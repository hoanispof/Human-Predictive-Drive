> **Trạng thái:** DRAFT — ví dụ minh họa cho Personal-Melody.md + Global-Melody.md

> **Mục đích:** Hình dung melody đời người qua 3 profiles PHỔ BIẾN NHẤT:
> ai đọc cũng có thể overlap, hình dung melody CỦA MÌNH

```

## 2. Người Center Có Nỗ Lực — "Melody Bình Thường Có Sprint Phase"

```

```

  │              │ ĐỀU          │ ĐỀU+ĐỈNH     │ ♩♯♯♯♯...    │
  ├──────────────┼──────────────┼──────────────┼──────────────┤
  │ Cortisol     │ THẤP-VỪA    │ CAO tạm      │ CAO bất ngờ  │
  │              │ (ổn định)    │ (tự chọn)    │ (bị buộc)    │
  ├──────────────┼──────────────┼──────────────┼──────────────┤
  │ Novelty      │ NHỎ đều      │ CAO tạm      │ NHỎ (bận     │
  │              │ (routine)    │ (sprint phase)│ survive)     │
  ├──────────────┼──────────────┼──────────────┼──────────────┤
  │ Connection   │ ỔN suốt     │ GIẢM tạm     │ CÓ THỂ TĂNG │
  │              │              │ (bận quá)    │ hoặc GIẢM    │
  ├──────────────┼──────────────┼──────────────┼──────────────┤
  │ Risk         │ THẤP         │ VỪA (tự chọn)│ CAO (bị buộc)│
  ├──────────────┼──────────────┼──────────────┼──────────────┤
  │ Kết quả      │ Ổn suốt     │ Đỉnh/thất bại│ Recover/kéo  │
  │              │              │ → settle     │ dài          │
  ├──────────────┼──────────────┼──────────────┼──────────────┤
  │ Yếu tố      │ Novelty nhẹ  │ Sleep +      │ Connection + │
  │ cần chú ý   │ (tránh chán) │ Connection    │ Support +    │
  │              │              │ (trong phase) │ Thời gian    │
  └──────────────┴──────────────┴──────────────┴──────────────┘

  ⭐ AI ĐỌC CÓ THỂ MIX:
    → 1 người CÓ THỂ trải qua CẢ 3: thuận lợi → nỗ lực → gặp khó
    → HOẶC: gặp khó → nỗ lực → thuận lợi (ngược lại)
    → HOẶC: thuận lợi suốt → bỗng gặp khó → nỗ lực recover
    → = 3 profiles = 3 "GIAI ĐOẠN" có thể xảy ra ở BẤT KỲ thời điểm
    → = KHÔNG phải "3 loại người" → "3 loại GIAI ĐOẠN" trong 1 cuộc đời


  ⭐ BÀI HỌC CHUNG:

    MỌI PROFILE:
      → Connection: yếu tố QUYẾT ĐỊNH melody BỀN hay KHÔNG
        (§1: connection ổn → melody ổn / §3: connection mất → melody khó recover)
      → Sleep: repair → body survive → melody TIẾP TỤC chơi
      → Body-listening: nghe body → biết mình ĐAng ở profile NÀO → adjust
      → = "Melody hay nhất = melody CHƠI ĐƯỢC LÂU NHẤT"
      → = KHÔNG phải peak CAO nhất → mà BỀN nhất

    SPECIFIC per-profile:
      ① Thuận lợi: cần TÌM novelty nhẹ → tránh "chán" → hobby, học thêm, explore
      ② Có nỗ lực: cần SLEEP + CONNECTION trong sprint phase → tránh Neural Wear
      ③ Gặp khó: cần SUPPORT + THỜI GIAN → connection = #1 priority khi khó
         "Ai ở bên khi khó = QUYẾT ĐỊNH melody recover hay không"
```
