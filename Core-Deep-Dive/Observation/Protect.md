# Protect — Observation Parameter

> My laptop breaks down — and I have enough money to buy a new one → "oh well."
> My laptop breaks down — and I can't afford a replacement → I resent the friend who broke it.
>
> Same laptop. Same friend. Same event — "it broke."
> What's different: CAN I REPLACE IT?
>
> "Protect" is not some primitive protective instinct.
> It is a pattern that emerges when the body detects: "something compiled into my baseline
> is at risk of becoming absent — and I'm not sure I can replace it."
>
> From "holding onto objects" → "holding onto people" → "holding onto ideas" → "holding onto identity" —
> same mechanism, different object, different intensity.
>
> BUT protect is not the only force.
> The body also has a giving dynamic — when direct reward ≈ 0 (already satiated),
> mirror reward (from sharing) > keep reward (from holding) → the body DRIVES giving.
> The gene that says "share when you have surplus" beats the gene that says "keep everything for yourself."
>
> This file: what Protect IS (ownership chunks + loss aversion),
> WHY loss hurts more than gain (asymmetric prediction-delta),
> the FORMULA for intensity, which hardware participates,
> the NATURAL COUNTERWEIGHT (giving dynamic),
> and WHEN protect is healthy vs when it becomes toxic.

### §1.1 — Ownership Chunks = Compiled Baseline

```
🟡 OWNERSHIP CHUNKS — BODY-LEVEL "MINE":

  DEFINITION:
    When the body repeatedly interacts with an object/person/idea →
    experience compiles into chunks at the body level (C+D zones).
    Pattern "X + me interacting" → Hebbian LTP → wired in.
    Result: the body has a BASELINE that includes X.
    → X is "mine" = X IS PART OF THE COMPILED BASELINE.

  PFC DOESN'T NEED TO DECLARE "MINE":
    → A 2-year-old guards a toy and won't share.
    → No concept of "ownership" at the PFC level yet.
    → Body has already compiled: "this toy + my hands = baseline."
    → Someone takes it = Chunk-Miss = dissonance = crying.
    → = Ownership chunk formation happens BEFORE language.

  COMPILE PATHWAY (Body-Feedback-Mechanism §5 Quality Baseline Shift):

    ① Interact once: proto-chunk weak
       → PFC aware "this is new" but body hasn't shifted baseline yet
       → Lost: PFC "slightly disappointed" > body dissonance (mild)

    ② Interact 3-10 times: chunk strengthening
       → Body baseline starts to include X
       → Lost: body dissonance CLEAR (Chunk-Miss appears)
       → Body-Feedback-Mechanism §5: SNC effect begins (Crespi rats downshift)

    ③ Interact 50-100+ times: fully compiled
       → X = HARD PART of baseline
       → Lost: body dissonance STRONG (SNC effect maximum)
       → Re-habituating (letting go) = slow, hard, painful
       → Body-Feedback-Mechanism §5 ④: re-habituate slow because strong compile = slow decay

    (Body-Feedback-Mechanism §5: compile_rate ≈ f(repetition × saliency × peak_valence
                               × multi_modal × contingency))

  EXAMPLES:
    → Car bought 1 day ago: gets scratched → "annoyed but ok"
    → Car driven 3 years: gets scratched → "who did this?!" (compiled deep)
    → Home of 20 years: forced to move out → real grief (body-level loss)

  ⭐ "MINE" = GRADIENT, NOT BINARY:
    → Just bought = ownership chunk weak
    → Used for a long time = ownership chunk strong
    → Used for a long time + emotional experiences = ownership chunk VERY STRONG
    → Same object, different compile history → different protect intensity
    → Explains: 2 people lose identical laptops but react DIFFERENTLY
      (person A: owned 1 month, used occasionally. Person B: owned 3 years, memories stored in it.)
```

### §1.2 — Loss Aversion = Asymmetric Prediction Delta

```
🟢 LOSS AVERSION — LOSING HURTS ~2× GAINING:

  🟢 Kahneman & Tversky 1979 (Prospect Theory):
    → Gaining $100: positive utility = X
    → Losing $100: negative utility ≈ 2X
    → = The body reacts to LOSS more strongly than GAIN of the same magnitude
    → = ASYMMETRIC prediction-delta

  THE MECHANISM IN FRAMEWORK TERMS:

    ① Compiled baseline = X (VTA habituated)
    ② Actual quality RISES to X+Δ:
       → VTA fires POSITIVE prediction-delta
       → Body signal: "better than expected" → reward
    ③ Actual quality FALLS to X-Δ:
       → VTA fires NEGATIVE prediction-delta
       → Body signal: "worse than expected" → dissonance
    ④ ⭐ ASYMMETRY: magnitude of ③ > magnitude of ② for the same Δ
       → = Loss aversion at the body level

    (Body-Feedback-Mechanism §5 PROPERTY ③: ASYMMETRIC —
     shift UP is easier than shift DOWN)
    (= Body resists quality decrease MORE STRONGLY than it accepts quality increase)

  WHY ASYMMETRIC? (evolutionary logic)
    → 🟡 Gaining something = nice but not survival-critical
    → Losing something already compiled = survival risk
    → Body has already invested resources compiling the pattern → losing it = waste
    → Body biases TOWARD KEEPING > TOWARD GAINING = conservative strategy
    → Conservative strategy survives BETTER in uncertain environments

  🟢 SNC EVIDENCE (Crespi 1942, Flaherty 1996):
    → Rats: high reward → switched to low → ate LESS THAN even the control group
    → = Not just "slightly disappointed" — reaction STRONGER than neutral
    → = Body-level mechanism, PFC NOT REQUIRED
    → = Loss aversion EXISTS in animals = hardware-level
    → (Body-Feedback-Mechanism §3.2: SNC = Chunk-Miss mechanism at animal level)

  🟢 Schultz 1997 (dopamine neurons):
    → Expected = X, Actual < X → dopamine SUPPRESSES below baseline
    → = Specific neural mechanism for negative prediction-delta
    → Dopamine suppression = ACTIVE signal "worse than expected"
    → Not "lack of reward" — it is an ACTIVE negative signal

  ⚠️ BUT: PREDICTION DELTA ≠ REWARD/PAIN
    (Clarification/Prediction-Error-Is-Not-Reward.md):
    → Prediction delta = VTA salience alert (doorbell: "something changed!")
    → REAL pain comes from 3 other sources: nociception / mismatch / recalibration
      (Body-Feedback.md: 3 Genuine Discomfort Sources)
    → Cortisol = sustainer/amplifier, NOT cause
      (Clarification/Cortisol-Amplifier-Not-Cause.md)
    → = Protect pain = BODY-BASE pain, not just "dopamine drop"
```

### §1.3 — Why "Mine" Exists at the Body Level

```
🟡 "MINE" IS NOT A PFC CONCEPT — IT IS A BODY STATE:

  EVIDENCE:

  ① Pre-verbal children (12-18 months) ALREADY protect their toys:
     → No language yet, no concept of "ownership" yet
     → Body has already compiled: "my hands + this toy = baseline"
     → Someone takes it = baseline violated = dissonance = crying/gripping tighter
     → = Ownership chunks compile BEFORE language

  ② Animals protect territory:
     → Dogs: guard a bone, growl when anyone approaches
     → Birds: guard the nest, attack intruders
     → No complex PFC → = body-level mechanism
     → 🟢 Prairie voles: vasopressin → territorial defense (Young & Wang 2004)

  ③ Phantom limb (🟢):
     → Lose an arm → body STILL fires the pattern "my arm is here"
     → = Compiled ownership chunks PERSIST after the object is gone
     → = Body does NOT "know" the arm is lost → continuous Chunk-Miss
     → Extreme case: body-level ownership of one's own body parts

  ④ Endowment Effect (🟢 Thaler 1980):
     → Give someone a mug → ask "what would you sell it for?" → average $7
     → Ask someone else "how much would you pay for it?" → average $3
     → SAME MUG. Different: who HAS been holding it.
     → JUST 30 SECONDS of holding → ownership chunk starts to form
     → = Body compiles fast when there is physical contact (multi-modal input)
     → = "Mine" begins at the body BEFORE PFC decides to keep/sell

  IMPLICATION:
    → "Letting go" through a rational PFC decision = TEMPORARY
    → Body-level ownership chunks STILL fire (compiled, not erasable)
    → Real letting go = re-habituate (Body-Feedback-Mechanism §5 ④:
      sustained absence → baseline shift DOWN) — slow, painful, but possible
    → = Explains why "knowing you should let go" while the body still holds on
    → PFC override is possible but COSTLY (fighting body signal continuously)
    → Duration of override = limited (PFC executive function = depletable)
```

### §1.3b — Compiled vs Fresh Protect (Inter-Body-Mechanism §3)

```
⭐ PROTECT HAS 2 PROCESSING MODES — CORRESPONDING TO THE COMPILED/FRESH AXIS:

  COMPILED PROTECT (automatic, cost ≈ 0):
    → Ownership chunks ALREADY compiled deeply → loss signal fires AUTOMATICALLY
    → Body reacts BEFORE PFC has a chance to evaluate
    → Example: someone touches your child → parent reacts IMMEDIATELY (compiled protection)
    → Example: hearing the boss is restructuring the team → stomach tightens IMMEDIATELY
      (compiled job ownership)
    → = Immediate protective reaction, NO THINKING REQUIRED
    → CAN overgeneralize: protect fires too strongly for things NOT actually threatened

  FRESH PROTECT (PFC evaluates, cost > 0):
    → Novel threat to valued entity → body alerts BUT PFC needs to evaluate
    → "Is this actually threatened? What happens if I lose it? Is it replaceable?"
    → Example: hearing the company might be acquired → PFC evaluates impact
    → Example: a friend says "I want to talk" → PFC evaluates: threat or not?
    → Cost: PFC executive function + uncertainty + possible suppression of body signal
    → = Slower but MORE ACCURATE than compiled protect

  FRESH → COMPILED TRANSITION (compilation path):
    → Repeated loss of the same type → pattern compiles → next time = automatic
    → Person betrayed 3 times → "the signs" compile → 4th time body reacts IMMEDIATELY
    → = Protect schemas LEARN + COMPILE over time
    → ⚠️ Trauma = forced compilation: 1 EXTREMELY POWERFUL event → compiles immediately
       → Post-trauma protect = overgeneralized compiled response

  🟡 Compiled/Fresh protect = framework synthesis (Inter-Body-Mechanism §3,
     consistent with Kahneman System 1/System 2).
```

### §1.4 — Cortisol: Amplifier, NOT Cause

```
⭐ CORTISOL IN PROTECT — FRAMEWORK CLARIFICATION:

  ⚠️ Mainstream + pop science: "loss → cortisol → pain → protect"
  ⚠️ Framework: "loss → 3 sources of REAL pain → cortisol AMPLIFIES + SUSTAINS"

  (Cortisol-Amplifier-Not-Cause.md — detailed clarification)

  CORTISOL IN THE PROTECT CYCLE:

    ① LOSS DETECTED → body-base pain fires (3 sources)
    ② Cortisol fires 20-30 minutes AFTER the NE+Adrenaline spike
       → = SUSTAINER, not initiator
    ③ Cortisol AMPLIFIES body sensitivity
       → ownership chunks fire MORE STRONGLY
       → loss signal PERSISTS longer
    ④ Cortisol INERTIA (Cortisol-Baseline.md §2.3):
       → Object recovered → cortisol DOESN'T drop immediately
       → Cortisol baseline takes weeks/months to re-calibrate
       → = Protect ANXIETY PERSISTS after the threat is resolved
       → Example: wallet found after being lost → still checking wallet
         obsessively for a week afterward

  DIRECTION GATE (Cortisol-Baseline.md §3, Chunk.md §2.4 Direction-At-Compile):
    → Same cortisol level, DIFFERENT direction:
    → Protect cortisol = threat-pull → chunks tagged AVOIDANCE
    → Novelty cortisol = domain-pull → chunks tagged APPROACH
    → → Protect-related chunks compile WITH avoidance tag
    → → When re-activated, body defaults to AVOID (not approach)
    → → = Explains: after a loss, body AVOIDS similar situations
         (avoidance learning = protect-compiled chunks firing)

  GYM ANALOGY (Cortisol-Baseline.md intro):
    → Cortisol = GYM for neurons
    → No training = atrophy (cortisol zero = Addison's disease)
    → Moderate training + adequate rest = stronger (acute stress + recovery = resilient)
    → Overtrain + no rest = injury (chronic cortisol + no repair = PFC damage)
    → = Protect acute (loss → recover → adapt) = HEALTHY
    → = Protect chronic (loss → no recovery → cortisol sustained) = PATHOLOGICAL
    → Key variable = REPAIR CAPABILITY (sleep), NOT cortisol level itself
```

### §2.1 — Perceived Replaceability (PFC assessment, often WRONG)

```
🟡 PERCEIVED REPLACEABILITY — PFC ASSESSMENT, NOT OBJECTIVE REALITY:

  IMPORTANT: "perceived" — NOT actual replaceability.

  PFC assesses: "if I lose X, can I get X back (or an equivalent)?"
  → This assessment CAN be wrong — and OFTEN IS:

  ① UNDERESTIMATING replaceability:
     → "No one understands me like this person" (reality: many people can)
     → "This job is one of a kind" (reality: many equivalent jobs exist)
     → = Compiled chunks + narrow experience → PFC lacks data
     → → Protect OVER-FIRES → controlling behavior, anxiety, jealousy

  ② OVERESTIMATING replaceability:
     → "There are plenty of friends" (reality: deep connection is rare)
     → "I can always earn more money" (reality: opportunity cost is real)
     → = Extensive surface-level experience → PFC overgeneralizes
     → → Protect UNDER-FIRES → let go too easily → later regret

  REPLACEABILITY FACTORS:
    → Resources (money, time, energy needed to replace)
    → Availability (is the replacement actually available?)
    → Uniqueness (does this thing have unique properties?)
    → Compilation cost (how long to re-compile?)
    → Emotional investment (is Imagine-Final tied to this object?)

  ⭐ COMPILATION COST = HIDDEN FACTOR:
    → Losing a laptop: buy new = money. BUT: re-setup, data, habits?
    → Losing a relationship: meeting someone new is possible. BUT: re-compile
      attachment chunks from scratch? 3-5 years? Longer?
    → Compilation cost is USUALLY IGNORED at the PFC level
    → Body KNOWS (because body already compiled → knows "how much it cost to compile")
    → → PFC says "replaceable" but body says "painful" = dissonance
    → = "I know it's replaceable but I don't want to lose it"
      = body is correct, PFC is wrong
      (body accounts for compilation cost, PFC doesn't)
```

### §2.2 — Attachment Chunks (compile depth)

```
🟡 ATTACHMENT CHUNKS — COMPILE DEPTH DETERMINES INTENSITY:

  Attachment chunks = how deeply ownership chunks have compiled.

  FACTORS AFFECTING COMPILE DEPTH
  (= Body-Feedback-Mechanism §5 compile_rate 5-parameter formula):

    ① Repetition: how many times interacted?
       → Once = proto-chunk → protect mild
       → 100 times = fully compiled → protect strong

    ② Saliency: was the experience noteworthy?
       → Laptop used routinely = moderate saliency
       → Laptop used to write a thesis under deadline stress = HIGH saliency
       → = Emotional peak → faster, deeper compile

    ③ Multi-modal: how many input channels?
       → Only seeing (an object sitting on a shelf) = 1 channel → weak
       → Holding + using + hearing + smelling = multi-modal → strong
       → = Physical contact ACCELERATES ownership chunk formation
       → (Thaler endowment: 30 seconds of holding is enough to shift baseline)

    ④ Peak valence: emotional peaks associated with the object?
       → Wedding ring = object + emotional peak + shared experience
       → Laptop = object + utility (moderate emotional)
       → → Wedding ring protect >> laptop protect (same monetary value)
       → (Valence-Propagation.md: valence TAG determines chunk weight)

    ⑤ Contingency: is the object LINKED DIRECTLY TO REWARD?
       → A tool that helps earn income = contingent (reward linked)
       → A decorative item = non-contingent (no direct reward link)
       → → Contingent objects compile FASTER (reinforcement)

  GRADIENT:
    Proto-chunk (weak) ────────────────── Fully compiled (strong)
    "I know it's mine"                    "IT'S PART OF ME"
    PFC-level only                        Body-level embedded
    Lost: "I'm sorry"                     Lost: genuine grief
```

### §4.1 — Vasopressin: Defense Side of Bonding

```
🟢 VASOPRESSIN = DEFENSE OF BONDED ATTACHMENT:

  🟢 Young & Wang (2004):
    → Oxytocin: "love" → bonding, trust
    → Vasopressin: "protect what you love" → guarding, territorial behavior
    → = 2 SIDES of the same bonding system
    → (Neurochemistry.md §8: "vasopressin = DEFENSE SIDE of oxytocin")

  🟢 PRAIRIE VOLE EVIDENCE (direct, animal):
    → Block vasopressin → STILL bonds but DOESN'T defend the partner
    → = Vasopressin ALONE doesn't create bonding
    → = Vasopressin DEFENDS an ALREADY EXISTING bond
    → = Defense mechanism for compiled attachment chunks
    → = Protect ≠ create bond. Protect = DEFEND existing bond.

  VARIATION (Neurochemistry.md §8):
    → Vasopressin HIGH: strong defense, jealousy, control
    → Vasopressin LOW: bonds but less defensive, lets go more easily
    → Oxytocin HIGH + Vasopressin HIGH = "loves deeply, holds tightly"
    → = NOT contradictory — 2 complementary systems both strong
    → Explains: "love + control" = oxytocin + vasopressin both elevated

  🟡 IN HUMANS — INDIRECT EVIDENCE:
    → 🟢 Vasopressin receptor gene (AVPR1A) variation →
      correlates with partner bonding patterns (Walum et al. 2008)
    → BUT: in humans, vasopressin → chunk processing → behavior
    → NOT directly "vasopressin → protect" as in voles
    → = Hardware bias (vasopressin) + compiled chunks = actual behavior
    → = Vasopressin SETS A TENDENCY, chunks DETERMINE expression

  FRAMEWORK POSITION:
    → Vasopressin = body-level BIAS TOWARD defending bonded objects
    → Not "the protect hormone" — it is the defense-side of the bonding system
    → Similar to serotonin for status: serotonin = certainty bias baseline,
      NOT the status decision-maker (Status.md §9)
    → Vasopressin = defense bias baseline, NOT the protect decision-maker
    → Chunks (compiled experience) = actual protect pattern
    → = Core-Software.md §9: "Protect parameter: hormone contribution
         vs chunk contribution ratio" (open research question)

### §4.2 — Cortisol Direction Gate: Protect = Threat-Pull

    → Novelty cortisol: curiosity body state → chunks tagged APPROACH
    → Threat/loss cortisol: anxiety body state → chunks tagged AVOIDANCE
    → Protect = almost always THREAT direction (loss = threat)

  IMPLICATIONS FOR PROTECT:

       → Body default: AVOID similar situations → future behavior avoids

       → Loss → cortisol spike → sustain → INERTIA

  PROTECT × CORTISOL COUPLING (Cortisol-Baseline §10.6):

### §7.1 — Mirror Reward Override (Agent-Mechanism.md §12.3)

```
