# Drive — Observation Parameter (Integration)

> A CEO decides to lay off 200 people (PFC ~85%, cortisol high, moral dissonance).
> Two hours later, he comes home and plays with his children (PFC ~5%, automatic,
> relaxed and happy).
>
> Same person. Same brain. DIFFERENT drives → COMPLETELY DIFFERENT mode.
>
> "Drive" is not a module that switches on and off — it is the NAME for
> energy + direction CURRENTLY EMERGING at the moment of observation.
>
> This file explains: why the same person behaves so differently,
> HOW PFC participates, and what happens when drives conflict.

```
  Core v7.5 (old): 3 drives (Novelty/Status/Protect) = L3 operators

  Drive IS NOT:
    ✗ "Decision → action" (wrong sequence)
      → Reality: body-need → drive → action → THEN "knowing" the decision was made
    ✗ "Consciousness controls behavior" (dualism)
      → Reality: unconscious = main engine (~95% of decisions)
    ✗ "Needs motivation to act" (simple motivation theory)
      → Reality: drives run CONTINUOUSLY; "lacking motivation" = drives IN CONFLICT
    ✗ "1 drive at 1 moment" (single-channel model)

  Drive IS:
    ○ Continuous — NEVER off (even during sleep: REM processing)
    ○ Multi-channel — many patterns run IN PARALLEL
    ○ Predictive — predicts + acts in advance (not purely reactive)

    ○ Emergent — action = OUTPUT, not INPUT

  HARDWIRED ARCHITECTURE (insects, simple animals):
    Hardwired circuits: food→eat, predator→flee, mate→approach.
    No "drive" needed — stimulus→response is FIXED.
    No PFC → no deliberation → no conflict.

    → Drive = EMERGENT OUTPUT from a general-purpose system:

  → Hardwired Architecture DOESN'T NEED the concept "drive" (stimulus→response suffices).

  🟡 Hardwired/Compilable Architecture = framework synthesis (Inter-Body-Mechanism.md §1.2).
```

## §1 — UNCONSCIOUS = ENGINE, PFC = OBSERVER-ORCHESTRATOR

```
⭐ WHY IS THE UNCONSCIOUS THE MAIN ENGINE?

  4 reasons the unconscious WINS against PFC in EVERY race:

  ① CAPACITY: ~95% of brain bandwidth
    → Unconscious: MILLIONS of signals/second
    → PFC: ~4±1 items simultaneously (Cowan 2001)
    → = PFC is a SMALL spotlight on a VAST stage

  ② SPEED: milliseconds vs seconds
    → Compiled chunks fire in ms
    → PFC: 200ms+ deliberation
    → = Unconscious HAS ALREADY ACTED before PFC "thinks"

  ③ EFFICIENCY: low energy cost
    → Compiled schemas = automatic = low energy
    → PFC deliberation = processing-intensive (~2% of mass, ~20% of energy)
    → = Brain SAVES ENERGY by maximizing unconscious use

  ④ EVOLUTION: 500 million years vs ~2 million years
    → Unconscious: present in ALL animals (500M years of refinement)
    → PFC: fully developed only in humans (~2M years)
    → = PFC BUILDS ON TOP OF the unconscious, does not REPLACE it

  → CONCLUSION:
    → Unconscious = DEFAULT mode → runs ALL THE TIME
    → PFC = EXCEPTION mode → runs WHEN NEEDED
    → Most of life = unconscious handles it → PFC rests
    → PFC is called when: new dissonance, significant novelty, new threat,
      drives conflict, Imagine-Final threatened


⭐ SIGNAL STRENGTH MODEL (replaces Layer Priority):

  v7.5 old: L0 > L1 > L2 > L3 (fixed priority)
  v7.8 new: STRONGEST signal wins (no fixed hierarchy)

  → L0 signals HAPPEN TO BE loudest (evolutionary design for survival)
  → BUT: not always. Context decides:
    → Normal: hunger (strong) overrides curiosity (weak) → seek food
    → BUT: Einstein skips meals to imagine (PFC overrides body-input)
    → BUT: Anne Frank sheltered (many reward chains > execution threat)
    → = Valence chain CAN override signal strength
      (Valence-Propagation.md §5③)

  → PFC CAN override (Drive-PFC-Override, §2) — but ALWAYS AT A COST
  → Sustained override → repair debt → body eventually wins

  ⭐ COMPILED/FRESH = THE REAL AXIS (Inter-Body-Mechanism.md §3):

    Unconscious vs PFC IS ACTUALLY a COMPILED vs FRESH spectrum:

    COMPILED DRIVE (Compiled — automatic, cost ≈ 0):
      → Chunks already compiled → body-feedback direct → action EMERGES
      → Unconscious auto → Drive-PFC-Absent/Monitor (§2)
      → Example: driving a familiar route (compiled route chunks fire → auto)
      → Example: a seasoned chef sees ingredients → "instantly knows" what to cook
      → = Most of life (~70-80%) = compiled drive

    FRESH DRIVE (Fresh — PFC drafts, cost > 0):
      → NEW situation / conflict → chunks don't match → PFC must draft
      → PFC deliberates → Drive-PFC-Spinning through Override (§2)
      → Example: first day at a new job → everything is new → PFC constantly drafting
      → Example: conflict between novelty vs threat → PFC arbitrates
      → = New + difficult situations = fresh drive territory

    TRANSITION Fresh→Compiled (Learning):
      → Fresh drive repeats + resolves → chunks compile → compiled drive
      → = "First day at work" (fresh) → "month six" (compiled)
      → Kahneman System 1 (compiled) / System 2 (fresh) equivalent
      → Details: Inter-Body-Mechanism.md §3.3

    → INSIGHT: Drive-PFC-Spinning vs Drive-PFC-Resolve = BOTH in fresh territory
      → DIFFER in chunk availability, NOT in activation level

  🟢 Unconscious processing dominance = neuroscience consensus
  🟢 PFC capacity limits = Cowan 2001 (4±1)
  🟢 PFC energy cost = Raichle 2006
  🟢 Evolutionary timeline = comparative neuroscience
  🟡 Compiled/Fresh drive distinction = framework synthesis (consistent with Kahneman)


⭐ AT ANY GIVEN MOMENT, MANY SIGNALS RUN SIMULTANEOUSLY:

  ┌────────────────────────────────────────────────────────┐
  │  Body-input signals:  which channels are unmet? how?   │
  │  Chunk patterns:      what matches? gaps? conflicts?    │
  │  Body-feedback:       reward or dissonance? intensity?  │
  │  Imagine-Final:       which action serves long-term?    │
  │                                                        │
  │  → Aggregate → STRONGEST SIGNAL WINS                  │
  │  → = energy + direction EMERGE → action               │
  └────────────────────────────────────────────────────────┘

  Most of the time: signals HARMONIZE → action is SMOOTH:
    → Body OK + chunks compiled-match context + no dissonance
    → = PFC NOT NEEDED → Drive-PFC-Absent/Monitor (§2)
    → = "Normal life"

  When signals CONFLICT or something is MISSING:
    → Body urgent + chunks don't match + dissonance high
    → = PFC is called → Drive-PFC-Spinning through Override (§2)
    → = "Difficult situations" — PFC participation required
```

## §2 — PFC PARTICIPATION SPECTRUM: 6 MODES

```
⭐ PFC = A CONTINUOUS SPECTRUM, NOT ON/OFF

  PFC participation = spectrum from 0% → 95%
  Framework discretizes into 6 MODES for communication.

  2 AXES:
    Axis 1: PFC ACTIVATION — "How BUSY is PFC?"
    Axis 2: PFC EFFECTIVENESS — "Is the busy-ness USEFUL?"

  ⚠️ CRITICAL INSIGHT:
    High Activation ≠ High Effectiveness
    Drive-PFC-Spinning: PFC 30-50% but NOT effective (lacks chunks)
    Drive-PFC-Resolve: PFC 20-40% but VERY effective (has chunks)
    → "Busier" ≠ "better" — CHUNKS determine PFC quality


2-AXIS DIAGRAM:

  PFC EFFECTIVENESS ▲
  (chunks quality)   │
                     │
    HIGH ────────────┤── [DRIVE-PFC-RESOLVE] ──── [DRIVE-PFC-STRATEGIC] ──
    (enough,         │    PFC 20-40%                PFC 60-80%
     relevant chunks)│    scan → match → act        hold → optimize
                     │                                       │
                     │                              [DRIVE-PFC-OVERRIDE]
                     │                                PFC 80-95%
                     │                                against body-base
                     │                                ⚠️ EXTREMELY HIGH cost
                     │
    LOW ─────────────┤── [DRIVE-PFC-SPINNING]
    (lacking,        │    PFC 30-50%
     wrong-level     │    try → fail → try → fail
     chunks)         │
                     │
    N/A ─────────────┤── [DRIVE-PFC-ABSENT] ─── [DRIVE-PFC-MONITOR]
    (PFC not         │    PFC 0-5%                PFC 5-15%
     needed)         │    fully automatic          background scan
                     │
                     └──────────────────────────────────────────►
                          0%     20%     40%     60%     80%    95%
                                    PFC ACTIVATION (% bandwidth)

  ⚠️ 6 modes = LABELED ZONES on a continuous spectrum, not 6 fixed boxes.


═══════════════════════════════════════════════════════
DRIVE-PFC-ABSENT (PFC ~0-5%)
═══════════════════════════════════════════════════════

  Fully automatic unconscious operation. PFC is "lights off."
  Conditions: safe context + chunks compiled + no dissonance

  Examples: playing with children, walking a familiar route, driving on autopilot
  = ~60-70% of waking time. "Normal," "natural."

  🟢 Automatic behavior dominance = neuroscience consensus


═══════════════════════════════════════════════════════
DRIVE-PFC-MONITOR (PFC ~5-15%)
═══════════════════════════════════════════════════════

  PFC observing, NOT intervening. "Security camera."
  Unconscious still running ~90%. PFC scans: "anything unusual?"
  If it detects dissonance → ESCALATES to Drive-PFC-Spinning+.

  Examples: driving a familiar route, a friend invites you out
    (PFC: "am I free? → yes → OK")

  🟢 Background monitoring = attention research (Posner 1980)


═══════════════════════════════════════════════════════
DRIVE-PFC-SPINNING (PFC ~30-50%, chunks INSUFFICIENT)
═══════════════════════════════════════════════════════

  ⭐ THE MOST UNCOMFORTABLE MODE — PFC active but NOT effective.
  "Knows there's a problem, doesn't know how to fix it."

  Mechanism:
    → PFC scans chunks → only FINDS solutions AT THE SAME LEVEL
    → Tries A → fails → A' → fails → A'' → fails
    → The CORRECT solution = DIFFERENT LEVEL — but chunks for that level
      DON'T EXIST

  Example: bored in front of the TV
    → PFC: change channel? scroll social media? eat a snack? → ALL 3 = same
      level (passive)
    → Correct solution: go outside, exercise, meet a friend
      (DIFFERENT LEVEL: active)
    → But the chunk for "going for a walk" may: not exist / be weak /
      be suppressed

  Why this is the most uncomfortable mode:
    → PFC active = HIGH processing load (serial bottleneck + catecholamine)
    → But: no resolution = zero reward
    → = HIGH cost + NO payoff → WORST experience

  ⚠️ PFC% is HIGHER than Drive-PFC-Resolve — because it's forcing a search,
     not because it's more skilled.

  🟡 Spinning mechanism = framework analysis
  🟢 PFC energy cost without reward = frustration research


═══════════════════════════════════════════════════════
DRIVE-PFC-RESOLVE (PFC ~20-40%, chunks SUFFICIENT)
═══════════════════════════════════════════════════════

  PFC active AND effective. "Knows the problem AND knows how to fix it."
  Scans → matches → acts → resolves → PFC disengages.

  Example: a doctor encounters a familiar symptom pattern → chunks match
    → diagnosis → done
  = "Expertise" = Drive-PFC-Resolve in a specific domain

  ⚠️ PFC% LOWER than Drive-PFC-Spinning because chunks are sufficient
     → scan is fast → done.
  → = Chunks determine spinning vs resolve, NOT "trying harder"

  🟢 Expertise = pattern recognition (Chase & Simon 1973)


═══════════════════════════════════════════════════════
DRIVE-PFC-STRATEGIC (PFC ~60-80%, chunks RICH + META)
═══════════════════════════════════════════════════════

  PFC HAS a solution → but HOLDS.
  Meta-observation: "Is the immediate solution OPTIMAL long-term?"

  = Drive-PFC-Resolve + META-COGNITIVE layer:
    → PFC evaluates PFC: "Am I thinking about this correctly?"
    → Metacognition (Flavell 1979)

  6 ways to add data while holding:
    → Reading/research, consulting experienced people, sleeping
      (REM consolidation), releasing → DMN (walking, showering → unconscious
      processes), observing further, using Simulation-Engine to run scenarios

  Example: a CEO holds before an investment decision → "need more data"

  🟢 Metacognition = Flavell 1979
  🟡 Drive-PFC-Strategic as distinct from Drive-PFC-Resolve = framework distinction


═══════════════════════════════════════════════════════
DRIVE-PFC-OVERRIDE (PFC ~80-95%)
═══════════════════════════════════════════════════════

  PFC WORKING AGAINST the body-feedback signal. Most expensive, rarest.

  → Skipping meals to hit a deadline
  → Tough love (overriding empathy for long-term good)
  → Staying in a dangerous situation because of duty

  ⚠️ ALWAYS HAS A COST (body protests):
    → Cost ∝ intensity of body drive × duration of override
    → Light, brief override → small cost, fast recovery
    → Heavy, sustained override → burnout, trauma
    → = "Daily override" = the road to burnout

  🟢 Executive function override = Miller & Cohen 2001


SUMMARY:

  ┌─────────────────────┬──────────┬──────────────┬─────────────────────────────┐
  │ Drive Mode          │ PFC %    │ Effectiveness│ When                        │
  ├─────────────────────┼──────────┼──────────────┼─────────────────────────────┤
  │ Drive-PFC-Absent    │  0-5%    │ N/A          │ Auto, safe, compiled        │
  │ Drive-PFC-Monitor   │  5-15%   │ N/A          │ Scanning, ready to activate │
  │ Drive-PFC-Spinning  │ 30-50%   │ LOW          │ Dissonance + no chunks      │
  │ Drive-PFC-Resolve   │ 20-40%   │ HIGH         │ Dissonance + sufficient     │
  │                     │          │              │ chunks                      │
  │ Drive-PFC-Strategic │ 60-80%   │ HIGH + META  │ Complex, needs optimizing   │
  │ Drive-PFC-Override  │ 80-95%   │ HIGH but COST│ Against body, last resort   │
  └─────────────────────┴──────────┴──────────────┴─────────────────────────────┘

  → MOST of life: Drive-PFC-Absent/Monitor (~70-80% of waking time)
  → Drive-PFC-Spinning vs Resolve = SAME situation, DIFFERENT chunks →
    DIFFERENT mode → DIFFERENT output
  → = Chunks = THE DECIDING VARIABLE
```

## §3 — DRIVE CONFLICT: 4 TYPES + RESOLUTION

```
⭐ WHY DOES CONFLICT MATTER?

  People are "complicated" because drives FREQUENTLY CONFLICT.
  Many signals SIMULTANEOUSLY + pointing different directions = conflict →
    "hard-to-understand" behavior
  Understanding conflict types → explains MOST "irrational" behavior


═══════════════════════════════════════════════════════
TYPE 1: SIGNAL STRENGTH (survival override)
═══════════════════════════════════════════════════════

  When a body-input signal is EXTREMELY STRONG → overrides EVERYTHING else.
  → Unconscious AUTO, PFC does not participate.

  Specific mechanism: NE flood → α1 receptors → PFC DISCONNECTS
    → PFC is literally cut off → CANNOT negotiate
    → Not "PFC chooses to yield" — body SHUTS DOWN PFC (Arnsten 2009)

  Examples:
    → Oxygen deprivation → breathe, regardless of what you're doing
    → Severe hunger → stop everything → find food
    → Tiger encounter → forget status → RUN

  🟢 α1 receptor → PFC offline = Arnsten 2009, LeDoux 1996


═══════════════════════════════════════════════════════
TYPE 2: NOVELTY vs THREAT (pull vs push)
═══════════════════════════════════════════════════════

  When Novelty (PULL) vs Threat (PUSH) point at the SAME target.

  → Novelty > Threat → EXPLORE (approach wins)
  → Threat > Novelty → STAY (avoidance wins)
  → Equal → FROZEN → PFC must arbitrate

  Example: wanting to change careers
    → Novelty: "new field, exciting, growth"
    → Threat: "lose income, family disapproves, risk of failure"
    → Resolution depends on chunks:
      → HAS chunks (network, savings, skills): Drive-PFC-Resolve → makes the change
      → LACKS chunks: Drive-PFC-Spinning → "wants to but can't commit"
      → META chunks: Drive-PFC-Strategic → "prepares 6 months in advance"

  🟢 Approach-avoidance conflict = Lewin 1935, Miller 1944


═══════════════════════════════════════════════════════
TYPE 3: BODY-FEEDBACK vs PFC (override territory)
═══════════════════════════════════════════════════════

  Body wants X NOW, PFC says NO.
  = PFC uses a DISTANT Imagine-Final to override a NEAR body-feedback.
  = Drive-PFC-Override — always carries a cost.

  Examples:
    → Sleep (body) vs Deadline (PFC) → stay up → cost: repair debt
    → Empathy (body) vs Strategy (PFC) → withhold help → cost: guilt
    → Fear (body) vs Duty (PFC) → stay → cost: trauma risk

  → Cost ∝ intensity × duration. Sustained = burnout.

  🟢 PFC override = executive function (Miller & Cohen 2001)


═══════════════════════════════════════════════════════
TYPE 4: INTRA-OBSERVATION (same level, different direction)
═══════════════════════════════════════════════════════

  When 2 observation patterns have equal signal strength but DIFFERENT direction.
  → Most COMPLEX — no automatic priority → PFC uses Imagine-Final as tiebreaker.

  Example: Speaking directly vs. keeping the peace
    → Integrity: "should speak directly"
    → Status: "speaking directly → offend someone"
    → PFC arbitrates: Imagine-Final "respected leader" → chooses directness
    → OR: Imagine-Final "harmonious relationship" → chooses diplomacy

  Example: Family obligations vs personal career goals
    → Schema empathy: "care for family, want to make them happy"
    → Schema growth: "want to develop, want challenge"
    → EQUALLY STRONG, BOTH "right" → PFC arbitrates based on Imagine-Final
    → NO single correct answer — framework EXPLAINS, does not prescribe

  🟡 Intra-observation conflict = framework classification
  🟢 Multiple social motives = Fiske 2004


═══════════════════════════════════════════════════════
RESOLUTION RULES — 5 rules in order
═══════════════════════════════════════════════════════

  1. EMERGENCY → strongest signal AUTO WINS (no PFC needed)

  2. NON-EMERGENCY → PFC arbitrates based on IMAGINE-FINAL
     → Imagine-Final clear → fast resolution (Drive-PFC-Resolve)
     → Imagine-Final unclear → spinning (Drive-PFC-Spinning)
       or deliberate (Drive-PFC-Strategic)

  3. SAME LEVEL → intensity + Imagine-Final relevance decide
     → Equal → FROZEN → need more information/time

  4. PFC OVERRIDE → possible but COSTLY
     → Use sparingly — "daily override" = burnout path

  5. NO IMMEDIATE RESOLUTION → 3 options:
     a) FROZEN → need more chunks, data, time
     b) STRATEGIC ACCEPT → accept dissonance temporarily (saves PFC)
     c) RELEASE → DMN → let the unconscious process → possible eureka

  ⚠️ CONFLICT CAN BE GOOD:
    → Conflict FORCES PFC participation → evaluates → grows
    → Resolve → NEW chunks compile → mode upgrades
    → = "Difficulty" = opportunity (if resolved)
    → = "Difficulty" = damage (if NOT resolved + sustained)
```

## §4 — PULL vs PUSH: WHY IT MATTERS

```
⭐ CORE: SAME BEHAVIOR → DIFFERENT DRIVE → DIFFERENT SUSTAINABILITY

  PULL drives (novelty, connection, experience):
    → "Want to do this" → enjoy the process → sustainable
    → Body-feedback: reward per step → chunks compile positively
    → Satisfaction: CAN fire → natural endpoint exists

  PUSH drive (threat):
    → "Have to do this" → endure the process → costs cortisol → not sustainable
    → Body-feedback: relief when threat temporarily lifts (≠ reward)
    → Satisfaction: HARD to fire → no natural endpoint
    → = "Avoiding threat ≠ body-need fulfilled"

  COMPARISON:

    ┌────────────────┬────────────────────┬─────────────────────┐
    │                │ PULL drives        │ PUSH drive          │
    ├────────────────┼────────────────────┼─────────────────────┤
    │ Direction      │ TOWARD reward      │ AWAY FROM harm      │
    │ Feeling        │ "Want"             │ "Have to"           │
    │ Process feel   │ Enjoy              │ Stress              │
    │ Endpoint       │ Clear (enough)     │ UNCLEAR             │
    │ Satisfaction   │ Fires ✅           │ Hard to fire ❌     │
    │ Sustainability │ High               │ LOW                 │
    │ Schema compile │ Positive           │ Negative            │
    │ Body cost      │ Low                │ HIGH                │
    │ When healthy   │ Default            │ Emergency           │
    │ When dominant  │ Flow territory     │ BURNOUT             │
    └────────────────┴────────────────────┴─────────────────────┘


DRIVE × NEW CONCEPTS (Concept Cascade):

  SIMULATION-ENGINE (Simulation-Engine.md v1.0):
    → Drive PREDICTION uses Simulation-Engine: PFC simulates outcome →
      body evaluates
    → "Want to do" = Simulation-Engine drafts → body REWARDS → approach
    → "Have to do" = Simulation-Engine drafts → body signals THREAT →
      avoidance or endurance

  ENTITY-ACCESS GRADIENT (Entity-Access.md v1.2):
    → Drives DIRECTED AT entities along the Level 0-5 gradient
    → Drive toward a Level 5 entity (child, self) = STRONGER than
      Level 1 (acquaintance)
```
