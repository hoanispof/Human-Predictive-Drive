### §2.1 — ① Real Body-Base Reward

```
⭐ PREREQUISITE 1: GIFT MUST TRULY FEED BODY-BASE

  (03-Reward.md §3 — 5 Body-Feedback-Preconditions)

  Gift → body-base reward only fires when Body-Feedback-Precondition IS MET:
    Precondition-1 Directed-Gap: body IS CURRENTLY NEEDING what is given
    Precondition-2 Chunk-Substrate: has enough chunks to decode the VALUE of the gift
    Precondition-3 Delta-Gate: gift creates LARGE ENOUGH change
    Precondition-4 Match-Range: Goldilocks zone (dynamic — not too familiar, not too foreign)
    Precondition-5 Compile-Gate: gift is not tagged as threat

  WHEN Body-Feedback-Precondition IS MET:
    → Body-base opioid release → REAL reward
    → PFC observes: "pleasant" → downstream components CAN fire

  WHEN Body-Feedback-Precondition FAILS:
    → Body-base "meh" → no opioid → PFC may KNOW the gift has value
      but body does NOT fire → gratitude does NOT emerge at body level
    → = "PFC says money is useful, but body-base says nothing"

  EXAMPLES:
    Walking for exercise (body comfortable):
      → Precondition-1 FAILS: schema pending ≈ 0 for money
      → Someone gives money → PFC knows it's useful → body "meh"
      → + Precondition-5 suspicion: "giving money out of nowhere → threat?" → mild cortisol
      → → Gratitude does NOT fire even though PFC knows money is useful

    Lacking bus money (body-base currently needs it):
      → Precondition-1 MET: schema pending FEEDS exactly what is lacking
      → Stranger gives money → real body-base reward → opioid
      → → Prerequisite ① MET → downstream components HAVE A CHANCE to fire

  ⭐ GAP DIRECTION LENS (Gap-Direction.md):
    Precondition-1 Directed-Gap = gap direction must EXIST pointing toward the gift type
    "Unknown = no gap": child who doesn't know the value of an antique car
      → no gap about antique cars → father gives antique car → body does NOT reward
      → gratitude IMPOSSIBLE even though gift is expensive
    = Gift must match GAP DIRECTION, not just "have objective value"

  🟢 Alliesthesia (Cabanac 1971): same stimulus, different body-state → different hedonic value
  🟡 Body-Feedback-Precondition as gratitude prerequisite = framework extension of established preconditions
```

## §3 — MECHANISM: ANTI-HABITUATION

### §3.1 — Root Problem: VTA Habituates → Hedonic Treadmill

```
⭐ VTA HABITUATES EVERYTHING — THIS IS THE ROOT PROBLEM:

  (Body-Feedback-Mechanism.md §5 — Quality Baseline Shift)
  (03-Reward.md §2.8 — Each loop raises baseline)

  MECHANISM:
    Gift first time → VTA fires delta → opioid → reward → "pleasant"
    Gift 2-10 times → VTA gradually habituates → reward DECREASES
    Gift 50-100+ times → VTA fully habituated → gift = baseline = INVISIBLE
    Lose gift → actual < compiled baseline → Successive Negative Contrast (SNC) → "anger" instead of "gratitude"

  = HEDONIC TREADMILL (🟢 Brickman & Campbell 1971, Brickman 1978):
    → Every improvement gradually becomes "new normal"
    → Body shifts baseline UPWARD → needs NEW improvement to get reward
    → = Endless loop: improve → habituate → invisible → need more

  WHY THIS IS "THE PROBLEM" FOR GRATITUDE:
    → Continuous gift → habituates → invisible → ZERO reward → ZERO gratitude
    → Parents cook 10,000 meals → habituated → "normal" → no gratitude
    → Oxygen all day → COMPLETELY habituated → no delta → no gratitude → ever
    → = Baseline shift = the INVISIBLE ENEMY of gratitude

  BUT — evolution does NOT let baseline shift destroy connection:
    → If everything habituates → lose gratitude → lose reciprocity
    → → Connection network decays → survival DECREASES
    → → Evolution SELECTS for ANTI-HABITUATION MECHANISMS
    → → = 3 mechanisms §3.2-§3.4

  🟢 Hedonic adaptation: Brickman 1978, Frederick & Loewenstein 1999
  🟢 VTA habituation: Schultz 1997 (habituated reward → no dopamine signal)
  🟢 SNC: Crespi 1942, Flaherty 1996
```
