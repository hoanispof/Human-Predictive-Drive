### 1.4 — Shareable vs Non-shareable Body-Knowing

```
⭐ "LOGIC vs INTUITION" = SHAREABILITY OF BODY-KNOWING

  MATHEMATICS (deterministic domain):
    Body-Knowing chunks: 2+2=4 (SHARED — same for everyone)
    Reproduce: expert A and B give same result (verifiable)
    Global label: "LOGIC" (because shared + reproducible)

  PSYCHOLOGY (probabilistic domain):
    Body-Knowing chunks: patterns from 1,000+ cases (UNIQUE to each expert)
    Reproduce: expert A and B MAY reach different conclusions (different cases)
    Global label: "INTUITION / FEELING" (because not perfectly reproducible)

  INSIDE: SAME PROCESS — compiled chunks fire → body-direct → "knowing"
    Mathematician: years compiling → chunks fire fast → Body-Knowing: "sees" the solution
    Therapist: years compiling → chunks fire fast → Body-Knowing: "sees" the pattern
    Experienced meditator: years compiling → chunks fire fast → Body-Knowing: "sees" body-state
    ALL THREE = Body-Knowing. ALL THREE = CONSISTENT over time.
    ALL THREE = follow their OWN "logic" (compiled pattern network)
    DIFFERENCE: WHICH chunks were compiled (domain × individual experience)

  THE DIFFERENCE:
    Math: subject = DETERMINISTIC → Body-Knowing CONVERGES across people
    Psychology: subject = PROBABILISTIC + INDIVIDUAL → Body-Knowing DIVERGES
    Body-state: subject = PRIVATE → Body-Knowing CANNOT be shared at all
    → NOT because psychologists are "guessing"
    → NOT because meditators are "arbitrary"
    → Domain not deterministic → own Body-Knowing = LEGITIMATE
    → Domain reality = patient outcomes / body-state clarity = final arbiter

  FRAMEWORK REFRAME:
    "Logic"     = Body-Knowing SHAREABLE (deterministic domain)
    "Intuition" = Body-Knowing NOT EASILY SHAREABLE (probabilistic domain)
    "Fresh"     = PFC currently drafting (not yet Body-Knowing — CAN articulate steps)
    DIFFERENCE: shareability. NOT DIFFERENT: quality of processing.

  ADDING A DIMENSION: ARC STABILITY (§3.2)
    Shareable + stable-arc Body-Knowing     = "logic" (math, shelf-life ≈ ∞)
    Non-shareable + stable-arc Body-Knowing = "intuition" (veteran therapist)
    Non-shareable + shifting-arc Body-Knowing = "gut instinct" (trader, shelf-life LIMITED)
    → Therapist: human psychology arc ≈ stable → Body-Knowing reliable long-term
    → Trader: market arc shifts → Body-Knowing reliable ONLY in current regime
    → Same non-shareable Body-Knowing but DIFFERENT shelf-life
    → "Gut instinct" from shifting-arc = deserves respect + NEEDS continuous domain verify

  WHY SHAREABLE BODY-KNOWING IS MORE TRUSTED:
    → Shareable = verifiable by others → social trust HIGH
    → Non-shareable = "only I know" → social trust LOW
    → But INSIDE THE BODY: both equally Body-Knowing, equally "knowing"
    → Expert "intuition" = expert "logic" — differs only in domain shareability

🟢 Expert intuition = compiled patterns (Kahneman 2011, Klein 1998).
🟡 "Shareable vs non-shareable" as organizing principle = framework synthesis.
```

## §2 — LOGIC/FEELING = OBSERVER LABELS FOR BODY-KNOWING

> **§1 establishes Body-Knowing and Compiled/Fresh spectrum.**

### 2.1 — Why Labels Emerge

```
⭐ OBSERVER PERSPECTIVE CREATES LABELS:

  Observer looks from OUTSIDE at processing:

  CASE A — Observer labels "Logic":
    Person articulates steps: "Because A, therefore B, so C."
    → OBSERVABLE: verbal chain, sequential, reproducible
    → Observer CAN follow → CAN verify → trust HIGH
    → Label: "LOGIC"
    → Inside: Body-Knowing (if compiled + shareable) OR Fresh (if novel PFC draft)

  CASE B — Observer labels "Feeling/Intuition":
    Person says: "I feel... can't explain it."
    → OBSERVABLE: no verbal chain, immediate, not reproducible by observer
    → Observer CANNOT follow → CANNOT verify → trust LOW
    → Label: "FEELING" or "INTUITION"
    → Inside: Body-Knowing (compiled body-direct — in expert domain)

  CASE C — Observer labels "Gut / Arbitrary":
    Person says: "I just want it that way" without domain expertise
    → OBSERVABLE: no chain + no track record + no domain verify
    → Observer: "person is guessing"
    → Inside: MAY be Self-Referencing Body-Knowing (wrong but feels smooth) OR truly random fresh

  → LABELS = f(observer's ability to UNDERSTAND + VERIFY)
  → SHAREABLE Body-Knowing → "logic"
  → NON-SHAREABLE Body-Knowing → "feeling/intuition"
  → UNVERIFIED → "gut instinct"
  → INSIDE: mechanism may be IDENTICAL across all 3 labels

  ⚠️ PRECISION — "LOGIC" LABEL COVERS 2 DIFFERENT PROCESSING PHASES:

    "Logic" is assigned by observers to BOTH:
      ① Compiled shareable Body-Knowing (Einstein solving a familiar problem → automatic)
      ② Fresh PFC processing (student solving a problem for the first time → effortful)

    Framework's precise claim:
      → ① (compiled shareable) and "intuition" (compiled non-shareable) = SAME mechanism
        (Body-Knowing, differ only in domain shareability)
      → ② (Fresh PFC draft) and ① (compiled Body-Knowing) = DIFFERENT processing phases
        (BUT: ② is TEMPORARY — through learning, Fresh → compiles → becomes ①)
      → = Observer label "logic" DOES NOT distinguish ① vs ②
      → = This is exactly WHY labels mislead about mechanism

    SUMMARY:
      "Logic" vs "Intuition" at compiled level = SAME mechanism, differ in shareability
      "Logic" at Fresh level vs "Intuition" at compiled level = DIFFERENT phase, same trajectory
      = Labels track OBSERVER PERSPECTIVE, not processing phase or mechanism

🟡 Labels as observer-perspective artifact = framework synthesis.
   Consistent with expertise research showing expert "intuition" = compiled patterns.
```

### 2.2 — Labels Still Useful — Just Not Mechanism

```
⭐ LABELS ≠ MEANINGLESS — labels signal SOMETHING REAL:

  "LOGIC" label useful when:
    → Need to COMMUNICATE reasoning (others can follow your steps)
    → Need to VERIFY (reproduce + check = trust building)
    → Need to TEACH (transfer Body-Knowing to others via articulated steps)
    → Need to ACCUMULATE (build rule on rule → collective knowledge)
    → Need to DEBATE (explicit claims → explicit counter-claims)
    = When SHAREABILITY matters for the task

  "FEELING/INTUITION" label useful when:
    → Need to TRUST BODY-KNOWING (compiled patterns too complex to articulate)
    → Need SPEED (waiting for fresh = too slow for situation)
    → Need CROSS-DOMAIN (Body-Knowing detects patterns fresh can't encode yet)
    → Need to DETECT ERROR (body signal "something's off" = Body-Knowing truth PFC missed)
    → Need QUALITY CHECK (body vote on PFC output — Dual Check)
    = When BODY-DIRECT ACCESS matters for the task

  "FRESH/LOGIC" specifically needed when:
    → Domain is NEW (no Body-Knowing yet — no compiled chunks)
    → Body-Knowing is WRONG (need override — unlearning)
    → Stakes are HIGH (need deliberate verify, not automatic)
    → Need to FORMALIZE (Body-Knowing → words → shareable → teachable)

  → Labels = PRAGMATIC TOOLS for communication about processing
  → Not ontological claims about "2 types of thinking"
  → Same person uses both labels at different moments for different purposes
```

### 2.3 — PFC = Lawyer Applies to BOTH Labels

```
⭐ NEITHER "LOGIC" NOR BODY-KNOWING = NEUTRAL TRUTH (Inter-Body §7):

  PFC = LAWYER for body-base (not a neutral judge):
    → Body-need fires FIRST
    → Body creates DRIVE toward action
    → PFC creates NARRATIVE ("reasons") for action the body ALREADY wanted
    → Person BELIEVES the narrative
    → 🟢 Gazzaniga split-brain: left hemisphere CONFABULATES
    → 🟢 Haidt 2001: moral judgment = intuition first, reasoning post-hoc
    → 🟢 Nisbett & Wilson 1977: people cannot report actual causes

  "LOGIC" output can be LAWYERING:
    → "I quit because of career growth" (actual: escape toxic boss)
    → "I bought the car because it's practical" (actual: status body-need)
    → PFC chain VALID but NOT SOUND (premise = hidden body-need)
    → = "Logic correct" ≠ "Logic honest"

  ⚠️ PFC = LAWYER ≠ PFC = USELESS:
    → PFC = Lawyer describes the DOMINANT dynamic (majority of everyday decisions)
    → PFC DOES have genuine processing capacity, especially when:
      a) Domain genuinely novel (no compiled base → no body-need to lawyer FOR)
      b) Formalization (converting Body-Knowing → shareable → teachable)
      c) Deliberate override when Body-Knowing detected as wrong (unlearning)
    → BUT even in genuine processing:
      → WHICH directions PFC explores is still influenced by body-needs
        (PFC explores directions body "finds interesting" → body steers attention)
      → PFC output still needs DOMAIN VERIFY (not self-sufficient)
    → = PFC = Lawyer + sometimes Genuine Explorer
    → = Both modes need domain arbiter (→ §3.3)
    → Inter-Body-Mechanism.md v2.0 §7.2: PFC Accuracy = spectrum 0→1

  BODY-KNOWING output can ALSO be distorted:
    → Compiled patterns CAN BE WRONG (trauma, Self-Referencing, wrong context)
    → Body-Knowing CAN BE based on PAST not CURRENT reality
    → "Body-Knowing feels right" ≠ "Actually right" (smooth ≠ accurate)

  → NEITHER track = pure truth source
  → DOMAIN REALITY = final arbiter for BOTH
  → Dual Check (Ask-AI v3.1 §6.1, Body-Knowing.md §7.6):
    Step 1: Body-feedback = quality controller ("is something off?")
    Step 2: Domain reality = final arbiter (outcome confirms/rejects)
  → = Both labels point to PARTIAL access. Domain = complete.

  ⭐ FEEL-EXPLANATION ≠ SELF-OBSERVATION (Self-Observation.md §8.2):
    Feel-Explanation = PFC creates narrative ("I am sad BECAUSE X") = CONSTRUCT
    Self-Observation = PFC reads body-state directly ("I am sad") = OBSERVE
    The more one explains, the FURTHER from genuine Self-Observation
    → Rumination = PFC Lawyer explain loop → REDUCES Self-Observation quality
    → Meditation = observe WITHOUT labeling → GOOD for Self-Observation
    → Details: Self-Observation.md v1.0 §8.2

🟢 Gazzaniga, Haidt 2001, Nisbett & Wilson 1977
🟡 "PFC = Lawyer for BOTH tracks" = framework synthesis
```

### 2.4 — Kahneman Reframe through Body-Knowing Lens

```
🟡 KAHNEMAN REFRAME:

  KAHNEMAN (2011):
    System 1: fast, automatic, intuitive, error-prone
    System 2: slow, deliberate, rational, effortful

  FRAMEWORK v4.0:
    System 1 ≈ BODY-KNOWING (automatic because compiled, not because "fast")
    System 2 ≈ FRESH processing (slow because novel, not because "deliberate is slow")

  FRAMEWORK ADDS:
    ① Speed = CONSEQUENCE of compilation, not defining feature
       → Body-Knowing CAN be slow (complex compiled chain)
       → Fresh CAN be fast (simple novel choice)
       → Speed misleads — compilation level is the real variable

    ② "System 1 biases" = BODY-KNOWING COMPILED FOR WRONG DOMAIN
       → Cognitive biases = compiled heuristics from ANCESTRAL environment
       → WORKED in ancestral context (domain verified by survival)
       → FAIL in modern context (domain CHANGED, compiled patterns OUTDATED)
       → = NOT "System 1 unreliable" — "Body-Knowing compiled for DIFFERENT domain"
       → Fix: re-compile for CURRENT domain (not "suppress System 1")

    ③ Kahneman + Klein (2009) synthesis SUPPORTS framework:
       → Intuition reliable WHEN: environment has patterns + learner has feedback
       → = Body-Knowing verified by domain → reliable
       → = Body-Knowing NOT verified → unreliable (Self-Referencing Trap)
       → = DOMAIN FEEDBACK as condition = our "Domain = Arbiter"

    ④ Framework rejects "System 2 corrects System 1" hierarchy:
       → BOTH can be wrong. BOTH can be right.
       → Fresh (System 2) = Lawyer drafting narrative (may confabulate)
       → Body-Knowing (System 1) = body truth (may be outdated)
       → DOMAIN = arbiter for both. Neither system "above" the other.

🟢 Kahneman 2011, Evans & Stanovich 2013
🟢 Kahneman + Klein 2009: conditions for reliable intuition
🟡 Reframe as compilation-level (not speed) = framework synthesis

### 3.1 — Basic Loop: Body-Knowing Anchor ↔ Fresh Explore

```
⭐ BASIC LOOP:

  ┌──────────────────────────────────────────────────────────────┐
  │                                                              │
  │  BODY-KNOWING ANCHOR       ←→       FRESH EXPLORE            │
  │  (compiled patterns)               (PFC drafts novel paths)  │
  │                                                              │
  │  "I KNOW: A→B→C"          ←→   "But body detects D..."     │
  │                                                              │
  │  FLOW:                                                       │
  │                                                              │
  │  ① Body-Knowing anchor: ALREADY VERIFIED chunks stay stable  │
  │     → Body-direct, cost ≈ 0, communicable (if shareable)    │
  │                                                              │
  │  ② Fresh explore: "What lies BEYOND current Body-Knowing?"   │
  │     → Body signals "something off" + PFC drafts novel link  │
  │     → Felt sense: "new pattern... not sure yet..."           │
  │     → Cost > 0 (PFC draft + uncertainty + maybe suppress)   │
  │                                                              │
  │  ③ Domain verify: check new pattern AGAINST DOMAIN REALITY   │
  │     → Correct → ACCEPT → compile → Body-Knowing EXPANDS     │
  │     → Wrong → REJECT → return to old anchor                 │
  │     → ⚠️ NOT "logic verify" — DOMAIN OUTCOME verify (§3.3)  │

    → Body-Knowing: some vague unease, quickly rationalized away
    → Conclusion: "Everything's fine."

  SOMATIC partner (Body-Knowing-biased):

    Somatic: "Are you okay?" Verbal: "Fine, nothing happened."

    Body WAS firing (tension, cortisol). Somatic partner's Body-Knowing mirrored it.

  WISE RESOLUTION:
    Verbal: "You're right, body is tense. Can't explain yet. Let me sit with it."
    Somatic: "I'll give space. My Body-Knowing didn't come from nothing."
    → Both modalities valid. Both needed. Neither sufficient alone.

🟡 Modality bias as stable individual difference = framework hypothesis.
