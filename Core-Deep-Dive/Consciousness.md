# Consciousness — "Knowing" as Functional Definition

> **You KNOW you are reading this sentence.**
> **You DON'T KNOW which grammar rules are processing it.**
> **Both fire. The difference: which one you KNOW.**
>
> **Consciousness = "KNOWING" — the portion of processing that is KNOWN.**
> **"Knowing" = f(signal strength × anchor quality × attention).**
>
> **The entire body + brain = unconscious (default).**
> **Consciousness = the STAGE — the small portion that "you know."**
> **GWT = HOW the stage works. "Knowing" = WHAT you experience.**

## §1 — DEFINITION: CONSCIOUSNESS = "KNOWING"

### §1.1 Functional Definition

```
⭐ DEFINITION HIERARCHY:

  PRIMARY DEFINITION (functional — use for analysis):
    Consciousness = the portion of processing that is KNOWN.
    = "Knowing."

  SECONDARY DESCRIPTOR (architectural — use when explaining position):
    Consciousness = enabling state of the chunk network.
    = Sits at meta-level above content.
    (See §1.5 for full explanation.)

  WHY THIS ORDER:
    "Knowing" answers: "what does consciousness DO?" → actionable
      → "Does this person KNOW X?" → check 3 parameters (§1.2)
      → "Why don't they know?" → signal weak? anchor missing? attention elsewhere?
      → "How to increase knowing?" → strengthen signal / improve anchor / direct attention
    "Enabling state" answers: "where does consciousness SIT in the framework?" → descriptive
      → Useful for architecture, not for analysis
    🟡 Hierarchy = framework design choice. Both descriptors are correct.


⭐ DEFINITION:

  CONSCIOUSNESS =
    The portion of processing that is KNOWN.
    = "Knowing."

  UNCONSCIOUS =
    The portion of processing that is NOT known.
    = "Not knowing."

  "Known" means the brain can:
    → Report: articulate it (even vaguely: "something feels off")
    → Remember: retain the experience
    → Use in decision: incorporate into choices
    → Act on it: behave based on it

  NOT required ALL SIMULTANEOUSLY — SPECTRUM:
    → "Something feels off" = vague report + brief memory + hard to use in decision
    → "I'm sad because I lost my job" = report + remember + decide + act
    → = DIFFERENT LEVELS of "knowing," not binary on/off

  ⭐ THE ENTIRE BODY + BRAIN = UNCONSCIOUS (default state).
    Consciousness = the STAGE — the small portion that "you know."
    ~95% of processing = unconscious (Core-Software.md §4).

  🟢 ACCESS CONSCIOUSNESS (Block 1995):
    "Information available for reasoning, reporting, and action control"
    → "Knowing" = framework's functional reframe of access consciousness

  ⚠️ SCOPE LIMITATION:
    → "Knowing" = Level 2 access consciousness ONLY (see §0)
    → Phenomenal consciousness ("why does knowing FEEL like something?")
      = Hard Problem (Chalmers 1995) → still OPEN QUESTION
    → This file does NOT address the Hard Problem

  🟢 Block 1995 | 🟢 Dehaene 2014: "global availability"
  🟡 "Knowing" = framework reframe — equivalent to access consciousness
```

### §1.2 "Knowing" = f(signal × anchor × attention)

```
⭐ "KNOWING" = NOT BINARY, NOT 1D — 3-PARAMETER MODEL:

  NOT KNOWING ─────────── THRESHOLD ─────────── FULL KNOWING

  Below threshold: processing occurs → you DON'T know
  Near threshold: "something feels off" = VAGUE knowing
  Above threshold: ignition → you KNOW → level depends on 3 PARAMETERS

  ⭐ LEVEL OF "KNOWING" = f(signal_strength × anchor_quality × attention):

    ① SIGNAL STRENGTH: how strong is neural activation?
       → Determines IF threshold can be crossed
       → Below threshold: unconscious regardless of other parameters
       → Sources: body-input (hardware), chunk re-firing (compiled),
         PFC activation (fresh)

    ② ANCHOR QUALITY: what type of anchor holds the pattern?
       → Determines HOW STABLY and CHAINABLY you know once conscious
       → Emotional only → vague | Verbal → clear | Full chain → full
       → Range: no anchor → emotional → contextual → ritual → verbal → multi

    ③ ATTENTION: is PFC directing resources to this processing?
       → Determines WHETHER signal gets broadcast to workspace
       → No attention: even strong signal may stay local (compiled efficiency)
       → Full attention: even moderate signal gets broadcast (deliberate pull)
       → = PFC → TRN gate (Neural-Processing-Flow.md §2.3)

  3 parameters CAN partially substitute:
    → Strong signal compensates weak anchor (intense pain → clear without label)
    → Good anchor compensates moderate signal (well-labeled memory → recalled clearly)
    → High attention compensates moderate signal (meditation → subtle sensations)
    → BUT: signal = 0 → always unconscious (no amount of anchor or attention helps)

  4 LANDMARK LEVELS (named TYPICAL parameter combinations):
    Vague knowing — "something feels off" — emotional anchor only
    Clearer knowing — "uneasy about something" — partial label
    Clear knowing — "uneasy because X" — verbal + causal
    Full knowing — "X because Y → need Z" — full chain + plan

  For detailed landmark descriptions: §2.2
  For detailed parameter analysis: §2.1

  🟡 3-parameter model = framework synthesis
  🟢 Individual parameters established (signal: Dehaene 2003; anchor: Chunk.md §5;
     attention: Corbetta & Shulman 2002)
```

### §1.3 "Knowing" vs "Broadcast" — Compatible, Different Level

```
⭐ "BROADCAST" = MECHANISM (HOW). "KNOWING" = FUNCTION (WHAT):

  GWT says: consciousness = information BROADCAST widely to many brain regions.
  Framework says: consciousness = YOU KNOW that information.

  BOTH COMPATIBLE:
    → Broadcast = HOW (mechanism: signal spreads widely → ignition)
    → Knowing = WHAT (result: processing is known)
    → GWT explains the MECHANISM underneath "knowing"
    → "Knowing" describes the RESULT from experiential perspective

  WHY "KNOWING" IS BETTER AS A DEFINITION:
    ① "Knowing" is naturally a SPECTRUM (vague/clear knowing = intuitive)
    ② "Knowing" doesn't depend on push vs pull (both → "knowing")
    ③ "Knowing" resolves counterexamples:
       Chronic stress = cortisol broadcasts body-wide
       → But you DON'T KNOW you're stressed → unconscious
       → "Broadcast" model needs extra explanation
       → "Knowing" model: don't know → unconscious. Simple.
    ④ "Knowing" handles local → conscious transitions
       "That dot ruins the painting" = local visual → but you know → conscious

  ANALOGY:
    "Gravity" = WHAT (objects fall down)
    General Relativity = HOW (spacetime curves)

    "Knowing" = WHAT (I know this information)
    GWT = HOW (broadcast, ignition, thalamo-cortical loop)

  🟢 GWT: Baars 1988, Dehaene 2014
  🟡 "Knowing" as primary definition = framework reframe
```

### §1.4 Consciousness ≠ Brain Region

```
⭐ REFRAME:

  WRONG: "Consciousness resides in the PFC" or "in the thalamus"
  RIGHT: Consciousness = NETWORK STATE

  Like "traffic jam" doesn't "reside in" any particular road.
  Traffic jam = state OF the road network.
  Consciousness = state OF the neural network.

  FRAMEWORK MAPPING:
    → Core-Software.md §0.3: "Follow mechanism, not organization"
    → Consciousness = ENABLING STATE (§1.5)

  NO neuron "is" consciousness.
  There IS a network state that, when it occurs, "knowing" emerges.

  PFC = primary contributor to gating and sustaining consciousness (§6)
  Thalamus = infrastructure for enabling and routing (§5)
  Cortex = generator of conscious content (Koch et al. 2016)
  → Consciousness = what happens when these interact,
    not a property of any one region.

  🟢 Consciousness = network state, not localized (Dehaene 2014)
  🟢 Content generated in posterior cortex (Koch et al. 2016, COGITATE 2025)
  🟡 Consciousness = enabling state of the chunk network = framework reframe
```

### §1.5 Meta-Level: Enabling State

```
⭐ CONSCIOUSNESS = ENABLING STATE:

  3 PROPERTIES OF THE ENABLING STATE:

  ① META-LEVEL: STATE, NOT CONTENT
    Chunk patterns = CONTENT: what appears on the consciousness stage
    Consciousness = STATE: determines whether content is known
    → Different meta-level: content exists WITHIN the state
    → Analogy: "lit stage" vs "actors on stage" — different categories

  ② PREREQUISITE FOR KNOWING:
    Patterns fire without consciousness (~95% unconscious, §1.1)
    Consciousness = when processing becomes KNOWN
    DO need consciousness ON to KNOW any pattern
    = Consciousness gates knowing

  ③ HAS IDENTIFIABLE MECHANISM:
    Thalamo-cortical loop, GWT, biased competition (§5)
    3-role thalamus, 5-checkpoint pathway (§5.1, §5.6)
    3 impairment mechanisms (§5.5)
    = The enabling state has a traceable physical substrate

  WHAT "ENABLING STATE" MEANS:
    Not a system (no new box). Not a component (no brain region).
    = The STATE that the chunk network CAN be in,
      where processing becomes known.
    Conscious/unconscious = labels for whether the state is ON or OFF
    for a specific piece of processing.


⭐ "KNOWING" HAS 2 DIMENSIONS — INTERNAL AND EXTERNAL:

  Dreaming reveals that "knowing" is not monolithic:

  ┌────────────────────┬────────────────────────────────────────────┐
  │ Dimension          │ Description                                │
  ├────────────────────┼────────────────────────────────────────────┤
  │ INTERNAL knowing   │ Experiencing WITHIN the state              │
  │                    │ = subjective experience present             │
  │                    │                                            │
  │ EXTERNAL knowing   │ Report, act, remember AFTER                │
  │                    │ = behavioral access to the experience      │
  └────────────────────┴────────────────────────────────────────────┘

  EVIDENCE FROM DREAMING:
    Normal dreaming: internal knowing PRESENT, external knowing ABSENT
      → You experience the dream, but cannot report or act on reality
      → dlPFC offline → no executive access
    Lucid dreaming: internal + external knowing BOTH PRESENT
      → You experience AND know you're dreaming → can report, control
      → dlPFC reactivated → executive access restored
      🟢 Voss et al. 2014: lucid dreaming → frontal gamma → meta-awareness

  WHY THIS MATTERS:
    → "Knowing" in this framework primarily means EXTERNAL knowing
      (Level 2 access = can report, use, act)
    → Internal knowing alone (dreaming) = consciousness EXISTS
      but is not ACCESSIBLE for behavioral use
    → Confirms: PFC = access ENABLER, not consciousness GENERATOR
      (consciousness exists without PFC → dreaming)

  Cross-ref: Core-Software.md §8, Consciousness-Accessibility.md

  🟡 Internal/external distinction = framework synthesis from dreaming evidence
  🟢 Dreaming consciousness without PFC: Hobson 2009, Nir & Tononi 2010
  🟢 Lucid dreaming + dlPFC: Voss et al. 2014
```

## §2 — SPECTRUM OF "KNOWING"

### §2.1 3-Parameter Model

```
⭐ "KNOWING" LEVEL = f(signal_strength × anchor_quality × attention):

  3 PARAMETERS — each contributes independently:

  ① SIGNAL STRENGTH: how strong is the neural activation?
     → Determines whether processing CAN reach consciousness
     → Below threshold: unconscious regardless of other parameters
     → Above threshold: can become conscious IF attention present
     → Range: subliminal → moderate → strong → overwhelming
     → Sources: body-input (hardware), chunk re-firing (compiled),
       PFC activation (fresh)

  ② ANCHOR QUALITY: what type of anchor holds the pattern?
     → Determines how STABLY and CHAINABLY you know once conscious
     → Emotional only → vague (hard to re-fire, can't chain)
     → + Verbal label → clear (easy to re-fire, can chain)
     → + Full causal chain → full (chain logic + plan)
     → Range: no anchor → emotional → contextual → ritual → verbal → multi

  ③ ATTENTION: is PFC directing resources to this processing?
     → Determines WHETHER signal gets broadcast to workspace
     → No attention: even strong signal may stay local (compiled efficiency)
     → Full attention: even moderate signal gets broadcast (deliberate pull)
     → = PFC → TRN gate (Neural-Processing-Flow.md §2.3)
     → ADHD reveals attention has DISTRIBUTION, not just magnitude:
       Typical: attention distributed across tasks (0-100% continuous)
       ADHD: attention bimodal — near-0% (default) or near-100% (hyperfocus)
       Threshold-gated, not linear — attention "switch," not "dial"
       🟡 Framework synthesis from ADHD drill evidence

  PARTIAL SUBSTITUTION:
    → Strong signal can compensate weak anchor:
      Intense pain → clear knowing even without verbal label
    → Good anchor can compensate weak signal:
      Well-labeled memory recalled deliberately → clear knowing with moderate signal
    → High attention can compensate moderate signal:
      Meditation detects subtle body sensation → knowing what would normally be missed
    → BUT: no amount of anchor or attention helps if signal = 0
      (Hardware-inaccessible: no upward pathway → parameter ① = 0 → always unconscious)

  WHY 3 PARAMETERS, NOT 1 DIMENSION:
    → Same observable level ("vague knowing") can come from different combinations:
      Strong signal + no anchor + partial attention = vague (needs labeling)
      Moderate signal + verbal anchor + no attention = unconscious! (needs attention redirect)
      Weak signal + emotional anchor + full attention = vague (needs signal strengthening)
    → INTERVENTION depends on which parameter is the bottleneck
    → 1D spectrum hides this — precision requires 3 parameters


⭐ TEMPORAL DYNAMICS — PARAMETERS CHANGE OVER TIME:

  f(signal(t) × anchor(t) × attention(t))

  The 3-parameter model is ALREADY DYNAMIC — parameters are not static snapshots.
  Each parameter has a temporal profile:

    Signal(t): decays without re-firing, spikes on new input
    Anchor(t): builds through labeling, degrades with disuse
    Attention(t): shifts between targets, depletes PFC budget over time

  TEMPORAL PHASES OF SAME MODEL (not a separate 4-stage model):
    Entry: signal crosses threshold at t₀ → ignition → conscious
    Resolution: anchor quality enriches at t₁ → richer knowing (not just "in or out")
    Duration: all 3 parameters sustain → content remains on stage
    Exit: any parameter drops below threshold → content leaves stage

  ADHD AS TEMPORAL PROFILE EVIDENCE:
    Parameters adequate at t₀ may decay rapidly by t₁.
    Not a "deficit" profile — a temporal DECAY profile:
    attention(t₀) = sufficient → attention(t₁) = insufficient (PFC sustain impaired)
    Hyperfocus: reward dopamine restores sustain → attention(t) becomes stable
    🟢 Barkley 1997: ADHD = deficit in sustained attention, not attention capacity
    🟡 Temporal decay framing = framework synthesis

  STRUCTURAL PATTERN — WHY 3 PARAMETERS ARE SUFFICIENT:
    Every candidate "4th parameter" tested maps to existing parameters:
      Chunk density → affects ALL 3 params simultaneously (signal + anchor + attention)
      Prediction-delta → modulates signal + attention via VTA dopamine
      Interoception → domain-specific signal (not separate parameter)
      ADHD temporal profile → temporal evolution of existing parameters
    = Depth factors BEHIND existing parameters, not new dimensions.
    Pattern: what LOOKS like a new parameter is a MODULATOR of existing ones.

  🟡 3-parameter model = framework synthesis (new framing)
  🟢 Individual parameters established:
     Signal strength: Dehaene et al. 2003 (ignition threshold)
     Anchor quality: Chunk.md §5 (5 anchor types)
     Attention: Corbetta & Shulman 2002, McAlonan et al. 2008 (TRN gate)
  🟡 Temporal dynamics framing = framework synthesis from dynamics drill
  🟢 ADHD sustained attention: Barkley 1997
```

### §2.1b Post-Threshold Resolution

```
⭐ CROSSING THRESHOLD ≠ END OF STORY — RESOLUTION ENRICHES CONTENT:

  Current understanding (Feeling.md §2.2b): clarity LOWERS threshold (entry).
  Additional dimension: clarity also ENRICHES post-threshold content (resolution).

  Chunk density enriches what APPEARS on stage, not just WHETHER it appears:

  EXPERT vs NOVICE — SAME INPUT, DIFFERENT KNOWING:
    Novice chess player looks at board:
      → Few chunks activated → signal above threshold → basic knowing
      → "I see pieces on a board"
    Expert chess player looks at board:
      → Many interconnected chunks activated → rich knowing
      → "I see a Sicilian Defense with a weak kingside"
    Same raw input, different chunk activation, different resolution.

  THIS IS NOT "BETTER CONSCIOUSNESS":
    → Same 3-parameter mechanism
    → Expert has: more anchors (②) → richer content on stage
    → Expert has: stronger aggregate signal (①) from chunk convergence
    → = Different content QUALITY through same mechanism, not different mechanism

  🟡 Post-threshold resolution = framework synthesis from dynamics drill
  🟢 Chunk expertise: Chase & Simon 1973 (chess expertise + chunking)
```

### §2.1c Prediction-Delta as Multi-Parameter Modulator

```
⭐ PREDICTION-DELTA MODULATES ALL 3 PARAMETERS SIMULTANEOUSLY:

  When reality ≠ prediction → prediction error (delta) occurs.

## §5 — MECHANISMS: HOW "KNOWING" WORKS

### §5.1 Mechanism Overview: Recurrence + Broadcast

```
⭐ MECHANISM PATHWAY — RECURRENCE THEN BROADCAST:

  signal → local processing (feedforward, ~50-150ms, unconscious)
  → RECURRENCE through primary cortex (RPT: V1/S1/A1 feedback loops, ~150-200ms)
  → IF recurrence SUFFICIENT → BROADCAST to global workspace (GWT: ~270-350ms, P3b)
  → "KNOWING"

  OLDER VIEW: signal → ignition → broadcast → conscious (GWT alone)
  REFINED VIEW: signal → recurrence → broadcast → conscious (RPT + GWT)

  RPT + GWT = COMPLEMENTARY, NOT COMPETING:
    → RPT (Recurrent Processing Theory, Lamme 2003):
      Consciousness requires FEEDBACK loops in primary cortex
      Higher cortex → lower cortex → higher cortex (local recurrence)
      = CP4 in 5-Checkpoint model (§5.6)
    → GWT (Global Workspace Theory, Baars 1988, Dehaene 2014):
      Consciousness requires BROADCAST to global workspace
      Signal spreads to PFC + parietal + temporal + cingulate
      = CP5 in 5-Checkpoint model (§5.6)
    → INTEGRATION: recurrence FIRST (local feedback), broadcast SECOND (global access)
      Recurrence = necessary precondition for broadcast, not alternative

  WHY RECURRENCE IS NEEDED BEFORE BROADCAST:
    → BLINDSIGHT: patient with V1 damage can DETECT but NOT KNOW
      = Signal reaches cortex (feedforward OK) but NO V1 recurrence
      = CP4 fails → CP5 never reached → no "knowing"
    → TMS AT ~100-120ms: disrupts recurrence → awareness abolished, processing intact
      = Timing shows recurrence at specific temporal window
    → BACKWARD MASKING: mask disrupts recurrence → stimulus processed but NOT known
    → COGITATE 2025: NEITHER GNW nor IIT fully correct alone
      Content = posterior cortex (not PFC). Access = PFC. Routing = thalamus.

  FRAMEWORK MAPPING:
    Global workspace = Thalamo-cortical network
    Spotlight = PFC → TRN gate (PFC-Function.md §4)
    Ignition = Feel-Consciousification (Feeling.md §2.2)
    Threshold = f(magnitude × clarity) (Feeling.md §2.2b)
    Workspace capacity = PFC ~4±1 slots (PFC-Function.md §2)

  🟢 Baars 1988, Dehaene & Changeux 2011, COGITATE 2025
  🟢 Lamme 2003: recurrent processing theory
  🟢 Ro 2021 (Current Biology): numbsense TMS
  🟡 RPT + GWT complementary integration = framework synthesis
```

```
⭐ NEURAL MECHANISM: THALAMO-CORTICAL LOOP RESONANCE:

  IGNITION = SUSTAINED THALAMO-CORTICAL LOOP:

    ① Signal enters cortex modality area → LOCAL processing (unconscious)
    ② If strong enough → signal RE-ENTERS thalamus (Layer 6 → thalamus feedback)
    ③ Thalamus relays back → cortex → thalamus → cortex (~40Hz gamma loop)

### §5.2 Biased Competition

```
🟢 SCHEMAS COMPETE CONTINUOUSLY:

  ① Compiled schemas fire based on activation strength

  ② PFC BIASES the competition (§6)

     → dlPFC resolves: amplify one, suppress the other
  ④ When NO conflict → compiled auto-runs
     → PFC releases → schemas run freely
     → = Highway hypnosis: compiled driving, PFC elsewhere

### §5.3 2 Pathways: Push + Pull

```
🟡 2 PATHWAYS TO "KNOWING":

  ① BOTTOM-UP (signal push):
     → Signal strong enough → self-ignites → broadcasts → knowing
     → Example: intense stomach pain → NO need for attention → know it hurts
     → Example: loud door slam → attention capture → knowing
     → = Signal PUSHES onto the stage

  ② TOP-DOWN (attention pull):
     → PFC deliberately directs attention → pulls signal up → knowing
     → Example: "attend to your breath" → compiled processing pulled up → knowing
     → Example: expert focuses → detects small signal → knowing
     → = PFC PULLS signal onto the stage

  → Both → SAME RESULT: you "know."
  → = Why "knowing" is BETTER than "broadcast" as definition:
    both push and pull → "knowing." No need to distinguish mechanism.

  🟡 "2 pathways" framing = framework synthesis
  🟢 Bottom-up attention capture: Corbetta & Shulman 2002
  🟢 Top-down attention: PFC → TRN gate (McAlonan et al. 2008)

### §5.4 Arousal vs Content

```
🟢 CONSCIOUSNESS = AROUSAL + CONTENT (2 separable components):

  ┌────────────────────────────────────────────────────────────────┐
  │ CONTENT (what)                        GENERATED BY:            │
  │ = "see WHAT, hear WHAT, think WHAT"   Cortex (Zone B) + PFC   │
  ├────────────────────────────────────────────────────────────────┤
  │ AROUSAL (wakefulness)                 CONTROLLED BY:           │
  │ = "awake or not"                      Brainstem RAS + Thalamus │
  └────────────────────────────────────────────────────────────────┘

  METAPHOR:
    Brainstem RAS = POWER PLANT (electricity ON/OFF)
    Thalamus = CENTRAL EXCHANGE (power distribution + signal routing + loop)
    Cortex = SCREENS (generate specific content)
    PFC = REMOTE CONTROL (selects channel via thalamic gate)

  WHY IT MATTERS:
    → Disable arousal (brainstem/thalamus) = TOTAL loss of consciousness (coma)
    → Disable 1 cortex area = loss of 1 CONTENT (blind, mute) but still conscious
    → Disable PFC = loss of CONTROL but still conscious
    → Arousal = necessary CONDITION for consciousness
    → Cortex = where consciousness CONTENT occurs
    → PFC = biased hub choosing WHICH content gets broadcast

  🟢 Moruzzi & Magoun 1949: RAS discovery
  🟢 Alkire et al. 2008: anesthesia + thalamus


  ⚠️ THALAMUS ≠ BRAINSTEM (distinct roles, often conflated):

    Brainstem RAS = ON/OFF SWITCH (arousal — system-wide power)
    Thalamus = CENTRAL EXCHANGE (arousal threshold + content routing + loop sustaining)

    → Brainstem provides POWER (ON/OFF)
    → Thalamus provides ROUTING + LOOP (which content reaches cortex + sustains)
    → Both OFF → coma, but for DIFFERENT reasons:
      Brainstem OFF → no power → nothing runs
      Thalamus OFF → no routing → content cannot reach cortex, loops cannot form

    🟢 Jones 1998: matrix (diffuse/arousal) vs core (specific/content) thalamic neurons
    🟢 Sherman 2007: first-order (relay) vs higher-order (cortico-cortical) thalamic nuclei

    └────────────────────┴──────────────────────┴──────────────────────┘

    Same thalamus, same anatomy — MODE determines consciousness.
    = Checkpoint 2 (CP2) in the 5-checkpoint model (§5.6).
    Tonic mode = necessary CONDITION, not sufficient.
    Tonic + all other checkpoints → consciousness possible.
    Burst mode → ALL signals blocked regardless of other factors.

```
