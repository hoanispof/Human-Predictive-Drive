# Consciousness — "Knowing" as Functional Definition

> **You KNOW you are reading this sentence.**
> **You DON'T KNOW which grammar rules are processing this sentence.**
> **Both are firing. The difference: which one you KNOW.**
>
> **Consciousness is NOT a "brain region."**
> **Consciousness is NOT a "system."**
> **Consciousness = "KNOWING" — the portion of processing the observer CAN access.**
>
> **"Knowing" = spectrum: vague knowing → clear knowing → chain logic → plan.**
> **The entire body + brain = unconscious (default).**
> **Consciousness = the stage — the SMALL portion that "you know."**
>
> **GWT = HOW "knowing" works (broadcast, ignition, thalamo-cortical loop).**
> **"Knowing" = WHAT we observe (I know this information).**
> **Two levels COMPATIBLE — they do not replace each other.**

## §1 — DEFINITION: CONSCIOUSNESS = "KNOWING"

### §1.1 Functional Definition

```
⭐ DEFINITION:

  CONSCIOUSNESS =
    The portion of processing that the OBSERVER CAN ACCESS.
    = "Knowing."

  UNCONSCIOUS =
    The portion of processing that the observer CANNOT access.
    = "Not knowing."

  "Knowing" = the observer can:
    → Report: articulate it (even vaguely: "something feels off")
    → Remember: retain the experience
    → Use in decision: incorporate into choices
    → Act on it: behave based on it

  NOT required ALL SIMULTANEOUSLY — SPECTRUM:
    → "Something feels off" = vague report + brief memory + hard to use in decision
    → "I'm sad because I lost my job" = report + remember + decide + act
    → = DIFFERENT LEVELS of "knowing," not binary on/off

  ⭐ THE ENTIRE BODY + BRAIN = UNCONSCIOUS (default state).
    Consciousness = THE STAGE = the SMALL portion that "you know."
    ~95% of processing = unconscious (Core-Software.md §4).

  🟢 ACCESS CONSCIOUSNESS (Block 1995):
    "Information available for reasoning, reporting, and action control"
    → "Knowing" = framework's functional reframe of access consciousness

  ⚠️ LIMITATION:
    → "Knowing" = access consciousness ONLY
    → Phenomenal consciousness ("why does knowing FEEL like something?")
      = Hard Problem (Chalmers 1995) → still OPEN QUESTION
    → This file does NOT address the Hard Problem

  🟢 Block 1995 | 🟢 Dehaene 2014: "global availability"
  🟡 "Knowing" = framework reframe — equivalent to access consciousness
```

### §1.2 "Knowing" = Spectrum

```
⭐ "KNOWING" = SPECTRUM, NOT BINARY:

  NOT KNOWING ─────────── THRESHOLD ─────────── FULL KNOWING

  Below threshold: processing occurs → you DON'T know
  Near threshold: "something feels off" = VAGUE knowing
  Above threshold: ignition → you KNOW → level depends on anchor quality

  5 LEVELS:
    ┌──────────────────┬─────────────────────────────────────────┐
    │ Level            │ Characteristics                         │
    ├──────────────────┼─────────────────────────────────────────┤
    │ Not knowing      │ Processing runs, observer has no access │
    │ ════ THRESHOLD ══│═════════════════════════════════════════ │
    │ Vague knowing    │ "Something's there" — emotional anchor  │
    │ Clearer knowing  │ "Uneasy about something" — partial label│
    │ Clear knowing    │ "Uneasy because X" — verbal + causal   │
    │ Full knowing     │ "X because Y → need Z" — chain + plan  │
    └──────────────────┴─────────────────────────────────────────┘

  KEY: Each level = STRONGER anchor → EASIER to re-fire → MORE chainable
  Training = MOVING patterns from "vague knowing" → "full knowing"

  🟡 Spectrum of "knowing" = framework synthesis
  Details + examples: Drill-Consciousness-Knowing.md §2
```

### §1.3 "Knowing" vs "Broadcast" — Compatible, Different Level

```
⭐ "BROADCAST" = MECHANISM (HOW). "KNOWING" = FUNCTION (WHAT):

  GWT says: consciousness = information BROADCAST widely to many brain regions.
  Framework says: consciousness = YOU KNOW that information.

  BOTH COMPATIBLE:
    → Broadcast = HOW (mechanism: signal spreads widely → ignition)
    → Knowing = WHAT (result: observer can access)
    → GWT explains the MECHANISM underneath "knowing"
    → "Knowing" describes the EXPERIENCE from the observer's perspective

  WHY "KNOWING" IS BETTER AS A DEFINITION:
    ① "Knowing" is naturally a SPECTRUM (vague/clear knowing = intuitive)
    ② "Knowing" doesn't depend on push vs pull (both → "knowing")
    ③ "Knowing" resolves counterexamples:
       Chronic stress = cortisol broadcasts body-wide
       → But you DON'T KNOW you're stressed → unconscious
       → "Broadcast" model needs extra explanation
       → "Knowing" model: don't know → unconscious. Simple.
    ④ "Knowing" handles local → conscious transitions
       "That dot ruins the painting" = local visual → but you know → conscious

  ⭐ ANALOGY:
    "Gravity" = WHAT (objects fall down)
    General Relativity = HOW (spacetime curves)

    "Knowing" = WHAT (I know this information)
    GWT = HOW (broadcast, ignition, thalamo-cortical loop)

  🟢 GWT: Baars 1988, Dehaene 2014
  🟡 "Knowing" as primary definition = framework reframe
  Detailed test cases: Drill-Consciousness-Knowing.md §1.3
```

### §1.4 Consciousness ≠ Brain Region

```
⭐ REFRAME:

  WRONG: "Consciousness resides in the PFC" or "in the thalamus"
  RIGHT: Consciousness = NETWORK STATE

  Like "traffic jam" doesn't "reside in" any particular road.
  Traffic jam = state OF the road network.
  Consciousness = state OF the neural network.

  FRAMEWORK MAPPING:
    → Core-Software.md §0.3: "Follow mechanism, not organization"
    → Observation parameters = names for patterns, NOT components
    → Consciousness = observation parameter for the broadcast state
    → Like "Drive" = observation label, "Novelty" = observation label
    → Consciousness = observation label for the "knowing" state

  NO neuron "is" consciousness.
  There IS a network state that, when it occurs, "knowing" emerges.

  🟡 Consciousness = observation parameter = framework reframe
  🟢 Consciousness = network state, not localized (Dehaene 2014)
```

### §3.1 Hardware-Inaccessible

```
① HARDWARE-INACCESSIBLE — too far, no upward pathway:

  → Gut brain (enteric nervous system: ~500 million neurons)
  → Brainstem autonomic (heart rate, blood pressure, digestion)
  → Spinal cord reflexes
  → Zone D hardware (Core-Hardware.md)
  → MECHANISM: no pathway UP to cortex (or extremely weak)
  → CANNOT "know" by willpower — UNCONSCIOUS DUE TO ARCHITECTURE

  Example: blood pressure 120/80, insulin levels, blood pH
  → Highly complex processing but NO PATHWAY to consciousness

  🟢 Interoceptive pathways: Craig 2002 (some have weak upward path)
```

### §3.2 Compiled-Efficient

```
② COMPILED-EFFICIENT — already known, no NEED to know anymore:

  → Grammar while speaking (Broca's area)
  → Maintaining balance (cerebellum)
  → Recognizing familiar faces (fusiform face area)
  → MECHANISM: chunks compiled DEEPLY → fire locally → task done
  → HAS anchor (motor/verbal) but PFC DOESN'T NEED to observe
  → CAN be pulled to consciousness if attention DELIBERATELY directed
  → = UNCONSCIOUS DUE TO EFFICIENCY (Compiled + consciousness OPTIONAL)

  SWITCHING example: breathing
    → Normal: brainstem automates = unconscious
    → Attend: attention → ignition → conscious
    → Stop attending: unconscious again
    = Compiled but CAN become conscious when attention directed

  🟢 Implicit learning: Reber 1967
  🟢 Compiled motor: cerebellum + basal ganglia (established)
```

### §3.3 Background-Invisible

```
③ BACKGROUND-INVISIBLE — too diffuse, no boundary:

  → Chronic stress building gradually over days/years
  → Cultural bias ("don't cause trouble for others")
  → Personality pattern ("I'm just that kind of person")
  → Implicit bias
  → MECHANISM: Background-Pattern = pattern that permeates EVERYTHING
    → NO boundary → PFC cannot "grasp" it
    → Background-Pattern.md §5: 7 REASONS PFC cannot observe:
      ① VTA habituated (constant → no delta)
      ② Too distributed (no boundary → can't capture)
      ③ Gist extracted (details faded)
      ④ Labeled as identity ("I'm just like that")
      ⑤ Compound signal (fires together with events)
      ⑥ Compiled suppress absorbed into baseline
      ⑦ PFC budget depleted by pattern itself
  → = UNCONSCIOUS DUE TO DIFFUSION (Background-Pattern)

  THIS IS THE HARDEST TYPE OF "NOT KNOWING":
    → No boundary → no natural anchor
    → NEEDS external catalyst: friends, therapy, AI
    → External catalyst = provides anchor → BEGINS "knowing"

  ⭐ CHRONIC STRESS COUNTEREXAMPLE:
    → Cortisol broadcasts THROUGHOUT the body (body-wide signal)
    → But you DON'T KNOW you're stressed
    → In "broadcast" model: signal broadcasts strongly yet unconscious?
    → In "knowing" model: don't know → unconscious. Simple.

### §5.2 Biased Competition

```
🟢 SCHEMAS COMPETE CONTINUOUSLY:

  ② PFC BIASES the competition (§6)

  ③ ACC DETECTS conflict

  ④ When NO conflict → compiled auto-runs
     → PFC releases → schemas run freely

### §5.3 2 Pathways: Push + Pull

```
🟡 2 PATHWAYS TO "KNOWING":

  ① BOTTOM-UP (signal push):
     → Signal strong enough → self-ignites → broadcasts → knowing
     → Example: intense stomach pain → NO need for attention → know it hurts
     → Example: loud door slam → attention capture → knowing
     → = Signal PUSHES onto the stage

  ② TOP-DOWN (attention pull):
     → PFC deliberately directs attention → pulls signal up → knowing
     → Example: "attend to your breath" → compiled processing pulled up → knowing
     → Example: expert focuses → detects small signal → knowing
     → = PFC PULLS signal onto the stage

  → Both → SAME RESULT: you "know."
  → = Why "knowing" is BETTER than "broadcast" as definition:
    both push and pull → "knowing." No need to distinguish mechanism.

  🟡 "2 pathways" framing = framework synthesis
  🟢 Bottom-up attention capture: established (Corbetta & Shulman 2002)
  🟢 Top-down attention: PFC → TRN gate (McAlonan et al. 2008)

### §8.3 Compilation Trajectory

```
🟡 FRESH CONSCIOUS → COMPILED UNCONSCIOUS:

  TRAJECTORY:
    Learning to drive:
      New: every step = Fresh → PFC hold → conscious → slow, tiring
      Experienced: every step = Compiled → local → unconscious → fast, effortless
    = PFC-Operations.md §1.2: Fresh → Compiled = learning

  RULE:
    → Compilation = moving processing from conscious → unconscious
    → Learning = automating → freeing workspace → PFC available for other tasks
    → Expert = more Compiled → more unconscious → LESS PFC cost → can do more

  REVERSE ALSO OCCURS:
    → Compiled → surprise (novelty spike) → pulled back to consciousness
    → Highway hypnosis → car ahead brakes hard → instantly conscious
    → = Novelty detection: prediction-delta → attention capture → ignition

  ⭐ IMPLICATION FOR TRAINING:
    → Somatic-Articulation-Loop: body-knowledge → explicit-knowledge
    → = REVERSE compilation: pulling compiled unconscious UP to consciousness (temporarily)
    → Purpose: inspect, refine, re-compile at higher quality
    → Meditation = reactive → proactive mode shift (Self-Observation.md §6)



  ⭐ RECALL VS DISCOVERY — 2 TASKS FOR REVERSE COMPILATION:

    §7.3 defines 2 compilation pathways. Pulling compiled patterns BACK
    to consciousness = 2 DIFFERENT tasks depending on which pathway
    created the compiled pattern:

    RECALL (from Explicit Pathway compiled):
      → Verbal anchor EXISTS (was created during original conscious processing)
      → Re-fire anchor → chunk activates → Knowing
      → FAST — anchor already present
      → Example: "capital of France?" → "Paris" (re-fire verbal anchor)

    DISCOVERY (from Implicit Pathway compiled):
      → NO verbal anchor (never consciously processed → never labeled)
      → Must CREATE new verbal anchor for something already compiled
      → = Reverse engineering: observe behavior → hypothesize rule → test
      → SLOW — requires Somatic-Articulation-Loop process (above)
      → Example: "describe your native grammar rules?" → ???
        (must discover what you use fluently every day)

    WHY DISCOVERY IS SLOW (3 reasons):
      ① No verbal anchor → must create from scratch (Chunk.md §6.2: labeling)
      ② Compiled fires too efficiently → completes before PFC can observe
        (PFC sees RESULT but not PROCESS — like watching a magic trick)
      ③ Pattern too complex to verbalize fully (tacit knowledge: Polanyi 1958)

    This is why linguistics takes centuries to systematize what a 3-year-old
    uses fluently — the task is DISCOVERY, not recall.

    = Karmiloff-Smith 1992 "representational redescription":
      Level I (implicit): knowledge IN the system, not TO the system
      Level E1 (partially explicit): accessible, partially verbalizable
      Level E2-3 (fully explicit): fully explicit, verbally transferable
      Transition I → E1 → E2-3 = the DISCOVERY process
      = Established developmental psychology framework for the same distinction


  ⭐ PREDICTION-DELTA TRACKS COMPILATION:

    The Schultz prediction error signal (Schultz et al. 1997) — called
    "prediction-delta" in this framework (Novelty.md §1.1) —
    maps directly onto the compilation trajectory:

    Learning start:
      → Pattern new → prediction-delta HIGH → VTA fires → dopamine → conscious
    Learning progress:
      → Pattern becoming familiar → prediction-delta DECREASING → dopamine decreasing
    Mastery:
      → Pattern fully predicted → prediction-delta ≈ 0 → dopamine silent → unconscious
    Surprise (violation):
      → Compiled schema violated → prediction-delta SPIKES → VTA fires
      → Pattern PULLED BACK to consciousness for re-evaluation
      → Already described above as "surprise → conscious"
      → This IS the Schultz prediction error mechanism

    = Prediction-delta is a TRACKABLE SIGNAL of where a pattern sits
      on the compilation trajectory. Not just a metaphor — measurable
      in dopamine neuron firing (Schultz 1997), striatal dopamine
      (Phillips 2024), and fMRI activation patterns (Poldrack 2005).


  🟡 Compilation trajectory analysis = framework synthesis
  🟡 Recall vs Discovery distinction = framework synthesis
    (components from Polanyi, Karmiloff-Smith, Reber — framing is new)
  🟡 Prediction-delta tracking = framework synthesis
    (mapping is new, individual components are 🟢)

  🟢 Prediction-delta: Schultz, Dayan & Montague 1997

### §8.4 Meta-Chunks: The Prerequisite for Learning

```
⭐ WHAT MUST EXIST BEFORE COMPILATION CAN BEGIN?

  §8.3 shows how Fresh processing compiles into unconscious patterns.
  But this trajectory ASSUMES the learner can engage with the material.
  What happens when there are ZERO chunks in a domain?


⭐ ZERO CHUNKS = LEARNING IMPOSSIBLE:

  → No chunks in domain = no prediction model for that domain
  → No prediction model = no prediction-delta possible
  → No prediction-delta = VTA cannot fire = domain is "invisible"
    to the dopamine system
  → Result: information in this domain = noise, not signal (§4.3)

  Example: elementary school student hearing "E = mc²":
    → No physics chunks → no prediction model → no prediction-delta
    → VTA silent → PFC does not engage → cannot learn
    → Not "too hard" — literally INVISIBLE to the learning system


⭐ META-CHUNKS = PREDICTION-DELTA BECOMES POSSIBLE:

  Meta-chunks = chunks ABOUT a domain (not IN the domain):
    → Know the name: "theory of relativity" (verbal anchor for domain)
    → Know the category: "physics" (contextual anchor)
    → Know one does not understand: self-assessment

  Meta-chunks create a MINIMAL PREDICTION MODEL:
    → "Domain X exists, I don't understand it"
  Now prediction-delta is POSSIBLE:
    → Encounter domain information →
      "this is something I don't understand"
    → Delta: "seeing a pattern" ≠ "expecting not to understand"
      → prediction-delta > 0
    → VTA fires (weakly) → PFC notices

  ⚠️ APPRAISAL STEP (PACE framework, Gruber & Ranganath 2019):
    Prediction-delta alone is NOT SUFFICIENT for curiosity.
    Lateral PFC evaluates: "is this prediction error worth pursuing?"
    → If appraised as valuable → curiosity → exploration → learning
    → If appraised as threatening → anxiety, avoidance
    → If appraised as irrelevant → ignore
    = Meta-chunks ENABLE prediction-delta, but appraisal FILTERS
      what passes through to curiosity


⭐ CHUNK-LEARNING SPECTRUM:

  Zero chunks → Meta-chunks → Domain chunks → Compiled mastery
  ─────────────────────────────────────────────────────────────
  No prediction   Minimal model    Rich model       Full model
  model           (coarse delta)   (fine delta)     (delta ≈ 0)
  VTA invisible   VTA can fire     VTA fires often  VTA silent
  No curiosity    Curiosity begins Curiosity peaks  Mastery
  Cannot learn    Learning begins  Deep learning    Automated


⭐ LIFECYCLE:

  ① Zero → meta-chunks: requires EXTERNAL catalyst
     (someone tells you the domain exists → meta-chunk forms)
  ② Meta-chunks → domain chunks: INTERNAL dopamine drives
     (curiosity → seek → learn → richer model →
      more gaps → more curiosity)
  ③ Domain chunks → compiled: dopamine DECLINES
     (prediction-delta → 0 → VTA silent → automated)
  = Lifecycle begins EXTERNAL, becomes INTERNAL, ends SILENT


⭐ CONNECTION TO §4 (Anchor = Consciousness Enablement):

  → Meta-chunks = the first anchors in a domain
  → Without meta-chunks: no anchor → no retrieval path → noise (§4.3)
  → With meta-chunks: anchor exists → retrieval possible → signal
  → = Meta-chunks are the PREREQUISITE for §4's anchor enablement mechanism
  → Cross-ref: Consciousness-Accessibility.md §5.2 (meta-chunks × accessibility)


  CONFIDENCE:
    🟡 Framework synthesis (spectrum framing, lifecycle,
       meta-chunks → prediction-delta mechanism)
    🔴 Meta-chunks → prediction model: logically consistent,
       no direct evidence that meta-knowledge creates
       dopamine-detectable prediction models specifically

  Upgraded from pure 🔴 after indirect but CONVERGENT support:
    🟢 Loewenstein 1994: "priming dose of knowledge greatly increases curiosity"
    🟢 Gruber & Ranganath 2019 (PACE): prediction error + appraisal → curiosity
    Still 🟡 not 🟢: no direct mechanism-level evidence
      for meta-chunks → VTA firing specifically

  SOURCES:
    Loewenstein 1994 (information gap theory of curiosity)
    Gruber & Ranganath 2019 (PACE: prediction error + appraisal → curiosity)
    Schultz 1997 (prediction-delta requires prediction model — implicit)
    Novelty.md v1.2 §1.1 ("VTA + chunks both required")
    Drill-Compiled-Fresh §5 (meta-chunks, zero chunks)
    Drill-Consciousness-Bridge §4 (meta-chunks × prediction-delta connection)
```
