## §1 — WHAT ARE EMERGENT PATTERNS

```
🟡 DEFINITION:

  Emergent Pattern = a stable relationship between the body and a domain entity,
  COMPILED from repeated domain interaction.

  NOT pre-designed modules:
    → There is no dedicated brain region for "enemy" or "nurturing"
    → Same set of mechanisms (Classify × Process × Valence × Body-base)
    → DIFFERENT input → DIFFERENT output → DIFFERENT patterns emerge

  NOT conscious decisions:
    → The body AUTOMATICALLY accumulates valence with every interaction
    → Patterns are COMPILED into Entity-Compiled chunks unconsciously
    → PFC only RECOGNIZES a pattern that has already formed ("yes, they're my friend")
    → PFC does NOT CREATE the pattern ("I've decided they're my friend")


  WHY CALL IT "EMERGENT":

    ① Nobody designed it in advance:
       → REPEATED interaction + ACCUMULATED valence = pattern SELF-GENERATES

    ② Emerges from the interaction, not from the individual:
       → 1 person CANNOT have a connection pattern alone (needs 2+)
       → The pattern lives BETWEEN the body and the entity, belonging to neither

    ③ Cannot be precisely predicted:
       → Depends on: accumulated valence + context + body-state + both parties


  STRUCTURE OF EACH PATTERN:

    ┌──────────────────────────────────────────────────────────────┐
    │ TRIGGER: domain interaction conditions that lead to pattern  │
    │ MECHANISM: how the body processes it (Self-Pattern-Modeling  │
    │             + Valence)                                       │
    │ COMPILED STATE: what the formed pattern looks like           │
    │ INFLUENCE: how the pattern shapes subsequent interactions     │
    │ BREAK CONDITION: when the pattern dissolves / changes        │
    └──────────────────────────────────────────────────────────────┘

    → ALL patterns in this file follow this structure
    → Bond-Architecture.md v2.0 FORMALIZES patterns into 4 bond types
    → Resonance-Sustainability.md v1.0 PREDICTS which patterns persist
```

## §3 — ENEMY / THREAT PATTERN

```
🟡 ENEMY = THE OPPOSITE FACE OF CONNECTION:

  Connection: sustained mutual POSITIVE valence → bond compiles
  Enemy:      sustained NEGATIVE valence → avoidance / fight pattern compiles

  BUT the enemy pattern HAS DISTINCT CHARACTERISTICS — it is NOT simply
  "connection in reverse":


  DEFINITION:

    Enemy Pattern = compiled negative-valence relationship
    between the body and a domain entity (agent OR object),
    when the entity CONTINUOUSLY affects body channels NEGATIVELY.


  ⭐ ASYMMETRY — Enemy forms FASTER than Connection:

    🟢 Negativity bias (Baumeister 2001, Rozin & Royzman 2001):
      → Negative events have STRONGER impact than positive events of the same magnitude
      → 1 betrayal > 10 acts of help (in terms of valence impact)
      → Evolutionary logic: missing 1 threat = DEATH. Missing 1 friend = just a missed opportunity

    Connection: requires MANY positive interactions → emerges gradually
    Enemy: can form from just 1 STRONG negative interaction → IMMEDIATE compile

    Connection: lost QUICKLY with 1 major violation (§7)
    Enemy: lost SLOWLY — requires MANY positive interactions to override (if even possible)

    → Asymmetric cost: false-negative enemy = death. False-positive = just costs energy.


  3 FORMS OF ENEMY PATTERN:

    ① AGENT ENEMY — sustained negative valence from an agent:
       → A bully, a toxic boss, a battlefield combatant
       What makes agent enemy special:
         → Agent HAS their own goals → unpredictable → adaptive
         → Agent KNOWS you are avoiding them → adjusts strategy → arms race
         → = Agent enemy is MORE DANGEROUS than object enemy

    ② OBJECT ENEMY — compiled negative valence from an object:
       → Poison, fire, heights: body-base threat compiled from experience/schema
       → Object = deterministic, not adaptive → EASIER to avoid than agent
       → BUT: schema CAN amplify excessively (phobia)

    ③ OBJECT-AS-PROXY — object reclassified because of agent schema:
       → A bush on a battlefield: object → "an agent might be hiding inside" → threat
       → Traps, landmines: object CARRIES agent intent → valence from agent, form of object
       → = Self-Pattern-Modeling Compiled fires "there's an agent here" even when there's only object


  ⭐ DEHUMANIZATION — reclassifying an agent as an object:

    🟡 The problem: agent enemy → Self-Pattern-Modeling STILL fires
      → Simulation generates internal model of agent's state
      → Causes dissonance when taking harmful action
      → = NATURAL BARRIER against violence

    Evolutionary solution: RECLASSIFY agent → object
      → Object classification → Self-Pattern-Modeling DOES NOT fire
      → = Can harm or destroy WITHOUT simulation dissonance

    Cases:
      "Gooks," "cockroaches" (Rwanda) = dehumanizing language
        → Relabels agent → vermin/object → bypasses Self-Pattern-Modeling
      Surgeon: patient → "the case" → Object mode
        → TEMPORARY, purpose is to HELP = functional override

    → Same mechanism, DIFFERENT context:
      Surgeon: override TEMPORARY, purpose to HELP = healthy
      War: override TOTAL, purpose to DESTROY = dangerous


  ENEMY × SCHEMA INHERITANCE:

    🟡 Enemy pattern CAN BE TRANSMITTED through schema WITHOUT direct experience:
      → "Snakes = dangerous" → enemy compiled BEFORE ever seeing a snake
      → "Group X = the enemy" → compiled BEFORE ever meeting anyone from that group
      → = Schema inheritance (Valence-Propagation v3.0): fast but LESS accurate

    Risk: overgeneralization
      → "1 person from Group X hurt me" → compiles "ALL of Group X = enemy"
      → = Prejudice, racism = enemy pattern inherited + overgeneralized
      → Same mechanism as §7 Violation response ② (overgeneralize after violation)
```

## §4 — NURTURING PATTERN

```
🟡 NURTURING = EMERGENT PATTERN, NOT A SEPARATE MODULE:

  DEFINITION:
    Nurturing = the drive to care for a high-vulnerability entity,
    emerging from: Self-Pattern-Modeling Compiled × vulnerability cues
    × perceived ability × existing body-base channels.


  4 FACTORS DETERMINING NURTURING INTENSITY (multiplicative, not additive):

    ① LIVING — must have life (or the perception of it):
       → Living → Self-Pattern-Modeling CAN fire
       → Non-living → Self-Pattern-Modeling CANNOT (except through anthropomorphism)

    ② VULNERABILITY — the more helpless → signal is STRONGER:
       → 🟢 Baby Schema (Lorenz 1943, Glocker 2009):
         Large eyes, round face → nucleus accumbens fires even in non-parents
       → Hardware trigger: vulnerability = potential death → auto-prioritize

    ③ EXPRESSIVENESS — the more clearly state is expressed → signal is STRONGER:
       → Human face: EXTREMELY expressive → Self-Pattern-Modeling receives rich input
       → Fish: near-zero expressiveness → Self-Pattern-Modeling receives poor input
       → 🟢 Facial EMG mimicry (Dimberg 2000): automatic, pre-conscious, 300ms

    ④ SIMILARITY / FAMILIARITY — the more similar/familiar → signal is STRONGER:
       → Dense compiled chunks → more accurate simulation → stronger signal
       → 🟢 In-group empathy bias (Xu 2009): ACC response stronger for in-group

    The 4 factors MULTIPLY each other:
      Own baby = Living ✅ × Vulnerability MAX × Expressiveness HIGH × Familiarity MAX
        → = STRONGEST nurturing response
      Pet fish = Living ✅ × Vulnerability LOW × Expressiveness ≈ 0 × Familiarity LOW
        → = Nurturing near ZERO → primarily aesthetic appreciation


  ⭐ UNIQUE REWARD — tied to the OTHER person's state:

    Every other pattern: reward comes from OWN body-state improving
    Nurturing:           reward comes from the OTHER person's body-state improving

    → "Seeing your child eat well and feeling satisfied yourself"
      = Self-Pattern-Modeling detects target's improvement
      → prediction-delta POSITIVE → body-feedback PLEASANT
    → Unique: reward arrives VIA simulation of another's state improvement

    → Bond-Architecture.md v2.0: parent→child bond = Entity-Compiled
      reusing the same mechanism, but with oxytocin-vasopressin dominant ratio


  CONFIRMING CASES:

    Mother caring for her own child: all 4 factors MAX + hardware-subsidy (oxytocin)

    Adults wanting to hold a stranger's baby:
      → Baby schema hardware trigger, NO Entity-Compiled bond needed
      → = PROOF that nurturing ≠ connection (bond is not required)

    Deep pet ownership: Vulnerability HIGH + perceived ability EXTREMELY HIGH
      (the pet is 100% dependent)

    An injured animal on the road:
      → Perceived ability HIGH → stop to help → sense of relief
      → Perceived ability LOW → turn away → CUTTING INPUT (defense mechanism)
      → = The clearest demonstration of perceived ability modulation
```

## §5 — THE "GIVING" PATTERN

```
🟡 "GIVING" = THE NATURAL RESOURCE DISTRIBUTION MECHANISM:

  DEFINITION:
    Giving = the drive to share resources (material, knowledge, action)
    that emerges when: OWN body-base is SUFFICIENT + Self-Pattern-Modeling
    detects SOMEONE ELSE is lacking + reward from sharing > reward from keeping.


  CORE MECHANISM:

    Own body-base is already FULL → giving MORE to myself: reward ≈ 0 (satiation)
    SOMEONE ELSE is LACKING → share → they improve →
      Self-Pattern-Modeling detects improvement → prediction-delta → body feels PLEASANT

    → THE SAME MANGO:
      I eat one more (already full): reward ≈ 0 (cyclic satiation — Gap-Body-Need v1.0)
      I give it to someone hungry: reward for ME (via Self-Pattern-Modeling) + reward for THEM
    → = The item generates GREATER TOTAL REWARD when distributed


  ⭐ "GIVING" IS NEVER COMPLETELY SELFLESS — 4 REWARD SOURCES:

    ① SELF-PATTERN-MODELING REWARD — seeing the recipient improve → body becomes pleasant
    ② STATUS REWARD — "I am the giver" → Costly Signal (Obligation.md v1.2)
    ③ CONNECTION REWARD — sharing → bond strengthens → future access (reciprocal)
    ④ SCHEMA REWARD — "good person" identity confirmed → satisfaction

    → = "Selfless giving" is actually many reward sources firing SIMULTANEOUSLY
    → = This is NOT bad — it is OPTIMAL DESIGN (evolutionary)
    → Protect.md v1.2 §7: giving dynamic = natural COUNTERBALANCE to protect


  ⭐ 3 VIOLATION TESTS — PROVING "GIVING" IS DRIVEN BY THE REWARD SYSTEM:

    🟡 If giving were truly selfless → these 3 tests would NOT have any effect.
    But in reality: EVERY case stops when any of the 3 is violated:

    ① OWN BODY-BASE IS INSUFFICIENT:
       → Giving from surplus = pleasant. Giving from deficit = PAINFUL.
       → Exception for justified override: mother going hungry so her child can eat
         — hardware-subsidy Level 0 (kin-bonding) overrides

    ② CORE SCHEMA IS VIOLATED:
       → Donating to charity → discovers it's fraudulent → STOPS IMMEDIATELY
       → Donating to a temple → the monk breaks a vow → STOPS IMMEDIATELY
       → = Schema drive is broken → drive DISAPPEARS

    ③ RECIPROCITY = 0 SUSTAINED:
       → Gives repeatedly → receives nothing in return → body compiles: "no reciprocate"
       → Connection reward DECREASES → total reward < cost → STOPS
       → 🟢 Reciprocal altruism (Trivers 1971): non-reciprocators are excluded


  SPECTRUM — HEALTHY TO PATHOLOGICAL:

    HEALTHY: body sufficient + share surplus → reward > 0, cost ≈ 0

    OVERRIDE A (justified): mother skips a meal so her child can eat
      → hardware priority → HAS a natural ceiling

    OVERRIDE B (schema hijack): donating until financially ruined — "it will bring blessings"
      → NO ceiling → HARMFUL
      → = Bug: mechanism is good for the group, but harmful to the individual

    → Boundary: A has a hardware ceiling, B does not → B = dangerous territory
```

## §7 — VIOLATION & RECOVERY

```
🟡 WHEN THE PATTERN BREAKS — domain reality CONFLICTS STRONGLY with compiled schema:

  DEFINITION:
    Violation = a domain event that CONTRADICTS a compiled pattern,
    with sufficient magnitude to force a schema update.

    → A close friend talks behind your back (connection pattern violated)
    → A religious figure violates their vow (schema "they are holy" violated)
    → A parent abandons their child (schema "parent = safety" violated)


  ⭐ IMPACT = INVESTMENT × VIOLATION SEVERITY:

    🟡 Not every violation carries equal weight:

    Investment HIGH × Violation SEVERE = ⭐ CRISIS
    Investment LOW  × Violation SEVERE = cut ties, move on
    Investment HIGH × Violation MILD   = absorb, adjust
    Investment LOW  × Violation MILD   = ignore

    ┌─────────────────────┬─────────────────────┬──────────────────────┐
    │                     │ Violation MILD       │ Violation SEVERE     │
    ├─────────────────────┼─────────────────────┼──────────────────────┤
    │ Investment LOW      │ Ignore               │ Cut ties, move on    │
    ├─────────────────────┼─────────────────────┼──────────────────────┤
    │ Investment HIGH     │ Absorb, adjust       │ ⭐ CRISIS            │
    └─────────────────────┴─────────────────────┴──────────────────────┘


  ⭐ 3 RESPONSES TO CRISIS (high investment × severe violation):

    ① RECALIBRATE — update schema for THIS SPECIFIC entity:
       → "Friend A betrayed me → A is NOT trustworthy, but friends in general can be"
       → = Adjust Entity-Compiled for that entity, preserve the overall pattern
       → HEALTHY but HARD: requires PFC budget + manageable cortisol + time
       → = "Maturity" = the capacity to recalibrate after violation

    ② OVERGENERALIZE — extend the violation to AN ENTIRE CATEGORY:
       → "Friend A betrayed me → ALL MEN/WOMEN are like that → I trust NO ONE"
       → Violation from 1 entity → generalized into a pattern FOR THE WHOLE TYPE
       → Evolutionary safety: false positive (trusting no one) = only missed opportunities
       → = Prejudice, trust issues = overgeneralization
       → 🟢 Negativity bias (Baumeister 2001): negative generalizes faster than positive
       → Same mechanism as §3 Enemy schema inheritance

    ③ COLLAPSE — HIGH-LEVEL schema is broken → worldview shakes:
       → "A monk violated their vow → RELIGION is a lie → I believe in nothing"
       → "My parent abandoned me → FAMILY is unsafe → the world is dangerous"
       → = Violation strikes at the ANCHOR SCHEMA (Anchor-Schema.md) → cascade
       → MOST DANGEROUS but RARE (requires violation at a very high schema level)
       → Anchor-Schema-Extreme-Example.md: excessive anchor + domain check skipped


  SCHEMA LEVEL DETERMINES IMPACT:

    ┌──────────────────────┬────────────────────────────────────────────┐
    │ Schema Level         │ Violation impact                           │
    ├──────────────────────┼────────────────────────────────────────────┤
    │ Specific Agent       │ Only affects relationship with A           │
    │ (lowest level)       │ Recovery: relatively fast                  │
    ├──────────────────────┼────────────────────────────────────────────┤
    │ Category             │ Affects ALL relationships within category   │
    │                      │ Recovery: slow, needs positive counter-evid.│
    ├──────────────────────┼────────────────────────────────────────────┤
    │ Organization/Domain  │ Affects all trust in the entire domain     │
    │                      │ Recovery: very slow, may be permanent       │
    ├──────────────────────┼────────────────────────────────────────────┤
    │ Worldview            │ Affects ALL domain interactions             │
    │ (highest level)      │ Recovery: extremely difficult, needs new    │
    │                      │   Anchor-Schema                            │
    └──────────────────────┴────────────────────────────────────────────┘

    → Schema level × Domain Types (Domain.md v2.0):
      Reality Domain violation: immediate feedback → recalibrate EASIER
      Abstract-Dynamic Domain violation: delayed/ambiguous feedback
        → overgeneralization EASIER, recovery HARDER
      → = Domain type affects recovery difficulty


  CASES:

    Romantic betrayal:
      Investment HIGH + Violation SEVERE
      → Common: overgeneralize ("I can't love again")
      → Healthy: recalibrate ("THIS PERSON wasn't right for me; love is still possible")
      → Entity-Compiled: grief Type A (active missing) + Type B (identity vacuum)

    Religious figure violating their vow:
      Anchor schema "this person is holy" broken → cascade risk HIGH
      "Blessings" CANNOT be body-confirmed (Abstract-Dynamic Domain) → stuck
      Dual Check (Domain.md v2.0): body smoothing is active, domain does NOT confirm

    Donation + religious figure violation (STACKED):
      "Giving" pattern BROKEN (§5) + faith schema BROKEN = DOUBLE violation
      → Recovery extremely difficult because 2 patterns broken SIMULTANEOUSLY


  RECOVERY PATHS:

    ① Rebuild WITH THE SAME entity (recalibrate):
       → Entity-Compiled chunks update, not erased
       → Rebuild is SLOWER than the original build (trust is scarred)
       → "A repaired crack" — functional but the scar remains in the schema

    ② Rebuild WITH A NEW entity:
       → Overgeneralized schema CAN BLOCK this ("too scared to love again")
       → Requires: new domain exposure + time + positive counter-evidence

    ③ Rebuild at a HIGH SCHEMA level (after collapse):
       → Requires a NEW ANCHOR (Anchor-Schema.md) — an entity/experience
         strong enough to reground
       → "1 anchor person is enough" = 1 genuinely trustworthy person →
         rebuilds "the world has good people" from a collapsed worldview
       → EXTREMELY SLOW + requires long-term consistency

## §8 — HONEST ASSESSMENT

```
  ESTABLISHED (🟢):

    🟢 Negativity bias (Baumeister 2001, Rozin & Royzman 2001):
       negative events carry stronger impact — supports enemy formation asymmetry
    🟢 Baby Schema (Lorenz 1943, Glocker 2009):
       nucleus accumbens fires for infant faces, even in non-parents
    🟢 Facial EMG mimicry (Dimberg 2000):
       automatic, pre-conscious, 300ms
    🟢 In-group empathy bias (Xu 2009):
       ACC response stronger for in-group members
    🟢 Reciprocal altruism (Trivers 1971):
       cooperation = optimal long-term strategy
    🟢 Kin selection (Hamilton 1964):
       helping gene-carriers = helping one's own genes survive
    🟢 Attachment theory (Bowlby 1969):
       connection = innate drive, proximity-seeking
    🟢 Social pain = physical pain (Eisenberger 2003):
       rejection activates the same neural pathways


  FRAMEWORK SYNTHESIS (🟡):

    🟡 "Emergent patterns = compiled valence relationships"
       — Logically consistent with attachment theory, schema therapy
    🟡 "Enemy forms FASTER than connection"
       — Supported by negativity bias research (🟢)
    🟡 "Dehumanization = agent→object reclassification → Self-Pattern-Modeling OFF"
       — Consistent with Haslam 2006, Leyens 2000
    🟡 "Nurturing = 4 factors multiplied"
       — Each factor individually documented; multiplicative model = synthesis
    🟡 "'Giving' is not selfless — 4 reward sources"
       — Consistent with warm glow (Andreoni 1990), costly signaling theory
    🟡 "3 violation tests" — if truly selfless, tests would have no effect. They do.
    🟡 "Violation impact = investment × severity"
       — Well established in relationship science
    🟡 "3 responses (recalibrate / overgeneralize / collapse)"
       — Consistent with schema therapy, Janoff-Bulman 1992
    🟡 "Schema level → violation impact (4-level table)"
       — Logical: higher schema = more dependent schemas below it


  HYPOTHESIS (🔴):

    🔴 "Object-as-proxy enemy" (a bush → container for a potential agent)
       — Logically sound but specific mechanism not yet mapped
    🔴 "Schema inheritance → enemy → prejudice" (full causal chain)
       — Each step has evidence; full chain = hypothesis
    🔴 "4 factors are MULTIPLICATIVE (not additive)" — could be additive or more complex
    🔴 "Collapse → needs a new Anchor" — consistent with therapy practice;
       "1 person is enough" = not empirically verified
    🔴 "Domain type affects recovery difficulty" — logical but untested


  ⚠️ OVERALL:
    v2.0 condensed v1.2 (~1,302L → ~560L). ~60% superseded → cross-referenced.
    REMAINING = unique content: Enemy + Nurturing + Giving + Violation.
    v7.8 aligned: Self-Pattern-Modeling, Entity-Compiled, signal strength.
    NEW in v2.0: Domain Types × violation recovery (§7).
    Strongest sections: Enemy (negativity bias 🟢), Violation (relationship science 🟢).
    Weakest: nurturing multiplication model, collapse recovery hypothesis.
