## §3 — ENEMY / THREAT PATTERN

```
🟡 ENEMY = MẶT ĐỐI DIỆN CỦA CONNECTION:

  Connection: sustained mutual POSITIVE valence → bond
  Enemy:      sustained NEGATIVE valence → avoid / fight / destroy

  NHƯNG enemy pattern CÓ ĐẶC ĐIỂM RIÊNG — KHÔNG phải "connection ngược":


  DEFINE:

    Enemy Pattern = compiled negative valence relationship
    giữa body và domain entity (agent HOẶC object),
    khi entity LIÊN TỤC ảnh hưởng NEGATIVE tới body channels.

    → Agent enemy: người đe dọa, đối thủ, kẻ hại
    → Object enemy: chất độc, bẫy, vũ khí (hướng về phía mình)
    → Object-as-proxy: bụi cây chiến trường, gói bưu phẩm lạ
      (object + schema "agent nguy hiểm có thể ẩn bên trong")


  ⭐ BẤT ĐỐI XỨNG QUAN TRỌNG — Enemy hình thành NHANH hơn Connection:

    🟢 Negativity bias (Baumeister 2001, Rozin & Royzman 2001):
      → Negative events ảnh hưởng MẠNH hơn positive events cùng magnitude
      → 1 lần bị phản bội > 10 lần giúp đỡ (về mặt valence impact)
      → = Evolutionary logic: miss 1 threat = CHẾT. Miss 1 friend = chỉ thiếu

    Connection: cần NHIỀU lần positive → dần dần emerge
    Enemy: có thể chỉ 1 lần negative MẠNH → NGAY LẬP TỨC compile

    Connection: mất NHANH khi 1 lần violation lớn
    Enemy: mất CHẬM — cần NHIỀU positive để override (nếu có thể)

    → = Asymmetric cost: false negative enemy = chết. False positive = chỉ tốn energy.
    → = Não THIẾT KẾ để enemy pattern NHẠY hơn connection pattern.


  CÁC DẠNG ENEMY PATTERN:

    ① AGENT ENEMY — sustained negative valence từ agent:
       → Kẻ bắt nạt: L0 threat + L2 humiliation + L3 status damage
       → Sếp toxic: L1 power over livelihood + L2 stress + autonomy--
       → Kẻ thù chiến trường: L0 THREAT CỰC CAO → kill or be killed

       Đặc biệt agent enemy:
         → Agent CÓ mục tiêu riêng → unpredictable → khó tránh
         → Agent BIẾT mình đang tránh → adjust strategy → arms race
         → = Agent enemy NGUY HIỂM HƠN object enemy vì adaptive

    ② OBJECT ENEMY — sustained negative valence từ object:
       → Chất độc, lửa, độ cao: L0 threat compiled từ experience/schema
       → Vũ khí hướng về mình: dao trong tay kẻ thù, bom
       → "Gián" / "chuột": L2 disgust compiled (schema inheritance từ culture)

       Object enemy = DỄ TRÁNH hơn agent:
         → Object = deterministic, không adaptive
         → Biết quy tắc (lửa = nóng) → tránh = xong
         → NHƯNG: schema CÓ THỂ khuếch đại (phobia = object enemy quá mức)

    ③ OBJECT-AS-PROXY — object được reclassify vì agent schema:
       → Bụi cây chiến trường: object → "có thể có agent ẩn" → L0 THREAT
       → Gói bưu phẩm lạ: object → "có thể là bom" → L0 THREAT
       → Bẫy gai, mìn: object MANG Ý ĐỒ agent → valence từ agent, dạng object
       → = Schema reclassify (agent ↔ object override)

       Case bụi cây Việt Cộng:
         → Bình thường: bụi cây = object, Logic mode, valence neutral
         → Chiến trường: schema "VC ẩn trong bụi" active →
            RECLASSIFY → potential agent container →
            SWITCH Modeling mode → L0 THREAT cực cao
         → = CÙNG entity, KHÁC schema → KHÁC hoàn toàn pattern


  ⭐ DEHUMANIZATION — reclassify agent → object để TIÊU DIỆT:

    🟡 Vấn đề: agent enemy → mirror VẪN fire (vì classified = agent)
      → Mirror fire → dissonance khi gây hại cho họ
      → = RÀO CẢN tự nhiên chống bạo lực (mirror mechanism)

    Giải pháp (từ góc nhìn evolution): RECLASSIFY agent → object
      → Object = không có state → mirror KHÔNG fire
      → = Có thể tiêu diệt MÀ KHÔNG bị mirror dissonance

    Cases:
      → "Gooks", "cockroaches" (Rwanda) = ngôn ngữ dehumanize
        → Relabel agent → vermin/object → bypass mirror
      → "Beef": con bò → thịt (agent → object reclassify)
        → Agent → object → eat without mirror guilt
      → Bác sĩ phẫu thuật: patient → "ca mổ" → Object-Logic mode
        → CHỨC NĂNG: giảm mirror → tập trung kỹ thuật (healthy context)

    → Dehumanization = Object-Agent OVERRIDE NGƯỢC (agent→object)
    → Mục đích: tắt mirror → cho phép hành động mà mirror sẽ block
    → = Cùng mechanism, KHÁC moral context:
      Bác sĩ: override TẠM THỜI, mục đích GIÚP = healthy
      Chiến tranh: override TOÀN DIỆN, mục đích DIỆT = dangerous


  ENEMY PATTERN × SCHEMA INHERITANCE:

    🟡 Enemy pattern CÓ THỂ TRUYỀN qua schema KHÔNG cần trải nghiệm:
      → Cha mẹ dạy: "rắn = nguy hiểm" → con compile enemy pattern MÀ CHƯA THẤY rắn
      → Văn hóa dạy: "nhóm X = kẻ thù" → compile TRƯỚC KHI gặp ai thuộc nhóm X
      → = Schema inheritance = nhanh nhất nhưng ÍT chính xác nhất
        (Valence-Propagation.md)

    Risk: enemy pattern inherited → OVERGENERALIZE:
      → "1 người nhóm X hại tôi" → compile "TOÀN BỘ nhóm X = enemy"
      → = Cùng mechanism §9 Violation (overgeneralize sau violation)
      → = Prejudice, racism = enemy pattern inherited + overgeneralized
```
