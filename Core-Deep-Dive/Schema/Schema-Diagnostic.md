> **Trạng thái:** DRAFT
> **Ngày:** 2026-03-19
> **Mục đích:** Quy trình THỰC DỤNG để map schema từ nông → sâu cho 1 người cụ thể
> **Reference:** Schema-Atlas.md (gradient body→domain), Body-Needs.md (6 nhóm)
> **Ai dùng:** AI (khi được hỏi), therapist, coach, hoặc TỰ bản thân
> **Nguyên tắc:** Map = ƯỚC LƯỢNG → KHÔNG chính xác tuyệt đối → cần VERIFY liên tục

## 1. Nguyên Lý Gốc

```
Schema = gradient từ body-need (sâu) → domain skill (nông)
  → NÔNG: observable, verbal-accessible, đổi nhanh
  → SÂU: invisible, body-embedded, đổi chậm
  → → Map = bắt đầu từ CÁI THẤY (nông) → suy ngược → CÁI ẨN (sâu)
  → → Giống bác sĩ: thấy TRIỆU CHỨNG (nông) → suy ngược → BỆNH (sâu)

⚠️ GIỚI HẠN:
  → Verbal mô tả ≈ 20% schema thật (multi-modal, unconscious)
  → Map = ước lượng GẦN ĐÚNG, không bao giờ CHÍNH XÁC tuyệt đối
  → Cùng hành vi → nhiều schema sâu KHÁC có thể gây ra
  → → LUÔN verify, LUÔN có thể sai, LUÔN sẵn sàng adjust
```

## 2. Quy Trình 5 Bước

### Bước 1: Thu Thập Schema Nông (Observable)

```
MỤC TIÊU: liệt kê 5-10 HÀNH VI lặp lại (patterns)

CÁCH LÀM:
  Hỏi (hoặc tự hỏi):
    "Hành vi nào bạn LÀM ĐI LÀM LẠI mà không cần nghĩ?"
    "Ở WORK bạn thường thế nào?"
    "Ở NHÀ bạn thường thế nào?"
    "Khi STRESS bạn thường làm gì?"
    "Khi RẢNH bạn thường làm gì?"
    "Điều gì khiến bạn KHÓ CHỊU nhất?"
    "Điều gì khiến bạn VUI nhất?"

  Liệt kê per domain:
    Work:         [hành vi 1] [hành vi 2]
    Relationship: [hành vi 3] [hành vi 4]
    Social:       [hành vi 5] [hành vi 6]
    Alone:        [hành vi 7] [hành vi 8]
    Stress:       [hành vi 9] [hành vi 10]

  Ví dụ output:
    Work:         "im lặng trong họp", "làm thêm giờ dù xong việc"
    Relationship: "nhắn partner nhiều lần/ngày", "nấu ăn cực kỹ"
    Social:       "tránh party", "chỉ đi nếu có bạn thân đi cùng"
    Alone:        "lướt MXH", "ăn nhiều"
    Stress:       "ăn bù", "nằm ngủ"

⚠️ QUAN TRỌNG: thu HÀNH VI, không thu LÝ DO
  → "Tôi im lặng trong họp" = hành vi ✅
  → "Vì tôi sợ nói sai" = lý do → CHƯA thu ở bước này (bước 3 mới hỏi)
  → Vì: lý do có thể là POST-HOC rationalization (PFC bịa) → body gốc khác
```

### Bước 2: Cluster → Tìm Rule Chung (Gradient Giữa)

```
MỤC TIÊU: gom hành vi → tìm PATTERN CHUNG xuyên domain

CÁCH LÀM:
  Nhìn 5-10 hành vi → có gì CHUNG?

  Ví dụ từ bước 1:
    "Im lặng trong họp" + "tránh party" + "chỉ đi nếu có bạn thân"
    → CHUNG: TRÁNH tình huống bị ĐÁNH GIÁ bởi người không quen

    "Nhắn partner nhiều" + "nấu ăn cực kỹ"
    → CHUNG: CỐ LÀM HÀI LÒNG người quan trọng

    "Ăn bù" + "lướt MXH" + "nằm ngủ"
    → CHUNG: TÌM COMFORT nhanh khi stress (dopamine/opioid shortcut)

  → TÌM được 2-3 RULES chung:
    Rule A: "tránh bị đánh giá" (across work + social)
    Rule B: "phải làm hài lòng người quan trọng" (across relationship)
    Rule C: "tìm comfort nhanh khi stress" (across alone + stress)

  → Rules = gradient GIỮA → POINTER tới schema sâu

⚠️ Nếu KHÔNG thấy pattern chung → hành vi có thể KHÔNG liên quan
  → Thu thêm hành vi → hoặc: hành vi đó = schema NÔNG riêng lẻ, không connect sâu
```

### Bước 3: Hỏi "TẠI SAO?" → Tìm Body-Need Gốc (Schema Sâu)

```
MỤC TIÊU: từ rules → drill down → body-need NÀO?

CÁCH LÀM — "TẠI SAO?" chain:

  Rule A: "tránh bị đánh giá"
    → "Tại sao tránh bị đánh giá?"
    → "Vì sợ người ta thấy tôi kém" → interpretation
    → "Tại sao sợ thấy kém?"
    → "Vì... tôi nghĩ mình không đủ tốt" → schema SÂU emerge!
    → Body-need: Nhóm 5 (Meaning: self-worth) hoặc Nhóm 2 (Connection: sợ bị reject)

    Phân biệt bằng câu hỏi thêm:
      "Nếu họ thấy bạn kém → bạn sợ điều GÌ xảy ra?"
      → "Họ sẽ không thích tôi" → Nhóm 2 (Connection) ✅
      → "Tôi sẽ thấy mình vô giá trị" → Nhóm 5 (Meaning) ✅
      → "Tôi sẽ mất việc" → Nhóm 1 (Survival) ✅
      → = Cùng rule → KHÁC body-need gốc tùy người → PHẢI hỏi tiếp

  Rule B: "phải làm hài lòng"
    → "Tại sao phải làm hài lòng?"
    → "Vì sợ họ bỏ" → Nhóm 2 (Connection: abandonment)
    → HOẶC: "Vì phải chứng minh mình tốt" → Nhóm 5 (Meaning: worth through service)

  Rule C: "tìm comfort nhanh"
    → "Tại sao cần comfort ngay?"
    → "Vì cảm giác stress quá chịu không nổi" → body CẦN opioid NGAY
    → = Nhóm 3 (Experience) bù → nhưng GỐC có thể Nhóm 1/2/5 đang thiếu

  Kết quả bước 3:
    Schema sâu CANDIDATE: "tôi không đủ tốt" (Nhóm 2 hoặc 5)
    Body-need: Connection (sợ mất người) VÀ/HOẶC Meaning (sợ mất giá trị)
    → Có thể CẢ HAI → body baseline = tổng hợp

BODY CHECK (quan trọng — xác nhận body level):
  Hỏi: "Khi nghĩ tới 'tôi không đủ tốt', body CẢM GÌ?"
    → "Ngực nặng" → somatic encode ✅ → schema SÂU thật (body-level)
    → "Vai gồng" → somatic encode ✅ → schema SÂU thật
    → "Không feel gì" → có thể: verbal only (chưa sâu) hoặc suppressed (rất sâu)
    → → Nếu "không feel" + hành vi VẪN lặp → suppressed (sâu, bị ẩn)
```

### Bước 4: Predict Gradient → Verify

```
MỤC TIÊU: dùng schema sâu PREDICT hành vi → check ĐÚNG không

CÁCH LÀM:
  Schema sâu: "tôi không đủ tốt" (Nhóm 2+5)

  PREDICT per domain CHƯA HỎI:
    "Khi thử cái MỚI (domain mới), bạn thường thế nào?"
    → Nếu schema đúng → predict: "sợ, chần chừ, hoặc tránh thử"
    → Hỏi → "Đúng! Tôi rất sợ thử cái mới" → MATCH ✅
    → HOẶC: "Tôi thích thử mọi thứ!" → KHÔNG MATCH ❌ → schema sâu CÓ THỂ KHÁC

    "Khi được KHEN, bạn cảm thấy sao?"
    → Nếu "không đủ tốt" đúng → predict: "không tin lời khen, discount"
    → Hỏi → "Ừ, tôi hay nghĩ họ nói xã giao" → MATCH ✅
    → HOẶC: "Vui lắm!" → KHÔNG MATCH → schema sâu có thể khác

  TỔNG HỢP verify:
    5 predictions → 4+ match → schema sâu LIKELY đúng ✅
    5 predictions → 2-3 match → schema sâu CÓ THỂ đúng PHẦN NÀO → adjust
    5 predictions → 0-1 match → schema sâu SAI → quay bước 3 → hỏi lại

⚠️ KHÔNG BAO GIỜ "chắc chắn đúng" — chỉ "likely đúng" → tiếp tục verify qua thời gian
```

### Bước 5: Xác Định Hướng Can Thiệp

```
MỤC TIÊU: biết fix Ở ĐÂU trong gradient

Khi schema sâu ĐÃ ước lượng → fix ĐÚNG LEVEL:

  FIX SÂU (body baseline — hiệu quả nhất, KHÓ nhất):
    → Body-level: exercise, sleep, nutrition (Nhóm 1 Body-Needs)
    → Somatic therapy: body-focused processing
    → Context change dài hạn: môi trường safe + warm
    → Timeline: tháng → năm
    → Effect: MỌI gradient levels TỰ adjust theo body mới

  FIX GIỮA (rules/strategies — hiệu quả vừa, KHÓ vừa):
    → CBT: challenge rules ("phải hoàn hảo" → "đủ tốt là ok")
    → Schema Therapy: identify + challenge deep rules
    → Timeline: tuần → tháng
    → Effect: domains ĐÃ CÓ adjust → domains MỚI cần train riêng

  FIX NÔNG (domain skills — nhanh nhất, ÍT cascade):
    → Skill training: learn kỹ năng domain cụ thể
    → Behavior practice: thử hành vi mới per domain
    → Timeline: ngày → tuần
    → Effect: CHỈ domain đó → domains khác KHÔNG đổi

  THỨ TỰ NÊN:
    → Fix nông TRƯỚC nếu: cần kết quả NGAY (crisis, deadline)
    → Fix sâu TRƯỚC nếu: muốn thay đổi BỀN VỮNG (long-term growth)
    → Ideal: fix sâu + nông ĐỒNG THỜI → body shift + behavior practice
    → = Body-Needs Nhóm 1 (baseline) + domain skill training (cụ thể)
```

## 3. Ví Dụ Map Hoàn Chỉnh

### 3.1 Người A — "Workaholic mà vẫn trống rỗng"

```
BƯỚC 1 — Thu thập hành vi:
  Work:         "Làm 12h/ngày, dù đã xong việc vẫn tìm thêm"
  Relationship: "Ít gặp partner, 'bận quá'"
  Social:       "Chỉ networking, không hangout thật"
  Alone:        "Scroll LinkedIn, check email"
  Stress:       "Làm nhiều hơn"

BƯỚC 2 — Cluster:
  Pattern A: "LUÔN LÀM VIỆC" (work + alone + stress → đều = work)
  Pattern B: "TRÁNH connection thật" (relationship + social → surface only)
  → Rule chung: "làm việc = giá trị, dừng = nguy hiểm"

BƯỚC 3 — Tại sao?
  "Tại sao không dừng được?"
  → "Vì dừng là lãng phí thời gian"
  → "Tại sao lãng phí = xấu?"
  → "Vì... nếu không productive thì tôi vô dụng"
  → Schema sâu: "giá trị = output" → Nhóm 4 (Progress) + Nhóm 5 (Meaning: identity = work)
  Body check: "ngực nặng khi nghĩ nghỉ?" → "Có, lo lắng ngay"
  → Somatic confirm ✅

BƯỚC 4 — Predict + verify:
  Predict: "khi bị ốm phải nghỉ, bạn cảm thấy sao?"
  → "Cực kỳ bứt rứt, tội lỗi" → MATCH ✅ (dừng = mất giá trị)
  Predict: "khi được khen 'bạn giỏi', feel sao?"
  → "Vui 5 phút rồi nghĩ phải giỏi HƠN" → MATCH ✅ (never enough)
  Predict: "nếu tài chính đã đủ, vẫn làm 12h?"
  → "Vẫn!" → Confirm: KHÔNG phải Nhóm 1 (survival) → Nhóm 4+5 ✅
  → 3/3 match → schema sâu LIKELY đúng ✅

BƯỚC 5 — Hướng fix:
  Sâu: fix "giá trị = output" → "giá trị = nhiều thứ, output chỉ 1 phần"
       Body: exercise (burn cortisol) + sleep + rebuild Nhóm 2 (connection thật)
  Giữa: challenge rule "dừng = lãng phí" → "nghỉ = recover → productive HƠN"
  Nông: practice "1 tối/tuần không làm gì" → tolerate → dần mở rộng

  → Xem thêm: Drive-Optimization.md (Pattern B → C)
```

### 3.2 Người B — "Dễ thương nhưng hay bị lợi dụng"

```
BƯỚC 1 — Thu thập hành vi:
  Work:         "Nhận thêm việc người khác nhờ, dù đã quá tải"
  Relationship: "Luôn nhường, không bao giờ nói 'không'"
  Social:       "Giúp mọi người trước, quên bản thân"
  Alone:        "Mệt, kiệt sức"
  Stress:       "Khóc 1 mình, không kể ai"

BƯỚC 2 — Cluster:
  Pattern: "LUÔN đặt người khác TRƯỚC mình" (work + relationship + social)
  Pattern: "KHÔNG bao giờ express khó chịu" (stress → khóc 1 mình)
  → Rule: "nhu cầu người khác > nhu cầu mình"

BƯỚC 3 — Tại sao?
  → "Tại sao không nói 'không'?"
  → "Vì sợ họ buồn... sợ họ không thích mình nữa"
  → Schema sâu: "nếu không hữu ích → bị bỏ" → Nhóm 2 (Connection: conditional love)
  Body check: "khi nói 'không', body cảm gì?"
  → "Tim đập nhanh, sợ" → somatic confirm ✅
  → Gốc có thể: tuổi thơ, tình thương bố mẹ CÓ ĐIỀU KIỆN ("ngoan = yêu")

BƯỚC 4 — Predict + verify:
  Predict: "khi ai đó giúp BẠN, cảm thấy sao?"
  → "Ngại, tội lỗi, sợ mang ơn" → MATCH ✅ (nhận = nợ, vì "phải hữu ích")
  Predict: "khi 1 mình, có thấy bình an không?"
  → "Có, nhẹ hẳn" → MATCH ✅ (1 mình = không ai cần phải please)
  → 2/2 match ✅

BƯỚC 5 — Hướng fix:
  Sâu: fix "hữu ích mới được yêu" → "tôi ĐÁNG được yêu DÙ không hữu ích"
       Body: oxytocin từ NHẬN (not just cho) — practice NHẬN giúp đỡ
  Giữa: practice "nói 'không' 1 lần/tuần" → body learn: "nói không ≠ bị bỏ"
  Nông: specific scripts per domain: "cám ơn nhưng tôi bận" (work skill)
```

## 4. Ngoại Lệ + Cảnh Báo

```
NGOẠI LỆ 1: Schema sâu ĐÚNG nhưng gradient KHÁC predicted
  → 2+ schema sâu TƯƠNG TÁC → gradient = emergent → predict khó
  → Fix: thu thêm hành vi → tìm schema sâu THỨ 2 → adjust model
  → Ví dụ: "không đủ tốt" + "cố cũng vô ích" = 2 schemas → gradient = BỎ CUỘC
    (predict "phải hoàn hảo" nhưng thực tế "mặc kệ luôn" = 2 schemas đối nghịch)

NGOẠI LỆ 2: Cùng schema sâu + KHÁC hardware → KHÁC gradient
  → DRD4 7-repeat + "không đủ tốt" = chuyển domain liên tục (scatter)
  → COMT Met/Met + "không đủ tốt" = perfectionism sâu trong 1 domain (stuck)
  → → Hardware profile ẢNH HƯỞNG gradient → cần ước lượng hardware NỮA
  → Chi tiết: PFC-Analysis.md §8 (Hardware Receptor Variants)

NGOẠI LỆ 3: Schema giữa ĐÃ compiled → TRÔNG NHƯ schema sâu
  → "Phải hoàn hảo" = rule (bước 3) nhưng compiled (BG automatic) → trông sâu
  → Test: therapy fix rule → body CŨNG đổi? → đúng sâu
                            → body KHÔNG đổi? → chỉ compiled rule, sâu HƠN vẫn ẩn
  → = COMPILED DEPTH ≠ BODY DEPTH → phải phân biệt

⚠️ CẢNH BÁO TỔNG QUÁT:
  → Map = ƯỚC LƯỢNG → có thể SAI → luôn verify
  → AI/self-diagnosis = BẮT ĐẦU → nhưng KHÔNG thay thế therapist cho trauma sâu
  → Schema sâu body-level = cần BODY intervention → verbal alone KHÔNG ĐỦ
  → File này cho HƯỚNG → không cho CHẨN ĐOÁN → "compass, not GPS"
```

## 5. Template Cho AI — Khi Được Hỏi "Phân tích tôi"

```
Khi ai đó thả file này + framework files → hỏi AI "phân tích tôi":

  BƯỚC 1: Hỏi hành vi per domain (5 câu hỏi §2 Bước 1)
  BƯỚC 2: Cluster hành vi → tìm 2-3 rules chung
  BƯỚC 3: Hỏi "tại sao?" chain → narrow body-need nhóm
  BƯỚC 4: Predict gradient → đặt câu hỏi verify → check match
  BƯỚC 5: Suggest hướng fix (sâu + giữa + nông)

  ⚠️ AI PHẢI nói rõ:
    "Đây là ƯỚC LƯỢNG dựa trên verbal — có thể sai"
    "Body-level cần bạn TỰ CẢM NHẬN hoặc therapist hỗ trợ"
    "Nếu prediction KHÔNG match → hỏi thêm, KHÔNG ép fit"

  ⚠️ AI KHÔNG NÊN:
    ❌ "Schema sâu của bạn LÀ X" (overclaim)
    ❌ Diagnose bệnh lý (không phải therapist)
    ❌ Ép fit framework khi data không match

  AI NÊN:
    ✅ "Dựa trên hành vi bạn mô tả, CÓ THỂ schema sâu thuộc Nhóm X"
    ✅ "Để xác nhận, bạn thử check: [câu hỏi verify]"
    ✅ "Nếu match → likely đúng. Nếu không → cần điều chỉnh"
```

## 5.5 Bổ Sung: Ước Lượng Calibrate Level

```
Khi diagnostic 1 người, NÊN ước lượng thêm: calibrate level khi switch context.

  → Body baseline (pressure nền, challenge nền, hormone quen) MANG THEO khi switch domain
  → Calibrate = khả năng ADJUST baseline khi context thay đổi
  → 4 levels (0=không biết → 3=tự động) → chi tiết: Threshold-Analysis.md §3.4

  Cách ước lượng:
    Hỏi: "Khi từ công ty về nhà, bạn cảm thấy thế nào?"
      → "Bình thường, thoải mái ngay" → Level 2-3 (calibrate tốt)
      → "Vẫn nghĩ về công việc, khó relax" → Level 0-1 (carry baseline)
      → "Cần 30 phút đi bộ mới thoải mái" → Level 2 (có routine, cần effort)

    Quan sát: hành vi KHÁC giữa contexts?
      → Rất khác (formal ở cty, thoải mái ở nhà) → calibrate TỐT
      → Giống nhau (căng thẳng ở CẢ 2) → calibrate KÉM → baseline carry mạnh

  Giá trị diagnostic:
    → Calibrate kém → predict: hành vi GIỐNG ở mọi context (1 mode cho tất cả)
    → Calibrate tốt → predict: hành vi KHÁC per context (adaptive)
    → Và: calibrate level CÓ THỂ TRAIN → recommend routine nếu level thấp
```

## 6. Câu Hỏi Mở

```
SD1: Bao nhiêu hành vi (bước 1) là ĐỦ để cluster reliable?
     → Hiện tại: 5-10 → có thể cần nghiên cứu optimal number

SD2: "Tại sao?" chain bao nhiêu bước mới đến gốc?
     → Ước lượng 3-5 "tại sao?" → có thể KHÁC tùy người (avoidant → nhiều hơn)

SD3: Verify accuracy rate bao nhiêu?
     → Chưa test thực tế → cần: pilot 10-20 người → đo prediction match rate

SD4: AI vs Therapist — accuracy KHÁC bao nhiêu?
     → AI: verbal only → miss somatic. Therapist: verbal + observe body.
     → Predict: therapist accuracy > AI → nhưng AI = starting point tốt

SD5: Self-diagnostic bias?
     → Tự phân tích mình → bias CAO (blind spots)
     → Cần: external check (friend, therapist, AI) để verify
```
