## §0 — MỤC ĐÍCH + VỊ TRÍ NOVELTY TRONG FRAMEWORK

```
NOVELTY TRONG 5-LAYER:

  L0 — Autonomic (tim đập, thở)
  L1 — Body-Base (đói, khát, nhiệt, ngủ)
  L2 — Sensory-Motor (cảm giác, vận động)
  L3 — Pattern (Novelty / Status / Protect) ← NOVELTY Ở ĐÂY
  L4 — PFC (imagine, simulate, meta-cognition)

  L3 = 3 drives xã hội, MỌI động vật xã hội đều có:
    Novelty  = EXPAND  — mở rộng, khám phá, cải thiện
    Status   = MAP     — vị trí trong nhóm, được công nhận
    Protect  = DEFEND  — bảo vệ bản thân, người thân, lãnh thổ

  Novelty = drive KÉO (pull toward new)
  Protect = drive ĐẨY (push away from threat)
  Status  = drive ĐỊNH VỊ (where am I in the group)

  → 3 drives KHÔNG loại trừ nhau — chạy ĐỒNG THỜI
  → Tỉ lệ drive nào MẠNH hơn = tùy context + hardware + chunks tích lũy


NOVELTY = GÌ?

  ĐỊNH NGHĨA:
    Novelty = cơ chế DETECT + RESPOND to BIẾN ĐỘNG
    so với trạng thái hiện tại (baseline)

    = Não PHÁT HIỆN cái gì ĐÃ THAY ĐỔI → signal → attend → explore

  KHÔNG PHẢI:
    ✗ "Thích cái mới" (pop psychology — quá đơn giản)
    ✗ Personality trait "tò mò" (trait theory — mô tả, không giải thích)
    ✗ Chỉ ở PFC / ý thức (tồn tại ở CẢ vô thức lẫn ý thức)
    ✗ Lúc nào cũng tốt (mechanism TRUNG TÍNH — có thể có lợi hoặc có hại)

  LÀ:
    ○ Brain mechanism — mọi người đều có
    ○ Habituation-based — detect THAY ĐỔI so với baseline, không phải "cái mới tuyệt đối"
    ○ VTA-driven — VTA = bộ phận detect, dopamine = signal
    ○ Threshold-filtered — DRD4 receptor quyết định "biến động nào ĐỦ LỚN để báo lên"
    ○ Multi-layer — hoạt động ở cả L2 (cảm giác), L3 (pattern), L4 (imagine)


CƠ CHẾ DETECT: VTA — "SEISMOGRAPH" CỦA NÃO

  VTA (Ventral Tegmental Area):
    → ~400,000 neurons ở midbrain
    → Chức năng: detect DELTA (biến động), KHÔNG phải absolute values
    → = "Seismograph" — đo RUNG CHẤN, không đo độ cao mặt đất

  Cơ chế (ĐƠN GIẢN hơn prediction-error theory):
    → Schema compiled → fire ỔN ĐỊNH → VTA quen (habituate) → IM LẶNG
    → Neurons mới sync (pattern mới/mở rộng) → RUNG KHÁC → VTA fire
    → Trở lại ổn định ở level mới → VTA habituate lại → IM LẶNG

    = "baseline → detect change → fire → new baseline → habituate"
    = KHÔNG CẦN cơ chế "predict → compare → error" phức tạp
    = Đơn giản: "quen → phát hiện khác-với-quen → fire"

  VTA fire → dopamine production (via tyrosine pathway)
  Dopamine travel qua axons (cable VẬT LÝ, hình thành từ thai nhi, CỐ ĐỊNH)
  → Chỉ LƯỢNG dopamine thay đổi, không phải topology kết nối
  → Dopamine bind vào DRD4 receptors trên PFC neurons

  🟢 VTA, dopamine pathway, habituation = neuroscience verified


DRD4 THRESHOLD — FILTER VÔ THỨC (KHÔNG PHẢI giới hạn PFC)

  ⚠️ CLARIFICATION QUAN TRỌNG (xem Core-v7.5-Draft.md §1.5):

  Threshold LỚN/NHỎ = ở VTA→DRD4 level (VÔ THỨC)
  = Filter "biến động nào ĐỦ LỚN để BÁO LÊN PFC"

  PFC khi ĐÃ FOCUS = xử lý MỌI chi tiết tinh vi
  → PFC KHÔNG bị giới hạn bởi DRD4
  → PFC bị giới hạn bởi CÁI GÌ ĐƯỢC BÁO VỀ (input filter)

  Ví dụ:
    → 7R (threshold CAO): VTA chỉ báo biến động LỚN
      → PFC ÍT bị interrupt → deep focus tốt hơn
      → Khi nhận signal = chunk LỚN → reward LỚN per detect

    → 4R (threshold THẤP): VTA báo MỌI biến động nhỏ
      → PFC LIÊN TỤC bị interrupt → biết nhiều, khó deep
      → Mỗi signal = chunk NHỎ → reward NHỎ per detect, nhưng LIÊN TỤC

  → KHÔNG PHẢI "7R sáng tạo hơn" hay "4R tò mò hơn"
  → = KHÁC DẠNG processing: depth (7R) vs breadth (4R)
  → Cả 2 đều CÓ Novelty → filter KHÁC → experience KHÁC

  🟢 DRD4 receptor variants = neuroscience verified
  🟡 Framework interpretation: threshold → depth/breadth distinction = derived


KẾT NỐI:
  → File NÀY = Novelty BẢN THÂN NÓ — define, mechanism, types, interactions
  → Novelty-Loop.md = khi Novelty + Imagine tạo VÒNG LẶP không tự dừng
  → Flow: hiểu Novelty (file này) → hiểu tại sao nó CÓ THỂ loop (file kia)

## §1 — 2 DẠNG NOVELTY: VÔ THỨC vs IMAGINE

```
TẠI SAO PHÂN BIỆT?

  Novelty KHÔNG CHỈ CÓ 1 DẠNG.
  Phân biệt 2 dạng giúp hiểu:
    → Tại sao xem MXH (scroll) ≠ nghiên cứu Einstein (imagine)
    → Tại sao 1 cái TỰ DỪNG, 1 cái KHÓ DỪNG
    → Tại sao cùng Novelty drive nhưng OUTPUT hoàn toàn khác

  2 dạng:
    ① Novelty Vô Thức (external-driven) — input từ BÊN NGOÀI
    ② Novelty từ Imagine (internal-driven) — input từ BÊN TRONG (PFC + vô thức)


═══════════════════════════════════════════════════════
① NOVELTY VÔ THỨC (External-Driven)
═══════════════════════════════════════════════════════

  = Phản ứng với input MỚI từ environment
  = VTA detect biến động từ giác quan → dopamine → attend → explore

  FLOW:
    External input (mắt, tai, da, mũi, lưỡi)
    → Neurons fire pattern MỚI (khác baseline)
    → VTA detect DELTA → fire
    → Dopamine → DRD4 filter → PFC attend (nếu vượt threshold)
    → Explore (tiếp cận, tương tác, thử)
    → Chunks tích lũy → pattern quen dần
    → VTA habituate → dopamine GIẢM
    → Chán → dừng → tìm thứ khác

  GIỚI HẠN TỰ NHIÊN — TẠI SAO TỰ DỪNG:
    → Cơ chế 1: HABITUATION
      → External input LẶP LẠI → VTA quen → im → Novelty tắt
      → = Input HẾT MỚI → VTA không còn detect delta → dừng

    → Cơ chế 2: BODY-BASE SATISFACTION
      → Body-base: "đủ rồi, dừng" (leptin, insulin, CCK,...)
      → Khi nhu cầu ĐÃ ĐÁP ỨNG → body dừng drive → Novelty tắt
      → Dù external CÒN input → body nói "đủ" = dừng

    → Cơ chế 3: INPUT DEPLETION
      → Environment có HẠN → input mới cạn → VTA im → dừng
      → Ví dụ: chỉ có 5 con cua → xem hết 5 → hết input

    → 3 cơ chế này = BUILT-IN BRAKE
    → Novelty vô thức HIẾM KHI kéo dài quá VÀI GIỜ cho 1 stimulus
    → Cycle: phút → giờ (exception: environment CỰC KỲ phong phú)


  ── VÍ DỤ 1: ĐỒ ĂN ──

    Tình huống A — Đồ ăn bình thường:
      → Đói (Body-Dissonance: thiếu dinh dưỡng)
      → Kiếm đồ ăn → ăn
      → Vị bình thường → VTA: delta NHỎ (quen rồi) → ít reward
      → No (Body-Satisfaction: leptin, insulin) → DỪNG
      → = Novelty vô thức BOUNDED bởi body-base satisfaction
      → Cycle: 10-30 phút

    Tình huống B — Đồ ăn NGON (lần đầu):
      → Ăn → vị MỚI, texture MỚI → VTA: delta LỚN → dopamine MẠNH
      → Body-Reward: melody mượt VƯỢT mong đợi → opioid
      → Tiếp tục ăn + thưởng thức (Novelty + Reward cùng fire)
      → No → Body-Satisfaction → DỪNG (dù còn ngon)
      → = Body-base satisfaction OVERRIDE Novelty reward

    Tình huống C — Đồ ăn ngon ĂN NHIỀU LẦN:
      → Lần 1: WOW → dopamine mạnh
      → Lần 3: ngon nhưng... bớt wow → dopamine giảm
      → Lần 10: quen rồi → VTA habituate → gần như không novelty
      → = HABITUATION — cùng input, VTA quen, delta → 0
      → Vẫn ngon (Body-Reward còn) nhưng KHÔNG CÒN MỚI (Novelty tắt)
      → = Phân biệt: Reward ≠ Novelty
        → Reward = "melody mượt" (có thể lặp lại)
        → Novelty = "melody KHÁC" (chỉ khi delta > 0)

    ⇒ Pattern: Body-Satisfaction + Habituation = 2 PHANH tự nhiên cho đồ ăn
    ⇒ Tại sao "nghiện đồ ăn" KHÓ HƠN "nghiện ma túy":
       → Đồ ăn: phanh body-base MẠNH (no = dừng) + habituation nhanh
       → Ma túy: HACK phanh body-base (override satisfaction) + novelty nhân tạo


  ── VÍ DỤ 2: CON CUA TRÊN BÃI BIỂN ──

    Đứa trẻ lần đầu thấy con cua:
      → Nhìn: hình dạng LẠ → VTA fire (delta visual)
      → Cua CHẠY NHANH → VTA fire thêm (delta movement)
      → Đuổi theo → cua đổi hướng → VTA fire (delta trajectory)
      → Bắt được → cua kẹp → VTA fire (delta somatic — đau!)
      → Mỗi lần cua làm gì MỚI → VTA detect → dopamine → tiếp tục

    Sau 20-30 phút:
      → Cua chạy? → Quen rồi (VTA habituate)
      → Cua kẹp? → Biết rồi (VTA habituate)
      → Cua trốn? → Quen rồi (VTA habituate)
      → ALL delta → 0 → VTA im → dopamine dừng
      → "Chán" = TÊN GỌI cho VTA habituated + dopamine giảm
      → Quay đi → tìm thứ KHÁC (= tìm delta MỚI)

    Pattern rõ:
      → Novelty vô thức BOUNDED bởi external input depletion
      → Con cua CHỈ CÓ bấy nhiêu behaviors → hết delta → hết novelty
      → Cycle: ~20-60 phút (tùy complexity của stimulus)

    ⇒ Nếu bãi biển có 100 loài KHÁC NHAU → cycle DÀI HƠN
    ⇒ Nhưng vẫn CÓ GIỚI HẠN — vì external input = HỮU HẠN
    ⇒ = Novelty vô thức luôn TỰ DỪNG (sớm hay muộn)


  ── VÍ DỤ 3: SCROLL MXH ──

    Tại sao scroll KHÔNG DỪNG ĐƯỢC (dù là "vô thức"):

      → Mỗi post = micro-delta MỚI → VTA fire nhẹ
      → Swipe → post mới → delta mới → VTA fire → dopamine nhỏ
      → KHÔNG habituation vì: mỗi post KHÁC post trước
      → = Input KHÔNG DEPLETED (algorithm cung cấp VÔ TẬN)
      → = HACK cơ chế 1 (habituation) + cơ chế 3 (input depletion)

    NHƯNG:
      → Mỗi delta RẤT NHỎ → reward RẤT NHỎ per scroll
      → Không có chunk TÍCH LŨY (mỗi post = isolated, không connect)
      → Body-base SAU 1-2 giờ: "THIẾU gì đó" (dissonance — vì nhiều
        delta nhưng KHÔNG có depth, không build melody)
      → = "Chán mà không dừng được" = VTA CÒN fire (delta) nhưng
        Body-Reward KHÔNG fire (không melody mượt)

    ⇒ MXH = EXPLOIT Novelty vô thức bằng cách:
      → Loại bỏ habituation (mỗi post khác)
      → Loại bỏ input depletion (algorithm vô tận)
      → Giữ delta > 0 LIÊN TỤC nhưng KHÔNG cho depth
    ⇒ Phanh duy nhất còn lại: Body-Satisfaction L1 (đói, mệt, buồn ngủ)
      hoặc ý thức CAN THIỆP (PFC override — nhưng khó vì delta vẫn fire)

    🟡 Analysis: consistent với observed behavior, cơ chế derived từ framework


  PATTERN CHUNG — NOVELTY VÔ THỨC:

    ┌─────────────────────────────────────────────────────────────┐
    │                                                             │
    │  External input → VTA detect delta → dopamine              │

    │  → habituation HOẶC satisfaction HOẶC input depletion      │
    │  → DỪNG                                                    │
    │                                                             │
    │  Cycle: phút → giờ (HIẾM KHI > 1 ngày cho 1 stimulus)    │
    │  Exception: environment LIÊN TỤC thay đổi (du lịch,       │
    │  MXH scroll) → kéo dài hơn nhưng NÔNG (shallow delta)     │
    │                                                             │
    │  3 PHANH TỰ NHIÊN:                                        │
    │    ① Habituation — VTA quen → im                          │
    │    ② Body-Satisfaction — nhu cầu đáp ứng → dừng           │

    → = Tại sao 1 expert CÓ THỂ nghiên cứu 1 domain CẢ ĐỜI
      mà VẪN tìm được cái mới

    🟡 "Combinatorial space tăng → imagine lâu hơn" = logical, chưa measured
    🟢 Combinatorial mathematics = verified
    🟢 Chunk compilation + hierarchy = neuroscience supported (expertise research)


  GIỚI HẠN CỦA NOVELTY IMAGINE — PHANH NÀO CÒN HOẠT ĐỘNG:

    3 phanh vô thức KHÔNG hoạt động hiệu quả:

      ③ Input Depletion? → KHÔNG (input = internal, gần vô tận)

      → L1 Body-Base: đói, khát, buồn ngủ → BẮT BUỘC dừng (sinh tồn)

        → Kiệt sức (exhaustion) → BẮT BUỘC dừng (repair)
      → Sleep/Repair: não CẦN ngủ để compile chunks + repair
        → Thiếu ngủ → PFC GIẢM chức năng → imagine GIẢM chất lượng
      → External interrupt: ai đó gọi, deadline, trách nhiệm
        → = PHANH TỪ BÊN NGOÀI, không phải internal mechanism

    → = Novelty Imagine có rất ÍT phanh tự nhiên
    → = Tại sao NÓ CÓ THỂ kéo dài hàng giờ, tháng, NĂM
    → = Khi không tự dừng + gây hại → = NOVELTY-LOOP (xem file riêng)


  ── VÍ DỤ 1: ĐỒ ĂN (IMAGINE MỞ RỘNG) ──

    So sánh với Novelty vô thức ở trên:

    Novelty vô thức: ăn → ngon → no → DỪNG
    Novelty Imagine: ăn → ngon → TẠI SAO ngon? → imagine...

      → Đầu bếp ăn món lạ → "vị gì đây?"
      → Chunks tích lũy: 100+ gia vị, 50+ kỹ thuật, 200+ món
      → Imagine: "nếu thay X bằng Y → melody vị thế nào?"
      → PFC simulate → body respond nhẹ (~40% fidelity)
      → "Hmmm, có thể..." → thử nấu → kết quả → chunk mới
      → Chunk mới + chunks cũ → imagine TIẾP → cycle

    → Đầu bếp KHÔNG ăn vì đói (L1 satisfaction không drive)
    → Đầu bếp imagine vì CHUNKS + COMBINATORIAL SPACE
    → Body-Satisfaction cho "ĐÓI" = đạt → nhưng Body-Satisfaction
