## §5 — CONTENT: §1 Three States of Awareness

```
⭐ §1.1 — DEFINITIONS:

  KNOWING = currently on the stage
    → Information broadcast → observer can access
    → Can report, remember, use in decision, act on it
    → = Consciousness.md §1.1 functional definition
    → = Currently in working memory workspace

  ACCESSIBLE = not currently on stage, but CAN be brought to stage
    → Pathway exists: compiled anchor, signal strong enough to pull
    → NOT currently in workspace (not "knowing" right now)
    → BUT a retrieval path exists → CAN become Knowing
    → = "Potential knowing" — latent but retrievable
    → Examples: grammar rules (not thinking about them, but CAN attend)
               capital of France (not knowing right now, but can recall)
               balance skills (not observing, but can attend to body)

  NOT-ACCESSIBLE = cannot be brought to stage (or very difficult)
    → No pathway, too diffuse, or beyond cognitive gap
    → CANNOT become Knowing through normal attention/recall
    → May need external intervention, training, or may be permanent
    → Examples: blood pressure regulation (no pathway — permanent)
               deeply buried cultural bias (too diffuse — needs catalyst)
               subliminal prime (too weak — needs signal strengthening)

  FORMAT: 3 definitions, each with mechanism + examples, ~30 lines


⭐ §1.2 — WHY "NOT-KNOWING" NEEDS SUBDIVISION:

  Consciousness.md §7.1 uses binary: Knowing / Not-Knowing
  The binary is USEFUL (simple, clear axis) but LOSES information:

  SAME "NOT-KNOWING", COMPLETELY DIFFERENT SITUATIONS:
    "I don't know grammar rules right now" (while speaking)
      → But I COULD attend to them if asked → ACCESSIBLE
    "I don't know my blood pressure right now"
      → And I CANNOT know it through willpower → NOT-ACCESSIBLE

  THE DIFFERENCE MATTERS FOR:
    → Training: can only train what's Accessible or can become Accessible
    → Therapy: must FIRST make Not-Accessible → Accessible → then Knowing
    → Learning design: must FIRST create meta-chunks (Not-Accessible → Accessible)
    → Understanding disorders: what was Accessible becomes Not-Accessible

  "Tiềm năng" (potential) naturally emerged during drill analysis:
    → 2-state model was inadequate to describe what we observed
    → Led to this 3-state refinement

  FORMAT: Concrete examples + practical implications, ~20 lines


⭐ §1.3 — THEORETICAL SUPPORT:

  BLOCK 1995 — ACCESS VS PHENOMENAL CONSCIOUSNESS: 🟢
    → Block distinguishes: access consciousness = "available for reasoning,
      reporting, and action control"
    → Key insight: "not currently reporting" ≠ "not accessible"
    → Information CAN be accessible to consciousness systems
      without being currently accessed
    → = "Accessible" in our framework: pathway exists, not currently active

  KARMILOFF-SMITH 1992 — REPRESENTATIONAL REDESCRIPTION: 🟢
    → Level I = implicit (procedural, cannot report)
      = NOT-ACCESSIBLE: knowledge embedded in procedure, no access path
    → Level E1 = partially explicit, accessible
      = ACCESSIBLE: can be accessed but not fully articulated yet
    → Level E2-3 = fully explicit
      = KNOWING: can report, reason about, chain logic
    → = Developmental psychology framework that INDEPENDENTLY
      describes the same 3-state progression
    → Discovery process = moving knowledge through these levels

  CONVERGENCE:
    → Block (philosophy), Karmiloff-Smith (developmental psychology),
      our framework (neuroscience-based) → same 3-state structure
    → = Cross-disciplinary convergence supports the distinction

  CONFIDENCE: 🟡 framework synthesis
    Philosophical + developmental support (🟢 individually)
    But 3-state AS A UNIFIED MODEL = new synthesis (🟡)

  FORMAT: 2 evidence blocks + convergence note, ~35 lines
```

## §8 — CONTENT: §4 Transition Modes

```
⭐ §4.1 — ACCESSIBLE → KNOWING: VTA-DEPENDENT (TOP-DOWN PULL):

  MECHANISM:
    → Prefrontal cortex (PFC) deliberately directs attention → needs dopamine to amplify
    → Ventral tegmental area (VTA) fires → dopamine to PFC → PFC can hold + sustain
    → = Goal-directed access: "I want to recall X" → PFC pulls X to stage

  CHARACTERISTICS:
    → Requires: VTA dopamine active + PFC functional
    → Energy cost: HIGH (PFC budget consumed)
    → Speed: varies (depends on anchor quality — verbal = fast, emotional = slow)
    → COLLAPSES when VTA dopamine is lost (Parkinson's disease, fatigue, depletion)

  EXAMPLES:
    → "What is the capital of France?" → PFC pulls compiled → "Paris"
    → Planning tomorrow → PFC holds goal → pulls relevant compiled patterns
    → Self-observation → PFC deliberately attends to internal states
    → Studying → PFC sustains attention on new material

  MAPS TO:
    → Consciousness.md §5.3 top-down pull pathway
    → Explicit Compilation Pathway products: have verbal anchors
      → PFC can easily pull via top-down
    → Self-Observation.md §6: meditation = training this mode

  FORMAT: Mechanism + characteristics + examples + cross-ref, ~25 lines
  CONFIDENCE: 🟡 (VTA-dependent framing is synthesis; dopamine × PFC = 🟢)


⭐ §4.2 — ACCESSIBLE → KNOWING: VTA-INDEPENDENT (BOTTOM-UP / CUE):

  MECHANISM:
    → Signal strong enough to self-ignite → broadcasts → Knowing
    → OR compiled anchor fires via external cue → ignition → Knowing
    → Does NOT require VTA dopamine or PFC deliberate pull

  CHARACTERISTICS:
    → Requires: signal strength above threshold OR matching cue
    → Energy cost: LOW (automatic, not PFC-driven)
    → Speed: FAST (direct activation, no PFC mediation)
    → SURVIVES when VTA dopamine is lost

  EXAMPLES:
    → Sudden pain → Knowing (signal pushes, no PFC needed)
    → Someone says "Paris" → "capital of France" fires (cue-triggered)
    → Smell of cookies → childhood memory (emotional cue → compiled fires)
    → Car brakes ahead → alert (compiled driving + strong signal)

  MAPS TO:
    → Consciousness.md §5.3 bottom-up push pathway
    → Implicit Compilation Pathway products: accessed via strong cues,
      not verbal pull (because no verbal anchor exists)

  FORMAT: Same structure as §4.1 for parallel comparison, ~25 lines
  CONFIDENCE: 🟡 (VTA-independent framing; bottom-up capture = 🟢)


⭐ §4.3 — NOT-ACCESSIBLE → ACCESSIBLE: CREATING ACCESSIBILITY:

  HOW DOES SOMETHING BECOME ACCESSIBLE?

  TYPE ③ (BACKGROUND) → ACCESSIBLE:
    → External catalyst creates first anchor
    → Friend points out pattern → "oh, I do do that!" → now Accessible
    → Therapy makes implicit explicit → first anchor created → Accessible
    → AI analysis surfaces background pattern → verbal anchor → Accessible
    → MECHANISM: external agent provides BOUNDARY for diffuse pattern
      → boundary → anchor possible → Accessible

  TYPE ④ (SUB-THRESHOLD) → ACCESSIBLE:
    → Training strengthens signal OR lowers threshold
    → Meditation → interoceptive accuracy increases → sub-threshold → Accessible
    → Expert training → more chunks → pattern recognition → previously
      invisible signals now Accessible
    → MECHANISM: signal/threshold ratio improves → crosses into Accessible range

  META-CHUNKS → ACCESSIBLE (for new domains):
    → Zero chunks in domain = Not-Accessible (invisible to dopamine system)
    → Meta-chunks (know domain exists) = minimal prediction model
    → Minimal model → prediction-delta possible → VTA can fire → Accessible
    → = Meta-chunks are the GATEWAY from Not-Accessible to Accessible
    → Cross-ref: Consciousness.md §8.4 (if implemented; see §11 of this plan)

  TYPE ① (HARDWARE) = PERMANENT NOT-ACCESSIBLE:
    → No pathway upward exists
    → Cannot become Accessible through any known mechanism
    → Only via external measurement (blood test, EEG, etc.)

  FORMAT: 4 type-specific paths, ~35 lines
  CONFIDENCE: 🟡 (types ②③④ transitions), 🟡🔴 (meta-chunks gateway)


⭐ §4.4 — EVIDENCE: DEPLETION CONFIRMS 2 MODES:

  BERRIDGE DEPLETION (98-99% DOPAMINE LOSS): 🟢
    → Wanting (self-initiated, goal-directed) = GONE
      = VTA-dependent mode blocked → cannot pull Accessible → Knowing
    → Liking (stimulus-triggered hedonic) = INTACT
      = VTA-independent mode works → cue-triggered Accessible → Knowing

  NISHIO 2007 (VTA SUPPLY CUT): 🟡

    → Language (stimulus-triggered compiled) = INTACT
      = VTA-independent mode works → cue activates compiled directly

  PATTERN:
    → VTA-dependent collapses → VTA-independent survives
    → = Same pattern across 2 different evidence sources
    → = Convergent support for 2 distinct transition modes
    → = FUNCTIONAL DISSOCIATION confirms 2 modes are real, not theoretical

  SOURCES:
    Berridge & Robinson 1998 (wanting/liking dissociation)
    Nishio et al. 2007, J Neurol Sci 260:271-274 (VTA infarction)
    Drill-Consciousness-Bridge.md v1.1 §3 (2 modes analysis)

## §9 — CONTENT: §5 Connections to Framework

```
⭐ §5.1 — × COMPILATION PATHWAYS:

  Explicit Compilation Pathway products:
    → Went THROUGH conscious stage → have verbal anchor
    → = Typically ACCESSIBLE with easy top-down pull
    → Pulling back to Knowing = RECALL (re-fire existing anchor)
    → RECALL = fast (anchor already present)

  Implicit Compilation Pathway products:
    → BYPASSED conscious stage → NO verbal anchor
    → = Often NOT-ACCESSIBLE initially
    → Pulling back to Knowing = DISCOVERY (create NEW anchor)
    → DISCOVERY = slow (reverse-engineer: observe → hypothesize → test)
    → = Karmiloff-Smith representational redescription process

  → Compilation pathway determines INITIAL accessibility state
  → Explicit → Accessible from the start
  → Implicit → Not-Accessible → needs discovery to become Accessible

  CROSS-REF: Consciousness.md §7.3 (if implemented; see §11 of this plan)
  FORMAT: 2 pathways × accessibility state, ~20 lines
  CONFIDENCE: 🟡 (pathway → accessibility mapping)


⭐ §5.2 — × META-CHUNKS:

  Meta-chunks = the gateway from Not-Accessible to Accessible for new domains:
    → Zero chunks → Not-Accessible (invisible)
    → Meta-chunks → Accessible (minimal model → delta possible)
    → Domain chunks → deeply Accessible (rich model → fine delta)
    → Compiled mastery → Accessible but "sleeping" (delta ≈ 0)

  = Meta-chunks control ENTRY into the Accessible pool for each domain
  = Without meta-chunks, a domain remains functionally Not-Accessible
    (not permanently like ① Hardware, but invisible until catalyzed)

  CROSS-REF: Consciousness.md §8.4 (if implemented; see §11 of this plan)
  FORMAT: Progression path, ~15 lines
  CONFIDENCE: 🟡🔴 (meta-chunks → prediction model = logically consistent,
