# Drill — Consciousness × Compiled/Fresh × Accessibility

> **Compiled ≠ vô thức. Fresh ≠ ý thức. Consciousness ≠ Knowing.**
> **Ba thứ khác nhau, tương quan nhưng KHÔNG đồng nhất.**
>
> **Ngữ pháp tiếng mẹ đẻ: compiled sâu, dùng hàng ngày — nhưng CHƯA BAO GIỜ ý thức.**
> **Expert nhìn code thấy bug: compiled sâu — nhưng Ý THỨC RÕ.**
> **Cùng compiled, khác consciousness. Vì đó là 2 trục khác nhau.**
>
> **"Biết" vs "có thể biết" vs "không thể biết" = 3 trạng thái, không phải 2.**
> **Kéo compiled lên ý thức = discovery, KHÔNG phải recall.**
> **Sân khấu yếu = 2 thứ khác nhau: mất điện vs quản lý mệt.**

```

### §2.2 Accessible = có thể lên sân khấu

```
⭐ ACCESSIBLE = CÓ PATHWAY ĐỂ LÊN SÂN KHẤU (dù hiện tại không trên sân khấu):

  = Có anchor (retrieval path tồn tại)
  = HOẶC có compiled pathway (có thể fire lại)
  = CÓ THỂ lên sân khấu khi: trigger, attention pull, imagine

  2 LOẠI ACCESSIBLE:

  ① ĐÃ TỪNG KNOWING (có lịch sử conscious):
     → Bài học tuần trước, kỹ năng đã compile
     → Có verbal anchor → re-fire dễ
     → = Recall (gọi lại thứ đã biết)
     → Ví dụ: nhớ tên thủ đô nước Pháp

  ② CHƯA TỪNG KNOWING (compiled qua implicit learning):
     → Ngữ pháp bản địa, thăng bằng, cultural norms
     → Compiled qua Con đường B (§3.2) — không qua sân khấu
     → Có motor/ritual anchor nhưng KHÔNG CÓ verbal anchor
     → CÓ THỂ kéo lên sân khấu = nhưng bằng DISCOVERY, không phải recall
     → Ví dụ: ngữ pháp tiếng Việt — biết dùng nhưng không biết rules

  ⭐ "ACCESSIBLE" CHÍNH XÁC HƠN "KNOWN":
    → "Known" (đã biết) ngụ ý lịch sử conscious → bỏ sót loại ②
    → "Accessible" (có thể tiếp cận) = bao gồm CẢ HAI loại
    → = Có pathway, bất kể đã từng conscious hay chưa

  ACCESSIBLE CŨNG CÓ SPECTRUM:
    ┌──────────────────────┬───────────────────┬──────────────────────┐
    │ Mức Accessible       │ Anchor quality    │ Effort để Knowing    │
    ├──────────────────────┼───────────────────┼──────────────────────┤
    │ Compiled sâu         │ Bền, motor/verbal │ Minimal (attention   │
    │ (ngữ pháp, lái xe)   │                   │ pull → re-fire)      │
    ├──────────────────────┼───────────────────┼──────────────────────┤
    │ Compiled vừa         │ Vừa, cần trigger  │ Trigger hoặc effort  │
    │ (bài học tuần trước) │                   │ nhẹ                  │
    ├──────────────────────┼───────────────────┼──────────────────────┤
    │ Anchor yếu           │ Yếu, đang decay   │ Cần cue hoặc có thể │
    │ (bài báo hôm qua)    │ (Ebbinghaus)      │ mất nếu không re-fire│
    └──────────────────────┴───────────────────┴──────────────────────┘

        Not identical but SAME PRINCIPLE: current report ≠ capacity to report.)

       Level I (implicit) → Level E1 (accessible, partially explicit) → Level E2-3 (fully explicit)
       = "Accessible" maps to Level E1: knowledge available to system but not yet fully explicit
     → UPGRADE từ 🔴 lên 🟡: có philosophical (Block) + developmental (Karmiloff-Smith) support
       Vẫn chưa 🟢 vì chưa có direct empirical test of 3-state utility vs 2-state
🟡 Concepts underlying (anchor, compiled, implicit learning) = established
```

### §2.3 Not-Accessible = không thể hoặc rất khó

```
⭐ NOT-ACCESSIBLE = KHÔNG CÓ PATHWAY, HOẶC PATHWAY QUÁ YẾU/PHÂN TÁN:

  ① HARDWARE-INACCESSIBLE (kiến trúc não):
     → Huyết áp, pH máu, insulin
     → Không có pathway upward đến cortex
     → KHÔNG THỂ biết bằng ý chí → chỉ qua external measurement
     → = Not-Accessible VĨNH VIỄN (do kiến trúc)

  ② BACKGROUND-INVISIBLE (quá phân tán):
     → Chronic stress, cultural bias, personality pattern
     → Không có ranh giới → PFC không thể "nắm bắt"
     → 7 reasons PFC cannot observe (Background-Pattern.md §5):
       Ⓐ VTA habituated (constant → no delta)
       Ⓑ Too distributed (no boundary → can't capture)
       Ⓒ Gist extracted (details faded)
       Ⓓ Labeled as identity ("tôi là người như vậy")
       Ⓔ Compound signal (fires together with events)
       Ⓕ Compiled suppress absorbed into baseline
       Ⓖ PFC budget depleted by pattern itself
     → = Not-Accessible nhưng CÓ THỂ SHIFT → Accessible nếu có
       external catalyst (bạn bè, therapy, AI)

  ③ NGOÀI GAP NHẬN THỨC (zero chunks):
     → Chưa có chunk nào trong domain

     → Không biết mình không biết
     → = Not-Accessible do thiếu substrate (→ §5.1)

  ⭐ NOT-ACCESSIBLE ≠ VĨNH VIỄN (trừ Type ①):
    → Type ② (background): external catalyst → shift sang Accessible
    → Type ③ (ngoài gap): tích lũy chunks → dần shift sang Accessible
    → Chỉ Type ① (hardware) = structurally Not-Accessible

## §6 — SÂN KHẤU: AROUSAL VS PFC BUDGET

### §6.1 Arousal = nguồn điện (phần cứng)

```
⭐ AROUSAL = SÂN KHẤU BẬT/TẮT:

  CONTROLLED BY: Brainstem RAS (Reticular Activating System) + Thalamus
  FUNCTION: giữ hệ thống TỈNH (wakefulness)
  = NGUỒN ĐIỆN của sân khấu

  BẬT (arousal ON):
    → RAS active → thalamus relay → cortex active
    → Sân khấu CÓ THỂ hoạt động
    → Consciousness POSSIBLE (nhưng không guaranteed)

  TẮT (arousal OFF):
    → RAS inactive → thalamus block → cortex silent
    → Sân khấu TẮT HOÀN TOÀN
    → Consciousness IMPOSSIBLE
    → = Hôn mê, gây mê, ngủ sâu (non-REM)

  GIẢM (arousal LOW):
    → RAS partially active → microsleep, drowsiness
    → Sân khấu CHẬP CHỜN
    → Consciousness intermittent
    → = Thiếu ngủ nặng, mê sảng

  ⭐ DISABLE THALAMUS → MẤT Ý THỨC (Alkire et al. 2008)
  DISABLE PFC → VẪN Ý THỨC (Gage 1848, lobotomy 1940s-60s)
  → Arousal = NECESSARY CONDITION. PFC = orchestrator, NOT source.

🟢 Moruzzi & Magoun 1949: RAS discovery
🟢 Alkire et al. 2008: anesthesia + thalamus
🟢 Consciousness.md v1.0 §5.4: Arousal vs Content
```

### §6.2 PFC budget = quản lý (phần mềm)

```
⭐ PFC BUDGET = CHẤT LƯỢNG QUẢN LÝ SÂN KHẤU:

  PFC budget = FINITE + SHARED across ALL activities (PFC-Operations.md §9):
    → Learning, changing behavior, self-observation, decision-making,
      suppression, social engagement — TẤT CẢ dùng chung 1 budget
    → Budget recharge: sleep (catecholamine restoration)

  BUDGET ĐẦY ĐỦ (PFC fresh):
    → Hold: giữ ~4±1 items trên sân khấu ổn định
    → Suppress: block pattern cạnh tranh hiệu quả
    → Gate: chọn lọc tốt cái gì lên sân khấu
    → Self-Observation: observe trạng thái bản thân (Level 2+)

  BUDGET CẠN (PFC depleted):
    → Hold yếu: items rơi khỏi sân khấu nhanh hơn
    → Suppress yếu: pattern cạnh tranh xông lên (intrusive thoughts)
    → Gate yếu: kém chọn lọc → noise lên sân khấu
    → Self-Observation MẤT ĐẦU TIÊN (PFC-Operations.md §9)

  ⭐ SÂN KHẤU VẪN BẬT KHI PFC DEPLETED:
    → Arousal ON → sân khấu bật
    → PFC depleted → quản lý mệt
    → = Sân khấu bật nhưng diễn viên sai lên, diễn viên đúng bị đẩy xuống
    → KHÔNG PHẢI sân khấu tắt → QUẢN LÝ yếu

🟢 PFC-Operations.md v1.3 §9: PFC Budget
🟡 "Quản lý mệt" vs "sân khấu tắt" distinction = framework clarification
```
