## §2 — Thread 12 Decomposed

Full sub-questions list (copy + expand từ master plan §2.12).

### §2.1 — Cycle phenomenology questions

**Q12.1** — Tại sao learning NEW chunks/connections tốn effort dù solve schema (rewarding)?

- Observation: "vừa sướng vì hiểu giá trị, vừa mệt sau vài tiếng"
- Hypothesis candidate: Reward và effort là 2 systems tách biệt, parallel firing
- Research: Csikszentmihalyi flow (reward + challenge balance), Sweller cognitive load (effort ≠ distress)
- Resolution: Reward from schema solve + effort from neural integration cost = 2 dimensions, không phải conflict

**Q12.2** — Sướng + mệt + dissonance đồng thời — 3 signals từ đâu, conflicting hay parallel?

- Sướng: body reward (opioid/dopamine) from successful schema match
- Mệt: cognitive load + metabolic cost (glucose? ATP? — debated)
- Dissonance nhẹ: pattern chưa smooth với existing schemas (melody lens)
- 3 signals parallel, measured on different dimensions, not contradictory

**Q12.3** — Draft pattern "chưa smooth on hardware" — cơ chế cụ thể?

- Synaptic efficiency: mới formed synapses fire weaker, noisier
- Myelination: long-term axon optimization (months-years)
- Network noise: cortical sync not yet established
- LTP early phase vs late phase (protein synthesis window)
- → Multiple substrates, not single mechanism

**Q12.4** — Melody integration cost (schemas khác phải sync) — cơ chế?

- New chunk/connection may conflict với existing schemas
- Conflict detection → dissonance signal
- Resolution requires either: update new chunk, update old schemas, or partition contexts
- Integration work done largely in sleep (active systems consolidation)

### §2.2 — Sleep mechanism questions

**Q12.5** — Role của sleep vs waking rest — khác nhau thế nào?

- Sleep: full offline, DMN + hippocampal replay active, cortical synchronization
- Waking rest (walking, meditation, shower): PFC offline, DMN active, but not full
- Walker 2017: sleep unique for synaptic homeostasis (SHY)
- Oppezzo 2014: walking boosts creativity (waking rest effect, different from sleep)
- → Both contribute, different contributions

**Q12.6** — Synaptic pruning during sleep — mất chunks nào, giữ chunks nào?

- SHY (Tononi-Cirelli): global downscaling, but **relative strengths preserved**
- Frequently used → relatively stronger after downscale
- Rarely used → relatively weaker, may fall below firing threshold
- = Pruning is **not selective deletion**, it's **selective preservation** via competition
- Chunks with emotional tag (amygdala) resist downscaling more

**Q12.7** — Cycle iteration count — sao một số thứ cần 1 đêm, một số cần nhiều tuần?

- Single-session learning: simple chunks, 1 sleep cycle enough
- Complex domains: many chunks + many integrations, weeks-months of cycles
- Deliberate practice research (Ericsson): 10,000 hours for expertise
- Cycle count depends on: novelty + integration distance + existing schema complexity (H8 cost formula)

### §2.3 — Baseline + adaptation questions

**Q12.8** — Baseline shift ("bình thường mới") — mechanism cho adaptation?

- Hedonic adaptation (Brickman 1978): new reward level becomes baseline, old feels boring
- Prediction error minimization: PE smaller → reward smaller → new baseline
- User's observation: "1 tuần trước thì nhẹ đầu, hiện tại thì mượt mà hơn" = baseline shift
- Framework implication: each cycle iteration moves baseline, eventually stable

**Q12.9** — Exhaustion threshold — khi nào "mệt quá" stop productive learning?

- User observation: "hồi xưa không quan tâm mệt nhẹ hay nặng, trừ khi mệt quá thì đi ngủ"
- Cognitive load exceeds capacity → noise > signal
- Brain fog (Chunk-Practical §4c) — infrastructure breakdown
- Cortisol chronic → neurons desensitize
- → Threshold is individual + state-dependent

**Q12.10** — Age effects — trẻ em học dễ hơn người lớn có liên quan đến cycle cost không?

- Children: more plasticity, less established schemas to integrate with = lower cost per cycle?
- Children: more sleep (esp SWS + REM), more consolidation time
- Adults: fewer sleep hours + more integration conflict = higher cost per cycle?
- → Plausible, needs research verification

### §2.4 — Individual variation questions

**Q12.11** — Expert vs beginner — reduced cycle cost or faster cycle?

- Expert: already has relevant schemas, smaller integration distance (H8 formula) → lower cost
- Expert: chunks compile faster (meta-chunks) → less total chunks to form
- Beginner: longer integration distance, more schemas conflict, higher cost per chunk
- → Both factors contribute

**Q12.12** — Individual variation — sao một số người exhaust fast, một số chịu được long sessions?

- Factors: age, sleep quality baseline, chronic stress level, domain expertise, motivation
- Genetic factors: COMT polymorphism (PFC efficiency), BDNF (neuroplasticity)
- Training factors: deliberate practice builds cycle tolerance
- → Multiple factors, not single axis

### §2.5 — Meta questions

**Q12.13** — Flow state vs struggle state — 2 modes khác nhau trong cycle?

- Flow (Csikszentmihalyi): high challenge + high skill match → smooth cycle, minimal friction
- Struggle: challenge exceeds skill → high effort, risk exhaustion
- → Flow is cycle within optimal zone; struggle is cycle near threshold

**Q12.14** — Framework evolution case study (user's own observation)

- User's meta-observation: "từ đợt phân tích profile bản thân, rồi thiết kế framework này, tôi để ý hơn tới trạng thái của bản thân"
- Hiện tại: "mượt mà hơn và nhẹ đầu hơn"
- Trước đó: "rất căng, nhưng vẫn cảm thấy có một số điểm sáng"
- Case study: framework-building is itself a long learning cycle, user observable case
- Meta-meta: using framework helps observe framework's claims — reflexive validation
