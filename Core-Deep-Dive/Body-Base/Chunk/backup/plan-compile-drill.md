### File 1: `10-Compile-Architecture.md` (~1,200-1,500L)

**Core thesis**: Brain có 1 compile engine + 3 modulator systems + 1 offline maintenance. Mọi compile đều đi qua exposure → Hebbian.

**Sections dự kiến**:

§1. **Unifying Principle**
- ALL compile = exposure → Hebbian strengthening
- Tại sao: Hebbian là cơ chế wiring DUY NHẤT (LTP/LTD)
- Citation: Hebb 1949, Bliss & Lømo 1973 (LTP discovery)
- Key insight: không có "trust compile mechanism" hay "expertise compile mechanism" — chỉ có 1 engine

§2. **Engine A: Exposure → Hebbian → Compile**
- compile_rate = f(exposure × saliency × contingency × peak_valence × multi_modal_richness)
- Exposure = tham số FIRST, các tham số khác = quality dimensions CỦA exposure
- Từ Chunk.md §2.2 hiện tại — giữ nguyên, reframe context
- Cross-ref: Chunk.md §2.1 (4 compile mechanisms)

§3. **Modulator B: Entity-Valence Bias**
- Automatic, compiled, cost ≈ 0
- Per-entity structural valence profile × trust meta-dimension
- Trust = AMPLIFIER, gradient Mức 0-5, KHÔNG phải gate binary
- Contradiction resolution: Chunk.md §2.3 "gate" vs Entity-Access "gradient"
- Cross-ref: Entity-Valence-Dynamics.md, Entity-Access.md Mức 0-5, Chunk.md §2.3

§4. **Source C: 3 Exposure Channels (Song song)**
- C1: External (body-input) — sensory, environmental
- C2: Internal-deliberate (PFC imagination/simulate) — nhẩm từ, tưởng tượng tình huống
- C3: Internal-automatic (spontaneous chunk fire, Background-Pattern, memory resurface)
- Tỉ lệ C1:C2:C3 biến thiên theo tình huống, LUÔN song song
- Cross-ref: Body-Feedback-Mechanism.md §1.2 (2 genuine sources), Simulation-Engine.md

§5. **Modulator D: PFC Hold + Suppress**
- Hold = amplify specific exposure (CAN compile)
- Suppress = block output route (CANNOT compile "not")
- Finite budget, fatigable, evolutionary mới
- Cross-ref: PFC-Operations.md §2.1-§2.3

§6. **Multi-Stream Compile**
- Mọi tình huống chạy ĐỒNG THỜI nhiều compile streams
- Content stream (luôn Experience-type)
- Value stream (trust-amplified hoặc không)
- Entity stream (Entity-Valence update)
- Behavior stream (approach/avoidance direction)
- Ví dụ: trẻ bị ép học → Content=Experience, Entity=negative update cho "mẹ-ép", Value=threat-driven

§7. **Feedback Loops**
- PFC → Entity-Valence: INDIRECT (via exposure, slow)
- Entity-Valence → PFC: DIRECT (automatic, fast, can override)
- "Muốn con hay chữ thì yêu lấy thầy" = virtuous loop
- Entity-Valence → Exposure selection: automatic attention bias
- PFC → Exposure: direct (deliberate practice, imagination)

§8. **3 Compile Types = Modulator Configurations**
- Experience = A dominant, minimal B/D (direct exposure learning)
- Trust = A + B dominant (Entity-Valence amplifies/blocks)
- Expertise = A + D dominant (PFC sustained hold)
- Mỗi type = config KHÁC NHAU của cùng 1 engine + modulators
- Bảng mapping: Type → Engine × Modulator B × Source C × Modulator D

§9. **Evolutionary Gradient**
- Insects: A only (pure Hebbian)
- Fish/reptiles: A + C1 + basic B
- Dogs/mammals: A + C1 + B basic + D basic
- Primates: A + C1,C3 + B + D
- Humans: A + C1,C2,C3 + B full + D full + S
- Mỗi bước tiến hóa = THÊM modulator/source, KHÔNG thay đổi engine

§10. **Edge Cases Verified (8+)**
- Trauma (single exposure, peak valence)
- Cult (Entity-Valence override PFC)
- Therapy (PFC-guided recompile qua new exposure)
- Trẻ bị ép học (multi-stream, forced exposure)
- "Biết mà không làm" (PFC compiled ≠ body compiled)
- Immersion language learning (C1 dominant, B minimal)
- Expert blind spot (D-compiled mạnh, B weak cho novice)
- "Yêu lấy thầy" (PFC→B virtuous loop)

§11. **Open Questions**
- Hebbian rate có uniform không? Hay khác nhau giữa cortical regions?
- Entity-Valence bias strength: tương quan tuyến tính hay có threshold?
- C3 (spontaneous chunk fire) có thể PFC-influenced không?

### File 2: `11-Compile-Sleep.md` (~800-1,000L)

**Core thesis**: Sleep = offline maintenance system ĐỘC LẬP. 6 mechanisms: chỉ ~1.5 là exposure-based. Sleep KHÔNG đơn thuần là "exposure source".

**Sections dự kiến**:

§1. **Sleep ≠ Exposure Source**
- Tại sao phân biệt: 4/6 sleep mechanisms KHÔNG tạo new exposure
- Sleep = maintenance + optimization + consolidation
- Position trong architecture: Component S, parallel với A-B-C-D, ĐỘC LẬP

§2. **6 Sleep Mechanisms — Classification**
Mỗi mechanism phân loại: IS exposure / NOT exposure / PARTIAL

§2.1 **SHY — Synaptic Homeostasis Hypothesis** (NOT exposure)
- Global synaptic downscaling, pruning weak connections
- Tononi & Cirelli 2003
- Tác dụng: signal-to-noise ratio, save metabolic cost
- KHÔNG tạo new exposure, KHÔNG strengthen specific pattern

§2.2 **Hippocampal Replay** (IS exposure — replay = internal exposure)
- Patterns re-fire 10-20x compressed in SWRs
- Wilson & McNaughton 1994
- GIỐNG engine A (exposure → Hebbian), nhưng source = replay not external
- Classify: Source C3 variant (automatic internal exposure)

§2.3 **Active Systems Consolidation** (NOT exposure)
- Hippocampus → cortex TRANSFER via spindle-ripple coupling
- Born & Wilhelm 2012
- KHÁC exposure: đây là architectural reorganization, move storage location
- Không strengthen pattern, mà RELOCATE pattern

§2.4 **REM Creative Linking** (PARTIAL exposure)
- Remote associations via altered neurochemistry (low norepinephrine)
- Cai 2009
- Tạo NEW connections giữa existing patterns = partial new exposure
- Nhưng mechanism khác Hebbian trực tiếp: random activation + weak gating

§2.5 **Emotional Decoupling** (NOT exposure)
- Strip emotional charge, preserve content
- Walker 2017
- MODIFY existing compile, không tạo new compile
- Important for: trauma processing, emotional regulation

§2.6 **Gist Extraction** (PARTIAL exposure)
- Abstract pattern from specific, detail fades
- Payne 2009
- Tạo new ABSTRACT pattern (= new exposure) nhưng LOSES specific detail
- Trade-off: generalization gain vs specificity loss

§3. **Sleep × Compile Architecture Interaction**
- Sleep enhances Engine A (via replay = more exposure cycles)
- Sleep enhances Modulator B (via emotional decoupling = Entity-Valence calibration)
- Sleep independent of Modulator D (PFC OFFLINE during sleep)
- Sleep modifies Source quality (via gist extraction = new C3 patterns)

§4. **Sleep Deprivation = Architecture Degradation**
- Disrupts ALL 6 mechanisms simultaneously
- PFC function degrades FIRST (modulator D weakened)
- Entity-Valence amplification goes unchecked (emotional reactivity)
- Cross-ref: stress/cortisol literature

§5. **Open Questions**
- Sleep mechanisms cạnh tranh hay sequential trong 1 đêm?
- Napping: mechanism nào active? (evidence: mainly replay + SHY)
- Individual variation: genetic influence on sleep mechanism efficiency?
- Dreaming (Revonsuo 2000): genuine simulation training hay epiphenomenon?
- Age-related changes: sleep architecture shifts → compile quality changes?
