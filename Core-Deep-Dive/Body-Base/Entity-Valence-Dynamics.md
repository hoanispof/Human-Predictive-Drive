## §1 — STRUCTURAL VALENCE vs CURRENT VALENCE

> Source: Drill-Entity-Valence-Dynamics v2.0 §1.

```
⭐⭐ 2 VALENCE TYPES — DIFFERENT LOCATION, SPEED, AND SATIATION TYPE:

  STRUCTURAL VALENCE (INSIDE Entity-Compiled):
    = Aggregate of per-channel valence tags across all compiled chunks
    = Each chunk carries a tag: approach / avoidance / neutral (compile-time lock)
    = Structural valence = SUM of all tags = valence PROFILE
    = SLOW to change (requires compile/decompile of chunks = months/years)
    = 3 subtypes (§2.1): positive-dominant / negative-dominant / MIXED
    
    STRUCTURAL VALENCE HAS A SATIATION PROFILE:
      Each channel has its own satiation type (Gap-Body-Need v2.0 §2):
      → Cyclic channels: approach/avoidance tags OSCILLATE (hunger → fill → off → return)
      → Tonic channels: approach tags form STABLE baseline (comfort, safety → slow build, slow fade)
      → Generative channels: approach tags RENEWABLE (curiosity → fill → new gap → repeat)
      → Structural valence = aggregate across ALL satiation types
      → Most deep entities = Tonic + Generative compound (§6)
    
    Example — mother (structural valence profile):
      Channel [cooking meals]: approach (+) — Cyclic (fill → return → fill)
      Channel [hugging]: approach (++) — Tonic (ongoing baseline)
      Channel [scolding]: avoidance (--) — Cyclic (episode → fade)
      Channel [sacrifice]: complex (+/schema) — Tonic (persistent)
      Channel [teaching skills]: approach (+) — Generative (each lesson → new gap)
      = Structural valence = POSITIVE-DOMINANT, Tonic-base + Generative-bursts
      
  CURRENT VALENCE (OUTSIDE Entity-Compiled, per-moment):
    = Valence-Momentary (Self-Pattern-Modeling) + activated Valence-Structural channels AT THIS MOMENT
    = Changes PER SECOND (context-dependent)
    = What PFC observes → what the person verbalizes
    
    CURRENT VALENCE "FEEL" = f(Reward-Signal-Architecture Evaluative:Direct-State ratio)
    (Reward-Signal-Architecture v2.1 §1):
      Direct-State dominant → "warm, comfortable" (opioid, CT afferents)
      Evaluative dominant → "enthusiastic, interesting" (evaluative, dopamine-adjacent)
      → Mother hugging child: current = Direct-State dominant → "warm"
      → Friend sharing an insight: current = Evaluative dominant → "interesting"
      → SAME positive valence, DIFFERENT "feel" because DIFFERENT Evaluative:Direct-State ratio
    
    Example — mother scolding right now:
      Valence-Momentary = NEGATIVE (Self-Pattern-Modeling simulates mother angry → body distress)
      Valence-Structural [scolding] channel = ACTIVE (avoidance, Cyclic)
      Valence-Structural [hugging, cooking] channels = DORMANT (not triggered)
      Current valence = NEGATIVE
      PFC label: "hate mom" (in this moment)
      
    Example — reading a poem about mother while away from home:
      Valence-Momentary = POSITIVE (poem → Self-Pattern-Modeling fires → body warmth)
      Valence-Structural [sacrifice, hugging, cooking] channels = ACTIVE (triggered by poem)
      Valence-Structural [scolding] channels = DORMANT (not in poem context)
      Current valence = INTENSE POSITIVE (Direct-State dominant → "love/care")
      
  ⭐ STRUCTURAL ≠ CURRENT:
    Structural = what's STORED (slow, aggregate across satiation types)
    Current = what's ACTIVATED (fast, context-dependent, Reward-Signal-Architecture
              Evaluative:Direct-State ratio determines "feel")
    Same entity, same structural valence → DIFFERENT current valence depending on moment
    → "Hate mom" (when being scolded) vs "love mom" (when reading the poem)
    → SAME structural (mixed) → DIFFERENT current (context activates different channels)
    
  🟡 Structural vs Current distinction + satiation type dimension = framework synthesis
  🟢 Per-channel valence: Valence-Propagation.md v4.1 §2 (explicit "does NOT average")
  🟢 Reward-Signal-Architecture Evaluative:Direct-State: Reward-Signal-Architecture v2.1 §1
```

### §2.1 — 3 Subtypes (spectrum)

```
⭐ 3 SUBTYPES — A SPECTRUM, NOT BINARY (Inter-Body-Mechanism.md §8.2):

  ① POSITIVE-DOMINANT (≈ old "Entity-Owned"):
     Majority channels positive. Presence = approach + reward. Loss = grief.
     Examples: long-time close friends, children, mother in a healthy relationship.

  ② NEGATIVE-DOMINANT:
     Majority channels negative. Presence = threat/dissonance. Loss = relief.
     BUT sub-case "obsession": loss = emptiness (losing a target).
     Examples: bully, abuser, obsessive rival.

  ③ MIXED (AMBIVALENT) — MOST COMMON:
     Significant BOTH positive AND negative channels SIMULTANEOUSLY.
     Behavior oscillates by STATE/TRIGGER/CONTEXT.
     Loss = COMPLEX grief (relief + pain simultaneously).
     Examples: strict parents, conflictual couples, frenemies.

  WHY IS ③ THE MOST COMMON?
    → Long relationships → many interaction types → many channels compile
    → THE CLOSER and LONGER → the more channels (both + and -)
    → "Purely positive" = rare luxury of LIMITED interactions
    → Paradox: CLOSE FOR A LONG TIME = deeper bond + deeper conflict potential

  🟡 3 subtypes = framework synthesis.
  🟢 Ambivalence = well-documented (Cacioppo & Berntson 1994).
```

### §2.4 — MECHANISM NOTE: Description vs Mechanism (from Drill-Body-Base-Boundary-Spectrum v1.2)

    → Entity OK → gaps fillable → I'm OK

### §2.4.1 — Embodied-Mirror → Gap-Predict: 2-STEP META-PROCESS

> Source: Drill-Agent-Feed-Channel v2.2 §5 + Entity-Compiled.md v1.2 §2.3.

```
⭐⭐ §2.4 DESCRIBES THE MECHANISM: gap-fillability prediction (Gap-Predict).
  BUT: Gap-Predict does NOT operate alone.
  Entity-Compiled §2.3 DESCRIBES Embodied-Mirror: body simulation.
  WHAT'S NEW: Embodied-Mirror → Gap-Predict = UNIFIED 2-STEP META-PROCESS.

  STEP 1 — Embodied-Mirror: EMBODIED SIMULATION (PRESENT mirroring):
    Entity-Compiled fires → body simulates entity's CURRENT state
    using OWN pain/joy circuits
    = Entity-Compiled §2.3: "my body = simulator for close persons"
    = Singer 2004: ACC + anterior insula fire when watching a partner in pain
    Scale: Entity-Compiled depth (self > close > stranger)

    Properties:
      PRESENT-oriented: mirrors entity state AS IT IS NOW
      Automatic: fires when Entity-Compiled is activated (no PFC required)
      Graded: intensity = f(Entity-Compiled depth)
      Uses OWN circuits: MY pain circuits simulate the ENTITY's pain

  STEP 2 — Gap-Predict: GAP-FILLABILITY PREDICTION (ANTICIPATORY signal):
    Entity state (from Embodied-Mirror output OR from direct information) →
    Self-Pattern-Modeling predicts: how will entity fill my gaps in the future? →
    Body generates signal:
      Entity OK → gaps fillable → relief, reassurance
      Entity threatened → gaps threatened → anxiety, worry
    = §2.4 above: "entity's state = my state" = compressed prediction
    Scale: Entity-Compiled depth × gap dependency breadth

  ⭐ Embodied-Mirror AND Gap-Predict = 2 DISTINCT LAYERS:
    Embodied-Mirror = MIRROR (body simulates entity state → feel entity's pain/joy)
    Gap-Predict = PREDICT (entity state → predict impact on MY gaps → feel MY concern)

    Embodied-Mirror fires → Gap-Predict CAN receive its output:
      Embodied-Mirror simulates "entity in pain" → Gap-Predict predicts "entity less available" → MY anxiety
    BUT: Gap-Predict ALSO fires INDEPENDENTLY when information arrives WITHOUT going through Embodied-Mirror

  DIFFERENCE FROM LAYER 0 ENTRAINMENT (Gap-Body-Need v2.0 §11.3):
    Entrainment = PHYSICS-LEVEL oscillator sync (unconscious, bidirectional)
    Embodied-Mirror = COGNITIVE simulation (Entity-Compiled fires → body mirrors)
    Embodied-Mirror REQUIRES Entity-Compiled. Entrainment DOES NOT.
    Embodied-Mirror MORE one-directional. Entrainment BIDIRECTIONAL.
    3 INDEPENDENT mechanisms: Entrainment (Layer 0) / Embodied-Mirror (simulation) / Gap-Predict (prediction)

  NOVELTY vs GAP-FILLABILITY: 2 MECHANISMS WITH DIFFERENT TIME HORIZONS:
    Novelty/Prediction-delta (Gap-Body-Need v2.0 §4.2) = INSTANTANEOUS:
      VTA fires when actual ≠ predicted → reward/pain
      Time-reference: NOW (entity behavior AT THIS MOMENT vs prediction)
      Example: friend says something surprising → delta → dopamine → "interesting!"

    Gap-Fillability Prediction (Gap-Predict) = ANTICIPATORY (future-oriented):
      Self-Pattern-Modeling predicts entity's FUTURE impact on my gaps
      Time-reference: FUTURE (entity state → predict gap-fill trajectory)
      Example: friend says "I'm moving away soon" → predict future loss → anxiety

    BOTH = Processing MECHANISMS (Gap-Body-Need v2.0 §4.2):
      Run ACROSS all hardware sources (unbounded, cross-source)
      DIFFERENT from hardware SOURCES (bounded, fires for 1 type of input)
    BUT: DIFFERENT time horizons (instantaneous vs anticipatory)

  🟢 Embodied-Mirror body simulation: Entity-Compiled §2.3. Singer 2004: ACC + insula.
  🟢 Gap-Predict gap-fillability: §2.4 above. Eisenberger 2003: social pain.
  🟡 Embodied-Mirror → Gap-Predict = unified 2-step meta-process = drill synthesis (Drill-Agent-Feed-Channel v2.2 §5)
  🟡 Novelty (instantaneous) vs Gap-Fillability (anticipatory) = drill synthesis

```

    Entity-Compiled fires → body simulates entity state → feel entity's pain/joy
    BUT: does NOT fire Gap-Predict (no gap prediction)
