### §2.2 — The Body-First Invariant (KEEP unchanged)

**This part is stable across Tier 1 và Tier 2**: Body → Feeling → PFC. Order NEVER reversed.

**4 evidences from Feeling-Accuracy.md §1**:

1. **Libet-style research** 🟢 (cần caveat về interpretation): Unconscious brain activity precedes conscious decision by 300-500ms. PFC "decides" AFTER body has prepared action.

2. **Damasio VMPFC patients** 🟢: Phineas Gage + modern cases. VMPFC damage → logic + memory + language INTACT, feeling-based decision LOST → paralysis. Iowa Gambling Task: normal subjects develop anticipatory SCR (skin conductance response) BEFORE consciously knowing which deck is bad. VMPFC patients: no anticipatory SCR, cannot learn.

3. **Interoception** 🟢 (Craig 2002, 2013): Afferent pathways body → brain continuous. Insula integrates body state BEFORE cortical processing. PFC receives ALREADY-INTEGRATED signal. **PFC does not access raw body signals directly.**

4. **Gendlin felt sense** 🟢: 50+ years clinical observation. "Felt sense = body knowledge pre-verbal. Cannot be generated by thinking. Must be ACCESSED, not created."

**Implications** (from Feeling-Accuracy.md §1):
> "A. 'Think yourself into feeling' = **impossible**. You can trigger schema that triggers body state (indirect). But you cannot directly will a feeling into existence.
>
> B. PFC can INHIBIT feeling expression, not generate source. Suppression works temporarily, but source feeling continues body-level.
>
> C. Awareness matters: Body computes whether PFC notices or not. PFC attention = whether feeling becomes explicit. Not attending ≠ not happening."

**→ INVARIANT**: No matter which definition (Tier 1 or Tier 2), **body computes first, feeling emerges, PFC observes after**. Theme A preserves this fully.

**Example 5 — Predictive interoception**:

**Example 6 — Gendlin felt sense**:

**Pattern across 6**: All are real feelings PFC observes. None fit clean "body-base" definition. All involve **unconscious processing beyond raw body signals** — schemas, predictions, simulation, memory retrieval.

### §2.5 — PFC's Role: Observer, Not Creator

Feel-Development.md §2 explicitly lists what PFC does and doesn't do:

**What PFC DOES**:
- **Observe** (read integrated upstream signals)
- **Label** (via chunks — error-prone at Layer 6-7)
- **Integrate** (combine multiple channel reads)
- **Plan** (strategic reasoning using chunks)
- **Report** (verbalize if chunks support)
- **Choose** between competing signals (executive function)
- **Direct attention** (top-down bias of what to observe)

**What PFC DOESN'T do**:
- **Execute interoception** (body does)
- **Run schemas** (cortex runs them automatically)
- **Generate feelings** (unconscious processing does)
- **Directly access raw body** (integration happens upstream)

**Imagine-Final.md §1 adds 3 specific PFC functions**:
1. **CONSCIOUS HÓA**: name unconscious Imagine-Final
2. **SIMULATE DOMAIN**: check feasibility, domain accuracy
3. **CHỌN**: executive choice between competing Imagine-Finals

**PFC = REACTIVE, not proactive** (Imagine-Final §1):
> "PFC KHÔNG tự khởi động → BODY GỌI PFC khi cần. Body MET → PFC không engage (autopilot). Body UNMET → dissonance signal → PFC BẬT."

**~95/5 rule** (Anchor-Schema §1):
- ~95% of life: compiled schemas auto-fire → auto-anchor → PFC KHÔNG tham gia
- ~5% of decisions: compiled insufficient → PFC engages (new situations, conflicts, long-term decisions)

**→ Answer Part 1 of user's question**: PFC does "quan sát vô thức rồi lựa chọn" — **framework explicitly confirms**. But PFC does more than just observe and choose:
- Observe (Layer 3-4 integration)
- Label via chunks (Layer 6, error-prone)
- Explain via narrative (Layer 7, confabulation risk)
- Choose between competing (executive)
- Direct attention (top-down bias)

### §2.6 — "PFC yêu cầu vô thức tính toán hoặc hold để chain": User's Hypothesis Check

User asks: **Is there a top-down "query mechanism" where PFC asks unconscious to continue computing or hold for chaining?**

**Framework's position** (partial support, needs refinement):

**Direct evidence in framework**:
- Imagine-Final.md §1 Tầng 2: PFC does "direct" operations — consciousify, simulate domain, choose
- Imagine-Final.md §1.5 Phase 4 RELOAD: "body trigger PFC engage lại với Imagine-Final SAVED" — top-down reload
- Anchor-Schema.md §1 bước ③: "PFC SCAN + IMAGINE: tìm anchor mới" — PFC actively generates candidates when compiled anchor insufficient

**Indirect support from research**:
- **Global Workspace Theory** (Baars 1988) 🟢: PFC broadcast → whole-brain sync. This IS a top-down mechanism.
- **Predictive processing** (Clark 2013, Friston): PFC sets high-level priors → propagate down to sensory cortex. Top-down attention biases what unconscious processes.
- **Miller & Cohen 2001** 🟢: PFC maintains goal representations → bias processing across brain. Not just passive observation.

**BUT**: Framework hasn't explicitly modeled **"PFC query → wait for response → chain"** as a mechanism.

**Closest framework concept — "Attention allocation"** (Feeling.md §2.5 Layer 4):
> "Layer 4 OBSERVATION requires PFC allocation (attention budget)."

**And Imagine-Final §1.5 BACKGROUND phase**:
> "'Cố nghĩ' KÉM HƠN 'để nó tự thấm'. Phần lớn sự 'tiến hóa' của Imagine-Final xảy ra Ở PHASE 3 (background) — KHÔNG khi PFC actively work."

**Key insight**: Framework says PFC can **allocate attention** (top-down) but **chain formation happens mostly in background** (not active PFC work). So user's intuition is **partially correct**:

- **YES**: PFC can direct attention, trigger reload, choose which simulation to run
- **NO** (or partial): PFC doesn't "query and wait" — instead, PFC sets attention, releases control to unconscious, receives refined result on reload

**Refined answer**: Framework architecture is:
1. PFC sets attention / anchor / question
2. PFC **releases** to unconscious processing (Background Phase)
3. Unconscious consolidates, spreads activation, refines chunks via REM + DMN
4. When body triggers (dissonance) or PFC queries (reload), PFC pulls result back
5. If unfinished, loop

**This is CLOSE to user's "yêu cầu vô thức tính toán hoặc hold"** but with nuance: PFC doesn't stay active waiting. It disengages and reloads later.

**→ Theme A confirms user's hypothesis with refinement**: Top-down query exists (attention allocation + anchor setting). Chain formation is NOT active PFC work — it's background processing. PFC reloads to see progress. User's intuition captures the direction-setting aspect but may overstate how "active" the wait is.

### §2.7 — Spectrum: Clear Signals → Fuzzy Imagine Layer

User describes spectrum: "từ PFC detect body-base cơ bản (đói, đau rõ ràng, vui rõ ràng,...) tới PFC detect qua lớp imagine mờ mờ trong vô thức".

**Framework's decomposition of this spectrum**:

#### §2.7.1 — Layer 3 Clarity Dimension (from Feeling.md §2)

The 7-layer structure applies to ALL feeling sources, not just body interoception:

| Layer | Clarity | Example clear signal | Example fuzzy signal |
|---|---|---|---|
| Layer 1 Raw | 100% body truth | Gastric signals | Schema partial fire |
| Layer 2 Integration | 95% | Insula integration | Multi-schema integration |
| Layer 3 Noticeable | 90% | "Đói rõ" | "Cảm thấy có gì đó..." |
| Layer 4 Observe | 85% | "Tôi đang đói" | "Tôi đang feel gì đó" |
| Layer 5 Locate | 70-90% | "Đói ở bụng" | "Ở chest nhưng mờ" |
| Layer 6 Label | 40-80% | "Đói" | "Uneasy" |
| Layer 7 Explain | 20-70% | "Vì chưa ăn" | "Vì... không biết" |

**Clear signals (đói, đau rõ)** reach Layer 5-6 reliably. Fuzzy signals stay at Layer 3-4.

**This is NOT two different mechanisms** — it's the **same observation interface** applied to signals of different strength/source.

#### §2.7.2 — Source Dimension

From Feeling-Sources.md §0-§10, 10 channels produce feelings. They differ in:

- **Source strength**: L1 physiological (high, clear) vs imagine-final preview (low, fuzzy)
- **Temporal stability**: L1 (clear over time) vs cognitive prediction (flickering)
- **Compilation depth**: L3 meaning (deep compile) vs novel cross-domain (no compile yet)

**"Clear đói rõ"**:
- Source: L1 interoception (gastric, ghrelin signals)
- Integration: clear — single-source dominant
- Layer 3 fidelity: high
- PFC labels easily: "đói"

**"Fuzzy imagine-final preview"**:
- Source: simulation of future state
- Integration: weak — preview is diluted version of "real" body response
- Layer 3 fidelity: lower
- PFC labels with uncertainty: "cảm thấy kế hoạch này sẽ work... chắc?"

**Same mechanism, different signal strength**. Framework handles both via same PFC observation interface.

#### §2.7.3 — Why "Fuzzy" Feels Different (Layer 3 felt sense)

**Gendlin's felt sense** (Feeling.md §2.4):
> "Layer 3 là điểm body đã compute XONG — nhưng PFC CHƯA label gì cả. 'Có gì đó', 'cảm thấy sao sao ấy', 'không biết nhưng cảm thấy...' = TRUTH đã sẵn sàng. PFC chưa có từ."

**Why some feelings feel "fuzzy"**:
1. **Source is upstream integration** (not single raw signal) → no single word maps
2. **Multiple channels firing weakly** → compression difficult
3. **PFC chunks insufficient** → no learned label for this specific pattern
4. **Novel combination** → schema doesn't have pre-compiled label

**Framework position** (from Feel-Development §2.4):
> "Q3 Logic-Feeling flow partially resolved: NOT Model A (PFC directly senses raw body), NOT Model B (PFC pure labels from outside), IS: **PFC reads integrated middle-ground signals, labels via chunks**."

**→ Fuzzy ≠ another mechanism**. Fuzzy = same PFC observation interface seeing a signal that:
- Is integrated (Layer 2-3) but
- Has no compiled Layer 6 label yet

**Training (meditation, Focusing)** = learning to trust Layer 3-4 without forcing premature Layer 6 labels.

### §2.8 — "Hay là cơ chế xử lý khác nữa?" — User's third question

User's third question: Is there another processing mechanism beyond "PFC observes vô thức + chooses + requests"?

**Framework's answer**: **Possibly, but framework commits to unified model** with bidirectional flow.

#### §2.8.1 — No separate "imagine layer" mechanism

Framework does NOT treat "imagine layer feelings" as separate mechanism:
- Imagine-Final previews = body simulation firing same pathways as real (Schema-Operations §8.5, cited in Feeling.md)
- Mirror "copy" = own body firing weaker version of other's state (Gallese embodied simulation)
- Schema-triggered = body fires as if real input (Feeling-Sources §7)

**Common pattern**: Body fires same pathways at different intensities, from different triggers. PFC observes the integrated result. ONE mechanism, many inputs.

#### §2.8.2 — Bidirectional flow is architectural complement

Anchor-Schema.md §0 adds the **bottom-up complement**:
- Top-down: PFC Imagine-Final → vô thức (user's intuition)
- Bottom-up: Vô thức Anchor-Schema requirement → PFC must engage
- PFC is **not always in control** — vô thức can pull PFC in

**This is NOT "another mechanism" — it's the reverse flow of same mechanism**.

#### §2.8.3 — Predictive Processing (open frontier)

Framework acknowledges (Feeling-Research §9, Feel-Development §2.5):
> "Predictive processing (Friston, Seth) offers framework. Interoceptive inference model (Seth 2013). Relationship to framework not fully worked out."

Predictive processing proposes: brain is a **prediction machine** that generates expectations + updates from errors. If true, there may be deeper unification:
- "Feeling" = PFC observing prediction error signals
- "Body-first" = body interoception happens upstream of prediction
- "Imagine-final" = forward model predictions
- "Anchor-schema" = high-level priors

**Theme A's stance on PP**: Accept as promising direction but don't commit. Theme A architecture is **compatible** with PP but doesn't require it.

**→ Answer Part 3**: No, there isn't a secret "third mechanism". Framework has:
- ONE observation interface (PFC observes integrated unconscious processing)
- BIDIRECTIONAL flow (top-down attention + bottom-up requirement)
- MANY sources feeding into integration (10 channels in Feeling-Sources)
- POTENTIAL deeper unification via predictive processing (not committed)
