# Action-Space — Per-Person Capability Landscape Observation

> **Two people with the same goal: escape poverty.**
> **Same neighborhood. Both 25 years old.**
> **Same gap. Same drive. Same body-need firing.**
>
> **Person A: speaks English, has a laptop, knows people in tech,**
> **knows they can learn to code.**
> **Person B: only speaks their local language, has no computer,**
> **only knows their neighbors, doesn't know what "IT" is.**
>
> **SAME GAP. COMPLETELY DIFFERENT OPTIONS.**
> **Person A sees 5 paths. Person B sees 0.**
>
> **Not because B is lazy. Not because B doesn't want it.**
> **B DOESN'T KNOW ANY PATH EXISTS.**
>
> **Gap-Distribution-Profile: "What do I WANT?" — demand side.**
> **THIS FILE: "What CAN I do?" — supply side.**
>
> **Actual behavior = f(what you want × what you can do).**
> **Missing either = you only see half the picture.**
>
> **And action space doesn't appear by itself.**
> **It is SHAPED by: collective infrastructure → social position →**
> **education + family → hardware + accumulated experience.**
>
> **This file: view action space as a PERSONAL PROFILE,**
> **understand WHERE it comes from, HOW to observe it, and WHAT it predicts.**

## §1 — DEFINITION: Action-Space = Supply-Side Profile

### §1.1 — From per-concept to aggregate

```
⭐ MECHANISM FILES DESCRIBE PER-CONCEPT. THIS FILE DESCRIBES THE ENTIRE SUPPLY-SIDE.

  The framework currently has:
    → Status.md: Resource Access Map (what I access from which agents)
    → Autonomy.md: Self-directed tendency (how much I prefer self-action)
    → Obligation.md: Cost to maintain access (what I pay for access)
    → Compliance-Floor.md: Legal limits (what I am NOT permitted to do)
    → Chunk.md: Accumulation mechanism (how chunks compile)
    → Entity-Compiled-Capacity: Dunbar ceiling (how many relationships)

  This file ADDS:
    → Each person has a CAPABILITY SPACE at every moment
    → This space = AGGREGATE: what they know + what they access +
      what they are permitted + what they are aware of
    → = AGGREGATE SUPPLY-SIDE PROPERTY
    → = PER-PERSON, PER-MOMENT, ALWAYS CHANGING

  EXAMPLE:
    Person A at time T:
      Compiled Capacity: speaks 3 languages, holds an IT degree,
                         10 years' experience
      Resource Access: monthly income, 200-contact network, owns property
      Freedom: occupational freedom, no debt, adult children
      Awareness: knows they can start a business, knows the market,
                 understands the risks
      → Action-Space A: VERY WIDE — many options available

    Person B at time T:
      Compiled Capacity: finished 9th grade, knows farming, physically healthy
      Resource Access: low income, knows neighbors, no capital
      Freedom: must support aging parents, bank debt, one occupation
      Awareness: unaware of scholarships, unaware of online courses
      → Action-Space B: VERY NARROW — few options available

  → A and B differ NOT because of IQ or "willpower"
  → But because their ACTION SPACE differs →
    different options → different behavior → different life
```

### §1.2 — Action-Space as distribution (not a single point)

```
⭐ EACH PERSON = A DISTRIBUTION, NOT A SINGLE POINT:

  Like Gap-Distribution §1.2 — Action-Space SHOULD NOT be described in one word:
    WRONG: "This person has a wide action space" (oversimplification)
    RIGHT: "This person has DEEP capacity in IT, MODERATE resource access,
           HIGH freedom, NARROW awareness outside their domain"

  Action-Space has a "PROFILE":
    → 4 axes INDEPENDENT — combined to create unique per-person profile
    → Some axes WIDE (many options), some axes NARROW (few options)
    → Each axis has STRONG zones and WEAK zones
    → Example: Einstein 1905 — capacity EXTREMELY DEEP (physics) +
      resource access NARROW (clerical job) + freedom OK (free time) +
      awareness HIGH (knew he was strong in physics)

  Action-Space ALWAYS CHANGING:
    → Every day compiles more chunks → axis ① expands
    → Earn/spend money → axis ② fluctuates
    → Laws change, obligations added/removed → axis ③ shifts
    → Meet someone new, read a book, hear a podcast → axis ④ can expand suddenly
    → = Dynamic, per-moment, NOT fixed

  Action-Space NOT FULLY SELF-PERCEIVED:
    → PFC = Lawyer (Inter-Body-Mechanism §7): self-assessment = BIASED
    → Axis ④ (Awareness) specifically describes the gap between ACTUAL and PERCEIVED
    → = Can only be OBSERVED, not MEASURED PRECISELY (like Gap-Distribution §1.2)
```

### §1.3 — Gap-Distribution × Action-Space = Behavioral Prediction

```
⭐⭐ ACTUAL BEHAVIOR = f(Gap-Distribution × Action-Space):

  Gap-Distribution (demand): "What do I WANT?" → where gaps cluster
  Action-Space (supply): "What CAN I do?" → where options are available
  Behavior: INTERSECTION of both — only where gap AND option MEET → action

  4 QUADRANTS:

    ┌──────────────────────────────┬───────────────────────────────────────┐
    │ Gap-Distribution HIGH +      │ Gap-Distribution HIGH +               │
    │ Action-Space HIGH            │ Action-Space LOW                      │
    │                              │                                       │
    │ = ACTIVE PURSUIT             │ = FRUSTRATION / STUCK                 │
    │ Want + can do → action       │ Want + can't → suffering              │
    │ Example: Researcher with lab │ Example: Aspiring artist who must     │
    │ Example: Entrepreneur with   │   work a factory job                  │
    │   capital                    │ → Chronic dissonance, possible        │
    │ → Productive, confident      │   learned helplessness                │
    ├──────────────────────────────┼───────────────────────────────────────┤
    │ Gap-Distribution LOW +       │ Gap-Distribution LOW +                │
    │ Action-Space HIGH            │ Action-Space LOW                      │
    │                              │                                       │
    │ = DRIFT / WASTE              │ = SIMPLE / TRAPPED                    │
    │ Can do + don't want to       │ Don't want + can't do                 │
    │ → NO DRIVE                   │ → Narrow life, stable but constrained │
    │ Example: Wealthy child with  │ Example: Isolated rural life — few   │
    │   everything but "bored"     │   gaps, few options, simple living   │
    │ → The lottery winner case    │ → Peaceful OR invisible trap         │
    │ → "Wealthy but don't know    │                                       │
    │    what they want"           │                                       │
    └──────────────────────────────┴───────────────────────────────────────┘

  ⭐ MOST IMPORTANT QUADRANT: Gap-Distribution HIGH + Action-Space LOW
    → This is the SUFFERING quadrant
    → Body fires real gaps (demand exists) but NO FILL PATH (supply is lacking)
    → = Chronic cortisol holding (Cortisol-Baseline §7: Role ② sustained)
    → = If prolonged → learned helplessness (Maier & Seligman 2016)
    → = Education + resource access = 2 primary levers to move toward Q1

  ⭐ Gap-Distribution AND Action-Space INFLUENCE EACH OTHER:
    → Action-Space expands → new domains accessible → new chunks compile →
      new gaps form → Gap-Distribution SHIFTS
    → Gap-Distribution shifts → new drives → effort builds capacity →
      Action-Space expands
    → = FEEDBACK LOOP — not one-directional
    → Education primarily = expand Action-Space ④ (awareness) →
      trigger feedback loop


🟡 Gap-Distribution × Action-Space 4-quadrant model = framework synthesis
🟢 Behavior = f(capability × motivation): Vroom 1964 Expectancy Theory
🟢 Frustration when blocked: Dollard et al. 1939
```

### §2.1 — Axis ① Compiled Capacity (what I know, what I can do)

```
⭐ COMPILED CAPACITY = TOTAL CHUNKS: KNOWLEDGE + SKILLS + PHYSICAL ABILITY:

  DEFINITION:
    → The "arsenal" available at moment T
    → = Total compiled chunks: domain knowledge, motor skills, languages,
        social skills, physical fitness, professional expertise
    → = f(years exposure × domain fit × compile depth)

  2 SUB-DIMENSIONS:

    DEPTH (deep in one domain):
      → Years of deliberate practice/exposure → chunks compile deep
      → Deep = see patterns others miss (Chase & Simon 1973)
      → Deep = high reward when gap fills (compound effect)
      → Examples: chess master 20 years, surgeon 15 years, physicist 30 years

    BREADTH (across many domains):
      → Exposure to many domains → chunks compile across them
      → Breadth = versatility, bridge function
      → Breadth = more options per situation
      → Examples: T-shaped designer, polyglot, generalist manager

    DEPTH × BREADTH ≈ CONSTANT (Drill-Entity-Compiled-Capacity §5):
      → Finite compilation resource — deep in 1 domain = less for others
      → = Tradeoff, NOT failure
      → Specialist: narrow-deep. Generalist: broad-shallow. T-shaped: hybrid.


  ⭐ INVISIBLE BOUNDARY — MOST IMPORTANT:

    "Unknown = no gap" (Gap-Direction §5.3) applies DIRECTLY:
      → Compiled Capacity = BOTH actual capability AND the edge of perception
      → Don't know Python exists → no gap "want to learn Python"
                                  → no option "use Python to solve this"
      → = INVISIBLE CONSTRAINT (§3.3): unaware the option exists → NO SIGNAL

    IMPORTANT DISTINCTION:
      → Actual capability (compiled chunks) = axis ① → measurable
      → Awareness of capability = axis ④ → perception gap
      → Axes ① and ④ RELATED but DIFFERENT:
        ① "I know Python" (fact)
        ④ "I know that I know Python" (meta-awareness)
        ④ "I DON'T KNOW that Go language exists" (invisible boundary)


  OBSERVABLE PROXIES:
    → Education level, certifications, years of experience
    → Skills demonstrated (not just claimed)
    → Languages spoken, tools mastered
    → Physical ability, health status
    → = Proxies — each captures ONLY PART

🟢 Expertise restructures perception: Chase & Simon 1973
🟢 Depth × Breadth tradeoff: Roberts et al. 2009 (Dunbar relationship layers)
🟢 "Unknown = no gap": Gap-Direction §5.3 (established framework principle)
```

### §2.2 — Axis ② Resource Access (what I can access)

```
⭐ RESOURCE ACCESS = STATUS.MD RESOURCE ACCESS MAP EXTENDED:

  DEFINITION:
    → Status.md §1: "what someone lets me take without fighting"
    → This file REFRAMES: resource access = ALL mobilizable resources
    → = NOT JUST social status — includes material + social + knowledge access

  3 RESOURCE TYPES:

    MATERIAL:
      → Money, assets, tools, equipment, workspace
      → Money = bridge technology (Money-Analysis §1.2):
        → Standardizes cross-domain costs
        → Converts implicit → explicit obligation
        → Scales community exchange
      → = Options that MONEY CAN BUY

    SOCIAL CAPITAL (network):
      → Who is willing to help? → Dunbar layers (Entity-Compiled-Capacity §1.2)
        S1 ~5: intimate — willing to help unconditionally
        S2 ~15: close — help when asked
        S3 ~50: friends — help within their means
        S4 ~150: active — introductions, information
      → Network BREADTH + DIVERSITY (Putnam 2000):
        Bonding capital: network of SIMILAR people → emotional support
        Bridging capital: network of DIFFERENT people → access to new domains
      → = Options that NETWORK UNLOCKS

    KNOWLEDGE ACCESS (access to information):
      → Internet, libraries, mentors, peer groups
      → DIFFERENT from axis ①: this is ACCESS (can be reached), not COMPILED (already known)
      → Example: having internet = POTENTIAL access (but not yet compiled)
      → Example: having a mentor = GUIDED access (accelerates compilation)
      → = Options that INFRASTRUCTURE PERMITS


  ⭐ MONEY = MULTIPLIER FOR AXIS ②, NOT EVERYTHING:
    → Money BUYS material resources → axis ② expands
    → Money BUYS some social access (events, clubs) → axis ② partial
    → Money CANNOT BUY compiled capacity → axis ① requires TIME
    → Money CANNOT BUY freedom directly → axis ③ requires legal/social change
    → Money CANNOT BUY awareness → axis ④ requires exposure + meta-cognition
    → = Money important but NOT sufficient for full Action-Space expansion

  ⭐ DIMINISHING RETURNS (Kahneman & Deaton 2010):
    → $0 → $30K/year: massive Action-Space expansion (survival → options)
    → $30K → $100K: moderate expansion (comfort → more options)
    → $100K → $1M: marginal expansion (premium access)
    → $1M+: near-zero expansion (luxury ≠ more options)
    → = Resource access has a CEILING — more money ≠ proportionally more Action-Space


  OBSERVABLE PROXIES:
    → Income, savings, assets
    → Network size + diversity
    → Access to mentors, experts
    → Infrastructure available (internet, library, transport)

🟢 Status = Resource Access Map: Status.md §1 (established framework)
🟢 Social capital: Putnam 2000, Bourdieu 1986
🟢 Income ↔ well-being plateau: Kahneman & Deaton 2010
🟢 Dunbar layers: Zhou et al. 2005
```

### §2.3 — Axis ③ Freedom (degree of freedom to act)

```
⭐ FREEDOM = EXTERNAL CONSTRAINTS: WHAT BLOCKS ME FROM USING ① AND ②:

  DEFINITION:
    → Compliance-Floor.md: "freedom = default, law = exception"
    → This axis: TOTAL external constraints currently limiting options
    → = Know (①) + Have (②) + PERMITTED (③) = options actually available

  4 CONSTRAINT SOURCES:

    LEGAL (law):
      → Compliance-Floor 4-layer: body-base + chunks + melody space + collective
      → "No killing, stealing, fraud, trespass" → appropriate constraints
      → "No business without license" → bureaucratic constraint
      → "No leaving the country" (some regimes) → oppressive constraint
      → = NECESSARY constraints (protect others) + EXCESS (over-restrict freedom)

    SOCIAL NORMS:
      → "Women shouldn't be engineers" (in many societies)
      → "Must marry before 30" (Collective-Schema-Pressure)
      → NOT law but FUNCTIONS LIKE law:
        Violating → social punishment (Collective-Body §2: compile paths)
      → = Invisible constraint when awareness is absent, visible as awareness grows

    OBLIGATIONS:
      → Obligation.md: compiled prediction about cost to maintain access
      → "Must support aging parents" → constrains career options
      → "Bank debt" → constrains spending/risk-taking
      → "Must keep job for the family" → constrains exploration
      → = Obligation REDUCES freedom BUT provides access (tradeoff)

    DEPENDENCY:
      → Children: dependent on parents → freedom ≈ 0 (appropriate, developmental)
      → Employee: dependent on employer → freedom in work = limited
      → Debt: dependent on creditor → freedom = constrained
      → = Dependency = INVERSE of autonomy (Autonomy.md §3.1)


  ⭐ PHYSICAL CONSTRAINT = EDGE CASE:

    Modern people are generally NOT physically constrained:
      → Watch videos of the ocean → have chunks about the ocean
        (shallower than a diver's, but present)
      → Internet + travel = access to almost any domain
      → = Everything = patterns in the brain

    WHEN physical constraint IS present (imprisonment, serious illness,
    severe disability):
      → Crushes axis ② (resource access) → ≈ 0
      → Crushes axis ③ (freedom) → ≈ 0
      → Axis ① (capacity) REMAINS (knowledge is still knowledge)
      → Axis ④ (awareness) REMAINS (still knows what could be done)
      → = Already captured in 4 axes — no 5th axis needed

    Example: Stephen Hawking — ①④ very high, ②③ crushed by ALS
      BUT: technology (wheelchair, voice synthesizer) = partial restore of ②③
      → = Physical constraint ≈ crashes ②③ but technology can partially compensate


  OBSERVABLE PROXIES:
    → Legal status (citizen, visa, undocumented)
    → Debt level, financial obligations
    → Family obligations (dependent parents, children)
    → Social norm pressure (cultural context)
    → Physical freedom (health, mobility, location)

🟢 Development = freedom: Sen 1999 (Capability Approach)
🟢 Compliance Floor = harm principle: Mill 1859
🟡 4 constraint sources = framework synthesis (integrating multiple files)
```

### §2.4 — Axis ④ Awareness (perception of one's own Action-Space)

```
⭐⭐ AWARENESS = GAP BETWEEN ACTUAL ACTION SPACE AND PERCEIVED ACTION SPACE:

  DEFINITION:
    → Axes ①②③ describe ACTUAL ACTION SPACE (objective)
    → Axis ④ describes HOW MUCH YOU KNOW ABOUT YOURSELF (subjective)
    → Gap between actual and perceived = OBSERVATION TARGET of this axis

  PFC = LAWYER (Inter-Body-Mechanism §7):
    → PFC is NOT a neutral judge
    → PFC = body-base's lawyer: creates narratives defending actions body ALREADY wants
    → → Self-assessment = BIASED — always
    → = Why axis ④ is SEPARATED from ①②③


  3 STATES:

    UNDER-ESTIMATE: "I can't" but actually CAN
      → Learned helplessness: vmPFC atrophied → DRN not suppressed →
        default "helpless" NOT OVERRIDDEN (Maier & Seligman 2016)
      → Compiled suppression: "acting independently = punishment"
        compiled from childhood (Autonomy.md §1.2: Child B scolded for trying)
      → → Actual Action-Space wider than perceived Action-Space → options BYPASSED
      → → Body STILL fires reward for self-action (hardware) but
        software override → "know I should act but don't dare"

    ACCURATE: perception ≈ reality — RARE
      → Requires: broad domain exposure + meta-cognition + feedback from domain
      → = "Knowing what I know, knowing what I DON'T know" (Socratic paradox)
      → = Optimal but ASYMPTOTIC — no one is 100% accurate

    OVER-ESTIMATE: "I'm sure I can" but chunks are INSUFFICIENT
      → Dunning-Kruger 1999: low competence + low meta-cognition →
        "I'm good" (because they don't know what they're missing)
      → Recent graduate: "4 years of university = expert" → reality shock
      → → Perceived Action-Space wider than actual Action-Space → TRY but FAIL →
        recalibrate
      → → Failure = PAINFUL but = recalibration mechanism


  ⭐⭐ INVISIBLE CONSTRAINT: ESPECIALLY IMPORTANT

    Axis ④ connects with §3.3 (Invisible constraint):

    Not knowing an option EXISTS = NO GAP for it:
      → Unaware of a scholarship → don't apply → opportunity missed
      → Unaware of online learning → don't study → capacity stuck
      → Unaware of career path X → don't pursue → life path limited

    WHEN awareness INCREASES → invisible BECOMES visible:
      → "Oh, there's a way to do this?" → option SUDDENLY available
      → 1 conversation → 1 book → 1 video → axis ④ expands SUDDENLY
      → = CHEAPEST to expand (no years required) but HARDEST to trigger
        (invisible = don't know what to search for)

    EDUCATION = TURNING INVISIBLE → VISIBLE:
      → Education does NOT only teach skills (axis ①)
      → Education PRIMARILY OPENS EYES (axis ④):
        "So this domain exists. This career path exists."
      → → Invisible constraint → Blocked/Dependent → SOLVABLE
      → → = WHY education = the most powerful intervention


  OBSERVABLE PROXIES:
    → Match between self-assessment and external assessment
    → Response to new information: "Oh, this is possible?" = expanding
    → Domain exposure breadth (how many fields they know EXIST)
    → Willingness to seek feedback + update self-model

🟢 Dunning-Kruger 1999: low competence → overestimate
🟢 Maier & Seligman 2016: learned helplessness = DEFAULT, vmPFC learns controllable
🟢 Bandura 1977: self-efficacy → behavior prediction
```

### §3.1 — Blocked: knows but not permitted

```
⭐ BLOCKED = FRUSTRATION:

  Mechanism:
    → Person KNOWS option X exists (axis ④ has awareness)
    → Person HAS capacity/resource for X (axes ①② sufficient)
    → BUT axis ③ BLOCKS: law prohibits, norm prohibits, others prevent
    → → Gap EXISTS + fill path EXISTS + BLOCKED = frustration

  EXAMPLES:
    → Wants to start a business but private enterprise is restricted
      (Vietnam before Đổi Mới 1986 economic reform)
    → Wants to pursue higher education but "daughters don't need to study" (norm)
    → Wants to leave a marriage but "think of the children" (obligation)
    → Wants to enter public life but "no connections" (status gate)

  BODY RESPONSE:
    → Frustration = cortisol sustained (gap fires + fill path blocked)
    → Over time: can → protest / circumvent / suppress
    → 🟢 Dollard et al. 1939: frustration → aggression hypothesis

  ⭐ BLOCKED = HAS ENERGY → can be RESOLVED:
    → Because they know the option + know they're blocked → HAS TARGET to address
    → Lobbying, advocacy, legal challenge, migration = strategies
    → = SOLVABLE (though costly)
```

### §3.2 — Dependent: knows but conditions not yet met

```
⭐ DEPENDENT = TEMPORARY ACCEPTANCE:

  Mechanism:
    → Person KNOWS option X exists (axis ④ has awareness)
    → Person LACKS sufficient resource for X (axis ② lacking)
    → OR hasn't yet built sufficient capacity (axis ① lacking)
    → → Gap EXISTS + fill path EXISTS + CONDITIONS NOT YET MET = temporary acceptance

  EXAMPLES:
    → Wants to open a restaurant but doesn't have capital yet → save → wait
    → Wants to pursue a PhD but needs more foundational knowledge → study → wait
    → Wants to leave home but not yet of age or financial independence → build → wait

  BODY RESPONSE:
    → Cortisol MODERATE (gap fires + fill path known + timeline unclear)
    → PFC can PLAN: "if I save X months → enough"
    → → Cortisol DECREASES when plan exists (certainty ↑)

  ⭐ DEPENDENT = HAS DIRECTION → MOTIVATING:
    → Knows the option + knows what's lacking → HAS DIRECTION to build toward
    → = Source of drive, not despair
    → Education, mentorship, resource building = strategies
    → = SOLVABLE given TIME
```

### §3.3 — Invisible: doesn't know the option exists

```
⭐⭐ INVISIBLE = MOST DANGEROUS — BECAUSE THERE IS NO SIGNAL AT ALL:

  Mechanism:
    → Person DOESN'T KNOW option X exists (axis ④ = 0 for X)
    → → "Unknown = no gap" (Gap-Direction §5.3)
    → → NO GAP for X → NO DRIVE toward X
    → → NO FRUSTRATION → NO ENERGY TO SEEK
    → → = "FALSE PEACE" — doesn't know they are constrained

  EXAMPLES:
    → 18th-century farmer: unaware "voting rights" exist → doesn't demand them
    → Controlled child: unaware "children have rights" → accepts the situation
    → Factory worker: unaware "free online courses" exist → doesn't learn
    → Patient: unaware "therapy X" exists → doesn't seek it

  BODY RESPONSE:
    → NONE. This is the PROBLEM.
    → Frustrated person SEEKS a solution (has energy).
    → Dependent person BUILDS toward a solution (has direction).
    → Invisible person DOES NOTHING — because there is NO GAP.
    → = WHY invisible is MOST DANGEROUS:
      Blocked = pain → find a way
      Dependent = wait → build
      Invisible = false peace → stuck FOREVER (unless someone "opens their eyes")

  ⭐ INVISIBLE → VISIBLE: THE "OH" MOMENT:
    → 1 conversation: "Did you know there's a study-abroad scholarship?"
      → ④ expands → gap FORMS → frustration/motivation BEGINS
    → 1 book: "This career path exists"
      → ④ expands → new option VISIBLE → decision now possible
    → Internet exposure: watching videos about other ways of living
      → ④ expands → awareness: "life could be different like this"
    → = CHEAPEST intervention (cost = 1 conversation)
      BUT HARDEST to trigger (invisible = can't search for what you don't know)

  ⭐ EDUCATION = ANTI-INVISIBLE MECHANISM:
    → Education does NOT only teach skills (axis ①)
    → Education PRIMARILY converts INVISIBLE → VISIBLE (axis ④)
    → → Options INVISIBLE → BLOCKED or DEPENDENT
    → → BLOCKED/DEPENDENT = SOLVABLE (because they now know the problem + direction)
    → → = WHY "education is the foundation of development" (Sen 1999)

🟢 "Unknown = no gap": Gap-Direction §5.3 (established)
🟢 Capability = opportunity set: Sen 1999 (Nobel)
🟡 Invisible constraint taxonomy + "false peace" = framework synthesis
🔴 Invisible constraint = most dangerous (logical, untested empirically)
```

## §4 — SPIRAL DYNAMICS

### §4.1 — Positive spiral (privilege, compound advantage)

```

  ① → ②: Knowledge EXPANDS resource access
    → Speaks English → wider market → income + network
    → Knows law → negotiates better → accesses legal resources

  ② → ①: Resource EXPANDS knowledge
    → Has money → accesses education → compiles faster
    → Has network → mentors → guided compilation → faster + deeper
    → Has tools → practices efficiently → compiles better

  ③ → ①②: Freedom ENABLES use of ① and ②
    → Time freedom → uses own capacity → builds more
    → Financial freedom → invests resources → grows portfolio
    → = Imprisoned → ①② become meaningless (capacity still there but unusable)

  ④ → ①②③: Awareness ACTIVATES everything
    → Knows options exist → seeks capacity, resources, freedom
    → Knows they can → USES ①②③ instead of ignoring them
    → = Not knowing they can → stuck (even if ①②③ are sufficient)

    ① ↗ → ② ↗ → ③ ↗ → ④ ↗ → ① ↗↗ → ② ↗↗ ...
```

### §6.4 — Axis ④ (Awareness) shift: SUDDEN POSSIBLE

### §7.1 — Core paradox: more awareness ≠ more capability

```
⭐⭐ PARADOX: EXPANDING ACTION SPACE IS NOT ALWAYS BENEFICIAL.

  §4 describes positive spiral: 4 axes rise → compound advantage.
  §6 describes shift mechanisms: how each axis expands.
  BUT: HOW you expand matters more than HOW MUCH.

  EXAMPLE:
    Person A: reads widely, knows IT, finance, design, marketing, cooking,
    music... each studied for a week then jumps to the next.
    → Axis ④ (awareness) WIDE: knows 10 domains exist
    → Axis ① (capacity) SHALLOW: each domain ≈ 0 actual execution ability
    → = KNOWS 100 DOORS, CAN OPEN 0

    Person B: same time → compiles DEEP in 1 domain only (coding).
    → Axis ④ NARROW: knows only 1-2 domains
    → Axis ① DEEP: coding proficient → CAN EXECUTE
    → = KNOWS 2 DOORS, CAN OPEN 1

    WHO ACHIEVES MORE?
    → Person B: fewer options but HAS REAL CAPACITY → acts → reward → compounds
    → Person A: more options but NO CAPACITY → paralysis → frustration

  ⭐ THE AWARENESS PARADOX:
    → Axis ④ expands → more domains VISIBLE → more gaps FORM
      (because "knowing a domain exists = gap begins forming" — Gap-Direction §5.3 reversed)
    → But axis ① does NOT expand at the same speed (compilation takes YEARS — §6.1)
    → = Awareness OPENS FAST, capacity builds SLOW
    → = Gap-Distribution expands (more gaps) + Action-Space mismatch
      (shallow capacity) = FRUSTRATION
    → = WORSE than invisible constraint:
      Invisible: don't know → false peace (§3.3)
      Broad-shallow: knows EVERYTHING they DON'T HAVE → chronic frustration

🟡 Awareness-capacity mismatch = framework synthesis
🟢 Choice overload → paralysis: Schwartz 2004, Iyengar & Lepper 2000
```

### §7.2 — Threshold as gate: "potential" vs "real" action space

```
⭐ NOT ALL AWARENESS IS "REAL" ACTION SPACE:

  Domain-Mapping-Drive §2.3 + Gap-Distribution §6.1 established:
    → THRESHOLD = critical mass of chunks needed for a gap to FORM
    → BEFORE threshold: no gap → no drive → no reward →
      PFC must FORCE every time
    → AFTER threshold: gap formed → drive → reward → SELF-SUSTAINING

  APPLIED TO ACTION SPACE:

    POTENTIAL Action-Space (pre-threshold):
      → Knows domain exists (axis ④ has it)
      → But hasn't compiled enough chunks to EXECUTE (axis ① insufficient)
      → Every attempt to use it → PFC must DRAFT from scratch → COSTLY
      → = "Knows Python exists" but can't code Python
      → = Options VISIBLE but NOT EXECUTABLE

    REAL Action-Space (post-threshold):
      → Has compiled enough → gap formed → reward loop → self-sustaining
      → Using this capacity = LOW COST (compiled, automatic)
      → = "Knows Python AND can code it"
      → = Options VISIBLE AND EXECUTABLE

  ⭐ ACTION SPACE QUALITY = POST-THRESHOLD / TOTAL AWARENESS:
    → Person A: 10 domains known, 0 post-threshold → quality ≈ 0
    → Person B: 2 domains known, 1 post-threshold → quality HIGH
    → Person C: 10 domains known, 3 post-threshold → quality GOOD (T-shaped)
    → = This file describes Action-Space SIZE (§2). This section adds Action-Space QUALITY.

🟢 Threshold → self-sustaining drive: Domain-Mapping-Drive §2.3

🟡 Potential vs Real Action-Space = framework synthesis
```

### §7.4 — Depth-first principle: roots before branches

```

    → After threshold: learning = self-sustaining drive = FREE

    → = Never crossing any threshold = every domain = costly = scattered

    → Deep compile → Background-Pattern forms "I = person of domain X"
      → STABLE IDENTITY
    → Stable identity → explore outward = LOW RISK (anchor to return to)
    → No stable identity → explore = HIGH RISK (drift, no anchor)

    → Deep compilation → meta-skills: "how to learn" not just "what I know"
    → Deep in physics → transfer pattern recognition TO biology, economics
    → Deep in code → transfer systematic thinking TO design, business
    → = Depth in 1 domain doesn't lock you there — it ARMS you for others

  ④ REWARD MEMORY SUSTAINS (Gap-Distribution §6.3 ②):
    → Deep domain → body compiles: [deep gap fill = MASSIVE opioid]

    → Scattered = no deep reward memory → no fuel → PFC must supply constantly

  ⭐ ANALOGY — THE TREE:
    → Deep roots + strong trunk → branches CAN EXPAND safely → wind won't topple it
    → No roots + branches everywhere → light wind → IT FALLS
    → Roots = depth (post-threshold domain). Branches = breadth (awareness expands).
    → = ROOTS FIRST, THEN BRANCHES.
```

## §8 — Gap-Distribution × Action-Space: BEHAVIORAL PREDICTION (extending §1.3)

```
⭐⭐ Gap-Distribution AND Action-Space INFLUENCE EACH OTHER — FEEDBACK LOOP:

  Action-Space → Gap-Distribution (supply shapes demand):
    → Action-Space expands → new domains accessible → new chunks compile →
      new gaps form → Gap-Distribution SHIFTS
    → Example: Student enters a lab → accesses new domain → chunks compile →
      gap "want to research more deeply" FORMS → Gap-Distribution shifts
      toward abstract
    → = Action space expansion CREATES new demand

  Gap-Distribution → Action-Space (demand shapes supply):
    → Gap-Distribution shifts → new gaps → drive to build capacity →
      Action-Space expands
    → Example: Gap "want to code well" → invest time learning → capacity ↑ →
      Action-Space expands
    → = Demand DRIVES supply expansion

  → = BIDIRECTIONAL FEEDBACK LOOP
  → = WHY initial conditions matter (Heckman 2006):
    Early Action-Space expansion → early Gap-Distribution shift → early feedback loop
      → compound advantage
    Late Action-Space expansion → late Gap-Distribution shift → late start →
      catching up is harder


  ⭐ Gap-Distribution × Action-Space MISMATCH = CORE OF MANY LIFE PROBLEMS:

    MISMATCH TYPE 1: "Want but can't"
      → Gap-Distribution: gap clear + fires strongly
      → Action-Space: options insufficient
      → = Chronic frustration → learned helplessness risk
      → Fix: expand Action-Space (education, resources, awareness)

    MISMATCH TYPE 2: "Can but don't want to"
      → Action-Space: options abundant
      → Gap-Distribution: gaps don't match available options
      → = Drift, waste, "bored despite having everything"
      → Fix: expose to new domains → Gap-Distribution shifts → gaps match Action-Space

    ALIGNED: "Want AND can"
      → Gap-Distribution gaps match Action-Space options → action → reward → compound
      → = Most productive, most fulfilling
      → = Flow state prerequisites: skill ≈ challenge (Csikszentmihalyi 1990)


🟡 Gap-Distribution ↔ Action-Space bidirectional feedback = framework synthesis
🟢 Flow: skill ≈ challenge: Csikszentmihalyi 1990
🟢 Initial conditions compound: Heckman 2006
```
