## §2 — "Why Mini Dissonance Is Constant" (5 Daily Cases)

### §2.0 — Meta-question + answer preview

**User's question** (§0.5.2):
> "Why does mini dissonance always exist?"

**Framework answer** 🟡:

> Mini dissonance = **byproduct** of the unconscious Imagine-Final scan (95%) (P1 §5.5 references Imagine-Final.md §1).
>
> The body's unconscious system is CONSTANTLY:
> 1. Receiving sensory input (Step 1 — interface loop)
> 2. Comparing against expected state (Imagine-Final at all scales)
> 3. Generating a gap signal when mismatch is detected
>
> → **Continuous scan → continuous gap detection → continuous mini dissonance**.

**Why the scan is constant**:
- Evolutionary advantage: organisms that detect gaps early survive better
- Homeostasis maintenance: body inputs drift constantly (temperature, glucose, social status, domain accuracy)
- Schema coherence check: new input is continuously checked against compiled patterns

**Mini ≠ pathological.** Mini dissonance is **healthy** — it signals that the body's coherence-monitoring system is working correctly.

**What IS pathological**:
- Mini dissonance that **accumulates without resolution** → becomes chronic
- Mini dissonance that **cannot be identified** (self-signal interoception failure, P1 §5.4)
- Mini dissonance that **overwhelms PFC capacity** (too many gaps simultaneously → overwhelm)

**5 cases decomposed below** — each reveals a different source mechanism of mini dissonance.

### §2.1 — Case B1.1: Yesterday was hot, today is cool — thinking about the awning

**User's scenario**:
> "Yesterday was really hot. Today it's cool — but who knows when it'll get hot again. Maybe I should redesign the awning or buy another fan."

**Mechanism decomposition** 🟡:

**Phase 1 — Past discomfort trace (yesterday's heat)**:
- Thermoreception (Body-Input-Enumeration §4.1) detected L1 body-state delta above baseline
- Cortisol elevated during heat exposure (coping response)
- **Chunk compile**: "hot day = uncomfortable" schema strengthened
- Post-event: schema persists in memory even after the environment cooled

**Phase 2 — Current relief (today's cool temperature)**:
- Thermoreception back to baseline
- Body-feedback layer: no thermal dissonance currently
- BUT: schema "hot = uncomfortable" was recently fired → residual activation remains

**Phase 3 — PFC anticipation (unknown when it will get hot again)**:
- PFC engages (Step 2 in the interface loop, dissonance threshold reached)
- **Imagine-Final simulates**: future hot-day preview
- **Body pre-feels**: opioid? NO. Mild cortisol preview (anticipation threat — Threat-Drive-Analysis §3.3 anticipation source)
- **Key**: Pre-feel is at ~20–60% fidelity (Imagination-Analysis) — weaker than real, but **body still responds**

**Phase 4 — Resolution seeking (redesigning the awning)**:
- PFC generates Imagine-Final candidate: "buy a fan / redesign the awning"
- **Anchor-Schema §1 flow**: scan compiled anchors (body-base thermal comfort) → activate → PFC generates solution options → body preview
- **Dissonance partially resolves** when the candidate Imagine-Final produces an opioid preview

| Anticipation cortisol | Threat source 3 (anticipation) | Threat-Drive-Analysis §3 |

**Why this is mini dissonance (not full threat)**:
- Current state: comfortable (no acute threat)
- Intensity: ~5–15% body resource ("nagging unease" threshold — micro dissonance)
- Substrate: anticipation loop (PFC level), no amygdala engagement
- Resolution: Imagine-Final candidate → mini reward (opioid preview) → dissonance resolves

**Key mechanism insight**: **Mini dissonance doesn't require a current problem**. Memory + anticipation + Imagine-Final preview → body responds AS IF the problem is still present (~20–60% fidelity).

### §2.2 — Case B1.2: Beautiful but expensive — let's not look anymore

**User's scenario**:
> "Walking past a clothing store, I saw an outfit that looked beautiful — but it was expensive. Let's not look anymore."

**Mechanism decomposition** 🟡:

**Phase 1 — Visual input triggers**:
- Walk past store → exteroception visual input
- The outfit on display → pattern fire → activates compiled chunks: "beautiful clothes," "style," "aesthetic appreciation"
- **Spreading activation** (Chunk.md §4 + Collins & Loftus 1975): pattern cascade triggers related schemas

**Phase 2 — Desire schema fires**:
- **Imagine-Final micro-scale** (Imagine-Final §1 Micro: seconds→minutes): "wearing this outfit → looking great → feeling confident"
- **Body pre-feels** opioid preview (~20–60% fidelity): "this will be pleasant"
- **prediction-delta detection** (PFC-Analysis §8.3 step 2): unexpected beautiful item → dopamine alert → PFC attention

**Phase 3 — Resource check (reality intrudes)**:
- PFC checks domain (§6 P1: PFC is THE ONLY system that checks domain accuracy)
- "Expensive" = domain constraint — cost exceeds resource schema
- **Imagine-Final candidate "buy this outfit"** evaluated against resource reality
- **Anchor-Schema §2 Trust ≥ Cost principle**: desire schema strength < resource schema strength → anchor doesn't form

**Phase 4 — Conflict → dismiss → residual**:
- Two competing schemas fire:
  - Desire: "want it"
  - Resource/prudent: "can't afford it"
- PFC executes "dismiss" (walk away, don't look)
- **Residual**: desire schema still has trace activation (not fully suppressed)
- **Nagging-unease residual**: micro-mismatch at the mildest threshold — does not require action

**Framework mapping**:

| Phase | Mechanism | Substrate |
|---|---|---|
| Visual trigger | Exteroception → chunks spreading activation | Body-Input-Enumeration §2 + Chunk.md §4 |
| Desire schema | Imagine-Final micro-scale preview | Imagine-Final §1 Micro |
| Opioid pre-feel | Body-base simulation response | PFC-Analysis §8.3 step 5 |
| Resource check | PFC domain verification | Chunk.md §3 PFC functions |
| Trust vs Cost | Anchor-Schema binding principle | Anchor-Schema §2 |
| Nagging-unease residual | Micro dissonance level | Chunk-Miss variant ⓐ (explicit) |

**Why this case is classic mini dissonance**:
- **Sub-threshold for PFC engagement**: "let's not look anymore" = PFC actively suppresses → mini signal remains but below conscious attention
- **Stays at Feel-Consciousification — Feel-Observation in the 7-layer model** (P1 §5.3): body fires signal, PFC notices briefly, doesn't label deeply, doesn't explain
- **Resolution via avoidance**: walk past = input removed → desire schema deactivates → residual dissonance fades within minutes

**User's specific insight**: "Let's not look anymore" is an **avoidance strategy** — it prevents further spreading activation of the desire schema. Very different from "pretend it doesn't exist" (suppression). It's smart resource management.

**Key mechanism insight**: **Mini dissonance can be designed to be self-resolving** when avoidance is possible. The brain evolved this — it prevents chasing unfeasible Imagine-Finals (Anchor-Schema §2 Trust < Cost → anchor collapses).

**Dark side**: The same mechanism can lead to **chronic avoidance** when applied to important matters. "Don't look" at debt, at health, at a failing relationship → short-term mini dissonance relief, long-term problem accumulation.

**Phase 4 — Imagine-Final BUILD (Phase 1 per §1.5)**:

### §2.6 — Meta-mechanism: Why mini dissonance is CONSTANT

Consolidating across all 5 cases:

> Mini dissonance is **not an aberration** — it is a **byproduct** of the healthy body-domain-schema integration system running continuously.

**3 sources of constant mini dissonance**:

**Source A — Historical traces** (B1.1 heat, B1.5 salary):
- Past experiences compile schemas
- Schemas fire when triggered → memory-based dissonance
- **Continuous** because memory doesn't fade quickly enough to prevent activation

**Source B — Imagine-Final spawning** (B1.2 outfit, B1.3 flowers):
- Novel exposure → new Imagine-Finals born
- Any new Imagine-Final → gap with current state
- **Continuous** because novel inputs happen daily (modern environment especially)

**Source C — Social comparison** (B1.4 Friend C, B1.5 salary):
- Other humans observed → Self-Pattern-Modeling triggers
- Status gaps detected → dissonance
- **Continuous** because the social environment is always ambient

**Union of 3 sources** = **constant background mini dissonance** in any non-isolated human life.

**Is this a bug or a feature?**

🟡 **Feature argument**:
- Without it: no growth, no adaptation, no ambition
- Drives interface loop iteration (Step 6 feedback)
- Signals the body's health (dissonance-free life = depression, alexithymia, or dissociation)
- Source of creativity (B1.3: spontaneous Imagine-Final birth)

🟡 **Bug risk**:
- Accumulation without resolution → chronic cortisol (Cortisol-Baseline §2)
- Modern environment over-stimulates all 3 sources → signal overload
- If self-signal interoception is weak → body fires but PFC cannot read → silent wear (P1 §5.4)

**Resolution, NOT elimination**:
- Fix mismatch (source-based) → signal clears (Cortisol-Baseline §2.5)
- Accept what cannot be fixed → schema update (e.g., "D earns more because the domain is different")
- Avoidance when appropriate (B1.2 "let's not look anymore" was smart)
- Meditation → train Feel-Consciousification — Feel-Observation felt sense access → better signal management
- Reduce modern overstimulation (fewer novel Imagine-Final spawn triggers)

**→ User's question answered**: Mini dissonance exists because the body-domain-schema system is working. The question is not "how to eliminate it" but "how to respond wisely."

## §3 — 3 Genuine Discomfort Sources (Cortisol-Baseline §2.5 Integration)

### §3.1 — Critical Reframe: Cortisol Does NOT Cause Pain

🟡 **Framework critical reframe** (Cortisol-Baseline §2.5):

> **Cortisol ≠ source of pain or discomfort**. Cortisol is a **response and amplifier**, not a cause.
>
> Misconception: "cortisol = stress hormone = bad" → widespread, even in research papers.
> Reality: cortisol arrives AFTER mismatch/damage is detected. Fix the source → cortisol self-reduces.

**Evidence** (Cortisol-Baseline §2.5):

🟢 **Cortisol injection** (healthy subjects): cortisol ↑ → NO pain or discomfort reported. Possibly some alertness. NOT "pain."

🟢 **Addison's disease** (cortisol ≈ zero): patients experience severe discomfort (chronic fatigue, weakness). → If cortisol were the source, zero cortisol would equal comfort. It's the opposite.

🟢 **Cushing's syndrome** (chronically high cortisol): mood changes (irritability), but "mood change" ≠ "pain."

**Implication**: "Reduce cortisol" is the wrong strategy. "Fix mismatch" is the right strategy.

### §3.2 — The 3 Real Sources

🟡 Framework commit — 3 actual sources of discomfort and pain:

#### §3.2.1 — Source 1: NOCICEPTION (physical damage)

**Mechanism**:
- Tissue damage → nociceptors detect → signal via A-delta fibers (fast, 50ms) + C fibers (slow, seconds)
- → Brain interprets as "pain"
- **Cortisol arrives AFTER** (response, not cause)

**Examples**:
- Needle prick → pain (nociceptor) → cortisol increase (response)
- Burn → pain → cortisol
- Headache → inflammation/pressure → nociceptor variants

**Block the nociceptor** (local anesthetic) → NO pain → NO cortisol increase. **This proves cortisol is a response.**

🟢 Research: established pain pathways — A-delta + C fibers (Melzack & Wall gate control theory).

#### §3.2.2 — Source 2: MISMATCH (schema ≠ reality) — PURE dissonance

**Mechanism**:
- Current schema expects state X
- Reality detected in state Y (≠ X)
- Body signal: "something is off"
- **Signal fires FIRST** → cortisol increase SECOND (to support the fix)

**Examples**:
- **Hunger**: body-need "stomach full" ≠ current "empty" → mismatch → signal → cortisol → find food
- **Being reprimanded**: self-schema "competent" ≠ feedback "incompetent" → mismatch → signal → cortisol → adjust
- **Deadline approaching**: schema "should be done" ≠ "not yet done" → mismatch → signal → cortisol → work
- **Eating something unfamiliar**: gut schema ≠ new food chemistry → mismatch → signal → cortisol → recalibrate gut

**Resolution**: Fix the mismatch → signal clears → cortisol self-reduces.

**This IS the core dissonance mechanism.** Almost all "dissonance cases" in the drill trace back to this source.

🟡 **MISMATCH SUB-TYPES** (Dissonance-Signal-Architecture.md v1.0 §4):
- **Hardware mismatch → Direct-State Dissonance**: temperature deviation, hunger, vestibular = body-state ≠ setpoint. Hardware pathways, present from birth.
- **Compiled mismatch → Evaluative Dissonance**: betrayal, career worry, identity crisis = schema ≠ reality. dACC + anterior insula, PFC modulated.
- "Mismatch" is too broad — mechanism sub-types must be classified for precision.
- Evaluative/Direct-State dimension: Dissonance-Signal-Architecture §1 (parallel to Reward-Signal-Architecture.md §1).

#### §3.2.3 — Source 3: RECALIBRATION (neurons adjusting pattern)

**Mechanism**:
- Mismatch detected → body activates **pattern change** (neurons trying new configurations)
- **Active recalibration** = temporarily unstable state
- Unstable = inherent discomfort during the process
- Analogy: muscle soreness from the gym comes FROM exertion, not from a problem

**Examples**:
- **Learning something new**: "mental strain" = neurons literally changing wiring
- **Eating something unfamiliar**: "gut unease" = gut recalibrating enzymes
- **Meeting someone new**: "awkward" = social schemas adjusting
- **Cultural transition**: "culture shock" = multiple schemas recalibrating simultaneously

**Cortisol role**: **Amplifies** recalibration speed. More cortisol → faster pattern change → MORE discomfort during the process but SHORTER duration.

**Without cortisol**: recalibration is slow → less intense discomfort but longer duration.

**Trade-off**: Intense and short vs mild and long. Body chooses based on urgency.

### §3.3 — Summary table

| Source | Mechanism | Cortisol role | Fix |
|---|---|---|---|
| **Nociception** | Nociceptors detect tissue damage | Response only (not cause) | Protect + repair the source |
| **Mismatch** | Schema ≠ reality | Kick-starts the fix | Update schema OR change reality |
| **Recalibration** | Neurons changing pattern | Amplifies speed | Endure — process ends when new pattern is stable |

### §3.4 — Implications for drill cases

**B1.1 heat/awning**: Mismatch (past thermal discomfort chunks) + Anticipation (future heat). NO active recalibration. Cortisol mild.

**B1.2 outfit**: Mismatch (desire ≠ resource). Minimal recalibration. Cortisol micro.

**B1.3 flowers**: Mismatch (new Imagine-Final ≠ current state). Imagine-Final formation = mild recalibration. Cortisol mild.

**B1.4 Friend C**: Mismatch (self ≠ simulated-self-as-C). Status schema (evaluative). Cortisol mild-moderate.

**B1.5 salary D**: Mismatch (self-value ≠ perceived market value) + potential L1 survival adjacency if money is tight. Cortisol moderate-high.

**B2.2 parent with switch (chronic)**: Mismatch (safety ≠ punishment) chronic + Recalibration attempts (PFC tries solutions, all fail) + eventual neural wear. Cortisol chronic.

**→ Almost all dissonance cases reduce to Source 2 (Mismatch) with Source 3 (Recalibration) as a common accompaniment.** Source 1 (Nociception) is specific to physical damage.

### §3.5 — Why this matters for treatment framing

🟡 **Framework implication**:

> "Reduce stress" (anti-cortisol) is the wrong frame. The right frame is "reduce mismatch."
>
> - Mismatch reduction = root cause fix
> - Cortisol reduction = symptom management

**Strategies that target mismatch reduction**:
- Solve the problem (obvious but often skipped)
- Update expectations (schema revision)
- Remove from environment (avoidance when appropriate)
- Accept what cannot be changed (acceptance = schema update about unchangeability)
- Meditation (not "reduce cortisol" but "develop better mismatch observation + tolerance")

**Strategies that ONLY reduce cortisol without fixing mismatch**:
- Substance use (alcohol, drugs)
- Dissociation
- Distraction (can help temporarily but doesn't resolve)
- **Risk**: mismatch continues → body fires again → more cortisol needed → escalation

**→ The Cortisol-Baseline reframe is foundational for all dissonance cases in the drill.**

### §4.2 — Threat = Body-Dissonance + Prediction of Harm

🟡 Framework commit (Threat.md §1):

> **Threat = Body-Dissonance + Prediction of Harm**
>
> Two simultaneous components:
> 1. **Body-Dissonance** (SIGNAL): "something is off"
> 2. **Prediction of Harm** (INTERPRETATION): unconscious/PFC predicts "if no action → bad consequence"

**Distinction from other dissonance types**:

| | Has "prediction of harm"? | Result |
|---|---|---|
| **Dissonance without prediction** | No | Mild discomfort, boredom, restlessness |
| **Dissonance + harm prediction** | Yes | **THREAT** |
| **Dissonance + opportunity prediction** | Yes, positive | **NOVELTY DRIVE** (not threat) |

🟡 Same body signal → different interpretation → different response mobilization.

### §4.3 — Why a signal instead of direct action?

**Evolutionary question**: Why does the body fire a signal (triggering PFC engagement) instead of just executing a compiled response?

🟡 **Framework answer**:

> Signal = **flexibility insurance**. Compiled responses work for familiar threats. A signal allows PFC to evaluate NOVEL threats + choose a response.

**Example**: Tiger chase vs boss reprimand at work
- Tiger: compiled response (run/fight/freeze) works
- Boss: no compiled response for "corporate politics" (recent evolution) → PFC needs to evaluate

**Body fires signal** → PFC gets attention → PFC uses chunks to craft a response. This is why Step 2 in the interface loop always includes signal generation, not just auto-response.

### §4.4 — 3 source mechanisms of threat (Threat-Drive-Analysis §3)

🟡 Framework:

Threats arise from 3 orthogonal pathways (same body signal, different upstream sources):

#### §4.4.1 — Physical threat (nociceptor + reflex)

**Pathway**: Sensory → thalamus → amygdala → reflex (bypasses cortex)
- **Speed**: 12ms (LeDoux 1996 🟢 low road)
- **Endorphin buffer**: body releases pain blockers during acute response
- **Recovery fast** after threat passes

**Examples**: fire, predator, physical attack, cliff edge

**Body state**: Acute cortisol spike + adrenaline + NE → fight/flight/freeze
**Recovery**: fast (within hours if no injury)

#### §4.4.2 — Social threat (no reflex, no endorphin buffer)

**Pathway**: Sensory (auditory/visual) → amygdala + schema → threat interpretation
- **Speed**: 200–500ms (requires schema processing)
- **NO endorphin buffer** (not designed for social pain)
- **Recovery slower**

**Examples**: Authority anger, rejection, public shame, betrayal, ostracism

**Body state**: Cortisol sustained + oxytocin ↓
**Recovery**: days–weeks (social rumination tendency)

🟢 **Eisenberger 2003**: Social pain uses the physical pain pathway (dACC activation) but has no endorphin buffer → hurts "longer."

#### §4.4.3 — Anticipation threat (PFC imagines future)

**Pathway**: PFC imagines hypothetical future → body simulates → simulated cortisol fires
- **Speed**: arbitrary (thinking-based)
- **Fidelity**: 20–60% (Imagination-Analysis-v2.md)
- **NO natural termination** (PFC can loop indefinitely)

**Examples**: Worry about a deadline, fear of future job loss, anxiety about health test results, rumination on past failures

**Body state**: Cortisol drip (continuous low-level) + low-level NE + sleep disruption
**Recovery**: depends on whether the future clarifies OR PFC stops imagining

🟡 **Key distinction**: Anticipation threat exists BECAUSE of PFC's ability to simulate futures. Animals have less anticipation threat (less future simulation capacity). Humans are prone to chronic anticipation threat → modern anxiety epidemic.

### §4.5 — Schema = Trigger, Hormone = Sustain (Threat-Drive-Analysis refined)

🟡 Framework critical refinement (Threat-Drive-Analysis §3):

> **OLD (simpler) model**: Hormones drive behavior.
>
> **NEW (refined) model**: Schema fire triggers the drive. Hormones arrive AFTER and SUSTAIN energy for the response.

**Timeline**:

```
t=0ms:    Schema fires ("TODAY IS EXAM DAY!")
t=100ms:  PFC attention
t=500ms:  NE spike → immediate energy
t=1-2s:   Adrenaline → body mobilized
t=2-5s:   Action begins
t=5-20min: Cortisol builds → sustains alert state
t=20-60min: Cortisol peak → maintenance mode
```

**Implication**: It is the schema that INITIATES, not the hormone. The hormone provides SUPPORT.

**Example**: Walking around enjoying the day → suddenly remembers "TODAY IS MY EXAM!"
- Schema fires IMMEDIATELY (instant)
- At that moment, cortisol is still low
- But you ALREADY know you need to hurry
- NE spike ~0.5–2s → energy
- Cortisol peaks later (5–20min) → sustains alertness

## §6 — Novelty → Dissonance (Goldilocks + Fatigue + Recalibration)

### §6.1 — User's question

> "Why can novelty also create dissonance? Can brief novelty still create mild dissonance (mild fatigue) — is that right?"

**Framework answer** 🟡: ✅ **YES, correct.** Novelty → dissonance through **3 simultaneous mechanisms**.

### §6.2 — Mechanism 1: Goldilocks Zone Violation

🟡 Framework (Why-Body-Knows §2):

> Body checks a new pattern against existing chunks. The match ratio determines the body response.

**3 zones**:

| Match ratio | Body response | Signal |
|---|---|---|
| **>90%** | "Too familiar → boring" | VTA habituates, dopamine low |
| **Goldilocks zone** | "Familiar enough yet novel → interesting" | Opioid reward + dopamine alert |
| **<20%** | "Too unfamiliar → uncomfortable" | Mild cortisol, uncertainty |

**Mechanism when match <20%**:
- Unconscious searches for matching patterns → finds nothing
- "Cannot decode" state → uncertainty → mild cortisol
- Body signal: "too unfamiliar, not enjoyable" (what a person might say)
- Note: "not enjoyable" for THIS person. A different person (different chunks) MIGHT find it "interesting"

**Examples**:
- First-time listener to atonal music: "sounds like noise" (match <20%)
- Someone with classical training: "beautiful" (chunks ≥40% match)
- Same stimulus → different dissonance vs reward based on chunks base

**→ Novelty dissonance when chunks base is INSUFFICIENT** to bring the match into the Goldilocks zone.

### §6.3 — Mechanism 2: PFC Processing Fatigue

🟡 Framework (Learning-Cycle §3.3 Stage 3 fatigue signal):

> Any chunk held in PFC working memory occupies the serial bottleneck + demands catecholamines. Novel chunks require MORE PFC bandwidth (not compiled to auto-fire). Extended novelty → processing load accumulates.

**Mechanism**:
- Novel input → PFC holds chunks in working memory (4±1 slots)
- Sustained hold → processing load: serial bottleneck occupied + catecholamine decline
- **Processing fatigue**: 🟢 Kurzban 2013 (opportunity cost), Arnsten 2009 (catecholamine inverted-U). Glucose model falsified: Hagger 2016 d=0.04
- **Regional blood flow**: PFC active → elevated metabolic demand

**"Brief novelty creates mild fatigue"** mechanism:
- Even brief novelty exposure requires PFC holding
- Seconds–minutes → processing load accumulates in PFC (catecholamine decline)
- Body signal: "slight mental fatigue" / "brain needs rest"
- **Intensity scales with novelty duration + magnitude + PFC capacity**

**Example**: 30-minute museum visit with lots of new art
- Not physical exertion but clear "brain fatigue"
- That's PFC processing fatigue from sustained novel input processing
- Resolution: rest, food, decrease novel input

### §6.4 — Mechanism 3: Recalibration Discomfort

🟡 Framework (Cortisol-Baseline §2.5 source 3 — integrated into §3 above):

> Neurons actively changing pattern = temporarily unstable = inherent discomfort.

**Mechanism**:
- New chunk → Hebbian LTP starting
- Existing neighboring chunks → perturbed by new wiring attempts
- **Active recalibration**: neurons trying new configurations → some fail, some succeed
- Unstable state → body "doesn't know which pattern is right" → discomfort

**Analogy**: Muscle soreness during gym. Not from a problem. From the **adaptation process**.

**Duration**: Cortisol amplifies recalibration speed. More cortisol → faster change → intense but short discomfort. Less cortisol → slow change → mild but long discomfort.

### §6.5 — All 3 mechanisms fire simultaneously in novelty

🟡 **Critical insight**: Any novel input (chunk, pattern, person, place, idea) triggers **all 3 mechanisms** to some degree:

| Mechanism | Speed | Intensity factor |
|---|---|---|
| Goldilocks | Instant (match detection) | Depends on chunks base |
| PFC fatigue | Slow buildup (seconds–minutes) | Depends on hold duration |
| Recalibration | Gradual (seconds–hours) | Depends on integration depth |

**"Brief novelty creates mild fatigue"** = all 3 mechanisms at LOW intensity combining:
- Mild Goldilocks mismatch (some parts unknown)
- Mild PFC processing load (short hold)
- Mild recalibration (shallow integration attempt)
- **Combined**: "mild fatigue" signal

**"Long novelty creates heavy fatigue"** = same mechanisms amplified:
- More mismatches to resolve
- More PFC hold → deeper catecholamine depletion
- More recalibration → more unstable neurons

**→ User's intuition correct.** Framework commits: novelty ALWAYS produces some dissonance even when also producing reward (Goldilocks middle zone). The question is the balance between reward and dissonance, not the presence of dissonance alone.

### §6.6 — Implications for drill cases

**Learning** (§7 next): 3 concurrent signals (reward + fatigue + dissonance) in Learning-Cycle all map to novelty mechanisms.

**Van Gogh** (P3 §6): chunks base insufficient → match <20% → cortisol not opioid → "just don't get it" experience.

**Flow state** (P3 §10): all 3 mechanisms MINIMIZED through optimal challenge + compiled skill → dissonance near zero while reward is active.

**Exploration / travel**: intentional novelty exposure → trades fatigue for chunks base expansion → long-term reward potential.

### §7.2 — The 3 Concurrent Signals (Learning-Cycle §2.3)

🟡 Framework commit:

> Learning = simultaneous firing of 3 signals in Stage 3:
> - **Reward** (schema match fires opioid)
> - **Fatigue** (PFC + network integration cost)
> - **Mild Dissonance** (pattern not yet synced with neighboring schemas)

**Why all 3 fire simultaneously**:
- Each measures a different dimension (Reward = value, Fatigue = cost, Dissonance = coherence)
- Body doesn't "net" them — each has an independent function
- Conscious experience: "pleasant-heavy" (pleasant but tired), "pleasant-but-not-done" (pleasant but needs follow-up)

### §7.3 — Connection to body-feedback architecture

Each of the 3 signals maps to the body-feedback layer:

**Reward signal**:
- Source: schema match → opioid (PFC-Analysis §8.3 step 5)
- Layer: Raw (body-feedback Feel-RawSignals — Feel-Integration) → PFC receives Feel-Consciousification — Feel-Observation
- Chunk tag: opioid (if novelty context) or cortisol-tinged (if threat context)

**Fatigue signal**:
- Source: PFC processing load + network load
- Layer: Body-feedback raw (energy depletion sensors) → PFC receives as "brain heavy"
- Interoception: §4.9 self-signal picks up metabolic state

**Dissonance signal**:
- Source: pattern coherence check (mismatch with neighbors)
- Layer: Body-feedback raw (insula integration) → PFC Feel-Consciousification — Feel-Observation ("something hasn't settled yet")
- Function: maintains pattern active in background for integration

### §7.4 — The 8-Stage Cycle (Learning-Cycle §3)

Full formalization in Learning-Cycle.md. Summary:

| Stage | State | Key mechanism |
|---|---|---|
| 1 Draft | Awake, PFC active | Hebbian LTP early phase, any-path connection |
| 2 Fire + schema check | Awake | Unconscious schema match → dopamine/opioid |
| 3 Concurrent signals | Awake | Reward + fatigue + mild dissonance in parallel |
| 4 Rest entry | Transition | DMN OR sleep onset |
| 5 Multi-mechanism refinement | Offline | 6+1 sleep mechanisms (SHY, replay, consolidation, REM creative, emotional decoupling, gist, dreaming) |
| 6 Output | End of sleep | Pruned + integrated pattern |
| 7 Wake | Morning | Clearer reward, less fatigue, dissonance decreased |
| 8 Iterate OR baseline | Decision | Loop OR end-state reached |

**Key insight**: Learning dissonance is **NORMAL + NECESSARY**. Not pathology. Sleep stages 4–6 resolve it via multi-mechanism refinement.

### §7.5 — Why learning dissonance is HEALTHY

🟡 Framework (Learning-Cycle §2.3 + §3.10):

- **Feature**: signal keeps pattern active for background integration → allows sleep mechanisms to refine
- **Cycle termination**: baseline shift when pattern is integrated → signal fades
- **Hormesis pattern**: dissonance + rest + repair → neural strengthening (Cortisol-Baseline §3)

**Contrast with unhealthy dissonance**:
- Trauma (B3): same signal but NO resolution path → cycle cannot terminate → signal becomes chronic → damage
- Hedonic trap (B4): schema drift faster than integration → multiple unresolved cycles → melody noise background

**Implication**: Learning dissonance and trauma dissonance use the SAME body signal. The difference is **whether a resolution path exists**, not the signal itself.

## §8 — Trauma Loop (Case B3)

### §8.1 — User's description

> "Dissonance from a heavy negative schema with no resolution path can disrupt the entire brain — the body only has one — it drafts every possible way to resolve the original schema — or after days and months of chaos, loses the original trauma schema — but generates many new dissonance patterns from the prolonged chaotic drafting. PFC drafts actively at first but gradually tires from poor-quality sleep. Creates a dangerous loop." 🟡

**Framework decomposition below** — this case extends B2.2 (parent with switch, chronic) to the full trauma trajectory.

### §8.2 — Stage 1: Initial Trauma Schema Compile

**Event**: Trauma occurs. Could be acute (single event) or cumulative (many events).

**Mechanism**:
- **Emotional peak path** (Chunk.md §2 mode 2): amygdala + NE → wires EXTREMELY FAST, once is enough
- 🟢 Brown & Kulik 1977 flashbulb memories
- Schema compiles deep in a single event if intensity is sufficient
- **Multiple modalities involved**: visual + auditory + body sensations + emotions all fire together
- Chunks get cortisol tag (high threat cortisol context per §3.5)

**Example substrates**:
- Abuse: schema "authority figure = danger" + "my body = not safe"
- Loss: schema "connection = pain to come" + "I cannot protect those I love"
- Accident: schema "safety = illusion" + "world = unpredictable harm"

**Post-event state** (hours–days):
- Acute response settles (NE drops, cortisol gradually normalizes)
- Schema compiled, awaiting integration
- **If processed well** (resolution path, social support, return to safety) → normal grief/recovery
- **If NOT processed** → enters Stage 2

### §8.3 — Stage 2: PFC Active Drafting, No Resolution Path

**PFC engages to find a solution**:
- Body dissonance persists → PFC reactive engagement (Imagine-Final §1 "PFC reactive")
- PFC scans chunks, simulates scenarios, tries to build an Imagine-Final for escape

**User's key phrase**:
> "Drafts every possible way to resolve the original schema"

**What PFC tries**:
- Re-narrate the event ("maybe it was my fault" → self-blame schema)
- Plan future avoidance ("I'll never trust authority again" → over-generalization)
- Suppress memory ("don't think about it" → effortful suppression)
- Rationalize ("they had reasons" → normalization)
- Seek meaning ("this will make me stronger" → silver-lining attempt)

**Problem**: None of these fully resolves. Each draft fires but doesn't compile into a stable schema because:
- **No external feedback confirms** (Steps 5–6 of the interface loop are incomplete)
- **Contradictory drafts** fire competing schemas
- **Core schema doesn't update** (trauma schema compiled too deep via emotional peak path)

**Stage 2 duration**: days–weeks initially. Can extend if the environment provides no resolution conditions.

### §8.4 — Stage 3: Sleep Quality Degradation

**User's observation**:
> "PFC drafts actively at first but gradually tires from poor-quality sleep."

🟡 **Framework mechanism**:

**Normal sleep**:
- Phase 3 background processing (Imagine-Final §1.5) integrates the day's chunks
- REM consolidates memory + emotional decoupling (Learning-Cycle §4 mechanism 5)
- SWS does systems consolidation + gist extraction
- Cortisol low at night → repair mode active (Cortisol-Baseline §3)

**Traumatized sleep**:
- Anticipation loop still active at sleep onset → cortisol stays elevated
- Nightmares (PFC's anxious content → REM uses it for processing)
- Fragmented sleep (arousals from cortisol/NE spikes)
- Reduced SWS → less consolidation + less glymphatic clearance
- Reduced REM → less emotional decoupling
- **Cycle doesn't complete** — dissonance stays instead of fading

**User's astute phrase**:
> "Haunting dreams"

This IS framework-predictable. PFC couldn't resolve while awake → unconscious tries during sleep → doesn't resolve (no new information) → dreams reflect the unresolved state.

**Critical**: Sleep quality degradation is the **primary accelerant** of the trauma loop. Not the trauma itself, but **impaired recovery** from each day's processing.

### §8.5 — Stage 4: PFC M2 Damage + Schema Drift + New Dissonance Patterns

**This is where the loop becomes self-perpetuating.** 🟡

#### §8.5.1 — PFC M2 damage mechanism

From PFC-Analysis §8.4 + Cortisol-Baseline §3.5:

**M2 mechanism** (chronic cortisol → PFC damage):
- Cortisol baseline chronically elevated (trauma → anxiety → anticipation loop)
- 🟢 Arnsten 2009: cortisol → PFC dendrite retraction
- 🟢 McEwen 2007: chronic stress → PFC structural changes
- 🟢 Radley 2005: reversible if stress is removed, may be permanent if prolonged

**Timeline** (Cortisol-Baseline §8 + clinical observation):
- 1 week of stress: M1 temporary recover + M2 baseline shift begins
- 1 month of stress: M2 measurable
- 1+ year of stress: M2 substantial
- Childhood trauma: M2 on developing PFC → architecture permanently altered (Teicher research)

**Impact on drafting**:
- Damaged PFC → worse pattern recognition
- Worse Imagine-Final building (blurry simulations)
- Worse domain checking (confabulation increases)
- **Worse ability to solve the very problem that caused the damage**

#### §8.5.2 — Schema drift — original schema gets lost

**User's observation**:
> "Or after days and months of chaos, loses the original trauma schema — but generates many new dissonance patterns from the prolonged chaotic drafting."

🟡 **Framework explanation**:

**Schema drift mechanism**:
- Original trauma schema: specific event → compiled
- Many failed drafts fire during Stages 2–3
- Each failed draft creates **new proto-schemas** (interpretation attempts)
- Some proto-schemas partially compile (emotional peak path fires from each rumination)
- **Original schema gets overlaid** with many derived schemas
- User describes this as "loses the original schema"

**BUT**: chunks don't delete (Chunk.md §2). Original schema is still there, just buried under the drift.

**Result**: Schema landscape becomes:
- Original trauma schema (deep)
- Derived schemas: self-blame, worldview changes, trust issues, body safety concerns, relationship patterns, authority distrust, etc.
- All firing independently → triggering dissonance from many angles
- PFC cannot track all sources → feels like "I'm a mess" / "everything is wrong"

**User's key phrase**:
> "Generates many new dissonance patterns from the prolonged chaotic drafting"

**→ Dissonance pattern proliferation** = hallmark of untreated chronic trauma. Single event → many dissonance sources.

#### §8.5.3 — Self-signal interoception collapse

**From P1 §5.4** (Body-Input-Enumeration §4.9):

> Chronic trauma → dissociation → self-signal reading capacity atrophies.

**Mechanism**:
- Body signals become dangerous (they fire anxiety + panic + memory triggers)
- Protective adaptation: reduce self-signal access
- Short-term: functional (can go to work, live)
- Long-term: cannot read own state → alexithymic presentation
- **Feeling layer collapses** → can't label emotions → "not sure what I feel"

**This is the KEY mechanism** linking trauma to adult alexithymia (Bird & Cook 2013 correlation with trauma history).

### §8.7 — Why it "Cannot Be Deleted"

🟡 Framework commit (Chunk.md §2):

> No one can delete a chunk. Chunks only STRENGTHEN / WEAKEN / MODIFY — they do NOT delete.

**Applied to trauma**:
- Trauma schema compiled deep → cannot delete
- Many derived schemas compiled during drift → cannot delete
- **Accumulation only** during the loop

**What therapy does**:
- NOT delete the trauma schema
- Creates NEW schemas that **override** or **reframe**
- Re-association of trauma chunks with a safer context (Cortisol-Baseline §3.5 path 1)
- Builds new compile paths (path 2 novelty hijack, path 3 AI bridge)

**Why therapy is HARD**:
- Must fight against active feedback loop interference
- Must build schemas against a damaged PFC
- Must sustain through dissonance without collapsing
- **Slow** by necessity (chunks compile slowly, especially reassociation)

### §8.8 — Implications for Treatment Framing

🟡 **Framework-predicted effective approaches** (consistent with empirical research):

**Intervention points per stage**:

**Stage 1 (acute)**:
- Immediate safety + social support
- Prevent Stage 2 progression
- 🟢 Psychological First Aid research supports

**Stage 2 (active drafting)**:
- Guided processing (therapy)
- Help PFC draft meaningful frames (not traumatic drafts)
- Prevent unresolved drafts from compiling → CBT/EMDR approaches
- 🟢 EMDR, CPT, TF-CBT empirically supported

**Stage 3 (sleep degraded)**:
- Sleep hygiene CRITICAL
- Prazosin for nightmares (🟢 PTSD research)
- Address anticipation loop specifically
- **Sleep-first approach** — restore sleep → enable re-processing of Stages 1–2

**Stage 4 (M2 damage)**:
- Physical exercise (🟢 BDNF restoration research)
- Nutrition (PFC repair needs substrates)
- Extended time (repair takes months–years, not weeks)
- **Somatic approaches** (SE, sensorimotor psychotherapy) to rebuild self-signal interoception
- 🟢 Van der Kolk body-based therapy research

**Critical**: Most trauma therapy works at Stage 2 (drafting). Stages 3–4 require different interventions. A single-modality approach is often insufficient for full recovery.

### §8.9 — Connection to H10 preconditions

H10 predicts (from §1.3 of 01-Foundation.md): "Trauma persistence" from:
- **Precondition 1 failed**: Schema locked pending without a clear resolution path
- **Precondition 2 degraded**: Chunks base inadequate for escape simulation (PFC damaged + no new frames)
- **Precondition 5 negative**: Chunks cortisol-tagged → future retrieval triggers fear

All 5 preconditions compromised → reward cannot fire → only dissonance → self-perpetuating.

**Recovery** = restoring preconditions:
- 1: Build new schemas with resolution paths
- 2: Compile new chunks (therapy, safe experiences)
- 5: Re-associate chunks via positive context exposure

## §9 — Hedonic Trap (Cases B4.1 + B4.2)

### §9.1 — User's scenarios

**B4.1** (money trap):
> "Like wanting a lot of money — pouring all energy into making money — chunks change strongly but neural wear is also heavy — schemas aren't smoothed out long enough — melody noise in the background — leading to emptiness when the money is finally made — because the old schemas are no longer clear — body-base doesn't give reward — old friends, old emotions — all different now. I focused on making money thinking the old schemas would still be there — but the prolonged disruption changed the old schemas too much — even the finest delicacies don't taste as good anymore." 🟡

**B4.2** (beautiful house trap):
> "Like wanting a beautiful house — work hard to earn money — but once you've bought it, you feel bored with it too." 🟡

### §9.2 — Framework NEW contribution: Schema Maintenance Cost

🟡 **This is a new framework concept** introduced in this drill. Not previously formalized.

**Definition**:

> **Schema maintenance cost** = the energy + attention required to keep compiled schemas **active + usable** over time. Without ongoing activation, schemas decay (Chunk.md §2 "use it or lose it"). Multiple schemas compete for limited maintenance bandwidth.

**Core claim** 🟡:

> A human has a **schema portfolio** — many compiled schemas across domains (relationships, hobbies, values, skills, aesthetic preferences, emotional regulation, social roles). Each requires ongoing activation to stay usable. **Intense pursuit of one domain** reallocates maintenance bandwidth → other schemas drift → eventual loss.

**Why this wasn't previously formalized**:
- Learning-Cycle focuses on **single chunk formation**
- Chunk.md §2 notes decay but doesn't model portfolio-level dynamics
- Hedonic treadmill research exists but doesn't connect to schema maintenance

**This drill formalizes the connection.**

### §9.3 — Schema Portfolio Model

🟡 **Model**:

| Schema category | Maintenance mechanism | Decay time without activation |
|---|---|---|
| **Survival skills** (walking, eating, basic motor) | Physical body use | Very long (years before decay) |
| **Language + basic cognition** | Daily use | Long (years) |
| **Professional skills** | Work activation | Medium (months–years) |
| **Relationships** | Social interaction + memory retrieval | Medium-short (months) |
| **Emotional patterns** | Body state retrieval | Medium |
| **Aesthetic preferences** | Sensory exposure | Short (weeks) |
| **Hobbies** | Active practice | Short |
| **Complex cognitive frameworks** | Mental retrieval | Medium |
| **Fine-grained taste/enjoyment** | Sensory experience + schema fire | Short (atrophies quickly) |

**Critical insight**: "Fine-grained taste/enjoyment" chunks are **most fragile** because they require rich sensory-attention activation to stay calibrated.

### §9.4 — Mechanism of the Hedonic Trap (B4.1 Decomposed)

**Phase 1 — Commitment**: "Pouring all energy into making money"
- Person compiles strong Imagine-Final: "wealth = happiness"
- Anchor-Schema active: "make money"
- Domain accuracy partial: money does enable things
- **BUT**: "happiness" mapping to "money" is evolution lag (Why-Body-Knows §6 ①)

**Phase 2 — Execution**: "Chunks change strongly"
- Heavy compile in the money-making domain
- Work skills + financial knowledge + strategic thinking → strong new chunks
- **Reallocation**: maintenance bandwidth shifts to money-domain schemas

**Phase 3 — Neglect**: Other schemas lose activation
- **Friends** (B4.1 "old friends"): less time → relationship schemas lose maintenance → chunks decay
- **Emotional attunement**: less practice at noticing → self-signal interoception atrophies
- **Simple pleasures**: sensory chunks decay from underuse
- **Hobbies / aesthetic preferences**: no exposure → decay

**Phase 4 — Neural wear**: "Neural wear is heavy"
- Intense pursuit → chronic cortisol elevation (Cortisol-Baseline §3 imbalance)
- Sleep quality compromised (working late, work-worry)
- PFC M2 damage begins
- **Repair deficit accumulates**

**Phase 5 — Melody noise**: "Schemas aren't smoothed out long enough"
- User's beautiful observation
- Without sleep consolidation (Learning-Cycle §4–§6), daily chunks don't integrate cleanly
- Rough patches in the melody persist
- **"Noise" background** = unresolved fragments accumulating

**Phase 6 — Arrival**: "Money finally made"
- External Imagine-Final goal achieved
- PFC expects reward
- Anticipates the happiness the Imagine-Final promised

**Phase 7 — Emptiness**: "Emptiness"
- **Body-base check** (PFC-Analysis §8.3 step 5): money arriving → does it fit body-need?
- Body-need was NEVER "money itself" — it was what money promised (experiences, freedom, love, status, security)
- **Body checks against NEGLECTED schemas**:
  - "Friends" schema → decayed → can't enjoy
  - "Simple pleasures" schema → atrophied → "even the finest delicacies don't taste as good"
  - "Emotional connection" → lost → feel disconnected
  - "Self" schema → modified through intense work → no longer recognizes own preferences

**Result**: Body doesn't fire reward despite achievement. PFC is confused — the promise was broken. Depression / existential dread risk.

### §9.5 — Why "Even the Finest Delicacies Don't Taste as Good"

🟡 **Framework mechanism** (user's insight + framework extension):

**Taste chunks** are complex:
- Not just tongue sensors firing
- Include: attention + memory + context + social + mood integration
- "Delicious" = body match between sensory input + rich taste chunks + social/memory context

**During intense money pursuit**:
- Eating fast, distracted, transactional
- Taste chunks not richly activated
- **Decay starts**: fine-grained taste discrimination fades
- Default: fewer distinctions → fewer patterns → less rich experience

**At "arrival"**:
- Finest food presented
- Body tries to taste
- **BUT chunks are atrophied** → can't discriminate fine flavors → can't feel complexity
- Surface taste (sweet, salty, umami) registered
- Subtle taste (complexity, balance, skill, tradition) → not registered
- Result: "mediocre" / "not that good"

**This is a real neurological phenomenon**:
- 🟡 Consistent with hedonic adaptation research (Brickman 1978)
- 🟡 Consistent with "savoring" research (Bryant & Veroff 2007) — skill is required for enjoyment
- 🟢 Neuroscience: sensory cortex specialization requires ongoing training

**Implication**: **Hedonic experience requires maintained chunks.** Wealth doesn't automatically restore atrophied chunks. You can buy the best food, but the body needs the **capacity to taste** — and that capacity **decays without practice**.

### §9.6 — B4.2: Beautiful House Trap (Simpler, Same Mechanism)

**User's scenario**:
> "Like wanting a beautiful house — work hard to earn money — but once you've bought it, you feel bored with it too."

**Same mechanism as B4.1 but with a different target**:

**Phase 1**: Imagine-Final "beautiful house = happiness"
**Phases 2–5**: Same pursuit → neglect → neural wear → melody noise
**Phase 6**: Arrives at "beautiful house"
**Phase 7**: "Feel bored"

**Specific to house**:
- **Hedonic adaptation** (🟢 Brickman 1978): major purchase → initial boost → adaptation → baseline returns
- **Domain reality**: house is a physical object, utility saturates once basic needs (shelter, space, location) are met
- **Body-base need** was actually about: stability, belonging, status, comfort, a space for memory-making
- If the person didn't cultivate RELATIONSHIPS to fill the space, or experiences to make memories in it, the house is an empty shell
- **"Bored"** = body reports "the thing is here, but nothing else fits"

### §9.7 — The Deep Misunderstanding (User's Astute Framing)

**User's critical phrase**:
> "I focused on making money thinking the old schemas would still be there — but the prolonged disruption changed the old schemas too much."

🟡 **This is the exact mechanism.**

**The assumption**: Schemas are "permanent" once formed. Like files on disk that stay until deleted.

**The reality** (framework): Schemas require **ongoing activation** to stay usable. Like plants that need watering. Neglect → decay.

**The trap**:
- Person thinks "I'll pursue goal X now, return to old life later"
- Old life = decayed schemas
- "Old life" is no longer accessible — not because it's gone but because **the capacity to engage with it** has atrophied
- **The person who "returns" is not the same person who "left"**

**Philosophical implication**:
- 🟡 Heraclitus' river applies to selves, not just environments
- Can't step in the same river twice — because you're different by the time you return
- **Identity maintenance** requires continuous activation

### §9.8 — Pursuing Money ≠ Pursuing a Passion Project

🟡 Framework key distinction:

**Passion project**:
- Imagine-Final clear + aligned with body-base + domain-accurate
- Work done under **novelty cortisol** (not threat cortisol)
- Chunks compile with **opioid tag** (Cortisol-Baseline §3.5)
- Process itself rewarding → sustains without burnout
- **Melody stays smooth** because the work fits the hardware

**Generic money pursuit**:
- Imagine-Final is a proxy ("money will bring happiness") — indirect, unreliable
- Work done under **threat cortisol** (deadline, competition, fear of loss)
- Chunks compile with **cortisol tag**
- Process itself not rewarding → sustained by external goal → burnout risk
- **Melody noise accumulates** because the work violates the hardware

**Same hours of effort. Different chunk tags. Different long-term outcomes.**

**Caveat**: Money pursuit CAN be a passion for some people — investment professionals who find markets fascinating, entrepreneurs who love building, etc. The distinction isn't about money specifically but about **body-base fit**.

**Framework prediction**:
- Money pursuit aligned with body-base → sustainable + rewarding
- Money pursuit NOT aligned → hedonic trap guaranteed

### §9.9 — Implications for Reward Drill (P3 preview)

P3 §4 will analyze reward cases. The hedonic trap is the **shadow twin** of reward cases:
- **Reward case**: body fires opioid because preconditions are met
- **Hedonic trap**: body FAILS to fire opioid despite apparent achievement → because preconditions (chunks base, actual body-need) were neglected during pursuit

H10 predicts: "Reward arrives only when 5 preconditions are met." Hedonic trap = preconditions compromised during pursuit itself → arrival doesn't fire reward.

Can the hedonic trap be reversed? Once chunks decay through neglect, can they rebuild — and if so, how long does it take?

### §10.5 — Q12: Dissonance signal — two-way resolution?

Does the dissonance signal itself have a two-way resolution? Can the signal fade WITHOUT the mismatch actually being fixed (just through habituation)?

**Framework partial answer**:
- Habituation reduces signal strength (unconscious adapts)
- But the underlying mismatch remains
- Silent accumulation is possible
- Signal returns if circumstances change (trigger reactivates schema)
- → Habituation ≠ resolution. Real resolution requires schema update OR environmental change.

### §10.6 — Q13: Self-signal interoception rebuild timeline

How long does self-signal interoception take to rebuild after trauma-induced collapse? What is the minimum effective practice dose?

**Current evidence**: MBSR 8 weeks shows measurable changes (Hölzel 2011 🟢). Full recovery may take longer for trauma-related collapse vs normal developmental atrophy.
