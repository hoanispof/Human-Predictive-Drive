**User's question** (§0.5.2):
> "Tại sao luôn tồn tại mini dissonance?"

**Framework answer** 🟡:

> Mini dissonance = **byproduct** của Imagine-Final vô thức scan 95% (P1 §5.5 referenced Imagine-Final.md §1).
>
> Body's unconscious system CONSTANTLY:
> 1. Receives sensory input (Step 1 interface loop)
> 2. Compares với expected state (Imagine-Final at all scales)
> 3. Generates gap signal nếu mismatch
>
> → **Continuous scan → continuous gap detection → continuous mini dissonance**.

**Why constant scan**:
- Evolution advantage: organism that detects gaps early survives better
- Homeostasis maintenance: body inputs drift constantly (temperature, glucose, social status, domain accuracy)

- Mini dissonance that **accumulates without resolution** (→ chronic)
- Mini dissonance that **cannot be identified** (self-signal interoception failure, P1 §5.4)
- Mini dissonance that **overwhelms PFC capacity** (too many gaps simultaneously → rối)

## §3 — 3 Genuine Discomfort Sources (Cortisol-Baseline §2.5 Integration)

### §3.1 — Critical reframe: Cortisol KHÔNG gây đau

🟡 **Framework critical reframe** (Cortisol-Baseline §2.5):

> **Cortisol ≠ source of pain/discomfort**. Cortisol is **response + amplifier**, not cause.
>
> Misconception: "cortisol = stress hormone = bad" → widespread even in research papers.
> Reality: cortisol arrives AFTER mismatch/damage detected. Fix source → cortisol tự giảm.

**Evidence** (Cortisol-Baseline §2.5):

🟢 **Cortisol injection** (healthy subjects): cortisol ↑ → NO pain/discomfort reported. Maybe alertness. NOT "đau."

🟢 **Addison's disease** (cortisol ≈ zero): patients VERY uncomfortable (chronic fatigue, weakness). → If cortisol were source, zero cortisol would = comfort. It's opposite.

🟢 **Cushing's syndrome** (chronic high cortisol): mood changes (irritable), but "mood change" ≠ "pain."

**Implication**: "Reduce cortisol" is wrong strategy. "Fix mismatch" is right strategy.

### §3.2 — The 3 Real Sources

🟡 Framework commit — 3 actual sources of discomfort/pain:

#### §3.2.1 — Source 1: NOCICEPTION (physical damage)

**Mechanism**:
- Tissue damage → nociceptors detect → signal via A-delta (fast, 50ms) + C fibers (slow, seconds)
- → Brain interprets as "đau"
- **Cortisol arrives AFTER** (response, not cause)

**Examples**:
- Kim châm → đau (nociceptor) → cortisol increase (response)
- Bỏng → đau → cortisol
- Đau đầu → inflammation/pressure → nociceptor variants

**Block nociceptor** (local anesthetic) → NO pain → NO cortisol increase. **Proves cortisol is response**.

🟢 Research: established pain pathways — A-delta + C fibers (Melzack & Wall gate control theory).

#### §3.2.2 — Source 2: MISMATCH (schema ≠ reality) — THUẦN dissonance

**Mechanism**:
- Current schema expects state X
- Reality detected in state Y (≠ X)
- Body signal: "something is off"
- **Signal fires FIRST** → cortisol increase SECOND (to support fix)

**Examples**:
- **Đói**: body-need "stomach full" ≠ current "empty" → mismatch → signal → cortisol → find food
- **Bị mắng**: self-schema "competent" ≠ feedback "incompetent" → mismatch → signal → cortisol → adjust
- **Deadline approaching**: schema "should be done" ≠ "not yet done" → mismatch → signal → cortisol → work
- **Ăn món lạ**: gut schema ≠ new food chemistry → mismatch → signal → cortisol → recalibrate gut

**Resolution**: Fix mismatch → signal hết → cortisol tự giảm.

**This IS the core dissonance mechanism**. Almost all "dissonance cases" in drill trace to this source.

#### §3.2.3 — Source 3: RECALIBRATION (neurons adjusting pattern)

**Mechanism**:
- Mismatch detected → body activates **pattern change** (neurons trying new configurations)
- **Active recalibration** = temporarily unstable state
- Unstable = discomfort inherent to process
- Analogy: muscle soreness from gym is FROM exertion, not from problem

**Examples**:
- **Học cái mới**: "căng đầu" = neurons literally changing wiring
- **Ăn món lạ**: "ruột cồn cào" = gut recalibrating enzymes
- **Gặp người mới**: "awkward" = social schemas adjusting
- **Cultural transition**: "culture shock" = multiple schemas recalibrating simultaneously

**Cortisol role**: **Amplifies** recalibration speed. More cortisol → faster pattern change → MORE discomfort during process but SHORTER duration.

**Without cortisol**: recalibration slow → less intense discomfort but longer duration.

**Trade-off**: Intense short vs mild long. Body chooses based on urgency.

### §3.3 — Summary table

| Source | Mechanism | Cortisol role | Fix |
|---|---|---|---|
| **Nociception** | Nociceptors detect tissue damage | Response only (not cause) | Protect + repair source |
| **Mismatch** | Schema ≠ reality | Kick-starts fix | Update schema OR change reality |
| **Recalibration** | Neurons changing pattern | Amplifies speed | Endure — process ends when new pattern stable |

### §3.4 — Implications for drill cases

**B1.1 nóng/mái hiên**: Mismatch (past thermal discomfort chunks) + Anticipation (future nóng). NO recalibration active. Cortisol mild.

**B1.2 áo đẹp**: Mismatch (desire ≠ resource). Minimal recalibration. Cortisol micro.

**B1.3 cây hoa**: Mismatch (new Imagine-Final ≠ current). Imagine-Final formation = mild recalibration. Cortisol mild.

**B1.4 bạn C**: Mismatch (self ≠ simulated-self-as-C). L3 status schema. Cortisol mild-moderate.

**B1.5 lương D**: Mismatch (self value ≠ perceived market value) + potential L1 survival adjacent if money tight. Cortisol moderate-high.

**B2.1 hổ đuổi**: Mismatch (safety ≠ predator present) + Recalibration (escape response fires). NOCICEPTION if injured. Cortisol very high, acute.

**B2.2 mẹ cầm roi chronic**: Mismatch (safety ≠ punishment) chronic + Recalibration attempts (PFC tries solutions, all fail) + eventual neural wear. Cortisol chronic.

**B3 trauma**: All 3 sources mixed. Nociception if abuse physical. Mismatch core. Recalibration attempts that cannot stabilize.

**B4 hedonic**: Mismatch (schema drift). Recalibration effects of schema maintenance cost.

**→ Almost all dissonance cases reducible to Source 2 (Mismatch) with Source 3 (Recalibration) as common accompaniment**. Source 1 (Nociception) is specific to physical damage.

### §3.5 — Why this matters for treatment framing

🟡 **Framework implication**:

> "Reduce stress" (anti-cortisol) is wrong frame. Right frame is "reduce mismatch."
>
> - Mismatch reduction = root cause fix
> - Cortisol reduction = symptom management

**Strategies that target mismatch reduction**:
- Solve the problem (obvious but often skipped)
- Update expectations (schema revision)
- Remove from environment (avoidance when appropriate)
- Accept what cannot be changed (acceptance = schema update about unchangeability)
- Meditation (not "reduce cortisol" but "develop better mismatch observation + tolerance")

**Strategies that ONLY reduce cortisol without fixing mismatch**:
- Substance use (alcohol, drugs)
- Dissociation
- Distraction (can help temporarily but doesn't resolve)
- **Risk**: mismatch continues → body fires again → more cortisol needed → escalation

**→ Cortisol-Baseline reframe is foundational for all dissonance cases in drill**.

### §4.2 — Threat = Body-Dissonance + Prediction of Harm

🟡 Framework commit (Threat.md §1):

> **Threat = Body-Dissonance + Prediction of Harm**
>
> Two components simultaneously:
> 1. **Body-Dissonance** (SIGNAL): "something is off"
> 2. **Prediction of Harm** (INTERPRETATION): vô thức/PFC predicts "if not action → bad consequence"

**Distinction from other dissonance types**:

| | Has "prediction of harm"? | Result |
|---|---|---|
| **Dissonance without prediction** | No | Mild discomfort, boredom, restlessness |
| **Dissonance + harm prediction** | Yes | **THREAT** |
| **Dissonance + opportunity prediction** | Yes, positive | **NOVELTY DRIVE** (not threat) |

🟡 Same body signal → different interpretation → different response mobilization.

### §4.3 — Why signal instead of direct action?

**Evolutionary question**: Why body fires signal (triggering PFC engagement) instead of just executing compiled response?

🟡 **Framework answer**:

> Signal = **flexibility insurance**. Compiled responses work for familiar threats. Signal allows PFC to evaluate NOVEL threats + choose response.

**Example**: Hổ đuổi vs cảnh cáo boss at work
- Hổ: compiled response (run/fight/freeze) works
- Boss: no compiled response for "corporate politics" (recent evolution) → needs PFC to evaluate

**Body fires signal** → PFC gets attention → PFC uses chunks to craft response. This is why Step 2 in interface loop always includes signal generation, not just auto-response.

### §4.4 — 3 source mechanisms of threat (Threat-Drive-Analysis §3)

🟡 Framework:

Threats arise from 3 orthogonal pathways (same body signal, different upstream sources):

#### §4.4.1 — Physical threat (nociceptor + reflex)

**Pathway**: Sensory → thalamus → amygdala → reflex (bypasses cortex)
- **Speed**: 12ms (LeDoux 1996 🟢 low road)
- **Endorphin buffer**: body releases pain blockers during acute response
- **Recovery fast** after threat passes

**Examples**: fire, predator, physical attack, cliff edge

**Body state**: Acute cortisol spike + adrenaline + NE → fight/flight/freeze
**Recovery**: fast (within hours if no injury)

#### §4.4.2 — Social threat (no reflex, no endorphin buffer)

**Pathway**: Sensory (auditory/visual) → amygdala + schema → threat interpretation
- **Speed**: 200-500ms (requires schema processing)
- **NO endorphin buffer** (not designed for social pain)
- **Recovery slower**

**Examples**: Authority anger, rejection, public shame, betrayal, ostracism

**Body state**: Cortisol sustained + oxytocin ↓
**Recovery**: days-weeks (social rumination tendency)

🟢 **Eisenberger 2003**: Social pain uses physical pain pathway (dACC activation) but no endorphin buffer → hurts "longer."

#### §4.4.3 — Anticipation threat (PFC imagines future)

**Pathway**: PFC imagines hypothetical future → body simulates → simulated cortisol fire
- **Speed**: arbitrary (thinking-based)
- **Fidelity**: 20-60% (Imagination-Analysis-v2.md)
- **NO natural termination** (PFC can loop indefinitely)

**Examples**: Worry about deadline, fear of future job loss, anxiety about health test results, rumination on past failures

**Body state**: Cortisol drip (continuous low-level) + NE low-level + sleep disruption
**Recovery**: depends on whether future clarifies OR PFC stops imagining

🟡 **Key distinction**: Anticipation threat exists BECAUSE of PFC's ability to simulate futures. Animals have less anticipation threat (less future simulation capacity). Humans prone to anticipation threat chronic → modern anxiety epidemic.

### §4.5 — Schema = Trigger, Hormone = Sustain (Threat-Drive-Analysis refine)

🟡 Framework critical refine (Threat-Drive-Analysis §3):

> **OLD (simpler) model**: Hormones drive behavior.
>
> **NEW (refined) model**: Schema fire triggers drive. Hormones arrive AFTER and SUSTAIN energy for the response.

**Timeline**:

```
t=0ms: Schema fire ("HÔM NAY THI!")
t=100ms: PFC attention
t=500ms: NE spike → immediate energy
t=1-2s: Adrenaline → body mobilized
t=2-5s: Action begins
t=5-20min: Cortisol builds → sustains alert state
t=20-60min: Cortisol peak → maintenance mode
```

**Implication**: It's schema that INITIATES, not hormone. Hormone provides SUPPORT.

**Example**: Đang đi chơi → bỗng nhớ "HÔM NAY THI!"
- Schema fire NGAY (instant)
- At this moment, cortisol still low
- But you ALREADY know you need to hurry
- NE spike ~0.5-2s → energy
- Cortisol peak later (5-20min) → sustains alert

## §6 — Novelty → Dissonance (Goldilocks + Fatigue + Recalibration)

### §6.1 — User's question

> "Tại sao novelty cũng có thể tạo dissonance? Novelty ngắn vẫn có thể tạo dissonance nhẹ (mệt mỏi nhẹ) <= có đúng không nhỉ?"

**Framework answer** 🟡: ✅ **YES, đúng**. Novelty → dissonance through **3 simultaneous mechanisms**.

### §6.2 — Mechanism 1: Goldilocks Zone Violation

🟡 Framework (Why-Body-Knows §2):

> Body check pattern mới against existing chunks. Match ratio determines body response.

**3 zones**:

| Match ratio | Body response | Signal |
|---|---|---|
| **>90%** | "Quá quen → chán" | VTA habituate, dopamine low |
| **40-70%** | "Vừa quen vừa lạ → HAY" | Opioid reward + dopamine alert |
| **<20%** | "Quá lạ → khó chịu" | Cortisol mild, uncertainty |

**Mechanism when match <20%**:
- Vô thức searches for matching patterns → finds nothing
- "Cannot decode" state → uncertainty → cortisol mild
- Body signal: "lạ quá, dở" (phrase người này might say)
- Note: "dở" with THIS person. Person khác (different chunks) CÓ THỂ thấy "hay"

**Examples**:
- First-time atonal music listener: "nghe như noise" (match <20%)
- Someone with classical training: "đẹp" (chunks ≥40% match)
- Same stimulus → different dissonance vs reward based on chunks base

**→ Novelty dissonance when chunks base INSUFFICIENT** to bring match into Goldilocks zone.

### §6.3 — Mechanism 2: PFC Glucose Fatigue

🟡 Framework (Learning-Cycle §3.3 Stage 3 fatigue signal):

> Any chunk hold in PFC working memory consumes glucose. Novel chunks require MORE PFC bandwidth (not compiled to auto-fire). Extended novelty → metabolic cost accumulates.

**Mechanism**:
- Novel input → PFC hold chunks in working memory (4±1 slots)
- Sustained hold → glucose consumption elevated
- **Metabolic fatigue**: 🟢 Gailliot & Baumeister 2007 ego depletion debated but glucose correlate documented
- **Regional blood flow**: PFC active → elevated metabolic demand

**"Novelty ngắn mệt nhẹ"** mechanism:
- Even short novelty exposure requires PFC hold
- Seconds-minutes → slight glucose depletion in PFC region
- Body signal: "hơi mệt đầu" / "brain cần nghỉ"
- **Intensity scales with novelty duration + magnitude + PFC capacity**

**Example**: 30-minute museum visit with lots of new art
- Not physical exertion but clear "brain fatigue"
- That's PFC glucose fatigue from sustained novel input processing
- Resolution: rest, food, decrease novel input

### §6.4 — Mechanism 3: Recalibration Discomfort

🟡 Framework (Cortisol-Baseline §2.5 source 3 — integrated into §3 above):

> Neurons actively changing pattern = temporarily unstable = inherent discomfort.

**Mechanism**:
- New chunk → Hebbian LTP starting
- Existing neighboring chunks → perturbed by new wiring attempts
- **Active recalibration**: neurons trying new configurations → some fail, some succeed
- Unstable state → body "không biết pattern nào đúng" → discomfort

**Analogy**: Muscle soreness during gym. Not from problem. From **adaptation process**.

**Duration**: Cortisol amplifies recalibration speed. More cortisol → faster change → intense but short discomfort. Less cortisol → slow change → mild but long discomfort.

### §6.5 — All 3 mechanisms fire simultaneously in novelty

🟡 **Critical insight**: Any novel input (chunk, pattern, person, place, idea) triggers **all 3 mechanisms** to some degree:

| Mechanism | Speed | Intensity factor |
|---|---|---|
| Goldilocks | Instant (match detection) | Depends on chunks base |
| PFC fatigue | Slow buildup (seconds-minutes) | Depends on hold duration |
| Recalibration | Gradual (seconds-hours) | Depends on integration depth |

**"Novelty ngắn mệt nhẹ"** = all 3 mechanisms at LOW intensity combining:
- Mild Goldilocks mismatch (some parts unknown)
- Mild PFC glucose dip (short hold)
- Mild recalibration (shallow integration attempt)
- **Combined**: "mệt nhẹ" signal

**"Novelty dài mệt mạnh"** = same mechanisms amplified:
- More mismatches to resolve
- More PFC hold → deeper glucose depletion
- More recalibration → more unstable neurons

**→ User's intuition correct**. Framework commits: novelty ALWAYS produces some dissonance even when also producing reward (Goldilocks middle zone). The question is balance between reward and dissonance, not presence of dissonance alone.

### §6.6 — Implications for drill cases

**Learning** (P2 §7 next): 3 concurrent signals (reward + fatigue + dissonance) in Learning-Cycle all map to novelty mechanisms.

**Van Gogh** (P3 §6): chunks base insufficient → match <20% → cortisol not opioid → "chả hiểu" experience.

**Flow state** (P3 §10): all 3 mechanisms MINIMIZED through optimal challenge + compiled skill → dissonance near zero while reward active.

**Exploration / travel**: intentional novelty exposure → trades fatigue for chunks base expansion → long-term reward potential.

## §7 — Learning Dissonance (Cross-Ref Learning-Cycle)

### §7.2 — The 3 Concurrent Signals (Learning-Cycle §2.3)

🟡 Framework commit:

> Learning = simultaneous firing of 3 signals in Stage 3:
> - **Reward** (schema match fires opioid)
> - **Fatigue** (PFC + network integration cost)
> - **Dissonance nhẹ** (pattern not yet synced with neighboring schemas)

- Body doesn't "net" them — each has independent function
- Conscious experience: "pleasant-nặng" (pleasant nhưng mệt), "pleasant-chưa-xong" (pleasant nhưng cần follow up)

### §7.3 — Connection to body-feedback architecture

Each of 3 signals maps to body-feedback layer:

**Reward signal**:
- Source: schema match → opioid (PFC-Analysis §8.3 bước 5)
- Layer: Raw (body-feedback L1-2) → PFC observes L3-4
- Chunk tag: opioid (if novelty context) or cortisol-tinged (if threat context)

**Fatigue signal**:
- Source: PFC metabolic cost + network load
- Layer: Body-feedback raw (energy depletion sensors) → PFC observes as "brain heavy"
- Interoception: §4.9 self-signal picks up metabolic state

**Dissonance signal**:
- Source: pattern coherence check (mismatch with neighbors)
- Layer: Body-feedback raw (insula integration) → PFC L3-4 ("có gì đó chưa settle")
- Function: maintains pattern active in background for integration

### §7.4 — The 8-Stage Cycle (Learning-Cycle §3)

Full formalization in Learning-Cycle.md. Summary:

| Stage | State | Key mechanism |
|---|---|---|
| 1 Draft | Awake, PFC active | Hebbian LTP early phase, any-path connection |
| 2 Fire + schema check | Awake | Unconscious schema match → dopamine/opioid |
| 3 Concurrent signals | Awake | Reward + fatigue + dissonance nhẹ parallel |
| 4 Rest entry | Transition | DMN OR sleep onset |
| 5 Multi-mechanism refinement | Offline | 6+1 sleep mechanisms (SHY, replay, consolidation, REM creative, emotional decoupling, gist, dreaming) |
| 6 Output | End of sleep | Pruned + integrated pattern |
| 7 Wake | Morning | Clearer reward, less fatigue, dissonance decreased |
| 8 Iterate OR baseline | Decision | Loop OR end-state reached |

**Key insight**: Learning dissonance **NORMAL + NECESSARY**. Not pathology. Sleep stages 4-6 resolve it via multi-mechanism refinement.

### §7.5 — Why learning dissonance is HEALTHY

🟡 Framework (Learning-Cycle §2.3 + §3.10):

- **Feature**: signal keeps pattern active for background integration → allows sleep mechanisms to refine
- **Cycle termination**: baseline shift when pattern integrated → signal fades
- **Hormesis pattern**: dissonance + rest + repair → neural strengthening (Cortisol-Baseline §3)

**Contrast with unhealthy dissonance**:
- Trauma (B3): same signal but NO resolution path → cycle cannot terminate → signal chronic → damage
- Hedonic trap (B4): schema drift faster than integration → multiple unresolved cycles → melody noise background

**Implication**: Learning dissonance and trauma dissonance use SAME body signal. Difference is **whether resolution path exists**, not signal itself.

Learning-Cycle.md already commits 3 concurrent signals mechanism fully. This drill adds:

1. **Body-feedback vs Feeling layer mapping** (P1 §5 integration)

## §8 — Trauma Loop (Case B3)

### §8.1 — User's verbatim

> "Dissonance từ schema tiêu cực nặng nề không lối giải có thể gây xáo trộn toàn não, body chỉ có một, draft mọi cách giải quyết schema ban đầu, hoặc sau nhiều ngày nhiều tháng lộn xộn mà lost mất schema trauma ban đầu, nhưng lại sinh rất nhiều pattern dissonance mới sau quá trình draft lộn xộn kéo dài. PFC tích cực draft ban đầu nhưng sau đó mỏi dần do giấc ngủ không chất lượng. Gây loop nguy hiểm" 🟡

**Framework decomposition** below — this case extends B2.2 (mẹ cầm roi chronic) to full trauma trajectory.

### §8.2 — Stage 1: Initial Trauma Schema Compile

**Event**: Trauma occurs. Could be acute (single event) or cumulative (many events).

**Mechanism**:
- **Emotional peak path** (Chunk.md §2 mode 2): amygdala + NE → wire CỰC NHANH, 1 lần đủ
- 🟢 Brown & Kulik 1977 flashbulb memories
- Schema compiles deep in 1 event if intensity sufficient
- **Multiple modalities involved**: visual + auditory + body sensations + emotions all fire together
- Chunks get cortisol tag (high threat cortisol context per §3.5)

**Example substrates**:
- Abuse: schema "authority figure = danger" + "my body = not safe"
- Loss: schema "connection = pain to come" + "I cannot protect those I love"
- Accident: schema "safety = illusion" + "world = unpredictable harm"

**Post-event state** (hours-days):
- Acute response settles (NE drops, cortisol gradually normalizes)
- Schema compiled, awaits integration
- **If processed well** (resolution path, social support, safety return) → normal grief/recovery
- **If NOT processed** → enters Stage 2

### §8.3 — Stage 2: PFC Active Draft, No Resolution Path

**PFC engages to find solution**:
- Body dissonance persistent → PFC reactive engagement (Imagine-Final §1 "PFC reactive")
- PFC scans chunks, simulates scenarios, tries to build Imagine-Final for escape

**User's key phrase**:
> "Draft mọi cách giải quyết schema ban đầu"

**What PFC tries**:
- Re-narrate event ("maybe it was my fault" → self-blame schema)
- Plan future avoidance ("I'll never trust authority again" → over-generalization)
- Suppress memory ("don't think about it" → effortful suppression)
- Rationalize ("they had reasons" → normalization)
- Seek meaning ("this will make me stronger" → silver-lining attempt)

**Problem**: None of these fully resolve. Each draft fires but doesn't compile into stable schema because:
- **No external feedback confirms** (Step 5-6 interface loop incomplete)
- **Contradictory drafts** fire competing schemas
- **Core schema doesn't update** (trauma schema compiled too deep by emotional peak path)

**Stage 2 duration**: days-weeks initially. Can extend if environment doesn't provide resolution conditions.

### §8.4 — Stage 3: Sleep Quality Degradation

**User's verbatim**:
> "PFC tích cực draft ban đầu nhưng sau đó mỏi dần do giấc ngủ không chất lượng"

🟡 **Framework mechanism**:

**Normal sleep**: 
- Phase 3 background processing (Imagine-Final §1.5) integrates day's chunks
- REM consolidates memory + emotional decoupling (Learning-Cycle §4 mechanism 5)
- SWS does systems consolidation + gist extraction
- Cortisol low at night → repair mode active (Cortisol-Baseline §3)

**Traumatized sleep**:
- Anticipation loop still active at sleep onset → cortisol stays elevated
- Nightmares (PFC's anxious content → REM uses it for processing)
- Fragmented sleep (arousals from cortisol/NE spikes)
- Reduced SWS → less consolidation + less glymphatic clearance
- Reduced REM → less emotional decoupling
- **Cycle doesn't complete** — dissonance stays instead of fading

**User's astute**:
> "Mơ ngủ ám ảnh"

This IS framework-predictable. PFC couldn't resolve while awake → unconscious tries during sleep → doesn't resolve (no new information) → dreams reflect unresolved state.

**Critical**: Sleep quality degradation is **primary accelerant** of trauma loop. Not the trauma itself but **impaired recovery** from each day's processing.

### §8.5 — Stage 4: PFC M2 Damage + Schema Drift + New Dissonance Patterns

**This is where loop becomes self-perpetuating** 🟡.

#### §8.5.1 — PFC M2 damage mechanism

From PFC-Analysis §8.4 + Cortisol-Baseline §3.5:

**M2 mechanism** (chronic cortisol → PFC damage):
- Cortisol baseline elevated chronically (trauma → anxiety → anticipation loop)
- 🟢 Arnsten 2009: cortisol → PFC dendrite retraction
- 🟢 McEwen 2007: chronic stress → PFC structural changes
- 🟢 Radley 2005: reversible if stress removed, may be permanent if prolonged

**Timeline** (Cortisol-Baseline §8 + clinical observation):
- Stress 1 week: M1 temporary recover + M2 baseline shift begins
- Stress 1 month: M2 measurable
- Stress 1 year+: M2 substantial
- Childhood trauma: M2 on developing PFC → architecture permanently altered (Teicher research)

**Impact on drafting**:
- Damaged PFC → worse pattern recognition
- Worse Imagine-Final building (blurry simulations)
- Worse domain checking (confabulation increase)
- **Worse ability to solve the very problem that caused damage**

#### §8.5.2 — Schema drift — lost mất schema ban đầu

**User's verbatim**:
> "Sau nhiều ngày nhiều tháng lộn xộn mà lost mất schema trauma ban đầu, nhưng lại sinh rất nhiều pattern dissonance mới sau quá trình draft lộn xộn kéo dài"

🟡 **Framework explanation**:

**Schema drift mechanism**:
- Original trauma schema: specific event → compiled
- Many failed drafts fire during Stage 2-3
- Each failed draft creates **new proto-schemas** (attempts at interpretation)
- Some proto-schemas compile partially (emotional peak path fires from each rumination)
- **Original schema gets overlaid** with many derived schemas
- User describes this as "lost mất schema ban đầu"

**BUT**: chunks don't delete (Chunk.md §2). Original schema still there, just buried under drift.

**Result**: Schema landscape becomes:
- Original trauma schema (deep)
- Derived schemas: self-blame, worldview, trust issues, body safety, relationship patterns, authority distrust, etc.
- All firing independently → triggering dissonance from many angles
- PFC cannot track all sources → feels "I'm a mess" / "everything is wrong"

**User's key phrase**:
> "Sinh rất nhiều pattern dissonance mới sau quá trình draft lộn xộn kéo dài"

**→ Dissonance pattern proliferation** = hallmark of untreated chronic trauma. Single event → many dissonance sources.

#### §8.5.3 — Self-signal interoception collapse

**From P1 §5.4** (Body-Input-Enumeration §4.9):

> Chronic trauma → dissociation → self-signal reading capacity atrophies.

**Mechanism**:
- Body signals become dangerous (fire anxiety + panic + memory triggers)
- Protective: reduce self-signal access
- Short-term: functional (can go to work, live)
- Long-term: cannot read own state → alexithymic presentation
- **Feeling layer collapses** → can't label emotions → "not sure what I feel"

**This is the KEY mechanism** linking trauma to adult alexithymia (Bird & Cook 2013 correlation with trauma history).

### §9.2 — Framework NEW contribution: Schema Maintenance Cost

🟡 **This is a new framework concept** introduced in this drill. Not previously formalized.

**Definition**:

> **Schema maintenance cost** = energy + attention required to keep compiled schemas **active + usable** over time. Without ongoing activation, schemas decay (Chunk.md §2 "use it or lose it"). Multiple schemas compete for limited maintenance bandwidth.

**Core claim** 🟡:

> Human has **schema portfolio** — many compiled schemas across domains (relationships, hobbies, values, skills, aesthetic preferences, emotional regulation, social roles). Each requires ongoing activation to stay usable. **Intense pursuit of one domain** reallocates maintenance bandwidth → other schemas drift → eventual loss.

**Why not previously formalized**:
- Learning-Cycle focuses on **single chunk formation**
- Chunk.md §2 notes decay but doesn't model portfolio-level dynamics
- Hedonic treadmill research exists but doesn't connect to schema maintenance

**This drill formalizes** the connection.

### §9.3 — Schema Portfolio Model

🟡 **Model**:

| Schema category | Maintenance mechanism | Decay time without activation |
|---|---|---|

| **Fine-grained taste/enjoyment** | Sensory experience + schema fire | Short (atrophies quickly) |

**Critical insight**: "Fine-grained taste/enjoyment" chunks are **most fragile** because they require rich sensory-attention activation to stay calibrated.

### §9.5 — Why "Sơn hào hải vị không ngon lắm"

🟡 **Framework mechanism** (user's insight + framework extension):

**Taste chunks** are complex:
- Not just tongue sensors firing
- Include: attention + memory + context + social + mood integration
- "Ngon" = body match between sensory input + rich taste chunks + social/memory context

**During intense money pursuit**:
- Eating fast, distracted, transactional
- Taste chunks not activated richly
- **Decay starts**: fine-grained taste discrimination fades
- Default: fewer distinctions → fewer patterns → less rich experience

**At "arrival"**:
- Expensive food presented (sơn hào hải vị)
- Body tries to taste
- **BUT chunks are atrophied** → can't discriminate fine flavors → can't feel complexity
- Surface taste (sweet, salty, umami) registered
- Subtle taste (complexity, balance, skill, tradition) → not registered
- Result: "cũng bình thường" / "không ngon lắm"

**This is real neurological phenomenon**:
- 🟡 Consistent with hedonic adaptation research (Brickman 1978)
- 🟡 Consistent with "savoring" research (Bryant & Veroff 2007) — skill required for enjoyment
- 🟢 Neuroscience: sensory cortex specialization requires ongoing training

**Implication**: **Hedonic experience requires maintained chunks**. Wealth doesn't automatically restore atrophied chunks. You can buy the best food, but body needs the **capacity to taste** — and that capacity **decays without practice**.

- **Hedonic adaptation** (🟢 Brickman 1978): big purchase → initial boost → adapts → baseline returns

### §9.7 — The Deep Misunderstanding (User's astute framing)

**User's critical phrase**:
> "Mình tập trung vào kiếm tiền vì nghĩ các schema cũ sẽ vẫn ở đấy, nhưng quá trình xáo trộn lâu dài đã khiến các schema cũ thay đổi quá nhiều"

🟡 **This is the exact mechanism**.

**The assumption**: Schemas are "permanent" once formed. Like files on disk that stay until deleted.

**The reality** (framework): Schemas require **ongoing activation** to stay usable. Like plants that need watering. Neglect → decay.

**The trap**:
- Person thinks "I'll pursue goal X now, return to old life later"
- Old life = decayed schemas
- "Old life" no longer accessible — not because it's gone but because **capacity to engage with it** has atrophied
- **The person who "returns" is not the same person who "left"**

**Philosophical implication**:
- 🟡 "Heraclitus river" applies to selves, not just environments
- Can't step in same river twice — because you're different by the time you return
- **Identity maintenance** requires continuous activation

### §9.8 — Why "Cố Gắng Kiếm Tiền" ≠ "Cố Gắng Cho Project Có Passion"

🟡 Framework key distinction:

**Passion project**:
- Imagine-Final clear + aligned with body-base + domain-accurate
- Work done under **novelty cortisol** (not threat cortisol)
- Chunks compile with **opioid tag** (Cortisol-Baseline §3.5)
- Process itself rewarding → sustains without burnout
- **Melody stays smooth** because work fits hardware

**Money pursuit (generic)**:
- Imagine-Final proxy ("money will bring happiness") — indirect, unreliable
- Work done under **threat cortisol** (deadline, competition, fear of loss)
- Chunks compile with **cortisol tag**
- Process itself not rewarding → sustained by external goal → burnout risk
- **Melody noise accumulates** because work violates hardware

**Same hours of effort. Different chunk tags. Different long-term outcomes.**

**Caveat**: Money pursuit CAN be passion for some people — investment professionals who find markets fascinating, entrepreneurs who love building, etc. The distinction isn't about money specifically but about **body-base fit**.

**Framework prediction**:
- Money pursuit that's aligned with body-base → sustainable + rewarding
- Money pursuit that's NOT aligned → hedonic trap guaranteed

### §9.9 — Implications for Reward Drill (P3 preview)

- **Reward case**: body fires opioid because precondition met
- **Hedonic trap**: body FAILS to fire opioid despite apparent achievement → because preconditions (chunks base, body-need actual) were neglected during pursuit

H10 predicts: "Reward arrives only when 5 preconditions met." Hedonic trap = preconditions compromised during pursuit itself → arrival doesn't fire reward.
