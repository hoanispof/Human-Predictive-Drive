## §1 — 3-FACTOR MODEL: ENGINE MODE × GAP-NEED × ACCESS CONFIDENCE

### §1.1 — Factor 1: Engine Mode

```
⭐⭐⭐ FACTOR 1 — ENGINE MODE: HOW brain processes entity

  TOOL ─────────── AGENT (shallow) ──────── AGENT (deep)
  predict           predict state            predict state +
  FUNCTION          "vui hay buồn"           trajectory +
  "giao hàng"                                optimize interaction

  ⭐ BINARY GATE + CONTINUOUS SPECTRUM:
    Tool-mode = GATE CLOSED → entity-access IMPOSSIBLE (regardless of F2, F3)
    Agent-mode = GATE OPEN → entity-access POSSIBLE → depth matters

    Tool: "Con ăn chưa?" / "Giao hàng xong chưa?" → predict FUNCTION
    Agent shallow: "Con có vẻ buồn" → predict STATE
    Agent deep: "Con đang muốn thử nhưng sợ sai" → predict TRAJECTORY
    Agent optimize: "Biết bạn đang rảnh nên rủ" → USE prediction to optimize

  ⭐⭐ ENGINE MODE = DYNAMIC (★ KEY v1.0):

    KHÔNG phải fixed property → shifts MOMENT-TO-MOMENT

    Mẹ SÁNG (ngủ đủ, rảnh):
      Agent-mode deep → "con hôm nay có vẻ lo" → optimize response
    Mẹ TỐI (mệt, deadline):
      Tool-mode → "ăn đi!" → predict FUNCTION only
    → CÙNG MẸ, CÙNG CON — KHÁC moment = KHÁC mode
    → PFC Resource State MODULATES Engine Mode directly

    Lãnh chúa → nô lệ:
      Normally: Tool-mode ("xong chưa?") → NO entity-access
      Rare: Agent-mode ("nô lệ này thú vị, giỏi") → entity-access POSSIBLE
      → Historical: master-slave bonds DID develop (power DISTORTS, not prevents)

  ENGINE MODE = PER-PAIR + PER-MOMENT:
    Bưu tá: TÔI access DỊCH VỤ (Tool toward service), VỢ bưu tá access CHỒNG (Agent)
    → Cùng 1 con người, KHÁC target: service vs individual
    → SPM MODE = property of OBSERVER's processing, NOT of entity
    → Bưu tá nào giao cũng ok → cá nhân = interchangeable component of service

  ⭐ Simulation Engine APPLICATION:
    SPM = (Other, Present, Simulate) trên Simulation Engine
    Tool-mode: SPM KHÔNG chạy (no state simulation)
    Agent-mode: SPM CHẠY (simulate state → predict → optimize)
    Depth = SPM quality × EC data available

  🟡 Engine Mode as dynamic gate + spectrum = framework synthesis (v1.0)
  🟢 SPM as Simulation Engine application: Drill-Simulation-Engine v1.0 §3
```
