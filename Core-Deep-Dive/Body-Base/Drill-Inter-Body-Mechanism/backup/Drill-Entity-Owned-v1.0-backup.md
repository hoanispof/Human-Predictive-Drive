## §1 — BƯU TÁ CORRECTION: ACCESS ≠ ENTITY-OWNED

```
⭐⭐⭐ PHẢN VÍ DỤ QUYẾT ĐỊNH:

  BƯU TÁ:
    Access: STABLE (đến mỗi ngày, đúng giờ)
    NHƯNG: KHÔNG CÓ Entity-Owned feeling
    → ACCESS STABILITY ALONE ≠ ENTITY-OWNED
    
  CÁI THIẾU Ở BƯU TÁ:
    ① SPM = TOOL-MODE:
       "Đúng giờ sẽ tới, giao thứ X, xong việc họ về"
       = Predict FUNCTION, KHÔNG predict internal state
       "Không cần đoán bưu tá vui hay buồn" = NO state modeling
       
    ② Gap-fill = SINGLE FUNCTION:
       Giao hàng = 1 gap duy nhất (receive package)
       Không fill: comfort, emotional, social, curiosity
       
    ③ Reward = TASK COMPLETION:
       "Package received" = functional completion
       NOT body-need fill (không opioid, không oxytocin)
       
    ④ EC = SHALLOW:
       Minimal chunks: face, schedule, route
       Không: personality, emotional history, multi-modal
       
  MẸ (contrast):
    ① SPM = AGENT-MODE: predict mẹ's state, emotion, behavior
    ② Gap-fill = MULTI-CHANNEL: food + comfort + safety + emotional + guidance
    ③ Reward = BODY-NEED FILL: opioid, oxytocin, genuine
    ④ EC = CỰC SÂU: 20+ years, multi-modal, daily
    
  → Entity-Owned = f(①②③④⑤) — NOT just access stability
  → Bưu tá fails ①②③④ → NO Entity-Owned regardless of access
  
  ⭐ INSIGHT: TOOL vs AGENT = SPM MODE, NOT ENTITY PROPERTY:
    Bưu tá CÓ internal states (is an agent biologically)
    NHƯNG: MY SPM process bưu tá in Tool-mode
    Same bưu tá: Tool-mode cho tôi, Agent-mode cho vợ bưu tá
    → = SPM MODE = PER-PAIR decision, not entity property
    
  🟡 Bưu tá correction = drill insight (breaks access-only model)
```

## §2 — 5-FACTOR MODEL

```
⭐⭐ ENTITY-OWNED = f(5 factors ĐỒNG THỜI):

  ① SPM MODE: Agent-mode (predict STATE) vs Tool-mode (predict FUNCTION)
     Agent-mode: model entity's internal states, emotions, predictions
     Tool-mode: model entity's functional output only
     → Tool-mode = NO Entity-Owned possible (regardless of other factors)
     → Agent-mode = Entity-Owned POSSIBLE (if ②-⑤ sufficient)
     → = FIRST FILTER — fail here → stop
     
  ② MULTI-GAP FILL: entity fill BAO NHIÊU loại gap body-need?
     Single-function: Tool even if Agent-mode (bác sĩ cho bệnh nhân)
     Multi-gap: comfort + emotional + safety + play + ...
     → More gaps filled → stronger Entity-Owned potential
     → = SECOND FILTER — single-gap → weak Entity-Owned
     → Mẹ = MOST multi-gap (food + comfort + safety + emotional + guidance)
     
  ③ FILL QUALITY: genuine (opioid) hay schema (relief)?
     Genuine: body CONFIRM → approach tag → deep, self-sustaining
     Schema: PFC-driven → relief tag → shallow, need bridge
     → Genuine fill → STRONG Entity-Owned
     → Schema fill → WEAK Entity-Owned (need schema maintain)
     → VD: "bạn gái xinh nên phải giữ" = schema → Entity-Owned FRAGILE
     → VD: "chơi với bạn vì vui thật" = genuine → Entity-Owned SOLID
     
  ④ ACCESS STABILITY: khi cần, entity có available?
     Stable + ①②③⑤ → Entity-Owned COMPILE into baseline
     Unstable + ①②③⑤ → ANXIOUS Entity-Owned (want but uncertain)
     → Access stability determines Entity-Owned QUALITY (secure vs anxious)
     → NOT determines Entity-Owned EXISTENCE (need ①②③ first)
     → Bưu tá: access stable + fails ①②③ → NO Entity-Owned
     
  ⑤ EC DEPTH: tích lũy đủ để predict chính xác?
     EC shallow → SPM predict MỜ → confidence LOW → Entity-Owned = MỜ
     EC deep → SPM predict RÕ → confidence HIGH → Entity-Owned = RÕ
     → "Chưa rõ lắm, nhưng thấy thích" = EC building, Entity-Owned forming
     → "Rõ ràng là của tôi" = EC deep, Entity-Owned compiled
     
  ALL 5 CẦN ĐỦ → compile vào baseline → Entity-Owned EMERGE:
  
    ┌────────┬──────────────────────────────────────────────────────────┐
    │ Thiếu  │ Kết quả                                                 │
    ├────────┼──────────────────────────────────────────────────────────┤
    │ ① fail │ Tool → NO Entity-Owned (bưu tá, ATM, nhà hàng)         │
    │ ② fail │ Single-purpose → NO Entity-Owned (bác sĩ, thợ sửa)     │
    │ ③ fail │ Schema-only → WEAK Entity-Owned ("phải thế" relationship)│
    │ ④ fail │ Unstable → ANXIOUS Entity-Owned (limerence, xa bạn)      │
    │ ⑤ fail │ Shallow → UNCLEAR Entity-Owned ("thích nhưng chưa rõ")  │
    └────────┴──────────────────────────────────────────────────────────┘
    
  🟡 5-factor model = framework synthesis from drill analysis
```
