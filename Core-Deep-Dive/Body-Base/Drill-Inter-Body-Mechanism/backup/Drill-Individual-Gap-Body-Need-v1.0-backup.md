# Drill: Individual Gap Body-Need — Feed Dynamics, Satiation, Entity Matching

> **Mỗi người = 1 LANDSCAPE riêng của gaps.**
> **Gap = hole in chunk network nơi structure PREDICT pattern C nên tồn tại nhưng C chưa compiled.**
> **"Chưa biết = không có gap." — Gap CẦN surrounding chunks tạo border.**
>
> **Gap types có FEED DYNAMICS khác nhau:**
> **Hunger: fill 15 phút → no → return 3-6h → CYCLE (sharp satiation).**
> **Comfort/ôm: fill ONGOING → habituate CHẬM → "mãi không chán" (slow habituation).**
> **Curiosity: fill → CREATE new gaps → never "done" (self-generating).**
>
> **Não dự đoán entity nào fill gap nào qua SPM + Imagine-Final.**
> **Bưu tá đến mỗi ngày = access stable NHƯNG = Tool-mode → không "của tôi."**
> **Mẹ = Agent-mode + multi-gap fill + genuine + stable access → "của tôi."**
> **SPM MODE (Agent vs Tool) quyết định entity có TIỀM NĂNG fill body-need hay chỉ fill function.**
>
> **Collective-arc BIAS gap landscape: install gaps ("phải học giỏi"),**
> **suppress gaps ("lớn không được chơi"), re-tag gaps ("ăn đường có hại").**
> **Schema ≠ hardware nhưng CAN OVERRIDE hardware priority.**

## §2 — 3 SATIATION TYPES

```
⭐⭐ GAP TYPES CÓ FEED DYNAMICS KHÁC CĂN BẢN:

  TYPE A — SHARP SATIATION:
  
    Fill → satiation CLEAR → gap DORMANT → return after interval
    
    Đặc điểm:
      → Hardware mechanism tắt reward khi fill đủ
      → "Full = stop" rõ ràng
      → Gap return theo hardware clock (predictable interval)
      → Entity fill type A = needed in BURSTS, not continuously
    
    VÍ DỤ:
      Hunger: fill 15-30 phút → leptin/ghrelin signal → stop
              → Return 3-6h → gap re-activate → CYCLE
      Temperature: hot→cool = SHARP relief → baseline met → INVISIBLE
              → Shift environment = immediate return
      Safety: threat removed → safe = baseline → DORMANT
              → Threat appears = gap INSTANT activation
      Sexual: fill minutes → refractory period → return hours/days
      
    🟢 Satiety hormones: leptin, ghrelin (established endocrinology)
    🟢 Thermoregulation: homeostatic baseline (established physiology)


  TYPE B — SLOW HABITUATION:
  
    Fill → habituate SLOWLY (Weber-Fechner) → NO sharp cutoff → eventually habituate
    
    Đặc điểm:
      → Opioid system: sustained low-level reward
      → "Mãi không chán" WHILE entity present
      → Weber-Fechner CHẬM (hours not minutes for social/touch)
      → No satiation hormone equivalent (unlike hunger)
      → Entity fill type B = needed ONGOING, not in bursts
      → Reset qua ABSENCE + return (de-habituate → re-fill = "ấm hơn")
    
    VÍ DỤ:
      Touch/comfort (ôm): ongoing opioid → hours → gradual decline
        → "Ôm mẹ mãi không chán" = no sharp cutoff
        → INTERRUPT bởi type A gap (đói) hoặc type C gap (curiosity)
      Social connection: ongoing while with agent → gradual decline
        → Person-dependent: hardware need varies (extrovert > introvert)
      Temperature maintained: initially type A (relief) → transitions to type B
        → "Mặc áo mát mẻ": first = sharp relief, then = baseline = invisible
    
    ⭐ TYPE A→B TRANSITION:
      Nhiều gaps START as type A (sharp relief) → TRANSITION to type B (baseline):
        Temperature: hot→cool (A) → maintained cool (B → baseline → invisible)
        Safety: threat→safe (A) → maintained safe (B → baseline → invisible)
      = "Relief → Baseline → Invisible" trajectory
    
    🟢 Weber-Fechner law: established psychophysics
    🟢 Opioid system + touch: CT afferent fibers (Löken 2009)
    🟢 Hedonic adaptation: Bao & Lyubomirsky 2013


  TYPE C — SELF-GENERATING:
  
    Fill → CREATE new gaps → never "done" → perpetual cycle
    
    Đặc điểm:
      → Fill gap → discover new gaps → chase → fill → discover more
      → Reward = VARIABLE (depends on prediction-delta per discovery)
      → High discovery = HIGH reward (dopamine burst)
      → Repeated known = LOW reward (prediction confirmed, no delta)
      → Entity fill type C = needed for NOVELTY, not for presence
    
    VÍ DỤ:
      Curiosity/novelty: learn X → see Y unknown → gap for Y → chase
        → "Học toán → thấy bài khó mới → muốn giải" = self-generating
      Domain mastery: solve problem → see bigger problem → gap expands
        → PhD: chapter → experiment → thesis = mini-arc decomposition
      Status (comparative): achieve → baseline shift → need MORE
        → "Lương $1M → adapt → $10M → vẫn thiếu" = escalating baseline
        
    ⭐ TYPE C + ENTITY:
      Entity RELEVANT khi entity CONTRIBUTE to novelty:
        → Bạn thân chơi cùng: nếu tạo NEW discoveries → type C reward
        → Bạn thân same routine: no new discovery → type C ↓ → "chán"
        → Partner support drive riêng: new by-products → type C reward
        → Partner suppress drive riêng: no new by-products → "chán"
    
    🟢 VTA habituation + prediction-delta: Schultz 1997
    🟢 Mini-arc decomposition: BFM §3.3d
    🟢 Self-expansion model: Aron & Aron 1996, 2000
    🟡 "Self-generating" as explicit gap satiation type = framework synthesis
```

    Gap depth depends on SURROUNDING CHUNKS (how much you already know)
    Expert: deep surrounding chunks → deep gap → fill = HIGH reward
    Novice: shallow surrounding → shallow gap → fill = MODERATE reward
    → "Chưa biết = không có gap" → expert has DEEPER gaps than novice

## §6 — IMAGINE-FINAL × ENTITY-GAP MATCHING

### §6.1 — 5-step entity-gap matching process

```
⭐⭐ NÃO DỰ ĐOÁN ENTITY NÀO FILL GAP NÀO:

  STEP 1 — GAP DETECT (ACC/insula):
    Body-Feedback-Mechanism §3.3a: ACC detect inconsistency
    "Thiếu gì đó" → body signal
    PFC optionally articulate → "tôi đói" / "tôi buồn" / vague "bứt rứt"
    → Felt sense PRECEDES verbal label (Gendlin 1978)
    
  STEP 2 — ENTITY SCAN (SPM on compiled entities):
    Brain scan Entity-Compiled library: "ai fill được gap này?"
    SPM evaluate EACH compiled entity:
      → "Entity X từng fill gap loại này?" (historical)
      → "Entity X có available?" (access prediction)
      → "Entity X fill quality thế nào?" (valence history)
    = Imagine-Final applied to entity-gap matching
    
  STEP 3 — MODE FILTER (Agent vs Tool):
    SPM classify entity:
      TOOL-MODE: predict FUNCTION only
        → "Bưu tá giao hàng" = single function, no state modeling
        → "Nhà hàng nấu ăn" = single function
        → = Entity CAN fill gap nhưng as TOOL, not as AGENT
        → = No body-need BEYOND function → no Entity-Owned potential
      AGENT-MODE: predict STATE + EMOTION + BEHAVIOR
        → "Mẹ đang vui → ôm sẽ ấm" = state modeling
        → "Bạn đang buồn → cần lắng nghe" = empathy engagement
        → = Entity fill gap VỚI body-state interaction
        → = Entity-Owned potential IF multi-gap + genuine + stable
    
  STEP 4 — PREDICT 3 DIMENSIONS:
    a) REWARD APPROACH/AVOIDANCE:
       "Gặp entity X → reward hay threat?"
       = Per-agent valence (❸) quyết định
       = Compiled from historical interaction quality
       
    b) FILL QUALITY:
       "Entity X fill gap NÀY chính xác không?"
       = SPM depth × EC depth quyết định prediction accuracy
       = EC nông → predict mờ → uncertain reward
       = EC sâu → predict rõ → confident reward
       
    c) ACCESS STABILITY:
       "Entity X có available khi tôi cần?"
       = EC access chunks quyết định
       = Mẹ ở nhà = access stable. Bạn xa = access unstable.
       
  STEP 5 — DRIVE GENERATION:
    Reward HIGH + quality HIGH + access STABLE → APPROACH drive
      = "Muốn gặp bạn" = high predicted reward + accessible
    Reward HIGH + quality HIGH + access UNSTABLE → APPROACH + ANXIETY
      = "Muốn gặp người yêu nhưng sợ bận" = reward uncertain
    Reward LOW + access any → NO drive
      = "Bưu tá mà, kệ" = tool-mode, no body-need fill
    Reward NEGATIVE + access any → AVOIDANCE drive
      = "Gặp kẻ thù → tránh" = threat prediction

  🟡 5-step entity-gap matching process = framework synthesis
  🟢 ACC gap detection: Bush et al. 2000
  🟢 Felt sense before verbal: Gendlin 1978
```
