# Obligation — Observation Parameter

> My parents work hard to raise me and pay for my education.
> I "have to" wash dishes, sweep the floor, study hard.
>
> The boss pays my salary.
> I "have to" do my work.
>
> Person B lends me money.
> I "have to" repay — but repay HOW? How much? When?
>
> The village elder is trusted by the community.
> He "has to" care for the people.
>
> The scientist is recognized by the field.
> He "has to" contribute.
>
> "Have to" — the same phrase. COMPLETELY DIFFERENT mechanisms.
>
> Some "have tos" are clear (repay money, do the work) — cost known upfront, endpoint known.
> Some "have tos" are vague (unspoken debt, identity) — cost UNKNOWN, NO endpoint.
>
> But NO "have to" is born from nothing.
> Every one is compiled from REAL LIVED EXPERIENCE:
> Mother makes me wash dishes → body registers the cost.
> Don't wash → mother scolds → body registers the consequence.
> Ask a friend for help → friend asks for something in return → body registers reciprocity.
> Repeated over and over → chunks COMPILE → PFC can now PREDICT:
> "If I receive → I will have to give something back."
>
> Obligation = the name for that prediction.
> NOT Status (Status = what I have).
> NOT Threat (Threat = what I fear losing).
> NOT morality (morality = a label society assigns).
>
> Obligation = COMPILED PREDICTION about ongoing cost
> to KEEP what I HAVE from agents.
> Tracking mechanism for EXCHANGE — the foundational mechanism
> that allows humans to COOPERATE, SHARE, and BUILD communities.

## §1 — WHAT OBLIGATION IS

### §1.1 — Definition v1.0

```
⭐ DEFINITION:

  Obligation = observable pattern when body detects COMPILED PREDICTION
  about "cost to pay / to do / to maintain" to MAINTAIN access
  to agent-as-external-tool.

  CAUSAL CHAIN:
    Agent provides value (body-base feed: virtual chunks, connection, status, ...)
      ↓
    Body registers: "THIS agent feeds me" → ❸ positive toward agent
      ↓
    Chunk history COMPILES: "when receiving from agent → usually must give back"
    (accumulated from MANY past real experiences — §2)
      ↓
    Chunk-Gap emerges (Body-Feedback-Mechanism §3.3):
    "I RECEIVE → I SHOULD give back → I HAVE NOT yet" = network inconsistency
      ↓
    Cortisol holding signal fires:
    "still needs to be resolved" → PFC is forced to return to it
      ↓
    PFC predicts: "give back WHAT? HOW MUCH? WHEN?"
      ↓
    IF predictable → plan → execute → Gap closes → cortisol stops
    IF NOT predictable → Gap remains → cortisol SUSTAINED
      ↓
    = OBLIGATION (observable: body tension, PFC monitoring, behavioral change)


  CORE CLAIM:
    Obligation = COMPILED PREDICTION, not invented from thin air.
    The body does NOT imagine obligation — the body ACCUMULATES it from real experience:
      → Mother makes me wash dishes, sweep the floor (real cost registered)
      → Don't wash → mother scolds (real consequence registered)
      → Ask a friend for help → friend asks for a favor in return
          (real reciprocity registered)
      → Repeated many times → chunks compile → PFC CAN NOW predict
    → Prediction = PRODUCT of compilation
    → A child without the experience → nothing compiled yet → no prediction → no obligation


  ⭐ COMPILABLE ARCHITECTURE — WHY OBLIGATION EXISTS:

    Inter-Body-Mechanism.md §1.2: Humans use Compilable Architecture
    (general-purpose reward + compilation + social hardware + PFC).

    Hardwired Architecture (insects): hardwired circuit for each need
      → food→reward, mate→reward — NO tracking needed, NO agents needed
    Compilable Architecture (humans): general-purpose, LEARNS from environment
      → Needs 15-20 YEARS of compilation → DEPENDENT on agents throughout childhood
      → Adults still dependent: specialization → CANNOT do everything alone
      → = NEED agents as external tools → NEED TRACKING mechanism

    Obligation = tracking mechanism EMERGENT from Compilable Architecture:
      → Not a separate "social module"
      → Not a hardwired "moral circuit"
      → = General-purpose compilation system ACCUMULATES reciprocity patterns
      → = Same mechanism that compiles "hot → pull away" also compiles
          "receive help → may need to give back"
      → = Obligation is NOT special — just compilation applied to the SOCIAL DOMAIN

    WHY Hardwired Architecture DOES NOT NEED obligation:
      → Insects: no extended dependency → no agent tracking needed
      → Each individual = self-sufficient (hardwired for ALL needs)
      → = No exchange → no obligation

    → Obligation = ARCHITECTURAL CONSEQUENCE of being general-purpose learners
    → = The price of flexibility: need agents → need to track cost → obligation emerges

    🟡 Compilable Architecture → obligation as emergent tracking = framework synthesis
      (Inter-Body-Mechanism.md §1.2)

    🟡 "Compiled prediction, not imagination" = framework synthesis
    🟢 Chunk compilation mechanism = established (Chunk.md, Body-Feedback-Mechanism.md)
    🟢 Reciprocity expectation = developmental research (Piaget, Kohlberg)
```

### §1.2 — What Obligation Is NOT

```
OBLIGATION IS NOT:

  ✗ Moral judgment ("have to = ethics"):
    → Ethics = society labels right/wrong
    → Obligation = body-level cost prediction, independent of moral label
    → Can FEEL obligation WITHOUT a moral component
      (borrow money → must repay, not "ethics" → just cost tracking)
    → Can NOT feel obligation DESPITE society saying "you should"
      (scientist who faked credentials → society says "must contribute"
       → body has not compiled anything → no obligation felt)

  ✗ Social norm only ("have to = society dictates"):
    → Social norms = EXTERNAL install → CAN contribute to obligation
    → BUT obligation CAN emerge WITHOUT social norm
      (2 children exchange candy → independently develop reciprocity expectation
       → no norm required)
    → Social norms = 1 of many sources of obligation chunks

  ✗ Status ("Status = WHAT you have, Obligation = WHAT IT COSTS to KEEP it"):
    → Status = Resource Access Map — snapshot of current state
    → Obligation = ongoing COST to maintain that map
    → 2 parameters ORTHOGONAL:
      Status HIGH + Obligation LOW = built and maintained by yourself (ideal)
      Status HIGH + Obligation HIGH = dependent on many agents (fragile)
      Status LOW + Obligation LOW = little access, little debt (simple)
      Status LOW + Obligation HIGH = trapped, burdened (worst case)

  ✗ Threat ("Threat = FEAR of loss, Obligation = KNOWING the ongoing cost"):
    → Threat = prediction of HARM (reactive, FEAR)
    → Obligation = prediction of COST (proactive, KNOWLEDGE)
    → Obligation EXISTS BEFORE threat:
      "I know I owe person B" ← no threat yet
      "Person B is calling in the debt" ← NOW threat fires (deadline, consequence)
    → Threat CAN BE a consequence of obligation failure
    → But obligation = compiled expectations, threat = when expectations ARE VIOLATED

  ✗ Guilt ("guilt = regret"):
    → Guilt = body response AFTER failing obligation
    → Obligation = prediction BEFORE/DURING
    → Guilt = output. Obligation = ongoing state.


  IS:

    ○ Compiled prediction chunks about ongoing cost to maintain agent access
    ○ Emergent from agent-as-external-tool architecture
    ○ Body-level: cortisol holding when obligation is pending
    ○ PFC-level: monitoring + predicting + planning fulfillment
    ○ Spectrum: explicit→implicit, bounded→unbounded, self-imposed→external
    ○ Developmental: builds over time through real experience (§2)
    ○ Tracking mechanism: bookkeeping system for social exchange (§5)
```

### §1.3 — Obligation Valence: When Positive, When Negative

```
⭐ OBLIGATION IS NOT ONLY NEGATIVE:

  NEGATIVE obligation (forced, high cost, unclear terms):
    → Washing dishes when you hate it → cortisol ↑
    → Unspoken social debt with unclear terms → sustained anxiety
    → Trapped in an unwanted role → resentment
    → = Most people associate "obligation" with a HEAVY FEELING

  NEUTRAL obligation (exchange, balanced):
    → Salary ↔ work → fair, body ≈ neutral
    → Bank loan → clear terms, monthly repayment → manageable
    → = Exchange mode (Status §5) → obligation SYMMETRIC → acceptable

  POSITIVE obligation (voluntary, high Valence-Structural buffer):
    → Caring for mother: obligation HIGH but Valence-Structural HIGH → net = JOY (§7)
    → Helping a close friend: obligation LOW + Valence-Structural HIGH → near pure reward
    → Scientist contributing because they love the field: identity obligation + meaning → fulfilling
    → = CHOOSING the obligation (efference copy) + Valence-Structural buffer → positive experience

  ⭐ KEY: 3 FACTORS determine valence:
    ① CHOSEN vs FORCED (Autonomy-Hardware §5: efference copy present/absent)
    ② Valence-Structural BUFFER (Entity-Compiled: agent = body-base extension?)
    ③ COST CLARITY (knowing what to give back → manageable; not knowing → anxiety)

    Chosen + high Valence-Structural + clear cost → POSITIVE obligation
    Forced + low Valence-Structural + unclear cost → NEGATIVE obligation
    → = Why "same obligation, completely different experience"

  🟡 Obligation valence = f(choice, Valence-Structural, cost_clarity) = framework synthesis
  🟢 Intrinsic vs extrinsic motivation (Deci & Ryan 2000): chosen = rewarding
  🟢 Self-Determination Theory: autonomy, competence, relatedness
```

### §2.2 — 4 Stages of Obligation Formation

```
⭐ 4 STAGES:

  STAGE 1 — REAL EXPERIENCE: body registers outcome
  ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
    Body GOES THROUGH IT (not imagines it):
      → "Mother makes me wash dishes, sweep the floor" → body registers: tired, bored
          = REAL COST
      → "Don't wash → mother scolds" → body registers: pain, fear = REAL CONSEQUENCE
      → "Ask friend for help with homework → next day friend asks for paper, eraser"
          → body registers: friend IS ASKING IN RETURN
      → "Friend loses eraser, acts casual" → body registers: UNFAIR → valence ↓

    KEY: Stage 1 = RAW DATA for compilation.
    No Stage 1 → nothing to compile.
    → Child living in a bubble → little raw data → little obligation prediction


  STAGE 2 — COMPILATION: multiple experiences → pattern compiles
  ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
    Many Stage 1 experiences → brain detects PATTERN:
      → Washing dishes time 1, 2, 5, 20... → compiled: "mother raises me → I must do chores"
      → Friend calls in the favor time 1, 3, 5... → compiled: "ask someone → may have to give back"
      → Borrow money → repay money → compiled: "borrow → must repay" (explicit, bounded)
      → Ask person B → person B asks for something difficult in return → compiled:
          "unspoken debt → unclear cost" (implicit)

    COMPILATION IS UNEVEN:
      → Many negative experiences (friend demands a lot back) → over-compile: "asking anyone = DEBT"
      → Many positive experiences (friend helps freely) → under-compile: "asking anyone = FREE"
      → Mixed experiences → balanced compilation → flexible prediction
      → = Why everyone is DIFFERENT (§8: parenting calibration)

    🟢 Pattern compilation = chunking (Miller 1956, Chase & Simon 1973)
    🟢 Experience-dependent learning = established neuroscience


  STAGE 3 — PREDICTION: compiled chunks → PFC CAN predict
  ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
    When enough compiled chunks exist, PFC can predict BEFORE receiving help:
      → Someone offers help → PFC scans chunk history
      → "Last time I asked person X → had to give back Y"
      → "Last time I asked someone like Z → cost was high/low"
      → → PFC generates OBLIGATION PREDICTION

    PREDICTION OFTEN WRONG:
      → Each agent is DIFFERENT → past patterns DO NOT guarantee the future
      → PFC fills with WORST-CASE when Cost_clarity is low (§3)
      → CANNOT ASK DIRECTLY (social norm: asking = offensive)
      → → PFC over-predicts → over-estimates obligation → AVOIDS help
      → → = Certainty premium (Kahneman): prefer CERTAIN cost over UNCERTAIN cost
      → → "Work extra hours myself" > "borrow from person B"
          = choosing a KNOWN cost even if it is HIGHER

    🟢 Prediction error is well-documented in cognitive science
    🟢 Certainty premium (Kahneman & Tversky 1979): prospect theory
    🟡 "PFC over-predicts obligation" = framework application


  STAGE 4 — DECISION: prediction drives behavior
  ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
    Obligation prediction → decision:
      → Predict HIGH obligation + UNCLEAR cost → REFUSE help
        ("thank you, I can manage on my own")
      → Predict LOW obligation + CLEAR cost → ACCEPT comfortably
        ("sure, I'll borrow, I'll return it tomorrow")
      → Predict obligation ≈ 0 → ACCEPT without thinking
        ("stranger gives bus fare → accept → will never see them again")

    DECISION = RISK ASSESSMENT:
      Expected_benefit (help received)
      vs
      Expected_obligation (predicted cost to give back)
      + Access_cost (cost of shifting mode — §6)
      ─────────────────────────────────────────────────────
      IF benefit > obligation + access_cost → ACCEPT
      IF benefit < obligation + access_cost → REFUSE or DO IT YOURSELF

    🟡 Decision framework = framework synthesis
    🟢 Cost-benefit analysis in social exchange = established (Homans 1958)
```

### §2.3 — Body Signals During Obligation

```
OBLIGATION PRODUCES REAL BODY SIGNALS:

  CORTISOL HOLDING — Role ② (Cortisol-Baseline §7.7, Body-Feedback-Mechanism §3.3):
    → Obligation pending = Chunk-Gap: "needs resolving"
    → Body fires cortisol holding signal → sustained low level
    → PFC FORCED to keep returning to it → "have I repaid? what's the plan?"
    → Cost_clarity LOW → holding can SHIFT to Role ③ (threat-sustained)
    → Cost_clarity HIGH → cortisol manageable, decreases when a plan exists

  PFC MONITORING:
    → Working memory allocates slot for "pending obligation"
    → = Cognitive load: obligation occupies PFC bandwidth
    → Many obligations simultaneously → PFC overload → anxiety
    → Resolve 1 obligation → slot FREED → relief

  BODY TENSION:
    → Obligation pending → subtle muscle tension (shoulders, jaw)
    → Compiled, no conscious awareness needed
    → Resolving → tension release → relief = reward signal


  ⭐ 3-COST MODEL FOR OBLIGATION FULFILLMENT (Inter-Body-Mechanism.md §4):

    Fulfilling obligation = 3 INDEPENDENT COST SOURCES:

    ① PFC DRAFT COST (processing load):
      → Plan HOW to fulfill → PFC chains novel paths → serial bottleneck occupied
      → TYPE 1 (financial): draft cost LOW ("repay $50" = simple plan)
      → TYPE 3 (unspoken social debt): draft cost HIGH
          ("what to give back to make it enough?" = long novel chain)
      → = f(chain_length × novelty_degree)

    ② SUPPRESS COST (efference mismatch):
      → Fulfilling obligation AGAINST what body wants → body RESISTS
      → "Washing dishes even though I hate it" = suppress "want to play" → efference mismatch
      → "Going to see person B even though I don't want to" = suppress "want to stay home" → cost
      → = f(intensity × duration) — chronic suppression → burnout

    ③ UNCERTAINTY COST (cortisol holding):
      → DON'T KNOW if fulfillment is enough → hold multiple options open
      → TYPE 3 (unspoken social debt): "what to give? is it enough? is person B satisfied?"
        → 3+ options × high stakes × weeks/months = cortisol SUSTAINED
      → TYPE 1 (financial): uncertainty ≈ 0 → cost ③ ≈ 0

    → TYPE 3 (unspoken social debt) = HEAVIEST because ALL 3 costs HIGH SIMULTANEOUSLY:
      PFC draft long (novel problem) + suppress (reluctance) + uncertainty (unclear terms)
    → TYPE 2 (exchange) = LIGHTEST because ALL 3 costs LOW:
      PFC draft short (clear terms) + suppress low (fair) + uncertainty low (clear)
    → = 3-cost model EXPLAINS why same obligation type can feel very different in weight

## §3 — FORMULA: 5 FACTORS + STATUS GAP MULTIPLIER

### §3.1 — 5-Factor Formula

```
⭐ OBLIGATION INTENSITY = f(5 factors):

  Obligation_intensity = Agent_value
                         × Replaceability⁻¹
                         × Cost_clarity⁻¹
                         × Autonomy_alternative⁻¹
                         × Endpoint_clarity⁻¹


  ① AGENT VALUE: how much does this agent FEED my body-base?
  ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
    HIGH → obligation HIGH (much to lose if agent withdraws)
    LOW → obligation LOW (losing them matters little)

    E.g.: Parents raising me (Agent_value EXTREMELY HIGH: food, shelter, safety, love)
    E.g.: Classmate (Agent_value LOW: play together, but easily replaceable)
    E.g.: Feudal lord (Agent_value EXTREMELY HIGH: controls land, livelihood)


  ② REPLACEABILITY⁻¹: how EASILY can this agent be replaced?
  ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
    HARD to replace → obligation HIGH (losing them = permanent loss)
    EASY to replace → obligation LOW (can find another source)

    E.g.: Only 1 feudal lord in the region → Replaceability EXTREMELY LOW → obligation ↑↑↑
    E.g.: 20 classmates → Replaceability HIGH → obligation ↓
    E.g.: One and only close friend → Replaceability LOW → obligation ↑

    → Cross-ref: Protect.md §2: f(replaceability × attachment)
      = SAME FACTOR but Protect = guard, Obligation = maintain cost


  ③ COST CLARITY⁻¹: how CLEAR is the cost of repaying?
  ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
    UNCLEAR → obligation HEAVY ("don't know how much to give back to make it enough")
    CLEAR → obligation MANAGEABLE (give back → done)

    ⭐ = KEY DIFFERENTIATOR: separates light obligation from heavy obligation

    E.g.: Borrow $50 → Cost_clarity HIGH → "repay $50 = done" → manageable
    E.g.: Person B informally lends money → Cost_clarity LOW → "what does person B want???" → anxiety
    E.g.: Parents raise me → Cost_clarity EXTREMELY LOW → "how much is enough?" → ∞

    → IF CANNOT ASK DIRECTLY (social norm: asking = insulting):
      PFC fills with worst-case → over-estimates → cortisol sustained
    → IF CAN ASK:
      Cost_clarity ↑ → obligation MANAGEABLE → cortisol ↓
    → = Why "open communication" reduces obligation burden


  ④ AUTONOMY ALTERNATIVE⁻¹: can I DO IT MYSELF instead?
  ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
    CANNOT do it alone → obligation HIGH (must depend on others)
    CAN do it myself → obligation LOW (escape option: do it yourself = no debt)

    E.g.: I don't know how to fix a car → MUST ask a mechanic → obligation
    E.g.: I can code → build the website myself → no obligation to anyone
    E.g.: Young child → CANNOT eat or learn alone → full dependency → but not yet detected

    → Cross-ref: Autonomy-Hardware.md: self-action = efference copy + opioid reward
    → Doing it yourself = DOUBLE reward: no obligation + efference copy reward
    → = Why people PREFER doing things themselves even when help is available (§6 access cost)


  ⑤ ENDPOINT CLARITY⁻¹: do I know when it will be DONE?
  ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
    NO endpoint → obligation SUSTAINED (forever? → cortisol forever)
    CLEAR endpoint → obligation BOUNDED (repaid = LIBERATED)

    E.g.: Pay off bank loan → CLEAR endpoint → cortisol STOPS → relief
    E.g.: "Unspoken debt" → NO CLEAR ENDPOINT → cortisol NEVER STOPS
    E.g.: Identity obligation (scientist "must" contribute) → NEVER an endpoint

    → Cost_clarity⁻¹ + Endpoint_clarity⁻¹ = 2 factors TOGETHER create uncertainty
    → Uncertainty = EXACT pattern of Internal-Predict (Threat.md):
      don't know what to give + don't know when it's over = LOOP with no exit


  FORMULA APPLICATION:
  ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄
    Bank loan of $50:
      Agent_value: + | Repl⁻¹: + (many banks) | Cost⁻¹: ++ (clear) |

      Autonomy⁻¹: ++++ (cannot do it alone) | Endpoint⁻¹: −−− (lord decides when it's enough)

## §6 — ACCESS COST: MODE SHIFT WHEN ASKING A HIGHER-STATUS AGENT

### §6.1 — What Access Cost Is

```
⭐ ACCESS COST = THE COST OF REACHING AN EXTERNAL TOOL:

  When asking an agent for help, total cost = 2 COMPONENTS:

    Total_cost = ACCESS COST + OBLIGATION
                 (upfront)     (future)

    ACCESS COST: cost AT THE MOMENT OF ASKING
      → Must shift mode (Exchange → Comply)
      → Must admit "I cannot do this myself" (autonomy loss signal)
      → Must accept social risk (being refused → embarrassment)
      → = UPFRONT cost, paid RIGHT WHEN ASKING

    OBLIGATION: cost AFTER receiving help
      → Compiled prediction: "what must I give back?"
      → = FUTURE cost, paid AFTER receiving


  WHY ACCESS COST EXISTS:

    Body PREFERS exchange mode (Status §5: positive-sum, manageable)
    → Asking a higher-status agent = shifting FROM Exchange → Comply
    → Body RESISTS comply mode (Status §9.2: serotonin ratchet)
    → → Resistance = ACCESS COST = "fear of having to ask"

    Autonomy-Hardware: self-action = efference copy + opioid reward
    → Asking an agent = ADMITTING you cannot self-act
    → = Autonomy loss signal → negative body response
    → = One more layer of access cost

    Social risk:
    → Agent might REFUSE → embarrassment
    → Agent might JUDGE NEGATIVELY → status drop
    → Especially: person new to a job with no prior experience with this agent
        → PFC over-predicts risk

  🟡 Access cost concept = framework synthesis
  🟢 Social evaluation threat (Dickerson & Kemeny 2004)
  🟢 Help-seeking avoidance research (Lee 1997, DePaulo 1983)
```

### §6.2 — Access Cost × Status Gap

```
⭐ HIGH STATUS GAP → HIGH ACCESS COST:

  EQUALS / PEERS (friends, same level):
    → Mode: already in Exchange → NO SHIFT NEEDED
    → "Hey, help me with this" → casual, easy
    → Refused: "okay, I'll ask someone else" → low cost
    → Access cost ≈ 0
    → → Why "asking friends is easy, mostly for small things"

  MANAGER (moderate gap):
    → Mode: partially Comply → must shift MORE when asking for EXTRAS
    → "May I ask you..." → formal, deliberate
    → Refused: embarrassment + possible status damage
    → Access cost: MODERATE
    → Especially: NEW TO THE JOB:
      → No chunk history with manager → prediction uncertain
      → PFC over-predicts rejection risk + judgment risk
      → → Access cost INFLATED by uncertainty
      → → "Afraid to ask the manager" = access cost + obligation prediction + novelty threat

  STRICT PARENTS (large gap, familiar):
    → Mode: Comply already compiled since childhood
    → "Mom, Dad, I want to ask..." → deliberate, considered
    → Refused: not embarrassment but CAN be scolded
    → Access cost: HIGH (because history of negative consequences is compiled)
    → → "Weighing it carefully" = PFC predicting consequence = access cost visible

  FEUDAL LORD (EXTREMELY LARGE gap):
    → Mode: deep Comply
    → Must go through intermediaries (officials, procedures)
    → Refused: may be punished (not just embarrassment)
    → Access cost: EXTREMELY HIGH
    → → Only ask when THERE IS NO OTHER OPTION


  ⭐ PATTERN:
    Access cost ∝ Status gap × History of consequence

    → Large gap + Negative history (scolded, punished) → access cost EXTREMELY HIGH
    → Large gap + Positive history (manager is nice) → access cost MODERATE
    → Small gap + Any history → access cost LOW (many alternatives)

  🟡 Access cost ∝ status gap = framework synthesis
  🟢 Help-seeking behavior varies with power distance (Hofstede)
```
