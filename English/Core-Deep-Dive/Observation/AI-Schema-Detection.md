# AI-Schema-Detection
## AI-Assisted Schema, Compile Type, and Collective Context Detection

> **v2.1** | GATEWAY FILE | Confidence: 🟢🟡🔴 per section

---

> *"I see several patterns in what you've shared over the past three sessions:*
> *you use 'must/should' words most frequently in the context of family — especially around your mother.*
> *Would you like to explore that together?"*
>
> *Client pauses.*
>
> *Expert: "When you hear that — what does your body do?"*
>
> *Client: "My chest feels a little tight... actually, yes. That's real."*
>
> → AI detected the verbal pattern (Level 1). Expert ran the body-check (Expert layer). Client self-verified (Client layer).
> → **This is the 3-Layer Model in action.**
>
> **BUT: AI can install patterns into you through trust.**
> AI says "you have pattern X" → you trust AI → you compile that hypothesis — whether it is real or not.
> = The same mechanism by which a parent installs "be a good child" (Chunk.md §2.3).
>
> **→ Tool for NAVIGATING — not a precise GPS.**
> This file is a guide for working with AI in this way — with eyes open about what AI can and cannot do.

## §1 — FUNDAMENTAL LIMITS

```
10 LIMITS — MUST KNOW BEFORE USING:

  ① Valence-Propagation chains = EXPLANATORY, NOT PROCESSING:
    → "A triggers B triggers C" = description, NOT mechanism
    → Brain does not execute a chain; the chain is our map
    → AI chain detection = hypothesis about the map, NOT the territory
    → ⚠️ Confusing map for territory = major error source

  ② Access to body-level compile is IMPOSSIBLE from external data:
    → Body-level patterns compile below verbal awareness
    → Body cues (breath, tension, micro-expressions) = partial signal only
    → Text + audio = LOW fidelity vs actual body state
    → Even expert in-person = LIMITED access (Feeling.md v2.0 §2)

  ③ AI does NOT have direct access to the body:
    → Text-only AI: ZERO somatic access
    → Audio AI: tone, pace = some signal, still low fidelity
    → Video AI: micro-expressions possible, still not equivalent to in-person
    → = AI body inference = inferential, not direct

  ④ Schema = dynamic, not static:
    → Same person: schema active in one context, quiet in another
    → Schema changes over time (Hebbian + sleep + experience)
    → AI snapshot = at one moment only
    → = AI hypothesis may be true yesterday, false today

  ⑤ Verbal output ≠ internal state:
    → 7-layer fidelity gradient: internal → verbal (20–70% fidelity) (Feeling.md v2.0 §2)
    → Person can talk about schema WITHOUT activating it
    → Activation = specific trigger (context, person, environment)
    → = Verbal analysis misses activation context

  ⑥ Implicit >> explicit (Polanyi 1958, Reber 1967):
    → Expert's real knowledge = mostly implicit pattern
    → Cannot be fully verbalized (Somatic-Articulation-Loop.md §1.4)
    → AI can only access the explicitly verbalized portion
    → = "Tip of the iceberg" problem: AI only sees what breaks the surface

  ⑦ Confabulation: PFC explains post-hoc (Nisbett & Wilson 1977):
    → Body acts → PFC constructs story → person believes story = real cause
    → AI analyzes the story → NOT the real cause
    → = AI chain analysis may reinforce an accurate confabulation narrative

  ⑧ Schema clusters = overlapping, not cleanly separated:
    → One pattern can belong to multiple clusters simultaneously
    → AI cluster suggestion = approximate categorization
    → Clean separation = useful heuristic, not reality

  ⑨ Collective context is often invisible in individual data:
    → "I always feel unmotivated at work" = individual or collective chain break?
    → Without knowing industry context, AI cannot distinguish
    → = Collective blindspot: AI may misattribute collective → individual

  ⑩ AI output = Trust Compile install into user via trust:
    ⭐ CORE PARADOX:
    AI detects schema → outputs hypothesis → user trusts AI → user compiles hypothesis
    = Simultaneously detecting schema AND potentially installing NEW schema
    = SAME mechanism as parent installing "be a good child" (Chunk.md §2.3)
    → Only difference: AI = new type of trust entity, with properties
      DIFFERENT from human trust sources
    → = The tool is subject to the same mechanism it is designed to detect
    → → See §8 for guardrails
```

## §2 — THE 3-LAYER MODEL

```
⭐ CORE ARCHITECTURE:

  ┌─────────────────────────────────────────────────────────┐
  │                   3-LAYER MODEL                         │
  │                                                         │
  │  Layer 1 — AI DETECTOR:                                 │
  │    → Verbal pattern tracking across sessions            │
  │    → Schema cluster suggestion                          │
  │    → Contradiction detection                            │
  │    → Chain hypothesis generation                        │
  │    → Compile type detection (Experience/Expertise/Trust)│
  │    → Collective context flag                            │
  │    → 3-Level detection tagging                          │
  │                                                         │
  │  Layer 2 — EXPERT FEEL-CHECKER:                         │
  │    → Self-Pattern-Modeling (simulate client's patterns) │
  │    → Body-vote: expert's body responds to AI hypothesis │
  │    → Compile type awareness                             │
  │    → Collective vs individual distinguishing            │
  │    → VERIFY or REJECT AI hypothesis                     │
  │                                                         │
  │  Layer 3 — CLIENT SELF-VERIFIER:                        │
  │    → Focusing (Gendlin 1981): body-check hypothesis     │
  │    → Body-check AI output directly                      │
  │    → Final arbiter: body confirms or does not           │
  └─────────────────────────────────────────────────────────┘


  2-LAYER SIMPLIFIED MODE (AI + Self — for mild cases):

  When safe to use WITHOUT expert:
    ✅ Mild self-exploration: "Why do I procrastinate?"
    ✅ General patterns: "Why do I avoid conflict?"
    ✅ Self-development focus (not clinical)
    ✅ Feeling Literacy ≥ Stage 3 (Feeling.md v2.0 §10)
    → See §7 Self-Drill Mode for full protocol

  When expert is REQUIRED (do NOT use 2-layer mode):
    ❌ Trauma active: PTSD, abuse memories, violent events
    ❌ Crisis: suicidal ideation, self-harm, acute anxiety
    ❌ Deep pathology: severe depression, psychosis, personality disorders, addiction
    ❌ Feeling Literacy < Stage 2
    ❌ Distress escalation during drill
```

## §3 — THE AI LAYER: 9 CAPABILITIES

```
AI INDIVIDUAL CAPABILITIES ①–⑨:
(Collective capabilities ⑩–⑭ → AI-Collective-Detection.md v1.0)


  ① VERBAL PATTERN TRACKING (🟢 NLP applied):

    What AI can do:
      → Track word frequency across multiple sessions
      → NLP sentiment analysis per topic
      → Identify recurring linguistic patterns:
          "always/never," "must/should," "what if," "I can't help it"
      → Track pattern shifts over time (increasing/decreasing frequency)

    Verbal signals to track:
      → Must/should:     → possible Trust Compile
      → Always/never:    → schema with wide trigger surface
      → I can't help it: → pattern feels involuntary (deep chunk)
      → What if:         → anticipation schema active
      → Everyone/nobody: → social generalization pattern

    Limit: verbal pattern ≠ internal state (Limit ⑤)
    → Pattern in text does not mean the pattern is currently active in body


  ② SCHEMA CLUSTER SUGGESTION (🟡 framework application):

    What AI can do:
      → Detect lateral connections across topics
      → Suggest: "These 3 patterns may share a common underlying schema"
      → Example: "avoidance of conflict" + "difficulty saying no" + "fear of rejection"
          → AI suggests: possible "approval-seeking" cluster

    Why useful:
      → Client + expert each see one pattern at a time
      → AI sees across sessions: broader pattern emerges
      → = Pattern that neither party notices alone

    Limit: cluster suggestion = hypothesis, NOT confirmed grouping
    → Client body-check required before treating cluster as real


  ③ CONTRADICTION DETECTION (🟢 computational linguistics applied):

    What AI can do:
      → Detect verbal vs behavioral contradictions:
          "I don't care what people think" + talks about others' opinions every session
      → Detect temporal contradictions:
          "I used to be confident" + describes current avoidance behavior
      → Track: what client says vs what client actually describes doing

    Why useful:
      → Contradiction = possible gap between verbal schema and body-compile
      → Gap = indicator that schema may be Trust Compile (not genuine Experience Compile)

    Limit: contradiction detection from text only = partial
    → Body-verbal mismatch (tone, posture) = richer data that text-only AI cannot access


  ④ SCHEMA CHAIN HYPOTHESIS GENERATION (🟡 framework application):

    What AI can do:
      → Generate Level 3 explanatory chain hypothesis:
          "Possible chain: fear of failure → avoidance behavior → postponement → self-criticism"
      → Suggest which link in the chain is the primary driver
      → Distinguish: schema is trigger point vs schema is downstream effect

    CRITICAL LIMIT (Limit ①):
      → Chain = explanatory model, NOT actual brain processing
      → AI chain hypothesis = map, NOT the territory
      → "This is a possible narrative that may explain the pattern"
      → NEVER "this is how your brain actually works"


  ⑤ ⭐ COMPILE TYPE DETECTION — Experience / Expertise / Trust (🟡 v2.0 NEW):

    Source: Compile-Taxonomy.md v3.0 §6

    3 primary compile types:

    EXPERIENCE COMPILE (direct bodily experience):
      Verbal signals:
        → Body-emotion language: "I feel tightness when...", "I get that knot in my stomach"
        → Direct cause: "When X happens, I always Y"
        → Spontaneous description (not prompted)
        → Past specific events: "When I was 7, my dad..."
      Profile: vivid, specific, body-anchored, unprompted

    EXPERTISE COMPILE (skill + knowledge compile):
      Verbal signals:
        → Technical/domain vocabulary fluency
        → "I know that..." + elaborate explanation
        → Metacognition: "I notice I tend to..."
        → Comparison: "Compared to standard practice, I..."
      Profile: structured, analytical, domain-specific, knowing-tone

    TRUST COMPILE (installed by trusted source):
      Verbal signals:
        → Must/should/have to (without personal rationale)
        → Attribution to authority: "My mom always said...", "Our culture says..."
        → Social default: "Everyone does this"
        → Justify only by "that's just how it is"
      Profile: rule-like, attributed, lacks personal body-anchor

    Practical output:
      → AI suggests: "Dominant compile type appears to be Trust Compile —
        'must/should' language predominates"
      → Expert verifies: matches body vote + compile type awareness
      → WHY THIS MATTERS: intervention differs by compile type (see §6 Step 5)

    Limit: mixed types are common → classification is always approximate
    → Body-verbal mismatch = key signal, but text-only AI misses body cues


  ⑥ ⭐ CHUNK DEPTH INFERENCE (🟡 novel, logical):

    Source: Modality.md v1.0 §3

    Principle: chunk depth = modality count (★1–★5)
    AI infers depth from verbal output:

    SHALLOW (★1–★2):
      → Simple, general language
      → No body description
      → No emotional color
      → Example: "I don't like conflict"

    MODERATE (★3):
      → Some specifics
      → Some emotional language
      → Example: "When there's conflict, I get anxious and withdraw"

    DEEP (★4–★5):
      → Rich, multi-modal description
      → Body + emotion + memory + behavior + relational
      → Spontaneous, detailed, unprompted
      → Example: "My chest tightens, I can't breathe, I go quiet,
        I feel small like I'm 8 years old again, I just want to disappear"

    ADDITIONAL INDICATORS:
      → Prerequisite inference: AI checks if client can describe prerequisite schemas
          If client cannot describe prerequisite schema → target schema likely not deeply compiled
      → Domain-specific limit: deep chunk shows in SPECIFIC domain
          "I'm very open in friendships, but completely shut down in romance"
          → domain-gated chunk
      → Gap detection: verbal understanding WITHOUT body-compile
          Client explains schema accurately BUT cannot feel it in body
          → verbal-body gap
      → Body-verbal mismatch: client says "I'm fine" but describes avoiding, freezing
          → possible compile gap or suppression
      → Modality depth profile: count modalities client uses to describe schema:
          VISUAL + KINESTHETIC + AUDITORY + EMOTIONAL + NARRATIVE = deep compile

    Limit: verbal complexity ≠ chunk depth always
    (People can use complex language about a schema they understand analytically
    but have not body-compiled)


  ⑦ ⭐ AI SURFACES HIDDEN SCHEMAS (🟡 Somatic-Articulation-Loop §5 applied):

    What AI can do:
      → Pattern surfacing:
          "You mention feeling drained after helping people —
           I notice you never mention your own needs"
      → Cross-domain connection: link patterns across domains client doesn't connect:
          "You describe similar feelings at work AND with your mother"
      → Articulation catalyst: provide language for patterns client has felt but couldn't name:
          "Could what you're describing be something like
           'I'm only valuable when I'm useful to others'?"
      → Prerequisite probing: ask about upstream schemas:
          "Before the avoidance pattern activates — what comes first?"

    Why this matters:
      → Implicit patterns (Polanyi 1958) cannot be verbalized without help
      → AI provides language + connection that makes implicit explicit
      → NOT AI creating the schema — AI providing a mirror for what is already there

    CRITICAL RISK (from §1 Limit ⑩):
      → AI surfaces schema → highly suggestible client compiles AI hypothesis as real
      → = AI "surfaces" a schema that did not previously exist
      → Defense: body-check REQUIRED before treating surfaced schema as real
      → "Does your body confirm this — or is this just intellectually interesting?"


  ⑧ ⭐ COLLECTIVE CHAIN BREAK DETECTION (🟡 v2.0 NEW):

    Source: Collective-Body.md v1.1 §5.2

    INDIVIDUAL vs COLLECTIVE DIAGNOSIS:
      → Individual pattern: unique to this person, individual schemas + history
      → Collective chain break: shared structural condition affecting many simultaneously
          → Structural unemployment, industry collapse, cultural norm shift
          → "Many people in your industry describe exactly this"

    3 TYPES OF COLLECTIVE CHAIN BREAK:
      → Economic: market collapse, job category disappearing (structural, not personal)
      → Institutional: system failure affecting all members (school, company, government)
      → Cultural: norm shift making old patterns non-adaptive (intergenerational)

    HOW AI DETECTS:
      → Pattern language: shared complaint across multiple clients/data
      → Industry/demographic context data: trends matching client's experience
      → Attribution error flag: client blames self → AI checks for structural cause:
          "Are others in your context experiencing the same thing?"

    WHY IT MATTERS FOR INTERVENTION:
      → Individual pattern → self-understanding → possible behavior change
      → Collective chain break → ADAPTATION, not therapy:
          "This is not a 'you' problem — this is a structural condition"
          → New skills, career pivoting, waiting for collective adjustment
      → Misdiagnose collective → individual = harmful:
          Client does therapy for a systemic problem → no improvement → self-blame worsens

    Limit: requires context data (industry, economy, social trends)
    → Distinguishing "many people complaining" (collective) vs "echo chamber" = difficult


  ⑨ ⭐ 3-LEVEL DETECTION TAGGING (🟡 v2.0 NEW):

    Source: Collective-Body.md v1.1 §1

    AI tags EACH detection finding with detection level:

    LEVEL 1 — DETECTABLE FROM BEHAVIORAL DATA:
      → Pattern visible in verbal behavior, frequency, context
      → AI confidence: moderate–high
      → Example: "Avoidance pattern: present across 8 of 10 sessions — Level 1"
      → → Intervention: direct work with pattern itself

    LEVEL 2 — INFERRED ACROSS CONTEXTS:
      → Pattern inferred from cross-context consistency + self-report
      → AI confidence: moderate
      → Example: "Approval-seeking cluster: inferred from 3 domains — Level 2"
      → → Intervention: cross-context exploration + validation

    LEVEL 3 — MECHANISTIC HYPOTHESIS:
      → Chain explanation: "A → B → C" explanatory model
      → AI confidence: low (chain = map, not territory — Limit ①)
      → Example: "Fear of failure → avoidance → postponement — Level 3 hypothesis"
      → → Intervention: treat as hypothesis, body-check before using


  ───────────────────────────────────────────────────────────────────
  COLLECTIVE CAPABILITIES ⑩–⑭:
    → See companion file: AI-Collective-Detection.md v1.0
    File THIS = individual (①–⑨). File THAT = collective (⑩–⑭).
    Together: continuous capability system ①–⑭.
  ───────────────────────────────────────────────────────────────────
```

## §4 — THE EXPERT LAYER

```
EXPERT LAYER — ROLE + PROCESS:

  ⭐ SELF-PATTERN-MODELING (Agent-Mechanism.md §0):
    Expert uses their own pattern system to simulate the client's patterns
    → "When I imagine being in that situation — what do I feel?"
    → Body-vote: expert's body responds to client's pattern
    → Expert's body-vote ≠ guarantee (expert's own patterns can interfere)
    → But body-vote adds a crucial signal: does this hypothesis feel alive or flat?

  COMPILE TYPE AWARENESS:
    → Expert recognizes compile type indicators in client's language
    → Trust Compile detected → explore: who installed this? When?
    → Experience Compile detected → explore: what was the originating experience?
    → Mixed types → caution: do not oversimplify to one type
    → Expert uses their own compile type awareness as a calibration tool

  COLLECTIVE VS INDIVIDUAL CHECK:
    → Before attributing pattern to individual schema:
        "Is this a collective chain break that many in this context experience?"
    → Expert asks: "Do others in your industry/community describe this?"
    → If YES → reframe: adaptation focus, not individual-pathology focus

  EXPERT 5-STEP PROCESS (within the 3-Layer Model):

    Step A — Receive AI hypothesis:
      → Read AI output: patterns, cluster, compile type, collective flag, 3-level tags
      → First response: does any of this match what I've observed?

    Step B — Body-vote:
      → Simulate client in that pattern
      → Does my body confirm the hypothesis (resonance) or reject it (flat/off)?
      → Resonance: hypothesis worth exploring with client
      → Flat: hypothesis may be wrong OR expert's own pattern is interfering

    Step C — Compile type integration:
      → Which compile type is dominant for this pattern?
      → If Trust Compile: explore origin (who installed it? when? still serving client?)
      → If Experience Compile: explore originating experience (safe to access now?)
      → Mixed: hold both — do not force single narrative

    Step D — Collective check:
      → Is this individual or collective?
      → If collective: do not present as individual pathology
      → Adjust intervention frame accordingly

    Step E — Present to client:
      → Tentative, not certain: "I notice a pattern here — what do you make of this?"
      → Invite body-check, not PFC agreement
      → "When you hear that — what happens in your body?"

  EXPERT LIMITS:
    → Expert's Self-Pattern-Modeling = powerful but fallible
    → Expert's own schemas can color the simulation
    → Expert's compile type blindspots = real risk
    → Supervision + peer consultation = defense against expert bias
    → Expert cannot access client's body directly —
      body-check always comes from the client
```

## §5 — THE CLIENT LAYER

```
CLIENT LAYER — FINAL ARBITER:

  ⭐ BODY-CHECK AS PRIMARY VERIFICATION (Gendlin 1981 — Focusing):
    → Schema hypothesis presented → client body-checks
    → "When you hear that — what does your body do?"
    → Body confirm (somatic response, felt sense): → hypothesis worth pursuing ✅
    → Body nothing (no response): → PAUSE ⚠️
        (hypothesis may be wrong OR body is suppressing)
    → PFC agree instantly: → RED FLAG 🔴
        (PFC agreed before body had time to respond — possible confabulation)
    → = Client's body is the final arbiter — not intellectual agreement

  BODY-CHECKING AI OUTPUT:
    → Client must also body-check AI's direct output
    → AI says "you have pattern X" → does my body confirm?
    → Do NOT accept AI hypothesis on trust alone
    → Body-check applies even when AI language sounds accurate
    → = §8 guardrails applied at the client level

  FEELING LITERACY PREREQUISITE (Feeling.md v2.0 §10):
    The client's ability to body-check depends on Feeling Literacy development:

    Stage 1 — Awareness:     notices body signals exist
    Stage 2 — Recognition:   can name basic body sensations
    Stage 3 — Differentiation: can distinguish signals (tight vs heavy vs racing)
    Stage 4 — Integration:   connects body signals to patterns and contexts
    Stage 5 — Articulation:  can describe body signals verbally with fidelity

    → Self-drill safe at ≥ Stage 3
    → Full 3-layer process works optimally at ≥ Stage 4
    → Below Stage 2: body-check is unreliable → expert required

  CLIENT LIMITS:
    → Client's body-check itself can be biased
    → If client strongly wants AI to be right → body "confirms" = confirmation bias
    → Dissociation: client may not have access to body signals at all
    → Cultural schemas: some clients trained NOT to trust body signals
    → = Body-check is the best available tool — not a perfect tool
```

## §6 — PRACTICAL PROCESS (5 STEPS, v2.0)

```
⭐ UPGRADED v2.0 — Compile Type + Collective Context integrated at each step

  ═══════════════════════════════════════════════════════════════
  STEP 1: COLLECT PATTERNS (AI)
  ═══════════════════════════════════════════════════════════════

  AI tracks across sessions:
    → Verbal patterns: recurring words, must/should, always/never
    → Topics: what subjects emerge repeatedly?
    → Emotional language: what emotions appear in what contexts?
    → Contradiction flags: what does client say vs describe doing?
    → Domain mapping: in which domains do patterns appear?

  Output from Step 1:
    → Pattern list with frequency and context
    → Preliminary flags: high-frequency patterns worth investigating
    → Domain map: which domains are rich vs sparse in pattern data


  ═══════════════════════════════════════════════════════════════
  STEP 2: CLUSTER + CHAIN + COMPILE TYPE + COLLECTIVE + 3-LEVEL TAG (AI)
  ═══════════════════════════════════════════════════════════════

  AI processes pattern list:

    A — CLUSTER:
      → Which patterns may share an underlying schema?
      → Example: "avoidance + saying yes when wanting no + fear of disappointing"
          → Suggested cluster: approval-seeking

    B — CHAIN HYPOTHESIS (Level 3):
      → Possible explanatory chain for each cluster
      → Tag as Level 3: "explanatory hypothesis only"
      → Example: "approval-seeking ← fear of rejection ← early experience of conditional love
          — Level 3 hypothesis"

    C — COMPILE TYPE CLASSIFICATION:
      → Dominant type per pattern based on verbal signals (capability ⑤)
      → Trust Compile:      must/should, attribution to authority
      → Experience Compile: body language, specific events, visceral descriptions
      → Expertise Compile:  analytical, domain vocabulary, metacognitive
      → Flag: mixed types (common — treat with caution)

    D — COLLECTIVE CONTEXT FLAG:
      → Does this pattern match a known collective chain break condition?
      → "Many people in [industry/demographic] describe this"
      → Flag: COLLECTIVE POSSIBLE → expert to verify with client

    E — 3-LEVEL TAG:
      → Each finding tagged: Level 1 (behavioral) / Level 2 (cross-context) / Level 3 (mechanistic)
      → AI confidence decreases from Level 1 → Level 3

  Output from Step 2:
    → Schema cluster hypotheses with compile type + collective flag + 3-level tags
    → Priority ranking: which hypotheses are most robust (Level 1 + multiple sessions)?


  ═══════════════════════════════════════════════════════════════
  STEP 3: BODY-CHECK WITH CLIENT (Expert + Client)
  ═══════════════════════════════════════════════════════════════

  Expert presents hypothesis:
    → Tentative framing: "I notice a pattern here — what do you make of this?"
    → NOT: "You have schema X" (too definitive — risk of Trust Compile install)
    → Include compile type framing: "This looks like it may have come from [source]"

  Client body-check:
    → Expert: "When you hear that — what happens in your body?"
    → Wait. Give body time (10+ seconds, not 2 seconds).
    → Client reports: sensation, location, quality

  3 outcomes:
    → Body confirm (somatic response, felt sense) → PROCEED with hypothesis ✅
    → Body nothing (no response)                 → PAUSE ⚠️
        (hypothesis may be wrong OR body suppressing)
    → PFC agree instantly                        → RED FLAG 🔴
        (body hasn't had time to respond — possible confabulation)

  If RED FLAG:
    → Do not proceed. Give time.
    → Return to the question after a few minutes: does body respond now?
    → If still nothing → hypothesis likely wrong
    → If body responds later → hypothesis likely right


  ═══════════════════════════════════════════════════════════════
  STEP 4: PREDICT + VERIFY (Expert + Client)
  ═══════════════════════════════════════════════════════════════

  If schema hypothesis is supported → generate predictions:
    → "If this schema is real, we should see it in other contexts too"
    → Expert + AI generate 5 predictions for domains NOT yet discussed

  PREDICTIONS DIFFER BY COMPILE TYPE:

    Experience Compile predictions:
      → Pattern activates when body perceives similar threat/reward
      → Activation = involuntary, fast, body-led
      → Predict: "In context Y, body reacts before you have time to think"

    Trust Compile predictions:
      → Pattern activates when social rule applies
      → May not feel uncomfortable — just "normal"
      → Predict: "When in context Y, you likely follow [rule] automatically"

    Collective chain break predictions:
      → Pattern appears across many people in same context
      → Ask: "Do others in your industry/group describe this feeling?"
      → Predict: "If collective, others in [group] should confirm this"

  Expert role in Step 4:
    → Ask about domains client has NOT yet discussed:
        "When you try something new, what usually happens?"
        "When you receive praise, how do you feel?"
    → Compare response vs prediction

  Client role:
    → Respond honestly → body-check if needed

  Scoring:
    → 5 predictions → 4+ match: schema hypothesis LIKELY correct ✅
    → 5 predictions → 2–3 match: PARTIALLY correct → adjust
    → 5 predictions → 0–1 match: hypothesis WRONG → return to Step 2

  ⚠️ NEVER "certainly correct" — only "likely correct"
    → Continue verifying over time
    → Schema changes → hypothesis needs updating


  ═══════════════════════════════════════════════════════════════
  STEP 5: INTERVENTION DIRECTION (Expert + Client)
  ═══════════════════════════════════════════════════════════════

  Schema hypothesis verified → where in the gradient to intervene:

  DEEP FIX (body baseline — most effective, slowest):
    → Body-level: exercise, sleep, nutrition, safe environment
    → Somatic therapy: body-focused processing
    → Timeline: months → years
    → Effect: ALL gradient levels adjust automatically

  MID-LEVEL FIX (schema chain — moderate effectiveness):
    → Identify chain links: A→B→C→body-base
    → STRENGTHEN or WEAKEN specific links
    → Competitive re-linking (Chunk.md v2.1 §4.3)
    → Timeline: weeks → months

  SURFACE FIX (domain skills — fastest):
    → Domain-specific skill training
    → Behavior practice: try new behavior in safe context
    → Timeline: days → weeks
    → Effect: that domain ONLY

  v2.0 — FIX DIFFERS BY COMPILE TYPE:

    Experience Compile (direct) → fix through experience: create NEW experiences
      → Example: fear of conflict → practice safe conflict → body recompiles
    Trust Compile (installed) → verify + possible uninstall:
      → "Where did this pattern come from?" → client recognizes source
        → choice to keep or change
      → Trust Compile + body confirm → keep (genuine + installed = compound strong)
      → Trust Compile WITHOUT body confirm → possible to uninstall
    Experience Compile under threat → address threat source:
      → Do NOT fix "the caring response" → fix WHAT IS CAUSING THE FEAR
    Collective chain break → adaptation, NOT therapy:
      → Career change, new skills, or wait for collective adjustment
      → Do NOT blame self → understand collective context

  SEQUENCE:
    → Crisis: surface fix FIRST (need results NOW)
    → Long-term: deep fix FIRST (sustainable)
    → Ideal: deep + surface SIMULTANEOUSLY

  AI role in Step 5:
    → Track progress: "client's negative language about X decreased 30% over 3 months"
    → Alert: "old pattern RE-APPEARING" → schema may not have resolved at body level
    → = AI monitors the TIMELINE that expert only sees at POINTS

## §7 — SELF-DRILL MODE (User + AI)

```
🔴 HYPOTHESIS — Logic consistent, not yet systematically tested, safety boundaries unclear

  ⭐ SELF-DRILL = 2-LAYER MODE IN PRACTICE:
  §2 described the 2-Layer Simplified Mode (AI + Self, without expert).
  THIS SECTION: HOW to implement + GUARDRAILS for safety.


  WHEN SAFE — 2-LAYER SELF-DRILL:

      → "Why do I feel bored with my work?"
      → "I keep procrastinating — what pattern is driving that?"
      → "Why do I enjoy cooking but hate cleaning?"
    ✅ General patterns:

  WHEN EXPERT IS REQUIRED — DO NOT SELF-DRILL:

    ❌ Trauma active: PTSD, abuse memories, violent events
    ❌ Crisis: suicidal ideation, self-harm, acute anxiety attacks
    ❌ Deep pathology: severe depression, psychosis, personality disorders, addiction
    ❌ Feeling Literacy < Stage 2: user CANNOT distinguish body signals
    ❌ Distress escalation: drill makes user MORE anxious/distressed, not less
    → AI MUST detect RED FLAGS and REFER immediately:
      "I notice patterns in your language that suggest you need professional support.
       This is NOT something to self-drill."


  SELF-DRILL PROCESS (simplified 3 steps):

    ┌──────────────────────────────────────────────────────────┐
    │ STEP 1: AI DETECT + SUGGEST                              │
    │   AI tracks verbal patterns → clusters → hypothesis     │
    │   AI classifies compile type                            │
    │   AI checks collective context                          │
    │   AI: "It looks like you may have pattern [X].          │
    │   Possible compile source: [Trust/Experience].          │
    │   How does that land for you?"                          │
    │                                                          │
    │ STEP 2: AI ASKS BODY-FOCUSED QUESTIONS                   │
    │   AI: "When you hear that — what does your body do?"    │
    │   AI: "Any sensation in chest, stomach, shoulders?"     │
    │   AI: "Take 10 seconds of silence — listen to body."   │
    │   = Do NOT ask "why" → ask "what does body feel"        │
    │                                                          │
    │ STEP 3: USER BODY-CHECK + REPORT                         │
    │   User: listens to body → reports sensation             │
    │   "Slight heaviness in chest" → something here → explore│
    │   "Nothing" → hypothesis may be wrong → try another     │
    │   "PFC agrees immediately" → RED FLAG: confabulation?   │
    └──────────────────────────────────────────────────────────┘


  GUARDRAILS — SAFE SELF-DRILL:

    ① AI OUTPUT = HYPOTHESIS, NOT DIAGNOSIS:
      → AI: "This is a PATTERN I detected — NOT a diagnosis."
      → AI: "I may be WRONG. Your body is the final measure."
      → = AI self-disclaims BEFORE EVERY suggestion

    ② BODY-CHECK = REQUIRED:
      → User MUST body-check → if nothing felt → hypothesis may be wrong
      → User must NOT "just agree with head"
      → = PFC agrees too fast = possible confabulation (PFC-Function.md §6)

    ③ DISTRESS = STOP:
      → If user feels anxious/distressed AFTER drilling → STOP IMMEDIATELY
      → AI: "You seem distressed. I suggest we stop here."
      → → Seek professional support if distress persists
      → = Self-drill should NOT cause additional pain

    ④ DEPTH LIMIT:
      → Self-drill = explore SHALLOW to MODERATE only
      → If drill touches trauma / deep childhood patterns / very strong patterns:
      → → AI: "This pattern appears DEEP. I suggest exploring with a professional."
      → = Knowing when to STOP = the most important skill


  COMPILE TYPE AWARENESS FOR USER:

    AI helps user RECOGNIZE which compile type is active:
    → "You use 'must/should' frequently → may be a pattern INSTALLED by someone else
       (Trust Compile)"
    → "You say 'like/dislike' with body match → may be genuine experience
       (Experience Compile)"
    → "You say 'everyone does...' → may be social default (Trust Compile social)"
    → "You say 'scared if I don't...' → may be threat avoidance (Experience Compile)"

    User recognizes compile source → empowering:
      "Ah — I 'have to help people' because my mom said so — not because I genuinely want to?"
    → Body-check: "Does my body genuinely WANT to help?"
    → If body confirms → compound Experience+Trust (ideal — keep)
    → If body does not respond → pure Trust Compile install → can re-evaluate

    ⚠️ CAUTION: AI classifying compile type IS ITSELF a Trust Compile install!
    → User trusts AI saying "this is Trust Compile" → user "ah, right!" → BUT body hasn't verified
    → = Body-check AI's CLASSIFICATION too (§8 guardrails)


  COLLECTIVE CONTEXT AWARENESS FOR USER:

    AI asks:
      → "Do many people in your industry experience something similar?"
      → "Do friends/colleagues describe the same thing?"

    If YES (many people in same context):
      → "This may be a collective chain issue — NOT 'your fault'."
      → Direction: adaptation (new skills, change direction)

    If NO (unique to user):
      → "This looks like an individual pattern. Explore further?"
      → Direction: self-understanding → possible behavior change

  🔴 Self-drill mode safety: not yet systematically tested
  🟡 Body-check as verification: framework principle applied
  🟢 Guided self-reflection: established (bibliotherapy, Pennebaker writing)
  🟢 Distress as stop signal: clinical practice standard

    → = "Knowing" ≠ "understanding."
      Knowing with PFC ≠ understanding with body.
    → = Needs TIME + body-check — not just PFC agreement

      → Millions of users across various cultures receive the SAME bias
