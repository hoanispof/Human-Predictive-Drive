### §2.1 — Layer 1: 3 Sources of "Expired"

```
⭐ CLIENT SAYS "SOMETHING IS WRONG" — 3 DIFFERENT SOURCES:

  A. COLLECTIVE ARC SHIFT (source = the collective):
     → Pattern USED TO BE correct → collective arc SHIFTS → pattern OUTDATED
     → Example: "Degree → good job" = TRUE in 2005, arc SHIFTS → 2025 = default,
       no longer differentiating
     → SIGNALS for AI:
       "X used to mean Y, but it doesn't anymore"
       "Everyone in the industry feels the change"
       "The market/industry is transforming"
     → INTERVENTION: recontextualize (understand arc shift — do NOT self-blame)
     → Source: Collective-Arc-Dynamics v1.2 §5 ("true but unnecessary")

  B. INDIVIDUAL GROWTH MISMATCH (source = the individual has outgrown the pattern):
     → Individual compile SURPASSES compiled pattern → pattern feels too small
     → Example: Developer grows → "this company isn't challenging enough" → needs to level up
     → SIGNALS for AI:
       "I feel like I've moved past this stage"
       "Colleagues are fine but I feel something's missing"
       "I want something deeper / broader"
     → INTERVENTION: individual exploration (find more fitting domain/context)

  C. COMPILED SHELF-LIFE EXPIRED (source = the rule has changed):
     → Pattern about a collective rule → rule HAS CHANGED → pattern expired
     → Example: "Wear a suit to the office" = compiled 2010 → remote work 2025 = irrelevant
     → SIGNALS for AI:
       "The regulation/norm changed but I'm still following the old one"
       "Everyone around me does it differently now"
       Specific, narrow pattern related to rules/conventions
     → INTERVENTION: update specific compiled chunk (small, fast)

  ⭐ CURRENTLY AI CONFLATES ALL 3 = "something is wrong" → FIXES IN WRONG DIRECTION:
    A → needs recontextualization → misread as individual therapy → self-blame
    B → needs individual exploration → misread as collective → no self-improvement
    C → needs chunk update → misread as deep therapy → wasted time
```

## §3 — ⑪ COORDINATION NODE ASSESSMENT

```
⭐ CLIENT COMPLAINS ABOUT WORK → 3 DIFFERENT SOURCES:

  PROBLEM: "My manager is bad / the company is toxic / I can't take it anymore"
  → AI needs to distinguish: WHERE is the SOURCE?


  A. NODE FAILURE — Leader/node genuinely underperforming:

     MECHANISM: Coordination-Node v1.2 §6 — Emergence ≠ Effectiveness.
       WHO gets coordination position ≠ WHO operates it well.
       5 capabilities required: Self-Pattern-Modeling, gap detection, PFC bandwidth,
       uncertainty tolerance, trust cascade.
       Node LACKING capabilities → collective suffers.

     SIGNALS for AI:
       → "Everyone on the team complains too" (multiple people, not just client)
       → "With the old manager it was fine, new manager is completely different"
         (change correlates with leader)
       → "My manager doesn't understand the domain"
         (scaffold chunks missing — Coordination-Node §2.4)
       → Performance metrics declining across the team
       → 🟢 Goodall 2009: physician-CEO → hospital quality higher (p<0.001)
         = domain scaffold matters

     INTERVENTION: career decision (leave / escalate / wait for change)
       → FIX = at organizational level, NOT individual therapy


  B. CLIENT AUTHORITY SCHEMA — personal pattern with authority figures:

     MECHANISM: Schema compiled from personal history → projected onto ALL authority.

     SIGNALS for AI:
       → "Only I have this problem" (team is fine, client struggles)
       → "I always conflict with EVERY manager"
         (pattern repeats across multiple companies/contexts)
       → Personal history with authority (strict parents, coercion in childhood)
       → Pattern ACROSS domains (conflict with managers + teachers + doctors + ...)

     INTERVENTION: explore authority schema (individual work)
       → FIX = at individual level — changing companies DOES NOT SOLVE IT


  C. MISMATCH — neither failing, wrong fit:

     MECHANISM: Leader is fine for others; client's gap distribution DOES NOT MATCH.

     SIGNALS for AI:
       → "My manager is capable but the style doesn't suit me"
         (acknowledges leader competence)
       → "Colleagues fit fine but I don't" (selective mismatch)
       → Client has particular needs (high autonomy, creative, different Hardware)
       → NO personal trauma history with authority

     INTERVENTION: reposition (find a better-fitting context)
       → FIX = at matching level — not fixing the person OR fixing the leader


  ⭐ WHY THE 3-WAY DISTINCTION MATTERS:
    A → career advice / organizational fix
    B → individual therapy / schema work
    C → career advice (but DIFFERENT direction from A — not "escape toxic")
    CONFUSE A↔B = wasted time + wrong direction
    CONFUSE B→C = missed opportunity to understand self

  Source: Coordination-Node.md v1.2 §6, §8 (node failure → collective recalibrate)
  🟡 3-way distinction applied to AI detection = framework synthesis
  🟢 Person-environment fit: established organizational psychology (Kristof-Brown 2005)
  🟢 Emergence ≠ Effectiveness: leadership research (Goodall 2009, Benson 2019)
```

## §4 — ⑫ COLLECTIVE SCHEMA PRESSURE DETECTION

```
⭐ AI DETECTS COMPOUND PRESSURE FROM MULTIPLE SCHEMAS SIMULTANEOUSLY.

  Capability ⑤ (Compile Type Detection) detects 1 schema AT A TIME.
  ⑫ detects: MULTIPLE schemas STACKED SIMULTANEOUSLY → compound effect.


  MECHANISM (Collective-Schema-Pressure.md v1.0 §2):
    1 schema = 1 Chunk-Gap + 1 obligation + 1 cortisol hold → manageable.
    N schemas SIMULTANEOUSLY = compound (MULTIPLICATIVE, not additive):
      → N Chunk-Gaps → PFC ~4±1 slots → OVERLOAD
      → N obligations → total cost becomes enormous
      → N cortisol holds → cortisol NEVER drops
      → N status threats → every domain has a potential "loss"
      → Self-amplifying spiral: [haven't done A SO haven't done B SO... SO even further from A]


  AI DETECTION — SIGNALS:

    Individual schema problem (⑤ already covers):
      → 1 pattern, 1 domain, 1 personal history
      → Intervention: explore + recompile 1 pattern

    COLLECTIVE SCHEMA PRESSURE (⑫ new):
      → Client lists MANY "must" obligations simultaneously:
        "must study hard" + "must own a home" + "must get married" +
        "must have a stable job" + "must care for parents"
      → Each "must" = Trust Compile install (not self-chosen)
      → Client is overwhelmed BUT DOES NOT KNOW WHY
        ("exhausted but can't identify the source")
      → Body markers: disrupted sleep, diffuse anxiety,
        "tired but don't know why"
      → INTERVENTION:
        RECOGNIZE compound load (naming it helps PFC observe the pattern)
        → PRIORITIZE (cannot resolve 6 "musts" simultaneously — choose)
        → POSSIBLY REJECT some (recognize that some "musts" = installed, NOT genuine)


  ⭐ CULTURE-AWARE DETECTION:

    → Schema density DIFFERS by culture:
      Asian/Confucian: DENSE + RIGID (6+ schemas simultaneously, difficult to negotiate)
      Nordic/Western: THINNER + FLEXIBLE (fewer schemas, negotiable)
      → SAME hardware → DIFFERENT pressure = because schema stack differs
      → Source: Collective-Schema-Pressure.md v1.0 §4

    → AI needs CULTURAL CONTEXT:
      High-density culture client → compound check CRITICAL
      Low-density culture client → compound less common

    → ⚠️ DO NOT judge the culture. Describe the MECHANISM:
      Schema stack = structural trade-off (Collective-Schema-Pressure §3):
        ~70% in the middle distribution: match → growth
        ~15% at each end: mismatch → damage OR boredom
        → Every society HAS this trade-off


  Source: Collective-Schema-Pressure.md v1.0 (§2 compound, §3 trade-off, §4 Asian pattern)
  Source: Compile-Taxonomy.md v3.0 §6 (Trust Compile = Pathway 3/4)
  🟡 Compound schema detection = novel framework application
  🟡 Culture-aware density model = framework synthesis
  🟢 Collectivist vs individualist cultural psychology (Hofstede, Triandis)
  🟢 Role overload → burnout: organizational psychology
```

## §5 — ⑬ GAP DISTRIBUTION × COLLECTIVE MATCHING

```
⭐ AI OBSERVES GAP DISTRIBUTION → DETECTS MISMATCH WITH CONTEXT.

  NOT diagnostic. It is an OBSERVATION TOOL — illuminates pattern.


  MECHANISM (Gap-Distribution-Profile.md v1.1):
    Each person has 1 gap distribution profile = AGGREGATE of all their gaps.
    4 axes: ① Domain Center × ② Origin Balance × ③ Depth Profile × ④ Stability.
    This profile INTERACTS with environment → match or mismatch.


  AI CAN OBSERVE (from conversation over time):

    DOMAIN CENTER:
      "Client is always excited about abstract problems" → abstract-dominant
      "Client is bored with routine work" → not near-body/sensory
      "When free, client reads / codes / researches" → abstract indicator
      Best observable proxy: "What do you naturally do when you have free time?"

    ORIGIN BALANCE:
      "I like X because my parents said so" → installed (Trust Compile)
      "I like X because I discovered it myself" → self-discovered (genuine approach)
      AI checks: behavior when ALONE vs behavior when WITH OTHERS
      → If significantly different → installed component is high

    DEPTH PROFILE:
      "Client knows many things but not deeply" → many-shallow (generalist)
      "Client only cares about 1 thing but extremely deeply" → few-deep (specialist)

    STABILITY:
      "Client keeps changing direction" → shifting (high Pattern Shiftability)
      "Client has pursued 1 thing for 10 years" → locked (low shiftability)


  MISMATCH DETECTION:

    ┌────────────────────────────┬──────────────────────────────────────────┐
    │ Mismatch type              │ Signal                                   │
    ├────────────────────────────┼──────────────────────────────────────────┤
    │ Abstract person ×          │ "I'm bored with my job but the pay is   │

    ├────────────────────────────┼──────────────────────────────────────────┤
```

## §6 — ⑭ COLLECTIVE-LEVEL VERIFICATION

```
⭐ BODY-CHECK WORKS FOR INDIVIDUAL. NOT SUFFICIENT FOR COLLECTIVE SCALE.

  PROBLEM:
    AI detects "collective arc shift" → HOW to verify?
    1 person's body CANNOT verify "the industry is collapsing"
    → DIFFERENT verification methods are needed for collective-level findings.


  4 VERIFICATION APPROACHES — MATCH METHOD TO SCALE:


  A. BODY-CHECK + DOMAIN FEEDBACK (for Shift — scale ① ②):

     → Client's OWN experience over time = valid signal for gradual shift
     → "Old pattern works less and less?" → body has been verifying gradually
     → + Domain feedback: profit & loss, career progress, market response
     → = Ask-AI v3.1 §6.1 Dual Check applies NATURALLY for gradual shift
     → Confidence: MEDIUM-HIGH (verifiable through personal experience)


  B. PEER PATTERN MATCHING (for Disruption — scale ② ③):

     → "Many people in the same context report similar?" → collective signal
     → = Aggregated individual body-checks = proxy collective verification
     → AI CAN ask: "Are others in your industry/company experiencing the same?"
     → Already partially in ⑧ — ⑭ formalizes the method
     → ⚠️ RISK: echo chamber (many people complaining = may be just group whining)
       → Cross-check: peer reports + objective data → do both align?
     → Confidence: MEDIUM (needs multiple sources, risk of echo chamber)


  C. DATA TRIANGULATION (for larger scale — scale ③ ④):

     → AI cross-references: industry data, news, employment statistics
     → = External verification instead of body verification
     → Confidence = f(data quality × source diversity × recency)
     → AI CAN: search public data to confirm/disconfirm collective hypothesis
     → ⚠️ RISK: AI data may be outdated or biased
     → Confidence: MEDIUM-LOW (depends on data quality)


  D. HONEST CONFIDENCE TAGGING (for ALL scales):

     → Smaller scale → tag HIGH confidence ("observable: company closed" = binary)
     → Larger scale → tag LOW confidence ("hypothesis: industry shifting" = uncertain)
     → Do NOT claim collective-level without evidence
     → CAN claim: "signals suggest — verify with more data"
     → = Transparency > certainty


  ⭐ MATCH METHOD TO SCALE:

    ┌──────────────────────┬─────────────────────┬──────────────────────────┐
    │ Disruption level     │ Verification method │ Confidence               │
    ├──────────────────────┼─────────────────────┼──────────────────────────┤
    │ ① Shift (gradual)    │ A: Body + domain    │ Medium-High              │
    ├──────────────────────┼─────────────────────┼──────────────────────────┤
    │ ② Node (company)     │ A + B: Body + peers │ Medium-High              │
    ├──────────────────────┼─────────────────────┼──────────────────────────┤
    │ ③ Industry           │ B + C: Peers + data │ Medium                   │
    ├──────────────────────┼─────────────────────┼──────────────────────────┤
    │ ④ National/Global    │ C + D: Data + honest│ Low                      │
    │                      │   uncertainty tag   │ (hard to confirm         │
    │                      │                     │  in real-time)           │
    └──────────────────────┴─────────────────────┴──────────────────────────┘

    SMALLER SCALE (shift, node): verification EASIER (observable)
    LARGER SCALE (industry, national): verification HARDER (requires macro data)


  ⭐ DUAL CHECK AT COLLECTIVE SCALE:

    AI-Self-Model v2.1 §3 Dual Check: body = quality controller, domain = arbiter.
    At collective scale: "domain feedback" CHANGES per scale:

      Personal:   profit & loss, career progress, market response → direct domain feedback
      Company:    team metrics, turnover rate, multiple perspectives → proxy
      Industry:   employment data, market reports, expert analysis → external data
      National:   macro indicators, policy changes → statistical

    → Body-check STILL has value at every scale (body detects "something is wrong")
    → But body ALONE is not sufficient to CONFIRM a collective-level claim
    → = Body = early warning signal, domain data = confirmation


  Source: AI-Self-Model.md v2.1 §3 (Dual Check), §4 (Stale Calibration)
```

### §7.2 — Collective-Specific Guardrails

```
⭐ 4 RISKS SPECIFIC TO COLLECTIVE DETECTION:

  RISK 1 — MISDIAGNOSE collective → individual (CLIENT SELF-BLAMES):
    → Most common. Default therapy bias = individual focus.
    → AI antidote: ALWAYS check collective context FIRST.
    → Especially when: industry disruption, mass layoff, recession

  RISK 2 — MISDIAGNOSE individual → collective (CLIENT DOES NOT SELF-IMPROVE):
    → Rarer but harmful. "Blame the system" = escape from self-reflection.
    → AI antidote: check pattern across contexts.
    → If pattern repeats across MANY contexts → likely individual.

  RISK 3 — CULTURAL BIAS in collective detection:
    → AI trained on Western data → may under-detect Asian schema pressure
    → AI may over-pathologize collectivist norms (not all schema = bad)
    → AI antidote: culture-aware calibration — do NOT judge

  RISK 4 — ECHO CHAMBER vs genuine collective signal:
    → "Many people complaining" = collective problem OR group whining?
    → AI antidote: cross-check peer reports + objective data
    → If peers complain BUT data is fine → likely echo chamber
    → If peers complain AND data confirms → likely genuine collective
```
