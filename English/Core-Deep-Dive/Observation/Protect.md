# Protect — Observation Parameter

> My laptop breaks down + I have enough money to buy a new one → "Oh well."
> My laptop breaks down + I don't have enough money → I resent the friend who broke it.
>
> Same laptop. Same friend. Same event of "breaking."
> What's different: CAN I REPLACE IT?
>
> "Protect" is not a primitive protective instinct.
> It is the pattern that emerges when the body detects: "something already compiled
> into the baseline risks becoming absent — and I am not sure I can replace it."
>
> From "holding onto objects" → "holding onto people" → "holding onto ideas" → "holding onto identity" —
> same mechanism, different objects, different intensity.
>
> BUT protect is NOT the only force.
> The body also has a giving dynamic — when direct reward ≈ 0 (already satisfied),
> mirror reward (sharing) > keep reward (holding) → the body ITSELF drives giving.
> The gene "share when surplus" WINS over the gene "keep everything for yourself."
>
> This file: WHAT protect IS (ownership chunks + loss aversion),
> WHY losing hurts more than gaining (asymmetric prediction-delta),
> the FORMULA for intensity, which hardware participates,
> the NATURAL COUNTERWEIGHT (giving dynamic),
> and WHEN protect is healthy vs when it becomes toxic.

### §1.1 — Ownership Chunks = Compiled Baseline

```
🟡 OWNERSHIP CHUNKS — BODY-LEVEL "MINE":

  DEFINITION:
    When the body repeatedly interacts with an object/person/idea →
    the experience compiles into chunks at the body-level (C+D zones).
    Pattern "X + me interact" → Hebbian LTP → hard-wired.
    Result: the body has a BASELINE that includes X.
    → X is "mine" = X IS PART OF COMPILED BASELINE.

  PFC DOES NOT NEED TO DECLARE "MINE":
    → 2-year-old holds toy tight and won't let others play.
    → No language yet, no PFC-level concept of "ownership."
    → Body HAS ALREADY compiled: "this toy + my hands = baseline."
    → Anyone taking it = Chunk-Miss = dissonance = crying.
    → = Ownership chunk formation BEFORE language exists.

  COMPILE PATHWAY (Body-Feedback-Mechanism §5 Quality Baseline Shift):

    ① Interaction 1: proto-chunk weak
       → PFC aware "this is new" but body hasn't shifted baseline yet
       → Losing it: PFC "slight regret" > body dissonance (mild)

    ② Interactions 3-10: chunk strengthening
       → Body baseline starts to include X
       → Losing it: body dissonance CLEAR (Chunk-Miss appears)
       → Body-Feedback-Mechanism §5: SNC effect begins (Crespi rats downshift)

    ③ Interactions 50-100+: fully compiled
       → X = HARDENED part of baseline
       → Losing it: body dissonance STRONG (SNC effect maximum)
       → Re-habituate (letting go) = slow, difficult, painful
       → Body-Feedback-Mechanism §5 ④: re-habituate slow because strong compile = slow decay

    (Body-Feedback-Mechanism §5: compile_rate ≈ f(repetition × saliency × peak_valence
                               × multi_modal × contingency))

  EXAMPLES:
    → Car scratched after 1 day of ownership: "annoyed but fine"
    → Car scratched after 3 years of driving: "who did this?!" (compiled deep)
    → Home lived in for 20 years, must move: real grief (body-level loss)

  ⭐ "MINE" = GRADIENT, NOT BINARY:
    → Just bought = ownership chunk weak
    → Used for a long time = ownership chunk strong
    → Used for a long time + emotional experiences = ownership chunk VERY STRONG
    → Same object, different compile history → different protect intensity
    → Explains: 2 people lose their laptops but REACT DIFFERENTLY
      (Person A: 1 month, rarely used. Person B: 3 years, memories stored there.)
```

### §1.2 — Loss Aversion = Asymmetric Prediction Delta

```
🟢 LOSS AVERSION — LOSING HURTS ~2× GAINING:

  🟢 Kahneman & Tversky 1979 (Prospect Theory):
    → Gaining $100: positive utility = X
    → Losing $100: negative utility ≈ 2X
    → = Body responds to LOSS more strongly than GAIN of the same magnitude
    → = ASYMMETRIC prediction-delta

  MECHANISM IN FRAMEWORK:

    ① Compiled baseline = X (VTA habituated)
    ② Actual quality RISES to X+Δ:
       → VTA fires POSITIVE prediction-delta
       → Body signal: "better than expected" → reward
    ③ Actual quality DROPS to X-Δ:
       → VTA fires NEGATIVE prediction-delta
       → Body signal: "worse than expected" → dissonance
    ④ ⭐ ASYMMETRY: magnitude ③ > magnitude ② for the same Δ
       → = Loss aversion at the body-level

    (Body-Feedback-Mechanism §5 PROPERTY ③: ASYMMETRIC — upward shift easier than downward shift)
    (= Body resists quality decrease MORE STRONGLY than it accepts quality increase)

  WHY ASYMMETRIC? (evolutionary logic)
    → 🟡 Gaining something = nice but not survival-critical
    → Losing something already compiled = survival risk
    → Body has already invested resources compiling the pattern → losing it = waste
    → Body biases TOWARD KEEPING > TOWARD GAINING = conservative strategy
    → Conservative strategy survives BETTER in uncertain environments

  🟢 SNC EVIDENCE (Crespi 1942, Flaherty 1996):
    → Rats: high reward → switch to low → eat LESS THAN even the control group
    → = Not just "slightly disappointed" — reaction is STRONGER than neutral
    → = Body-level mechanism, NO PFC NEEDED
    → = Loss aversion EXISTS in animals = hardware-level
    → (Body-Feedback-Mechanism §3.2: SNC = Chunk-Miss mechanism at animal level)

  🟢 Schultz 1997 (dopamine neurons):
    → Expected = X, Actual < X → dopamine SUPPRESSED below baseline
    → = Specific neural mechanism for negative prediction-delta
    → Dopamine suppression = ACTIVE signal "worse than expected"
    → Not "missing reward" — it is an ACTIVE negative signal

  ⚠️ BUT: PREDICTION DELTA ≠ REWARD/PAIN
    (Clarification/Prediction-Error-Is-Not-Reward.md):
    → Prediction delta = VTA salience alert (doorbell: "something changed!")
    → REAL pain comes from 3 different sources: nociception / mismatch / recalibration
      (Body-Feedback.md: 3 Genuine Discomfort Sources)
    → Cortisol = sustainer/amplifier, NOT cause
      (Clarification/Cortisol-Amplifier-Not-Cause.md)
    → = Protect pain = BODY-BASE pain, not just "a dopamine drop"
```

### §1.3 — Why "Mine" Exists at Body-Level

```
🟡 "MINE" IS NOT A PFC CONCEPT — IT IS A BODY STATE:

  EVIDENCE:

  ① Pre-verbal infants (12-18 months) ALREADY protect toys:
     → No language yet, no concept of "ownership"
     → Body HAS ALREADY compiled: "my hands + this toy = baseline"
     → Anyone taking it = baseline violated = dissonance = crying/clutching
     → = Ownership chunk formation BEFORE language

  ② Animals protect territory:
     → Dogs: guard a bone, growl when someone approaches
     → Birds: defend nests, attack intruders
     → NO complex PFC → = body-level mechanism
     → 🟢 Prairie vole: vasopressin → territorial defense (Young & Wang 2004)

  ③ Phantom limb (🟢):
     → Lose a hand → body STILL fires pattern "my hand is here"
     → = Compiled ownership chunks PERSIST after the object is gone
     → = Body does NOT "know" the hand is lost → continuous Chunk-Miss
     → Extreme case: body-level ownership of one's own body parts

  ④ Endowment effect (🟢 Thaler 1980):
     → Person given a mug → values it higher than someone who doesn't have one
     → JUST 30 SECONDS OF HOLDING → ownership chunk begins to form
     → = Body compiles fast with physical contact (multi-modal input)
     → = "Mine" starts at the body BEFORE PFC decides to keep/sell

  IMPLICATION:
    → "Letting go" via rational PFC decision = TEMPORARY
    → Body-level ownership chunks STILL fire (compiled, not erasable)
    → Genuine release = re-habituate (Body-Feedback-Mechanism §5 ④: sustained absence →
      baseline shift downward) — slow, painful, but possible
    → = Explains why "knowing you should let go" but the body still holds on
    → PFC override is possible but COSTLY (continuously fighting body signal)
    → Duration of override = limited (PFC executive function = depletable)
```

### §1.3b — Compiled vs Fresh Protect (Inter-Body-Mechanism §3)

```
⭐ PROTECT HAS 2 PROCESSING MODES — CORRESPONDING TO COMPILED/FRESH AXIS:

  Compiled PROTECT (automatic, cost ≈ 0):
    → Ownership chunks ALREADY compiled deep → loss signal fires AUTOMATICALLY
    → Body reacts BEFORE PFC can evaluate
    → Example: someone touches your child → mother reacts INSTANTLY (compiled protection)
    → Example: hear news that boss is restructuring the team → gut tightens IMMEDIATELY
        (compiled job ownership)
    → = Immediate protective response, NO NEED to think
    → CAN overgeneralize: protect fires too strongly for things NOT actually threatened

  Fresh PROTECT (PFC evaluate, cost > 0):
    → Novel threat to a valued entity → body alerts BUT PFC needs to evaluate
    → "Is this actually a threat? What happens if I lose it? Can I replace it?"
    → Example: hear news that the company may be acquired → PFC assesses the impact
    → Example: friend says "I need to talk to you" → PFC evaluates: threat or not?
    → Cost: PFC executive function + uncertainty + possible suppression of body signal
    → = Slower but MORE ACCURATE than compiled protect

  Fresh → Compiled TRANSITION (compilation path):
    → Repeated loss of the same type → pattern compiles → next time = automatic
    → Person betrayed 3 times → "warning signs" compile → 4th time body reacts IMMEDIATELY
    → = Protect schemas LEARN + COMPILE over time
    → ⚠️ Trauma = forced compilation: 1 EXTREMELY INTENSE event → compiles immediately
       → post-trauma protect = overgeneralized compiled response

  🟡 Compiled/Fresh protect = framework synthesis (Inter-Body-Mechanism §3, consistent with Kahneman S1/S2).
```

### §1.4 — Cortisol: Amplifier, NOT Cause

```
⭐ CORTISOL IN PROTECT — FRAMEWORK CLARIFICATION:

  ⚠️ Mainstream + pop science: "loss → cortisol → pain → protect"
  ⚠️ Framework: "loss → 3 REAL pain sources → cortisol AMPLIFIES + SUSTAINS"

  (Cortisol-Amplifier-Not-Cause.md — detailed clarification)

  CORTISOL IN THE PROTECT CYCLE:

    ① LOSS DETECTED → body-base pain fires (3 sources)
    ② Cortisol fires 20-30 minutes AFTER the NE+Adrenaline spike
       → = SUSTAINER, not initiator
    ③ Cortisol AMPLIFIES body sensitivity
       → ownership chunks fire MORE STRONGLY
       → loss signal PERSISTS longer
    ④ Cortisol INERTIA (Cortisol-Baseline.md §2.3):
       → Object recovered → cortisol DOES NOT DROP immediately
       → Cortisol baseline takes weeks/months to re-calibrate
       → = Protect ANXIETY PERSISTS after the threat is resolved
       → Example: found a lost wallet → still checking wallet constantly for a week after

  DIRECTION GATE (Cortisol-Baseline.md §3, Chunk.md §2.4 Direction-At-Compile):
    → Same cortisol level, DIFFERENT direction:
    → Protect cortisol = threat-pull → chunks tagged AVOIDANCE
    → Novelty cortisol = domain-pull → chunks tagged APPROACH
    → → Protect-related chunks compile WITH avoidance tag
    → → When re-activated, body defaults to AVOID (not approach)
    → → = Explains: after a loss, body AVOIDS similar situations
         (avoidance learning = protect-compiled chunks firing)

  GYM ANALOGY (Cortisol-Baseline.md intro):
    → Cortisol = GYM for neurons
    → No training = atrophy (cortisol zero = Addison's disease)
    → Moderate training + enough rest = stronger (acute stress + recovery = resilient)
    → Overtrain + no rest = injury (chronic cortisol + no repair = PFC damage)
    → = Acute protect (loss → recovery → adapt) = HEALTHY
    → = Chronic protect (loss → no recovery → cortisol sustained) = PATHOLOGICAL
    → Key variable = REPAIR capability (sleep), NOT cortisol level itself
```

### §2.1 — Perceived Replaceability (PFC assessment, often WRONG)

```
🟡 PERCEIVED REPLACEABILITY — PFC ASSESSMENT, NOT REALITY:

  IMPORTANT: "perceived" — NOT actual replaceability.

  PFC assesses: "if I lose X, can I get X (or something equivalent) back?"
  → This assessment can be WRONG — and OFTEN is:

  ① UNDER-ESTIMATE (underestimates how replaceable it is):
     → "No one understands me like this person" (reality: many people could)
     → "This job is the only one for me" (reality: many equivalent jobs exist)
     → = Compiled chunks + narrow experience → PFC lacks data
     → → Protect OVER-FIRES → controlling behavior, anxiety, jealousy

  ② OVER-ESTIMATE (overestimates how replaceable it is):
     → "There's no shortage of friends" (reality: deep connection is rare)
     → "Money can always be earned again" (reality: opportunity cost is real)
     → = Many surface-level experiences → PFC overgeneralizes
     → → Protect UNDER-FIRES → lets go too easily → regret later

  REPLACEABILITY FACTORS:
    → Resources (money, time, energy needed to replace)
    → Availability (is a replacement readily available?)
    → Uniqueness (does this thing have unique properties?)
    → Compilation cost (how long to re-compile the attachment?)
    → Emotional investment (is Imagine-Final tied to this object?)

  ⭐ COMPILATION COST = HIDDEN FACTOR:
    → Losing a laptop: buying a new one = money. BUT: setting up again, lost data, habits?
    → Losing a relationship: meeting someone else is possible. BUT: re-compiling
      attachment chunks from scratch? 3-5 years? More?
    → Compilation cost is TYPICALLY IGNORED at the PFC level
    → Body KNOWS (because the body has compiled → knows "how much compiling costs")
    → → PFC says "it's replaceable" but body says "it hurts" = dissonance
    → = "Knowing it's replaceable but not wanting to lose it" = body correct, PFC wrong
      (body accounts for compilation cost, PFC doesn't)
```

### §2.2 — Attachment Chunks (compile depth)

```
🟡 ATTACHMENT CHUNKS — COMPILE DEPTH DETERMINES INTENSITY:

  Attachment chunks = how deeply ownership chunks have compiled.

  FACTORS AFFECTING COMPILE DEPTH
  (= Body-Feedback-Mechanism §5 compile_rate 5-parameter formula):

    ① Repetition: how many times have you used/interacted with it?
       → 1 time = proto-chunk → mild protect
       → 100 times = fully compiled → strong protect

    ② Saliency: was the experience memorable or notable?
       → Laptop used normally = moderate saliency
       → Laptop used to write a thesis, deadline, under stress = HIGH saliency
       → = Emotional peak → faster, deeper compile

    ③ Multi-modal: how many input channels were involved?
       → Only visual (saw it on the shelf) = 1 channel → weak
       → Hold + use + hear + smell = multi-modal → strong
       → = Physical contact ACCELERATES ownership chunk formation
       → (Thaler endowment: 30 seconds of holding is enough to shift)

    ④ Peak valence: were emotional peaks tied to the object?
       → Wedding ring = object + emotional peak + shared experience
       → Laptop = object + utility (moderate emotional)
       → → Wedding ring protect >> laptop protect (same monetary value)
       → (Valence-Propagation.md: valence TAG determines chunk weight)

    ⑤ Contingency: is the object DIRECTLY TIED TO REWARD?
```

### §4.1 — Vasopressin: Defense Side of Bonding

```
🟢 VASOPRESSIN = DEFENSE OF BONDED ATTACHMENT:

  🟢 Young & Wang (2004):
    → Oxytocin: "love" → bonding, trust
    → Vasopressin: "protect what you love" → vigilance, territorial defense
    → = 2 FACES of the same bonding system
    → (Neurochemistry.md §8: "vasopressin = DEFENSIVE FACE of oxytocin")

  🟢 PRAIRIE VOLE EVIDENCE (direct, animal):
    → Block vasopressin → still bonds but DOES NOT protect mate
    → = Vasopressin ALONE does not create bonding
    → = Vasopressin DEFENDS existing bonding
    → = Defense mechanism for compiled attachment chunks
    → = Protect ≠ create the bond. Protect = DEFEND the existing bond.

  VARIATION (Neurochemistry.md §8):
    → Vasopressin HIGH: strong protection, jealousy, controlling behavior
    → Vasopressin LOW: bonded but less protective, lets go more easily
    → Oxytocin high + Vasopressin high = "loves much, holds tight"
    → = NOT contradictory — 2 complementary systems both strong
    → Explains: "loving + controlling" = oxytocin + vasopressin both high

  🟡 IN HUMANS — INDIRECT EVIDENCE:
    → 🟢 Vasopressin receptor gene (AVPR1A) variation →
      correlates with partner bonding patterns (Walum et al. 2008)
    → BUT: in humans, vasopressin → chunk processing → behavior
    → NOT directly "vasopressin → protect" as in voles
    → = Hardware bias (vasopressin) + compiled chunks = actual behavior
    → = Vasopressin SETS TENDENCY, chunks DETERMINE expression

  FRAMEWORK POSITION:
    → Vasopressin = body-level BIAS TOWARD defending bonded objects
    → Not a "protection hormone" — it is the defense side of the bonding system
    → Similar to serotonin for status: serotonin = certainty bias baseline,
      NOT a status decision-maker (Status.md §7)
    → Vasopressin = defense bias baseline, NOT a protect decision-maker
    → Chunks (compiled experience) = the actual protect pattern
    → = Core v7.8 §8: "Protect parameter: hormone contribution
         vs chunk contribution ratio" (open research question)
```

### §4.2 — Cortisol Direction Gate: Protect = Threat-Pull

```
🟡 CORTISOL DIRECTION — WHY PROTECT = AVOIDANCE-TAGGED:

  (Cortisol-Baseline.md §7, Chunk.md §2.4 Direction-At-Compile)

  DIRECTION PRINCIPLE (Source > Level):
    → SAME cortisol level, DIFFERENT direction → DIFFERENT chunk tag:
    → Novelty cortisol: curiosity body state → chunks tagged APPROACH
    → Threat/loss cortisol: anxiety body state → chunks tagged AVOIDANCE
    → Protect = almost always THREAT direction (loss = threat)

  IMPLICATIONS FOR PROTECT:

    ① Chunks compiled during protect = AVOIDANCE-tagged:
       → Body defaults to: AVOID similar situations → future behavior avoids
       → Example: deceived out of money → chunks compile "deception" with avoidance
       → → Automatically AVOIDS similar situations (even if rational analysis says it's safe)
       → = Protect-compiled chunks CREATE avoidance patterns

    ② Protect cortisol = MODERATE-HIGH (Cortisol-Baseline §0.1):
       → Higher than novelty (LOW-MODERATE) → body MORE mobilized
       → = Protect reactions = STRONGER, FASTER than novelty reactions
       → = Explains: body protects > body explores (survival priority)

    ③ Cortisol inertia applies (Cortisol-Baseline §2.3):
       → Loss → cortisol spike → sustained → INERTIA
       → Object recovered → cortisol DOES NOT DROP immediately
       → = Protect patterns PERSIST beyond actual threat
       → = Hyper-vigilance period after loss/near-loss

  PROTECT × CORTISOL COUPLING (Cortisol-Baseline §10.6):
    → Same loss, DIFFERENT cortisol baseline → DIFFERENT experienced intensity
    → Person A (high baseline): lost wallet = "panic"
    → Person B (low baseline): lost wallet = "annoying but fine"
    → → Protect intensity = objective loss × personal cortisol baseline
    → → Anxious people (high cortisol) protect MORE STRONGLY for the same object
    → → Not "they're too sensitive" — it's a cortisol baseline difference
```
