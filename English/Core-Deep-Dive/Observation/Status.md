# Status — Observation Parameter

> 20 monkeys living together. Each time they want to eat fruit → they have
> to fight.
> 20 eating occasions × 20 monkeys = 400 fights EVERY DAY.
> Injury, blood loss, infection, death. Cost EXTREMELY HIGH.
>
> Evolution finds a CHEAPER way: fight once → remember the result →
> reuse it.
> Alpha wins → eats first, sleeps in the best spot, mates freely.
> Everyone else DEFERS — no need to fight again each time.
>
> That is STATUS.
> Not "who is higher than whom" — but "WHO GETS WHAT WITHOUT HAVING TO FIGHT."
>
> Body gives REAL REWARD when status rises — because status rise =
> BETTER FOOD, BETTER SLEEP, BETTER SAFETY.
> Not an abstract feeling.
>
> In monkeys: 1 dimension (strong/weak), 1 hierarchy.
> In humans: MANY dimensions (skill, knowledge, network, emotional...),
> PER-PERSON, PER-CONTEXT. But THE PRINCIPLE DOES NOT CHANGE:
> status = WHAT I CAN ACCESS FROM THIS AGENT.
>
> This file: WHY status exists, HOW the mechanism works,
> from simple animals → complex society,
> and why understanding status = understanding most of human social behavior.

## §0 — WHY STATUS EXISTS: EVOLUTIONARY FOUNDATION

```
⭐ THE CORE QUESTION: Why does evolution select for status assessment?

  PRINCIPLE: Status exists because it REDUCES THE COST of resource allocation
  within a social group, while body gives REAL REWARD when status rises
  because status rise = resource access rise = body-base fed better.

  CAUSAL CHAIN:

    Animals living in groups → share resources (food, sleeping spots, mates)
      → Each allocation requiring a fight → cost EXTREMELY HIGH
        → Evolution selects for a cheaper mechanism:
          → Fight once → remember the result → reuse many times
            → = PRE-CALIBRATED RESOURCE ALLOCATION
              → = STATUS

    Higher status → access to more resources
      → Body-base fed better (better food, better sleep, safety, mates)
        → Body gives REAL REWARD for status upgrade (opioid, serotonin rises)
          → Animals with a status assessment mechanism → survival advantage
            → Evolution selects → mechanism prevalent across ALL social species


  🟢 EVIDENCE:

    ① Sapolsky (30+ years studying baboon troops):
       → Alpha: low cortisol, high serotonin, high testosterone
       → Subordinate: higher cortisol, lower serotonin
       → Alpha removed → beta serotonin RISES IMMEDIATELY (before winning)
       → = Body PRE-REWARDS for a status upgrade opportunity

    ② Pecking order (chickens, 1920s, Schjelderup-Ebbe):
       → New chicken added → a few fights → rank established → STABLE afterward
       → Cost: a few fights. Benefit: thousands of feeding occasions without fighting
       → = Cost-benefit ratio EXTREMELY HIGH for pre-calibration

    ③ Lobster serotonin (Huber & Kravitz 2005):
       → Winning lobster → serotonin rises → expansive posture → confident behavior
       → Losing lobster → serotonin drops → contracted posture → timid behavior
       → = FUNDAMENTAL MECHANISM, present in a species 350 million years removed

    ④ 🟢 Social Baseline Theory (Coan 2015):
       → Proximity to allies → metabolic cost DROPS
       → Having allies = having backing = status maps expand further
       → = Status system INTEGRATED into energy management

    → Cross-species: chickens, lobsters, fish, wolves, monkeys, humans — ALL have status
    → = ANCIENT mechanism, at minimum 350 million years of evolution


  ⭐ REAL BODY REWARD — NOT ABSTRACT:

    Alpha monkey:
      → Eats WHATEVER (best fruit, best meat)
      → Goes ANYWHERE (best sleeping position, safest spot)
      → Mates FREELY (most partners)
      → Everyone DEFERS (no fighting required)
      → = ALL body-base needs met at the HIGHEST level

    Lowest-rank monkey:
      → Eats LEFTOVERS (what remains, what was left)
      → Movement RESTRICTED (dares not enter alpha's zone)
      → Mates LIMITED (must do so covertly)
      → Must DEFER to everyone of higher rank
      → = Body-base needs met at the LOWEST level

    → Higher status = wider resource access = REAL body-base reward
    → Body rewards for upgrade = CORRECT, because upgrade = genuinely beneficial
    → Body resists status decrease = CORRECT, because decrease = real resource loss
    → = Reward mechanism IS ACCURATE — not an illusion


  ⭐ COMPILABLE ARCHITECTURE — WHY STATUS IS EMERGENT
     (Inter-Body-Mechanism.md §1.2):

    HARDWIRED ARCHITECTURE (insects, simple animals):
      Each social behavior = 1 dedicated hardwired circuit
      → Status? NOT NEEDED — response hardwired: encounter stronger → retreat,
        encounter weaker → attack
      → Hormone drives DIRECTLY: testosterone → approach, cortisol → retreat
      → = No "status tracking" — only stimulus → response

    COMPILABLE ARCHITECTURE (complex mammals, humans):
      Hardwired: ONLY reward mechanism + compilation + social hardware + PFC
      Content: LEARN from environment → compile → body-need
      → Status = EMERGENT when general-purpose system TRACKS resource access:
        ① Body NEEDS resources (food, mate, safety) → universal
        ② Body LEARNS "which agent gives what access" → compilation
        ③ Compiled maps = STATUS SYSTEM (no dedicated module needed)
      → = Status has NO "status module" in the brain
      → = Status = the name for the OUTPUT of the general-purpose learning system
        when applied to social resource tracking

    🟡 Hardwired/Compilable Architecture naming = framework synthesis.
       Status = emergent from Compilable Architecture = consistent with no dedicated
       "status module" found in neuroscience (unlike, e.g., FFA for faces).


    🟡 This is why the Serotonin Ratchet exists (§9.2):
       Ready to rise when opportunity presents (grab resource quickly)
       Resists dropping when threatened (hold resources already acquired)
       = Body OPTIMIZES resource access through the status marker
```

### §0.1 — Cost of status calibration

```
🟡 CALIBRATING STATUS = COSTLY, BUT CHEAPER THAN FIGHTING EVERY TIME:

  COST OF FIGHTING EVERY TIME (without status):
    → Injury (blood loss, broken bones, infection)
    → Death (total loss)
    → Energy drain (each fight = cortisol spike + muscle drain)
    → Time (fighting = not eating, not sleeping, not mating)
    → = Cost PER INTERACTION = VERY HIGH

  COST OF STATUS CALIBRATION (once, reused many times):
    → A few fights initially (or display + posture competition)
    → PFC drafts status maps for every member (bandwidth cost)
    → Temporary cortisol rise (uncertainty phase)
    → = Up-front cost = MODERATE

  BENEFIT OF CALIBRATED STATUS:
    → Each resource allocation = ZERO fighting cost
    → PFC cost ≈ 0 (map already compiled)
    → Thousands of interactions/year × zero cost = ENORMOUS SAVINGS
    → = Cost-benefit ratio >> 1

  → = Evolution selects because: up-front cost SMALL × long-term benefit LARGE
  → = Like "buying insurance" — pay once, use many times
  → = Explains why ALL social species have a status system
```

  Agent-Mechanism.md §12: Body NEEDS social presence like it needs food, water.
  Agents = EXTERNAL TOOLS helping body-base meet needs:

    Mother → feeds, cares for → body-base directly (survival)

    Colleague → cooperates → products impossible to create alone
    Spouse/Partner → intimacy + co-regulation → cortisol drops, oxytocin rises

    → = RESOURCE ACCESS MAP per agent

  Feudal lord → takes ANYTHING from slave (no need to reciprocate)
  Slave → dares not resist (clearly knows the gap is too large)
  = EXTREME case: status gap ENORMOUS = one-directional resource access

  Close friends → exchange knowledge, emotion, time (mutual)
  = COMMON case: status gap SMALL = bidirectional resource access

  → Status is NOT abstract "who is higher than whom"
  → Status = "WHAT CAN I ACCESS FROM WHO" = concrete, measurable, body-real
```

    → Does not reside in 1 step — observed ACROSS the cycle

### §1.1 — Resource Access Map: concrete, not abstract

```
⭐ STATUS = RESOURCE ACCESS MAP:

  MOST CONCISELY:
    Status = "With THIS PERSON, in THIS CONTEXT:
    what can I ACCESS, and what must I GIVE?"

  "Access" here = CONCRETE:
    → Eat first or eat last? (food access)
    → Speak or stay silent? (voice access)
    → Propose or wait for instructions? (initiative access)
    → Embrace or maintain distance? (intimacy access)
    → Ask for help or manage alone? (support access)
    → Decline or comply? (refusal access)

  EXAMPLE — the same person, 3 DIFFERENT agents:

    With SPOUSE:
      → Voice: speak freely, propose comfortably ✅
      → Intimacy: embrace, kiss, be close ✅
      → Initiative: suggest outings, purchases ✅
      → Refusal: decline requests (gently) ✅
      → = Map WIDE, many resources accessible

    With BOSS:
      → Voice: consider before speaking, choose words carefully 🟡
      → Intimacy: maintain professional distance ❌
      → Initiative: can propose, but must frame correctly 🟡
      → Refusal: hard to decline, must have good reason 🟡
      → = Map NARROWER, many resources restricted

    With STRANGER:
      → Voice: polite, cautious ❌
      → Intimacy: none ❌
      → Initiative: limited ❌
      → Refusal: easy (no obligation) ✅
      → = NARROWEST map but refusal access HIGH (no commitment)

  → SAME PERSON, DIFFERENT AGENT → COMPLETELY DIFFERENT MAP
  → Each agent = 1 UNIQUE map
  → Status is NOT 1 number — it is a MULTI-DIMENSIONAL MAP per-agent


  WHY "RESOURCE ACCESS MAP" NOT "HIERARCHY":

    Hierarchy = only 1 DIMENSION: high/low
      "I am rank 5, they are rank 8 → I am lower → comply"
      → Explains: military, companies (clear rank)
      → DOES NOT EXPLAIN: family, romantic partners, friends, context switching

    Resource Access Map = MANY DIMENSIONS:
      "Technically I am >> them (access: propose, lead)
       Socially they are >> me (access restricted: defer to them)
       Physically we are equal (symmetric access)"
      → Explains: ALL interactions

    → Hierarchy = the SIMPLEST SPECIAL CASE
      (when context flattens to 1 dimension: rank)
    → Resource Access Map = the GENERAL CASE
```

## §2 — THE SCAN MECHANISM: UNCONSCIOUS BODY PROCESS

```
🟢 WHEN MEETING AN AGENT, BODY UNCONSCIOUSLY SCANS (milliseconds → seconds):

  EVOLUTIONARY FUNCTION OF THE SCAN:
    → Determine quickly: what can I ACCESS from this agent
    → So that: fighting is NOT NEEDED each time → saves cost (§0.1)
    → Speed matters: slow scan = high PFC cost = high energy = disadvantage

  5 STEPS OF THE SCAN:

  ① IN WHAT WAY ARE THEY AHEAD OF ME?
     → Strength? Skill? Knowledge? Resources? Network?
     → = "What do THEY HAVE that I need/want"
     → Input: visual cues (posture, clothing, gaze), auditory (voice tone),
       context (setting, role), compiled chunks (if already acquainted)

  ② IN WHAT WAY AM I AHEAD OF THEM?
     → = "What do I HAVE that they need/want"
     → Input: compiled self-assessment chunks + domain chunks

  ③ WHAT CAN I RECEIVE FROM THEM?
     → Without giving anything? (I am >> them → "take" mode)
     → Must exchange? (roughly equal → "exchange" mode)
     → Must request? (I am << them → "comply" mode)

  ④ WHAT CAN THEY RECEIVE FROM ME?
     → Can I decline? (equal/above)
     → Can I NOT decline? (below)

  ⑤ IS EXCHANGE POSSIBLE?
     → "I want to embrace, they also want to" → schema "embrace" ACCESSIBLE
     → "I want to embrace, they do not" → schema "embrace" BLOCKED

  RESULT OF SCAN = Resource Access Map for THIS AGENT:
    → List of resources ACCESSIBLE
    → List of resources BLOCKED
    → = Status position WITH THIS AGENT (not in general)
```

### §2.1 — Scan mechanism: chunk-based, not "calculation"

```
🟡 SCAN = CHUNK PATTERN MATCHING (fast, cheap, unconscious):

  Encounter agent → visual/auditory/context inputs fire
    → Spreading activation through compiled status chunks
      → Chunks match → map LOADS quickly (familiar: milliseconds)
      → Chunks do not match → PFC drafts new map (unfamiliar: seconds → minutes)

  FAMILIAR AGENT (compiled maps):
    Encounter spouse → SPOUSE map compiled → load → unconsciously knows access
    → PFC cost ≈ zero (like driving a familiar road)
    → = Explains why being near familiar people = COMFORTABLE (PFC freed)

  UNFAMILIAR AGENT (not yet compiled):
    Encounter new boss → no map → PFC scans:
      Visual cues → estimate domain expertise
      Voice cues → estimate confidence
      Context → estimate authority
    → PFC DRAFTS temporary map → mild cortisol rise (uncertainty)
    → After a few encounters → map COMPILES → next time = unconscious
    → = Explains "exhausted on the first day at a new job"
        = PFC drafting maps continuously

  MULTIPLE AGENTS SIMULTANEOUSLY:
    Party of 20 strangers → 20 maps need drafting → PFC OVERLOAD
    → Cortisol rises → exhausted → want to leave
    → Party of familiar friends → maps compiled → PFC cost low → comfortable
    → ⚠️ "Introvert afraid of parties" = MULTI-CAUSAL, not just PFC cost:
      Status scan cost is only 1 of 6 axes determining momentary capacity.
      Detail: Connection.md §10.2 — Momentary Capacity (6 axes:
      hardware arousal × cortisol × Self-Pattern-Modeling library ×
      environment match × 8-pathway reward ratio × threat level)

  → Scan speed = survival advantage (faster knowing → correct reaction → survival)
  → Compiled maps = energy saving (reuse → no PFC cost)
  → = Evolution optimizes: FAST + CHEAP once familiar, COSTLY but ACCURATE when new


  ⭐ Compiled / Fresh IN STATUS SCANNING
     (Inter-Body-Mechanism.md §3, Self-Pattern-Modeling.md v3.0):

  COMPILED STATUS (FAMILIAR agent):
    → Maps ALREADY compiled through many interactions → body-direct
    → Encounter spouse → map LOADS instantly → KNOWS what to access → cost ≈ 0
    → "I am the expert in domain X" → compiled → CONFIDENCE automatic
    → = Kahneman System 1: fast, automatic, effortless
    → Compilation level determines, NOT content type

  FRESH STATUS (UNFAMILIAR agent / NEW context):
    → No compiled map yet → PFC DRAFTS temporary map → COSTLY
    → Encounter new boss → PFC scans visual/auditory/context → drafts access map
    → "Where am I in this new group?" → fresh → CONSUMES PFC bandwidth
    → = Kahneman System 2: slow, deliberate, effortful
    → PFC draft NOT yet body-verified → accuracy lower than compiled

  TRANSITION — Fresh → Compiled (status compilation):
    → Repeated interaction + verify OK → Hebbian strengthen → compile
    → "First day at a new job" = Fresh (each interaction costs PFC)
    → "After 3 months" = Compiled (maps compiled, unconsciously knows access)
    → = Explains "getting used to it" = Fresh→Compiled transition in status domain
    → = §2.1 "familiar agent" vs "unfamiliar agent" = Compiled vs Fresh PRECISELY

### §3.1 — Layer 1: Solitary animals (no need for status)

```
  THE TIGER HOLDING TERRITORY:

    Territory = a land area with prey = direct resource access
    → Stronger tiger → holds it → eats enough
    → Weaker tiger → loses it → must find another area or die

    Calibration: 1 confrontation → know who is stronger → done
    Dimensions: 1 (physical strength)
    Agents: 1 vs 1 (rarely more)
    Durability: until one side changes (grows larger/grows old)

    → NO complex status system needed
    → Only needs: "stronger → hold, weaker → leave"
    → Hormone drives directly: testosterone + cortisol → fight/flight

    BUT: holding territory itself = HOLDING RESOURCE ACCESS
    → Tiger does not "know" territory = food
    → Hormone drives it to hold, and the RESULT = body-base fed
    → = Proto-status: resource access through physical dominance
```

### §3.2 — Layer 2: Simple groups (1-dimensional hierarchy)

```
  PRIMATE TROOP (20-50 MEMBERS):

    Living together → sharing resources → STATUS SYSTEM NEEDED
    → Alpha: full access (food, sleep spots, mates, movement)
    → Beta: much access but defers to alpha
    → ...
    → Omega: least access, leftover food, poor sleep spot

    Calibration: fighting at first → hierarchy STABLE
    Dimensions: primarily 1 (strength + alliances)
    Agents: 20-50 (each member knows the rank of EVERY other)
    Durability: months → years (until disruption — §6)

    🟢 Sapolsky data:
      → Alpha: low cortisol when stable, high serotonin
      → Subordinate: higher cortisol, lower serotonin
      → = Body MARKS rank with REAL biochemistry

    IMPORTANT FEATURES:

## §4 — 3 INTERACTION MODES: TAKE / EXCHANGE / COMPLY

```
⭐ FROM RESOURCE ACCESS MAP → 3 NATURAL INTERACTION MODES:

  Once body scan is complete (§2), result = resource access gap between
  oneself and the agent. That gap NATURALLY leads to 1 of 3 modes:

  ┌──────────────────────────────────────────────────────────────────┐
  │ Access gap             │ Mode         │ Resources flow           │
  ├────────────────────────┼──────────────┼──────────────────────────┤
  │ I >> THEM (large gap)  │ TAKE         │ I → take from them       │
  │                        │              │ No need to reciprocate   │
  │                        │              │ They dare not decline    │
  ├────────────────────────┼──────────────┼──────────────────────────┤
  │ I ≈ THEM (roughly      │ EXCHANGE     │ I ↔ them                 │
  │ equal)                 │              │ Give each other, receive │
  │                        │              │ from each other          │
  │                        │              │ Negotiate, mutual benefit│
  ├────────────────────────┼──────────────┼──────────────────────────┤
  │ I << THEM (large gap)  │ COMPLY       │ They → take from me      │
  │                        │              │ I follow, defer          │
  │                        │              │ I dare not decline       │
  └────────────────────────┴──────────────┴──────────────────────────┘

  3 MODES = CONTINUOUS SPECTRUM, NOT 3 FIXED BOXES:
    Take ←←←←← Exchange →→→→→ Comply
    → Each domain can fall DIFFERENTLY on the spectrum
    → Example with the same 2 people:
      "Technically I am >> them (take/lead)
       + socially they are >> me (comply/learn)
       + athletically we're equal (exchange/play together)"
    → = 3 modes OVERLAPPING per-domain


  WHY 3 MODES EXIST (evolutionary logic):

    TAKE MODE (status >> other):
      → Evolutionary function: alpha accesses resources QUICKLY, no need to negotiate
      → Cost-saving: negotiating TAKES TIME, taking is FAST
      → BUT: only durable when the gap is GENUINELY large
      → If gap is not genuinely large → the other agent RESISTS → conflict

    EXCHANGE MODE (status ≈ other):
      → Evolutionary function: both access EACH OTHER'S resources
      → POSITIVE-SUM: total resource access INCREASES for both
      → = THE DOMINANT MODE in modern humans (§5)
      → Requires: negotiation, communication, accurate calibration

    COMPLY MODE (status << other):
      → Evolutionary function: avoid unwinnable conflicts
      → Cost-saving: fighting when certain to lose = wasteful + dangerous
      → Body resists complying (Serotonin Ratchet §9.2) → but accepts when gap is too clear
      → BUT: body also seeks OPPORTUNITIES to escape comply (status upgrade)
```

### §5.1 — Specialization creates non-overlapping domains

### §5.2 — Exchange = positive-sum → body rewards BOTH parties

    → BOTH parties body-rewarded

  → Knowledge-Flow.md §1: Output separates → shared → nothing is lost
  → = Knowledge = resource that DOES NOT DIMINISH when shared
  → = Exchanging knowledge = purely POSITIVE-SUM
  → = Explains why cooperation FEELS GOOD: body rewards both

  Agent-Mechanism.md §12.3: Mirror reward override
    → When already sufficient (direct reward ≈ 0), sharing creates
        mirror reward > keeping
    → Body NATURALLY drives sharing when surplus → amplifies exchange
    → = Biology has its own mechanism driving cooperation

  BODY REWARDS FOR COOPERATION:
    → Helping someone → mirror reward (Self-Pattern-Modeling simulates their joy
        → body receives the echo)
    → Being helped → direct reward (body-base fed by agent input)
    → = BOTH THE GIVER AND RECEIVER body-rewarded

## §6 — DISRUPTION → RE-CALIBRATE → NEW EQUILIBRIUM

```
⭐ DISRUPTION CYCLE — A COMMON AND IMPORTANT PATTERN

  Every status system is STABLE → until hit by DISRUPTION.
  Disruption → all maps INVALID → re-calibrate → new equilibrium.
  This cycle repeats at ALL SCALES — from a primate troop to nations.
```

### §6.1 — Primate troop: alpha dies → chaos → new equilibrium

```
🟢 SAPOLSKY DATA — THE FUNDAMENTAL MECHANISM:

  BEFORE DISRUPTION (stable):
    → Alpha eats first, sleeps in the best spot, everyone defers
    → Every member KNOWS their rank → maps compiled → PFC cost ≈ 0
    → Serotonin: alpha HIGH, subordinates LOWER
    → Cortisol: alpha LOW (when stable), subordinates HIGHER
    → = System STABLE, predictable, energy-efficient

  DISRUPTION (alpha dies or is removed):
    → ALL maps become INVALID
    → "Who eats first? Who sleeps in the best spot? Who mates?"
    → = EXTREMELY HIGH UNCERTAINTY → cortisol SPIKES across the whole troop

  RE-CALIBRATION (contestation):
    → Beta, gamma, etc. all CLAIM Take mode
    → Fighting, displays, alliances → DETERMINING who is >> whom
    → Cost HIGH: injuries, energy drain, prolonged cortisol
    → Duration: days → weeks (depending on troop size + clarity of gap)
    → 🟢 Sapolsky: beta serotonin RISES IMMEDIATELY when alpha is removed
      → Body PRE-REWARDS for the upgrade opportunity (before actually winning)

  NEW EQUILIBRIUM:
    → Winner = new alpha → maps RESET across the whole troop
    → Every member compiles NEW maps → PFC cost gradually drops
    → Cortisol drops → serotonin settles → behavior predictable again
    → = NEW SYSTEM, possibly DIFFERENT from before, but STABLE

  CYCLE:
    Stable → Disruption → Re-calibrate → New equilibrium → ...
    → Repeats each time: alpha dies, gets sick, is successfully challenged
```

### §6.2 — Humans: same pattern, more complex

```
🟡 SAME CYCLE, DIFFERENT COMPLEXITY:

  ① COMPANY — CEO replaced:
     Stable: everyone knows the hierarchy, maps compiled, PFC freed for tasks
     Disruption: new CEO → "What is the new boss like? Who is trusted? Who will
       lose their job?"
       → All status maps INVALID
       → Cortisol rises across the whole company
       → PFC bandwidth shifts from TASK → SOCIAL SCAN
       → Productivity DROPS (PFC busy re-calibrating)
     Re-calibrate:
       → Vice presidents: compete for positions close to new CEO
       → Employees: observe + draft new maps
       → Duration: weeks → months
     New equilibrium:
       → Maps compiled → PFC freed → productivity recovers
       → May be DIFFERENT from before (team A rises, team B falls)

  ② FAMILY — Parents divorce:
     Stable: children know "father decides X, mother decides Y" → maps clear
     Disruption: divorce → "Who decides what? Who do we live with? What are
       the new rules?"
       → Family maps ALL invalid
       → Children: cortisol spike + Chunk-Miss (old home absent) +
           Chunk-Gap (new home unknown)
     Re-calibrate: months → years (children compile SLOWER)
     New equilibrium: when new maps compiled (living with mother = map A,
                      visiting father = map B)

  ③ NATION — Leader dies/is overthrown:
     Stable: hierarchy clear, law stable, everyone knows "who decides what"
     Disruption: leader dies → power vacuum
       → LOWER-RANKING LEADERS compete for the position:
         → Alliances, betrayals, coups, emergency elections
         → Possible BLOODSHED (when rules are not strong enough to regulate)
       → CITIZENS: anxious, stockpile, protest or fall silent
       → = The whole nation re-calibrates status maps

     Re-calibration — 3 SCENARIOS:

       A. FAST — smooth transition (strong institutions):
          → Democracy with clear rules: election → new leader → maps compile
          → Clear chain of command: deputy takes over → minimal disruption
          → Duration: weeks → months

       B. SUCCESS AFTER TURBULENCE — unification:
          → Multiple groups competing, none yet strong enough
          → Each group calibrates INTERNAL EQUILIBRIUM (its own hierarchy within)
          → But the WHOLE PICTURE is not yet clear → prolonged stasis
          → Ends with: 1 decisive conflict (1 side wins clearly)
            OR successful negotiation (power-sharing, alliance)
          → → Unification = 1 new hierarchy → maps compile nationally
          → Duration: years → decades

       C. SEPARATION — new nation (= Conflict-Dynamics resolution):

          PROCESS (maps precisely to Conflict-Dynamics.md §1 + §6):

          Step 1 — Conflict calibrates strength:
            → 2+ groups initially claim Take mode (think they are >>)
            → Fight one or more battles → calibrate ACTUAL strength
            → Result: "no group is clearly >>"
            → = Status calibration through conflict → clearly know the real gap

          Step 2 — Stalemate (when no one is strong enough):
            → Each group calibrates INTERNAL EQUILIBRIUM
                (its own hierarchy within the group)
            → BUT the overall picture is still unclear → prolonged stalemate
            → Can smolder for many decades (neither side dares commit to full assault)
            → Ongoing cost: chronic cortisol nationwide, PFC busy threat scanning

          Step 3 — Agreed separation = BREAKING OVERLAP:
            → Both recognize: cost of continuing > benefit of winning
            → Reach AGREED COMMITMENT on clear division of interests
            → = Conflict-Dynamics §6 ①: BREAK OVERLAP (most durable)
              Group A accesses territory/resource X
              Group B accesses territory/resource Y
              Overlap → 0 → Conflict → 0

          Step 4 — New equilibrium:
            → Each entity = its own hierarchy → maps compile SEPARATELY
            → Maps clear → certainty restored → cortisol drops
            → = Re-calibration SUCCESSFUL — the outcome is just 2 entities

          ⭐ SEPARATION IS NOT "FAILED CALIBRATION":
            → Separation = calibration successfully completed through conflict
                + negotiation
            → Both bodies know: "we are EQUAL, DIFFERENT domains"
            → = Same mechanism as friends dividing domains (§5.1):
              "you're good at tech, I'm good at music → each person has a niche"
            → Different scale, same principle: BREAK OVERLAP → peace

          Examples:
            → Yugoslavia → 7 nations (after conflict, gap clear, separated)
            → Sudan → Sudan + South Sudan (referendum = formal agreed separation)
            → North-South Korea (separated but no deep agreement → still tension)
            → Czechoslovakia → Czech + Slovakia ("Velvet Divorce" — separation
                WITHOUT conflict)
          → Duration: years → decades (calibration + negotiation process)

     ⭐ RULE: ALL national-scale disruptions ALWAYS end with A, B, or C.
        There is no "permanent chaos" — because:
          → Body NEEDS certainty (§9) → drives re-calibration
          → Uncertainty = chronic cortisol → NOT sustainable
          → Eventually MUST settle: unification OR separation
        The only question is: how long + how much cost before settling

  ④ SCHOOL CLASS — Classmate transfers in/out:
     Lightest but STILL same pattern:
     → "The new student" arrives → everyone scans → maps shift slightly
     → "The most popular student" transfers out → maps restructure significantly
     → Duration: days → weeks (children calibrate FASTER in this context)
```

## §9 — SEROTONIN: CERTAINTY BIAS BASELINE

```
⚠️ THIS SECTION CLEARLY SEPARATES: FUNCTION (certainty) vs HORMONE (candidate).

  FUNCTION "certainty bias baseline" = CERTAIN IT EXISTS:
    ✅ Everyone experiences "overall, confident or not confident"
    ✅ Baseline STABLE, carries across contexts
    ✅ Changes SLOWLY (weeks → months) as experience accumulates
    ✅ BIAS, not DECISION (PFC can override — §9.1)

  HORMONE "serotonin = chemical carrier" = ONLY A CANDIDATE:
    → Evidence EXISTS but INDIRECT (§9.3)
    → Framework binds to FUNCTION, not the hormone
    → If research proves a different hormone → function DOES NOT CHANGE
```

### §9.1 — 2 layers: Bias baseline + PFC decision

```
🟡 SEROTONIN DOES NOT "DECIDE" — PFC DECIDES:

  LAYER 1 — Certainty bias baseline (body-level, slow, unconscious):
    = BASELINE LEVEL OF CERTAINTY for the whole system
    → High: body FEELS "overall, things are okay" → BIASES schemas to expand
    → Low: body FEELS "overall, not certain" → BIASES schemas to narrow
    → Changes SLOWLY (weeks → months) as experience accumulates
    → = BASELINE BIAS — the default frame when there is NO SPECIFIC DATA

  LAYER 2 — PFC + Compiled chunks (per-context, fast):
    = THE ACTUAL DECISION about which schema to deploy
    → PFC evaluates: "SPECIFICALLY for this task, can I access it or not?"
    → Compiled schema: "done it 100 times → certain" → auto execute
    → = OVERRIDES the certainty bias baseline WHEN DATA IS STRONG ENOUGH

  INTERACTION:
    Bias HIGH + PFC certain  → ✅✅ very confident (bias + data align)
    Bias HIGH + PFC uncertain → 🟡 dare to try (bias pushes, data lacking)
    Bias LOW + PFC certain   → ✅ still does it (data overrides bias)
    Bias LOW + PFC uncertain → ❌ timid (bias + data both block)

  → When SPECIFIC data is strong → certainty bias BARELY MATTERS
  → Only when there is NO DATA (new, unfamiliar, ambiguous) → bias DECIDES
  → = Explains: same person, same low serotonin:
    At work (strong compiled data) → still confident
    With a crush (no data) → timid
    → SAME certainty bias, DIFFERENT outcome per-context → because PFC data differs
```

### §9.2 — Serotonin Ratchet: easy up, hard down

```
🟡 GROUNDED IN EVOLUTIONARY LOGIC (§0):

  Body OPTIMIZES resource access through certainty bias:

  ① STABLE (default):
     Certainty bias HOLDS its current level
     → Avoids oscillation → avoids costly PFC re-calibration of all maps
     → = Energy efficient: "I know where I stand, hold steady"

  ② READY TO RISE (when opportunity presents):
     Opportunity → bias RISES quickly:
       Being praised → rises
       Rival loses → "vacant position" → rises
       Winning a competition → rises
     → 🟢 Sapolsky: monkey beta serotonin RISES IMMEDIATELY when alpha is removed
     → = Body ready to GRAB wider resource access (§6.4)
     → Evolutionary logic: fast grab = advantage

  ③ RESISTS DROPPING (when threatened):
     Status lowered → body RESISTS before accepting:
       Phase 1 — Denial: "no, I'm still good" (bias holds)
       Phase 2 — Rationalize: "they don't understand me" (PFC explains away)
       Phase 3 — Fight back: "I'll prove it" (behavioral resistance)
       Phase 4 — Accept: bias DROPS when evidence is OVERWHELMING
     → Evolutionary logic: maintain wider access = maintain resources = survival

  → = RATCHET EFFECT: easy up, hard down
  → Explains:
    CEO dismissed: prolonged denial (ratchet resists) → prolonged pain
    Newly wealthy: adjusts QUICKLY (ratchet ready to rise)
    Newly impoverished: adjusts SLOWLY (ratchet resists the drop) → prolonged suffering
```

### §9.3 — Evidence limitations: honest assessment

```
⚠️ FRAMEWORK IS HONEST ABOUT EVIDENCE LIMITATIONS:

  EVIDENCE EXISTS (but indirect):
    🟢 Monkeys: rank correlates with BLOOD serotonin (Raleigh 1991)
       → BUT: blood serotonin ≠ brain serotonin
       → 95% of serotonin is in the GUT → measuring blood ≈ measuring gut primarily
    🟢 Monkeys: rank change → cortisol + behavior change (Sapolsky 30+ years)
       → BUT: primate troop = 20-50 members, 1 SIMPLE hierarchy
       → Humans = millions, MANY hierarchies → not directly equivalent
    🟢 Humans: job rank → health change (Whitehall Studies)
       → BUT: does NOT measure serotonin — only measures cortisol + health outcomes
    🟢 Humans: SSRIs (raise serotonin) → more confidence, social behavior changes
       → Supports serotonin connection → but SSRIs affect MANY systems

  EVIDENCE MISSING:
    ❌ No study yet measuring brain serotonin DIRECTLY in humans by status
    ❌ No study comparing serotonin: CEO vs employee
    ❌ Blood serotonin ≈ brain serotonin = not yet confirmed
    ❌ Monkeys (20 members) ≠ humans (millions, many hierarchies)

  FRAMEWORK POSITION:
    "Certainty bias FUNCTION = certain it exists (observed + somatic confirmed)
     Serotonin = CANDIDATE hormone (indirect evidence from monkeys + SSRIs)
     Could be serotonin + other hormones in combination
     Framework binds to FUNCTION — does not assert hormone"

  SEROTONIN HAS MANY FUNCTIONS:
    Function 1 — Certainty bias baseline: status-related
    Function 2 — Mood regulation: irritable / calm / patient
    Function 3 — OCD link: uncertainty → compulsive checking loop
    Function 4 — Sleep regulation: serotonin → melatonin pathway
    Function 5 — Gut signaling: 95% of serotonin is in the gut
    → 1 hormone, 5+ functions → FURTHER reason to call it only a CANDIDATE
```

### §10.1 — Chunk-Shift: status re-evaluated

```
🟡 CHUNK-SHIFT = SAME CHUNKS, DIFFERENT EVALUATION:

  DEFINITION: Chunks about SELF or the AGENT remain unchanged
  — the EVALUATION (valence) changes.
  = Access map DOES NOT change in structure, but valence SHIFTS.

  CASES:

  ① Betrayed (social):
     Chunks "person A trusts me" → betrayal occurs
     → Valence SHIFTS: "safe" → "deceived"
     → Access map WITH THAT PERSON restructures entirely
     → Body-feedback: strong dissonance (Body-Feedback-Mechanism §3.1)

  ② Discovering oneself less capable than thought:
     Thought oneself good at domain X → real feedback: not good
     → Chunks "I am good at X" still there, valence SHIFTS negative
     → Position DROPS in that domain → access maps narrow
     → Body: dissonance ("I am not who I thought I was")

  ③ Unexpectedly recognized (positive shift):
     Didn't know one was good → boss praises in front of everyone
     → Chunks "I did X" still there, valence SHIFTS positive
     → Position RISES → access maps expand
     → Body: reward (surprise + validation)

  CHARACTERISTICS:
    → Content unchanged (I am STILL who I am)
    → Evaluation changed (the ASSESSMENT OF ME differs)
    → Speed: can happen very quickly (1 event is enough)
    → BUT: certainty bias (serotonin) adjusts SLOWLY → lag → stress
    → = Chunk valence shift FAST, serotonin adjusts SLOWLY = mismatch period
```

### §10.2 — Chunk-Miss: status absent

```
🟡 CHUNK-MISS = COMPILED MAP NOT BEING FIRED:

  DEFINITION: Status maps already compiled but NOT ACTIVATING.
  = Previously had access → LOST (or left that context).

  CASES:

  ① Leaving a familiar group:
     Compiled maps for group A → leave the group → maps ABSENT
     → VTA: actual (not firing) < baseline (fires daily) → negative delta
     → = "Missing the old group" = Chunk-Miss (maps compiled but not firing)

  ② CEO dismissed:
     Compiled maps: "everyone complies, I give directions, access FULLY OPEN"
     → Dismissal → maps ABSENT (no one complies anymore)
     → VTA: massive negative delta (dozens of maps lost simultaneously)
     → + Serotonin Ratchet RESISTS (§9.2) → prolonged denial
     → = GENUINELY LOST ACCESS → body response EXTREMELY STRONG
         (proportionate to real loss)

  ③ Transferring schools:
     Maps for old group → COMPLETELY LOST (Chunk-Miss)
     + New context has no maps yet → must draft (Chunk-Gap)
     → = COMPOUND: Miss (old absent) + Gap (new not yet present) — §10.4

  CHARACTERISTICS:
    → Duration depends on: compiled depth × how long one had it ×
        alternative source availability
    → Deep compiled (10 years in the group) → Miss lasts long (months → years)
    → Shallow compiled (3 months) → Miss short (weeks)
    → CAN fade: when new maps compile sufficiently → old maps gradually deactivate
```

### §10.3 — Chunk-Gap: status never had

```
🟡 CHUNK-GAP = NEVER HAD ACCESS, BUT NETWORK DETECTS THE GAP:

  DEFINITION: NEVER compiled access X, but network KNOWS X exists.
  = Wanting access never experienced (aspiration without experience).

  CASES:

  ① Wanting to be respected (never experienced):
     Grew up being looked down on → all compiled maps are "comply"

     → Body: dissonance (wanting without a compiled basis)

  ③ Children wanting to "be adults":
     Sees adults having wide access → wants it
     → No compiled maps for adult context
     → = "Wanting to grow up fast" = Gap detection for access maps

  CHARACTERISTICS:
    → Gap ≠ Miss: Miss = HAD it then lost, Gap = NEVER HAD it
    → Gap → can transition to Miss (Body-Feedback-Mechanism §3.3):
      when Imagine-Final preview repeats + stabilizes → compiles baseline →
      reality doesn't match baseline → Miss
    → Gap MORE durable (doesn't fade because there's nothing to replace)
    → Gap CAN resolve: when compiling real maps (through experience, not imagination)

  Low           Maps STABLE → body-need easily met → calm
  Moderate      Maps STABLE + ACTIVE calibration GOOD (optimal zone)

  Very high     Maps NEARLY CLOSED → "worthless" → isolate

  WHY CORTISOL NARROWS MAPS:
    → High cortisol = body signals "danger, conserve resources"
