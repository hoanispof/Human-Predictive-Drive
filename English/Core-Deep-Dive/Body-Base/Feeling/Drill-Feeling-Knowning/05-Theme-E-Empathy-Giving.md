### §2.2 — The 3 Mechanisms Merged as "Mirror / Empathy"

Mirror-Neuron-Analysis.md §Refined Hypothesis decomposes:

**① PATTERN MATCHING** (limbic, near-innate, from birth):
- Detects acoustic/visual patterns matching own experience
- Example: contagious crying in newborns
- Mechanism: "sound of crying → similar to when I cry → body reacts"
- **NOT agent awareness** — arousal contagion, not empathy
- Evidence: Dondi 1999 newborn cry discrimination 🟢

**② AGENT MODELING** (learned, emerges 14–24 months):
- Brain builds prediction model for unpredictable agents
- Own-state mapping onto other
- "Mother has her own state; that state is SIMILAR to how I have states"
- **TRUE empathy begins here**
- Evidence: Warneken 2006–2007 helping, rouge test 18–24m, Svetlova 2010 🟢

**③ SCHEMA SIMULATION** (compiled, mature adult):
- Compiled agent model fires WITHOUT real input
- PFC creates virtual agents (deities, departed loved ones, "inner voice")
- Explains: religion, statues, placebo, memory photographs, lockets
- Evidence: Piaget animism, religious neuroscience, placebo research

**→ "Empathy" in everyday language merges all 3.** Framework's Theme E uses
Resonance (mechanism ② primarily) as the core.

**Developmental trajectory**:
- 0–6m: Pattern matching only (arousal contagion, NOT empathy)
- 6–12m: Social referencing, early agent detection
- **14–24m**: AGENT MODELING emerges (TRUE empathy begins with rouge test +
  empathic helping)
- 2–7y: Over-apply agent model (animism)
- 7y+: Refine + mature empathy

**Evidence support**:
- Svetlova 2010: instrumental helping at 14m, empathic helping at 18–24m 🟢
- Meltzoff "Like Me" hypothesis: own template applied outward 🟢
- Rouge test: self-recognition 18–24m → enables self-other mapping 🟢

### §2.3 — "Thiện Cảm" (Warm Regard) — Framework Construction

User asks: "What is warm regard (thiện cảm)? When does an agent trigger it?"

**Framework does NOT have "thiện cảm" as an explicit term.** Theme E **constructs**
it from framework vocabulary.

**"Thiện cảm" = compound positive valence toward an agent**

[Thiện cảm: the Vietnamese term for the warm, positive feeling of affinity toward
another person — no single English word captures it fully; encompasses "warm regard,"
"positive affinity," "goodwill."]

**Proposed decomposition** (Theme E synthesis):

**Thiện cảm = convergence of multiple positive signals**:

1. **L0 safety** (threat minimal): agent doesn't trigger threat alarms — no danger
   signals
2. **Mirror reward positive**: Resonance simulation returns positive fit-judgment
   (they seem well, in a good state)
3. **Valence compiled positive** (Feeling-Sources §9): prior interactions compiled
   into positive valence chunk
4. **Schema match positive**: agent fits "safe/good person" schema
5. **Agent input fulfilled** (Feeling-Sources §5): person provides co-regulation,
   warmth, or social benefit

**When ANY subset of the above fires POSITIVE, body generates "warm regard"
feeling.**

**Contrast with negative feelings**:
- Antipathy / dislike = L0 threat + mirror negative + valence compiled negative
- Neutral = no strong signals either way
- Thiện cảm / warm regard = positive convergence

**→ Answer Q4 "What is thiện cảm?"**: Compound positive valence toward an agent,
arising from multiple positive signals converging (safety + mirror + valence + schema
match + agent input). NOT a single dedicated feeling — a **label for positive
convergence.**

**When does warm regard activate?** (user's Q5):

Framework answer — **when any of these conditions fire**:

1. **Safe agent signals**: calm body language, smiling, open posture → Resonance
   returns "not threatening"
2. **Previous positive interactions**: compiled valence chunks fire positive
3. **Moral alignment** (schema match): agent's actions match one's value schema
   (ethical, kind-natured)
4. **Resemblance to loved one**: triggers relationship schema activation (mother-
   resemblance case — see §2.10)
5. **Reciprocal gift** ("the other person gave me something"): triggers fairness
   schema + reciprocal altruism
6. **Community belonging cue**: agent shares group identity (Uncle Ho + soldiers
   case — see §2.9)
7. **Vulnerability cue** (Empathy-Mirror §4): agent is weak/young → Empathy-Mirror
   fires stronger
8. **Competence cue**: agent shows skill/kindness that signals trustworthiness

**Why do you "need warm regard" before helping?**

Because **giving is driven by a positive body-level signal, not by abstract duty
alone.**

Framework evidence (Empathy-Mirror §6.5):
> "Share > keep BECAUSE mirror reward is real biochemistry. Body NATURALLY drives
> sharing when there is surplus — because mirror reward > keep reward."

**If no warm regard** → no mirror reward anticipation → body doesn't drive giving →
"I could help but don't feel like it."

**If warm regard present** → mirror reward anticipation → body drives giving → "I
want to help."

**→ "Needing warm regard"** is framework's **positive trust precondition**
(Anchor-Schema §2). Without positive valence, giving becomes forced and unsustainable.

### §2.4 — Surplus Mango Tree Case: Natural Resource Distribution

**User's Case A**: "The house has a mango tree with ripe fruit you can't eat fast
enough, so you give to the neighbors."

**Framework's DIRECT coverage** — [Empathy-Mirror.md §6.5 Evolutionary

> "EMPATHY-MIRROR = NATURAL RESOURCE DISTRIBUTION MECHANISM. No laws needed. No
> ethics needed. Body NATURALLY drives sharing when there is surplus — because mirror
> reward > keep reward."

**Mechanism breakdown**:

```
Step 1: Body-base sufficient (L1 met — mangoes already plentiful for self)
         → One more mango for self: reward ≈ 0 (already satiated)

Step 2: See neighbors (potential recipients)
         → Resonance simulator runs
         → Visualize: "they eat the mango → they're happy"
         → Mirror fire: own body pre-experiences neighbor's pleasure (weakly)

Step 3: Body-level calculation (unconscious)
         → Keep 1 more mango: reward ≈ 0
         → Give 1 mango → mirror reward: weak positive
         → Mirror reward > keep reward (since keeping is already satiated)

Step 4: Drive emerges naturally
         → "Give to neighbors" action arises from body's positive calculation
         → NOT conscious "moral duty" deliberation
         → Body drives this naturally

Step 5: Give → see neighbor receive → real mirror fire
         → Resonance: neighbor happy
         → Own body: genuine mirror reward (stronger than preview)

```

**Optimal conditions** (Empathy-Mirror §6.5):
> "Empathy-mirror optimizes when: ① Own body-base already sufficient, ② Perceived
> ability high, ③ Mirror input present."

Mango tree case fits all 3:
- ① Body-base sufficient ✓ (mangoes in surplus, already well-fed)
- ② Ability ✓ (easy to give — no real cost)
- ③ Mirror input ✓ (see neighbor receive)

**Result**: **Pure positive reward**, no cost. This is "healthy giving" in
framework's §8.5 spectrum:
> "Healthy (cost ≈ 0): House has surplus fruit → give to neighbors → mirror joy →
> cost ≈ 0. → Own body-base unaffected → PURE reward."

**→ Mango tree case = framework's paradigm of healthy giving.** Body naturally drives

**Evolutionary logic** (§6.5):

### §2.5 — Communal Banana Tree Case: Schema-Driven Giving

**User's Case B**: "The house has a banana tree with ripe fruit (you'd eventually
eat it all yourself, you still need it) — but because of communal habit you have to
share. Not sharing makes you feel guilty [áy náy], because the banana tree is planted
in the visible front garden and everyone who passes knows. So you still share a
portion with neighbors."

**Key difference from mango tree**: body-base is NOT satiated (bananas are needed for
self), but you still give due to **community norm** + **guilt/social discomfort**
[áy náy].

**Framework's analysis** — **mixed trust case**, requires decomposition:

**Mechanism breakdown**:

```
Step 1: Body-base not fully satiated
         → Keeping bananas: positive reward (needed for self)
         → Giving bananas: SOME cost to self

Step 2: Community visibility factor
         → "Banana tree planted in visible front garden — everyone passing knows"
         → Social proof active: neighbors see what you have
         → External inject (Anchor-Schema §3 Source ④ Social pressure)

Step 3: Community norm schema fires
         → Compiled chunk: "community expects sharing"
         → Schema activates automatically
         → Violating norm → schema triggers guilt/social discomfort [áy náy]

Step 4: Trust evaluation
         → Keep all → positive body reward (L1 need met)
         → Keep all → NEGATIVE schema trust violation (guilt/social discomfort)
         → Give some → moderate body cost + mirror reward + schema compliance
         → Balance: schema compliance outweighs body cost

Step 5: Resolve via partial sharing
         → "Share A PORTION" (not all)
         → Maintains community schema compliance
         → Reduces body cost (still keeps some for self)
         → Balances positive + negative trust
```

**Framework's trust analysis**:

This is **NEGATIVE TRUST dominant** (Anchor-Schema §2):
- User's phrasing: "not sharing makes me feel guilty" = **avoidance of guilt/shame**
- Motivation: avoid schema violation → cortisol drop relief ("phew, avoided the
  guilt")
- NOT: positive anticipation of mirror reward

**Contrast with mango tree**:
- Mango tree: POSITIVE trust (reward ahead) → opioid-type reward ("pleasant")
- Banana tree: NEGATIVE trust (avoid harm) → cortisol drop relief ("phew, escaped")

**Sustainable?** Mixed:
- In traditional community: sustainable via community reinforcement
- In modern breakdown: becomes burdensome ("why do I have to do this?")
- Framework warning (§8.5): "Schema hijack override" if body-base suffers
  significantly

**→ Banana tree case = schema-driven giving with mixed trust.** Framework recognizes
this as:
- Community-typical (not pathological)
- Driven by social schema + external inject (Source ④)
- NEGATIVE trust component (avoiding guilt)
- Sustainable WITH community reinforcement
- Can become burdensome WITHOUT

**→ Answer Q2 (Case B)**: Sharing even when body doesn't benefit = social schema
(community norm) + guilt avoidance (negative trust). Framework labels this distinctly
from pure mirror reward (mango tree case).

### §2.7 — Discomfort from Giving: 4 Sub-Cases

**User's Q8**: "When does helping feel uncomfortable?"

**User's 4 sub-cases**:
A. Don't want to do it (too tedious, too unpleasant, cost too high)
B. You give but the person criticizes
C. You give but the reaction is far lower than expected
D. You help and then discover you were exploited

**Framework decomposes each**:

#### Case A — Cost too high / task unpleasant

**Mechanism**:
- Body-base cost > anticipated mirror reward
- Either: task itself generates dissonance (bored, tired, unpleasant)
- Or: recipient not triggering warm regard → no mirror reward anticipation
- **Net body calculation**: NEGATIVE → don't want to do it
- If forced anyway: dissonance accumulates

**Framework's Resonance §4 "Case 2 State-gated mismatch"**:
> "I'm sad + I see everyone happy → I don't care."

Generalized: "Own body drained + task cost high → can't summon mirror reward →
giving feels like a burden."

**Sustainable?** No — cost > reward over time = resentment.

#### Case B — Recipient Rejects Gift ("person criticizes")

**Mechanism**:
- Expected Resonance output: "recipient happy → mirror positive"
- Actual output: "recipient dissatisfied → mirror negative"
- **Negative prediction delta** (Schultz 🟢) — VTA: actual < expected
- Trust violation (Anchor-Schema §2): "I expected positive response, got negative"
- **Negative feelings**: disappointment + hurt + possibly anger

**Chunk dynamics** (Body-Feedback-Mechanism.md §3):
- **Chunk-Shift**: valence of "giving to this person" SHIFTS negative
- **Chunk-Miss**: expected mirror reward ABSENT (compiled preview ≠ reality)
- = Compound: Shift + Miss → stronger dissonance than either alone

**Worse**: rejection signals low valuation by recipient:
- Implicit: "your gift wasn't good enough"
- Status signal hit: being valued low (evaluative signal)
- Compounds negative feeling

**Framework evidence**:
- Eisenberger 2003: social rejection = physical pain circuits 🟢
- Rejection of a gift = rejection of your effort/care

#### Case C — Low Reaction vs. High Expectation

**Mechanism**:
- Expected mirror reward: high (anticipated strong gratitude, visible joy)
- Actual: low (polite acknowledgment, muted response)
- **Prediction delta magnitude = expectation - reality**
- If expectation was high → delta large → dissonance large
- Framework: **same mechanism as Case B** but weaker (polite acknowledgment ≠
  rejection)

**Why expectation matters — Gap→Miss transition at micro-scale**
(Body-Feedback-Mechanism.md §3.3):
- PFC preview "give → they're happy → I'm happy" (Imagine-Final preview)
- Preview repeated (during preparation to give) → body compiles into baseline
- Baseline = "mirror reward HIGH" = expected
- Reality: low reaction → actual < compiled baseline
- = **Chunk-Miss** (preview already compiled into expectation, reality misses)
- Dissonance proportional to delta (higher expectation → greater disappointment)
- = Micro-scale version of Gap→Miss transition (minutes instead of months/years)
- Body feels "investment without return"

**User's own words**: "they don't smile or say thanks when beforehand you imagined
they'd be happy."

**"The higher the expectation, the greater the disappointment"** = precisely this
mechanism (Body-Feedback-Mechanism.md §3.3 ⑦).

#### Case D — Discovery of Exploitation

**Mechanism** (most complex):
- Past action: gave based on positive trust ("I believed they were worthy")
- New information: recipient was manipulating / exploiting
- Trust violation (Anchor-Schema §2):
  > "Trust violation → RIPPLE EFFECT onto related schemas. Schema 'X is trustworthy'
  > IS DAMAGED."
- Retroactive reinterpretation: past giving actions now tagged as "wasted"
- Valence compiled about this person FLIPS from positive to negative
- Anger + betrayal + self-blame compound

**Chunk dynamics** (Body-Feedback-Mechanism.md §3 + §4 Compound):
- **Chunk-Shift**: valence FLIP on entire relationship chunks (content preserved,

- **Chunk-Miss**: positive connection LOST (compiled relationship baseline severed)

- = **Triple compound** — comparable to "close friend betrayed a significant trust"

**Feelings produced**:
- **Anger** at recipient (betrayal) — Chunk-Shift primary
- **Shame** at self (for being fooled) — Chunk-Gap (identity)
- **Hypervigilance** going forward (harder to trust again) — compiled threat chunks
- **Mourning** the lost positive relationship — Chunk-Miss

**Framework quote** (Anchor-Schema §2 Build/Break):
> "Trust violation → ripple effect. 'Having been betrayed once' → threshold HIGHER
> for trusting again. Requires: domain feedback ✓ MANY TIMES + body confirmation
> MANY TIMES."

**Recovery** is slow: Anchor-Schema §2 Rebuild section.
Negativity bias (🟢 Baumeister 2001): threat chunks compile FAST, reward chunks
compile SLOWLY → recovery is ASYMMETRIC.

### §2.8 — Selfless Giving (Cho đi vô tư) Decomposed

**User's final question**: "Giving freely without expectation — if they like it you're
happy, if they don't that's fine, give to someone else another time. Is this a trained
skill, or is there always some default probability of this kind of selfless giving?
Or has the person developed a broader schema that can't easily be violated by a
single individual?"

**Three hypotheses from user**:
1. Trained skill (learned via practice)
2. Default probabilistic behavior (random occurrence)
3. Broader schema not easily violated by individuals

**Framework's answer**: **Primarily option 3 with elements of 1.** Not option 2.

**Decomposition of selfless giving**:

**What it means phenomenologically**:
- Give freely
- Recipient's response doesn't strongly affect giver's state
- If they like it: mild positive
- If they don't: mild neutral or mild disappointment, but NOT deep hurt
- Overall: stable giving disposition

**Mechanism for stability** (framework synthesis):

**Option 3 elaborated — Broader schema**:

The giver has anchored positive trust in a **broader schema** such as:
- **Religious / spiritual schema**: "I give as practice, not for recipient reaction"
  (Buddhist dana, Christian charity)
- **Community schema**: "I give because that's who we are, regardless of individual
  response"
- **Personal values schema**: "I give because it reflects MY values, independent of
  recipient"
- **Abundance schema**: "I have enough — giving doesn't deplete me"

**Why this stabilizes against individual rejection**:
- Positive trust isn't anchored in the RECIPIENT specifically
- Positive trust is anchored in the SCHEMA/FRAMEWORK of giving itself
- Individual recipient response is a small input to a large anchor

### §2.11 — Positive vs. Negative Trust Applied to Giving

expectation = optimal giving anchor.

**Healthy progression**:
- Early stage: negative trust common (guilt, duty)

- Master stage: broad schema anchor (give regardless of individual response)

This is consistent with spiritual traditions' emphasis on cultivating non-attached
giving (Buddhist dana pāramitā, Christian agape).
