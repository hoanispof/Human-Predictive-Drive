### §1.2 — Activation Dynamics → Feeling Properties

```
🟡 CHUNK ACTIVATION DYNAMICS → FEELING GRADIENT:

  Activation dynamics directly produce feeling properties:

  ACTIVATION PROBABILITY DISTRIBUTION → FEELING CLARITY:
    → Strong activation (1-2 dominant chunks) → CLEAR feeling
    → Scattered activation (many weak chunks) → VAGUE feeling
    → = Probability distribution in chunk system →
      gradient position in Feeling-Mechanism-Deep §5
    → = SAME mechanism producing both chunk pattern AND feeling quality

  ACTIVATION SPEED → FEELING TIMING:
    → Fast activation (well-compiled, well-linked chunks) → INSTANT feeling
    → Slow activation (weak links, new domain) → DELAYED feeling
    → Expert: chunks fire fast → feeling IMMEDIATE ("gut feeling")
    → Beginner: chunks fire slow → feeling DELAYED or absent

  ACTIVATION COMPETITION → FEELING AMBIVALENCE:
    → Two chunks compete for probability share (Chunk-Activation-Dynamics §3)
    → PFC receives BOTH signals → ambivalent feeling
    → "I'm happy and also sad" = 2 chunk patterns competing
    → Resolution: one wins probability share → feeling clarifies
    → = Mixed feelings = chunk competition, not "confused emotions"

  TRIGGER SURFACE SIZE → FEELING SENSITIVITY:
    → Large trigger surface → many cues activate chunk
    → = HIGH sensitivity to related stimuli
    → Small trigger surface → few cues activate
    → = LOW sensitivity
    → Trauma: large trigger surface → HYPER-sensitivity to triggers
    → = Feeling sensitivity IS trigger surface size
    → = Explains why trauma survivors "overreact" — large surface, appropriate activation
```

### §1.3 — Compile Context → Feeling Quality

```
🟡 CHUNK COMPILE CONTEXT → ADULT FEELING QUALITY:

  HOW chunks were compiled → HOW feelings feel in adulthood:

  COMPILED UNDER SAFETY (curiosity-driven, opioid-tagged):
    → Chunks: multi-modal, deeply linked, rich associations
    → Feelings: accurate, nuanced, approach-oriented
    → = "Expert feeling" — large library, healthy baseline
    → = Feeling-Mechanism-Deep §6.1: quality model factor ②

  COMPILED UNDER THREAT (forced, cortisol-tagged):
    → Chunks: threat-driven, narrow, defensively linked
    → Feelings: biased toward threat detection, avoidance-oriented
    → = "Trauma-shaped feeling" — distorted body signals
    → = Feeling-Mechanism-Deep §6.4: 4 distortion patterns

  COMPILED WITH RICH VOCABULARY (Label-As-Handle strong):
    → Chunks: verbal-feeling links abundant
    → Feelings: easy to identify, communicate, use in planning
    → = "Literate feeling" — training to Stage 4

  COMPILED WITHOUT VOCABULARY (Label-As-Handle absent):
    → Chunks: body-level only, no verbal retrieval path
    → Feelings: present but cannot be identified or communicated
    → = "Implicit feeling" — stuck at Feel-Consciousification–Feel-Observation

  → = Compile context IS the history that produces current feeling quality
  → = Chunk system IS feeling system, viewed from the angle of compile history
  → = Feeling-Literacy-Training explains: training = updating compile context
```

## §2 — Feeling → Chunk: Feedback Mapping

> **Direction B: Feelings MODIFY chunk operations.**
> Feeling is not just output — feeling FEEDS BACK into the chunk system.
> This is the direction that most people miss.

### §2.1 — How Feeling Modifies Chunk Activation

```
⭐ FEELING → CHUNK FEEDBACK (5 mechanisms):

  MECHANISM ① — BODY VOTE → LINK STRENGTHENING / WEAKENING:
    → Step ⑦ body vote fires (smooth / resist / empty)
    → SMOOTH → current chunk interpretation REINFORCED
      → Link probability INCREASES (Hebbian: "neurons that fire together wire together")
    → RESIST → current interpretation WEAKENED
      → Link probability decreases relatively
      → Alternative chunks searched → new links may form
    → = Body vote = SELECTION PRESSURE on chunk links
    → = EVERY feeling episode = training data fed back to chunk system
    → = This is why paying attention to feelings builds expertise:
        each accurate body vote strengthens the right links

  MECHANISM ② — EMOTIONAL STATE → PROBABILITY BIAS:
    → Current feeling → mood-congruent activation
      (🟢 Bower 1981: feeling "sad" → sad-compiled chunks get probability BOOST)
    → Feeling "energized" → achievement-compiled chunks BOOSTED
    → = Feeling state BIASES which chunks fire next
    → = Circular loop: feeling → chunk activation bias → more of same feeling
    → = Explains mood persistence: sad → sad chunks → deeper sadness → etc.
    → Breaking the cycle: external input or deliberate PFC attention shift

  MECHANISM ③ — FEELING DETECTION → CURIOSITY DRIVE:
    → Vague feeling detected → convergence zone present
    → Convergence zone → VTA fires → curiosity drive activated
    → Curiosity → deliberate search for matching chunks
    → = Feeling DRIVES the chunk discovery process
    → = "I feel something's off" → search → new chunks activated → clarify
    → = Feeling = ENGINE of the discovery cycle
    → = This is why "trust your vague feeling" + investigate is the right approach

  MECHANISM ④ — REWARD FEELING → CHUNK TAGGING:
    → Opioid reward fires (Feeling-Mechanism-Deep §4) → chunk tagged "valuable"
    → Future activations: valuable chunks get priority in probability distribution
    → = Reward feeling signals: "THIS pattern matters — keep it"
    → = Without reward → chunk not specially tagged → may fade over time
    → = Feeling system SELECTS which chunks survive long-term
    → = This is why positive emotional experiences accelerate expertise:
        opioid reward → chunk tagged valuable → future activation easier

  MECHANISM ⑤ — TRAUMA FEELING → CHUNK DISTORTION:
    → Extreme negative feeling → cortisol-tagged chunks (Mechanism-Deep §6.4)
    → Future activations: cortisol-tagged chunks → avoidance feeling fires
    → = Feeling distortion PROPAGATES:
      distorted feeling → distorted chunk activation → more distorted feeling
    → Breaking the cycle: competitive re-linking via new positive experiences
      that create alternative pathways competing for probability share
    → = This is why trauma therapy requires NEW experiences, not just insight:
        new opioid-tagged chunks must outcompete old cortisol-tagged ones
```

## §3 — Integration Points (Where the 2 Systems MEET)

### §3.1 — PFC as Observation Interface

```
🟡 PFC = WHERE CHUNK SYSTEM AND FEELING SYSTEM MEET:

  PFC role IN THE CHUNK SYSTEM:
    → Hold chunks in working memory (~4-7 items simultaneously)
    → Type 4 deliberate linking (logical, intentional connections)
    → Search, plan, compare, evaluate
    → = Logic side of PFC function

  PFC role IN THE FEELING SYSTEM:
    → Observe integrated body state (Step ④ in 8-step flow)
    → Label feeling (Step ⑥, Label-As-Handle)
    → Interpret body vote (Step ⑦)
    → Generate explanation (Feel-Explanation)
    → = Feeling side of PFC function

  → PFC does BOTH — not two separate functions in separate systems
  → Same PFC that holds chunks ALSO observes feelings
  → = Logic-Feeling.md: Compiled/Fresh axis → one PFC, two modes
  → = When PFC observes chunk activation → that IS feeling
  → = When PFC manipulates chunks deliberately → that IS reasoning
  → = Feeling and thinking = two aspects of PFC observing the same chunk system
  → = Not "left brain vs right brain" — one integrated observer, two functions
```

### §3.2 — Body Vote = The Meeting Point

```
⭐ BODY VOTE = WHERE CHUNK AND FEELING SYSTEMS MEET:

  Body vote (Step ⑦) is the CRITICAL integration point.

  FROM THE CHUNK SYSTEM PERSPECTIVE:
    → PFC holds a candidate chunk interpretation
    → Sends interpretation DOWN to body-base for simulation
    → Body simulates: "does this match body-need?"
    → Result = body vote (smooth / resist / empty)
    → = Chunk evaluation mechanism: is this interpretation correct?

  FROM THE FEELING SYSTEM PERSPECTIVE:
    → Body generates feeling signal
    → PFC attempts to label the signal
    → Label sent DOWN to body for verification
    → Body responds: smooth (correct) / resist (incorrect) / empty (unclear)
    → = Feeling verification mechanism: is this label accurate?

  → SAME MECHANISM described from two angles
  → Body vote = WHERE chunk interpretation and feeling detection CONVERGE
  → = This is WHY Gendlin's "resonating" step (§7.2 of Feeling-Accuracy) works:
    → Try label → body vote → smooth = both systems agree
  → = This is WHY expert "gut feeling" can be reliable:
    → Body vote on situation = chunk-based pattern match →
      verified by body simulation → feeling report


  ⭐ BODY VOTE APPLIES TO BOTH SYSTEMS SIMULTANEOUSLY:

    When making a decision:
      CHUNK SIDE: "available chunks suggest option A fits"
      FEELING SIDE: "body feels smooth about option A"
      BODY VOTE: "smooth" = BOTH systems agree = proceed

    When chunks conflict with feeling:
      CHUNK SIDE: "logic says option A"
      FEELING SIDE: "body resists option A"
      BODY VOTE: "resist" = systems DISAGREE = investigate before committing

    → Dissonance = chunk system and feeling system out of alignment
    → Resolution: either update chunks (new information corrects interpretation)
      or update feeling interpretation (misattribution correction)
    → = Body-Dissonance.md: dissonance = chunk-feeling misalignment
    → = When you "feel right" AND "think right" = aligned systems = high confidence
    → = When you "feel wrong" but "think right" (or vice versa) = investigate
```

### §3.3 — Convergence Zone = Feeling BEFORE Chunk

```
🟡 CONVERGENCE ZONE = THE DYNAMIC TRANSITION POINT:

  Chunk-Discovery-Lifecycle §2 defined convergence zone:
    = Intersection of many activation paths
    = Before a dedicated chunk / label exists for the pattern

  IN BRIDGE CONTEXT:

  CONVERGENCE ZONE = where the feeling system detects something
    that the chunk system has NOT YET compiled into a dedicated chunk:

    → Many chunks fire weakly → activation OVERLAPS at their intersection
    → Body integrates the convergent signals → "something here" (a feeling)
    → But NO single chunk yet represents this specific pattern
    → = Feeling system DETECTS what chunk system hasn't NAMED
    → = "Vague" = body knows, chunks don't have it formalized yet
    → = This is the structural explanation for "I know something is wrong
        but I can't say what"


  RESOLUTION (convergence zone → chunk):
    → PFC dwells on the convergence zone (Gendlin Focusing, or natural incubation)
    → Pattern clarifies → new chunk COMPILED from convergence zone signal
    → = Feeling was FIRST → chunk comes AFTER
    → = Feeling = PREVIEW of the chunk that doesn't exist yet
    → = This is WHY trusting a vague feeling (in expert domain) is justified:
      body is detecting a REAL pattern that the chunk system hasn't formalized


  AFTER RESOLUTION:
    → New chunk exists → label available → feeling becomes CLEAR
    → Something that was vague is now named and accessible
    → Convergence zone → chunk = vague feeling → named feeling
    → = Bridge moment: feeling system FEEDS chunk system with raw material →
      chunk system FORMALIZES it →
      feeling system can now READ it clearly


  → = CONVERGENCE ZONE is the DYNAMIC BOUNDARY between
    "feeling-system-knows" and "chunk-system-has-compiled"
  → = The frontier where the two systems are most visibly distinct —
      and where the integration happens
```
