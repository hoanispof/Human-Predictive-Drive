## §1 — Feeling = PFC Observation of Body-Chunk Interaction

### §1.1 Core Claim

```
⭐ FEELING IS NOT A SEPARATE SYSTEM.

  Framework claim (integrated from Chunk-Analysis + Feeling.md):

  Feeling = WHAT PFC SEES when it observes body-chunk interaction.

  There is no separate "feeling module" in the brain.
  There is no "emotion center" isolated from body processing.

  WHAT EXISTS:
    → Body systems fire continuously (body-base, mirror, schema, imagine-final...)
    → Chunks activate according to probability distribution
    → Body integrates signals at insula + ACC + VMPFC (🟢 Craig 2009)
    → PFC observes the integrated state → THAT IS FEELING

  Feeling = INTERFACE between body computation and PFC awareness.
  Body = processes (compiled patterns fire).
  Feeling = result that surfaces for PFC to see.
  PFC = observes + labels + explains (with decreasing fidelity at each step).

⭐ FOUR IMPORTANT CONSEQUENCES:

  ① Feeling DOES NOT CREATE new information.
     → Body has ALREADY finished computing before feeling appears
     → Feeling ONLY reports — it is messenger, not source
     → Temporal order: Body computes → Feeling emerges → PFC observes
     → 🟢 Damasio 1994, 1999: somatic markers PRECEDE conscious decision

  ② Feeling CANNOT BE WRONG at the raw level
     (Feel-RawSignals through Feel-Consciousification)
     → Body truth IS body truth — body does not deceive itself
     → ERRORS occur at PFC level: wrong label, wrong explanation
       (Feel-Labeling through Feel-Explanation)
     → = Fidelity decreases as it moves up the 7-layer stack (Feeling.md §2.9)

  ③ Feeling is NOT "emotion narrow"
     → Emotions (joy/sadness/anger/fear) = 1 small subset
     → Feeling = ALL body-state changes that PFC can observe
     → 7 types: body-physical, emotional, social reading, cognitive,
       predictive, valence, schema-triggered (Feeling.md §1.2)

  ④ Feeling quality DEPENDS ON chunk library
     → Expert: large, domain-real chunk library → accurate feeling
     → Beginner: small library → unreliable feeling
     → Trauma-compiled: distorted body-state direction → distorted feeling
     → = Feeling quality is NOT innate — it is a function of chunk history
```

### §1.2 Body-First Invariant (Temporal Order)

```
🟢 ESTABLISHED — body computes FIRST, PFC observes AFTER:

  EVIDENCE:
    → Damasio VMPFC patients: PFC intact, feeling broken → cannot decide
      (Iowa Gambling Task: logic OK but "gut feeling" absent → random choices)
      🟢 Bechara et al. 1994, 1997
    → Libet 1983: brain activity (~350ms) BEFORE conscious awareness
    → Galvanic skin response: autonomic body change BEFORE conscious detection
      🟢 Bechara 1997: skin conductance changes BEFORE conscious "bad deck" knowledge
    → LeDoux 1996: amygdala pathway (~12ms) BYPASSES cortex entirely
    → Body signals via vagus nerve → brainstem → thalamus → insula
      → PFC only gets INTEGRATED result (Craig 2002, 2009)

  FRAMEWORK FORMALIZATION:

    ┌─────────────────────────────────────────────────────────────┐
    │  T=0ms    Body systems fire (body-base + mirror + schema...)│
    │  T=50ms   Integration begins (insula + ACC)                │
    │  T=100ms  Integrated state forms                           │
    │  T=150ms  Consciousification threshold                     │
    │           (Feel-Consciousification)                        │
    │  T=200ms+ PFC observation begins (Feel-Observation+)       │
    │  T=300ms+ Label attempt (Feel-Labeling)                    │
    │  T=500ms+ Explanation generation (Feel-Explanation)        │
    └─────────────────────────────────────────────────────────────┘

    → Body leads. PFC follows. ALWAYS in this order.
    → PFC CAN direct attention (top-down) — but body STILL computes first.
    → = PFC = camera, NOT director. Body = scene, NOT controlled by PFC.

  ⚠️ PFC CAN INFLUENCE THE NEXT CYCLE:
    → PFC observes → labels → plans action → acts → new body state
    → PFC does NOT change CURRENT feeling (already computed)
    → PFC ONLY changes NEXT feeling (via action → new input → new body state)
    → = Loop, not linear (Feeling.md §3 Feeling Loop)
    → = PFC's power = INDIRECT, through action, not through override
```

## §2 — Signal Integration: Multi-Channel → Unified Feeling

### §2.1 The Integration Problem

```
⭐ WHY INTEGRATION IS NECESSARY:

  The body has THOUSANDS of signals firing simultaneously:

    Exteroceptive: 120M rods + 6M cones + 16K hair cells + 10M olfactory
      neurons + 10K taste buds + millions of tactile receptors
    Proprioceptive: muscle spindles + joint receptors + vestibular
    Interoceptive: cardiovascular + respiratory + visceral + metabolic +
      hormonal + nociception + thermal + self-signal meta

  PFC CANNOT process thousands of streams individually.
  PFC working memory capacity = ~4-7 items (🟢 Cowan 2001).

  → Body MUST compress thousands of signals → FEW integrated states
  → Integration = the compression mechanism

  NOT information LOSS (at integration level):
    → Body does not "discard" signals
    → Body INTEGRATES: many → one coherent state
    → Like an orchestra: 80 instruments → 1 musical experience
    → Each instrument is still "there" — but the listener hears MUSIC, not 80 parts
```

### §2.2 Integration Hardware

```
🟢 ESTABLISHED — 3 primary integration hubs:

  ① ANTERIOR INSULA — "interoceptive cortex"
     → 🟢 Craig 2002, 2009: "seat of subjective awareness"
     → Receives: ALL interoceptive afferents (via lamina I → thalamus)
     → Receives: exteroceptive signals (cross-modal)
     → Produces: unified "how do I feel RIGHT NOW" representation
     → Right anterior insula: moment-to-moment body-state awareness
     → = KEYSTONE hub for feeling integration

  ② ACC (Anterior Cingulate Cortex) — "conflict + salience monitor"
     → 🟢 Botvinick 2004: conflict monitoring
     → 🟢 Shackman 2011: pain, negative affect, cognitive control overlap
     → Detects: CONFLICT between signals (schema mismatch)
     → Detects: SALIENCE (something requires attention)
     → = "Alarm system" — flags when integrated state is NOT coherent
     → = Vague detection mechanism

  ③ VMPFC (Ventromedial Prefrontal Cortex) — "value integrator"
     → 🟢 Damasio 1994: somatic marker integration
     → 🟢 Bechara 1994: Iowa Gambling Task → VMPFC essential for gut feeling
     → Integrates: body state + learned values + context
     → Produces: approach/avoid valuation
     → = "Should or shouldn't?" assessment

  HOW THE 3 HUBS WORK TOGETHER:

    ┌──────────────────────────────────────────────────────────────┐
    │  Body signals (1000s) ──→ THALAMUS (relay + filter)         │
    │                              │                               │
    │                    ┌─────────┼─────────┐                    │
    │                    ↓         ↓         ↓                    │
    │              INSULA      ACC       VMPFC                    │
    │           "how I feel" "conflict?" "value?"                 │
    │                    │         │         │                    │
    │                    └─────────┼─────────┘                    │
    │                              ↓                               │
    │                    INTEGRATED STATE                          │
    │                  (unified body feeling)                      │
    │                              ↓                               │
    │                     PFC OBSERVATION                          │
    │                   (Feel-Observation+ begins)                 │
    └──────────────────────────────────────────────────────────────┘

    → 3 hubs communicate CONTINUOUSLY (reciprocal connections)
    → Insula provides RAW body state
    → ACC flags CONFLICTS + SALIENCE
    → VMPFC adds VALUE assessment
    → Together = UNIFIED feeling that PFC can observe

🟡 FRAMEWORK NOTE — self-signal interoception:

  Self-signal interoception = architectural KEYSTONE (Body-Input-Enumeration §4.9):
    → = The capacity to PREDICT body state → match actual → generate error signal
    → 🟢 Seth 2013 "interoceptive inference": brain PREDICTS body state;
      feeling = prediction-delta between expected and actual
    → = Insula does not just READ body — insula PREDICTS body
    → = Feeling = delta between "I expect to feel X" and "I actually feel Y"
    → Explains WHY meditators "feel more" — they predict better →
      detect smaller deltas → finer-grained feelings
```

### §2.3 Integration Mechanism: From Many to One

```
🟡 FRAMEWORK SYNTHESIS — HOW integration produces unified feeling:

  STEP 1 — PARALLEL FIRING:
    → All body systems fire SIMULTANEOUSLY
    → Each system → its own signals → converge at integration hubs
    → NOT sequential — PARALLEL (all at once)
    → Timing: continuous, not discrete events

  STEP 2 — TEMPORAL BINDING:
    → Signals arriving within ~30-50ms window = BOUND TOGETHER
    → 🟢 Binding problem (Treisman 1996, Engel 2001)
    → Gamma oscillation (~40Hz) synchronizes: "these signals belong together"
    → = Same mechanism as chunk binding
    → Signals from DIFFERENT modalities → bound into ONE experience

  STEP 3 — PREDICTIVE MATCHING:
    → Insula has a PREDICTION of current body state (Seth 2013)
    → Actual signals COMPARED to prediction
    → Match → "as expected" → low salience (no feeling shift)
    → Mismatch → "something different" → HIGH salience → feeling shifts
    → = Feeling reflects CHANGE more than absolute state
    → = Why you "get used to" things — body adapts, prediction matches, feeling fades

  STEP 4 — CONFLICT DETECTION:
    → ACC monitors for INCONSISTENCY between signals
    → All signals coherent → smooth state → "fine" feeling
    → Some signals conflict → ACC flags → "something's off" feeling

  STEP 5 — VALUE TAGGING:
    → VMPFC adds approach/avoid valuation
    → Integrated state + value → DIRECTIONAL feeling
    → "I feel X" + "X is good/bad" → approach or avoid tendency
    → = Body vote on current state (body-first, value-laden)

  → RESULT: 1000s signals → 1 unified, valued, directional body state
  → THAT IS the "unified feeling" — ready for PFC to observe.

⭐ KEY INSIGHT — why complex feelings feel UNIFIED, not like a list:

  Example: 9+ channels fire simultaneously (Feeling.md §1.3):
    L0 safe, L1 relaxed, novelty-reward (evaluative), meaning (evaluative),
    agent-satisfied, mirror positive, imagine-final positive,
    valence positive, schema active

  Integration mechanism:
    → 9 channels fire simultaneously
    → Temporal binding: all within ~30-50ms → BOUND
    → Insula integrates → "body state: VERY positive, multi-domain"
    → ACC: no conflict → smooth
    → VMPFC: high value → strong approach
    → = 1 unified feeling
    → PFC compresses it to a few words.
    → Body experience = 9 dimensions.
    → = Lossy compression at Feel-Labeling (Feeling.md §2.7)
```

## §3 — Feeling Operational Flow (8 Steps)

> **This is the "Processing System"** — operational flow from body signal to behavior.
> = Discovery Cycle applied specifically to FEELING.
> = The missing file the framework did not have.

### §3.0 Overview

```
⭐ 8-STEP FEELING OPERATIONAL FLOW:

  ┌────────────────────────────────────────────────────────────────┐
  │                                                                │
  │  ① BODY SIGNALS FIRE                                           │
  │     (body-base + mirror + schema + imagine-final + valence...) │
  │                              ↓                                 │
  │  ② VTA DETECTS DELTA                                           │
  │     (dopamine = salience ALERT — NOT reward)                   │
  │                              ↓                                 │
  │  ③ SIGNAL INTEGRATION                                          │
  │     (insula + ACC + VMPFC aggregate → unified state)           │
  │                              ↓                                 │
  │  ④ PFC OBSERVATION THRESHOLD                                   │
  │     (signal strong enough → conscious awareness                │
  │      or not → background processing)                          │
  │                              ↓                                 │
  │  ⑤ CHUNK PATTERN-MATCH                                         │
  │     (activated chunks → context for feeling: what IS this?)   │
  │                              ↓                                 │
  │  ⑥ LABEL SEARCH                                                │
  │     (label found → planning enabled /                          │
  │      not found → vague feeling → dwell or dismiss)            │
  │                              ↓                                 │
  │  ⑦ BODY VOTE ON INTERPRETATION                                 │
  │     (smooth = "correct" / resist = "not right" / empty)       │
  │                              ↓                                 │
  │  ⑧ ACTION OR CONTINUED PROCESSING                              │
  │     (labeled → plan + act / vague → dwell + iterate)          │
  │     └──────→ new body state → back to ① (continuous loop)    │
  │                                                                │
  └────────────────────────────────────────────────────────────────┘

  TIMING: Steps ① → ④ = ~150-200ms (automatic, mostly unconscious)
          Steps ⑤ → ⑧ = variable (milliseconds to hours, depending on complexity)

  LOOP: Step ⑧ → Step ① = continuous. The Feeling Loop never stops.
```

### §3.1 Step ① — Body Signals Fire

```
TRIGGER: Any change in body or environment.

  SOURCES (multi-channel, simultaneous):
    → L0 Alive: threat detection, safety monitoring
    → L1 Body-inputs: interoceptive, exteroceptive, proprioceptive
    → Novelty (evaluative): VTA habituation-based delta detection
    → Meaning (evaluative): schema fire, status, mastery
    → Mirror: agent observation → own body resonance
    → Schema: compiled patterns fire on trigger
    → Imagine-Final: preview outcomes → preview body states
    → Valence: compiled evaluations fire
    → Cognitive prediction: PFC expectations → body anticipation

  KEY PROPERTIES:
    → PARALLEL: all channels fire SIMULTANEOUSLY, not sequentially
    → CONTINUOUS: 24/7, including sleep (reduced but not off)
    → AUTOMATIC: body fires without PFC permission
    → MULTI-SCALE: milliseconds (reflex) → seconds (acute) →
      hours (mood) → years (trait)

  → = Feel-RawSignals in the Feeling.md 7-layer model
  → = INPUT for the entire feeling pipeline
```

### §3.2 Step ② — VTA Detects Delta

```
🟡 VTA = "seismograph" — detects CHANGE, not absolute state.

  MECHANISM:
    → Body fires stably → VTA habituates → IGNORES (no signal)
    → Body fires DIFFERENTLY → VTA detects DELTA → dopamine release
    → Dopamine = "SOMETHING CHANGED — pay attention!"

  ⚠️ CRITICAL — DOPAMINE ≠ REWARD:
    → Dopamine = SALIENCE ALERT ("the doorbell rang")
    → = "Something changed, pay attention!" — motivates SEARCH
    → ≠ "Something valuable" — that is opioid (separate step, see §4)
    → 🟢 Berridge & Robinson 1998: wanting (dopamine) ≠ liking (opioid)

  PREDICTION-DELTA TYPES:
    → External delta: environment changed (new stimulus arrived)
    → Internal delta: body state changed (hunger onset, fatigue shift)
    → Cognitive delta: schema mismatch detected (prediction-delta)
    → Social delta: agent behavior changed (mirror circuit shift)

  DRD4 RECEPTOR FILTER:
    → 4R variant (sensitive): small delta → noticed → fine-grained attention
    → 7R variant (less sensitive): only large delta → noticed → coarser attention
    → = Per-person difference in feeling sensitivity
    → = Same event → different people → different feeling intensity

  → prediction-delta → dopamine → §3 signal integration BOOSTED
  → VTA no delta → no dopamine → signals integrate at baseline level
```

### §3.3 Step ③ — Signal Integration

```
= The §2 mechanism APPLIED as a step in the operational flow.

  WHAT HAPPENS:
    → All current signals (from Step ①) converge at integration hubs
    → Insula: "HOW body feels right now" (interoceptive map)
    → ACC: "IS THERE conflict between signals?" (conflict monitor)
    → VMPFC: "IS THIS good or bad?" (value assessment)
    → = Parallel processing at 3 hubs → unified integrated state

  IF VTA FIRED (Step ②):
    → Dopamine BOOSTS integration → signals stronger → feeling more salient
    → PFC receives: "pay attention to this state"
    → = Amplification, not new information

  IF VTA DID NOT FIRE:
    → Integration happens at BASELINE level
    → Most signals → sub-threshold → background hum
    → Only strong signals break through → "ambient" feeling
    → = Why even during "normal" moments you still HAVE feeling — just faint

  INTEGRATION RESULT = 1 unified body state:
    → Valence: positive ↔ negative (approach ↔ avoid)
    → Arousal: high ↔ low (activated ↔ calm)
    → Coherence: smooth ↔ conflicted (ACC peaceful ↔ flagging)
    → Specificity: localized ↔ diffuse (clear source ↔ vague)

  → Unified state READY for PFC observation (Step ④)
```

### §3.4 Step ④ — PFC Observation Threshold

```
⭐ TRANSITION POINT: unconscious → conscious.

  = Feeling.md Feel-Consciousification → Feel-Observation transition.
  = When the integrated state is STRONG ENOUGH for PFC to notice.

  THRESHOLD DETERMINANTS:
    → Signal strength: strong signals → cross threshold easily
    → VTA boost: dopamine amplification → lowers effective threshold
    → PFC attention direction: top-down query → lowers threshold for target
    → Contrast: sudden CHANGE → crosses threshold (delta detection)
    → Current PFC load: busy PFC → HIGHER threshold → more signals missed

  2 OUTCOMES:

    ⬆️ ABOVE THRESHOLD → Conscious feeling:
       → PFC notices: "something is happening in me"
       → Feeling becomes available for Feel-Observation–Feel-Explanation processing
       → = Explicit feeling — person CAN report if asked
       → Proceeds to Step ⑤

    ⬇️ BELOW THRESHOLD → Background processing:
       → Body continues computing → influences behavior ANYWAY
       → 🟢 Unconscious priming (Bargh 1996): below-threshold signals
         still affect choices, preferences, and judgments
       → Person CANNOT report → but behavior SHOWS the influence
       → = "I don't know why I chose this" — body knew, PFC didn't notice
       → = Stays in ①→②→③→④ loop, may cross threshold later

  ⚠️ TRAINING EFFECT:
    → Meditation: lowers observation threshold → notice more signals
      🟢 Farb 2013: meditators have higher interoceptive accuracy
    → Alexithymia: raised threshold → miss most signals
      🟢 Taylor 2000: ~10% population spectrum
    → Expert domain: domain-specific threshold lowered → "sixth sense"
      🟢 Klein 1998: expert firefighters detect danger before articulating
    → = Threshold is NOT fixed — trainable, per-domain, per-person
```

### §3.5 Step ⑤ — Chunk Pattern-Match

```
⭐ PFC "SEES" FEELING → SEARCHES FOR MATCHING CHUNKS:

  When PFC observes feeling (Step ④ above threshold) → automatic:
    → Current body state → spreading activation
    → Activation propagates to MATCHING chunks
    → Chunks that MATCH current state → fire stronger
    → = "Context search": which past experiences MATCH this feeling?

  MECHANISM:
    → Body state = activation pattern
    → Pattern activates chunks with SIMILAR body-state tags
    → Emotional state match = probability modifier
       (🟢 Bower 1981 mood-congruent memory)
    → = "Feeling afraid → fear-compiled chunks fire → danger everywhere"
    → = "Feeling joyful → reward-compiled chunks fire → opportunities everywhere"

  PATTERN-MATCH OUTCOMES:

    STRONG MATCH (1-2 chunks fire clearly):
      → PFC gets CONTEXT: "this feeling is LIKE my experience X"
      → = Clear feeling → proceed to label (Step ⑥)
      → = Gradient position ①-② (Body Signal / Named Emotion)

    MODERATE MATCH (few chunks, partial):
      → PFC gets APPROXIMATE context: "sort of like X but not quite"
      → = Moderate clarity → attempt label (Step ⑥) or search more
      → = Gradient position ③ (Gut Feeling)

    WEAK MATCH (many chunks, scattered):
      → Multi-Weak-Signal-Convergence (§5.3)
      → PFC detects "something" but CANNOT isolate specific source
      → = Convergence Zone (§5.4): intersection of many paths,
          no single node yet
      → = Vague feeling → struggle with label → dwell
      → = Gradient position ④-⑥ (Intuition → Pre-monition)

    NO MATCH (new territory):
      → Current state doesn't match ANY chunk
      → PFC: "completely unfamiliar feeling"
      → = First time → no reference → anxiety or curiosity (depending
          on threat level)
      → = Very rare for adults (most experiences HAVE some match)

  → Pattern-match GIVES feeling its CONTEXT and SPECIFICITY
  → Without chunks: feeling exists (body fires) but PFC has no context
  → = Why experts FEEL better — larger chunk library → richer matches
```

### §3.6 Step ⑥ — Label Search

```
🟡 LABEL = "TICKET IN" FOR THE LOGIC-PLANNING SYSTEM.

  = Feel-Labeling in the 7-layer model
  = Label = prerequisite for logic-planning enablement
  = Label-As-Handle: label = retrieval path, NOT content

  MECHANISM:
    → PFC has feeling + chunk context (from Step ⑤)
    → PFC searches for LABEL that matches this experience
    → = Verbal search: "this feeling IS ___"

  LABEL 3-TIER (Logic-Planning §2):
    → Individual: "that particular sensation" — only I understand it; fidelity HIGHEST
    → Group: "like that time before" — group understands; fidelity MEDIUM
    → Global: "anxiety" — everyone understands; fidelity LOWEST
    → Paradox: more GLOBAL label → easier to communicate → more LOSSY

  SEARCH OUTCOMES:

    LABEL FOUND:
      → Feeling becomes NAMEABLE → logic-planning system CAN access it
      → "I am sad" → PFC can now: plan, reason, communicate, compare
      → = Label = ticket into the planning system
      → = Chunk shifts from body-only → body + logic (both pathways available)
      → Proceed to Step ⑦ (body vote on label accuracy)

    LABEL NOT FOUND:
      → Feeling remains VAGUE — PFC knows "something" but cannot name it
      → 2 responses:
        a) DWELL: PFC holds attention on feeling → more chunks activate
           → convergence zone sharpens → label MAY eventually emerge
           = Gendlin Focusing (1978): deliberately attend to felt sense
           = Somatic-Articulation-Loop applied to feeling
        b) DISMISS: PFC gives up → attention moves elsewhere
           → Feeling continues influencing behavior BELOW PFC awareness
           → = "Not nothing" — still affects choices, mood, reactions
      → Proceeds to Step ⑧ (continued processing for DWELL,
        or Step ① restart for DISMISS)

  ⚠️ LABEL ≠ ACCURACY:
    → Label CAN BE WRONG: "I'm angry at my partner" when actually = tired + hungry
    → Label is PFC's BEST GUESS, not body truth
    → = Feeling.md §2.7: Feel-Labeling fidelity 40-80%
    → = Step ⑦ (body vote) is WHERE accuracy gets checked
```

### §3.7 Step ⑦ — Body Vote on Interpretation

```
⭐ BODY VOTE = THE MECHANISM BY WHICH PFC ACCURACY IS CHECKED.

  = "Body vote" concept from Chunk-Analysis throughout
  = Body's response when PFC proposes a label or interpretation

  MECHANISM:
    → PFC proposes label: "I'm anxious"
    → Label activates ASSOCIATED body states (via chunk links)
    → Body SIMULATES: "if 'anxious' → what would body feel?"
    → Body's ACTUAL current state → COMPARED with simulated state
    → Comparison → body vote: MATCH or MISMATCH

  3 BODY VOTE OUTCOMES:

    ✅ SMOOTH (body vote: "yes, that's it"):
       → Label matches body state → no conflict → feeling COHERENT
       → Body relaxes slightly → "landing" sensation
       → Gendlin "body shift" (1978): when right word found → body releases
       → = "Yes, I am anxious" → body confirms → proceed to action
       → = Label = VALID interpretation (at this moment, for this person)

    ❌ RESIST (body vote: "not quite right"):
       → Label MISMATCHES body state → conflict → body TIGHTENS or REJECTS
       → "I'm angry at my partner" but body says "not exactly" → label wrong
       → PFC should: reconsider, try different label, go back to Steps ⑤-⑥
       → = Body PROTECTING truth against PFC confabulation
       → = Why therapy works: therapist helps find the label body ACCEPTS

    ⬜ EMPTY (body vote: "not sure"):
       → Body doesn't respond clearly either way
       → Label may be PARTIALLY right — or feeling too weak to compare
       → PFC in ambiguous territory → keep searching or accept uncertainty
       → = Most common in daily life — many feelings lack a clear body vote

  ⚠️ BODY VOTE CAN BE DISTORTED:
    → Trauma-compiled chunks: body votes based on PAST threat, not CURRENT reality
    → = Direction-At-Compile: body-state direction at compile → persists → vote BIASED
    → Chronic stress: elevated cortisol baseline → body votes "danger"
      even in safe contexts → anxiety disorders
    → Alexithymia: weak interoceptive connection → body vote MUTED
    → = Body vote quality = f(compile history + current state)
```

### §3.8 Step ⑧ — Action or Continued Processing

```
STEP ⑧ — CONVERGENCE TO ACTION (or iteration):

  PATH A — LABELED + BODY VOTE SMOOTH → ACTION:
    → Feeling identified → PFC can plan
    → Logic-planning system engaged
    → Plan → execute → new body state → back to Step ①
    → = "I'm sad because I'm sleep-deprived" → plan: sleep earlier → execute → feel better

  PATH C — UNLABELED + DWELL → CONTINUED PROCESSING:

    → = "Incubation" (Wallas 1926): unconscious processing → insight later

    → PFC gives up attending → but feeling does NOT DISAPPEAR

    → May resurface later (when trigger appears again)
    → = "I don't know why I keep snapping" — unlabeled feeling driving behavior
    → = Most dangerous path: no awareness → no regulation → accumulation

  ⚠️ ALL PATHS → BACK TO STEP ①:
    → Action → new environment/body state → new signals → new feeling
    → Re-interpretation → new body vote → new signals
    → Dwelling → new chunk activation → new feeling nuance
    → Dismissal → feeling continues → eventually retriggers
    → = LOOP IS CONTINUOUS. Feeling is NEVER "done."
    → = Feeling.md §3: Feeling Loop = core framework loop

⭐ OPERATIONAL FLOW × DISCOVERY CYCLE:

  8-step feeling flow = Discovery Cycle applied TO FEELING:

    Accumulate    = Step ① body signals (raw material)
    Vague Emerge  = Step ④ threshold + Step ⑤ weak match

  = Self-similar structure: macro cycle ↔ micro feeling flow = same pattern

## §4 — Reward Mechanism (Corrected, Unified, Definitive)

> **This is the DEFINITIVE statement** on reward in the feeling system.
> Replaces all scattered references elsewhere.
> v1.1 audit (2026-04-17): 8 files corrected for dopamine/opioid confusion.

### §4.1 The Fundamental Distinction

```
⭐ REWARD = 2-LEVEL MECHANISM, NOT 1 SIGNAL:

  LEVEL 1 — DOPAMINE ALERT (VTA):
    → "SOMETHING CHANGED — pay attention!"
    → Signal: salience, NOT value
    → Function: direct attention, motivate SEARCH
    → Neurochemical: dopamine (VTA → PFC via mesolimbic pathway)
    → Subjective: "huh? something!" — curiosity, alertness
    → 🟢 Berridge & Robinson 1998: WANTING system

  LEVEL 2 — OPIOID REWARD (body-base):
    → "Chunk MATCHES body-need → ACTUAL value confirmed"
    → Signal: actual reward, pleasure, satisfaction
    → Function: reinforce pattern, tag chunk as "valuable"
    → Neurochemical: endogenous opioids (endorphins, enkephalins)
    → Subjective: "pleasant," "satisfied," "yes, that's it"
    → 🟢 Berridge 2003, Fields 2007: LIKING system

  CORE ANALOGY:
    ┌───────────────────────────────────────────────────────────┐
    │  Dopamine = DOORBELL                                      │
    │    → "Someone's at the door!" — alerting, not evaluating  │
    │    → Doorbell ringing ≠ gift at the door                  │
    │                                                           │
    │  Opioid = THE GIFT BEHIND THE DOOR                        │
    │    → Open door → gift = reward. Open door → empty = no   │
    │      reward.                                              │
    │    → Gift = body-need match confirmed                     │
    │                                                           │
    │  Cortisol drop = RELIEF (consequence, not cause)          │
    │    → IF threat was pending → resolved → cortisol drops    │
    │    → Relief = BONUS on top of opioid, not reward itself   │
    │    → IF no threat → no cortisol drop                      │
    │    → = Einstein (no threat): opioid only, no relief bonus │
    └───────────────────────────────────────────────────────────┘

🟢 BERRIDGE EXPERIMENTS (definitive evidence):

  Rats with dopamine blocked:
    → STILL liked sugar (ate it when given, displayed "pleasant" facial expression)
    → Did NOT seek sugar (lost motivation to look for it)
    → = Dopamine = wanting. Block dopamine → lose wanting, KEEP liking.

  Rats with dopamine artificially stimulated:
    → SOUGHT MORE (approach behavior increased)
    → Were NOT more pleased (facial expression unchanged)
    → = Stimulate dopamine → increase wanting, does NOT increase liking.

  → Dopamine and opioid = 2 SEPARATE SYSTEMS.
  → Framework: doorbell (dopamine) ≠ gift (opioid). CORRECT.
```

### §4.2 Reward in the 8-Step Feeling Flow

```
🟡 REWARD MAPS ONTO THE 8-STEP FLOW (§3):

  Step ① Body fires → signals include: body-need states (pending needs)
  Step ② prediction-delta → dopamine = ALERT ("something might fill a need")
  Step ③ Integration → insula + VMPFC assess: "does this match a need?"
  Step ④ Threshold → PFC notices: "something potentially valuable"
  Step ⑤ Chunk match → chunks activate: "this pattern = X"

  ⭐ REWARD HAPPENS HERE (between Step ⑤ and ⑥):

    → PFC holds matched chunk
    → PFC sends chunk DOWN to body-base: "simulate → what would body feel?"
      (🟢 Jeannerod 1995: motor imagery uses motor cortex)
      (🟢 Kosslyn 1994: visual imagery uses visual cortex)
    → Body simulates:
      → Chunk MATCHES body-need → OPIOID releases → REAL REWARD
      → Chunk does NOT match → body "meh" → PFC discards

  Step ⑥ Label → IF reward: "pleasant!", "satisfying!", "eureka!"
  Step ⑦ Body vote → SMOOTH (reward confirmed) → reinforce pattern
  Step ⑧ Action → chunk tagged "valuable" → strengthened → baseline shift

  → REWARD = BETWEEN Step ⑤ and ⑥ — at body-base check point
  → NOT at Step ② (dopamine alert ≠ reward)
  → NOT at Step ⑥ (label ≠ reward; label is AFTER reward)
```

### §4.4 Eureka: Reward Varies by Context

    → Deeper schemas = more body-need match = more intense

### §5.2 What Determines Position on the Gradient

```
🟡 3 VARIABLES DETERMINE POSITION ON THE GRADIENT:

    → No label → VAGUER (stuck at Feel-Observation, cannot articulate)

    Clear feeling = FEW signals × STRONG × LABELED

    Vague intuition = MANY signals × WEAK × UNLABELED

  Feeling quality = f(chunk_library_size × compile_context × current_state)

  ① CHUNK LIBRARY SIZE (in relevant domain):
     → Large library → many reference points → accurate matches
     → Small library → few references → unreliable matches
     → = Expert vs beginner difference

  ② COMPILE CONTEXT (HOW chunks were compiled):
     → Compiled under curiosity/safety → opioid-tagged → healthy feeling

       a) Emotional state at compile (calm / stressed / terrified)

     → Fatigue → PFC threshold raised → quality drops

  = Feeling quality NOT innate → NOT "some people naturally feel better"
  = Feeling quality = developmental product of chunk compilation history
  = CHANGEABLE through new experience, therapy, training

### §6.2 Expert Feeling vs Beginner Feeling

```
🟡 EXPERT vs BEGINNER — SAME MECHANISM, DIFFERENT LIBRARY:

  EXPERT (10,000+ hours in domain):
    → Massive domain-specific chunk library
    → Many compiled patterns → strong probability links
    → Convergence zones resolve QUICKLY (many reference points)
    → "Gut feeling" = FAST chunk match → RELIABLE intuition
    → 🟢 Klein 1998: expert firefighters make decisions via
      pattern recognition, NOT deliberate analysis
    → 🟢 Chase & Simon 1973: chess masters = ~50,000 patterns compiled

  Expert feeling in action:
    → Firefighter enters building → "something's wrong" → EVACUATE
    → Body: multiple weak cues (heat pattern, sound, air flow...)
    → Convergence zone: resolves FAST → "this = flashover pattern"
    → Feeling: CLEAR and URGENT → immediate action
    → = Multi-Weak-Signal-Convergence → convergence FAST because library HUGE

  BEGINNER (few hours in domain):
    → Small chunk library
    → Few patterns → weak links → slow/unreliable matching
    → Convergence zones PERSIST (not enough references to resolve)
    → "Gut feeling" = more noise than signal
    → = Same Multi-Weak-Signal-Convergence mechanism — small library → unreliable convergence

  Beginner feeling in action:
    → Student enters lab → "something's wrong" → CANNOT identify what
    → Body: same weak cues as expert receives
    → Convergence zone: STAYS VAGUE → no resolution
    → Feeling: VAGUE and UNACTIONABLE → freeze or ignore
    → = Multi-Weak-Signal-Convergence → convergence FAILS because library TOO SMALL

⭐ IMPLICATION FOR TRUSTING FEELING:

    WHEN TO TRUST GUT FEELING:
      → In domain with large chunk library → HIGH trust
      → Expert's "something's off" = statistically reliable
      → 10,000+ hours of Domain-Checked experience → body knows patterns
        PFC cannot yet articulate

    WHEN TO BE SKEPTICAL OF GUT FEELING:
      → In unfamiliar domain → LOW trust (small library = noise)
      → After trauma in domain → DISTORTED (cortisol-tagged chunks bias)
      → Under strong emotion → BIASED (mood-congruent activation)
      → = Same mechanism, different reliability based on library quality

    → "Trust your gut" = GOOD advice for experts in their domain
    → "Trust your gut" = DANGEROUS advice for beginners or traumatized
```

### §6.4 Trauma and Feeling Distortion

```
🟡 HOW TRAUMA DISTORTS FEELING (Direction-At-Compile + competitive re-linking):

  MECHANISM:
    → Traumatic event → chunks compile under EXTREME emotional state
    → Emotional weight at compile → EXTRA STRONG links
    → 1 trauma event CAN create stronger links than 100 neutral repetitions
    → These chunks = cortisol-tagged (Precondition-5 violated from inception)
    → = Large "trigger surface": many cues → fire trauma chunk

  4 FEELING DISTORTION PATTERNS:

    ① FALSE POSITIVE (over-detection):
       → Trauma chunk fires → body state = "DANGER"
       → Current context = SAFE but body says "DANGER"
       → Feeling: anxiety, fear, freeze — WITHOUT current threat
       → = PTSD core mechanism: trauma chunk fires inappropriately
       → = Large trigger surface: many cues activate it

    ② NUMBING (under-detection):
       → Chronic threat → body DAMPENS signals to survive
       → PFC threshold RAISED → fewer feelings cross into awareness
       → = "I don't feel anything" — signals exist but below threshold
       → = Alexithymia-like but ACQUIRED (not developmental)

    ③ MISLABELING (wrong interpretation):
       → Body signal correct (actual need) → but trauma chunk HIJACKS label
       → Label comes from trauma pattern, not current reality
       → "I'm angry at my partner" when actually = trauma activation + fatigue
       → = Feel-Labeling/Feel-Explanation distortion driven by chunk bias

    ④ BODY-STATE DIRECTION LOCKED (Direction-At-Compile):
       → Chunks compiled at extreme → body-state direction PERSISTS
       → Future activation → body goes TO THAT DIRECTION
       → Even when context has changed → old direction fires
       → = "I always feel unsafe" — regardless of actual current safety

  RECOVERY MECHANISM (competitive re-linking):
    → NEW experiences compile NEW chunks with NEW body-state direction
    → New chunks COMPETE with old trauma chunks for probability share
    → Over time: new links GROW → old links RELATIVELY weaker
    → But: old links NEVER deleted → stress/fatigue can REACTIVATE them
    → = "Relapse when exhausted" = PFC weakened → old pathway temporarily wins

    3 UPDATE PATHS:
      a) Re-associate: new context → old chunk + new link → modified response
      b) Novelty override: very strong new experience → overpowers old link
      c) Therapy: reconsolidation window (🟢 Nader 2000) → modify during labile state
    → = Feeling quality CAN improve — but requires active intervention
```
