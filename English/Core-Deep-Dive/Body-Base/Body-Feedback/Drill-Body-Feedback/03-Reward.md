### §1.2 — Reward definition reframe

🟡 Framework commitment (consolidating from PFC-Analysis §8.3 + Why-Body-Knows §3 + Cortisol-Baseline §3.5):

> **Reward is NOT a single signal.** It is a **2-level mechanism** with a critical distinction:
>
> **Level 1 — Dopamine ALERT (VTA)**: "SOMETHING HAS CHANGED — pay attention!" — signals salience, NOT value.
> **Level 2 — Opioid REWARD (body-base)**: "chunk MATCHES body-need → genuine value." — the actual reward.
>
> VTA dopamine = **doorbell** (signals something is at the door). Opioid = **the gift behind the door** (the actual reward).

**Working hypothesis validation**:

The hypothesis stated: "VTA fires dopamine when chunks link + schema dissonance is resolved → reward."

**Framework agrees with one refinement** ✅:
- ✅ VTA fires on chunk linking (detects delta / coherence change)
- ✅ Reward requires schema pending (needs body-need gap)
- ✅ Reward requires resolution (chunk fills the gap)
- 🟡 Refinement: **VTA firing ≠ reward itself**. VTA = alert signal. Reward = body-base opioid release AFTER body confirms the fit.

**This distinction is foundational** because it explains:
- Why scrolling a phone can fire dopamine but feel empty (VTA fires, body doesn't confirm → no opioid)
- Berridge's wanting vs liking research 🟢
- Why reward is per-person (body-base needs differ between individuals)
- The 5-preconditions structure (§3)

### §1.3 — Berridge wanting vs liking (research validation)

🟢 **Established research** (Berridge & Robinson 1998, 2003):

> Dopamine = **WANTING** (motivational seeking). NOT **LIKING** (pleasure when received).
> Liking = primarily the opioid system.

**Classic experiments**:
- Rats with dopamine blocked: STILL liked sugar (ate it when given, showed facial pleasure expressions) but did NOT seek it
- Rats with dopamine stimulated: SOUGHT non-stop but did NOT experience more pleasure

**Framework interpretation** 🟡:

> Dopamine (VTA) = "this might be valuable — go check" → motivates SEARCH
> Opioid (body-base) = "this IS valuable" → reward for OUTCOME

**Application to compulsive-use behavior**:
- Slot machines, social media, scroll-based apps: dopamine-heavy design (constant alerts)
- Low opioid (most events don't actually fill any body-need)
- Result: **"wanting without liking"** → compulsive seeking + empty feeling afterward
- Recognizable pattern: "I scroll because I can't stop, but I'm left feeling worse."

**Working hypothesis refined**:
- "VTA fires" = correct — signals chunk linking detection
- "Reward is generated" = correct mechanism IF body-need is actually filled
- **Not every VTA firing → reward** — depends on body-base confirmation at Step 5

## §2 — VTA Detection Mechanism (7-Step Loop Consolidated)

### §2.1 — Source: PFC-Analysis §8.3

🟡 Framework formalization in `Imagination/PFC-Analysis.md` §8.3 "VTA Detection Mechanism — Delta Detection + PFC Spotlight Loop." This section consolidates the 7-step loop and validates the working hypothesis against each step.

### §2.2 — 7-Step Loop Overview

```
┌────────────────────────────────────────────────────────────────┐
│                                                                │
│  Step 1: UNCONSCIOUS — neurons fire 24/7, self-organizing      │
│          patterns form via Hebbian synchronization             │
│                              ↓                                 │
│  Step 2: VTA DETECTS DELTA — habituation-based                 │
│          (prediction-delta, NOT "prediction error")            │
│          Stable old schema → VTA habituated → silent           │
│          New pattern → "CHANGE DETECTED!" → VTA fires → dop.  │
│                              ↓                                 │
│  Step 3: RECEPTOR FILTER — DRD4 determines threshold           │
│          4R receptor: sensitive → small delta detected         │
│          7R receptor: less sensitive → only large delta        │
│                              ↓                                 │
│  Step 4: PFC SPOTLIGHT — top-down boost to target region       │
│          PFC: "something happening there" → NE + ACh boost     │
│          Target region fires more clearly → "spotlighted"      │
│                              ↓                                 │
│  Step 5: BODY-BASE CHECK ⭐ — does chunk fit body-need?        │
│          PFC sends chunk to body: "simulate → how does it feel?"│
│          Body MATCHES → OPIOID release → REAL REWARD           │
│          Body "meh" → PFC discards                             │
│                              ↓                                 │
│  Step 6: REINFORCE + LOOP — pattern stabilizes                 │
│          Successful chunk → strengthen pathways                │
│          VTA habituates to new pattern → "new baseline"        │
│          Unconscious continues trying → new delta → Step 2     │
│                              ↓                                 │
│  Step 7: CLEAR DOPAMINE — COMT enzyme breaks down              │
│          Signal fades → ready for next detection               │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

### §2.3 — Step 1: Unconscious Self-Organization

**Mechanism**:
- 🟢 86 billion neurons fire continuously (24/7, even during sleep)
- Each neuron connects to ~7,000 neighbors → signals propagate
- Neurons synchronize (fire together) via Hebbian rule
- Self-organizing process — no central controller
- Patterns emerge = "chunks" (groups of neurons firing synchronously)

**Critical context**: Cortisol baseline amplifies this (Cortisol-Baseline §1):
- Low cortisol → fewer pattern attempts → slow adaptation
- Moderate cortisol → active pattern-trying → optimal learning
- High cortisol → excessive firing → noise + overload

**Connection to working hypothesis**: "Chunks linking together" happens continuously in the unconscious. The question is WHEN VTA detects the linking.

### §2.4 — Step 2: Prediction-Delta Detection (Habituation-Based)

🟡 **Critical framework claim** (PFC-Analysis §8.3 — diverges from Schultz "prediction error"):

> VTA = **"seismograph"** — detects prediction-delta (change from habituated baseline), not absolute intensity, not "prediction error" computation.
>
> Mechanism: HABITUATION.
> - Region X fires stably → VTA habituates → ignores it
> - Region X fires differently → "SOMETHING CHANGED!" → VTA fires → dopamine
> - Region X stabilizes at new level → VTA habituates again → back to ignoring
> - Region Y changes → "DIFFERENT CHANGE!" → VTA fires again

**Key insight**: The framework's habituation model (prediction-delta) is **simpler** than Schultz 1997 "prediction error" theory yet produces the same observable result.

**Schultz model** (standard): Brain predicts outcome → compares to actual → dopamine fires on the error.
**Framework model**: Brain habituates to stable patterns → new activity differs → fires on the difference.

Both give the same prediction: dopamine fires on novelty, silence on routine. But the framework model is **computationally cheaper** (no prediction computation needed, just habituation detection).

🟢 **Research support for habituation mechanism**: established across multiple species and systems.

### §2.5 — Step 3: DRD4 Receptor Filter

**Mechanism** (PFC-Analysis §8.3):
- Dopamine released by VTA travels via axons to PFC
- Binds DRD4 receptors on PFC neurons
- DRD4 variants affect detection threshold:
  - **4R (sensitive)**: Small delta → enough dopamine → PFC notices → fine-grained attention
  - **7R (less sensitive)**: Only large delta → enough dopamine → PFC notices → coarse-grained attention

**Framework characterization** (PFC-Analysis §8.1):
- 7R = "big chunks / big delta" — high threshold, misses subtleties, needs strong novelty to notice
- 4R = "small chunks / small delta" — low threshold, notices everything, detail-oriented

**Implication for reward**: Same pattern → different DRD4 variant → different PFC detection → different reward experience. Hardware-level source of individual difference.

### §2.6 — Step 4: PFC Spotlight (Top-Down Boost)

**Mechanism**:
- PFC detects dopamine → knows "something in region X is happening"
- PFC sends top-down signal: NE + ACh boost to target region
- Target region neurons fire MORE strongly (amplification)
- "Spotlight effect" — region becomes clearer to PFC

**Key**: PFC doesn't "tell" the unconscious what to do. PFC simply "increases volume" of that region, and neurons there follow their existing connections.

**Spreading activation**: Boosted region → neighbors fire → related chunks activate → associative cascade (Collins & Loftus 1975 🟢).

### §2.7 — Step 5: Body-Base Check ⭐ (Critical Step)

🟡 **This is the key step for reward** (PFC-Analysis §8.3 Step 5):

> "PFC receives chunk → integrates into workspace → PFC sends chunk DOWN to body-base: 'simulate → how does the body feel?'
> Body-base simulates:
> → Chunk MATCHES body-need → **opioid release → REAL REWARD**
> → Chunk does NOT match → body 'meh' → PFC discards
>
> → Real reward = body-base confirmation, NOT dopamine
> → Dopamine = signal 'something changed' (doorbell)
> → Opioid = reward 'genuine value confirmed' (the gift)"

**Unpacking**:

**Substep 5a — PFC sends chunk to body for simulation**:
- PFC holds chunk in working memory
- Activates associated body states via cross-pathway firing
- 🟢 Jeannerod 1995: motor imagery uses motor cortex the same as real motion
- 🟢 Kosslyn 1994: visual imagery uses visual cortex the same as real sight
- Body can "feel" a simulated action at ~20–60% fidelity

**Substep 5b — Body-base checks body-need match**:
- Current body state has active needs (hunger, social connection, exploration, safety, etc.)
- Simulated chunk outcome → does it fit ANY active need?
- Match = body-need decremented → opioid release
- Mismatch = body state unchanged → no opioid

**Substep 5c — Opioid release (the REAL reward)**:
- Endogenous opioids (endorphins, enkephalins, dynorphins)
- 🟢 Berridge 2003, Fields 2007 — the liking system
- Body experiences the "pleasant" signal
- This is what is subjectively experienced as pleasurable

**Substep 5d — PFC receives opioid feedback**:
- PFC notes: this chunk produced body reward
- Tags chunk as "valuable, keep"
- Reinforces chunk in memory
- **This is where chunk tags are assigned** (opioid vs cortisol association — Cortisol-Baseline §3.5)

🟡 **Evaluative/Direct-State Note (v1.1)**: Step 5 opioid check = primarily the **EVALUATIVE** pathway.
Direct-State Reward (touch, exercise, warmth) bypasses cortical evaluation → different, shorter pathway.
Full Evaluative/Direct-State dimension detail: Reward-Signal-Architecture.md §1–§3.

**Working hypothesis validated with addition**:
- Hypothesis: "Chunks link → VTA fires → reward"
- Framework: "Chunks link → VTA fires (alert) → PFC check → body simulates → body-need matched → **opioid (actual reward)**"
- The hypothesis's "reward" = framework's "opioid response after body-base confirmation"

### §2.8 — Step 6: Reinforce + Loop

**Mechanism**:
- Successful chunk (got opioid) → pathway strengthened via Hebbian LTP + late LTP
- Schema updates → more stable pattern
- VTA habituates to new stable pattern → ignores it → baseline shifted
- **Unconscious tries next pattern** → new delta → Step 2 → loop continues

**Key insight**: Each successful loop **raises the baseline**. Next reward requires new novelty. This is the **hedonic treadmill** at the neural level (Brickman 1978 🟢).

### §2.9 — Step 7: Clear Dopamine

**Mechanism**:
- COMT enzyme breaks down dopamine in the synaptic cleft (PFC)
- Signal fades → synapse ready for next signal
- COMT variants affect clearance speed:
  - Val/Val (fast COMT) → rapid clear → short-lived dopamine → "improviser" cognitive mode
  - Met/Met (slow COMT) → slow clear → persistent dopamine → "specialist" cognitive mode

**Application**: Personal hardware (genetics) affects reward signal duration, and therefore reward experience intensity.

### §2.10 — Critical takeaway: Reward is body-base opioid, not VTA dopamine

🟡 **Framework core commitment**:

> The subjective "reward" experience = Step 5 opioid release.
> VTA dopamine = Steps 2–4: alert + attention direction.
> These are **different signals, different systems, different neurochemicals, different subjective experiences**.

**Why the confusion is common**:
- Research often measures dopamine → calls it "reward signal" → conflates with subjective reward
- Pop science: "dopamine hit" → everyone thinks dopamine = pleasure
- Framework: dopamine = alert, opioid = pleasure; they cooperate but are NOT the same signal

**Falsification test**: If dopamine = reward, then:
- Blocking dopamine → no reward → no liking → subject won't eat sugar
- Stimulating dopamine → more reward → more liking → subject enjoys more

**Actual findings** (Berridge):
- Blocking dopamine → subject STILL likes sugar (eats it when given, shows facial pleasure) but WON'T SEEK it
- Stimulating dopamine → subject SEEKS more but DOESN'T enjoy more

→ **Dopamine separated from liking. Liking is opioid. Framework vindicated.**

## §3 — Reward Preconditions Matrix (H10 Core)

### §3.1 — H10 Statement (refined v2)

🟡 **H10 — Body Signal Precondition Hypothesis v2**:

> Body signal (reward OR dissonance) = function of **5 preconditions** on the interface loop:
>
> 1. **Schema pending status** — is there a schema waiting to be filled / resolved / upgraded?
> 2. **Chunks base adequacy** — is there sufficient chunks substrate to decode the incoming pattern?
> 3. **prediction-delta threshold** — is the change large enough for VTA to detect?
> 4. **Goldilocks zone position** — does match ratio fall in the optimal zone (dynamic per person/context)?
> 5. **Chunk association tag** — what is the chunk tag direction (opioid vs cortisol)?
>
> **Claim**: Signal fires correctly (i.e., full reward) WHEN ALL 5 preconditions are met. Absence of ANY → signal absent or wrong direction.

**Falsifiability**: Each precondition can be manipulated independently. Failure modes are predictable per precondition violated.

### §3.2 — Precondition 1: Schema Pending Status

🟡 **Definition**:

> The body must have an **unresolved body-need** OR a **schema seeking upgrade** — an "opening" in the schema landscape where new information can fit.

**Mechanism**:
- Body-needs (Body-Base L1 + Body-Input-Enumeration §4): hunger, social connection, exploration, safety, etc.
- Schema-level needs: coherence gap, incomplete knowledge, unresolved problem
- "Pending" = active signal indicating the gap exists

**Detection by body**:
- Unconscious scans continuously (vs Imagine-Final expectations)
- When current state ≠ expected → signals "need"
- Signal persists until need is met

**When precondition MET** (schema pending present):
- Relevant chunks fire → body checks if they fit the pending need → opioid if match
- Example: hungry person eats → food chunks fit "hunger need" → opioid

**When precondition FAILED** (no schema pending):
- Chunks fire but there is no active "need" to match against
- VTA may still fire on delta, but body doesn't confirm value
- No opioid
- Example: a full person offered more food → no hunger need → no pleasure

**Research validation**:
- Satiety effects: same food is less pleasurable when full (Cabanac 1971 "alliesthesia" 🟢)
- Motivation and reward: seeking requires an active pending need state

**Working hypothesis match**: ✅ Exactly "schema waiting to be resolved" = schema pending status.

### §3.3 — Precondition 2: Chunks Base Adequacy

🟡 **Definition**:

> The body must have **sufficient chunks substrate** to decode the incoming pattern and recognize it as relevant to a pending need.

**Mechanism**:
- Chunks = compiled patterns from past experience (Chunk.md §2)
- New input → unconscious searches for matching chunks
- Match → recognition → can evaluate fit with need
- No match → "cannot decode" → uncertainty → mild cortisol (NOT opioid)

**When precondition MET** (chunks base sufficient):
- Input recognized via matching chunks
- Can be evaluated against active needs
- Reward possible if match found

**When precondition FAILED** (chunks base insufficient):
- Input cannot be decoded
- Body signals confusion or strangeness
- NO reward, even if the object is theoretically valuable
- Example: Van Gogh painting shown to an untrained viewer → chunks base insufficient → "I don't understand it" (§6 drill)

**Developmental implication** (P1 §5.5):
- Infants have minimal chunks → many things seem strange
- Adults have rich chunks → more things are decodable
- Domain experts have rich domain-specific chunks → more fine-grained evaluation possible

**Working hypothesis extension**: "Schema ready to be upgraded" = chunks base adequate FOR upgrade = this precondition.

### §3.4 — Precondition 3: Prediction-Delta Threshold

🟡 **Definition**:

> The change (delta) must be **large enough** to exceed VTA habituation threshold so that dopamine fires and PFC attention is triggered.

**Mechanism**:
- VTA habituated to stable patterns → ignores them
- Requires change exceeding the habituation threshold → fires
- Threshold affected by DRD4 variant (§2.5) + arousal + recent history

**When precondition MET** (delta sufficient):
- VTA fires → dopamine → PFC attention → downstream reward possible

**When precondition FAILED** (delta too small):
- VTA stays silent → no alert → PFC doesn't attend → no downstream reward even if other preconditions are met
- Example: routine daily events slip by without being registered as "rewarding" even when they are objectively positive

**Classic manifestation** (hedonic adaptation):
- First time eating chocolate: large delta → intense reward
- 100th time: pattern is stable → small delta → muted reward
- 1,000th time: fully habituated → no delta → no reward despite the same food

**Implication**: Reward requires novelty at some level. Pure repetition without variation loses reward potency.

### §3.5 — Precondition 4: Goldilocks Zone Position

🟡 **Definition**:

> Match ratio between the input pattern and existing chunks must fall in the **Goldilocks zone** (dynamic per person/context). Too familiar (>90%) → boredom. Too alien (<20%) → confusion. Goldilocks zone → optimal reward.

**Mechanism** (Why-Body-Knows §2):
- <20% match: too novel → cannot decode → mild cortisol → dissonance
- Goldilocks zone: recognizable + surprising → opioid + dopamine → **pleasurable**
- >90% match: too familiar → habituated → dopamine silent → boredom

**When precondition MET**:
- Input falls in Goldilocks zone
- Combined with Preconditions 1–3 → maximum reward

**When precondition FAILED**:
- **Too alien** (<20%): No decoding → confusion (Van Gogh to an untrained viewer)
- **Too familiar** (>90%): No novelty → boredom (same food 1,000 times)

**Research validation** 🟢:
- Zajonc 1968 mere exposure effect (familiarity → preference, but there is a limit)
- Berlyne 1960 optimal novelty (inverted-U with arousal)
- North & Hargreaves 1995 music preference × familiarity inverted-U

### §3.6 — Precondition 5: Chunk Association Tag (Novelty vs Threat)

🟡 **Definition** (Cortisol-Baseline §3.5 integration):

> Chunks being activated must have an **appropriate association tag** (opioid vs cortisol) for the direction of reward being sought. Threat-tagged chunks fire discomfort even when other preconditions are met. Opioid-tagged chunks fire sustainably.

**Mechanism**:
- Chunks compile under specific cortisol context (novelty vs threat)
- Tag persists through retrieval (Cortisol-Baseline §3.5)
- Future activation fires the tag association alongside the content

**When precondition MET**:
- Chunks opioid-tagged → activation is pleasant → sustainable reward
- Example: music learned during a curiosity-driven exploration → re-listening is pleasant

**When precondition FAILED**:
- Chunks cortisol-tagged → activation is uncomfortable → avoidance
- Example: math learned under threat → adult retrieval fires discomfort
- Or: chunks compiled in survival mode → functional but not enjoyable

**Genius Paradox relevance** (P2 §5.6):
- Newton / Tesla / Einstein had high baseline cortisol
- But their learning-domain chunks compiled in a curiosity state (threat source ≠ learning domain)
- → Learning-domain chunks became opioid-tagged
- → Reward accessible despite high baseline

🟡 **P5 extended (v1.1)**: P5 determines not just approach/avoid direction but reward QUALITY.
4-Pathway × P5 Tag Model: Hardware Fit → approach; Trust → moderate; Social → neutral; Threat → avoidance.
Full detail: Reward-Signal-Architecture.md §8.4.

### §3.7 — Preconditions Matrix Summary

| # | Precondition | Failure mode | Classic example |
|---|---|---|---|
| 1 | Schema pending | No need → no reward even when offered | Full person offered more food |
| 2 | Chunks base adequate | Cannot decode → confusion not reward | Van Gogh painting to untrained viewer |
| 3 | prediction-delta threshold | No alert → no attention | Routine events go unregistered |
| 4 | Goldilocks zone | Too alien OR too familiar → wrong direction | New music with no chunks (alien) or the same song 1,000 times (familiar) |
| 5 | Chunk tag | Content recognized but discomfort-tagged → avoidance | Math learned under sustained threat |

### §3.8 — All 5 Required Claim

🟡 **Strong claim**:

> Full reward (subjective pleasant experience) requires **ALL 5** preconditions to be met. Absence of ANY → signal reduced or in the wrong direction.

**Why the strong claim**:
- Each precondition addresses a different mechanism layer
- Missing any layer → corresponding failure mode
- No precondition is redundant

**Falsifiability**:
- Violate P1: full person, offer food → should not show reward → tested (alliesthesia research)
- Violate P2: show Van Gogh to untrained viewer → should show confusion not reward → observable
- Violate P3: boring routine → should show habituation not reward → observable
- Violate P4: extreme novelty or extreme familiarity → should NOT produce reward → Goldilocks research
- Violate P5: math-avoidant adult given "interesting" math problem → still feels aversion → clinical observation

**Framework prediction**: Cases where "should reward but doesn't" = one precondition missed. Cases where "shouldn't reward but does" = either mislabeling OR threshold effect (all 5 barely met).

## §4 — 7 Reward Cases Analyzed (C1.1-C1.7)

### §4.1 — Case C1.1: Teacher Calls Student's Name (Habituation vs Novelty)

**Original scenario**:
> "Focused on an assignment, the teacher suddenly calls your name — chunks link powerfully, activating a cascade of related chunks, VTA signals clearly, head looks up → startled; OR completely routine and habituated — no reward."

**Mechanism decomposition** 🟡:

#### §4.1.1 — Dual outcome analysis

The scenario identifies **2 outcomes** for the same stimulus:
- Outcome A: Startle / attention spike
- Outcome B: No reaction — habituated

Framework explains via precondition analysis:

**Outcome A (startle) — Preconditions state**:
- P1 Schema pending: focused on assignment → not specifically attending to name-calls
- P2 Chunks base: ✅ "teacher calling name = attention event" chunk is solid
- P3 prediction-delta: HIGH (unexpected name call while deeply focused elsewhere)
- P4 Goldilocks: within zone (expected teacher sounds, but not own name specifically)
- P5 Tag: neutral-positive (name calls not threatening in this context)

Result: VTA fires HARD → dopamine spike → attention interrupted → "startled."
BUT → body-base check at Step 5: name call doesn't fill any active body-need → **NO OPIOID → no subjective reward** despite VTA activity.

**→ VTA alert ≠ reward.** Being startled is "attention grabbed," not "pleased."

**Outcome B (habituated, no reaction) — Preconditions state**:
- P3 prediction-delta: LOW (habituated — being called in class is a stable pattern)
- VTA doesn't fire strongly → no interrupt → no reaction
- All other preconditions become irrelevant when P3 fails

Result: No attention shift, no surprise, routine continues.

#### §4.1.2 — Why this case is instructive

This case reveals the **2-level separation** between VTA alert and reward:
- VTA CAN fire without reward (Outcome A: spike but no opioid)
- Reward CANNOT fire without VTA alert (Outcome B: no alert → no downstream)

→ Dopamine is **necessary but not sufficient** for reward.

Framework validates: VTA fires (attention grabbed), but Step 5 body-base check finds no fit → no opioid.

### §4.2 — Case C1.2: Teacher Praises Public Solution (Social Schema Fill)

**Original scenario**:
> "Teacher asks if you can solve this problem. You say yes, solve it at the board, teacher praises you → pleasant (social recognition schema = natural desire, shaped by seeing peers praised, parent wanting public praise — teacher's praise activates many related chunks simultaneously, firing strongly)."

**Mechanism decomposition** 🟡:

#### §4.2.1 — Precondition analysis

**P1 Schema pending ✅**:
- Social recognition schema is clearly pending and active
- Observational learning: saw peers praised → desire formed
- Social approval schema compiled from: observing others + parental preference + cultural context
- **Need state**: "I want social recognition" — active and pending

**P2 Chunks base ✅**:
- Rich chunks about: "public praise" + "social approval" + "competence display" + "teacher's judgment"
- Can decode the situation fully
- Understands meaning at multiple levels (direct praise + social positioning + family pride)

**P3 prediction-delta ✅**:
- Surprise factor: praise (positive) exceeds baseline expectation
- Multiple simultaneous chunks firing ("many related chunks, firing strongly")
- Spreading activation cascade → large neural delta
- VTA fires strongly

**P4 Goldilocks ✅**:
- Match 50–70%: recognized situation but with positive novelty
- Familiar enough to decode ("public praise I understand")
- Novel enough to be salient (praise specifically for me, right now)

**P5 Chunk tag ✅**:
- "Praise" chunks opioid-tagged (compiled from observing pleasant outcomes)
- Social approval chunks positively tagged from childhood (parent's happy response)

**ALL 5 preconditions MET** → full reward mechanism engages:
- Step 5 body-base check: chunks fire "social schema need satisfied"
- Opioid releases STRONGLY
- Subjective experience: pleasurable

#### §4.2.2 — Multi-schema activation insight

The observation that praise activates "many related chunks simultaneously, firing strongly" is exactly **spreading activation cascade** (Collins & Loftus 1975 🟢).

**Cascade includes**:
- "Teacher praises" → "peers admire" → "parent would be proud" → "I am capable" → "bright future" → "safety"
- Multiple body-need schemas all "hit" in one event
- Each contributes an opioid signal
- **Summation**: experience is stronger than any single need fill would produce

**Framework prediction**: A single event activating a multi-schema cascade produces **compound reward** proportional to how many schemas are satisfied simultaneously.

This case is "hot" precisely because it fills **many needs at once**:
- Competence need
- Social belonging need
- Parental approval need (imagined)
- Future security signal
- Current status signal

**→ Multi-schema fill = compound reward** (drilled further in §7 spreading activation).

### §4.3 — Case C1.3: Monthly Salary + Lottery Jackpot (Pending Schema + Delta Extreme)

**Original scenario**:

**P2 Chunks base**: Strong — understands money, work, reward cycle
**P3 prediction-delta**: **LOW to moderate** — expected event, not surprising
**P4 Goldilocks**: HIGH match (routine pattern)

**P1 Schema pending**: Yes — resource need (always pending for most people)
**P2 Chunks base**: Strong — understands money, lottery, winning
**P3 prediction-delta**: **EXTREME** — unexpected massive event exceeding all predictions
**P4 Goldilocks**: within zone (understand lottery but have never experienced winning)

#### §4.3.3 — Caveat (hedonic adaptation + research finding)

🟢 **Brickman 1978** classic lottery winner study:
- Initial spike: massive happiness boost
- Adaptation: within 6–12 months, baseline returns to roughly pre-lottery levels
- **Why**: schema landscape adjusts → new baseline → reward habituates

**Framework mechanism**:
- Post-winning, resource-need chunks transform
- Some old needs disappear (security met)
- New needs emerge (what to do with wealth, avoiding loss, trust issues)
- **Schema portfolio shifts** — similar to hedonic trap (P2 §9)
- Each new pending need → new potential dissonance source

**Key insight**: Peak reward is **instantaneous maximum**. Sustained reward is different — requires ongoing schema fit, not one-time massive delta.

This explains: "Lottery jackpot is intensely pleasant" (peak) ≠ "lottery win produces lasting happiness" (sustained). Different mechanisms.

### §4.7 — Case C1.7: "The Longer the Effort, the Stronger the Reward" (Meta-Pattern)

**Key insight**: **Effort-proportional reward requires eventual resolution.** Struggle without resolution is trauma, not growth.

## §5 — Car Paradox 5-Scenario Matrix (C2.1-C2.5)

### §5.1 — Scenarios recap

Five scenarios with paradoxical outcomes:

- **C2.1**: Father buys the exact car wanted → INTENSELY PLEASANT
- **C2.2**: Father buys a motorbike → NOT HAPPY
- **C2.3**: Father buys an airplane → CONFUSED (never imagined it, can't use it)
- **C2.4**: Father buys a Van Gogh painting worth millions → NOT APPRECIATED
- **C2.5**: Father buys a vintage car (more expensive) → Son NOT HAPPY, father loves it

Key insight: same PERSON (father giving the gift) → different response per item. Same ITEM (expensive) → different response per person (father vs son).

**Framework explanation via H10 5 preconditions** — different scenarios violate different preconditions:

### §5.2 — Scenario C2.1: Ideal Car (All Preconditions Met)

**State**:
- **P1 Pending**: ✅ Schema "want a car" active — watching cars, longing
- **P2 Chunks base**: ✅ Rich — car chunks (models, colors, features, desirability)
- **P3 prediction-delta**: ✅ HIGH — surprise + expectation exceeded
- **P4 Goldilocks**: ✅ within zone — recognized (a car) + novel (this specific model)
- **P5 Tag**: ✅ Positive — car chunks opioid-tagged from admiration/play context

**All 5 preconditions met strongly** → **intensely pleasant**.

**Additional amplifiers**:
- Parental love signal (connection schema satisfied)
- Status signal (showing off the car to peers)
- Autonomy signal (can use car for desired activities)
- Multi-schema cascade activated

### §5.3 — Scenario C2.2: Motorbike (Wrong Category)

**State**:
- **P1 Pending**: ❌ No schema "want a motorbike" — the pending schema is specifically for "a car"
- **P2 Chunks base**: ✅ Has chunks about motorbikes (can decode what it is)
- **P3 prediction-delta**: Moderate — unexpected but wrong category
- **P4 Goldilocks**: N/A — wrong category, cannot be evaluated properly
- **P5 Tag**: Neutral

**P1 violated** → body-base check Step 5: chunk doesn't fit pending need ("I wanted a car, not a motorbike") → **NO OPIOID → no reward**.

**Subjective experience**: "Not happy" — may be polite acknowledgment, but no body-level reward. Possibly mild disappointment (pending unresolved + confusing new object).

**Framework insight**: Reward is **category-specific**. Even close substitutes (both motor vehicles) don't transfer, because the pending schema "want a car" is not "want a motor vehicle" — it is specific.

### §5.4 — Scenario C2.3: Airplane (Wrong Category + No Chunks)

**State**:
- **P1 Pending**: ❌ No "want a personal airplane" schema
- **P2 Chunks base**: ❌ Insufficient — has limited/no chunks about personal airplane ownership
- **P3 prediction-delta**: EXTREME — completely unexpected, strange, overwhelming
- **P4 Goldilocks**: ❌ <20% match — too alien
- **P5 Tag**: N/A

**Multiple preconditions violated**:
- P1 missing (no pending schema)
- P2 missing (can't decode the meaning)
- P4 missing (too alien to interpret)

**Subjective experience**: Confusion, not reward. Precise articulation:
> "Never imagined this, doesn't know how to fly a plane, can't use it for everyday outings."

Framework: Child thinks "it exists, it's mine(?), but what do I do with it?" → Step 5 body check finds no fit for any active body-need → NO OPIOID.

**Additional mild distress possible**: Cost schema may fire "this seems wasteful" or confusion about implications. Net result may even be slightly negative.

**Framework principle**: Reward requires **interpretability**. A gift must match the recipient's chunks base to be meaningful. "Too much" gift can miss reward entirely when chunks are insufficient.

### §5.5 — Scenario C2.4: Van Gogh Painting (Wrong Substrate Value)

**State**:
- **P1 Pending**: ❌ No "want a painting" schema (for a child typically)
- **P2 Chunks base**: ❌ No art appreciation chunks (cannot decode visual as "masterpiece")
- **P3 prediction-delta**: LOW — may not even register as special (looks like "just a painting")
- **P4 Goldilocks**: ❌ <20% match (no art chunks → cannot decode the "quality" signal art experts perceive)
- **P5 Tag**: Neutral at best

**All preconditions fail** despite the painting being "valuable" (worth millions).

**Subjective experience**: Precisely captured — "wavy lines, messy, don't understand it, not beautiful at all."

Framework: "Value" (domain: monetary) ≠ "reward" (body: chunks match). Receiving a Van Gogh painting → no body-level reward.

**The paradox fully explained**: Object has high domain value, zero body reward. Schema serves body-base (P1 §7) — without body-base match, no reward regardless of external value markers.

**§6 below drills this case more deeply** — training gradient shows HOW someone can develop appreciation through chunks compile.

### §5.6 — Scenario C2.5: Vintage Car (Wrong-Era Chunks)

**Most interesting scenario** because the same item produces opposite outcomes for father vs son.

**Son's state**:
- **P1 Pending**: ❌ No "want a vintage car" schema (wants modern)
- **P2 Chunks base**: ❌ No "vintage car value" chunks (doesn't know the historical significance)
- **P3 prediction-delta**: Moderate — unusual car (not modern) → registers as "strange"
- **P4 Goldilocks**: ❌ Son may see <20% match with "what I want" (modern aesthetics)
- **P5 Tag**: Neutral or negative ("outdated" chunks)

**Son**: No reward. Possibly mild disappointment.

**Father's state**:
- **P1 Pending**: ✅ Has schema "want a vintage car" (collector interest)
- **P2 Chunks base**: ✅ Rich chunks about vintage cars (history, craftsmanship, rarity, provenance)
- **P3 prediction-delta**: HIGH — seeing this specific car and knowing what it is
- **P4 Goldilocks**: ✅ within zone (recognizable + specific)
- **P5 Tag**: ✅ Opioid-positive (vintage chunks tagged with authenticity, history, craftsmanship)

**Father**: Full reward.

**→ SAME ITEM → opposite reward** based on different chunks bases.

**The key question**:
> "But the father loves this vintage car more than newer, cheaper models — and so do his friends. Why doesn't the son feel the same way?"

**Framework answer**: Reward is a personalized function of chunks base. Father and his friends share cultural chunks about vintage cars → common appreciation. Son lacks those chunks → no appreciation.

**Can son learn to appreciate it?** Yes — by building chunks base (studying car history, collecting examples, listening to his father's stories). But **reward cannot be transferred directly**.

**Key framework claim**:

> **Reward is NOT intrinsic to the object. Reward is computed per-person based on their individual chunks base + current schema pending state.**

**Philosophical implication**: There is no "objectively better" object. "Value" in a domain can be measured (monetary, historical, expert consensus). But "reward" is always **subject-dependent**.

Reward = direction-match quality, not just "gap filled or not." The Car Paradox C2.1–C2.5 = perfect test cases for gap direction: same "car" but DIFFERENT direction → DIFFERENT reward.

## §6 — Van Gogh Appreciation Paradox (C3) — Training Gradient

### §6.1 — The core question

> "Another example of Van Gogh's painting — honestly it looks to me like a child drew it: wavy lines, messy, don't understand it, not beautiful at all. But it's considered an artistic masterpiece. Does that mean the people who appreciate it must also have formed many chunks similar to Van Gogh's in order to perceive its value?"

**Framework answer** 🟡: ✅ **YES, precisely correct.** This intuition matches the framework exactly.

### §6.2 — Framework explanation

This case is a **specific application** of the §5.8 failure mode table:
- Violates **P2** (chunks base insufficient)
- Violates **P4** (Goldilocks <20% match = too alien)
- Result: "don't understand it" experience

**Core claim** 🟡:

> Art appreciation requires a **matching chunks base** with the artist's chunks (or at least compatible ones). Without those chunks, match <20% → dissonance/confusion, not reward.

The intuition: "Those who appreciate it must also have formed many chunks **similar to** Van Gogh's."

Framework: Not exactly like Van Gogh's (no one can have the same exact chunks), but in the **same neighborhood** — chunks about:
- Post-impressionism technique
- Expression of emotional states via color and brushstroke
- Historical context (Van Gogh's life, era, artistic movements)
- Art history evolution (why his deviation was significant)
- Emotional resonance with his themes (isolation, nature, emotional intensity)

### §6.3 — Why match ratio moves INTO the Goldilocks zone with training

**Beginner's state**:
- Art chunks: minimal (perhaps "colors look nice," "painting = representation")
- Van Gogh input match: <20% (strokes don't represent cleanly, colors are unrealistic)
- Body signal: "strange, ugly" — P4 failed
- **Result**: Dissonance, confusion, no reward

**Intermediate learner** (after some art study):
- Chunks about: brushstroke techniques, color theory, artistic movements
- Van Gogh input match: approaching Goldilocks zone (recognizes technique, still finds emotional intensity excessive)
- Body signal: "interesting, not sure if beautiful" — approaching Goldilocks
- **Result**: Mild interest, mixed feelings

**Advanced learner** (years of study):
- Chunks about: Van Gogh's life, historical influence, technical innovation, comparative artists
- Van Gogh input match: within Goldilocks zone
- Body signal: opioid release — "beautiful, moving, significant"
- **Result**: Full reward, aesthetic appreciation

**Expert / artist**:
- Chunks: can replicate/analyze technique, understand historical position, personal emotional resonance
- Van Gogh input match: 60–80% — deep Goldilocks
- Additional: appreciation triggers own creative chunks (simulating being Van Gogh)
- **Result**: Profound reward, possibly tears (multi-schema cascade)

### §6.4 — Training gradient formalized

🟡 **Framework model**:

| Stage | Chunks base | Match ratio | Body signal | Subjective experience |
|---|---|---|---|---|
| **Beginner** | Minimal | <20% | Dissonance | "Wavy, messy, don't understand" |
| **Beginner + exposure** | Some art basics | 20–30% | Mild dissonance | "Strange, unimpressive" |
| **Intermediate** | Art movements + techniques | 30–50% | Mixed | "Interesting, mixed feelings" |
| **Advanced** | Historical + technical + context | 50–70% | **Opioid** ✅ | "Beautiful, meaningful" |
| **Expert** | Deep + personal resonance | 60–80% | **Strong opioid** ✅✅ | "Profound, moving" |

**Key observation**: Training moves the person through the Goldilocks transition → appreciation emerges at a certain threshold.

**Not linear**: Progression has plateaus + breakthrough moments. Building a chunks base is not smooth.

### §6.5 — 2-Layer Calibration applied (Why-Body-Knows v1.1 §3)

**How the chunks base builds**:

🟢 **Layer 1 Darwinian** (genetically encoded): Some aesthetic sensitivity is wired in (symmetry, color contrast, face recognition — universal)

🟡 **Layer 2 Hebbian — Path 2a Domain Contact** (direct experience):
- Museum visits, art classes, looking at many artists' work
- Personal practice with visual art, direct exposure

🟡 **Layer 2 Hebbian — Path 2b Trust-Injected** (learned through trusted sources):
- "Van Gogh was a pioneer" — meta-information (books, teachers, documentaries)
- Stories of his life, critical frameworks from art history
- AI providing background, technique analysis, historical context on demand

⚠️ Culture, teachers, and AI = INPUT SOURCES into the same Hebbian mechanism (Layer 2),
   NOT separate "layers." They differ in the INPUT PATH, not the underlying mechanism.

**Implication**: Appreciation can be **accelerated** via Path 2b trust-injected chunks + Path 2a direct exposure combined.

### §6.6 — Why "everyone appreciates Van Gogh eventually" phenomenon

**Common observation**: People who dismiss post-impressionist art at age 20 often come to love it by age 40.

**Framework explanation** 🟡:
- Age 20 → chunks base thin → match <30% → dismiss
- Ages 20–40 → accumulated cultural exposure (friends, media, museums, incidental learning)
- Age 40 → chunks base sufficient → match in Goldilocks zone → appreciation emerges

**NOT** because the art "became better" or "person became smarter." Chunks base grew via passive + active exposure → crossed the appreciation threshold.

### §6.7 — Comparison: Van Gogh vs children's drawings

"Honestly it looks to me like a child drew it."

**Are they actually similar?** Physically (to an untrained eye): some similarity. Wavy lines, bold colors, unconventional.

**Why trained viewers experience reward for Van Gogh but typically not for children's drawings**:
- Van Gogh chunks include: **historical significance** chunks ("this was done intentionally in 1889 when realism dominated")
- **Technical chunks** ("this represents emotional state mapped onto color")
- **Narrative chunks** ("this artist struggled with mental illness; this work captured his inner experience")
- **Comparison chunks** (can compare to other post-impressionists)

Children's drawings:
- Few historical significance chunks
- No intentionality chunks
- No technical innovation context
- Minimal comparison framework

**Same visual surface** → **different chunks cascade** → **different match ratio** → **different reward**.

### §6.8 — The deeper question: "objective" vs "trained" beauty

🔴 **Speculative framework position**:

Is Van Gogh "objectively" a masterpiece? Framework answer: **No "objective" status.** Beauty is a relationship between the object and the observer's chunks base.

**BUT**: Beauty can be **consensus** if enough observers have built a similar chunks base (cultural alignment).

**Expert consensus** (art history academics) share similar chunks → consistent appreciation → "masterpiece" emerges from shared appreciation, not from the object alone.

**Implications**:
- "Masterpiece" is meaningful but subject to chunk-base distribution across a population
- People can legitimately not appreciate Van Gogh (chunks base mismatch) — not ignorance, just different chunks
- Training can develop appreciation — but the person chooses whether to invest
- "Beauty standards" are chunks distributions shaped by culture

An honest self-report ("it looks like a child drew it") is **valid data**. Not wrong — just reflects current chunks state. Could change with exposure, or could remain genuinely uninterested.

### §6.9 — Implications for drill cases

**Reward is trainable but cannot be forced**:
- Building chunks base is effortful
- Body doesn't lie about current state
- Cannot fake appreciation

**Parenting implication**: Exposing children to "high culture" early doesn't guarantee appreciation. Must build chunks + give autonomy to develop genuine preference.

**Cross-reference**:
- **C5 schema update latency** (§8 below): chunks base for a new self-schema takes time to build → reward latency
- **Hedonic trap** (P2 §9): chunks base decays without activation → loss of appreciation even if previously present

## §7 — Spreading Activation Multi-Schema (C4)

### §7.1 — The question

> "Does new sensory input (vision, smell, hearing…) OR new chunk linking (imagination) simultaneously activate many schemas?"

**Framework answer** 🟢 + 🟡: ✅ **YES, this is spreading activation** — established neuroscience (Collins & Loftus 1975) integrated into the framework.

### §7.2 — Spreading Activation Mechanism

🟢 **Collins & Loftus 1975**:

> Concepts in semantic memory are nodes in a network. When one node activates, activation spreads to connected nodes, gradually fading with distance.

**Modern formulation**:
- Neurons (chunks) are nodes
- Synaptic connections = edges, weighted by connection strength
- Activation of chunk X → propagates to chunks Y, Z connected to X
- Activation decays with distance (fewer hops = more activation)
- Strong connections transmit more activation than weak ones

**Applied to new input**:

```
┌────────────────────────────────────────────────────────────┐
│                                                            │
│  SINGLE INPUT (e.g., visual of someone you love)           │
│          ↓                                                 │
│  Initial chunk fires (e.g., "face of person X")            │
│          ↓                                                 │
│  Spreading activation cascade:                             │
│          ├→ Memory chunks about person X                   │
│          ├→ Emotional chunks associated with person X      │
│          ├→ Context chunks (where you met, shared moments) │
│          ├→ Future chunks (plans with person X)            │
│          ├→ Body state chunks (sensations from interaction)│
│          ├→ Values chunks (what person X means to you)     │
│          ├→ Social network chunks (friends, connections)   │
│          └→ ... hundreds to thousands of chunks            │
│                                                            │
│  → Multi-schema cascade → multi-need potential fill        │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### §7.3 — Why multi-schema activation amplifies the signal

🟡 **Framework claim**:

> When a single input activates many related chunks, the body's Step 5 check runs against **multiple body-needs simultaneously**. If several needs fit the cascade → compound opioid release → amplified reward.

**Example** — Case C1.2 revisited:
- Single event: teacher praises student
- Cascade activated:
  - "Competence validated" → competence need filled
  - "Social approval received" → belonging need filled
  - "Parent would be proud" → family connection filled (imagined)
  - "Teacher likes me" → safety in school environment filled
  - "Peers see me as capable" → status need filled
  - "Future prospects" → long-term security signal
- **Each** contributes an opioid signal
- **Compound result**: much stronger reward than any single-need fill would produce

**Framework prediction**: Reward intensity ∝ number of body-needs simultaneously filled via the cascade.

### §7.4 — Directionality of cascade

🟡 Cascades can go positive OR negative:

**Positive cascade**: chunks with opioid tags activate → reward compounds
- Seeing a loved partner → love chunks + safety + future hopes → strong opioid

**Negative cascade**: chunks with cortisol tags activate → dissonance compounds
- Seeing an ex after a painful breakup → loss chunks + rejection + what-might-have-been → strong dissonance

**Mixed cascade**: both tags activate → ambivalent feeling
- Bittersweet moments (graduation, wedding, funeral)
- Same event fires many conflicting schemas

**Framework explanation for "complex emotions"**: They are cascade products where chunks carry mixed tags.

### §7.5 — Why new input activates many schemas

For meaningful input, cascade activation is **always true**:

> Any input strong enough to register consciously has likely triggered a cascade. "Meaningful" = activates more chunks than a routine stimulus.

**Examples**:

**Strong cascade triggers**:
- Emotional events (joy, grief, surprise)
- Meeting someone after a long separation
- Returning to a childhood home
- Music with personal history
- A smell triggering an old memory (🟢 Proust effect, olfactory → hippocampus direct pathway)

**Weak cascade triggers**:
- Background television
- Routine grocery shopping
- Repeated daily tasks

**Why a rich cascade = meaningful**:
- Activation reaches many memory chunks
- Past + present + future integrated
- Multi-modal (visual + auditory + emotional + somatic)
- Body signals: "this matters"

### §7.6 — Imagine-Final and cascade

🟡 **Framework claim**: **Imagine-Final formation uses spreading activation.**

**Mechanism** (Imagine-Final §1.5 + §7.2 above):
- PFC sets up goal/theme
- Activation spreads to chunks related to the theme
- Chunks recombine via cascade
- Novel combinations emerge as Imagine-Final candidates
- Body pre-feels each candidate

**Creative insight** = cascade producing a novel chunk combination not previously compiled. The "aha moment" literally is the cascade hitting a new valid pattern.

**Example** (spontaneous creative emergence):
- Exposure to beautiful flowers → flower chunks activate
- Later: DMN mode, cascade spontaneously activates flower chunks + "my courtyard" chunks
- Combination: "flowers in the courtyard" emerges as a new Imagine-Final
- This is **cascade-driven creativity**

### §7.7 — Limitations of cascade

🟡 Cascade is not infinite:

- **Activation decays with distance**: remote chunks don't fire
- **Requires strong initial input**: a weak trigger = a weak cascade
- **Competing cascades interfere**: multiple simultaneous strong triggers → cacophony
- **Chunks base matters**: a sparse chunks base = shorter cascade = less reward potential

**Implication**: Rich chunks base → rich cascade potential → rich emotional life.

This connects to earlier sections:
- Hedonic trap (P2 §9): chunks decay → cascade shrinks → less reward even for positive inputs
- Van Gogh (§6): chunks base insufficient → cascade limited → no appreciation
- Expertise: rich chunks → cascade hits many relevant associations → expert "sees" more in each stimulus

### §7.8 — Framework commitment on C4

> "Does new sensory input OR new chunk linking (imagination) simultaneously activate many schemas?"

**YES.** Framework fully commits. Mechanism: spreading activation via synaptic network. Richer chunks base → richer cascade → richer experience.

**Implication**: Quality of life is proportional to chunks base depth + diversity. Building a chunks base = building capacity for richer experience.

## §8 — Schema Update Latency (C5 Shift Status)

### §8.1 — The case

> "Shift status: why can't you believe it yet? And if you can't believe it, doesn't it feel slightly contradictory? Why does it take time to become normal and then feel confident? — A student from a rural school wins first prize at the provincial academic competition. There's joy when hearing the news, but still disbelief. Only after some time does it feel normal, and confidence and pride set in."

**Framework answer** 🟡: This case reveals the **reward latency mechanism** — positive information doesn't instantly produce full reward. Schema update takes time.

### §8.2 — Why disbelief happens despite positive information

**Key insight** 🟡:

> Reward isn't triggered by information ALONE. It's triggered by **body-level coherence**. The self-schema must update before full reward can fire.

**Immediate receipt of good news**:
- External info: "You won first prize at the provincial competition"
- Auditory input → PFC receives → attempts to label
- **BUT**: self-schema "I'm from a rural school, less capable than city students" still compiled
- Contradiction: new info ≠ existing self-schema
- Body detects mismatch → **cognitive dissonance fires**
- **Mild dissonance despite positive content** — "can't quite believe it" = mini dissonance from coherence gap

### §8.3 — Framework mechanism decomposition

**Phase 1 — Immediate information receipt**:

- External announcement arrives
- Auditory processing → concept "first prize at provincial level" → PFC workspace
- **P3 prediction-delta**: HIGH (unexpected positive event)
- **P2 Chunks base**: partial — student understands "first prize" = good thing
- **P1 Pending**: mixed — may have wanted to compete but held self-doubts
- **P4 Goldilocks**: poor match — self-schema doesn't align with the information
- **P5 Tag**: positive content
- **Initial reward**: partial opioid — joy on hearing the news

Mechanism: Enough dopamine + some opioid → recognizable as positive. But full reward is gated by self-schema coherence.

**Phase 2 — Coherence check fails**:

- PFC attempts integration: "I am a 'provincial competition first prize winner'"
- Body check: does this fit the self-schema?
- Self-schema: "I'm from a rural school, not the type who wins provincial competitions"
- **Mismatch detected** → dissonance signal
- **"Can't quite believe it"** = body reporting coherence gap

**Critical**: The doubt is NOT about the truth of the information. It is about the **self-identity schema** not yet updated.

**Phase 3 — Suspended state (joy + disbelief simultaneously)**:

- PFC cannot resolve immediately
- Two schemas firing in parallel:
  - New info: "I won"
  - Old self: "I am the type who doesn't win"
- Subjective: **complex mixed feeling** — partial joy + disbelief + confusion
- Precisely captured: "pleased when hearing the news, but still disbelieving"

**Phase 4 — Gradual schema update (days to weeks)**:

External reinforcement compounds:
- Family congratulates → social validation
- Teachers mention it → authority validation
- Classmates acknowledge → peer validation
- Tangible reminder (trophy, certificate) → physical evidence

Each reinforcement → **schema update pressure** (Chunk.md reconsolidation mechanism activated)
- Old self-schema gets nudged
- New self-schema ("I am someone who can win") compiles via emotional peak path (success event)
- **Multiple compile pathways** (Chunk.md §2): repetition + emotional peak + multi-modal

**Phase 5 — Schema integration complete**:

After days to weeks:
- Self-schema updated: "I can win"
- Old rural-school-inferiority schema weakened but not deleted (Chunk.md §2 "no delete")
- New schema becomes primary self-reference
- **NOW coherence check passes** — information + self match
- **Full reward fires**: confident and proud

Precisely captured: "Only after some time did it feel normal, with confidence and pride."

**"Feels normal"** = new schema accepted as baseline. No longer special because it has been integrated into self-identity.

### §8.4 — Why schema update takes time

🟡 Framework mechanisms (Chunk.md §2 + Learning-Cycle §3–§4):

**Chunks don't update instantly**:
- Compile pathways require time + activation
- Emotional peak path (one-shot compile) works only for high-intensity events
- Most schema update = repetition + multi-modal input over multiple iterations

**Sleep consolidation critical**:
- Each night's sleep refines the schema
- Takes multiple cycles (Learning-Cycle §4 6+1 mechanisms) to fully integrate
- Reconsolidation window (Nader 2000): recall → unstable → modify → re-stabilize

**Why days to weeks**:
- ~3–5 nights for initial integration
- ~2–3 weeks for self-schema level changes
- Months to years for deep identity shifts (e.g., career change)

**Implication**: Cannot force instant "belief" in new positive information. Body needs integration time.

### §8.5 — Real documented alternatives

Framework provides 5 well-documented real examples of the same mechanism:

#### §8.5.1 — C5.alt1: Impostor Syndrome Post-Promotion

🟢 **Research**: Clance & Imes 1978 (foundational), subsequent extensive research.

**Phenomenon**: High-achievers (often first-generation executives, minority hires, academics) receiving an unexpected position/recognition feel "undeserving" despite clear merit.

**Common experience**:
- Month 1–3: "They made a mistake hiring me"
- Month 3–6: Anxiety about being "found out"
- Month 6–12: Gradually accepting the role
- Year 2+: Comfortable with the identity

**Mechanism mapping** to framework:
- External: high position assigned
- Self-schema: "not the kind of person who belongs here"
- Coherence gap: persistent
- Resolution: daily execution + social feedback + successful performance → schema update

**Framework prediction**: Impostor syndrome resolves (for most) via **repeated positive feedback compiling a new self-schema**. Some never fully resolve → stay in mild chronic dissonance.

#### §8.5.2 — C5.alt2: Lottery Winner Disbelief

🟢 **Research**: Brickman et al. 1978 (classic hedonic adaptation study), subsequent replications.

**Phenomenon**: Lottery winners report an initial "surreal" phase — "it doesn't feel real."

**Common timeline**:
- Day 1–7: "Dream-like, not processing fully"
- Week 2–4: Practical implications dawn (taxes, family, lifestyle changes)
- Month 2–6: Gradual acceptance + identity shift
- Year 1+: New baseline emerges

**Mechanism mapping**:
- External: wealth acquired
- Self-schema: "ordinary person with ordinary financial situation"
- Coherence gap: massive
- Resolution: practical engagement with new reality forces schema update

**Brickman's famous finding**: Despite the initial peak, life satisfaction returns toward baseline within 12–18 months. Full schema integration happens, but hedonic adaptation also kicks in → no sustained massive happiness boost.

**Framework interpretation**: Schema updates to new baseline → new pending schemas emerge → back to normal dissonance/reward patterns. The peak was a transitional phase, not a permanent state.

#### §8.5.3 — C5.alt3: Kuhn Paradigm Shift Acceptance

🟢 **Research**: Kuhn 1962 "The Structure of Scientific Revolutions."

**Phenomenon**: Scientists trained in one paradigm literally cannot "see" a new paradigm even when the evidence is compelling.

**Classic examples**:
- Continental drift theory (Wegener 1912): rejected for decades until evidence + generational change
- Plate tectonics (1960s): accepted after the old guard retired
- Helicobacter pylori causing ulcers (1980s): Marshall had to drink the bacteria to prove it

**Kuhn's famous observation** (via Max Planck):
> "Science advances one funeral at a time."

Meaning: Established scientists never truly update. They die, and a new generation (raised with the new paradigm) replaces them.

**Mechanism mapping**:
- External: new evidence + theory
- Self-schema: "I am a [old paradigm] expert"
- Coherence gap: accepting the new paradigm = invalidating career investment
- Resolution: for most, never — only generational turnover resolves it

**Framework interpretation**: The strongest case of schema update resistance. Self-schema tied to professional identity + life's work → updating = identity threat + loss of expertise capital.

**Latency**: Years to decades. Sometimes never.

#### §8.5.4 — C5.alt4: Out-of-League Relationship Disbelief

🟡 **Phenomenon** (research supports, common clinical observation):

A person receiving unexpected romantic interest from a partner perceived as "higher status" → "why would they want me?"

**Timeline**:
- Initial: euphoria + disbelief
- Weeks 1–3: testing the relationship's reality
- Months 1–6: gradual acceptance + security building
- Year 1+: comfortable

**Mechanism mapping**:
- External: mutual attraction + relationship
- Self-schema: "I am not the kind of person who deserves this partner"
- Coherence gap: desire ≠ self-worth
- Resolution: continued partner commitment + personal growth → update

**Common failure mode**: Self-sabotage — schema protects itself by creating situations that confirm the old self ("see, I told you I'm not worthy"). Relationship collapses → schema preserved.

**Successful resolution requires**: partner patience + personal growth work + time + consistent positive evidence.

#### §8.5.5 — C5.alt5: The Underdog Narrative Genre

🟡 **Cultural phenomenon**: An entire narrative genre built around this mechanism.

- Underdog rises
- Gradual belief in self
- External + internal journey
- Final transformation

**Why it resonates universally**: Humans recognize the mechanism. Schema update IS a narrative — it takes time, has stages, requires both external catalyst and internal growth.

Examples: Cinderella, Rocky, Slumdog Millionaire, countless biopics, coming-of-age stories.

**Framework insight**: The "underdog story" structure reflects actual neuroscience of self-schema update. Not arbitrary storytelling — it mirrors lived experience.

### §8.6 — Why C5 is critical for the drill

🟡 **Key framework insight**:

> **Reward latency** matters because it shows reward is **NOT instant** even when "it should be." Body has its own timeline determined by schema integration cost.

**Implications**:
- Cannot "talk yourself into" feeling successful — body needs to experience the integration
- Good news received but not yet celebrated internally = normal, not abnormal
- Gradual acceptance ≠ weakness — it is architectural
- Full integration takes weeks to months for significant identity shifts

**Practical application**:
- Celebrating achievements matters (allows full schema integration)
- Social validation accelerates compile (multi-modal reinforcement)
- "Fake it till you make it" works partially — action forces schema exposure to new evidence
- Being patient with oneself during transitions is architecturally appropriate

### §8.7 — H10 precondition analysis for C5

**Phase 1 state** (immediate good news):
- P1 Pending: Complex — partial pending (competitive motivation) + new pending emerging ("be a successful student")
- P2 Chunks: INSUFFICIENT for the new self — chunks about "winner identity" not yet compiled
- P3 Delta: HIGH
- P4 Goldilocks: POOR match between new information and current self-schema
- P5 Tag: Positive content but triggers uncertainty tag

**Reason for "can't quite believe it"**: **P2 + P4 failed** at the self-schema level. Body signals "information exists but doesn't fit self" → coherence gap → mild dissonance despite positive content.

**Phase 5 state** (after integration):
- P2 Chunks: New "winner" chunks compiled
- P4 Goldilocks: New self-schema matches new identity
- All preconditions met
- Body fires full reward on any reminder of the achievement

**Framework prediction**: Schema integration time ∝ (magnitude of self-schema change) × (chunk maintenance rate) / (reinforcement rate).

### §8.8 — Connection to trauma loop (anti-case)

Interesting inverse: **Trauma = schema update failure in the negative direction.**

**Positive case (C5)**:
- New positive info → gradual integration → new healthier self-schema
- Takes time but ends well

**Trauma case (P2 §8)**:
- New negative event → compiled deeply (emotional peak)
- Tries to integrate but cannot resolve
- Gets stuck at Phase 2–3 indefinitely
- Never reaches integration

**Difference**: Positive schema update has a clear positive destination (compile a new better self). Trauma has no clear destination (no "good" interpretation available) → cycle instead of resolution.

**Framework prediction**: Positive schema updates typically succeed (given reinforcement). Negative schema updates from trauma often fail without intervention.

## §9 — Chunk Association Tag Deep Dive

### §9.1 — Framework source

This section drills **Cortisol-Baseline §3.5** — the Novelty Cortisol vs Threat Cortisol distinction — which was identified as **H10 Precondition 5** in §3.6.

### §9.2 — Core claim restated

🟡 **Framework commitment**:

> Chunks compile under specific neurochemical context. That context leaves a **tag** that persists through chunk retrieval. Same content can be **opioid-tagged** (from novelty context) OR **cortisol-tagged** (from threat context). The tag determines future usability.

**Critical implication**: Chunk content alone is insufficient to predict future use. Must also know the tag.

### §9.3 — Mechanism: Novelty Cortisol vs Threat Cortisol

Both involve cortisol, but:

**Novelty cortisol**:
- Curiosity, exploration, learning interest
- Accompanied by: dopamine + opioid preview
- Body state: engaged, focused, eager
- Neurons fire in **directed pattern** (attention on the novelty source)
- Calibration has direction (purposeful)
- Sleep quality: good (task done, cortisol drops)

**Threat cortisol**:
- Fear, coercion, ongoing danger
- Accompanied by: NE + adrenaline
- Body state: tense, alert, defensive
- Neurons fire in **scattered pattern** (scanning for danger)
- Calibration is anxious without clear direction
- Sleep quality: poor (cortisol stays elevated)

### §9.4 — The tagging mechanism 🟡

**During chunk compile**:
- New chunks form via Hebbian LTP (neurons fire together wire together)
- **Neighboring neurochemical activity** at time of compile gets wired in
- If environment is novelty-rich → dopamine + opioid co-active → wire in
- If environment is threat-rich → NE + cortisol co-active → wire in
- The "tag" = association with the co-active neurochemistry at compile time

**During chunk retrieval**:
- Retrieving the chunk activates the same network
- **Associated tags fire too** (lingering neurochemical association)
- Body experiences: original content + emotional association
- Opioid-tagged chunk: "remembering feels good"
- Cortisol-tagged chunk: "remembering feels uncomfortable"

**🟡 Framework claim**: This explains why "the same factual knowledge" can feel completely different to different people — compile context differs.

### §9.5 — 4 Threat Intensity Levels

Cortisol-Baseline §3.5 commits to **4 threat intensity levels** with different chunk outcomes:

#### §9.5.1 — Mild threat (60:40 threat:novelty)

**Example**: "Finish your homework → then you can play" parenting

**Mechanism**:
- Mild threat (losing play time)
- Some intrinsic novelty (possibly an interesting task)
- Cortisol: low
- Opioid: present (anticipating the play reward)
- **Chunk tag**: mostly opioid, slight cortisol

**Result**: Chunks compile as usable. Future retrieval: functional, slight wariness. Good foundation.

✅ **Framework verdict: valuable** — gets chunks compiled + minimal long-term damage.

#### §9.5.2 — Moderate threat (80:20)

**Example**: "Must finish the project this week — otherwise working through the weekend"

**Mechanism**:
- Moderate threat (losing weekend)
- Some task interest
- Cortisol: moderate
- Opioid: minimal
- **Chunk tag**: mostly cortisol, some opioid

**Result**: Chunks compile but "reluctant." Adult retrieval: functional but accompanied by dread. Can do it but doesn't enjoy it.

✅ **Framework verdict: acceptable foundation** — trade-off with long-term motivation.

#### §9.5.3 — Heavy threat (95:5)

**Example**: "Failure to study means punishment every day for the entire year"

**Mechanism**:
- Heavy threat, daily intensity
- Cortisol: HIGH, sustained
- Opioid: absent
- **Chunk tag**: deep cortisol

**Result**: Chunks have content but **body-base objects to retrieval**:
- Opening a textbook → automatic body discomfort
- Sitting at a desk → muscles tense
- **Compiled avoidance response** alongside the content

Adult outcome: May know the material but cannot enjoy using it. "Competent but averse" pattern.

⚠️ **Framework verdict: low value** — chunks present but problematic to use.

#### §9.5.4 — Extreme threat (violence, humiliation, trauma)

**Example**: Extreme abuse contexts

**Mechanism**:
- EXTREME cortisol
- Trauma-level compile
- Other schemas damaged beyond just learning content
- **Chunk tag**: TRAUMA-level cortisol

**Result**: Chunks present but RETRIEVAL triggers panic/freeze:
- Seeing a textbook → trauma recall response
- Teacher's voice → body freezes
- **Chunks become triggers** for trauma response

Adult outcome: Dangerous to engage with the content. May require therapy to reprocess.

❌ **Framework verdict: net negative** — content compiled, but cost to access outweighs any benefit.

### §9.6 — 4-Level Summary Table

| Level | Threat:Novelty | Chunk tag | Content accessible? | Usability | Framework verdict |
|---|---|---|---|---|---|
| **Mild** (60:40) | Low threat | Opioid majority | ✅ Easy | ✅ Good | Valuable |
| **Moderate** (80:20) | Moderate threat | Cortisol majority | ✅ Yes | ⚠️ Reluctant | Acceptable |
| **Heavy** (95:5) | Heavy threat | Deep cortisol | ⚠️ Avoidant | ❌ "Competent but averse" | Low value |
| **Extreme** | Trauma | Trauma-level | ❌ Triggers panic | ❌ Dangerous | Net negative |

**Critical insight**: There is no linear "more threat = more damage" — there are **threshold effects** at each level. The difference between mild and moderate is gradual. Between heavy and extreme is qualitative (content becomes a trauma trigger).

### §9.7 — Genius Paradox Deep Dive

**Introduced in P2 §5.6**, this section drills deeper.

**Phenomenon**: Newton, Tesla, and Einstein had high cortisol baselines (trauma, anxiety, difficult childhoods). Yet they produced brilliant work showing flow states and deep engagement. Why didn't their learning-domain chunks all become cortisol-tagged → avoidant?

🟡 **Framework resolution**:

> **The source of cortisol matters more than its level.**

**Mechanism**:

- High cortisol baseline → body in change-readiness mode continuously
- BUT the threat source ≠ the learning domain
- → Body has energy available for pattern change → DIRECTS that energy toward novelty in any available domain
- → Learning domain receives novelty cortisol + opioid preview + directed attention
- → Chunks there get OPIOID tag
- → Long-term usable for flow and creative work

### §9.8 — Case studies

#### §9.8.1 — Newton

**Trauma source**: Mother's abandonment (maternal figure lost through remarriage, severe attachment wound).

**Cortisol baseline**: Chronically high.

**Learning behavior**:
- Read voraciously (books = replacement connection?)
- Mathematical fascination (abstract world = safe)
- Physics inquiry (understanding nature = filling a gap?)

**Chunks in math/physics**: Compiled with OPIOID tag because engagement was **chosen refuge**, not compelled.

**Result**: Revolutionary work. Chunks accessible, enjoyable, generative.

**Social chunks**: Partially cortisol-tagged (lifelong social difficulties, reclusiveness).

#### §9.8.2 — Tesla

**Trauma source**: Brother died young, childhood illness, family grief.

**Cortisol baseline**: High.

**Learning behavior**:
- Escaped into vivid imagination (safe mental space)
- Inventions emerged from imagination play
- Self-directed experimentation

**Chunks in engineering/physics**: OPIOID-tagged — pursuit of invention = self-rescue from a painful reality.

**Result**: Prolific inventor. Work was enjoyed, though life remained troubled.

#### §9.8.3 — Einstein

**Trauma source**: Social difficulties (late to speak, school mismatch, unusual behavior patterns).

**Cortisol baseline**: Moderate-high.

**Learning behavior**:
- Disliked formal school (school chunks became cortisol-tagged)
- LOVED physics (self-directed, curiosity-driven)
- Thought experiments as primary method

**Chunks in physics**: OPIOID-tagged — chosen pursuit, fascinating, fills coherence-gap need.

**Chunks in school environment**: Cortisol-tagged — lifelong frustration with institutional education.

**Result**: Relativity breakthrough. Lifelong struggle with teaching environments.

### §9.9 — General pattern

🟡 **Framework pattern**:

> High baseline cortisol + threat source ≠ chosen domain + chosen domain has intrinsic interest → chosen domain chunks get opioid tag → flow possible → brilliance possible.

**Contrast**: High baseline cortisol + threat source = learning domain itself → all learning chunks cortisol-tagged → cannot enter flow → potential suppressed.

**Practical implication**: "Gifted trauma survivors" often excel in self-chosen domains while struggling with imposed ones. Pattern consistent with framework.

### §9.10 — Re-association: 3 Paths (Cortisol-Baseline §3.5)

Critical framework claim: **Chunk tags CAN be changed via re-association**.

3 documented paths:

#### §9.10.1 — Path 1: Re-associate via new experience

**Mechanism**:
- Use chunks in a NEW positive context
- Repeat many times
- Opioid association gradually overlays the cortisol association
- Chunks don't delete (Chunk.md §2) but the tag changes

**Example**: An adult who "hates math" (threat-tagged chunks) starts doing data analysis for a game development hobby. Uses the same math chunks but in a novelty context (fun, self-directed). Over months to years, the association shifts.

**Timeline**: Slow (months to years). The threat association never fully disappears but weakens.

**Success factors**:
- Consistent new positive context
- Sufficient repetition
- Patience with initial discomfort
- No re-traumatization

#### §9.10.2 — Path 2: Novelty hijack

**Mechanism**:
- Novelty drive (intrinsic curiosity) spontaneously activates cortisol-tagged chunks for a NEW purpose
- If the new purpose is exciting → chunks get "hijacked" into the novelty context
- Dopamine + opioid associations form around the new use

**Example**: Someone who hated chemistry class (cortisol-tagged chunks) becomes fascinated with cooking. Realizes "chemistry = understanding food reactions." Chemistry chunks suddenly activate in a positive food context → association shifts.

**Faster than Path 1** because the novelty drive provides active motivation.

**Requirement**: Genuine novelty drive + a context that permits hijacking.

#### §9.10.3 — Path 3: AI support (new era)

🟡 **Framework extension**:

- AI provides CONTEXT for cortisol-tagged chunks in a new way
- Conversational, non-judgmental, infinitely patient
- Meets the person where they are
- Bridges chunks to new purposes

**Example**: An adult with math anxiety uses an AI tutor to understand statistics for a hobby. AI explains patiently, links to interesting topics, creates opioid-context engagement. Over sessions, math chunks re-tag.

**Advantages**:
- Infinite patience (no frustration transmitted)
- Personalized pace
- Context-adaptive
- Available 24/7

**Risks**:
- AI gives incorrect information → builds wrong chunks
- AI cannot replace body-based experience entirely
- Risk of over-reliance without real-world action

**Framework position**: AI is a genuinely new path not previously available. Emerging research needed.

### §9.11 — Implications for trauma recovery

P2 §8.8 trauma treatment can be connected here:

Re-association paths offer a framework for trauma chunk recovery:
- Path 1: Somatic therapy (SE, sensorimotor) — re-experiencing in a safe context
- Path 2: Finding meaning (Viktor Frankl tradition) — novelty hijack for trauma
- Path 3: AI-supported exposure — emerging

None is instant. All require sustained effort. But all can work.

**Key**: Chunks don't delete. Must overlay with new associations. This is architectural, not a limitation — it is HOW memory systems work.

## §10 — Reward Sustainability

### §10.1 — Central question

Previous sections showed WHAT produces reward. This section asks: **How is reward SUSTAINED?** Why does some reward last, while other reward quickly fades?

### §10.2 — Sustainability by reward type

🟡 Framework distinction:

| Reward type | Source | Sustainability | Example |
|---|---|---|---|
| **Novelty reward** | prediction-delta + opioid from chunks compile | **High** — as long as chunks continue accumulating | Curious exploration, learning domain |
| **Relief reward** | Cortisol drop after threat resolves | **Low** — diminishing returns, needs escalating threat | Procrastination relief, avoiding punishment |
| **Mastery reward** | Competence schema confirmation | **High** — deep opioid tagging + self-efficacy | Skilled craft, expertise |
| **Social approval reward** | Connection schema fill | **Medium** — requires ongoing social context | Praise, status, recognition |
| **Sensory reward** | Direct body-need fill | **Low** — satiety quickly, habituation | Food, sex, drugs (without connection) |
| **Meaningful work reward** | Multi-schema cascade filling | **Highest** — compound effect, value-aligned | Einstein-level passion projects |

**Key insight**: **Sustainability** depends on how reward integrates with the ongoing schema landscape, not just on intensity.

### §10.3 — Flow state mechanism

🟢 + 🟡 Flow state research (Csikszentmihalyi) integrated into framework:

**Conditions for flow**:
- Challenge ≈ Skill (not too easy, not too hard)
- Clear goals (Imagine-Final present)
- Immediate feedback (domain response is quick)
- Deep engagement (PFC focus sustained)

**Framework decoding**:
- **Challenge ≈ Skill** = Goldilocks zone maintained (P4 ✅)
- **Clear goals** = Schema pending active (P1 ✅)
- **Immediate feedback** = prediction-delta frequent (P3 ✅)
- **Deep engagement** = chunks base sufficient for the task (P2 ✅)
- Activity chosen = opioid-tagged (P5 ✅)
- **All 5 preconditions CONTINUOUSLY met**

**Result**: Continuous small reward signals, no large gaps, body stays engaged → subjective experience of "time flying," "effortless effort," deep satisfaction.

🟡 **Framework claim**: **Flow = optimal maintenance of all 5 preconditions continuously.** It cannot be forced; it emerges from conditions.

### §10.4 — Why some rewards sustain, others don't

**Sustaining rewards**:
- Embedded in activity that builds chunks base → more future reward possible
- Aligned with deep body-needs (not just surface)
- Opioid-tagged (not relief-based)
- Meaningful in multi-schema cascade
- Examples: passion work, craft mastery, deep relationships

**Non-sustaining rewards**:
- Surface stimulation without chunks building
- Relief-based (diminishing returns)
- Cortisol-tagged
- Isolated (no cascade)
- Examples: endless scrolling, addiction, pure status without meaning

### §10.5 — Opioid vs Relief (critical distinction)

🟡 Framework commits:

**Opioid reward** (positive trajectory):
- Anticipatory + arrival + integration
- Builds on itself
- Sustainable
- Grows capacity for future reward
- Example: finishing a hard project → proud + chunks compiled → can do something harder next time

**Relief reward** (negative-valence reward):
- Only at arrival (threat gone)
- Wears off
- Diminishing returns
- May erode capacity
- Example: procrastinating → deadline stress → rushed completion → relief → but damages self-regulation chunks

**Mapping to prior cases** (P2 §8):
- B2.3 Father Promises Toy Car: opioid reward trajectory (intrinsic motivation preserved)
- B2.2 Forced Punishment Learning: relief reward only — unsustainable

### §10.6 — Burnout mechanism

🟡 Framework prediction:

**Burnout** = extended period where:
- Reward is primarily relief-based (sustained threat)
- Chunks are cortisol-tagged
- Repair deficit accumulates (Cortisol-Baseline §3)
- PFC M2 damage begins
- Feeling layer collapses (can't enjoy even what once worked)
- Self-signal interoception degrades

**Result**: Person cannot enter flow even in previously-loved activities. Reward capacity itself is damaged.

**Recovery**:
- Rest (reduce cortisol load)
- Sleep quality (allow consolidation)
- Re-association of chunks (Paths 1–3 from §9.10)
- Gradual rebuilding of reward sensitivity
- May take 6 months to 2 years for severe burnout

### §10.7 — Diminishing returns (hedonic treadmill)

🟢 Research (Brickman 1978) + 🟡 framework:

**Mechanism**:
- Repeated reward → VTA habituates → P3 delta drops
- New baseline expectation → requires MORE to trigger reward
- Addiction spiral: need more quantity/intensity for the same effect

**Counterintuitive**: Same absolute input level → less subjective reward over time.

**Framework insight**: Explains why:
- Money doesn't bring lasting happiness (habituation to luxuries)
- New purchases lose their appeal
- The same food / same music / same success becomes routine
- Novelty-seeking behavior is universal in humans

**Mitigation**:
- Variety (break habituation via new chunks in cascade)
- Breaks (allow VTA to reset)
- Gratitude practice (deliberately attend to inputs, reset habituation)
- New dimensions of the same activity (deeper expertise in an existing love)

### §10.8 — Peak vs sustained reward — final synthesis

🟡 **Framework commitment**:

**Peak reward** (intense moment):
- Often result of long-accumulated pending + sudden large delta
- Examples: breakthrough, winning, finally getting something long-desired
- **Cannot be sustained** — peak is by definition transient
- Intensity is its characteristic

**Sustained reward**:
- Continuous moderate reward from ongoing activity
- Examples: flow, deep work, meaningful relationships, craft
- **Can be sustained** indefinitely if conditions are maintained
- Integration is its characteristic

**A meaningful life** combines both:
- Occasional peaks (achievements, milestones)
- Sustained flow (daily engaged activities)
- Not either/or

**Framework warning**: Chasing only peaks → hedonic trap (P2 §9). Peaks are not replicable without new pending. Need sustained base + occasional peaks.

## §11 — Open Questions

### §11.1 — Emergent questions from P3 drill

**Q14**: **Can reward sensitivity be measured per-precondition?** Could we diagnose which specific precondition is "broken" in anhedonia or depression?

**Current status**: Limited research on decomposed reward failure. Most studies measure "overall reward sensitivity." Framework predicts decomposition is possible via careful experimental design.

### §11.2 — Chunk tag neurochemistry

**Q15**: **What is the exact molecular mechanism** of chunk association tags? How do opioid vs cortisol associations get "wired in" during compile?

**Current status**: Framework level is a functional model. Exact neurochemistry is underdetermined. Candidates:
- Co-release of neurotransmitters during LTP
- Epigenetic markers
- Different gene expression during compile
- Emerging research direction.

### §11.3 — Minimum re-association dose

**Q16**: **How much positive re-exposure is needed** to shift cortisol-tagged chunks to opioid-tagged? A research-based clinical question.

**Current observation**: Varies by:
- Original tag intensity
- Individual flexibility
- New context quality
- Time available
- Generally months to years for significant shifts

### §11.4 — Flow state trainability

**Q17**: **Can flow state be trained** more directly, or does it only emerge spontaneously from conditions?

**Framework prediction**: Flow emerges. Direct training of flow itself doesn't make sense (it is a state resulting from conditions). But training of **prerequisites** (chunks base, skill, task selection) enables flow.

### §11.5 — Social reward specifics

**Q18**: **How does social approval reward interact with self-schema?** Do different personality types weight social vs internal reward differently?

**Current status**: Research shows individual differences. Framework predicts this correlates with:
- P5 tag patterns (how approval was tagged during development)
- P1 pending configuration (which needs are primary)
- Cultural context (what is considered approval-worthy)

### §11.6 — Hedonic adaptation mitigation

**Q19**: **What is the longest documented sustained peak reward without significant adaptation?** Is a permanent happiness boost achievable?

**Current research**: Brickman shows lottery winners return to baseline. But some studies of meaningful work + community + autonomy (Self-Determination Theory, Deci & Ryan) show more sustained subjective well-being.

**Framework hypothesis**: Sustained reward requires continuous chunks base growth + cascade richness. Pure input-based reward adapts. Growth-based reward doesn't adapt the same way.

### §11.7 — Cross-cultural universality

**Q20**: **Are H10 preconditions universal** across cultures, or culture-shaped?

**Framework position**: Underlying mechanism (neurobiology) is universal. Content (what satisfies each precondition) is culture-specific.
- P1 pending: culture shapes which schemas are typically pending
- P2 chunks: culture determines what chunks are commonly built
- P3 delta: mostly universal
- P4 Goldilocks: culture shapes what counts as "familiar" vs "alien"
- P5 tag: culture shapes the contexts in which chunks get compiled
