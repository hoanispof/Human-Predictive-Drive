## §2 — Dual-Pull Principle (Hardware × Domain)

### §2.1 — Original articulation (§0.5.1 verbatim)

> "Schema (chunks or melody when viewed through the melody lens) is pulled simultaneously by both Hardware and Domain. That is, all schemas at any given moment must run smoothly on hardware (a non-smooth melody is dissonance). And the final outcome humans aim toward is mapping a broader real domain."

This is the **architectural starting point** of the entire drill. The framework has committed to it (🟡) in `Personal-Melody.md` §5 "Two-Axis Tension" — this file formalizes it for dissonance/reward decoding.

### §2.2 — Hardware Pull (internal, conservative, wants smooth)

**Definition** 🟡:

> The pulling force drawing schema toward **body-base comfort** — neurons fire familiar patterns, no recalibration required, no discomfort generated.

**Substrate mechanism**:
- **Hebbian LTP**: neurons that fire together wire together → familiar patterns fire more strongly over time
- **Habituation**: VTA grows accustomed to steady pattern → stops firing → no attention demanded → smooth
- **Metabolic efficiency**: compiled patterns = low energy cost (§3 Chunk.md, Chunk-Practical brain fog analysis)
- **Body-base homeostasis** (L1 Body-Base reframe): body-inputs maintain adaptive baseline — signal stays quiet when pattern = baseline

**Features when hardware pull dominates**:
- Comfort zone, routine, habit, contentment (🟡 Boredom-Analysis §2)
- Cortisol IDLE mode (Cortisol-Baseline §1)
- Personal melody smooth, no dissonance

**Dark side**: Hardware pull too strong → stagnation → domain goes unmapped → "living in the past" → evolution lag (Why-Body-Knows §6 ①: sugar, social media)

**Metaphor** (from Personal-Melody §5):
> "The audience wants to hear THE OLD FAVORITE"

### §2.3 — Domain Pull (external, demands adaptation, maps reality)

**Definition** 🟡:

> The pulling force drawing schema toward **domain reality mapping** — new chunks must be loaded even when body resists, because domain reality does not care whether the melody runs smoothly.

**Substrate mechanism**:
- **Novelty drive** (Novelty.md): genetic drive to explore
- **prediction-delta detection** (PFC-Analysis §8.3): new pattern → VTA fires dopamine → PFC attention
- **Prediction error updating** (Friston/Clark): mismatch with expectation → schema update pressure
- **Evolutionary advantage**: species that fail to map reality do not survive → genes eliminated (Why-Body-Knows v1.1 §3 Layer 1 Darwinian)

**Features when domain pull dominates**:
- Learning, exploration, curiosity, growth
- Cortisol ACTIVE/FOCUSED mode
- Melody builds via arcs (Melody-Arc §2)

**Dark side**: Domain pull too strong → burnout → neural wear → melody noise (§3 Cortisol-Baseline §3 repair deficit)

**Metaphor**:
> "The stage demands A NEW PIECE — the audience isn't ready yet"

### §2.4 — The Tension: Evolutionary FEATURE, Not Bug

**Critical insight** 🟡:

These two forces are NOT a design flaw. This is **architectural tension intentionally evolved**:

- **Hardware only**: organism fails to adapt → domain changes → death
- **Domain only**: organism hyperlearns to burnout → resources exhausted → death
- **BOTH**: oscillation between consolidation (hardware) and exploration (domain) → survive + thrive

→ Like the **immune system**: remembers old pathogens (memory cells) while remaining ready to respond to new ones (naive cells). Neither side can be dropped.

The body's 3 signals map directly onto dual-pull:

| Signal | Direction | Dominant force |
|---|---|---|
| **Body-Reward** | "Good, keep going" | Both aligned → melody builds |
| **Body-Satisfaction** | "Enough, stop" | Hardware met (baseline reached) |
| **Body-Dissonance** | "Not right yet, change" | Hardware OR Domain unmet → tension |

### §2.5 — "Good Melody" 4 Criteria (Personal-Melody §5)

🟡 Framework commit:

> "Good melody" ≠ the melody most pleasant for the body. "Good melody" = a melody that meets **ALL 4 criteria simultaneously**:
>
> ① **Runs smoothly on one's own hardware** (body-base comfortable, plays without strain — person-specific)
> ② **Maps domain accurately** (reflects reality correctly, no pleasant delusions)
> ③ **Maps domain as broadly as possible** (wide coverage = flexibility, adaptable to many situations)
> ④ **Plays sustainably over the long term** (durable, livable, developable in the real world)

**Implication**: Evaluating a schema/melody requires NOT just "does the body find it pleasant?" (① only) or "is it correct?" (② – ③ only). All 4 criteria are needed.

**Examples of incomplete good melodies**:

| Case | ① Hardware | ② Accurate | ③ Broad | ④ Sustainable | Outcome |
|---|---|---|---|---|---|
| Addiction (heroin) | ✅ max | ❌ | ❌ | ❌ | Body reward intensely pleasant but domain punishes |
| Comfort zone | ✅ | ✅ partial | ❌ | ⚠️ stable but narrow | No expansion |
| Workaholism | ❌ burnout | ✅ | ✅ | ❌ neural wear | Domain mapped but body collapses |
| Delusion | ✅ feels right | ❌ wrong | ❌ | ❌ reality clash | Body pleasant + domain punishes (Imagine-Final-Evaluation §4) |
| Passion (sweet spot) | ✅ | ✅ | ✅ | ✅ | Jensen Huang / Einstein cases |

### §2.6 — Body-Base Reward ≠ Domain Reward (critical split)

🟡 Framework commit (Personal-Melody §5):

> **Body-base reward** pays reward when melody is SMOOTH. BUT smooth does NOT necessarily mean domain-correct. Body accuracy ~90% (Why-Body-Knows.md) → 10% CAN BE WRONG.
>
> - Addiction = body reward STRONG → domain PUNISHES (health)
> - Comfort zone = body smooth → domain stagnates (no progress)
>
> **Domain reward** pays TRUE reward (survival, development). Always correct BUT not always pleasant.
>
> - Learning something hard = body uncomfortable → domain rewards (new skill)
> - Waking early to exercise = body resists → domain rewards (health)

**Why this split matters for the drill**: All "reward cases" in the drill (C1.1–C1.7, C2.1–C2.5, C3, C4) must be analyzed on **2 separate axes**:
- What does body-base reward pay? (opioid, smooth melody preview)
- What does domain reward pay? (skill, resource, status, survival)

Einstein breakthrough (C1.5) = both aligned. Heroin user = body only. Workaholism = domain only. Right car (C2.1) = both aligned, person-specific.

**The optimum = both simultaneously** = "Passion" = complete Imagine-Final (Imagine-Final §1 "body wants × domain confirms").

### §2.7 — "Passion" = 2 Forces Aligned

🟡 Framework:
> Melody that runs smoothly (body enjoys) AND maps domain well (domain accepts) = "I LOVE doing what the world ALSO NEEDS"

Cases the framework cites as passion achieved:
- **Jensen Huang + GPU**: 30 years consistent. Hardware (curiosity about graphics) + Domain (AI boom accelerated GPU importance) aligned from early on
- **Einstein + physics**: Hardware (body felt "Newton physics lacks coherence") + Domain (reality confirms relativity via experiments)

Cases the framework cites as failed alignment:
- **Child forced to become doctor**: Parent's Imagine-Final (externally domain-driven) + body doesn't feel it → empty "skilled but hates it"
- **Earning money "because you must"**: Domain goal (money) + body doesn't find money accumulation meaningful → hedonic trap (B4.1, drill P2 §9)

> **Externalization note**: Dual-Pull does not only exist in the brain — it PROPAGATES into every system humans create (software, law, medicine, organizations). Same tension, different form because domain differs. See **Collective-Body.md §3.5** — Dual-Pull Propagation.

## §3 — Interface Loop (6-Step Formalization)

### §3.1 — Original articulation (§0.5.1 verbatim)

> "Interaction interface: domain => body => body feedback (PFC processes more carefully or unconscious) => schema update => Action to domain => domain acts on body => body feedback reward/dissonance => ..."

This is a **continuous cycle** — no start or end point. It describes the continuous state of human ↔ world interaction.

The framework has partially committed to this through multiple files:
- `Why-Body-Knows.md` §6 "Complete cycle"
- `Chunk.md` §3 (input → body check → compile → action)
- `Anchor-Schema.md` §1 6-step flow
- `Learning-Cycle.md` §3 8-stage cycle (for specific learning cases)

This file **consolidates** into a generic 6-step loop.

### §3.2 — 6-Step Loop Diagram

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  Step 1: DOMAIN → BODY sensory                                  │
│     (external reality → sensory organs)                         │
│                             ↓                                   │
│  Step 2: BODY → FEEDBACK                                        │
│     (unconscious 95% OR PFC gateway 5%)                         │
│     Body check: match known patterns? coherence?                │
│     Output: body signal (reward / dissonance / neutral)         │
│                             ↓                                   │
│  Step 3: FEEDBACK → SCHEMA update                               │
│     (compile new chunks / modify existing / reconsolidate)      │
│     Hebbian LTP + sleep consolidation                           │
│                             ↓                                   │
│  Step 4: SCHEMA → ACTION                                        │
│     (compiled schemas fire → motor output)                      │
│     PFC executes or unconscious auto                            │
│                             ↓                                   │
│  Step 5: ACTION → DOMAIN                                        │
│     (body physically affects world)                             │
│                             ↓                                   │
│  Step 6: DOMAIN → BODY feedback                                 │
│     (reality responds → new sensory input)                      │
│     Body reward (if action worked) / dissonance (if not)        │
│                             ↓                                   │
│                        [LOOP back to Step 1]                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### §3.3 — Step 1: Domain → Body Sensory

**Mechanism**:
- External reality (light, sound, touch, temperature, smell, taste, internal state) → sensory organs → neural transduction → cortex
- **Exteroception** (5 channels): Vision, Audition, Olfaction, Gustation, Tactile
- **Proprioception** (3 channels): Body position, Vestibular, Kinesthetic
- **Interoception** (9 channels, per Body-Input-Enumeration §4): Thermoreception, Nociception, Respiratory, Cardiovascular, Visceral, Metabolic, Hormonal-sensed, Sleep/Circadian, **Self-signal interoception §4.9**

**Key point**: Step 1 is a **boundary-crossing** — domain reality must pass through the body interface. There is NO "direct access" from PFC to reality. All knowledge about the domain is filtered through body sensors.

🟢 Established: Perception is reconstructive, not a direct copy (Helmholtz, Gregory, modern predictive processing).

### §3.4 — Step 2: Body → Feedback

**Mechanism**:
- Sensory signal → unconscious processing (95%) or PFC gateway (5%)
- Body **checks** signal: does it match known patterns? coherence with base chunks? fit with Imagine-Final expectation?
- Output: body signal
  - **Reward** (opioid) if it fits body-need (§5 PFC-Analysis §8.3 step 5)
  - **Dissonance** (signal "not right yet") if mismatch
  - **Neutral** if pattern = baseline, no delta

**Who handles it**:
- **Unconscious (95%)**: compiled schemas auto-check, prediction-delta detection, body-base simulation
- **PFC (5%)**: when dissonance signal is strong enough → PFC engages (Imagine-Final §1 "PFC reactive, body calls")

**Why the split**: PFC bandwidth is too small for the full signal volume — unconscious handles background scan, PFC handles novelty/conflict/decisions only.

🟡 Framework (Chunk.md §3): "Unconscious = primary operator (~95%), PFC = secondary operator (~5%)".

### §3.5 — Step 3: Feedback → Schema Update

**Mechanism**:
- Body signal → triggers schema modification
- 4 compile pathways (Chunk.md §2):
  - **Repetition**: slow + stable (Hebbian LTP, Bliss & Lømo 1973 🟢)
  - **Emotional peak**: fast + deep (amygdala + NE, Brown & Kulik 1977 🟢)
  - **Multi-modal**: strong (wires across multiple cortical areas)
  - **Sleep consolidation**: offline integration (Walker 2017 🟢)
- Reconsolidation window: recall → temporarily unstable → can modify → re-compile (Nader 2000 🟢)

**Chunk.md §2 key claim** 🟡: "No one can delete a chunk. Chunks only STRENGTHEN / WEAKEN / MODIFY — they do NOT delete."

**Implication for the drill**: Trauma loop (B3), hedonic trap (B4), status shift (C5) — all involve the schema update mechanism. The old schema cannot be "reset"; only a new schema compiled STRONGLY ENOUGH to override.

### §3.6 — Step 4: Schema → Action

**Mechanism**:
- Compiled schemas fire → motor output
- 2 paths:
  - **Unconscious auto**: compiled skill (driving, walking, eating) — PFC not involved
  - **PFC direct**: novel situation, decision, deliberate action — PFC spotlight + top-down boost

**Framework** (Anchor-Schema §1 step ②): "Most of life: step ② → anchor already exists → skip to ⑤. Only when ② fails → THEN PFC is called."

🟡 ~80% of behavior = compiled schemas auto-firing (Anchor-Schema §3 comparison table).

### §3.7 — Step 5: Action → Domain

**Mechanism**:
- Body movement → physical effect on world
- Speech → social effect
- Tool use → environmental modification

**Key point**: Step 5 is the **second boundary-crossing** — body → reality. Action ends on the body side; effect occurs on the domain side.

### §3.8 — Step 6: Domain → Body Feedback (loop closure)

**Mechanism**:
- Domain reality responds to action → new sensory input via Step 1
- **Reward signal** if action achieved intended effect
- **Dissonance signal** if action failed / produced unintended effect
- **Partial signal** if mixed result

**Loop closure**: Step 6 triggers Step 1 again for a new cycle. The cycle never stops during waking life.

**Example (simple)**:
1. Hungry (Step 6 domain feedback: body-state delta)
2. Body signals hunger (Step 2)
3. Schema "find food" fires (Step 3 retrieval)
4. Walk to kitchen, open fridge (Step 4–5)
5. Eat food (Step 5)
6. Body satisfied (Step 6) → loop continues

**Example (complex — Einstein C1.5)**:
1. Read Newtonian physics (Step 1–6 many iterations)
2. Body dissonance "not coherent" (Step 2)
3. Build chunks about physics (Step 3, years)
4. Discuss with Grossman (Step 4–5)
5. Domain feedback: new formula matches reality (Step 6 external confirm)
6. Body reward extremely strong (Step 2 back)
7. Loop continues to breakthrough

### §3.9 — Loop timing + who handles each step

| Step | Timing | Primary handler | PFC role |
|---|---|---|---|
| 1 Domain → Body | ms (sensory latency) | Body sensors | None |
| 2 Body → Feedback | ms–sec | Unconscious 95% | 5% (when dissonance significant) |
| 3 Feedback → Schema | sec–days (immediate compile + sleep consolidation) | Unconscious (compile), PFC (direct create) | Create new chunks, reconsolidate recall |
| 4 Schema → Action | ms–sec | Unconscious auto 80%, PFC 20% | Choose + execute novel action |
| 5 Action → Domain | varies (physical) | Body motor | Monitor execution |
| 6 Domain → Body | varies (reality response) | Body sensors | Observe outcome |

**Key insight**: PFC is a **gateway**, not a **controller**. PFC observes + directs attention + makes novel decisions, but unconscious runs the loop 95% of the time.

### §3.10 — Why the loop cannot be short-circuited

🟡 Framework claim:

> Every step in the loop is **architecturally necessary**. Cannot be skipped or compressed.

**Examples of attempted short-circuits that fail**:

- **"Learn from books, no experience needed"** → Skips Steps 5–6 (no action, no domain feedback)
  → Fake explicit knowledge (verbal-only chunks), body does not confirm → fragile, unusable in practice (Somatic-Articulation-Loop §1.3)

- **"Only need PFC thinking, not body"** → Skips Step 2 (no body signal)
  → No direction → existential boredom / decision paralysis (Imagine-Final §1 "dormant when harmony, fires when dissonance")

- **"Only need body feeling, no verification"** → Skips Steps 3–5 (no schema update, no action test)
  → Body rewards coherence (not truth) → can be wrong domain (Why-Body-Knows §6 ①)

**Implication for the drill**: Cases like trauma loop (B3), hedonic trap (B4) typically involve **loop breakdown** — one step encounters a problem → entire loop degrades → dissonance accumulates.

## §4 — Why Body "Knows" (2-Layer Calibration)

### §4.1 — The meta-question

> The framework says "listen to the body." But **why is the body trustworthy**?

Without answering this → the framework rests on something unexplained. Like "trust the GPS" without knowing why GPS knows the route.

`Why-Body-Knows.md` §1 poses 3 core questions; this file consolidates the answers.

### §4.2 — Layer 1: Darwinian Selection (millions of years, genetically wired)

🟢 Research-backed:

> Natural selection = "trial-error at MASSIVE SCALE" — millions of organisms × millions of years × millions of trials. Genes that "reward correct patterns" propagate. Genes that "reward incorrect patterns" are eliminated (their owners died).

**Examples of wired-in patterns**:
- Fire hot → pain → avoid → gene "fire = pain" propagates
- Ripe fruit → sweet → nutrition → gene "sweet = reward" propagates
- Snake → fear → avoid → gene "snake = danger" propagates

**Framework interpretation** 🟡:
> Body reward system = "survivor's GPS" — tested across millions of actual journeys, retaining only the correct routes.

**Limit**: Evolution calibrates SLOWLY (thousands–millions of years). Environment changes FAST (a few hundred years for industrial society) → **evolution lag** (§4.6 below).

### §4.3 — Layer 2 Path 2a: Domain Contact (lifetime experience)

🟢 Established + 🟡 framework extension:

> Each person CONTINUES calibrating their body against the CURRENT domain. Each experience = 1 cycle of "try → domain feedback → body calibrates."

**Mechanism**: accumulated chunks via DIRECT EXPERIENCE → body evaluates with increasing granularity:
- Little experience: body evaluates "good/bad" (binary, coarse)
- More experience: body evaluates "this part good, that part bad" (detailed)
- Extensive: body evaluates "if changed this way → better" (simulate variants)

→ "Expert" = body has more measuring instruments, not "smarter."

**Relation to body-feedback vs feeling** (§5 below): Domain contact = substrate for feeling layer compilation. Chunks from experience = material PFC uses to label body states.

### §4.4 — Layer 2 Path 2b: Trust-Injected — Culture (generational transmission)

🟡 Framework:

> Knowledge accumulated across generations — transmitted through oral tradition + books + education. Shared calibration: one person tries → everyone learns.
> ⚠️ Culture = INPUT SOURCE for Layer 2 Hebbian, NOT a separate layer.
> Same compile mechanism, different entry route (via TRUST AMPLIFIER rather than direct domain contact).

**Examples**:
- Parents teach "don't touch fire" → child doesn't need to try it (trust parent → compile)
- School teaches "gravity = F=mg" → student doesn't need to drop a stone 1,000 times
- Books teach "history of war → lessons" → no need to experience it

**Mechanism**: External inject (Anchor-Schema §3 source 4) → trust amplifier → Hebbian compile → body-usable.

**Limit**: External inject can compile INCORRECTLY if source is wrong → children believe "Santa Claus is real," adults believe "rich = happy." Trust-injected can transmit both truth and error.

### §4.5 — Layer 2 Path 2b: Trust-Injected — AI Era (unlimited chunks)

🟡 Framework claim:

> AI = amplifies Path 2b enormously. Contains knowledge of all humanity, accessible instantly. Chunks transmit speed ×1000 compared to books.
> ⚠️ AI = one additional Trust-Injected SOURCE, same Hebbian mechanism as culture, books, teachers.

**Mechanism**: Human asks AI → receives chunks → trust amplifier → body check → Hebbian compile. AI = unlimited + instant chunk source.

**Limit**: AI only provides chunks — body STILL must check. AI can be wrong → domain contact (Path 2a) is always the final arbiter.

**Implication**: AI does not replace Layer 1 or Path 2a. It **amplifies** the speed + breadth of Path 2b specifically.

### §4.6 — Coherence Check ≠ Truth Check

🟡 Critical distinction (Why-Body-Knows §3):

> Body checks **COHERENCE** (does the new pattern match known patterns?), NOT **TRUTH** (is the pattern correct in the real domain?).
>
> - Coherence matches → opioid → "pleasant / understood / good"
> - Doesn't match → mild cortisol → "strange / uncomfortable / wrong"
> - Unknown → dopamine → "curious / explore more"

**Why coherence ≈ truth (most of the time)**:
- 2-layer calibration (Darwinian + Hebbian) has tested coherence against reality millions of times
- Most coherence = truth (because base chunks are already correct)
- Body accuracy ~90% 🟡

**Why coherence ≠ truth (the 10% gap)**:
- Flat Earth: "looks flat straight ahead" → coherent with visual experience → body OK → WRONG in real domain
- Addiction: body reward strong → unconscious OK → but domain punishes
- Limerence: body intensely pleasant → "perfect" → domain not yet tested over time

**Key claim**: Body rewards **COHERENCE**, not **TRUTH**. External checking is necessary for the 10%.

### §4.7 — 3 Failure Modes (Why-Body-Knows §6)

🟡 Framework commits to 3 ways body can err:

**① Evolution Lag** — body reward system correct for old environment, wrong for new:
- Sugar: rare in ancestral environment → reward "sweet = precious" → modern easy access → sugar addiction
- Social media: ancestral social approval rare → reward "likes" → modern unlimited → scroll addiction
- Fat/Salt: ancestral scarcity → reward → modern junk food → obesity epidemic

**② Wrong Base Chunks** — coherence matches but foundation is wrong:
- Flat Earth (classic)
- "Cortisol = stress hormone" (widespread misconception, Cortisol-Baseline §2.5 corrects this)
- Outdated scientific consensus

**③ Schema Override** — knows it's wrong but still follows:
- Addiction: body "knows" sugar is bad but schema "sweet = reward" compiled too strongly → overrides
- Procrastination: PFC knows work is needed but compiled "scroll = reward" overrides
- Addiction generally: Schema override body check

**Implication for the drill**: Cases in P2 (dissonance) and P3 (reward) must consider all 3 failure modes. Trauma loop (B3) involves ② + ③. Hedonic trap (B4) involves ① + baseline shift.

## §5 — ⭐ Body-Feedback vs Feeling Architecture (CORE)

> **This is the LARGEST section of P1 Foundation.** Foundation for the entire feeling system's complexity. Directly addresses the architectural distinction.

### §5.1 — Definition distinction

🟡 Framework commit (consolidated from Somatic-Articulation-Loop §1 + Theme A §2.4 + Chunk.md §3):

| | **Body-Feedback layer** | **Feeling layer** |
|---|---|---|
| **What** | Raw body signal | PFC observation of integrated signals + chunk-based labeling |
| **Latency** | Automatic, pre-verbal | Requires chunks + observation time |
| **Substrate** | Body sensors + insula + ACC upstream integration | PFC + working memory + compiled labels |
| **Who does it** | Unconscious 100% | Unconscious integration → PFC observation interface |
| **Developmental** | Present from birth | Builds over years via mirroring + chunks |
| **Verbal** | Pre-verbal (no words needed) | Verbal (language emerges) |
| **Training** | Genetic + maturation | Mirroring + chunks + meditation |
| **Layer in 7-layer model** | Feel-RawSignals — Feel-Integration | Feel-Consciousification — Feel-Explanation |

**Critical**: These are NOT 2 separate systems. They are a **gradient** — body-feedback is the substrate, feeling layer builds on top via chunks accumulation.

### §5.2 — 3 States of Knowledge (Somatic-Articulation-Loop §1)

🟢 + 🟡 Framework commit:

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  IMPLICIT ──────── TRANSITIONAL ──────── EXPLICIT           │
│  (body only)       (felt sense)         (verbal + body)     │
│                                                             │
│  "don't know"       "something is        "I know that..."   │
│  but has             there" but          and can explain    │
│  compiled            can't phrase it                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### §5.2.1 — IMPLICIT ("Body knows, PFC doesn't know it knows")

Knowledge compiled into body (schemas, motor programs, emotional patterns, chunks) but PFC has no direct verbal access.

**Examples** (from Somatic-Articulation-Loop §1.1):
- **Motor**: riding a bicycle — can do it but cannot explain "how to maintain balance"
- **Language**: native speaker uses correct grammar but cannot explain the rules
- **Social**: "walked into the room and felt the heavy atmosphere" — body already processed hundreds of micro-signals
- **Emotional**: "met someone and didn't trust them, didn't know why"
- **Aesthetic**: "saw a beautiful painting, cannot explain exactly why"
- **Expertise**: chess master "sees" a good move without calculating

🟢 Research support:
- Implicit learning (Reber 1967+)
- Tacit knowledge (Polanyi 1958: "We can know more than we can tell")
- Expertise chunks (Chase & Simon 1973)
- Somatic markers (Damasio 1996)

#### §5.2.2 — TRANSITIONAL ("Felt sense" — Gendlin 1978)

Body-knowledge begins surfacing into PFC awareness, but without words yet. PFC senses "something" (coherence signal) but doesn't know "what."

**Phenomenology** (Somatic-Articulation-Loop §1.2):
- **Vague** (unclear, not sharp)
- **Coherent** (there is something unified, not random noise)
- **Bodied** (felt somewhere — chest, stomach, shoulder, throat)
- **Resistant to labels** ("it's not joy, but it's not sadness either")
- **Directional** (has direction — positive/negative, open/tight)
- **Unstable if not articulated** — dissipates if not captured
- **Satisfying when articulated correctly** — body "settles"

**Examples**:
- Creative impasse: "something is wrong" — don't yet know what
- Decision ambivalence: "offer looks good on paper but body is hesitant"
- Relationship dissonance: "not comfortable around that person, don't know why"

🟢 Gendlin's research at Rogers lab (Chicago): clients with felt sense access had significantly better therapy outcomes. Measurable "felt shift" — galvanic skin response changes during successful Focusing sessions.

#### §5.2.3 — EXPLICIT ("Verbal + body both know")

Knowledge that can be verbalized, communicated, taught, argued about. PFC has access; body still has signal. **BOTH** know.

🟡 **Critical distinction** — TRUE EXPLICIT vs FAKE EXPLICIT:

| | TRUE EXPLICIT | FAKE EXPLICIT (verbal-only) |
|---|---|---|
| PFC | Has words | Has words |
| Body | Has corresponding signal | No signal |
| Applicable | Usable in real action | Theory only |
| Stable | Consistent over time | Fragile, easily lost when PFC is busy |
| Example | "I love my mother" (body warm when thinking of her) | "Must love others" (PFC label, no body feel) |

**Examples of fake explicit** (Somatic-Articulation-Loop §1.3):
- Child taught "must love others" but never experienced it → verbal exists, felt sense missing → cannot apply in real emotional moments
- Student memorizes math formulas but doesn't understand → symbolic explicit, conceptual implicit missing → cannot apply to new problems
- Reading self-help books → knows all frameworks but can't apply them → "knowing but unable to do" pattern

→ Fake explicit = chunks from source ④ External Inject (Anchor-Schema §3) without filling ①②③
→ True explicit = chunks through FULL loop: body experience → implicit → transitional felt sense → articulation → explicit with body-confirm

### §5.3 — 7-Layer Feeling Observation Model (Theme A §2.4+§2.7)

🟡 Framework commit (Feeling.md + Theme A):

> Feeling = what PFC observes from unconscious processing. PFC has NO direct access to raw signals. Integration happens UPSTREAM (insula + ACC + VMPFC) before PFC sees.

**7-layer gradient** (same mechanism, different signal strength):

| Layer | Clarity | Description | Clear example (hunger) | Fuzzy example |
|---|---|---|---|---|
| **Feel-RawSignals** | 100% body truth | Pre-integration signal | Gastric ghrelin signal | Schema partial fire |
| **Feel-Integration** | 95% | Upstream integration (insula + ACC + VMPFC) | Multi-channel stomach+blood glucose | Multi-schema integration |
| **Feel-Consciousification** | 90% | Signal strong enough to break into awareness | "Clearly hungry" | "Feeling like something..." |
| **Feel-Observation** | 85% | PFC notices signal exists | "I am hungry" | "I'm feeling something" |
| **Feel-Location** | 70–90% | PFC localizes signal | "Hungry in the stomach" | "In the chest but vague" |
| **Feel-Labeling** | 40–80% | PFC applies verbal label via chunks | "Hungry" | "Uneasy" |
| **Feel-Explanation** | 20–70% | PFC explains via narrative + chunks | "Because I haven't eaten breakfast" | "Because... don't know" |

**Key claims** 🟡:

**Claim 1**: Body-feedback layer = Feel-RawSignals — Feel-Integration (pre-PFC). Feeling layer = Feel-Consciousification — Feel-Explanation (PFC observation).

**Claim 2**: **Clear signals** (hunger, sharp pain) reach Feel-Location — Feel-Labeling reliably. **Fuzzy signals** stay at Feel-Consciousification — Feel-Observation. These are NOT 2 different mechanisms — **same observation interface seeing signals of different strength**.

**Claim 3**: PFC has NO direct access to Feel-RawSignals raw. Integration upstream → Feel-Consciousification surfaces. PFC only observes Feel-Consciousification+ output.

**Claim 4** (Theme A §2.5): PFC performs 3 functions (OBSERVE + LABEL + CHOOSE), does NOT generate feelings:
- OBSERVE (read integrated upstream signals)
- LABEL (via chunks — error-prone at Feel-Labeling — Feel-Explanation)
- CHOOSE (executive function between competing signals)
- NOT: execute interoception, run schemas, generate feelings, access raw body directly

### §5.4 — Self-Signal Interoception KEYSTONE (Body-Input-Enumeration §4.9)

🟡 Framework commit — ARCHITECTURAL KEYSTONE:

> **Self-signal interoception = body's capacity to READ its own internal state.** Meta-level sensing. Without it, other inputs fire but body "cannot hear itself."
>
> = Prerequisite for feeling layer development, regardless of how many other chunks exist.

**Substrate**:
- **Anterior insula (AI)**: primary interoceptive hub (Craig 2002, 2009 🟢)
- **Anterior cingulate cortex (ACC)**: affective-motivational dimension, valence assignment
- **VMPFC**: integration of bodily feelings into decisions (Damasio somatic marker hypothesis 🟢)
- **Posterior insula**: first-order interoceptive representation
- **Predictive interoception** (Seth 2013, Barrett 2017 🟢): brain continuously generates predictions about body state, matches against actual

**3 levels of awareness** (Garfinkel et al. 2015 🟢):
1. **Interoceptive accuracy**: objective performance on heartbeat/respiratory detection tasks
2. **Interoceptive sensibility**: subjective self-report of body awareness
3. **Interoceptive awareness** (metacognitive): match between accuracy + sensibility = "knowing what you know about your body"

All 3 dimensions **trainable + variable + clinically relevant**.

**Why this is keystone**:
- Body signals fire from §4.1–§4.8 (thermoreception, nociception, cardiovascular, metabolic,...)
- BUT if §4.9 reading capacity is compromised → PFC cannot access signal → **silent deficit**
- Silent deficits accumulate without PFC awareness → feeling layer cannot develop
- → Architectural prerequisite, not a nice-to-have

**Compromise pathways** (§4.9 (c)):

🟢 **Clinical alexithymia** (Bird & Cook 2013): inability to identify + describe emotions — documented condition affecting **~10% of population**. Strong correlation with:
- Autism (Bird research: alexithymia, not autism per se, drives emotional difficulties)
- Depression
- Somatic symptom disorders
- Eating disorders (cannot distinguish hunger from emotional states)

🟢 **Dissociation**: trauma adaptation, disconnection from body signals. Protective acutely, damaging chronically.

🟢 **Trauma-induced suppression**: body signals become dangerous (PTSD, complex trauma) → suppression learned, "body = enemy territory" (van der Kolk "The Body Keeps the Score")

🟡 **Chronic codependency**: constant attunement to others' states → own state awareness atrophies

🟡 **Overprotective parenting deprivation**: children whose states are always intercepted by others → don't develop their own interoceptive reading → alexithymic adults

🟡 **Modern screen culture**: constant external attention → minimal self-attention → atrophy of interoceptive capacity at population level

**Rebuild pathways** (§4.9 (d)):

🟢 **Meta-cognitive body awareness** (mindfulness practices): Farb research — measurable improvements in interoceptive accuracy

🟢 **Meditation documented effects**: Vipassana, MBSR improve insula thickness + interoceptive accuracy. Hölzel 2011: 8-week MBSR → measurable insula changes on fMRI

🟢 **Gendlin Focusing technique**: specific technique for accessing felt sense — proto-verbal body knowledge

🟢 **Somatic therapy** (SE, sensorimotor psychotherapy): trauma-focused body-based therapy rebuilds interoceptive connection

🟢 **Contemplative traditions**: Buddhist body scan, yoga, Tai Chi — explicit interoceptive training methods

**Developmental origin** (§4.9 (b)):

🟡 **Caregiver mirroring**: infants learn interoceptive awareness through being mirrored — caregiver labels baby's state ("you're hungry," "you're tired," "that hurt") → baby learns to read own states.

🟡 **Emotional labeling culture**: languages with rich emotional vocabulary support interoceptive development.

🟢 **Secure attachment**: foundation for interoceptive trust (Fonagy mentalizing research).

**→ Critical claim**: Self-signal interoception is **developmentally scaffolded** by caregiver mirroring during a critical period in infancy. Adult training can partially compensate but cannot fully replace early scaffolding.

### §5.5 — Developmental Gradient (Examples Mapped)

🟡 Verbatim examples (§0.5.6) mapped to framework:

#### §5.5.1 — Urination gradient

| Stage | Body-feedback state | Feeling layer state | 7-layer mapping |
|---|---|---|---|
| Newborn | Bladder full → auto urinate | None (no verbal concept) | Feel-RawSignals — Feel-Integration only (raw motor reflex) |
| Slightly older child | Post-hoc notice "just urinated" | Emerging Feel-Consciousification ("something happened") | Feel-Consciousification emerges |
| Child who can call out | Pre-signal "about to urinate" → calls caregiver | Feel-Observation — Feel-Location ("I need to go") + Feel-Labeling ("bathroom") | Feel-Observation — Feel-Labeling compiled |
| Adult | Full awareness, can hold, can plan | Feel-Explanation ("2 hours after drinking water I'll need to go") | Full Feel-RawSignals — Feel-Explanation |

**Mechanism**: Chunks accumulate via repeated experience + language (caregiver labeling: "go to the bathroom now"). Without caregiver mirroring, Feel-Consciousification+ develops more slowly or incompletely.

#### §5.5.2 — Pain gradient

| Stage | Behavior | Framework explanation |
|---|---|---|
| Newborn | Leg hurts → general crying | Body-feedback Feel-RawSignals — Feel-Integration only. No localization. Pain = generalized distress signal. |
| Older child | "It hurts here" → points to location | Feel-Location compiled via caregiver prompts ("where does it hurt?") |
| Verbal child | "I fell and hurt my knee" | Feel-Labeling — Feel-Explanation (label + explain) via chunks about body parts + causation |
| Adult | "Lower back pain on the left, probably from sitting wrong yesterday" | Full Feel-Explanation with causal chunks + predictive |

**Critical**: Same raw pain signal. Body-feedback identical. Feeling layer resolution dramatically different because chunks base differs.

#### §5.5.3 — Discomfort differentiation gradient

| Stage | PFC access | Example |
|---|---|---|
| Young child | General "uncomfortable" | Cry without knowing cause |
| Older child | Differentiated causes | "Uncomfortable when sick" / "uncomfortable when it's sunny" / "uncomfortable with hard math problems" |
| Adult | Fine-grained attribution | "Dissonance when code won't compile" ≠ "dissonance when losing someone" (framework-level distinction) |

**Mechanism**: Accumulated chunks allow PFC to label body signals with increasing precision. Dissonance signal itself unchanged — PFC resolution improves.

#### §5.5.4 — Sadness gradient (meta-emotion)

| Stage | Awareness | Example |
|---|---|---|
| New adult | "Sad about something" | Mood dip, no specific cause identified |
| Adult with chunks | "Sad because partner didn't meet expectations" / "sad because manager's assessment" / "sad because unresolved problem" | Specific attribution via chunks about relationships, work, problems |

**Framework insight**: Same body dissonance signal. Attribution differs based on accumulated chunks. **Adult without rich emotional chunks** (alexithymia-spectrum) = stuck at "something is wrong" without resolution.

#### §5.5.5 — Meta-principle from gradient

🟡 **Body-feedback is INVARIANT across developmental stages** (raw signal doesn't change much — same ghrelin, same nociceptors, same cortisol).

🟡 **Feeling layer is TRAINABLE** — through 3 pathways:
1. **Caregiver mirroring** (infancy critical period) — scaffolds Feel-Consciousification — Feel-Observation emergence
2. **Chunks accumulation** (lifetime) — populates Feel-Location — Feel-Explanation labels
3. **Meditation/somatic training** (adult) — refines access to Feel-Consciousification — Feel-Observation (felt sense)

🟡 **Critical implication**: "Not feeling" ≠ "no body signal." Body ALWAYS fires signals. The question is whether PFC can OBSERVE them (Feel-Consciousification+) and LABEL them (Feel-Labeling+).

→ **Alexithymia** = feeling layer underdeveloped despite body-feedback intact. Not "no emotions" — emotions present at Feel-RawSignals — Feel-Integration, absent at Feel-Observation — Feel-Explanation.

### §5.6 — Training Pathways (Feeling Layer is Trainable)

🟡 Framework claims 6 pathways:

**① Caregiver mirroring** (infancy, critical period):

**④ Somatic therapy** (trauma recovery):

- Trauma-focused body-based approaches

**⑤ Gendlin Focusing** (specific technique):

## §6 — Unconscious vs PFC Division of Responsibility (Chunk.md §3)

### §6.1 — The architectural split

🟡 Framework commit (Chunk.md §3):

> Chunk = DATA. **2 operators** work with the same database:
> - **Unconscious = PRIMARY OPERATOR** (~95% of processing)
> - **PFC = SECONDARY OPERATOR** (~5%, but determines direction)

### §6.2 — What the Unconscious CAN DO (from Chunk.md §3)

- **Detect**: body-base state (current state)
- **Compile**: patterns from experience (match/mismatch)
- **Run**: compiled schemas automatically
- **Create Imagine-Final**: expectation patterns at every scale (micro → long-term)
- **Check Imagine-Final vs body-base match**: "does reality match expectation?"
- **Dispense reward/punishment**: opioid when match, cortisol when mismatch

**→ Unconscious is good at**: "What does the body feel RIGHT NOW?" + "Is it going in the RIGHT DIRECTION?" — answers IMMEDIATELY, ACCURATELY — at the **body-base** level.

### §6.3 — What the Unconscious CANNOT DO

- **Check domain accuracy**: "is this smooth state actually correct for the domain?"
- **Simulate future**: "what will things look like in 5 years?"
- **Compare with memory**: "last time this pattern appeared → poor outcome"

**→ Unconscious ONLY measures body-base** → does NOT measure domain accuracy.

**Implication**: If body is pleasant → unconscious says "OK" → **EVEN IF the domain could be WRONG**.

**Examples** (Chunk.md §3):
- Addiction = body pleasant → unconscious "OK" → domain punishes
- Limerence = body intensely pleasant → unconscious "perfect" → domain not yet tested
- Comfort zone = body smooth → unconscious OK → domain stagnates

### §6.4 — What PFC CAN DO (unconscious cannot)

- **Simulate future**: "if this continues → what outcome?"
- **Compare with memory**: "last time this pattern appeared → poor"
- **Check domain accuracy**: "body likes this BUT is it actually sustainable in reality?"
- **Check Imagine-Final domain**: "body wants this direction → does the domain permit it?"
- **Choose between competing Imagine-Finals**: short-term vs long-term
- **Create new evaluative chunks** → load into unconscious → UPDATE database

**→ PFC is THE ONLY SYSTEM** that can verify whether Imagine-Final is feasible + domain-correct.

### §6.5 — What PFC CANNOT DO (unconscious can)

- **Feel directly**: PFC does not sense — it must RECEIVE from body
- **Process 95% background**: PFC bandwidth too small
- **Auto-compile**: PFC only drafts → unconscious is what truly compiles

### §6.6 — Why BOTH are needed

🟡 Critical architectural claim:

> Unconscious is sufficient for animals (simple domain, body-base well-calibrated via evolution). Humans evolved PFC TO COMPENSATE for the 10% where body-base can be wrong. PFC = the domain checker that unconscious lacks.

**Consequence when one side is lost**:

**PFC switched off** (alcohol, drugs, extreme stress, limerence):
- Loses the only domain checker
- Body-base unchecked → can follow wrong direction
- Examples: limerence, addiction, comfort zone, impulsive decisions

**Unconscious switched off** (impossible in practice, but hypothetical):
- PFC alone cannot feel → no sense of what is wanted
- No compiled skills → everything at slow PFC pace → not viable
- Animals suggest → cognition without self-aware feeling system

**Balance when both function well** = "good melody" (§2.5) = body pleasant + domain correct.

**Implication for the drill**:
- Trauma loop (B3): PFC M2 damage → unconscious drafts without quality → feeling layer degrades
- Hedonic trap (B4): PFC pursues domain goal (money) → unconscious body-base drifts → mismatch detected too late
- Car paradox (C2): different chunks base → same object → unconscious checks body-base differently → different reward
- Schema update latency (C5): PFC receives positive information → unconscious hasn't updated compiled self-schema → coherence gap

## §7 — Architectural Claim: Schema Serves Body-Base

### §7.1 — The claim

🟡 Framework commit (PFC-Analysis §8.3 step 5 + Architecture v7.5):

> **Schema serves body-base.** Body-base is the **SOURCE** of reward/dissonance. Reward = **evaluation function** — body decides which schemas are worth keeping.

**NOT**:
- ❌ "Schema drives behavior, body is output"
- ❌ "PFC decides, body follows"
- ❌ "Rational agent with body as constraint"

**IS**:
- ✅ "Body drives, schemas serve"
- ✅ "Unconscious primary, PFC observer + director"
- ✅ "Embodied agent with PFC as extended capability"

### §7.2 — VTA/Opioid split confirms

PFC-Analysis §8.3 7-step loop step 5:
> "PFC receives chunk → integrates into workspace → PFC sends chunk DOWN to body-base: 'simulate → what does body feel?' Body-base simulates: Chunk MATCHES body-need → **opioid release → REAL reward**. Chunk DOESN'T match → body 'meh' → PFC discards."

**Key claim** 🟡:
- **Dopamine** (VTA) = signal "something has changed" (**doorbell**)
- **Opioid** (body-base) = reward "this has real value" (**gift behind the door**)

VTA detects delta → alerts PFC → PFC checks chunk → sends to body → body verifies. **Body is final arbiter of value**.

### §7.3 — Implication: Reward is PERSONALIZED

🟡 Corollary:

> Schema serves one's OWN body-base. Different person → different body-base state → different reward for same stimulus.

**Example** (Car paradox C2):
- Person A: body-need "want a car" pending → sees beautiful car → fills body-need → opioid
- Person A: body-need "want a car" → motorbike doesn't fit → no opioid
- Person A: no body-need for airplane → no reward
- Person A: no body-need for Van Gogh → no reward
- Person B: body-need "vintage car with historical value" pending → vintage car → opioid
- Person B: doesn't see "new model car" filling any body-need

→ **Reward = PERSONALIZED function** of (body-need × chunks base × schema pending). Drill P3 §5 analyzes the 5-scenario matrix.

### §7.4 — Schema update requires body-base confirmation

🟡 Framework commit:
> Schema update (Step 3 in interface loop) is not just PFC "deciding to believe X." It requires body-base confirmation through repeated experience.

**Why C5 (status shift) takes time**:
- External information received (Feel-RawSignals — Feel-Integration)
- PFC attempts label (Feel-Labeling)
- BUT body-base self-schema unchanged
- Body "doesn't confirm" → PFC label is fragile
- Repeated external evidence + schema drift over time → body gradually confirms → schema stabilizes

→ **Schema update has latency.** Cannot "decide to believe" instantly. Body must verify through multiple interface loop iterations.

### §7.5 — Cross-ref H10

H10 preconditions (§1.3) all relate to body-base:
- **Precondition 1** (schema pending) = body-base has a gap
- **Precondition 2** (chunks base) = body can decode the pattern

- **Precondition 4** (Goldilocks) = body checks coherence match ratio
