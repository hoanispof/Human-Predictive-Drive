### §1.1 — What Background-Pattern is

```
⭐ BACKGROUND-PATTERN = ACCUMULATED CHUNK-NETWORK PATTERN

  Formal definition:
    Background-Pattern = the distributed network pattern that emerges when
    MANY CHUNKS compile in the SAME DIRECTION over an EXTENDED PERIOD
    (months to years), such that Link Density becomes high enough to
    PERMEATE subsequent experience.

  Key properties:
    ① NOT a single chunk — a distributed NETWORK PATTERN
    ② NOT formed by one heavy event — formed by THOUSANDS of micro-events
    ③ NOT observable by PFC — too distributed, no single access point
    ④ NOT fixed — can shift, but requires extended time and experience
    ⑤ BIASES all new processing — new chunks link into the existing network

  Analogy:
    A single heavy rock dropped into a pond → 1 large ripple (individual event).
    Thousands of small pebbles dropped consistently over 15 years →
    the pond floor reshapes → ALL new pebbles fall into the established pattern.
    Background-Pattern = the reshaped pond floor.
```

## §2 — 2D MODEL: COMPILE DEPTH × LINK DENSITY

### §2.1 — Two independent dimensions

```
⭐ 2D MODEL — TWO INDEPENDENT DIMENSIONS:

  DIMENSION 1 — COMPILE DEPTH (from Schema.md §4.1):
    = How deeply each individual chunk is compiled
    = f(repetitions × modalities × emotional weight)
    Range: Low (surface rule) → High (deep schema)
    Determines: How resistant each chunk is to competitive re-linking

  DIMENSION 2 — LINK DENSITY (new in this file):
    = How many OTHER chunks link to this chunk-network
    Range: Low (isolated event) → High (Background-Pattern territory)
    Determines: How many entry points exist to activate the pattern
    → How pervasive the pattern's influence is on new experience

  WHY INDEPENDENT:
    High Depth + Low Density = possible
      (e.g., single heavy trauma — deeply compiled, few links to daily life)
    Low Depth + High Density = possible
      (e.g., cultural norm — each interaction shallow, but thousands of them)
    Both high = Background-Pattern territory (most persistent, hardest to shift)
    Both low = easily revised (ordinary experience)
```

## §3 — FORMATION MECHANISM

### §3.1 — Natural accumulation (the primary pathway)

```
⭐ BACKGROUND-PATTERN FORMATION = NATURAL ACCUMULATION:

  Background-Pattern does NOT form via a special mechanism.
  It forms through the ORDINARY chunk compile process — repeated over time.

  5-STEP FORMATION PROCESS:

  Step 1 — REPEATED EXPOSURE IN SAME DIRECTION:
    Same type of experience → same chunk direction compiled repeatedly
    Each event: individually light → no single amygdala peak
    But: same direction × consistent context × time
    Example: "effort → not enough" fired thousands of times over 15 years

  Step 2 — COMPETITIVE ADVANTAGE ACCUMULATES:
    Each new chunk in the same domain competes with existing chunks
    Existing Background-Pattern chunks → HIGH activation probability
    New chunks → LOW activation probability (just compiled)
    → New chunks LINK TO existing network (not replace it)
    → Network grows denser with each new experience

  Step 3 — LINK DENSITY CROSSES THRESHOLD:
    Below threshold: pattern exists but isolated
    Above threshold: pattern PERMEATES new chunks in the same domain
    → New experience automatically filtered through existing network
    → "First impression" = Background-Pattern fires before PFC processes

  Step 4 — GIST EXTRACTION ABSTRACTS THE PATTERN:
    Sleep: gist extraction compresses specific events → abstract rule
    Specific events: details fade
    Abstract rule: persists → "effort → not enough" (no specific memory needed)
    → Background-Pattern survives even when original memories fade

  Step 5 — NEOCORTEX EMBEDDING:
    Active systems consolidation (Born & Wilhelm 2012):
    Hippocampal memories → neocortex transfer over months-years
    High-frequency pattern → embedded in neocortex
    → No longer needs hippocampus for activation
    → Reconsolidation window: unclear for neocortex-embedded patterns (Q1)

  PFC NOTE:
    PFC provides initial seeds (deliberate attention) but does NOT
    create a separate formation mechanism.
    Background-Pattern = output of ordinary compile × time × repetition.
```

### §3.2 — Link Density growth trajectory

```
🟡 LINK DENSITY GROWTH OVER TIME:

  T = 0 (first event):
    1 chunk compiled in direction [effort → not enough]
    Link density: minimal
    Pattern: isolated, low activation probability

  T = 1 year:
    ~hundreds of chunks in same direction
    Link density: LOW-MODERATE
    Pattern: recognizable but not pervasive
    PFC can still override consistently

  T = 5 years:
    ~thousands of chunks in same direction
    Link density: MODERATE-HIGH
    Pattern: begins to permeate new experience
    PFC override: costly, requires sustained attention

  T = 15 years:
    ~tens of thousands of chunks in same direction
    Link density: VERY HIGH
    Pattern: fully permeates → Background-Pattern territory
    PFC override: near impossible (1 new chunk vs. thousands)

  GROWTH CURVE: Sigmoid (S-curve)
    Early: slow (each chunk is isolated)
    Mid: accelerating (each new chunk has more existing links to attach to)
    Late: plateau (system-level — new experience AUTO-LINKED)
    → 🔴 Exact sigmoid parameters: speculative

  SLEEP AMPLIFICATION:
    Each sleep cycle: gist extraction abstracts current pattern
    → New gist ADDS to existing pattern density
    → Background-Pattern formation is SLEEP-AMPLIFIED
    → Sleep × time × same-direction exposure = compound accumulation
```

```

     Sleep gist extraction removes specific details
     → What remains: abstract directional bias with no episodic handle

     Background-Pattern fires sub-threshold across many body signals simultaneously

     → Example: "I feel vaguely tense" ← thousands of chunks at low activation

  ⑦ PFC BUDGET REDUCED BY BACKGROUND-PATTERN (PFC-Operations.md v1.0):
     Background-Pattern (threat direction) → cortisol baseline elevated
```
