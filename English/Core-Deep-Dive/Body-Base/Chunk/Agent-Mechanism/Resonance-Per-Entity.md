# Resonance-Per-Entity — Per-Relationship Resonance Dynamics

> **Resonance = mutual by-product gap fill (By-Product-Gap-Resonance v1.4 §1).**
> **BUT: resonance with a PARENT ≠ resonance with a FRIEND ≠ resonance with a PARTNER.**
>
> **Same mechanism, DIFFERENT dynamics because DIFFERENT gap types are filled:**
> **Parent→child: existence-based gap (Tonic, hardware-boosted) → "seeing the child is joy."**
> **Child→parent: episodic multi-gap (Cyclic+Tonic) → lifecycle shift → "growing up and needing parents less."**
> **Close friend: Generative dominant → novelty-dependent → decay without contact.**
> **Romantic: limerence (hormone mask) → post-limerence (resonance or dissolution).**
>
> **Compilation chain: Fresh (cautious) → Compiled (automatic) → baseline → dynamics.**
> **Hardware-subsidy = ANTI-HABITUATION: parent→child MAXIMUM, close friend NONE, romantic TEMPORARY.**
>
> **Per-entity resonance is observed through 3 LAYERS:**
> **Layer 1 SUPPLY: gap fill profile + Cyclic/Tonic/Generative + Reward-Signal-Architecture Evaluative:Direct-State ratio.**
> **Layer 2 MECHANISM: hardware-subsidy + compilation path + 3-cost + 2-layer/2-stream.**
> **Layer 3 DYNAMICS: novelty architecture + decay + phantom + technology fill.**
>
> **ENGINE/ROAD/VEHICLE applied: resonance = 2 VEHICLES on the same ROAD.**
> **Specialization → each VEHICLE becomes more specialized → MORE DEPENDENT on other entities.**
> **Technology fills sensory gaps → modern resonance frontier = social + abstract.**

## §1 — COMPILATION CHAIN: Fresh → Compiled → BASELINE → DYNAMICS

> Inter-Body-Mechanism v1.0 §5 (compilation chain) + §4 (3-cost model) + PFC-Operations v1.2 (Compiled/Fresh spectrum).

```
⭐⭐ 4 STAGES OF RESONANCE COMPILATION:

  STAGE 1 — Fresh DOMINANT (first interactions):
    Cost: HIGH — 3-cost model (Inter-Body-Mechanism v1.0 §4):
      ① PFC draft: novel chains for EVERY prediction → processing load
      ② Suppress: override stranger-avoidance → efference mismatch
      ③ Uncertainty: "Who is this? Good/bad?" → cortisol holding
    Reward: UNCERTAIN (don't know if by-product match exists)
    Self-Pattern-Modeling: Fresh dominant → effortful → cautious evaluation
    Duration: first weeks-months of interaction
    = "First meeting: cautious, testing whether we're compatible"

    CHARACTERISTICS:
      → Every interaction = EXPERIMENT (will by-products match?)
      → Mismatch → abandon (total cost > reward → drift apart)
      → Match → continue → enter Stage 2
      → "Click" = early by-product match confirmed → approach drive ↑
      → Cost ② (suppress) HIGHEST at Stage 1
        — "must be proper, not yet comfortable being yourself"

  STAGE 2 — Compiled DEVELOPING (repeated success):
    Cost: DECREASING — 3-cost:
      ① PFC draft: DECREASES (Hebbian compiles successful patterns)
      ② Suppress: DECREASES (know what's safe → less override needed)
      ③ Uncertainty: DECREASES GRADUALLY (predict more accurately → less holding)
    Reward: INCREASING + PREDICTABLE
    Self-Pattern-Modeling: Compiled growing → less effort → more accurate
    Duration: months-years of regular interaction
    = "Getting familiar, meeting feels more comfortable, less effort"

    CHARACTERISTICS:
      → Interaction patterns COMPILE → automatic
      → "Understanding each other without words" = Compiled predicts accurately
      → Entity-Compiled DEEPENING across channels
      → Trust building → Self-Pattern-Modeling calibration improving
      → COMPILED QUALITY matters (PFC-Operations v1.2):
        Genuine compilation = from repeated VERIFIED match → STRONG
        Schema compilation = from "supposed to be close" belief → FRAGILE

  STAGE 3 — BASELINE (compiled bond):
    Cost: NEAR-ZERO — 3-cost:
      ① ≈ 0 (compiled path, no PFC draft)
      ② ≈ 0 (no suppress needed — "natural")
      ③ ≈ 0 (predict accurately — "know each other so well")
    Reward: BASELINE (VTA habituated → reward = invisible)
    Self-Pattern-Modeling: Compiled dominant → compiled → cost ≈ 0
    Duration: years of established bond
    = "Meeting a close friend = comfortable, no need to think"

    CHARACTERISTICS:
      → Entity presence = part of BASELINE → VTA habituated (R1 Schultz 1997)
      → Reward = INVISIBLE (PFC "notices nothing special")
      → Entity ABSENT → baseline violated → Chunk-Miss → "missing them"
      → Entity-Compiled established (Entity-Compiled v1.2) → structural inclusion
      → "Actually it was so much fun" = only visible through ABSENCE

  STAGE 4 — DYNAMICS (post-baseline):
    3 possible trajectories:

    4A — SUSTAINED RESONANCE:
      Both entities GROW independently → new by-products → new prediction-delta
      → Reward continues (Generative: self-generating)
      → = "Long-term close friends are still enjoyable"
        = ongoing novelty through growth
      → Requires: own drives maintained + shared novel experiences

    4B — HABITUATED FLAT:
      No new by-products → prediction confirmed → delta = 0
      → VTA: "exactly as expected" → no reward signal
      → Entity-Compiled STILL INTACT (baseline intact) but reward = 0
      → = "Bored but don't want to lose it"
      → Tonic intact (comfort), Generative dead (no novelty)

    4C — RESONANCE DEATH:
      Compiled suppresses own drives → loses drive → loses by-product
      → By-product match STOPS (nothing to produce)
      → Valence-Momentary ≈ 0. Valence-Structural habituated.
        Entity-Compiled = structural only.
      → = "Completely bored" = resonance dead
      → = Bond-Architecture v2.0 §4 Compiled-Suppress (primary mechanism)
      → Fix: awareness + uncompile suppress + rebuild own drives

  ⭐ COMPILATION RATE VARIES PER ENTITY TYPE:
    Parent→child: SKIP Stage 1-2 (hormone priming → hardware compiled)
    Close friend: STANDARD (R6 Hall 2018: ~200h → close friend)
    Romantic: ACCELERATED by limerence (hormone fast-track)
    Colleague: SLOW if mostly Tool-mode (limited Agent-mode time)
    → Rate depends on: hardware-subsidy + interaction frequency + Agent-mode proportion

  🟢 Fresh→Compiled compilation: Inter-Body-Mechanism v1.0 §5
  🟢 VTA habituation: R1 Schultz 1997
  🟢 3-cost model: Inter-Body-Mechanism v1.0 §4 (PFC draft, suppress, uncertainty)
  🟢 Compiled Quality: PFC-Operations v1.2 (genuine vs schema)
  🟡 3-cost mapping per stage + rate variation per entity = framework synthesis
```
