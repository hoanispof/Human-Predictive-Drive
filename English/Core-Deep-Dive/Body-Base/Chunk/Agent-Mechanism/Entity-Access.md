# Entity-Access — Gradient Model of Predictive Access-Relationships

> **Entity-Access = general mechanism:**
> **Brain builds PREDICTIVE ACCESS-RELATIONSHIP with entity for gap-filling.**
>
> **3-FACTOR MODEL:**
> **Compiled: Engine Mode — HOW brain processes entity (Tool ↔ Agent, DYNAMIC per-moment).**
> **Fresh: Gap-Need Profile — WHY I need access (A:B:C:D × breadth × compound).**
> **F3: Access Confidence — CAN I access (Entity-Compiled depth × stability × hardware-subsidy).**
>
> **Entity-Access GRADIENT (Level 0-5):**
> **Level 0: Tool/Service → Level 1-2: UNLABELED → Level 3: Compilable-dominant "close friend" →**
> **Level 4: Entity-Owned (PFC label) → Level 5: Excess territory.**
>
> **Entity-Owned = PFC LENS (observation), NOT mechanism.**
> **Mechanism = entity-access compilation (3-factor → baseline).**
> **Most entity-access = UNLABELED by PFC (fuzzy middle of gradient).**
>
> **MAJOR REFRAME: Entity-Owned (binary) → Entity-Access (gradient).**
> **Same mechanism at ALL levels. "Close friend" and "parent→child" = SAME mechanism, DIFFERENT depth.**

### §1.1 — Factor 1: Engine Mode (HOW brain processes entity)

```
⭐⭐⭐ Compiled — ENGINE MODE: Tool ↔ Agent (DYNAMIC per-moment)

  TOOL ─────────── AGENT (shallow) ──────── AGENT (deep)
  predict           predict state            predict state +
  FUNCTION          "happy or sad"           trajectory +
  "deliver package"                          optimize interaction

  BINARY GATE + CONTINUOUS SPECTRUM:
    Tool-mode = GATE CLOSED → entity-access IMPOSSIBLE (regardless of Fresh, F3)
    Agent-mode = GATE OPEN → entity-access POSSIBLE → depth matters

  ⭐⭐ ENGINE MODE = DYNAMIC (shifts MOMENT-TO-MOMENT):

    Parent MORNING (rested, free time): Agent-mode deep → "the child seems worried today"
    Parent EVENING (exhausted, deadline): Tool-mode → "just eat!" → predict FUNCTION only
    → SAME parent, SAME child — DIFFERENT moment = DIFFERENT mode
    → PFC Resource State MODULATES Engine Mode directly

  ENGINE MODE = PER-PAIR + PER-MOMENT:
    Postal worker: I access the SERVICE (Tool toward service), their spouse accesses their PARTNER (Agent)
    → Same 1 person, DIFFERENT observer: service vs individual
    → Self-Pattern-Modeling mode = property of OBSERVER's processing, NOT of entity

  Simulation-Engine application:
    Self-Pattern-Modeling = (Other, Present, Simulate) on Simulation-Engine
    Tool-mode: Self-Pattern-Modeling DOES NOT run (no state simulation)
    Agent-mode: Self-Pattern-Modeling RUNS (simulate state → predict → optimize)

  🟡 Engine Mode as dynamic gate + spectrum = framework synthesis (v1.0)
```

## §2 — ENTITY-ACCESS GRADIENT (LEVEL 0-5)

### §2.1 — Full gradient

```
⭐⭐⭐ ENTITY-ACCESS = GRADIENT, NOT BINARY:

  LEVEL 0 — NO ENTITY-ACCESS (3 sub-types):

    0a — MACHINE/SYSTEM: target = machine, system → HAS NO internal state
      → ATM, vending machine, website, app
      → Entity-access impossible BY DEFINITION

    0b — SERVICE ACCESS: target = SERVICE, individual = invisible interchangeable component
      → Postal worker, taxi driver, chain-store employee
      → Whether that individual is happy or sad = IRRELEVANT → swap for another → NO impact
      → "Small talk" = 0b WITH COMPILED SCRIPTS, NOT Level 1
        (scripted driver + scripted passenger = protocol exchange, DOES NOT predict STATE)
      → CAN shift to Level 1 if the same individual repeats enough (rare)

    0c — TOOL-MODE TOWARD SPECIFIC INDIVIDUAL: knows entity but only predicts function
      → Manager under deadline: "Has employee A finished?" → function only
      → Exhausted parent in the evening: "Just eat!" → KNOWS it's their child, but in this moment = Tool
      → DIFFERENT from 0b: entity IS ALREADY RECOGNIZED → shift to Agent EASIER
      → Tired parent: 0c = PAUSE (entity-access already exists, temporarily offline)

  LEVEL 1 — AGENT-MODE SHALLOW (gate open, early):
    "Interesting person" / "good to talk to"
    → Someone just met at a party, new colleague, new neighbor

  LEVEL 2 — PARTIAL COMPILE (developing):
    "Good acquaintance" / "want to meet again" / "somewhat familiar"
    → Friend of a few months, regular barista, neighbor who chats often

  LEVEL 3 — DEEP COMPILE, B-DOMINANT (friendship):
    "Close friend" / "important" / "someone I miss"
    → Entity-access STRONG but NO "ownership" feeling
    → Exit easy → PFC does NOT call it "possession"

  LEVEL 4 — DEEP COMPILE + HARDWARE/LOW EXIT (Entity-Owned):
    "Part of my life" / "CANNOT LOSE"
    → PFC label: "Entity-Owned" — parent→child, long-term spouse
    → CAN be healthy (A+B dominant) or heading toward excess (C+D rising)

  LEVEL 5 — COMPILE + C DOMINANT + NO EXIT (Excess):
    "MINE" / "MUST stay here" / "MUST do what I say"
    → Entity-access MAXIMUM + distorted → detail §6

  ⭐ SAME MECHANISM AT ALL LEVELS:
    Level 1 and Level 4 = SAME 3-factor model, DIFFERENT compilation depth
    "Close friend" and "parent→child" = SAME mechanism, DIFFERENT starting mode + hardware

  🟡 Entity-Access Gradient Level 0-5 = framework synthesis (v1.0)
```
