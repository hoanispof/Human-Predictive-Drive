## §1 — ANCHOR-SCHEMA: THE ORCHESTRA SYNCHRONIZATION MECHANISM

```
WHAT IS ANCHOR-SCHEMA — THROUGH THE MELODY LENS:

  The unconscious = AN ORCHESTRA:
    → Thousands of "musicians" (compiled schemas) ALWAYS playing
    → Each musician = 1 schema auto-firing on its own trigger
    → Playing SIMULTANEOUSLY, in PARALLEL, without stopping
    → = An orchestra that PLAYS ITSELF — no conductor needed for most of it

  Anchor-Schema = THE MAIN PLAYER:
    → 1 (or a few) musicians are "turned up" louder
    → The entire orchestra LISTENS and gradually SYNCS to the main player
    → = Sync point — the reference point for the entire system
    → = The "musical piece" the whole orchestra plays together

  No Main Player:
    → Each musician plays their own piece independently
    → = Cacophony → body signal: dissonance
    → = "Boredom" / "lost" / anxiety (Boredom-Analysis.md §1)

  Main Player Present But WRONG:
    → Orchestra syncs BUT the piece doesn't suit the hardware / domain
    → = Melody IS present but NOT smooth
    → = Mismatch / Delusion (Imagine-Final-Evaluation.md §4)


HOW ANCHOR-SCHEMA OPERATES:

  ┌──────────────────────────────────────────────────────────────┐
  │                                                              │
  │  BASELINE STATE (smooth melody → no strong anchor needed):  │
  │                                                              │
  │    → Body-base MET → schemas auto-fire harmoniously         │
  │      → smooth melody                                        │
  │    → Anchor-Schema DORMANT (present but no amplification    │
  │      needed)                                                 │
  │    → E.g.: lying on the beach → everything's fine           │
  │      → no need for a "direction"                            │
  │    → = Anchor-Schema exists in the BACKGROUND but isn't     │
  │      ACTIVE                                                  │
  │                                                              │
  │  WHEN DISSONANCE (disrupted melody → ANCHOR NEEDED):        │
  │                                                              │
  │    → Body-base UNMET → schemas CONFLICT → disrupted melody  │
  │    → Anchor-Schema ACTIVATES — amplifies → syncs orchestra  │
  │    → E.g.: urgent deadline → Anchor "finish it" amplifies   │
  │      → whole body syncs: sits down, focuses, types code     │
  │    → = Anchor-Schema is MOST NEEDED when dissonance is      │
  │      STRONGEST                                               │
  │                                                              │
  │  THE KEY PRINCIPLE:                                          │
  │    → Anchor-Schema strength ≥ dissonance level → SYNC HOLDS │
  │    → Anchor-Schema strength < dissonance level → SYNC LOST  │
  │    → = "Is THIS difficulty worth THAT direction?"           │
  │    → = If anchor is STRONGER → "worth it, keep going"       │
  │      → continues                                             │
  │    → = If anchor is WEAKER → "not worth it" → stops/fades  │
  │                                                              │
  └──────────────────────────────────────────────────────────────┘


THE 6-STEP FLOW — FROM DISSONANCE TO SYNC:

  ① BODY-BASE DISSONANCE: "something's missing"
     → Some channel unmet → dissonance signal
     → The orchestra starts to "drift" — some musicians fire in conflict
     → = TRIGGER for the entire flow

  ② ANCHOR-SCHEMA SCAN: "is there an anchor already available?"
     → The unconscious CHECKS: is there a compiled schema already acting
       as an anchor?
     → IF YES (compiled deep, familiar situation):
       → Anchor ACTIVATES immediately → skips to ⑤
       → E.g.: hungry + familiar "open the fridge" → auto sync → action
       → E.g.: manager assigns familiar task + compiled "career" Anchor
         → auto sync
       → = MOST OF LIFE → anchor already READY → PFC NOT NEEDED
     → IF NO (new situation, conflict, old anchor insufficient):
       → Proceed to ③ — CALL THE PFC

  ③ PFC SCAN + IMAGINE: "find a new anchor"
     → PFC is called (because the unconscious can't handle it alone)
     → PFC generates multiple Imagine-Finals (simulates options):
       → "Eat?" / "Scroll?" / "Call a friend?" / "Go outside?"
         / "Change jobs?"
     → Hippocampus simulates a preview of each option
     → Body PRE-FEELS each preview → opioid preview or not
     → = PFC is creating CANDIDATES for the Anchor-Schema position

  ④ BODY EVALUATE + SELECT: "which one is GOOD ENOUGH?"
     → Body-base compares each candidate:
       → Which preview → body feels "melody will be smoother?"
       → Which candidate → body confirm is STRONGEST?
     → PFC ALSO evaluates: domain fit? hardware fit? feasible?
     → INTERSECTION: body confirm + PFC confirm → SELECT
     → = THIS Imagine-Final → BECOMES the Anchor-Schema
     → ⚠️ IF no candidate is sufficient:
       → PFC: "haven't found one yet" → returns to ③ (tries more)
       → OR: stuck → "boredom" (Boredom-Analysis §1 ④)

  ⑤ ANCHOR AMPLIFIES → UNCONSCIOUS SYNCS:
     → Selected schema → AMPLIFIES (PFC spotlight + body commits)
     → The whole orchestra begins LISTENING to the main player
     → Related schemas: fire MORE strongly (supporting the anchor)
     → Unrelated schemas: fire MORE softly (yielding bandwidth)
     → Actions EMERGE: whole body syncs → behavior serves the anchor
     → E.g.: Anchor "call friend B" → hand reaches for phone
       → finds the name → calls

  ⑥ DOMAIN FEEDBACK → RE-EVALUATE:
     → Action → result → domain feedback
     → Feedback ✓ (friend B is happy) → Anchor CONFIRMED → Trust INCREASES
     → Feedback ✗ (friend B is busy) → Anchor WEAKENS → re-scan
       (returns to ②③)
     → Feedback OPEN (not yet clear) → Anchor MAINTAINED → keep trying
     → = Continuous LOOP — anchor is NOT FIXED, continuously re-evaluated

  ⭐ STEP ② IS THE MOST IMPORTANT STEP:
    → Most of life: step ② → compiled anchor FOUND → skips to ⑤
    → = The unconscious handles ~95% (Drive.md §1)
    → ONLY when ② fails (no compiled anchor available) → THEN calls PFC (③④)
    → = PFC = backup, not primary
    → = Anchor-Schema is a concept BELONGING TO the unconscious;
      PFC only SUPPLIES when needed


MULTI-SCALE ANCHOR — MULTIPLE ANCHORS SIMULTANEOUSLY:

  Like multi-scale Imagine-Final, Anchor-Schema is also multi-scale:

    Micro-Anchor (seconds → minutes):
      → "Eat this bite" → hand holds chopsticks → brings to mouth
      → = Compiled schema, automatic, PFC ~0%

    Short-Anchor (minutes → hours):
      → "Finish the assignment" → sit, write, focus
      → = Semi-compiled schema, PFC ~5–15% monitoring

    Daily-Anchor (day):
      → "Complete today's tasks" → organize, prioritize, execute
      → = PFC ~15–30% (planning + navigating)

    Life-Anchor (years → lifetime):
      → "Career success" / "Happy family" / "Life purpose"
      → = Deep compiled OR PFC-built Imagine-Final
      → = PFC ~variable (dormant when smooth, active during crisis)

  HIERARCHY:
    → Micro-Anchors SERVE short-Anchors → SERVE daily → SERVE life
    → "Open laptop" (micro) → "finish task" (daily)
      → "get promoted" (yearly) → "career success" (life)
    → = ALIGNMENT across scales → smooth drive → "flow"
    → = MISALIGNMENT → "busy but going nowhere"
      (micro-anchors running but NOT CONNECTED to the life-anchor)

  🟢 Goal hierarchy = established (Carver & Scheier 1998, control theory)
  🟢 PFC maintains goal representations → biases processing
       (Miller & Cohen 2001)
  🟢 Global Workspace Theory: PFC broadcast → whole-brain sync (Baars 1988)
  🟢 Predictive processing: PFC sets high-level priors (Clark 2013, Friston)
  🟡 "Anchor-Schema = bottom-up requirement" = new framework concept
  🟡 "6-step flow" = framework model, derived from established components
  🟡 "Step ② is most important" = framework emphasis
```

## §2 — TRUST: THE BINDING FORCE BETWEEN SYSTEM AND ANCHOR

```
WHAT IS TRUST — FRAMEWORK DEFINITION:

  Trust = The degree to which the system (unconscious + PFC) COMMITS to
          the Anchor-Schema, sufficiently to MAINTAIN sync through dissonance.

  NOT:
    ✗ Interpersonal trust (trusting people) — that is 1 specific form
    ✗ Blind faith — trust CAN BE evidence-based
    ✗ Willpower (PFC-only) — trust is MOSTLY in the unconscious
    ✗ A property of Imagine-Final — trust belongs to BINDING, not content

  IS:
    ○ The BINDING FORCE between the system and the Anchor-Schema
    ○ = "How firmly is THIS anchor holding?" (firm → ship holds;
        loose → ship drifts)
    ○ Mostly UNCONSCIOUS (compiled trust = body-level, no PFC deliberation
        needed)
    ○ CAN BE conscious (PFC evaluates and chooses to trust)
    ○ CHANGES over time (builds, breaks, rebuilds)

  ORCHESTRA METAPHOR:
    → Anchor-Schema = main player (WHICH piece is playing)
    → Trust = VOLUME of the main player (HOW LOUD)
    → Trust HIGH → main player LOUD → whole orchestra HEARS CLEARLY
      → sync GOOD
    → Trust LOW → main player SOFT → orchestra HEARS FAINTLY → sync WEAK
    → Trust = 0 → main player SILENT → orchestra loses sync → cacophony


ANCHOR STRENGTH — HOW DEEPLY COMPILED IN THE UNCONSCIOUS:

  Trust (above) = TOTAL binding (conscious PFC commit + unconscious compiled)
  Anchor Strength = SPECIFICALLY: how DEEPLY is THIS schema compiled in
                    the unconscious?
  = STATE VARIABLE — snapshot at a single moment, CONTINUOUSLY CHANGING

  3 LEVELS:

    FAINT (schema just recently formed):
      → Just heard an idea → "interesting" → new schema, not yet compiled
      → Fires VERY WEAKLY → easily overridden by stronger schemas
      → E.g.: read an article about startups → "maybe I should try?"
        → faint, not yet an anchor
      → = PFC must HOLD it active continuously → consumes bandwidth

    BUILDING (schema accumulating):
      → Thought about it many times + small attempts + partial results
        → fires MORE STRONGLY
      → E.g.: 3 months of research + prototype → "seems promising"
        → building
      → = Not yet deeply compiled but has momentum → PFC still needs
          to support it

    COMPILED (schema has gone deep, fires automatically):
      → No PFC holding needed → the unconscious FIRES AUTOMATICALLY
        → auto-anchor
      → E.g.: Jensen Huang 2024: "GPU = AI" = compiled EXTREMELY DEEP
      → E.g.: A "calling" = compiled deep from thousands of body-base
          confirmations
      → = PFC does NOT NEED to deliberate → low energy cost → sustainable

  3 PATHS TO INCREASING ANCHOR STRENGTH:
    ① Internal: reflection + experience → fires many times → gradually compiled
    ② External install: parents, society, religion repeat → compiles into
        the unconscious
    ③ PFC holds → months of body confirmation → unconscious AGREES → compiled
    → = Maps to 4 sources (§3): ① = sources ①+③, ② = source ④,
        ③ = source ①→③

  DECREASE:
    → Domain feedback ✗ → weakens
    → Competing schema stronger → overrides
    → Time without firing → fades (decay)


COST — HOW STRONG DOES AN ANCHOR NEED TO BE TO HOLD?

  §1 principle: "Anchor strength ≥ dissonance level → SYNC HOLDS"
  REFINED: dissonance level ≈ f(COST)

  Cost = TOTAL price to pay for the Anchor-Schema to reach its outcome:
    → Time: time invested
    → Energy: body-base resources consumed (cortisol, PFC bandwidth)
    → Risk: probability of LOSS if it fails (money, status, relationship, health)
    → Opportunity cost: what else must be GIVEN UP to pursue this anchor
    → Accumulated dissonance: total "discomfort" that must be endured along the way

  → Cost ≠ purely temporal (time):
    → Short + high cost: changing jobs = 1-day decision but ENORMOUS risk
    → Long + low cost: reading every evening = many years but LOW cost per evening
    → = Time is WITHIN cost, but cost is BROADER than time

  THE COST SPECTRUM:

    Cost ≈ 0: "simple problem → can do it right away"
      → Trust needed: ALMOST NONE (because dissonance ≈ 0)
      → = Anchor activates → execute → done → no time to think about "faith"

    Low cost: "finish the assignment → get to play games"
      → Trust needed: LIGHT (a few hours effort, reward is clear)

    Medium cost: "complete the course → gain a skill"
      → Trust needed: MODERATE (months of effort, reward is delayed)

    High cost: "build a startup → achieve product-market fit"
      → Trust needed: STRONG (years of effort + money + health +
          relationship risk)

    Extreme cost: "Einstein → unified physics" / "Musk → Mars"
      → Trust needed: EXTREMELY STRONG, or deeply compiled, or faith
      → = Cost CAN BE = an entire lifetime + NO GUARANTEE of achievement

  ⭐ THE PRINCIPLE: Trust ≥ Cost → anchor HOLDS; Trust < Cost → anchor COLLAPSES

    → High cost + High trust → anchor HOLDS through major dissonance
      E.g.: founder endures 3 years of losses because trust "the product
      will fit" is strong
    → High cost + Low trust → anchor COLLAPSES early
      E.g.: "want to change careers but the risk is too high... forget it"
    → Low cost + Low trust → still OK (because strong trust isn't needed)
      E.g.: "try cooking a new dish" → failing is fine
    → = Cost DETERMINES "how strong the anchor needs to be"

  🟡 "Trust ≥ Cost" = framework principle
  🟡 Cost = composite (time × risk × opportunity × dissonance) = framework model
  🟡 Anchor Strength = unconscious depth snapshot = refined Trust concept


WHY TRUST DIFFERS FROM CLARITY AND QUALITY:

  3 INDEPENDENT DIMENSIONS OF IMAGINE-FINAL → ANCHOR-SCHEMA:

    DIMENSION 1 — CLARITY (Imagine-Final.md):
      = "How CLEAR is the Imagine-Final?"
      = How VIVIDLY can the body simulate it?
      = Spectrum: Wish (0%) → Obsession (100%)
      = Measured by: 14 thresholds, body fidelity

    DIMENSION 2 — QUALITY (Imagine-Final-Evaluation.md):
      = "Is the direction GOOD or BAD?"
      = Domain fit? Hardware fit?
      = Spectrum: Fantasy → Delusion → Mismatch → Sweet Spot
      = Measured by: 2 axes, domain feedback, body signals

    DIMENSION 3 — TRUST (THIS file):
      = "How committed is the system?"
      = How tightly does the system BIND to the anchor?
      = Spectrum: 0 (no binding) → ∞ (unfalsifiable faith)
      = Measured by: behavior when dissonance occurs
        (stops immediately = low trust; endures long = high trust)

  3 INDEPENDENT DIMENSIONS — DEMONSTRATED:

    Clarity HIGH + Quality ✓ + Trust HIGH = STRONGEST:
      → Jensen Huang 2024: SEES the GPU future (clarity) + domain ✓
        + 100% commitment (trust)
      → = Full sync → 30 years of consistency → NVIDIA

    Clarity HIGH + Quality ✓ + Trust LOW = "KNOWS BUT DOESN'T DARE":
      → "I CLEARLY SEE I should change careers" (clarity) + domain ✓

      → = Imposter syndrome, self-doubt

    Clarity LOW + Quality ? + Trust HIGH = "BELIEVES WITHOUT SEEING":
      → Religious believer: DOESN'T see heaven clearly (low clarity)
        + domain ? (unfalsifiable) + ABSOLUTE BELIEF (trust ∞)
      → = Anchor-Schema STRONG despite DIM Imagine-Final
      → = Faith, calling, gut conviction

    Clarity HIGH + Quality ✗ + Trust HIGH = MOST DANGEROUS:
      → Cult leader: clear vision + WRONG domain + 100% commitment
      → = Orchestra syncs STRONGLY but direction is COMPLETELY WRONG
      → = Delusion + high trust = runs far + wrong direction + cannot stop


TRUST SPECTRUM — FROM MICRO TO INFINITE:

  ┌──────────────────────────────────────────────────────────────────┐
  │                                                                  │
  │ TRUST = 0         LOW TRUST      MODERATE TRUST    HIGH TRUST   │

  │                   dissonance                                     │

  │ E.g.: don't know  E.g.: "let's   E.g.: "I trust    E.g.: Jensen │
  │ what you want     try and see,   this direction    Huang + GPU  │
  │                   not sure"      is right"                      │

TRUST DYNAMICS — BUILD, BREAK, REBUILD:

  ── BUILD (trust INCREASES) ──

    ① Positive domain feedback (results confirm):
      → Anchor "build startup" → MVP launches → users arrive
        → trust INCREASES
      → = "The direction is really right" → body confirms → binds
          MORE TIGHTLY
      → = THE MOST DURABLE WAY TO BUILD TRUST (evidence-based)

    ② Body-base confirm (feels right):
      → Each day working → body reward → "worth it"
      → = Body says "RIGHT" → trust increases at the UNCONSCIOUS level
      → = Why "experiencing" builds trust more than "thinking"

    ③ Repetition → compile (repetition → compiled deep):
      → Ritual, habit, routine → schema compiles → trust becomes IMPLICIT
      → PFC deliberation no longer needed → body TRUSTS AUTOMATICALLY
      → E.g.: weekly religious ritual → trust compiled deep
      → E.g.: daily routine "wake → code → ship" → trust compiled

    ④ Social reinforcement (others trust the same thing):
      → "Everyone else believes it too" → one's OWN trust increases
      → = Mirror neurons + social proof → amplifies
      → E.g.: startup team shares the same vision → trust MULTIPLIES
      → E.g.: congregation prays together → trust MULTIPLIES

  ── BREAK (trust DECREASES / LOST) ──

    ① Negative domain feedback (results ✗):
      → Anchor "startup" → 12 months → 0 users → trust GRADUALLY DECREASES
      → = "Seems like the wrong direction" → body doubts → binding LOOSENS
      → ⚠️ BUT: rationalization CAN RESIST domain feedback
        → "The market isn't ready yet" → trust maintained despite feedback ✗
        → = Delusion mechanism (Imagine-Final-Evaluation.md §4)

      → Each day working → body drains → "not worth it"
      → = Body says "WRONG" → trust decreases at unconscious level
      → = Meaningless dissonance (≠ difficulty of a hard task)
          → trust erodes

    ③ Trust violation (promise broken):
      → "Dad promised to buy a toy" → over the weekend DID NOT buy
      → = Trust BROKEN → not just for THIS event
        → Schema "dad promises → dad does" is DAMAGED
        → Next time dad makes a promise → trust is LOWER
      → = Trust violation → RIPPLE EFFECT onto related schemas

    ④ Alternative exposure (sees something better):
      → Trusting Anchor A → sees option B looking MORE APPEALING
      → = Trust in A DECREASES (relative, even though A hasn't changed)
      → E.g.: settled at a company → sees a friend's startup succeed
        → Trust in "stable career" DECREASES (even though situation
            didn't change)
      → = Modern social media: constant exposure → existing anchor trust
          continuously tested

  ── REBUILD (trust after breaking) ──

    → Trust breaks → CAN be rebuilt BUT more slowly than building it
        the first time
    → "Being betrayed once" → threshold HIGHER for trusting again
    → Requires: domain feedback ✓ MANY TIMES + body confirm MANY TIMES
    → = "Trusting again" = rebuilding the schema "X is trustworthy"
        from a damaged state
    → = Why a second chance REQUIRES TIME + ACTION, not just words


POSITIVE TRUST vs NEGATIVE TRUST:

  ── POSITIVE TRUST — "I trust the reward is AHEAD" ──

    → Trust TOWARD: good results will come if one continues
    → Anchor-Schema: directed TOWARD something good
    → Body feeling when dissonance: "hard but WORTH IT"
    → Reward when achieved: body opioid → "deserved" → smooth melody
    → Sustainable: YES (reward per step → trust reinforced → positive cycle)

    Examples:
      → "Dad will buy a toy car this weekend" → positive trust
        → Dissonance (homework) → endured because reward ahead is clear
      → "Build product → users love it" → positive trust
        → Dissonance (hard code) → endured because vision is clear
      → "A harmonious partnership → a happy family" → positive trust
        → Dissonance (difficulty) → endured because trust in each other

  ── NEGATIVE TRUST — "I trust I will AVOID harm" ──

    → Trust TOWARD: if I act → I will NOT BE harmed
    → Anchor-Schema: AVOIDS what is bad (not directed TOWARD what is good)
    → Body feeling when dissonance: "hard, but if I stop it will be WORSE"
    → Reward when achieved: threat resolved → cortisol GRADUALLY WITHDRAWS
        → relief → "Phew, escaped"
      (NOT opioid "worth it" — but relief from threat cessation = "pain ends")
    → Sustainable: POOR (relief is diminishing, threat must ESCALATE
        to maintain it)

    Examples:
      → "If I don't study, mom will scold me" → negative trust
        → Dissonance (homework) → endured because of FEAR of harm
        → Done → "Phew, mom isn't angry" (relief, not "worth it")
      → "If I don't study, I'll get beaten" → stronger negative trust
        → Dissonance → endured because of FEAR of pain (survival trust)
        → Done → "didn't get beaten" (intense relief but NOT positive)

  ── COMPARISON ──

    ┌──────────────────┬──────────────────┬──────────────────────┐
    │                  │ POSITIVE TRUST   │ NEGATIVE TRUST       │
    ├──────────────────┼──────────────────┼──────────────────────┤
    │ Trust toward     │ Reward ahead     │ Avoid harm           │
    │ Anchor direction │ TOWARD good      │ AWAY FROM bad        │
    │ Dissonance feel  │ "Hard but worth  │ "Hard, but if I stop │
    │                  │  it"             │  it will be worse"   │
    │ Reward type      │ Opioid           │ Relief ("pain ends") │
    │                  │ ("pleasant")     │                      │
    │ Sustainable      │ YES (reinforcing)│ POOR (diminishing)   │
    │ Long-term        │ Smooth melody    │ Neural wear          │
    │ body-base        │                  │ (chronic cortisol)   │
    │ Reference        │ Novelty reward   │ Threat relief reward │
    │                  │ (Novelty.md)     │ (Threat.md §3)       │
    └──────────────────┴──────────────────┴──────────────────────┘

    → BOTH are trust → BOTH create Anchor-Schema → BOTH sync the orchestra
    → BUT: outputs DIFFER (melody vs neural wear)
    → = "Child obeys because SCARED" vs "child obeys because WANTS TO"
    →   Both "obey" → differ in long-term body-base cost

    ⚠️ NUANCE: most trust = a MIX of positive + negative:
      → Graduate: trust "escape poverty" (negative)
        + trust "prove my worth" (positive) → MIX
      → Startup founder: trust "product vision" (positive)
        + trust "out of money if I stop" (negative) → MIX
      → = Purely positive or purely negative = RARE
      → = Assessing: the positive/negative ratio → predicts sustainability


HOW TO CHECK TRUST LEVEL:

  BEHAVIOR WHEN FACING DISSONANCE = THE BEST MEASUREMENT:

    High trust:
      → Encounters difficulty → still acts → "I know this will pass"
      → Negative feedback → absorbs + adjusts → DOES NOT QUIT
      → Loses a night's sleep over something difficult → still works
          in the morning (anchor holds)

    Moderate trust:
      → Encounters difficulty → hesitates → "I'm not sure..." → then acts
      → Negative feedback → wobbles → needs reinforcement (mentor, peer)
      → = Anchor HOLDS but needs support

    Low trust:
      → Encounters difficulty → "forget it, this probably isn't right"
          → stops
      → Negative feedback → "EXACTLY, I KNEW it was wrong" → quits immediately
      → = Anchor LOST immediately when dissonance appears

    Trust = 0:
      → Doesn't even encounter difficulty — just doesn't act (no anchor to drive it)
      → = "Boredom" / "lost" — not laziness

  BODY SIGNALS:
    → High trust: body CALM when thinking about the anchor
        (compiled → implicit)
    → Low trust: body ANXIOUS when thinking about the anchor
        ("probably not right...")
    → = The body REFLECTS trust level more accurately than PFC self-report
    → = "I SAY I trust" ≠ "my BODY trusts" (they can differ)

  🟢 Commitment → goal attainment (Locke & Latham 1990)
  🟢 Intrinsic vs extrinsic motivation outcomes (SDT: Deci & Ryan 2000)

  🟢 Trust violation → ripple effect = trust repair research (Kim et al. 2004)
  🟢 Social proof → belief amplification (Cialdini 2001)
  🟢 Repetition → implicit learning / compiled (Reber 1993)
  🟡 "Trust = binding force" = new framework concept
  🟡 "3 independent dimensions" (Clarity × Quality × Trust) = framework model
  🟡 Positive vs negative trust = framework mapping onto approach/avoidance
  🟡 Trust dynamics (build/break/rebuild) = framework compilation
  🟡 "Body reflects trust level more accurately than PFC self-report"
      = framework claim, consistent with Somatic Marker Hypothesis
      (Damasio 1994)
```

```
═══════════════════════════════════════════════════════

  CHARACTERISTICS:
    → RICHEST: cross-domain, temporally distant, abstract

    → FLEXIBLE: PFC can build, adjust, discard, rebuild

    → NEW situations (no compiled schema matches)

      = Imagine-Final → Anchor-Schema (if trust is sufficient)

      = PFC simulates both options → compares

      = PFC scans chunks (which field? where? salary?)

      = SELECTS 1 → Anchor-Schema for the coming period

  LIMITATIONS:
    → REQUIRES sufficient chunks (too few chunks → DIM simulation

    → REQUIRES healthy PFC (tired, stressed → PFC impaired
        → Imagine-Final impaired)
    → REQUIRES time (PFC is slow → not instant)
    → = Source RICHEST but SLOWEST + many CONDITIONS

  ⭐ DETAILED FLOW — IMAGINE-FINAL → ANCHOR-SCHEMA:

    This is NOT a 1-step instant process — it's a LIFECYCLE over time:

      PFC BUILDS Imagine-Final (initial draft)
          ↓ SAVES when PFC switches tasks
      Imagine-Final dormant in storage (proto-schema state)
          ↓ BACKGROUND processing (unconscious processes related chunks)
      RELOADS when body triggers → new chunks available → REFINES
          ↓ Multiple cycles loop
      Imagine-Final gradually "ripens" — clarity increases +
          body confirms continuously
          ↓ Sufficient repetitions + domain feedback ✓
      COMPILES into a Schema (enters the deep unconscious)
          ↓ Schema grows strong enough
      Becomes an ANCHOR-SCHEMA (auto-fires, no PFC holding needed)

    → = Imagine-Final → Anchor-Schema = A LONG PROCESS (months–years)
    → = NOT an instant PFC "decision"
    → = Requires a lifecycle (Imagine-Final.md §1.5) through many cycles
    → = Jensen Huang example: 30 years of cycles → eventually compiled
        deep

    BUILD → SAVE → BACKGROUND → RELOAD → REFINE → loop →

SOURCE ② — HIPPOCAMPUS SIMULATION (mid-level)
═══════════════════════════════════════════════════════

  CHARACTERISTICS:
    → FASTER THAN PFC: hippocampus fires automatically, no deliberation needed

    → Spatial navigation (finding the way, going home)
    → Familiar situations with slight variations
    → Immediate planning (maze, route, sequence)

    → = Source FAST but NARROW
```
