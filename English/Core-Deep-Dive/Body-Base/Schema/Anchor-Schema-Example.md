## MICRO-ANCHOR (seconds → minutes)

> Mostly INVISIBLE — happens too fast to "feel" the trust.
> BUT: still an Anchor-Schema (the unconscious syncs to a specific schema).
> = Tier 0–1 (Anchor-Schema.md §4).

```
── M1. REACHING FOR A GLASS OF WATER ──

  Situation: Thirsty → sees a glass of water → hand reaches out → grasps → drinks.

  Anchor: schema "hand → glass → grasp → drink" (compiled sensorimotor)
  Trust: positive, IMPLICIT (no doubt that "hand will touch the glass")
  Strength: COMPILED (thousands of repetitions → automatic)
  Cost: ≈0 (seconds, no risk, no sacrifice)
  Domain: real ✓ (physics — hand touches glass = domain confirmed every time)
  Source: ③ Compiled (motor schema from thousands of repetitions)
  Outcome: micro-smooth melody → done → next anchor

  ⭐ NOTE: Is this REALLY an Anchor-Schema?
    → YES — whole body syncs (eyes track glass, arm coordinates, fingers prepare)
    → BUT: too small, too fast → invisible → no one thinks "I TRUST my hand
        will touch it"
    → = Anchor-Schema at its MOST PRIMITIVE level — pure compiled, zero PFC


── M2. PRESSING A KEY → CHARACTER APPEARS ON SCREEN ──

  Situation: Coding/writing → press a key → character appears → continues.

  Anchor: schema "finger → key → character" (compiled typing)
  Trust: positive, IMPLICIT
  Strength: COMPILED (experienced typist: millions of repetitions)
  Cost: ≈0
  Domain: real ✓ (keyboard → display = domain confirmed on every keystroke)
  Source: ③ Compiled
  Outcome: micro-smooth melody → continuous flow

  ⭐ NOTE: When keyboard LAGS → anchor BREAKS INSTANTLY:
    → Types → character DOESN'T appear → body JOLTS (micro-dissonance)
    → = Schema "press → character" REFUTED by domain feedback
    → = Trust BREAKS at micro-level → body dissonance → PFC jumps:
      "keyboard broken?" → debug
    → = PROVES: even micro-anchors HAVE trust,
      and trust IS BROKEN when domain feedback ✗


── M3. STEPPING DOWN A STAIRCASE ──

  Situation: Walking down the stairs → steps → foot lands on step → continues.

  Anchor: schema "step → foot lands → gravity stable" (compiled motor)
  Trust: positive, IMPLICIT
  Strength: COMPILED
  Cost: ≈0
  Domain: real ✓
  Source: ③ Compiled + ② Hippocampus (spatial map of staircase)
  Outcome: smooth melody

  ⭐ NOTE: Missing a step → EXTREMELY STRONG JOLT:
    → Schema "foot will land on the next step" → foot DOESN'T land → body SHOCK
    → = Micro-anchor trust BREAKS → whole body emergency response
    → = NE spike, adrenaline, heart pounds → even from just one missed step
    → = PROVES: body INVESTS trust in EVERY micro-anchor
      → when trust is refuted = response is STRONG (even for "small" things)
    → = Why VR headsets cause dizziness:
      body trusts "step → hit the floor" → VR says OTHERWISE → micro-trust
      CONFLICT


── M4. TODDLER REACHING ARMS OUT TO BE HELD ──

  Situation: 1–2 year old → sees mom → reaches arms out → waits to be picked up.

  Anchor: schema "arms out → mom picks up" (compiled from hundreds of times)
  Trust: positive, IMPLICIT (body KNOWS mom will pick them up)
  Strength: COMPILED (attachment schema — deep)
  Cost: ≈0 (reaching out = extremely small effort)
  Domain: real ✓ (mom picks up = domain confirmed hundreds of times)
  Source: ③ Compiled (attachment schema) + ② hippocampus (recognizes mom's face)
  Outcome: mom picks up → smooth melody → oxytocin

  ⭐ NOTE: Mom DOESN'T pick up → trust BREAKS in the child:
    → Reaches out → mom turns away → body: DISSONANCE (attachment violation)
    → IF repeated many times → compiled schema "mom DOESN'T pick up"
      → Trust DAMAGED → attachment style CHANGES (insecure)
    → = Anchor-Schema.md §2 Trust violation → ripple effect
    → = Attachment theory (Bowlby): repeated micro-trust BREAKS
      → compile into long-term attachment schema
    → 🟢 Verified: attachment theory (Bowlby 1969, Ainsworth 1978)


── M5. DRIVING TO A FAMILIAR INTERSECTION ──

  Situation: Driving a familiar route → green light → turn right → continue.

  Anchor: schema "this intersection → turn right → go home" (compiled route)
  Trust: positive, IMPLICIT
  Strength: COMPILED (passed through hundreds of times)
  Cost: ≈0
  Domain: real ✓ (road is still there, lights still work)
  Source: ③ Compiled + ② Hippocampus (spatial map of route)
  Outcome: smooth melody → PFC is thinking about something ELSE (auto-pilot)

  ⭐ NOTE: Road is CHANGED (construction, blocked) → PFC JUMPS:
    → Compiled route → domain feedback ✗ (road blocked) → anchor BREAKS
    → PFC: activates → scans alternative → builds new micro-anchor
    → = Step ② fails → ③④ activate (Anchor-Schema.md §1 flow)
    → = Why a familiar road being changed is ANNOYING (micro-trust break)
```

## SHORT-ANCHOR (minutes → hours)

> PFC BEGINS to participate (lightly). Trust BEGINS to be "felt."
> Cost > 0 but STILL low. Domain feedback FAST (minutes–hours).
> = The "normal" Anchor-Schema that everyone experiences dozens of times a day.

```
── S1. FINISHING HOMEWORK → GETTING TO PLAY GAMES ──

  Situation: Child, age 8. Dad says "finish your homework → you can play games
  for 1 hour."

  Anchor: schema "finish homework → games" (reward schema injected by dad)
  Trust: positive ("games are CERTAIN once it's done")
    → Trust source: authority inject (dad) + experience confirmation
        (dad has kept his word before)
  Strength: FAINT → BUILDING
    → First time dad says it: faint (not yet compiled)
    → Dad keeps his word 5 times: building (pattern forming)
    → Dad keeps his word 50 times: compiled ("dad says = true")
  Cost: LOW (1–2 hours of effort, no risk)
  Domain: real ✓ (games ARE real, dad CAN allow it)
  Source: ④ External Inject (dad) → ③ gradually compiles
  Outcome: finishes → plays games → smooth melody → anchor CONFIRMED

  ⭐ NOTE: Dad DOESN'T keep his word → trust BREAKS:
    → "Done!" → dad: "it's too late, play tomorrow"
    → Trust violation → next time dad makes a promise → trust LOWER
    → Repeated many times → compiled "dad promises ≠ dad does"
    → = Anchor-Schema.md §2 Trust violation → ripple effect
    → = Impact NOT JUST on games — but on ALL promises from dad


── S2. COOKING DINNER FOR THE FAMILY ──

  Situation: Parent cooks dinner. 45 minutes. Family eats together.

  Anchor: schema "cook → family eats well → joy" (compiled routine)
  Trust: positive + mixed (empathy-driven: reward = OTHERS being happy)
  Strength: COMPILED (thousands of meals → auto-pilot)
  Cost: LOW (45 minutes, familiar effort, no risk)
  Domain: real ✓ (cook → food → family eats = domain confirmed every meal)
  Source: ③ Compiled (daily routine) + empathy reward (seeing family eat)
  Outcome: smooth melody (routine + empathy reward)

  ⭐ NOTE: Empathy-driven anchor:
    → The MAIN reward is NOT "I eat well"
    → But: "seeing spouse/children eat well → mirror reward → MY OWN opioid"
    → = Anchor-Schema reward through the Empathy-Mirror
    → = Why parents cooking DOESN'T feel like "a burden" despite the effort
      (because reward = empathy, not self-reward)
    → = The "anchor for OTHERS" pattern


── S3. BORED → PFC SCAN → CALL A FRIEND ──

  Situation: Sitting at home on Sunday afternoon. Bored. Scrolling the phone.
  Still bored. PFC scans → "call friend B?" → body previews → "seems fun" → calls.

  Anchor: schema "call friend B → fun" (PFC-built, body preview confirms)
  Trust: positive, LIGHT (haven't called yet → not confirmed → just enough
      trust to try)
  Strength: FAINT (just formed from PFC scan → not yet compiled)
  Cost: LOW (a phone call = small effort, risk ≈ 0)
  Domain: real ✓ (friend B IS REAL, calling IS POSSIBLE)
  Source: ① PFC (generates option) → body preview (confirms) → becomes anchor
  Outcome:
    → Friend B answers, conversation is fun → anchor CONFIRMED → trust BUILDS
    → Friend B is busy → anchor BREAKS → PFC re-scans → next option

  ⭐ NOTE: §1 Flow steps ②③④⑤ play out IN A FEW MINUTES:
    → ② scan compiled: "scroll? watch TV?" → compiled but NOT ENOUGH
        (still bored)
    → ③ PFC generates: "call friend B?" (micro Imagine-Final)
    → ④ body evaluates: previews "talking with B" → light opioid → SELECTS
    → ⑤ anchor amplifies: hand reaches for phone → calls
    → = SHORT flow but COMPLETE 6 steps


── S4. FINISHING A CHAPTER → UNDERSTANDING THE CONCEPT ──

  Situation: Reading a framework book. Hard chapter. 2 hours.
  "Finish this chapter → will understand the concept."

  Anchor: schema "read → understand" (compiled learning experience +
      Imagine-Final "aha moment")
  Trust: positive ("have read → understood → reward before", compiled experience)
  Strength: COMPILED (regular reader: compiled "reading = will eventually
      understand")
    → BUT: each NEW chapter = micro-uncertainty (don't know exactly when
        understanding will click)
  Cost: LOW–MEDIUM (2 hours focus, PFC bandwidth, energy)
  Domain: semi-real ("read → understand" USUALLY holds but NOT guaranteed
      per chapter)
  Source: ③ Compiled (reading experience) + ① PFC (navigates difficult passages)
  Outcome: understanding → "aha!" → body reward → trust CONFIRMED → continues
      to next chapter

  ⭐ NOTE: The "aha moment" = micro-anchor COMPLETE:
    → Reading → dissonance (don't understand yet) → continues → suddenly GETS it
    → = Anchor "read → understand" CONFIRMED → mini-smooth melody
    → = Why "aha" feels RICHER than simply "information received"
      → Because: it = ANCHOR COMPLETE + trust confirm + dissonance resolved
    → = Melody-Arc.md: mini-arc complete → harmony


── S5. 30 MINUTES AT THE GYM ──

  Situation: Going to the gym. 30 minutes cardio. Sweat.

  Anchor: schema "exercise → healthy body → endorphins" (compiled + body-level)
  Trust: positive (body CONFIRMS every time → compiled trust)
  Strength: DEPENDS ON THE PERSON:
    → Long-time exerciser: COMPILED (automatic — "time to go → go" →
        no deliberation needed)
    → New exerciser: FAINT → BUILDING (needs PFC to hold "must go exercise")
  Cost: LOW–MEDIUM (30 minutes + physical effort + getting to the gym)
  Domain: real ✓ (exercise → endorphins → body health = physiology confirmed 🟢)
  Source: ③ Compiled (experienced person) OR ① PFC (new person → deliberate decision)
  Outcome: finishes → endorphins → body reward → trust INCREASES

  ⭐ NOTE: NEW exerciser vs EXPERIENCED exerciser:
    → New: Anchor = PFC-held (①) → strength FAINT → cost FEELS high → easy to quit
    → Experienced: Anchor = compiled (③) → strength COMPILED → cost FEELS low → automatic
    → = SAME behavior (30 min at gym) → DIFFERENT anchor strength → DIFFERENT experience
    → = Why "21 days to form a habit" (oversimplified but DIRECTIONALLY right):
      PFC-held (①) → repeat → compile (③) → anchor strength INCREASES → automatic
    → = The ACTUAL cost of an anchor DECREASES as strength increases
```

## MEDIUM-ANCHOR (days → months)

> Trust NEEDS TO BE STRONGER because cost INCREASES + domain feedback IS SLOWER.
> PFC participates VISIBLY (plans, navigates, holds through longer dissonance).
> REAL risk begins + REAL opportunity cost begins.
> = Where "belief" BEGINS TO BE SERIOUSLY TESTED.

```
── D1. COMPLETING A 3-MONTH ONLINE COURSE → GAINING A NEW SKILL ──

  Situation: Developer wants to learn Rust. Enrolls in a 3-month course.
  2 hours every evening. "Finish course → gain new skill → better career options."

  Anchor: schema "learn Rust → skill → career ↑" (PFC-built Imagine-Final)
  Trust: positive, MIXED sources
    → Evidence: "already learned Python successfully before" → compiled trust
        "I can learn"
    → Social: "Rust is trending" → social proof
    → PFC: "market needs Rust" → rational evaluation
  Strength: BUILDING (just starting → a few weeks → gradually clearer)
  Cost: MEDIUM
    → Time: 3 months × 2 hours/evening = ~180 hours
    → Energy: PFC bandwidth every evening (after a day of work = tired)
    → Opportunity: 180 hours NOT gaming / watching films / resting
    → Risk: low (course fee + time, no career risk)
  Domain: real ✓ (Rust = real language, skill = real value)
    → BUT: "better career options" = semi-real (likely, not guaranteed)
  Source: ① PFC (decides) + ④ social (trending) + ③ compiled ("I can learn")
  Outcome:
    → Week 1–2: novelty → body reward → "this is great!" → trust BUILDS
    → Week 3–6: difficulty increases (ownership/borrowing) → dissonance
      → Trust SUFFICIENT? → continues. Trust NOT SUFFICIENT? → "too hard" → quits.
    → Month 3: completes → builds mini-project → "I DID IT!" → anchor COMPLETE

  ⭐ NOTE: "The Valley" (Melody-Arc.md §3.3) APPEARS HERE:
    → Mid-course = furthest from start AND finish → dissonance AT ITS HIGHEST
    → Trust NEEDS ≥ Cost at this point → if not → GIVES UP
    → = Why many people quit online courses IN THE MIDDLE (not at the start
        or end)
    → = Initial trust (novelty) NOT ENOUGH for the cost IN THE MIDDLE
        (dissonance peak)


── D2. COMPLETING A MAJOR PROJECT AT WORK — 2 MONTHS ──

  Situation: Manager assigns a major project. 2-month deadline. Team of 5.
  "Complete it → manager's good assessment → bonus/promotion."

  Anchor: schema "complete project → recognition + reward" (mixed sources)
  Trust: MIXED (positive + negative simultaneously)
    → Positive: "done → bonus → career ↑" (reward ahead)
    → Negative: "not done → poor assessment by manager → layoff risk" (threat)
    → = 2 types of trust TOGETHER SUPPORTING one anchor → anchor STRONGER
        than single trust
  Strength: BUILDING → GRADUALLY COMPILED
    → Day 1: PFC holds ("must finish the project")
    → Week 2: daily routine forms → gradually compiles
    → Month 2: near deadline → compiled + threat amplifies → EXTREMELY STRONG
  Cost: MEDIUM → HIGH
    → Time: 2 months of focus
    → Energy: PFC intensive (coordinate team + solve problems)
    → Risk: career risk if fails
    → Opportunity: sacrifices personal time, connection
  Domain: real ✓ (project = real deliverable, manager's assessment = real domain
      feedback)
  Source: ④ External (manager assigns) + ① PFC (plans) + ③ gradually compiles
      (daily routine)
  Outcome: completes → reward → smooth melody → trust CONFIRMED for next project

  ⭐ NOTE: Positive + Negative trust SIMULTANEOUSLY:
    → = Threat.md + Novelty.md: 2 drives SAME DIRECTION → intensity HIGH
    → = Why deadlines are EFFECTIVE: PUSH (threat) + PULL (reward) = same anchor
    → = BUT: if there is ONLY negative trust (threat only → no reward):
      → Done → relief ("Phew") but NOT "worth it"
      → = Unsustainable long-term (Anchor-Schema.md §2 negative trust)


── D3. SAVING FOR 6 MONTHS → BUYING SOMETHING WANTED ──

  Situation: Wants a new laptop. 6 months saving 3 million VND/month.
  "6 months → enough money → buy → upgrade workflow."

  Anchor: schema "save → buy laptop → upgrade" (PFC plan + body preview)
  Trust: positive ("arithmetic is certain: 3M × 6 = 18M = enough")
    → Trust PARTICULARLY HIGH because: domain = ARITHMETIC (extremely real,
        extremely verifiable)
    → Different from "effort → success" (semi-real): "save → enough money"
        = CERTAIN
  Strength: BUILDING → COMPILED
    → Month 1: PFC holds "resist spending"
    → Month 3: sees balance INCREASING → body confirms → trust INCREASES
        → compiles
    → Month 5: nearly enough → compiled + excitement → anchor EXTREMELY STRONG
  Cost: MEDIUM
    → Time: 6 months
    → Sacrifice: curbing spending (monthly opportunity cost)
    → Risk: EXTREMELY LOW (money is still money, laptop is still for sale)
  Domain: real ✓ (arithmetic + commerce = extremely clear domain)
  Source: ① PFC (plans) → ③ gradually compiles (monthly savings routine)
  Outcome: buys laptop → body reward (new tool) → smooth melody

  ⭐ NOTE: ARITHMETIC domain = HIGHEST trust:
    → Because: 100% verifiable, no uncertainty
    → "3 × 6 = 18" = domain feedback CERTAIN every month (check the balance)
    → = Why FINANCIAL goals are easier to trust than ABSTRACT goals
      ("save 18M" = clear vs "become a better person" = dim)
    → = Cost concept: cost LOW (risk ≈ 0) + trust HIGH (domain real)
        = easy to hold


── D4. 3 MONTHS AT THE GYM → VISIBLE BODY CHANGE ──

  Situation: Starts going to the gym. Goal: 3 months → visible body change.

  Anchor: schema "consistent training → body changes" (PFC + social + domain
      knowledge)
  Trust: positive, BUT domain feedback IS SLOW
    → Domain: physiology = real ✓ (exercise → muscle growth)
    → BUT: visible change = 6–12 WEEKS (not immediate)
    → = Trust must HOLD through 6 weeks WITH NO visible feedback
    → = "Trust gap" — the period when trust IS NOT REINFORCED by results
  Strength: FAINT → BUILDING (if they don't quit)
  Cost: MEDIUM
    → Time: 3 months × 4–5 sessions/week × 1 hour
    → Energy: physical effort + soreness + routine change
    → Opportunity: 60+ hours
  Domain: real ✓ (exercise science = verified 🟢)
  Source: ① PFC (decision) + ④ social (gym culture, before/after photos)
          + ③ gradually compiles (if routine forms)
  Outcome:
    → Week 1–2: soreness → "it hurts so much" → trust TESTED
    → Week 3–6: NO visible change yet → "is this even working?" → trust DECREASES
      → = Trust gap = MOST DANGEROUS
      → IF social support (gym buddy, trainer) → bridges through the gap
      → IF solo + no results visible → QUITS (anchor collapses)
    → Week 8–12: visible change → body confirms → trust JUMPS → compiles

  ⭐ NOTE: "Trust Gap" = the space between START and FIRST DOMAIN FEEDBACK:
    → Domain real ✓ + BUT feedback SLOW → trust must hold WITHOUT evidence
    → = The EASIEST PERIOD TO QUIT (because trust builds on evidence →
        NO evidence yet)
    → = Why "support" (trainer, buddy, social) is IMPORTANT during this period:
      → Social trust (source ④) BRIDGES through the trust gap → until
          domain confirms
    → = Melody-Arc.md §4 ④ Social Mirror: "someone who's been through it
        sharing their progress"


── D5. STUDENT — FINAL EXAMS, STUDYING FOR 3 WEEKS ──

  Situation: 3 weeks before final exams. Many subjects. Must study.
  "Study enough → pass → advance → career path continues."

  Anchor: schema "study → pass → advance" (compiled academic + threat + reward)
  Trust: STRONG MIX (positive + negative on the SAME ANCHOR, same direction)
    → Positive: "pass → proud + parents happy + career continues" (reward)
    → Negative: "fail → lose a year + parents upset + embarrassment" (threat)
    → = 2 types of trust REINFORCE → anchor STRONG
  Strength: COMPILED (academic schema from 12+ years of schooling)
    → "Exam → study" = compiled deep → auto-activates when exam approaches
    → BUT: each NEW exam = micro-uncertainty (how hard is this subject?)
  Cost: MEDIUM
    → Time: 3 weeks intensive
    → Energy: PFC bandwidth (memorize + understand + practice)
    → Sacrifice: social time, entertainment, sleep (!)
    → Risk: moderate (failing = real consequence, but NOT life-threatening)
  Domain: real ✓ (study → high score = strong correlation, verified per exam)
  Source: ③ Compiled (12+ years) + ④ External (educational system, parents)
          + ① PFC (concrete study plan per subject)
  Outcome: passes → trust CONFIRMED → compiles MORE → "I can do it" → next arc

  ⭐ NOTE: SACRIFICING SLEEP = anchor overrides body-base:
    → Anchor "pass the exam" STRONG → overrides L1 "need sleep"
    → = Anchor-Schema CAN override body-base IF trust is strong enough
    → = Threat.md §5 ④: threat overrides body-base warning
    → = EFFECTIVE short-term (really passes) but BODY COST (neural wear)
    → = Why staying up all night studying = "works but is not sustainable"
```
