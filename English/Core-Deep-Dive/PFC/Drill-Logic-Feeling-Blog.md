### 0.2 — Core thesis (precise claim)

```
⭐ WHAT THE BLOG CLAIMS:

  "Logic" and "Intuition" are NOT two different cognitive mechanisms.
  They are OBSERVER LABELS for the same mechanism:
  compiled body-direct processing (Body-Knowing).

  The real axis: Compiled (automatic, body-direct, cost ≈ 0)
             vs. Fresh (PFC-drafted, working-memory-dependent, costly).

  What determines the label:
    SHAREABILITY of the domain output.
    Math (deterministic) → Body-Knowing converges across people → "logic"
    Psychology (probabilistic) → Body-Knowing diverges → "intuition"
    Body-state (private) → Body-Knowing cannot be shared → "feeling"

  Labels = f(observer's ability to verify), NOT mechanism difference.

⭐ WHAT THE BLOG DOES NOT CLAIM:

  ✗ "Logic is useless" (Fresh processing essential for novel domains)
  ✗ "Trust your gut always" (Body-Knowing can be Self-Referencing)
  ✗ "Rationality is overrated" (formalization, teaching, knowledge accumulation = vital)
  ✗ "There is no difference between System 1 and System 2"
     (there IS a difference — but it's compilation level, not mechanism type)
  ✗ "Kahneman was wrong" (Kahneman was RIGHT about the separation —
     the blog reframes WHAT the separating variable is)

⭐ CONFIDENCE LEVELS:

  🟢 Expert intuition = compiled pattern recognition (Klein, Chase & Simon, Kahneman & Klein)
  🟢 PFC creates post-hoc narratives (Gazzaniga, Haidt, Nisbett & Wilson)
  🟢 Body-feedback necessary for decisions (Damasio)
  🟡 "Logic/Intuition = observer labels for same mechanism" = framework synthesis
  🟡 "Shareability determines label" = framework synthesis (logically sound, not tested)
  🟡 "Compiled/Fresh = the real axis" = novel reframe (consistent with data, not proven)
  🔴 Exact ratios (95/5, etc.) = illustrative, not measured
```

#### R1: KAHNEMAN & KLEIN 2009

```
FULL CITATION:
  Kahneman, D. & Klein, G. (2009). Conditions for intuitive expertise:
  A failure to disagree. American Psychologist, 64(6), 515-526.

TYPE: Theoretical/integrative paper (NOT experiment).
  Dialogue between two opposing traditions:
  Kahneman (heuristics & biases, skeptical of intuition) +
  Klein (naturalistic decision making, champion of expert intuition).

KEY FINDING — 2 CONDITIONS FOR RELIABLE INTUITION:
  ① High-validity environment — stable, learnable regularities
     (chess, firefighting, nursing = high validity;
      stock picking, long-range politics = low validity)
  ② Adequate opportunity to learn — sufficient practice with
     prompt, reliable feedback

  Both conditions NECESSARY. Expert in unpredictable environment
  = confident but unreliable. Novice in predictable environment
  = not yet compiled.

  CRITICAL ADDITIONAL POINT:
  Subjective confidence is NOT a reliable indicator of accuracy.
  Experts in low-validity environments feel JUST AS CONFIDENT
  as experts in high-validity environments.

WHY STRONGEST FOR OUR THESIS:
  → Shows expert intuition = compiled expertise when conditions met
  → "Valid environment + feedback" = our "domain feedback = only arbiter"
  → Both researchers AGREED: intuition reliability depends on
     compilation conditions, not on content type (logic vs feeling)
  → Directly supports: the variable is compilation quality, not mode

LIMITATIONS:
  → Conceptual framework, not formal model with sharp boundaries
  → "High validity" vs "low validity" = judgment call, not precise cutoff
  → Gray zones acknowledged (clinical psychology, some medicine)
  → Fractionation: expert may be reliable for SOME aspects, not all

BLOG USAGE: Core citation. Anchor for "compilation conditions = real variable."
  Quote: "The confidence people have in their intuitions is not
  a reliable guide to their validity."
```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

#### R2: CHASE & SIMON 1973

```
FULL CITATION:
  Chase, W.G. & Simon, H.A. (1973). Perception in chess.
  Cognitive Psychology, 4(1), 55-81.

  Builds on: de Groot, A.D. (1946/1965). Thought and Choice in Chess.

TYPE: Experimental. N=3 (master, Class A player, beginner).
  Two tasks: reconstruct board positions from (a) real games,
  (b) randomized pieces.

KEY FINDING:
  Masters reconstructed meaningful game positions with far fewer
  glances and higher accuracy. On RANDOM arrangements,
  masters performed NO BETTER than beginners.

  → Expertise stored as meaningful patterns (chunks), not raw
     perceptual superiority
  → Estimated ~50,000 chunks in master's library
     (later revised upward to ~100,000 by some researchers)

WHY IMPORTANT FOR OUR THESIS:
  → Proves expertise = compiled pattern library
  → Same chess board, same eyes — different compiled base → different performance
  → "Seeing" the right move = pattern recognition, not deliberation
  → Directly maps to: compiled processing = Body-Knowing, regardless of domain

LIMITATIONS:
  → N=3 (extremely small, even for era)
  → Replicated and extended by many studies (Gobet & Simon 1996, etc.)
  → "Chunking" theory refined → "templates" (Gobet & Simon 1996):
     templates have fixed core + variable slots (more flexible than rigid chunks)
  → 50,000 chunk estimate revised, exact number uncertain
  → de Groot (1946) made the original observation; Chase & Simon added mechanism

BLOG USAGE: Evidence for "expertise = compiled patterns."
  Senior dev "seeing" bug = chess master "seeing" move = same mechanism.
  Note: cite de Groot 1946 for original observation, Chase & Simon for chunking.
```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

#### R3: KLEIN 1998 (Recognition-Primed Decision Model)

```
FULL CITATION:
  Klein, G.A. (1998). Sources of Power: How People Make Decisions.
  MIT Press.

  Original research:
  Klein, G.A., Calderwood, R., & Clinton-Cirocco, A. (1986/2010).
  Rapid decision making on the fire ground.
  Human Factors, 28(5), 576-585.

TYPE: Naturalistic Decision Making (NDM). Field interviews with
  experienced fireground commanders using Critical Decision Method
  (structured retrospective interviews).

KEY FINDING:
  In >80% of decision points, experienced commanders used
  pattern recognition to directly identify the situation as
  typical of a known prototype. They did NOT generate and
  compare multiple options.

  RPD process:
  ① Pattern match → recognize situation type
  ② Mental simulation → evaluate if candidate action works
  ③ If simulation reveals problems → modify or try next prototype
  = Serial evaluation, NOT parallel comparison

WHY IMPORTANT FOR OUR THESIS:
  → Experts DON'T "think logically" (weigh options)
  → Experts RECOGNIZE (compiled patterns fire → immediate action)
  → Same mechanism across domains: firefighting, nursing, military
  → Observer labels this "intuition" — but it's compiled expertise
  → Directly supports: "expert logic" = compiled Body-Knowing

LIMITATIONS:
  → Evidence base primarily QUALITATIVE (retrospective interviews)
  → Some decision researchers note evidence is "fairly thin" by
     experimental psychology standards
  → RPD EXPLICITLY requires extensive domain experience (not universal)
  → RPD fails in genuinely novel situations (no matching patterns)
  → Does not account for emotional distortion of pattern recognition
  → RPD assumes rapid, reliable feedback loops — fails in domains
     with delayed/ambiguous feedback (long-term policy, rare diseases)
  → "Intuition is reliable" = ONLY within Kahneman & Klein 2009 conditions

BLOG USAGE: Core evidence for "experts recognize, don't deliberate."
  Pair with Kahneman & Klein 2009 for conditions.
  Firefighter example → senior dev example → same mechanism.
```

#### R4: EINSTEIN/HADAMARD 1945

```
FULL CITATION:
  Einstein, A. (1945). "A Testimonial from Professor Einstein."
  Appendix II in Hadamard, J., An Essay on the Psychology of
  Invention in the Mathematical Field. Princeton University Press.
  pp. 142-143.

TYPE: Primary source. Personal letter responding to Hadamard's
  questionnaire about mathematical thinking processes.

EINSTEIN'S EXACT WORDS:
  "The words of the language, as they are written or spoken,
  do not seem to play any role in my mechanism of thought.
  The psychical entities which seem to serve as elements in
  thought are certain signs and more or less clear images which
  can be 'voluntarily' reproduced and combined...

  The above-mentioned elements are, in my case, of VISUAL and
  some of a MUSCULAR type. Conventional words or other signs
  have to be sought for laboriously only in a SECONDARY STAGE,
  when the mentioned associative play is sufficiently established
  and can be reproduced at will."

  Also to Wertheimer: "never thought in logical symbols, but in
  images, feelings, and even musical architectures"

WHY IMPORTANT FOR OUR THESIS:
  → The most famous "logical thinker" in history describes his
     thinking as NON-VERBAL, body-based (visual + muscular)
  → Words/formalization = "secondary stage" = Fresh processing
  → The "logic" (math) came AFTER body-based pattern discovery
  → Directly challenges: "Einstein used logic" → he used Body-Knowing

LIMITATIONS — CRITICAL FOR LW AUDIENCE:
  → N=1 introspective self-report
  → Exactly the kind of evidence Nisbett & Wilson 1977 would warn
     about: people may not accurately report their own cognitive processes
  → Other scientists in Hadamard's survey gave DIFFERENT answers
     (Hadamard: vague visual; Polya: more verbal)
  → Einstein may be reporting the OUTPUT of unconscious processing,
     not the processing itself
  → HOWEVER: subsequent research on mental imagery (Kosslyn,
     Shepard & Metzler mental rotation) and embodied cognition
     DOES support non-verbal, visuospatial roles in cognition

  ⚠️ IRONY TO ACKNOWLEDGE IN BLOG:
  We cite Einstein's introspection about thinking non-verbally,
  while also citing Nisbett & Wilson showing introspection is unreliable.
  Blog should NOTE this tension explicitly — it signals intellectual honesty.

BLOG USAGE: Vivid illustration, NOT primary evidence.
  Use as opening hook or strong example.
  Pair with: "Note: this is introspective self-report, which has
  known limitations (Nisbett & Wilson 1977). We include it as
  illustration, not proof. The research evidence follows."
```

#### R5: GAZZANIGA — Split-Brain Confabulation

```
FULL CITATION:
  Gazzaniga, M.S. & LeDoux, J.E. (1978). The Integrated Mind.
  Plenum Press. pp. 132-161. (Introduces "interpreter" concept)

  Gazzaniga, M.S. (2000). Cerebral specialization and interhemispheric
  communication. Brain, 123, 1293-1326.

  Gazzaniga, M.S. (2005). Forty-five years of split-brain research
  and still going strong. Nature Reviews Neuroscience, 6, 653-659.

TYPE: Experimental studies with corpus callosotomy patients
  (epilepsy surgery severing corpus callosum).

THE CLASSIC EXPERIMENT:
  Patient shown: chicken claw (right visual field → left hemisphere)
                 snow scene (left visual field → right hemisphere)
  Right hand pointed to chicken (matching claw).
  Left hand pointed to snow shovel (matching snow).

  When asked WHY left hand pointed to shovel:
  Patient did NOT say "I don't know."
  Patient said: "And you need a shovel to clean out the chicken shed."

  Left hemisphere CONFABULATED a coherent but completely false explanation.
  → The "Interpreter" — compulsively generates causal narratives,
     even with zero access to actual cause. Never says "I don't know."

WHY IMPORTANT FOR OUR THESIS:
  → Literal demonstration of PFC = Lawyer
  → Narrative construction ≠ truth reporting
  → "Logical explanation" can be complete confabulation
  → Supports: you cannot trust "I know why I did that" at face value

LIMITATIONS:
  → Patient population EXTREMELY small (<12 well-studied worldwide)
  → Patients had severe epilepsy + radical surgery
  → Generalizing to healthy intact brains requires caution
  → "Interpreter" = Gazzaniga's theoretical construct,
     not precisely localized brain module
  → The concept developed across decades (1978→2005), not single study
  → Roser & Gazzaniga 2004 debated extent to which split-brain
     confabulation maps onto everyday rationalization

BLOG USAGE: Use as compelling illustration of PFC = Lawyer.
  MUST note: "These are split-brain patients — a tiny population
  with radical surgery. The generalization to intact brains is
  debated. However, the pattern — generating confident explanations
  without access to actual causes — has been observed in healthy
  populations by Nisbett & Wilson (1977) and Haidt (2001)."
```

#### R6: HAIDT 2001 — Social Intuitionist Model

```
FULL CITATION:
  Haidt, J. (2001). The emotional dog and its rational tail:
  A social intuitionist approach to moral judgment.
  Psychological Review, 108(4), 814-834.

TYPE: Theoretical/review paper (NOT experiment).
  Synthesizes evidence from social, cultural, evolutionary psychology
  and neuroscience against rationalist models of moral judgment.

KEY FINDING:
  Moral judgment primarily driven by rapid automatic intuitions ("dog").
  Moral reasoning typically post-hoc rationalization ("tail") —
  constructed AFTER judgment to persuade others.
  6-link model includes "social persuasion" link:
  reasoning serves social/rhetorical functions, not truth-seeking.

  "Moral dumbfounding" paradigm: participants condemn harmless-but-
  repugnant scenarios but struggle to articulate reasons when
  researcher removes stated justifications.

WHY IMPORTANT FOR OUR THESIS:
  → "Reasoning" in moral domain = post-hoc, not causal
  → Supports PFC = Lawyer (narrative serves body-level judgment)
  → Even in domains where people BELIEVE they're reasoning,
     the judgment arrives first

LIMITATIONS — IMPORTANT:
  → "Moral dumbfounding" data was NEVER formally published with
     full methodological details
  → Royzman, Kim & Leeman (2015) — careful replication showed
     many participants DO have articulable reasons. Participants
     rejected the researcher's stipulation that scenario was harmless.
     They weren't "dumbfounded" — they disagreed with premises.
  → Rationalist critics (Kennett & Fine 2009, Jacobson 2012):
     Haidt conflates "fast" with "non-rational." Compiled expertise
     CAN produce fast judgments that are reason-responsive.
  → Haidt's model DOES allow for occasional reasoning-to-judgment
     (Links 5 and 6) — often overlooked by citers.

  ⚠️ NOTE: The Kennett & Fine critique ACTUALLY SUPPORTS our thesis:
  "fast judgments can be reason-responsive" = "compiled processing
  can be accurate" = Body-Knowing with domain verification.

BLOG USAGE: Use for PFC = Lawyer argument.
  MUST note: "Haidt's model has been debated — particularly the
  'moral dumbfounding' evidence (Royzman et al. 2015). However,
  the core thesis — that moral reasoning often follows rather than
  precedes moral judgment — is supported by converging evidence
  from Gazzaniga and Nisbett & Wilson."
```

#### R7: NISBETT & WILSON 1977

```
FULL CITATION:
  Nisbett, R.E. & Wilson, T.D. (1977). Telling more than we can know:
  Verbal reports on mental processes.
  Psychological Review, 84(3), 231-259.

TYPE: Review paper + original experiments.

THE STOCKING EXPERIMENT:
  Researchers displayed 4 identical stockings, asked passersby
  to select best quality. Strong position effect: rightmost
  preferred 4:1. Participants gave 80 different reasons
  (knit, weave, sheerness, elasticity). NOT ONE mentioned position.

KEY FINDING:
  People frequently cannot accurately introspect on actual causes
  of their behavior. They generate plausible causal theories
  based on a priori beliefs about what SHOULD have influenced them.
  "Telling more than they can know."

WHY IMPORTANT FOR OUR THESIS:
  → Demonstrates PFC narrative ≠ actual cause
  → Even when people BELIEVE they're reporting logic, they're
     constructing post-hoc explanations
  → Supports: observer label "logic" may describe narrative,
     not actual processing

LIMITATIONS:
  → Strong claim ("little or no direct introspective access")
     has been SIGNIFICANTLY QUALIFIED since 1977
  → Access exists on a CONTINUUM, not all-or-nothing
  → Stocking experiment shows lack of access to positional/anchoring
     biases specifically — not necessarily ALL cognitive processes
  → Ericsson & Simon (1980, 1993): concurrent verbal reports
     (think-aloud protocols) CAN be valid. N&W critique applies
     most to after-the-fact explanations, not concurrent reports.
  → No sustained empirical research program continued the exact
     line of work — influence more conceptual than methodological

BLOG USAGE: Use for PFC = Lawyer argument.
  MUST qualify: "Nisbett & Wilson showed that retrospective
  explanations are often confabulated. Subsequent work (Ericsson
  & Simon, 1980) showed that concurrent reports can be more valid.
  The critique is strongest for after-the-fact explanations —
  which is precisely the context of 'why did you decide that?'"
```

#### R8: DAMASIO 1994 — Somatic Marker Hypothesis

```
FULL CITATION:
  Damasio, A.R. (1994). Descartes' Error: Emotion, Reason,
  and the Human Brain. Putnam/Grosset.

  Bechara, A., Damasio, A.R., Damasio, H., & Anderson, S.W. (1994).
  Insensitivity to future consequences following damage to human
  prefrontal cortex. Cognition, 50, 7-15. (Iowa Gambling Task)

TYPE: Clinical case studies + experimental (Iowa Gambling Task).

KEY CASE — PATIENT ELLIOT:
  vmPFC damage from tumor. IQ = 97th percentile.
  Logic, language, memory, attention: INTACT.
  Real-world decisions: CATASTROPHIC.
  Choosing between 2 appointment dates → 30 minutes of pros/cons
  without reaching conclusion. Choosing restaurant → entire afternoon.
  Lost job, marriage, savings.

IOWA GAMBLING TASK:
  Healthy participants develop anticipatory skin conductance responses
  (SCRs) before selecting from "bad" decks — body signals danger
  BEFORE conscious awareness. vmPFC patients did NOT develop
  anticipatory SCRs → continued choosing disadvantageously.

WHY IMPORTANT FOR OUR THESIS:
  → Logic intact + body-feedback broken = decision paralysis
  → Demonstrates: "pure logic" is INSUFFICIENT for real decisions
  → Body-feedback (Body-Knowing) is not optional — it's structural
  → Supports: Fresh processing alone cannot function

LIMITATIONS — SIGNIFICANT:
  → Dunn, Dalgleish & Lawrence (2006) — major critical evaluation
     in Neuroscience & Biobehavioral Reviews: concerns about
     novelty, parsimony, specification of the hypothesis
  → Maia & McClelland (2004, PNAS): IGT participants had MORE
     conscious knowledge than Damasio claimed. With sensitive
     questioning, participants COULD articulate which decks were bad.
     Undermines claim that decisions driven by unconscious somatic signals.
  → vmPFC patients show changes in personality, social behavior,
     emotional processing broadly — not purely decision deficit
  → "Somatic" in somatic marker debated: Damasio later acknowledged
     "as-if body loop" where brain simulates body states without
     actual peripheral feedback
  → Replication across patient groups inconsistent

BLOG USAGE: Use for "logic alone insufficient" argument.
  MUST note: "Damasio's somatic marker hypothesis has been
  qualified — Maia & McClelland (2004) showed participants had
  more conscious access than originally claimed. However, the
  core finding — that vmPFC patients with intact reasoning
  struggle with real-world decisions — remains clinically
  established and points to body-feedback as structurally
  necessary for decision-making."
```

#### R9: EVANS & STANOVICH 2013

```
FULL CITATION:
  Evans, J.St.B.T. & Stanovich, K.E. (2013). Dual-process theories
  of higher cognition: Advancing the debate.
  Perspectives on Psychological Science, 8(3), 223-241.

TYPE: Theoretical review and defense paper.

KEY ARGUMENTS:
  → Consensus defining features:
    Type 1 = autonomous (does NOT require working memory)
    Type 2 = requires "cognitive decoupling" + loads on working memory
  → EXPLICITLY ABANDON old laundry list (fast/slow, unconscious/conscious)
    as DEFINING features. Speed, consciousness = typical correlates, NOT definitional.
  → Default-interventionist architecture: Type 1 generates defaults;
    Type 2 can intervene (but often doesn't)
  → Type 1 is NOT one system — it's a COLLECTION of autonomous processes
    (TASS: The Autonomous Set of Systems)

WHY THIS MATTERS — CHALLENGE TO OUR THESIS:
  → If Type 1 and Type 2 are defined by WM dependence,
    and WM dependence is a REAL processing difference,
    then claiming "same mechanism" seems problematic.
  → LW audience knows Evans & Stanovich. They WILL raise this.

FRAMEWORK RESPONSE: See §2 below (dedicated section).

CRITICAL NOTES:
  → Evans & Stanovich explicitly DO NOT equate "intuitive" with
    "irrational." Type 1 CAN implement learned rules and logical
    patterns. THIS SUPPORTS OUR THESIS.
  → Ongoing debate: Melnikoff & Bargh (2018) — defining features
    don't consistently co-occur. De Neys (2012, 2018) — conflict
    detection is ITSELF a Type 1 process, blurring boundary.
  → Pennycook et al. (2015) — some "intuitive" responses are
    actually CORRECT (against bias that Type 1 = error-prone)

BLOG USAGE: MUST address directly (§2 of this drill file).
  Do NOT ignore — LW readers will see this as a gap.
  Address as: "Evans & Stanovich correctly identify working memory
  dependence as a real processing difference. We argue this
  difference reflects compilation stage, not mechanism type."
```

#### R10: LIBET 1983/1985

```
FULL CITATION:
  Libet, B., Gleason, C.A., Wright, E.W., & Pearl, D.K. (1983).
  Time of conscious intention to act in relation to onset of
```

## §2 — EVANS & STANOVICH CHALLENGE + FRAMEWORK RESPONSE

> **This is the most important section for LW/HN audience.**
> **If the blog doesn't address this, it will be dismissed.**

### 2.1 — The challenge

```
EVANS & STANOVICH (2013) CLAIM:
  Type 1 processing = autonomous, does NOT require working memory
  Type 2 processing = requires cognitive decoupling, loads on WM

  This is a REAL processing difference — not just speed or consciousness.
  Working memory dependence is measurable, falsifiable, and robust.

  IF Type 1 and Type 2 are defined by this difference,
  AND this difference is a mechanism difference,
  THEN claiming "same mechanism" seems contradictory.

  LW READER WILL ASK:
  "You say logic and intuition are the same mechanism.
  But Evans & Stanovich show they differ in working memory
  dependence — that's a real difference. How do you reconcile?"
```

### 2.2 — Framework response

```
⭐ THE RESPONSE: WM DEPENDENCE = COMPILATION STAGE, NOT MECHANISM TYPE

  ANALOGY (tech audience):

    Consider COMPILED CODE vs INTERPRETED CODE.
    Compiled code: runs directly on hardware, fast, no interpreter needed.
    Interpreted code: runs through interpreter, slower, requires runtime.

    Are these "two different mechanisms"?
    NO — they're the SAME code at different compilation stages.
    The compiled version no longer needs the interpreter (= working memory).
    The interpreted version still requires it.

    Working memory ≈ interpreter/compiler runtime.
    Compiled processing bypasses it because the path is already built.
    Fresh processing requires it because the path is being constructed.

  PRECISE ARGUMENT:

  ① MECHANISM: Hebbian pattern matching + prediction error detection
     This is the SAME mechanism for both Type 1 and Type 2.
     Both ultimately rely on patterns (compiled or in-construction)
     being matched against current input.

  ② COMPILATION STAGE:
     Type 1 (autonomous) = pattern ALREADY compiled → fires automatically
       → bypasses WM because path exists → cost ≈ 0
     Type 2 (WM-dependent) = pattern being CONSTRUCTED → requires WM
       → needs WM as workspace for novel combination → cost > 0

  ③ LEARNING TRAJECTORY:
     What is Type 2 today becomes Type 1 tomorrow.
     Math student: algebra = Type 2 (effortful, WM-heavy).
     Math professor: algebra = Type 1 (automatic, no WM needed).
     SAME person, SAME domain, DIFFERENT compilation stage.
     = The processing MOVED from Type 2 to Type 1 through compilation.

  ④ WHY THIS MATTERS:
     If Type 1 and Type 2 were genuinely different mechanisms,
     you couldn't LEARN your way from one to the other.
     But you can — and that's what expertise IS.
     Klein's firefighters START as Type 2 (deliberate analysis)
     and BECOME Type 1 (pattern recognition) through experience.
     = Evidence that the mechanism is the same; the stage differs.

  ⑤ WHAT WE AGREE WITH EVANS & STANOVICH ON:
     → WM dependence IS a real, measurable processing difference
     → Type 1 CAN be error-prone (when compiled for wrong domain)
     → Type 2 CAN correct Type 1 (when domain feedback available)
     → The two "types" interact (default-interventionist architecture)

  ⑥ WHERE WE DIVERGE:
     → We argue WM dependence = CONSEQUENCE of compilation stage
     → Not a DEFINING feature of a separate mechanism
     → The "two types" are two STAGES of one learning process
     → Speed, consciousness, effort = correlates of compilation level

  ⑦ ANTICIPATED COUNTER TO ANALOGY:
     "Compiled code and interpreted code run on DIFFERENT execution
     environments (machine code vs interpreter runtime). Same source
     code ≠ same execution mechanism."

     RESPONSE:
     → Fair point — the analogy is imperfect (as all analogies are).
     → We DON'T claim Type 1 and Type 2 are identical in OPERATION.
        They clearly differ: WM involvement, speed, conscious access.
     → What we claim: they represent STAGES of the SAME LEARNING PROCESS,
        not two PERMANENT cognitive types.
     → The decisive evidence is NOT the analogy — it's the TRANSITION:
        A specific skill MOVES from Type 2 → Type 1 through practice.
        If they were truly different MECHANISMS (like vision vs hearing),
        this transition through mere practice would not be possible.
     → The analogy serves to make "same process, different stage"
        intuitive for tech audience. The evidence is the learning trajectory.
     → Blog should use analogy as ILLUSTRATION, then pivot to EVIDENCE
        (Klein, Schneider & Shiffrin, Logan, Fitts & Posner, Dreyfus).

  ⚠️ IMPORTANT: Evans & Stanovich THEMSELVES abandoned the old
  "fast/slow, unconscious/conscious" attributes as DEFINING features.
  They kept WM dependence as the defining feature.
  Our argument: WM dependence is ALSO a correlate (of compilation),
  not a mechanism difference.

  ⚠️ HONEST CAVEAT:
  This is a 🟡 framework synthesis claim, not proven.
  It's consistent with the data, but not definitively established.
  The blog should present this as: "We propose that WM dependence
  reflects compilation stage rather than mechanism type.
  This is testable: [falsification criteria]."

🟢 Evans & Stanovich 2013: Type 1/2 distinction
🟢 Expertise trajectory: Klein 1998, Ericsson 1993
🟡 WM dependence as compilation-stage correlate = framework synthesis
```

### 2.3 — Supporting evidence for the response

```
EVIDENCE THAT TYPE 2 → TYPE 1 THROUGH COMPILATION:

  🟢 Schneider & Shiffrin (1977): controlled → automatic processing
     through practice. Foundational demonstration that effortful
     processing becomes automatic with repetition.

  🟢 Logan (1988) — Instance Theory of Automatization:
     "Automaticity is memory retrieval." Each instance of performing
     a task is stored; with enough instances, retrieval replaces
     computation. = Compilation in our terms.

  🟢 Anderson (1982) — ACT theory: declarative → procedural

  🟢 Fitts & Posner (1967) — 3 stages of motor learning:
     Cognitive (explicit, WM-heavy) → Associative → Autonomous
     = Fresh → partially compiled → fully compiled

  🟢 Dreyfus & Dreyfus (1980/2005) — 5-stage expertise model:
     Novice → Advanced Beginner → Competent → Proficient → Expert
     Novice = rule-following (Type 2). Expert = intuitive (Type 1).
     = Continuous compilation trajectory.

  ALL show the SAME pattern: effortful → automatic through experience.
  = Type 2 → Type 1 = compilation, not mechanism switch.
```

```
  They're not a cognitive type. A human USING formal logic still
  relies on compiled pattern recognition to navigate the formal system.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  RESPONSE:
  It's not. The framework VALUES rationality — specifically:
  → Fresh processing is ESSENTIAL for novel domains (no compiled base)
  → Formalization enables teaching, accumulation, collective progress
  → Domain verification (the scientific method) IS the arbiter
  → "Trust but verify" applies to BOTH logic and intuition

  What the framework challenges: the HIERARCHY that places
  "rational/logical" above "intuitive/emotional" as a processing mode.
  Both are partial. Both can be wrong. Both are needed.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

⭐ OBJECTION 5: "Kahneman showed biases come from System 1"

  RESPONSE:
  Kahneman showed that CERTAIN HEURISTICS produce systematic errors
  in CERTAIN CONTEXTS. The framework reframes this:

  Cognitive biases = Body-Knowing compiled for ANCESTRAL environment,
  applied in MODERN environment where the patterns no longer match.

  Example: availability heuristic = Body-Knowing that
  "easily recalled examples = frequent." This was ACCURATE
  in small-group ancestral environment (what you personally
  encountered WAS representative). It's INACCURATE in
  media-saturated modern environment (what you recall is
  determined by media selection, not actual frequency).

  The fix is NOT "suppress System 1."
  The fix IS "re-compile for current domain"
  (which is what debiasing training attempts to do).

  Note: Kahneman HIMSELF (with Klein, 2009) acknowledged that
  System 1 is reliable under the right conditions.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

⭐ OBJECTION 6: "What about formal mathematics/logic as a domain?"

  RESPONSE:
  Formal systems (propositional logic, set theory, proof systems)
  are CULTURAL TOOLS — externalized verification procedures
  developed over centuries. They're domain-specific artifacts,
  not a cognitive type.

  A mathematician USING formal logic still relies on:
  → Compiled pattern recognition (seeing proof structure)
  → Body-Knowing (the "sense" that a proof direction is right)
  → Fresh processing (constructing novel proof steps)

  Einstein: "words sought laboriously in a secondary stage"
  = formalization comes AFTER body-based pattern discovery.

  The formal system provides a VERIFICATION FRAMEWORK —
  a way to check whether Body-Knowing arrived at a correct result.
  This is enormously valuable. But the processing USED to navigate
  the formal system is still compiled pattern matching.

  Analogy: a ruler is a measurement tool.
  Using a ruler doesn't mean your hand operates differently
  than when you estimate by eye. Same hand, different tool.
```
