## §2 — Anchor/Label System: 3 Tiers

Chunks exist BEFORE labels (F1 evidence: infants at 6 months have chunks, no words yet). Labels ATTACH to already-existing chunks, enabling chunks to participate in logic chains.

### §2.1 — Tier 1: Individual Anchors (only 1 person knows)

```
= Anchors that ONLY THE INDIVIDUAL holds, cannot share (or very hard to share)

Examples:
  • Felt sense "something is off" → no label yet, only body knows
  • Personal memory image: "the sunset that evening" → only I saw it
  • Grandmother's house scent → triggers a childhood chunk, only I have that association
  • Body sensation when hearing a particular song → personal body-state association
  • Internal image: "how I imagine the machine working" → not yet drawn

Properties:
  → CANNOT communicate (or requires large effort to describe)
  → CANNOT chain well into logic (PFC struggles to hold what has no clear label)
  → BUT: this is the FOUNDATION — every label BEGINS here
  → Somatic-Articulation Loop: Individual anchor → find label → becomes Group/Global

  ⭐ Einstein example:
  "I ride a light beam — what do I see?" = Individual anchor (body simulation)
  → YEARS of articulation → "spacetime curvature" = Global anchor (physics label)
  → Transition: Individual → Global via Somatic-Articulation process
```

### §2.2 — Tier 2: Group Anchors (shared within a group)

```
= Anchors that A GROUP shares through common experience, BUT without a formal label
  in global knowledge

Examples:
  • "That bug in the garden" → 2 siblings both know it, saw it, can say "the bug
    from before" → UNDERSTOOD, but a scientist would use a different name
  • "That thunderclap" → a group of friends all heard it, remember it, use it as a
    reference ("remember that huge thunder?") → shared experience = shared anchor
  • "The tree at the corner" → neighbors know it, use it for directions:
    "turn left at THE TREE" → local shared anchor
  • Team jargon: "the chicken bug" → dev team knows, outsiders do not
  • Family: "dad's way of doing things" → siblings understand immediately, outsiders do not
  • "The theme music from game X" → fan community knows it, humming it = understood

Properties:
  → CAN communicate within the group (shared experience = shared chunk)
  → CANNOT communicate outside the group (requires translation → loses nuance)
  → CAN chain logic (within the group):
    "The garden bug → it crawled onto our hand that day → got bitten → swelled up"
    → Both understand the plan: "next time we see that bug → stay away"
  → This is how HUMANS PLANNED BEFORE FORMAL LANGUAGE EXISTED:
    Two people see a tiger → point → run = Group anchor plan
    NO NEED to know the word "tiger" — only need to BOTH SEE + BOTH KNOW

  ⭐ Group anchor = MOST DAILY COMMUNICATION (% = illustrative, not measured):
  → Couples talking: 80% = group anchors (shared experience)
  → Team meeting: 60% = group jargon + shared context
  → Friends: "remember THAT DAY?" = group anchor trigger
  → Formal language only NEEDED when talking to PEOPLE OUTSIDE THE GROUP
```

### §2.3 — Tier 3: Global Anchors (shared across all of humanity)

```
= Anchors that have been FORMALIZED — all trained people understand them the same way

Examples:
  • Word "chó" (Vietnamese), "dog" (English), "狗" (Chinese) → Global verbal label
  • Number "7", symbol "+", formula "F = ma" → Global math label
  • Code "if (x > 0)" → Global programming label
  • Icon 🏠 → Global visual label
  • Note C₄ → Global music label

Properties:
  → CAN communicate GLOBALLY (requires matching format training)
  → Logic chain EXTREMELY POWERFUL: all formal knowledge = chain of global anchors
  → ACCUMULATE: books, papers, code, sheet music = stored global anchors
  → VERIFIABLE: anyone can check (peer review, unit test, proof)

  ⭐ Global anchors = accumulated "HUMAN KNOWLEDGE":
  → Each global anchor = 1 chunk that has been formalized + shared
  → All science, math, literature, code, music = LIBRARY of global anchors
  → AI accesses THIS ENTIRE library → AI excels in Global anchor territory
```

### §2.4 — Synthesis of 3 Tiers + Flow

```
FLOW BETWEEN 3 TIERS:

  Individual ──articulate──→ Group ──formalize──→ Global
  (only me)   (Somatic-     (group  (research,    (human-
               Articulation  knows)  education,    kind
               Loop)                 publication)  knows)

Examples:
  Einstein FEELS something about light [Individual]
  → Tells Besso, Grossmann "I think..." [Group — physics friends]
  → Writes paper, formalizes "E = mc²" [Global — humanity knows]

  Child FEELS pain [Individual]
  → Points to mother "hurts here" [Group — family]
  → Learns label "stomachache" [Global — vocabulary]

  Game dev FEELS "something off in the code" [Individual]
  → Tells team "this smells" [Group — team jargon "code smell"]
  → Reads book "ah, this is the God Object anti-pattern" [Global — design patterns]

REVERSE DIRECTION ALSO EXISTS:

  Global ──learn──→ Group ──internalize──→ Individual

  Learns label "anxiety" from a book [Global]
  → Team says "you're so anxious" [Group]
  → Body recognizes "ah, THIS is what anxiety feels like" [Individual — label → felt sense match]

  → This is why education EXISTS: install Global anchors into the Individual
  → And this is why AI ASSISTS: AI helps find the Global anchor for an Individual feeling

⭐ IMPORTANT:
  → Individual anchor = RICHEST (body has INFINITE patterns)
  → Group anchor = MOST of daily communication
  → Global anchor = MOST PRECISE but NARROWEST (only what has been formalized)
  → Most body-knowledge NEVER reaches the Global level
  → = "What I feel but have no words for" = Individual anchor not yet formalized
```

## §3 — Communication Formats: 5 Main Label Systems

> **Full analysis: [Language-Structure-Analysis/](Chunk-Analysis/Chunk-External-Development/Language-Structure-Analysis/) 5 files**

When an Individual/Group anchor is formalized into Global, it is encoded in one of the following **Communication Formats**:

| # | Format | Encodes | Optimal for | Breadth | Precision | Example |
|---|---|---|---|---|---|---|
| 1 | **Natural Language** (verbal/writing) | Meaning | Communication on any topic | ⭐ Widest | Low-Medium | "the dog runs fast" |
| 2 | **Mathematical Notation** | Quantity + Relation | Calculation, proof | Narrow | ⭐ Highest | F = ma, ∫₀^∞ e^(-x²) dx |
| 3 | **Musical Notation** | Temporal-Affective | Emotion × time | Narrow | High (pitch) | ♩♩♩𝅗𝅥 (Beethoven 5th) |
| 4 | **Visual/Diagram** | Spatial Relationships | Spatial overview | Medium | Medium-High | Flowchart, map, UI |
| 5 | **Programming Language** | Executable Behavior | Machine execution | Medium | ⭐ Highest | if (x > 0) return x; |

**Each format = 1 way to label chunks for a SPECIFIC domain:**
- Communicate → Natural Language
- Calculate → Math
- Encode temporal emotion → Music
- Show spatial → Visual
- Machine execution → Code

**No format covers EVERYTHING.** Experts use a MIX of formats:
- Doctor: Language (speaking to patients) + Math (dosage) + Visual (X-ray)
- Game developer: Code + Math + Visual + Language + Music (ALL 5!)
- Physicist: Math + Language + Visual (diagram) + Code (simulation)

## §4 — Planning: Chain labeled chunks → Imagine-Final

> **Full analysis: [Imagine-Final.md](Imagination/Imagine-Final.md) + [Chunk.md v2.0](Chunk-Analysis/Chunk.md) + [PFC-Analysis.md](Imagination/PFC-Analysis.md)**

### §4.1 — Planning mechanism

```
PLAN = sequence of actions TO TRANSFORM "current state" INTO "Imagine-Final" (the goal)

  Mechanism:
  ① Imagine-Final forms (body pre-feels the target — feeling territory)
  ② PFC compares: current state vs Imagine-Final → detects GAP
  ③ PFC DRAFTS plan: chains LABELED chunks → bridges the gap
  ④ Body TESTS: "is this plan ok?" → feeling check (body vote)
  ⑤ EXECUTE: carry out plan step by step
  ⑥ Body MONITORS: each step → body checks "on track?" (feeling feedback)
  ⑦ ADJUST: if wrong → PFC re-drafts → new plan
  ⑧ COMPLETE: current state ≈ Imagine-Final → body signals "done" (satisfaction)

  → ⭐ Plan = Logic chain (③-⑤) BUT foundation = Feeling (①②④⑥⑧)
  → Imagine-Final comes from FEELING (body pre-feels the future state)
  → Plan checking also uses FEELING (body approves/rejects)
  → Logic is IN THE MIDDLE: chains steps, but bookended by feeling
```

### §4.2 — Chunks = building blocks for the plan

```
CHUNKS (Chunk-Analysis/Chunk.md v2.0):
  = Strength-weighted associative network compiled through experience
  = Brain SEARCHES database, does NOT CALCULATE
  = PFC holds ~3-5 chunks in WM simultaneously

  5 ways to compile a chunk (Chunk.md v2.0 §2):
  ① Repetition (repeat — slow, durable)
  ② Emotional peak (once, strong — amygdala + NE)
  ③ Multi-modal simultaneous (multiple modalities at once)
  ④ Sleep consolidation (sleep — hippocampal replay)
  ⑤ External install (F3: social referencing, instruction, Direction B)

PLAN CHAIN CHUNKS:
  Step 1: retrieve chunk A (rule/fact/pattern)
  Step 2: retrieve chunk B
  Step 3: chain A → B → derive C
  Step 4: retrieve chunk D
  Step 5: chain C → D → derive E (conclusion / action)

  → Each step = PFC search → find chunk → hold in WM → chain
  → Expert: many meta-chunks (compiled chains) → plans FASTER
  → Beginner: individual chunks → plans SLOWER (more steps)
  → "Expert = someone who has compiled many chains into 1 chunk"
```

### §4.3 — Plans Are Domain-Specific (no universal template)

```
⭐ NO "universal solution for every problem"

  Each domain = its own chunks + its own rules + its own patterns
  → Plan for domain A ≠ plan for domain B
  → Must HAVE the domain's chunks to be able to chain for that domain

  Examples:
  Solve equation:  identify type → select formula → calculate → check
  Cook pho:        prep → simmer bones → prepare spices → cook broth → pho
  Debug game:      reproduce → isolate → trace → fix → test
  Treat illness:   ask → examine → test → diagnose → prescribe
  Write article:   brainstorm → outline → draft → revise → publish

  → EACH = different chain, different chunks, different format
  → Expert at cooking pho ≠ expert at debugging games (chunks are DIFFERENT domains)
  → "General intelligence" = MANY domain chunks, not 1 super-algorithm

  ⭐ This is why "experts" exist:
  Expert = someone who has compiled MANY chunks + meta-chunks IN 1 DOMAIN
  → Looks → recognizes pattern (L1) → chains instantly (skips L3)
  → Beginner is NOT MISSING PFC — is missing CHUNKS (small database)
  → Education = install chunks into the database
  → Experience = compile chunks into meta-chunks
```

## §5 — Why Plans Must Be Complex

```
"Language + plans are complex → because domain reality is complex."

Domain reality:
  • Physics: falling objects, waves, electricity, heat, quantum → each = distinct rules
  • Biology: cells, organs, systems → each = distinct rules
  • Social: relationships, economics, politics → each = distinct rules
  • Math: numbers, geometry, calculus, statistics → each = distinct rules
  • Code: architecture, algorithm, UI, networking → each = distinct rules
  • Game: design, art, code, music, UX → MIX of many domains

→ Plans MUST be complex because the domain is complex
→ Simplifying the plan = dropping domain detail → plan WRONG
→ This is why AI IS USEFUL: AI holds more rules than humans
→ Humans: ~3-5 chunks in WM simultaneously
→ AI: millions of chunks in "WM" simultaneously
→ AI chains MORE rules, MORE ACCURATELY, FASTER

BUT — AI LACKS:
  → AI does not FEEL domain reality (no body-input)
  → AI cannot detect novel patterns outside training data
  → AI cannot coherence-check using the body (only using rules)
  → = AI = an EXTREMELY DETAILED map — but still just a map (Logic territory)
  → = Humans still need a COMPASS (Feeling) to know whether the map IS CORRECT
```

## §6 — AI-Assisted Logic: why AI excels here

### §6.1 — AI excels in Logic territory

```
AI OUTPERFORMS HUMANS AT:

  ① FINDING LABELS:
    Human: "I sense something... vague... about the light..."
    AI: "Are you describing the phenomenon of 'diffraction'? Or 'refraction'?"
    → AI = Somatic-Articulation CATALYST (per Somatic-Articulation-Loop.md §5)
    → AI helps articulate Individual anchor → Global label

  ② BUILDING PLANS:
    Human: "I want to make a game but don't know where to start"
    AI: chains domain chunks → ordered plan → "step 1: concept, step 2: prototype,..."
    → AI chains MANY steps + MANY domains + MANY constraints simultaneously

  ③ CHECK LOGIC:
    Human: "is my logic correct?"
    AI: verifies chain, finds gaps, tests edge cases
    → AI = extremely powerful logic verifier

  ④ CROSS-REFERENCE:
    Human: "how does this relate to that?"
    AI: searches ENTIRE knowledge → finds connections
    → AI cross-domain referencing better than any individual

  ⑤ FORMAT TRANSLATE:
    Human: "explain it visually / in code / in math"
    AI: translates the same concept across multiple formats
    → AI = multi-format encoder
```

### §6.2 — AI CANNOT operate in Feeling territory

```

  ❌ "Something is off" signal (no body dissonance)

→ AI = LOGIC AMPLIFIER, not FEELING REPLACER
→ Humans must STILL feel first → then use AI to logic chain + plan
→ = Compass (human) + Map (AI) = optimal combination

⭐ FRAMEWORK IMPLICATION:

  → Core value of the framework = UNDERSTANDING FEELING
  → So people CAN KNOW:
```

### §7.3 — Somatic-Articulation Loop: Feeling → Label transition

> **Full analysis: [Somatic-Articulation-Loop.md](Imagination/Somatic-Articulation-Loop.md)**

```
3 states of knowledge:

  IMPLICIT:      Body knows, PFC does not. No label.
                 → Volatile, hard to chain, hard to communicate
                 → "Feels something but cannot say it"

  TRANSITIONAL:  Body knows, PFC is articulating. Label IS FORMING.
                 → Loop: body signal ↔ try label ↔ body feedback "close/not yet/yes!"
                 → Catalyst needed: a listener, AI, writing, drawing

  EXPLICIT:      Body knows, PFC HAS a label. Can chain + communicate.
                 → Stable, chainable in logic, shareable

  → TRANSITION = Somatic-Articulation Loop
  → AI = EXTREMELY POWERFUL catalyst for transition (available 24/7, patient, multi-format)
  → BUT: body must FEEL first — AI only HELPS articulate, does not REPLACE feeling
```

### §7.4 — 2-Direction Unified Flow: Feel-first vs Label-first

```
⭐ SAME DESTINATION (chunk with label, usable in plans), 2 ENTRY DIRECTIONS:

  DIRECTION A — FEEL-FIRST (bottom-up):
    Pattern fires enough → PFC receives vague feel → PFC attends → chunk gradually clarifies
    → feeling clarifies → find label → label matches → chunk explicit → use in plan

    Examples:
      Child: body signals bladder → feels "something" → gradually recognizes → "pee" (label)
      Adult: "something off" about code → feels mismatch → articulates → "God Object" (label)
      Einstein: feels something about light → years later → "spacetime curvature" (label)

    = Somatic-Articulation-Loop.md: implicit → transitional → explicit
    = PRIMARY direction for novel pattern / body-knowledge / creative insight


  DIRECTION B — LABEL-FIRST (top-down):
    Receive label (via education/language) → practice/experience → body compiles chunk
    → feeling matches label → chunk explicit → use in plan

    Examples:
      Student: learns label "gravity" → drops object → sees it fall → chunk compiles → genuinely understands
      Doctor: learns label "arrhythmia" → listens to patient's heart → recognizes it → chunk compiles
      Reading: encounters word "anxiety" → recognizes "ah, THIS is what I've been feeling!" → label match

    = Education mechanism: install label FIRST, chunk compiles LATER
    = PRIMARY direction for formal knowledge / domain training


  WHY BOTH DIRECTIONS EXIST:
    Direction A: for WHAT IS NEW (no one has a label yet → must feel-first → create label)
    Direction B: for WHAT IS ALREADY KNOWN (someone already created the label → install via education)

    Education = batch install Direction B labels → accelerate chunk formation
    Research/Creation = Direction A → creates NEW labels for what no one yet knows
    AI catalyst: accelerates BOTH directions
      → Direction A: AI helps find label for feeling ("are you describing X?")
      → Direction B: AI helps explain label for experience ("X means...")
```
