## §7 — FEEDBACK CALIBRATION LOOP

PR's primary architectural function: **feedback calibration** cho SPM library.

### §7.1 — Loop architecture

```
  [Self-Pattern state t₀]
          ↓
  [SPM fires forward prediction]
          ↓
  [Interaction proceeds]
          ↓
  [Outcome observed]
          ↓
  [PR inferred retrospectively]
          ↓
  [Library update]:
    → Confirmed chunks strengthened (Hebbian)
    → Misfired chunks weakened
    → New target-specific chunks created
    → Retrieval heuristics refined per-target
    → Schema links adjusted
          ↓
  [Updated SPM library → next prediction BETTER]

  v3.0 NOTE — PFC=Lawyer complicates loop:
    "Outcome observed" filtered through PFC narrative
    → PFC may DISTORT outcome to fit prior prediction (confirmation bias)
    → Deep friendship = enough episodes to calibrate PAST NARRATIVES
    → F1 Compiled path LESS susceptible (body doesn't need PFC narrative to fire)
    → F2 Fresh path MORE susceptible (PFC constructs both prediction AND interpretation)
```

### §7.2 — 4 library update types

```
  ① CHUNK STRENGTHENING:
    Prediction matched → chunks reinforced → future retrieval priority ↑
    "Friend prefers practical gifts" → confirmed → strengthened

  ② CHUNK WEAKENING:
    Prediction failed → chunks reduced weight
    "Colleague frown = upset" → actually tired → weakened

  ③ NEW CHUNK CREATION:
    Target unexpected behavior → new chunk formed
    "Didn't know she loved jazz" → new "friend's music: jazz" chunk

  ④ RETRIEVAL HEURISTIC REFINEMENT:
    Meta-learning: WHICH chunks for WHICH target
    "For this colleague, prioritize verbal-cognitive — affective cues subtle"
```

### §7.3 — Time scales

```
  Immediate:    mid-conversation micro-adjustments (seconds)
  Short-term:   post-interaction reflection (hours-days)
  Medium-term:  relationship trajectory observation (weeks-months)
  Long-term:    deep relationship compilation (years)
  Cross-target: lessons transfer to similar targets
```

### §7.4 — Bird & Cook 2013: loop break
