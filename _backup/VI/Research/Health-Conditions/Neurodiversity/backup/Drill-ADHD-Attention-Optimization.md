### I1 — Threshold = Hardware = Không Thay Đổi Được

```
🟡 FRAMEWORK SYNTHESIS:

    → KHÔNG thay đổi bằng training, therapy, hoặc willpower

  Đúng:
    ✅ Hiểu threshold → design environment ABOVE threshold
    ✅ Chọn domain → where natural gaps EXCEED threshold
    ✅ Medication → temporarily shift threshold (chemical compensation)
    ✅ Strategies → work AROUND threshold (external scaffolding)
```

### I3 — Chunk Accumulation → Internal Novelty Source

```
🟡 FRAMEWORK SYNTHESIS (applying Novelty.md §2):

  Novelty.md §2: VTA detects DELTA (change from prediction).
  External novelty: new input from environment → delta → fire.
  Internal novelty: RECOMBINATION of existing chunks → delta → fire.

  ADHD IMPLICATION:
    → External novelty: REQUIRES environment to provide (fragile, exhaustible)
    → Internal novelty: GENERATED from chunk library (sustainable, infinite)

  Progression model:
    Phase A — External-dependent (shallow chunks):
      → New domain → everything novel → VTA fires → engagement HIGH
      → As chunks compile → novelty DECREASES → engagement DROPS
      → = "Honeymoon phase" → then "boring" → then switch domain
      → = MOST ADHD people stuck here → "can't stick with anything"

    Phase B — Transition (building depth):
      → FORCE through boring phase (external scaffolding needed)
      → Chunks accumulate → gaps emerge → gaps have direction
      → Some internal novelty starts → engagement FLUCTUATES
      → = "Sometimes interesting, sometimes boring" → unstable

    Phase C — Internal-driven (deep chunks):
      → Deep chunk library → MANY gaps → RICH recombination
      → Internal novelty > external novelty → self-sustaining
      → = "Expert" territory → ADHD = genuine advantage here
      → Cross-domain connections (ADHD topic-jumping) = ASSET
      → = "I never get bored in my field — always something new"

  CRITICAL TRANSITION: A → B → C
    → A → B = hardest (external novelty fading, internal not yet started)
    → = "The Valley" (Self-Created-Threat.md §5.5)
    → Duration: weeks to months depending on domain complexity
    → MOST ADHD people QUIT HERE
    → Scaffolding needed SPECIFICALLY for this transition
    → Once past B → C: self-sustaining, scaffolding can withdraw

  APPLICATION:
    → Don't tell ADHD person "just stick with it" (= ignore threshold)
    → DO: provide scaffolding SPECIFICALLY during A→B transition
    → DO: make transition VISIBLE ("you're in phase B now, almost there")
    → DO: use mini-arcs with tangible output during boring phase
    → Once C reached: "congratulations, your ADHD is now an advantage here"

  🔴 CẦN VERIFY:
    -> A→B→C model = framework construct
    -> "Internal novelty" concept — neuroscience basis?
    -> Duration of transition — any longitudinal data?
```

### I5 — Condition 2: Arc Decomposition with Tangible Output

```
🟡 FRAMEWORK SYNTHESIS:

  ADHD big-arc problem:
    → Big goal (write thesis, build app, learn skill)
    → Progress = invisible (no tangible delta until END)
    → Invisible progress → BELOW threshold → "boring" → abandon
    → = "ADHD people start 100 things, finish 3"

  SOLUTION: Arc decomposition (Gap-Body-Need §6 mini-arcs):
    → Break big-arc → mini-arcs (3-7 chunks each)
    → Each mini-arc has TANGIBLE output (visible, concrete, shareable)
    → Tangible output → EXCEEDS threshold → reward fires → sustain drive
    → Reward → compile [progress = real] → motivation sustains

  DESIGN PRINCIPLES cho mini-arcs (ADHD-specific):

    ① SIZE: smaller than neurotypical mini-arcs
      → ADHD dopamine signal SHORT → reward window NARROW
      → Neurotypical mini-arc: 1-2 weeks → tangible output
      → ADHD mild: 2-3 days → tangible output
      → ADHD moderate: 1 day → tangible output
      → ADHD severe: hours → tangible output
      → = Adjust granularity to match dopamine signal duration

    ② TANGIBILITY: must EXCEED threshold
      → "Made progress" ≠ tangible (abstract, invisible)
      → "Wrote 500 words" = tangible (countable, visible)
      → "Built working feature" = tangible (demonstrable)
      → "Organized 10 files" = tangible (before/after visible)
      → Rule: if you can't SHOW someone → not tangible enough

    ③ TRANSITION: clear end of mini-arc + clear start of next
      → Ambiguous endings → "is this done?" → PFC cost → procrastinate
      → CRISP boundaries: "this mini-arc = done when X exists"
      → Neurotypical: can self-define transitions
      → ADHD: need EXTERNAL definition (checklist, partner confirms)

    ④ VARIETY: alternate mini-arc TYPES within big-arc
      → Same type mini-arc × repetition → habituate → below threshold
      → Mix: research → write → design → test → research...
      → = Keep novelty WITHIN the big-arc

  EXTERNAL SCAFFOLDING OPTIONS:
    → Project management tools (visual progress: Kanban, progress bars)
    → Accountability partner ("show me today's output")
    → Gamification (streak counters, achievement badges)
    → AI pair-programming / writing partner (immediate feedback loop)
    → Physical artifact per mini-arc (print, hang on wall, check off)

  🔴 CẦN VERIFY:
    -> Mini-arc optimal size per ADHD severity — any research?
    -> Gamification effectiveness specifically for ADHD — meta-analysis?
    -> "Tangible output" concept — psychology of concrete vs abstract goals?
```

## ROUND 3 — MICRO-REWARD ENGINEERING

### I8 — Feast-or-Famine → Steady Mini-Arc Rewards

```
🟡 FRAMEWORK SYNTHESIS:

  ADHD REWARD PATTERN (current):
    → BIG reward when hyperfocus + success = FEAST (dopamine flood)
    → NOTHING when below-threshold task = FAMINE (dopamine absent)
    → = Binary: everything or nothing
    → = Tại sao ADHD person oscillates between
      "most productive person" and "can't do basic tasks"

  OPTIMIZATION: Convert feast-or-famine → steady micro-rewards:

  Step 1 — DETECT current reward pattern:
    → Track 1 week: when do you feel engaged? when dead?
    → Map: task × engagement level
    → Identify: which tasks = feast, which = famine
    → = Diagnostic baseline

  Step 2 — DESIGN mini-arcs for famine tasks:
    → Famine task = below threshold → needs ENGINEERING to exceed
    → Options:
      a) Attach to above-threshold meta-task
         "Clean desk" = boring → "Clean desk in 5 min while music plays" = gamified
      b) Pair with immediate reward
         "After this email, 5 min break for X"
      c) Create artificial novelty
         Same task, different approach each time
      d) Social accountability
         "I'll show you my completed X at 3pm"

  Step 3 — IMPLEMENT reward schedule:
    → Pomodoro-like but ADHD-adapted:
      → Standard: 25 min work / 5 min break
      → ADHD mild: 15-20 min / 5-10 min break
      → ADHD moderate: 10-15 min / 5 min break
      → ADHD severe: 5-10 min / 5 min break
      → KEY: break must be GENUINE reward (not "rest" — ACTIVE reward)
    → Each work block = mini-arc with TANGIBLE completion
    → Each break = REWARD that body RECOGNIZES
    → = Artificial Cyclic satiation pattern overlaid on tasks

  Step 4 — MONITOR and adjust:
    → Body feedback: are mini-arcs too long? (signs: dread, avoidance)
    → Body feedback: are mini-arcs too short? (signs: momentum broken)
    → DYNAMIC adjustment — not fixed formula
    → = Body = final arbiter (Ask-AI.md §6.1 dual check)

  IMPORTANT: This is SCAFFOLDING, not fix:
    → In above-threshold domain → scaffolding UNNECESSARY (natural flow)
    → Scaffolding = for NECESSARY below-threshold tasks only
    → Goal: minimize time in below-threshold, not "fix" threshold

  🔴 CẦN VERIFY:
    -> Pomodoro × ADHD adaptation — formal studies?
    -> Micro-reward timing per ADHD severity — any dose-response?
    -> "Active reward break" vs "passive rest" — effectiveness?
```

## ROUND 4 — EXTERNAL PFC STRATEGIES

### I10 — "External PFC" Concept: Externalize What PFC Can't Sustain

```
🟡 FRAMEWORK SYNTHESIS:

  CORE INSIGHT:
    ADHD = PFC under-fueled → can't sustain certain functions internally
    SOLUTION: externalize those functions to environment/tools/people
    = "External PFC" — không fix internal PFC, BUILD external equivalent

  5 PFC FUNCTIONS TO EXTERNALIZE:

  ① WORKING MEMORY (Hold Duration):
    Internal: PFC holds ~4±1 items, ADHD hold duration SHORT
    External options:
      → Physical notes, sticky notes, whiteboards
      → Digital: Notion, Obsidian, task managers
      → Voice memos (capture thought IMMEDIATELY before fade)
      → AI: "remember this for me"
    KEY: capture system must be FRICTIONLESS
      → High friction → won't use → items lost
      → Low friction → habitual → items preserved
      → Phone voice memo = lowest friction for most

  ② INHIBITION (Suppress):
    Internal: PFC suppress impulses, ADHD suppress WEAKER
    External options:
      → Website blockers (Cold Turkey, Freedom)
      → Phone away during work (physical barrier)
      → Partner: "tell me if I'm drifting"
      → Environment design: remove distractors physically
    KEY: external inhibition = PHYSICAL BARRIER, not willpower
      → "Don't check phone" (willpower) = PFC cost → fails
      → Phone in another room (physical) = no PFC cost → works
      → = Leverage environment, not PFC

  ③ TIME PERCEPTION:
    Internal: PFC maintains time model, ADHD = time blind
    External options:
      → Visual timers (Time Timer: RED shrinks visually)
      → Phone alarms (multiple, per-task)
      → Calendar blocking (assign time → make visible)
      → AI: "it's been 45 minutes since you started"
    KEY: make time VISIBLE (ADHD doesn't perceive time passing)
      → "You have 2 hours" = abstract → body ignores
      → Timer counting DOWN visually = concrete → body registers

  ④ PLANNING (Sequential):
    Internal: PFC sequences steps, ADHD = jump to interesting
    External options:
      → Written step-by-step (1. 2. 3. — visible order)
      → Partner/manager: pre-sequence tasks
      → Checklist (physical: cross off → tangible reward)
      → AI: "break this project into steps for me"
    KEY: WRITTEN, not mental
      → Mental plan = PFC hold → fade → lost
      → Written plan = persistent → can return after interruption

  ⑤ EMOTIONAL REGULATION:
    Internal: PFC modulates emotional response, ADHD = binary
    External options:
      → Pre-agreed response scripts ("when frustrated, walk 5 min")
      → Partner: "you seem escalating, let's pause"
      → Body practices: breath counting (physical anchor)
      → Post-event processing with therapist/partner
    KEY: pre-agreed protocols COMPILED, not real-time decisions
      → Real-time regulation = PFC under load → fails
      → Pre-compiled protocol = automatic → works

  META-PRINCIPLE:
    → Every externalized function = PFC budget FREED
    → Freed PFC → available for ACTUAL cognitive work
    → = "External PFC" doesn't just compensate — it OPTIMIZES
    → ADHD person with full external PFC scaffolding → 
       PFC fully available for thinking → potentially OUTPERFORMS neurotypical
       (who wastes PFC on functions that could be externalized)
    → = Why best ADHD strategies = "things everyone should do"

  🔴 CẦN VERIFY:

    -> Effectiveness of each strategy category — meta-analyses?
    -> "Outperform neurotypical with scaffolding" = strong claim, evidence?
```

## ROUND 7 — BIG-ARC MANAGEMENT

### I15 — Valley Prediction + Preparation

```
🟡 FRAMEWORK SYNTHESIS (applying Self-Created-Threat.md §5.5):

  Self-Created-Threat §5.5: Full Drive Cycle:
    ARC START: mostly PULL (novelty + curiosity → drive high)
    ARC MIDDLE ("The Valley"): PULL fades, PUSH needed
    ARC END: PULL returns (final near → dopamine rises)

  ADHD-SPECIFIC VALLEY:
    → Valley DEEPER for ADHD (novelty depleted → BELOW threshold → ZERO drive)
    → Valley WIDER for ADHD (takes LONGER to reach "final near" from middle)
    → Valley HARDER TO EXIT (PFC under-fueled → can't self-push effectively)

  PREDICTION + PREPARATION STRATEGY:

  ① PREDICT WHEN VALLEY ARRIVES:
    → Most projects: ~20-40% completion = valley entry
    → Novelty of start GONE, satisfaction of end FAR
    → KNOW IN ADVANCE: "I will hit the valley, it's ARCHITECTURE not failure"
    → = Cognitive preparation → reduces cortisol when valley arrives
    → "The valley is not me failing — it's the gap between novelty and completion"

  ② PRE-POSITION SCAFFOLDING:
    → Before valley: set up accountability (partner expects progress)
    → Before valley: break valley-section into EXTRA-small mini-arcs
    → Before valley: prepare REWARDS for valley mini-arcs
    → = Don't wait until valley to build scaffolding (PFC too weak then)

  ③ INCREASE PULL, MINIMIZE PUSH:
    → Drill-Trade-Off I17-I18: ADHD optimal = 90:10 pull:push
    → In valley:
      → Re-visualize Imagine-Final (reconnect with WHY)
      → Share progress with someone who cares (social reward)
      → Find NOVEL APPROACH to same task (refresh novelty)
      → AVOID: "just push through" (= PFC suppress → will fail)

  ④ PLANNED ARC SWITCHING (not impulsive):
    → If valley too deep → PLANNED switch to different arc temporarily
    → "I'll do Project B for 2 hours, then return to Project A"
    → = NOT abandoning A — STRATEGIC rotation
    → Return to A: novelty partially refreshed (absence → delta)
    → KEY DISTINCTION:
      → Impulsive switching: no plan to return → arc abandoned
      → Planned switching: return time SET → arc preserved
    → External scaffolding: timer/calendar for return

  ⑤ ARC GRAVEYARD PREVENTION:
    → "Arc graveyard" = pile of unfinished projects
    → Each unfinished arc = [can't finish] BP reinforcement
    → Prevention:
      → BEFORE starting: "Am I committing to valley? Who's my scaffolding?"
      → DURING valley: use strategies above
      → IF abandoning: CONSCIOUS decision ("this arc isn't worth finishing because X")
        → NOT drift → DECIDE → reduces BP reinforcement
        → "I chose to stop" ≠ "I couldn't finish" → different self-model chunk

    -> Project abandonment patterns specifically in ADHD?
    -> Planned vs impulsive task-switching effects?
```
