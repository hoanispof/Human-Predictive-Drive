# Drill — Trust × Entity-Valence × Gap-Fill: 2 Observations Của Cùng 1 Process

> **Trust: "tin thầy cô dạy đúng."**
> **Valence: "thích thầy cô — hoặc ghét."**
>
> **2 thứ KHÁC NHAU. Nhưng thường ĐI CÙNG NHAU.**
> **Thích ai → dễ tin. Tin ai → dễ thích.**
> **Power cao → cả trust LẪN valence positive → TẠI SAO?**
>
> **Đáp án: Trust và Valence là 2 OBSERVATIONS KHÁC NHAU**
> **của CÙNG 1 quá trình: entity × gap-fill interaction.**
> **Valence = "entity fill gap TỐT hay XẤU?" (outcome)**
> **Trust = "entity fill gap ỔN ĐỊNH và ĐÚNG không?" (prediction)**
>
> **Cùng quá trình → cùng data → dễ tương quan.**
> **NHƯNG đo 2 thứ khác → CÓ THỂ diverge.**
> **SỢ Chúa (valence negative) nhưng TIN Chúa tha tội (trust positive).**
> **THÍCH bạn (valence positive) nhưng KHÔNG TIN bạn dạy toán giỏi (trust thấp trong domain).**

## §4 — 2 CHIỀU ĐỘC LẬP — TẠI SAO NÓI "ĐỘC LẬP"

```
⭐⭐ TRUST VÀ VALENCE ĐỘC LẬP VÌ:

  ① ĐO THỨ KHÁC NHAU:
    Valence = "entity ảnh hưởng body-base CỦA TÔI thế nào?"
           = OUTCOME assessment (đã xảy ra, body ghi nhận)
    Trust  = "entity có ĐÁNG TIN CẬY cho tương lai không?"
           = PREDICTION meta-tag (dựa trên lịch sử, hướng tới tương lai)

  ② CÓ THỂ DIVERGE HOÀN TOÀN:
    SỢ Chúa (valence -) + TIN Chúa tha tội (trust +) → case ② Guilt Drill §3.1
    THÍCH bạn (valence +) + biết bạn KHÔNG authority (trust - in domain) → case ⑤
    GHÉT sĩ quan (valence -) + biết sĩ quan CÓ quyền tha (trust +) → case ④

  ③ UPDATE KHÁC TỐC ĐỘ:
    Valence current: thay đổi per-SECOND (context-dependent)
    Trust: thay đổi CHẬM (compiled, cần multiple experiences để update)
    → Có thể valence flip (mẹ la → ghét lúc này) mà trust KHÔNG đổi

  ④ MECHANISM KHÁC:
    Valence emerge từ: body-feedback (L0, L1, evaluative channels)
    Trust emerge từ: compiled PREDICTION about entity's reliability
    → Valence = reaction. Trust = expectation.


  CÓ THỂ HIGH-TRUST + NEGATIVE-VALENCE:
    Military commander: soldier knows commander has authority (trust)
    BUT commander makes harsh decisions → soldier stressed (valence negative)
    → Trust ≠ liking. Trust = belief about capability/authority.

  CÓ THỂ LOW-TRUST + POSITIVE-VALENCE:
    New colleague: fun, friendly, interesting (valence positive on social/novelty)
    BUT chưa có track record → trust LOW (chưa verify reliability)
    → "Thích nhưng chưa tin" = early relationship state.


🟡 Trust vs valence independence = framework synthesis
🟢 Trust ≠ liking: Colquitt et al. 2007 (meta-analysis separates trust from affect)
```

### §4.1 — ASYMMETRY: VALENCE OSCILLATE, TRUST = STABLE BASELINE

```
⭐⭐ TRUST VÀ VALENCE CÓ ASYMMETRY VỀ TỐC ĐỘ UPDATE:


  ENTITY-VALENCE — 2 TẦNG, 2 TỐC ĐỘ:

    Current valence (Entity-Valence-Dynamics v1.1 §1):
      → Thay đổi per-SECOND (context-dependent)
      → "Mẹ la → ghét mẹ LÚC NÀY" (current negative)
      → "Mẹ ôm → thương mẹ LÚC NÀY" (current positive)
      → = OSCILLATE tùy context + state

    Structural valence (Entity-Valence-Dynamics v1.1 §1):
      → Thay đổi per-MONTH/YEAR (aggregate tích lũy)
      → = SUM of all compiled valence tags
      → CHẬM vì cần compile/decompile chunks

    → Valence = có phần NHANH (current) + phần CHẬM (structural).
    → PFC verbalize = current valence ("ghét mẹ lúc này").
    → Body-base TRUE assessment = structural valence ("thương mẹ sâu").


  TRUST — CONSERVATIVE, STABLE BASELINE:

    Build: CHẬM — months/years tích lũy qua multiple gap-fill outcomes.
      → "Tin thầy cô" = build qua hàng trăm lần dạy đúng.
      → Trust = aggregate COMPILED prediction → PFC invisible khi stable.

    Maintain: ỔN ĐỊNH — compiled = fire tự động, không cần reinforce mỗi ngày.
      → "Tin mẹ tốt" = stable baseline dù current valence oscillate.
      → = Trust là NỀN TẢNG, valence oscillate TRÊN NỀN đó.

    Collapse: CÓ THỂ NHANH — 1 betrayal = violent flip (VP §3 update ③).
      → "Bạn thân phản bội → trust COLLAPSE ngay" (Freyd 1996).
      → Build chậm, destroy nhanh = ASYMMETRIC.
      → Negativity bias: negative information × trust = overweight (Baumeister 2001).


  ⭐ CONSEQUENCE CỦA ASYMMETRY:

    ① "Ghét mẹ lúc này" (current valence -) nhưng "vẫn tin mẹ tốt" (trust +):
       → Trust KHÔNG oscillate theo current valence.
       → Trust = stable PREDICTION, valence = momentary REACTION.
       → = Tại sao trẻ con la "ghét mẹ!" → 5 phút sau ôm mẹ.
         Current valence flip NHANH. Trust = unchanged.

    ② "Tin giờ đi gặp bạn sẽ vui" (trust + = prediction):
       → Prediction dựa trên HISTORY of positive valence.
       → Trust = COMPILED aggregate of many valence outcomes.
       → = Trust TÍCH LŨY từ valence data, nhưng CHẬM HƠN valence respond.

    ③ Trust collapse ≠ valence change:
       → Betrayal: trust COLLAPSE (nhanh), valence CÓ THỂ vẫn mixed.
       → "Vẫn thương nhưng không tin nữa" = structural valence positive (slow decay)
         + trust ZERO (immediate collapse).
       → = 2 chiều TÁCH RỜI rõ nhất khi betrayal xảy ra.


  ⭐ ANALOGY:

    Valence ≈ NHIỆT ĐỘ đo mỗi giây (fluctuate theo thời tiết).
    Trust ≈ KHÍ HẬU trung bình (ổn định, thay đổi rất chậm, nhưng CÓ THỂ biến đổi đột ngột nếu catastrophe).
    → Nhiệt độ hôm nay lạnh ≠ khí hậu đã đổi.
    → Mẹ la hôm nay ≠ trust vào mẹ đã đổi.
    → NHƯNG: nếu "lạnh" kéo dài NHIỀU NĂM → khí hậu THAY ĐỔI thật.
    → Nếu mẹ harm LIÊN TỤC NHIỀU NĂM → trust DẦN erode thật.


🟡 Trust/valence asymmetry = framework synthesis (combining EVD §1 + VP §3 ③)
🟢 Negativity bias × trust: Baumeister et al. 2001 ("Bad is stronger than good")
🟢 Trust asymmetry (easier to destroy): Slovic 1993 ("Trust is hard to gain, easy to lose")
```
