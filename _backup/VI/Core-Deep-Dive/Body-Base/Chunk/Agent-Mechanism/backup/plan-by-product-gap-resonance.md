### Definition
- **Resonance = mutual by-product gap fill giữa 2+ entities**
- Emerge GIỮA entities — không ai sở hữu
- Chỉ có thể quan sát, ước lượng — KHÔNG đo chính xác tuyệt đối
- Cả 2 entity PHẢI có gap riêng VÀ by-product feed lẫn nhau
- 1 chiều = KHÔNG phải resonance (parasocial, tool-use, exploitation)

### 2 tình huống resonance
```
① TRỰC TIẾP (observable, confidence cao):
  → By-product match xảy ra TỰ NHIÊN
  → Hardware (Stream 1: mẹ-bé, beauty) HOẶC Compiled SPM (Stream 2: bạn thân)
  → Cost ≈ 0. Tự vận hành. Quan sát rõ.
  → "Gặp nhau là vui. Không nghĩ gì cả."
  → Hardware habituates. Compiled anti-habituates.

② PFC-FRESH MEDIATED (khó quan sát, confidence thấp):
  → PFC bỏ chi phí để TẠO ĐIỀU KIỆN cho match
  → PFC-Fresh BẢN THÂN NÓ ≠ resonance. PFC = cost tạo điều kiện.
  → Resonance = cái EMERGE khi match thành công
  → 3 sub-cases:
    a) Genuine exploration (Loại A) → CÓ THỂ compile → thành ①
    b) Forced effort (Loại B: suppress) → compiled suppress → KHÔNG thành ①
    c) Strategic (Tool-mode giả Agent-mode) → dissolves khi goal ends
  → Khó phân biệt a/b/c từ bên ngoài
  → ② → ① = compile thành công (Hebbian). Kẹt ở ② = burnout risk.
```

### Tool ↔ Agent = Processing Mode Spectrum
```
INSIGHT MỚI (session này):
  → Tool-mode: predict action → outcome. SPM OFF. Transactional.
  → Agent-mode: predict entity state → response. SPM ON. Can deepen.
  → CÙNG entity → KHÁC mode tùy context/thời điểm/mục tiêu
  → 2D model: Tool difficulty × Agent engagement
  → Tool ↔ Agent INTERACT (feed nhau): Agent inform Tool, Tool inform Agent
  → VD: đoán ý sếp = Tool(predict outcome) + Agent(model mood) feed nhau

  → Phổ LIÊN TỤC, không binary:
    Ném đá:        Tool=low, Agent=0
    Dùng AI:       Tool=mid, Agent=0~low
    Đồng nghiệp:  Tool+Agent mix, shift tùy lúc
    Bạn thân:      Tool≈0, Agent=high
    Đoán ý sếp:    Tool=high, Agent=high
```

### 5 trục ước lượng xác suất resonance
```
TRỤC 1: GAP OVERLAP (Hardware ↔ Collective-Arc)
  → Gap distributions 2 entity overlap bao nhiêu?
  → Overlap CAO → by-product match probability CAO
  → Gap-Distribution-Profile §2 = reference
  → VD: 2 bạn cùng thích game = cao. IT vs công nhân = thấp dần.

TRỤC 2: PROCESSING MODE (Tool ↔ Agent)
  → SPM engagement level
  → Agent-mode = phần chính giữ resonance BỀN VỮNG
  → Tool-mode = transactional, dissolves khi goal ends

TRỤC 3: GAP QUALITY (Approach ↔ Avoidance)
  → Gap mỗi entity feed bằng approach (novelty) hay avoidance (threat)?
  → Approach × Approach = highest quality, self-reinforcing
  → Avoidance × Avoidance = lowest quality, break khi threat removed/exceeded
  → VD: con thích học (approach) vs con học tránh đòn (avoidance)

TRỤC 4: MUTUALITY (1 chiều ↔ 2 chiều)
  → Cả 2 entity gap fill?
  → 2 chiều: genuine resonance
  → 1 chiều: NOT resonance (parasocial, exploitation)
  → Asymmetric nhưng cả 2 có fill: VẪN là resonance (mẹ-bé)

TRỤC 5: CONSTRAINT (Tự do ↔ Bắt buộc)
  → Proximity tự chọn hay bị buộc?
  → Tự chọn: genuine quality
  → Bị buộc: CÓ THỂ toxic (fill gap cũ, tạo gap mới)
  → Sources: survival dependency, economic, legal, social norm
  → Break khi: entity có giải pháp khác + gap fill từ ở lại < từ rời đi
```
