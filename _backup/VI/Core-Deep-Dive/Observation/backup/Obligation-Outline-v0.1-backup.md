# Obligation — Observation Parameter (OUTLINE)

> Bố mẹ nuôi tôi ăn học. Tôi "phải" học giỏi.
> Sếp trả lương. Tôi "phải" làm việc.
> Anh B cho mượn tiền. Tôi "phải" trả — nhưng trả GÌ? Bao nhiêu? Khi nào?
> Trưởng làng được tin tưởng. Ông ấy "phải" lo cho dân.
> Nhà khoa học được công nhận. Ông ấy "phải" cống hiến.
> Vợ ở nhà nội trợ. Cô ấy "phải" giữ lửa gia đình.
>
> "Phải" — cùng 1 từ. KHÁC mechanism hoàn toàn.
>
> Có "phải" rõ ràng (trả tiền, làm việc) — cost biết trước, endpoint biết trước.
> Có "phải" mờ ảo (ân tình, identity) — cost KHÔNG biết, endpoint KHÔNG CÓ.
>
> Obligation = tên gọi cho pattern này.
> KHÔNG phải Status (Status = có gì). KHÔNG phải Threat (Threat = sợ mất gì).
> Obligation = COST LIÊN TỤC để GIỮ những gì mình CÓ từ agents.

## PLAN SECTIONS (drill chi tiết session sau)

### §0 — VỊ TRÍ TRONG FRAMEWORK

```
OUTLINE:

  Obligation KHÁC Status:
    Status = "tôi CÓ access gì TỪ agent NÀY" (Resource Access Map — snapshot)
    Obligation = "tôi PHẢI LÀM GÌ để GIỮ access đó" (ongoing cost — process)
    → Status = WHAT you have. Obligation = WHAT IT COSTS to keep it.
    → 2 parameters ORTHOGONAL: có thể Status CAO + Obligation THẤP,
      hoặc Status CAO + Obligation CAO

  Obligation KHÁC Threat:
    Threat = "hậu quả xấu nếu không hành động" (harm prediction)
    Obligation = "cost liên tục để MAINTAIN current state" (upkeep cost)
    → Threat CÓ THỂ là consequence CỦA obligation failure
    → Nhưng obligation TỒN TẠI TRƯỚC threat fire
    → Obligation = compiled expectations. Threat = khi expectations BỊ VIOLATED.

  Obligation liên hệ Autonomy:
    Autonomy = self-action preferred (efference copy reward)
    Obligation = those things you CANNOT do alone → NEED agent → cost emerges
    → Autonomy CAO → Obligation THẤP (tự làm = no debt to anyone)
    → Autonomy THẤP → Obligation CAO (phụ thuộc agents = must maintain)

  Obligation inverse Gratitude:
    Gratitude ∝ (1 / Obligation)
    Cùng 1 gift: obligation ≈ 0 → pure gratitude.
    Cùng 1 gift: obligation CAO → gratitude BỊ ĐÈ bởi debt anxiety.
    → Gratitude.md §5: Gift-Obligation Spectrum
```

### §1 — OBLIGATION LÀ GÌ

```
OUTLINE:

  Definition draft:
    Obligation = observable pattern khi body detect COMPILED EXPECTATION
    về "phải trả / phải làm / phải duy trì" để MAINTAIN access
    tới agent-as-external-tool.

  Core mechanism:
    Agent provide value (body-base feed qua virtual chunks, connection, status,...)
    → Body compile: "agent NÀY feed tôi" → ❸ positive
    → Body CŨNG compile: "để GIỮ feed → tôi phải X"
    → X = obligation content (explicit hoặc implicit)
    → KHÔNG fulfill X → agent CÓ THỂ withdraw feed → threat

  KHÔNG PHẢI:
    ✗ Moral judgment ("phải = đạo đức")
    ✗ Social norm only ("phải = xã hội quy định")
    ✗ Status (Status = CÓ gì, Obligation = TỐN gì để GIỮ)
    ✗ Threat (Threat = SỢ mất, Obligation = BIẾT phải duy trì)

  LÀ:
    ○ Compiled chunks về ongoing cost to maintain agent access
    ○ Emergent từ agent-as-external-tool architecture
    ○ Body-level: cortisol monitoring khi obligation pending
    ○ Spectrum: explicit→implicit, bounded→unbounded, self-imposed→external
```

### §2 — FORMULA + 5 FACTORS

```
OUTLINE:

  Obligation_intensity = f(Agent_value × Replaceability⁻¹
                           × Cost_clarity⁻¹ × Autonomy_alternative⁻¹
                           × Endpoint_clarity⁻¹)

  ① Agent_value: agent FEED body-base tôi bao nhiêu?
     → Cao = obligation CAO (nhiều thứ để mất)
     → Thấp = obligation THẤP (mất cũng không sao)

  ② Replaceability⁻¹: agent này DỄ thay thế không?
     → Khó = obligation CAO (mất = mất vĩnh viễn)
     → Dễ = obligation THẤP (tìm source khác)
     → Protect.md §2: f(replaceability × attachment)

  ③ Cost_clarity⁻¹: cost trả ơn RÕ RÀNG không?
     → Mờ = obligation NẶNG (không biết "trả bao nhiêu cho đủ")
     → Rõ = obligation QUẢN LÝ ĐƯỢC (trả xong = xong)
     → KEY DIFFERENTIATOR: ân tình MỜ > nợ tiền RÕ

  ④ Autonomy_alternative⁻¹: có thể TỰ LÀM thay không?
     → Không = obligation CAO (phải phụ thuộc)
     → Có = obligation THẤP (tự làm = escape option)

  ⑤ Endpoint_clarity⁻¹: biết khi nào XONG không?
     → Không endpoint = obligation SUSTAINED (vĩnh viễn?)
     → Endpoint rõ = obligation BOUNDED (trả xong = giải phóng)
```

### §3 — OBLIGATION SPECTRUM: 6 TYPES

```
OUTLINE — drill chi tiết từng type:

  TYPE 1 — FINANCIAL (explicit, bounded):
    VD: Vay tiền, trả rõ ràng
    Cost_clarity: CAO. Endpoint: RÕ. Cortisol: manageable.
    Chunk dynamics: Chunk-Gap → fill khi trả xong

  TYPE 2 — EXCHANGE (symmetric, ongoing):
    VD: Lương ↔ làm việc, mua bán
    Cost_clarity: CAO. Endpoint: hợp đồng kết thúc.
    Body-base: neutral ("công bằng, không nghĩ gì")
    → Status.md §4-5: mode "Trao đổi"

  TYPE 3 — SOCIAL / ÂN TÌNH (implicit, unbounded):
    VD: Anh B cho mượn tiền — ân tình, không rõ terms
    Cost_clarity: THẤP. Endpoint: KHÔNG RÕ.
    Body-base: cortisol SUSTAINED (Threat.md: Internal-Predict, no endpoint)
    ⭐ = TYPE NẶNG NHẤT — unpredictable cost + no endpoint
    → Tại sao "tự đi làm thêm" > "mượn anh B" khi có choice

  TYPE 4 — ROLE (trust-based, ongoing):
    VD: Trưởng làng, manager, leader
    Status NHẬN TỪ collective agent → obligation MAINTAIN trust
    Cost: performance liên tục, scope RỘNG
    Endpoint: role kết thúc (hoặc vĩnh viễn nếu role = identity)
    → Nếu fail → Status.md §6 disruption cycle

  TYPE 5 — IDENTITY (self-imposed, no external enforcement):
    VD: Nhà khoa học cảm thấy "phải" cống hiến
    Source: SELF — Anchor-Schema "tôi = nhà khoa học" drive behavior
    Cost: self-verify liên tục
    Endpoint: KHÔNG BAO GIỜ (identity = ongoing)
    ⭐ Không phải ai cũng feel (anchor strength varies)
    → Anchor yếu → obligation dissolve → "nhà khoa học" chỉ là label

  TYPE 6 — RELATIONAL (compound: exchange + social + role):
    VD: Vợ nội trợ — compound nhiều obligation type:
      Exchange: chồng kiếm tiền ↔ vợ nội trợ (nhưng không rõ "công bằng")
      Social: schema cài "phụ nữ phải giữ lửa" (installed obligation)
      Role: "vợ" + "mẹ" = role obligation ongoing
      + Autonomy GIẢM MẠNH (financial dependency)
      + Endpoint: KHÔNG RÕ (ly dị = threat lớn)
      → = COMPOUND nhiều types → obligation CỰC NẶNG

    VD đối lập: Vợ mới cưới xinh đẹp
      Exchange: chồng value → obligation ≈ 0 ("chỉ cần vui vẻ")
      Cost ≈ 0 (sẵn xinh = no effort needed)
      Autonomy: CAO (tự do làm gì muốn, miễn giới hạn)
      → NHƯNG: nếu "xinh" habituate → obligation SẼ TĂNG dần
      → = Baseline shift áp dụng cả cho obligation


  SPECTRUM TỔNG HỢP:

  EXPLICIT, BOUNDED                                  IMPLICIT, UNBOUNDED
  ←──────────────────────────────────────────────────────────────────→
  Financial  Exchange   Role        Social/Ân tình    Identity
  (rõ, trả   (symmetric, (trust,     (mờ, no          (self-imposed,
   = xong)    mutual)    ongoing)    endpoint)         vĩnh viễn)

  Cortisol:  LOW       ≈ 0        MODERATE    HIGH sustained    MODERATE-chronic
  Endpoint:  RÕ        Ongoing    Role-bound  KHÔNG RÕ          KHÔNG BAO GIỜ
```

### §4 — OBLIGATION × AGENT-AS-EXTERNAL-TOOL

```
OUTLINE:

  Core insight từ session:
    Agent-Mechanism.md §12: agents = external tools cho body-base
    → Autonomy: tự làm → KHÔNG CẦN agent → no obligation
    → NHƯNG: không phải mọi thứ tự làm được → PHẢI nhờ agent → obligation emerge

  Cost asymmetry khi "nhờ qua lại":
    → Tôi giỏi X, bạn giỏi Y
    → Tôi giúp bạn X (cost thấp cho tôi)
    → Bạn nhờ tôi Y (cost CAO cho tôi — không phải domain tôi giỏi)
    → = CROSS-DOMAIN trade = cost MISMATCH
    → = Tại sao tiền (money) phát minh: STANDARDIZE cost → eliminate asymmetry

  Người giúp VÔ TƯ (chain DÀI):
    → "Giúp 1 người = giúp cả cộng đồng" → chain reward từ cộng đồng tốt hàng ngày
    → Obligation imposed lên người nhận ≈ 0
    → = Gratitude PURE cho người nhận

  Người giúp chain NGẮN (đòi reciprocity):
    → "Giúp → nhờ trả ơn trực tiếp" → exchange gần
    → Obligation imposed = CAO + có thể unpredictable
    → = Gratitude MIXED cho người nhận
```

### §5 — OBLIGATION × GRATITUDE INTERACTION

```
OUTLINE:

  Reference: Gratitude.md §5 Gift-Obligation Spectrum

  Core: Obligation ở MẪU SỐ của Gratitude formula
    → Obligation ↑ → Gratitude ↓
    → Obligation ≈ 0 → Gratitude maximum
    → Obligation >>> feed → Gratitude → 0, thay bằng debt/anxiety

  "Vô điều kiện" = GRATITUDE MẠNH NHẤT — vì obligation ≈ 0
  Tôn giáo exploit: "Chúa cho vô điều kiện" → obligation ≈ 0 → gratitude max

  Khi obligation VƯỢT feed:
    → Gift mà debt > reward → NET NEGATIVE → avoid gift
    → = Tại sao "tự đi làm thêm" > "mượn người thực dụng"
```

### §6 — OBLIGATION × AUTONOMY INTERACTION

```
OUTLINE:

  Autonomy-Hardware: self-action = reward (efference copy)
  Obligation = things you CANNOT self-action → NEED agent

  Spectrum:
    Full autonomy → obligation = 0 → reward = efference copy + opioid
    Partial dependency → obligation MODERATE → reward mixed
    Full dependency → obligation MAX → autonomy GIẢM → helplessness risk

  Developmental:
    Bé nhỏ: full dependency on parents → obligation "vô hình" (chưa có chunks detect)
    Lớn lên: autonomy TĂNG → obligation GIẢM → nhưng MỚI patterns emerge
    Adult: chose dependencies (work, marriage, loans) → obligation CHỌN (efference copy!)

  ⭐ CHỌN obligation ≠ BỊ ÉP obligation:
    → Chọn: efference copy + obligation = NET OK (tự nguyện)
    → Bị ép: no efference copy + obligation = NET NEGATIVE (resentment)
    → = Tại sao "tự chọn đi làm" khác "bị bắt đi làm" dù cùng work
```

### §7 — OBLIGATION VỚI VIRTUAL AGENT

```
OUTLINE:

  Đeo ảnh Đức Mẹ:
    → Schema: "Đức Mẹ bảo vệ tôi" → Agent_value CAO
    → Obligation: đọc kinh = cost DUY TRÌ trust chain
    → Anchor STRONG: lười đọc → identity dissonance → guilt
    → Anchor WEAK: lười đọc → schema decay → ít dissonance → obligation dissolve

  Cầu nguyện = obligation toward virtual agent:
    → "Tôi PHẢI cầu nguyện" = obligation
    → NHƯNG cost THẤP (vài phút) + reward CAO (anchor sync + comparison + cortisol ↓)
    → = Obligation LOW-COST + HIGH-REWARD = sustainable

  → Obligation = f(anchor strength) cho virtual agents
```
