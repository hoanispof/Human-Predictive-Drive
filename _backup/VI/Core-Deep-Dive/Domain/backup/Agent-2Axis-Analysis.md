## §1 — OBJECT-AGENT REFRAME: Unified Model

### Insight gốc

User đặt câu hỏi: "Object-Agent có thật sự binary không?" và đề xuất reframe:
- Mọi thứ (bố, mẹ, ti vi, tủ lạnh, cục đá, pho tượng, mẫu ảnh Đức Mẹ) đều sờ được, nhìn thấy, có chunks logic định nghĩa
- Một số entities được THÊM modeling overlay (bố, mẹ, bạn bè)
- Một số entities trigger schema mà không cần modeling (tượng thiêng)

### Unified Model

```
🟡 3-LAYER MODEL cho mọi entity trong domain:

  LAYER 1 — OBJECT CHUNKS (foundation, mọi entity có):
    → Physical properties: kích thước, vị trí, cấu trúc, modality input
    → Logic rules: cách entity hoạt động tại level physical
    → Có thể touch, see, hear (hoặc imagine thêm nếu khuất)
    → Compiled qua repeated observation
    → = "Cái gì tôi nhìn thấy, sờ được, định nghĩa được bằng rules"

  LAYER 2 — LOGIC PROCESSING (predict via rules):
    → Dựa trên object chunks → predict behavior
    → Works KHI: entity behavior deterministic theo rules
    → Sufficient cho: most objects, schema-driven entities

  LAYER 3 — MODELING OVERLAY (optional, add khi cần):
    → Add KHI: behavior unpredictable, need own-state mapping
    → Contains: agent model, mirror prediction, state tracking
    → Works BY: simulate "nếu tôi ở vị trí họ → tôi feel/want gì"
    → Varies in depth: minimal (cây, côn trùng) → deep (mẹ ruột)

  LAYER 4 — SCHEMA OVERRIDE (can bypass or reshape):
    → Community/cultural rules override natural classification
    → Can add "agent" to objects (religious icons)
    → Can remove "agent" from people (dehumanization)
```

### Examples across spectrum

```
Cục đá:        LAYER 1 + 2 (logic đủ) — 0% modeling
Tủ lạnh:       LAYER 1 + 2 — 0% modeling
Cái bút:       LAYER 1 + 2 — 0% modeling
Cây:           LAYER 1 + 2 + MINIMAL LAYER 3 (proto-state: khỏe/héo)
Côn trùng lạ:  LAYER 1 + 2 + MINIMAL LAYER 3 (basic "escape/eat")
Chó quen:      LAYER 1 + 2 + PARTIAL LAYER 3 (state tracking)
Người lạ:      LAYER 1 + 2 + DEFAULT LAYER 3 (species overlap)
Bạn thân:      LAYER 1 + 2 + DEEP LAYER 3
Mẹ ruột:       LAYER 1 + 2 + DEEPEST LAYER 3
Tượng Đức Mẹ: LAYER 1 + 2 + SCHEMA OVERRIDE (Mode 1: 0% modeling,
                                              Mode 2: imagined modeling)
```

### Insight sâu

**Không phải "Object vs Agent binary", mà là "Object base + spectrum of modeling overlay"**

- Hardware VTC classification vẫn binary (animate/inanimate) 🟢
- Nhưng schema level là CONTINUOUS — modeling requirement từ 0% → 100%
- "Agent" = entity CẦN modeling để predict
- "Object" = entity KHÔNG CẦN modeling (logic đủ)
- Schema có thể add hoặc remove modeling requirement

## §2 — ĐỨC MẸ CASE: Pure Schema vs Imagined Modeling

### Core insight

User phát hiện: Đức Mẹ / Thiên Chúa có thể được processed theo 2 modes khác nhau.

### Mode 1 — Pure Schema (0% modeling)

```
🟡 MODE 1 — Logic/Schema thuần:

  Schema rule: "Đức Mẹ LUÔN yêu thương, LUÔN bảo vệ"
  → = ZERO uncertainty về state Đức Mẹ
  → Không bao giờ "Đức Mẹ hôm nay có giận mình không?"
  → Vì schema đã ĐỊNH NGHĨA: state = CONSTANT positive
  → = Entity được DEFINE HOÀN TOÀN bằng logic (qua schema)
  → = 100% predictable → modeling KHÔNG CẦN
  → Cầu nguyện = trigger schema, không phải communicate với agent

  Đặc điểm:
    → Zero cognitive load
    → Zero uncertainty → maximum comfort
    → Daily devotion, rote prayer
    → Schema fire → body comfort → done

  Tại sao tôn giáo "work" cho Mode 1 users:
    → Real agents (mẹ thật) có uncertainty → disappointment possible
    → Schema agent (Đức Mẹ) zero uncertainty → never disappoint
    → "Placebo of certainty" — real body response, schema-driven
```

### Mode 2 — Imagined Modeling (optional overlay)

```
🟡 MODE 2 — Schema + imagined modeling:

  Tín đồ TƯỞNG TƯỢNG Đức Mẹ như agent có state:
    → "Đức Mẹ đang NHÌN tôi, đang BUỒN vì tôi phạm lỗi"
  → Body fire modeling circuits: simulate Đức Mẹ's "feelings"
  → Mirror FIRE trên virtual agent (compiled qua PFC)
  → = Mechanism ③ Schema Simulation + added modeling layer

  Đặc điểm:
    → Deeper emotional intensity
    → Higher cognitive load
    → Mystical experience, intense personal prayer, guilt experiences
    → Fill agent input need via imagined agent presence

  Trade-off:
    Mode 1: lighter, routine, maximum efficiency
    Mode 2: deeper bond, deeper cognitive load
    → Tín đồ có thể switch giữa 2 modes tùy context
```

### Framework implication

**Schema Simulation (Domain-Mechanisms.md §1.2 mechanism ③) KHÔNG bắt buộc có modeling overlay.**

Entity 100% schema-based có thể:
- Fire PURE logic (Mode 1) — đủ cho comfort
- Fire logic + imagined modeling (Mode 2) — deeper emotional

Cần refine Domain-Mechanisms.md §1.4 (Agency Spectrum) để clarify 2 modes này.

## §3 — WHY ENTITIES NEED MODELING: 4 Reasons

```
🟡 4 LÝ DO entity cần LAYER 3 (modeling overlay):

  REASON A — UNPREDICTABLE BEHAVIOR:
    → Behavior không theo rules simple
    → VD: mẹ thật (hôm nay vui/buồn/mệt varied)
    → Solution: build agent model via interaction
    → Result: compiled model → predict state → reduce uncertainty

  REASON B — HIGH-STAKES INTERACTION:
    → Entity có thể impact body-base significantly
    → Kể cả predictable behavior vẫn worth modeling
    → VD: sếp (routine behavior nhưng impact career → worth model)
    → Solution: deeper tracking để optimize interaction

  REASON C — SOCIAL BOND / AGENT INPUT NEED:
    → Body CẦN social presence → cần feel "connected"
    → Modeling = way để feel connection
    → VD: bạn thân — even routine behavior, modeling feels good
    → Solution: ongoing mutual modeling → connection pattern

  REASON D — IMAGINED PRESENCE:
    → Entity không có real state → nhưng người imagine cho nó
    → VD: Đức Mẹ Mode 2, người đã mất, character văn học
    → Solution: PFC simulate virtual agent
    → Benefit: social presence without real agent
    → Cost: no real feedback calibration

  NO MODELING NEEDED:
    → Physics-deterministic (cục đá, bàn, tường)
    → Schema-defined absolute (Đức Mẹ Mode 1: "luôn yêu → done")
    → Routine-complete (đồng nghiệp gặp chào hỏi xong đi)
```

## §4 — AGENT 2-AXIS MODEL

### Đề xuất gốc

User đề xuất framework Agent bằng 2 axes:

```
AXIS 1: UTILITY (multi-channel impact on body-base)
  → Entity này affect body-base channels thế nào?
  → Scale: highly negative ← 0 → highly positive
  → Multi-channel hoặc single channel

AXIS 2: MODELING REQUIREMENT (không phải mirrorability)
  → Cần modeling bao nhiêu để predict entity?
  → Scale: 0% (logic đủ) → 100% (modeling dominant)
```

### Tại sao "Modeling Requirement" chứ không phải "Mirrorability"

User insight: Mirrorability sẽ gây confusion cho schema entities.

```
Ví dụ Đức Mẹ:
  → Nếu axis 2 = "mirrorability" → Đức Mẹ LOW mirrorability
  → Gây confusion: như thể người không hiểu được Đức Mẹ
  → Thực tế: không CẦN mirror — schema đã đủ
  → "Mirrorability LOW" ≠ "processing failure"

"Modeling Requirement" chính xác hơn:
  → Cục đá: 0% (không cần modeling)
  → Đức Mẹ: 0% (Mode 1) hoặc 30% (Mode 2, optional)
  → Chó quen: 40% (some modeling helps)
  → Mẹ: 80% (deep modeling needed)
```

### 4 Quadrants

```
🟡 4 QUADRANTS = 4 TYPES OF ENTITY RELATIONSHIPS:

  ┌──────────────────────────────────────────────────────────────┐
  │                                                                │
  │  HIGH UTILITY × LOW MODELING  │  HIGH UTILITY × HIGH MODELING │
  │                                │                                │
  │  Schema agents:                │  Real relationships:           │
  │  • Đức Mẹ (Mode 1)             │  • Mẹ, bố ruột                 │
  │  • Thiên Chúa                  │  • Bạn thân                    │
  │  • Tượng thiêng                │  • Vợ/chồng                    │
  │  • Lucky charm                 │  • Con                         │
  │  • Amulet / locket người yêu  │  • Chó quen sâu sắc            │
  │                                │                                │
  │  Tools đơn giản:               │                                │
  │  • Tủ lạnh (food access)       │                                │
  │  • Xe máy (di chuyển)          │                                │
  │  • Tiền (flexible utility)     │                                │
  │                                │                                │
  │  = ROLE-BASED RELATIONS        │  = AGENT RELATIONSHIPS          │
  │  Schema + logic đủ              │  Need active modeling           │
  │  Low cognitive cost             │  High cognitive cost            │
  │                                │                                │
  ├────────────────────────────────┼────────────────────────────────┤
  │                                │                                │
  │  LOW UTILITY × LOW MODELING    │  LOW UTILITY × HIGH MODELING   │
  │                                │                                │
  │  Background objects:           │  Obsessive fixation:            │
  │  • Cục đá random               │  • Crush không đáp lại          │
  │  • Tường, trần nhà             │  • Celebrity tôn sùng           │
  │  • Côn trùng thoáng qua        │  • Enemy ám ảnh                 │
  │  • Người lạ qua đường          │                                │
  │                                │  Weak bonds:                    │
  │  Background living:            │  • Bạn cũ lâu không gặp         │
  │  • Cây xa lạ                   │  • Thầy giáo cũ                 │
  │  • Chó hàng xóm không tương    │                                │
  │    tác                          │  = HIGH MODELING, LOW PAYOFF    │
  │                                │  Often unhealthy pattern        │
  │  = BACKGROUND NOISE             │                                │
  │  Minimal cognitive investment  │                                │
  │                                │                                │
  └────────────────────────────────┴────────────────────────────────┘
```

### Insight từ 4 quadrants

1. **Top-left (HIGH util × LOW model) = UNDERVALUED category** trong framework cũ
   - Schema agents + simple tools
   - Very efficient: high benefit, low cognitive cost
   - Đức Mẹ, tủ lạnh, xe máy = cùng loại "reliable utility"
   - Tôn giáo exploit exactly this: maximum utility, minimum uncertainty

2. **Top-right = Real relationships** — cost nhiều modeling nhưng payoff cao
   - Deep bond patterns (EP.md §2 Connection)

3. **Bottom-left = Background noise** — brain filter out
   - Most things in environment

4. **Bottom-right = Unhealthy patterns** — high investment, low return
   - Obsession, celebrity parasocial, enemy fixation
   - Modeling cost > benefit → net negative

### Dynamic movement

Relationships NATURALLY move giữa quadrants over time:

- New relationship → Top-right (learning)
- Compiled → Top-left (routine established)
- Crisis → Top-right (re-modeling needed)
- Post-conflict → Bottom-left (background)

## §7 — LOGIC-MODELING → LOGIC-FEELING? (Major consideration)

### User's deep insight

User challenge: "Modeling" là từ AI terminology, có đúng cho não người không?

```
🟡 "MODELING" PROBLEM:
  → Borrowed from AI/computational neuroscience
  → Implies explicit computation, step-by-step
  → Implies conscious algorithmic process
  → Implies programmatic representation

  Nhưng khi bạn "hiểu" mẹ:
  → KHÔNG chạy algorithm
  → KHÔNG compute explicit model
  → KHÔNG conscious step-by-step
  → Chỉ: NHÌN mẹ → FEEL "mẹ đang lo"
  → FEEL fire TRƯỚC khi PFC verbalize
  → = "Modeling" = description SAI về HOW it feels
  → = "Modeling" = description của OUTPUT, không phải PROCESS
```

### Einstein example supports "Feeling" framing

Einstein KHÔNG "model" tia sáng mathematically first. Ông FELT what it would be like to ride one.
- "Tôi cưỡi tia sáng, tôi thấy gì?"
- = Body-based simulation (mirror mechanism)
- Chỉ SAU đó mới verify bằng math
- = FEELING trước → LOGIC sau

### Research supports "Feeling" framing

- 🟢 Damasio somatic marker hypothesis (1994, 1996, 1999)
- 🟢 Gendlin felt sense / Focusing (1978)
- 🟢 Craig interoception research (2002, 2013)
- 🟢 Gallese embodied simulation (2007)
- 🟢 Kahneman System 1 (2011)
- 🟢 Panksepp affective neuroscience
