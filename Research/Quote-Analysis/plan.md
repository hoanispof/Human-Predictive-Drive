## §1 — MỤC ĐÍCH

```
3 GIÁ TRỊ CỦA QUOTE ANALYSIS:

  ① CHO NGƯỜI ĐỌC QUOTE:
    Quote hay thường được trích dẫn nhưng apply CỨ​NG NHẮC.
    "Stay Hungry" → workaholism. "Know the goal" → micro-manage.
    Analysis giúp: HIỂU mechanism → biết KHI NÀO apply, KHI NÀO KHÔNG.
    = Quote trở thành TOOL, không phải SLOGAN.

  ② CHO FRAMEWORK:
    Stress test: nếu framework giải thích được TẠI SAO 20-30 câu quote
    từ nhiều ngành, nhiều thời đại lại đúng — bằng CÙNG bộ mechanism
    → evidence MẠNH cho explanatory power.
    Nếu có quote framework KHÔNG giải thích được → gap cần xem lại.

  ③ CHO NGƯỜI VIẾT:
    Mỗi quote = 1 bài tập apply framework vào real-world wisdom.
    = Luyện ability: nhìn surface observation → trace mechanism underneath.
```

## §2 — TIÊU CHÍ CHỌN QUOTE

```
⭐ KHÔNG PHẢI MỌI QUOTE ĐỀU ĐÁNG PHÂN TÍCH:

  ✅ CHỌN KHI:
    ① Quote MÔ TẢ pattern thực sự (không chỉ motivational fluff)
    ② Framework CÓ mechanism giải thích (không phải re-describe)
    ③ Có FAILURE MODES rõ ràng (khi nào quote SAI)
    ④ Có CALIBRATION value (giúp người đọc apply tốt hơn)
    ⑤ Quote phổ biến đủ để nhiều người benefit từ analysis

  ❌ KHÔNG CHỌN KHI:
    ✗ Quote chỉ là wordplay đẹp nhưng không có mechanism underneath
    ✗ Quote quá specific cho 1 domain (framework = general mechanism)
    ✗ Quote mà framework chỉ re-describe (không ADD insight)
    ✗ Quote trivially đúng (không cần analysis)

  PRIORITY:
    Tier 1: Quote có PARADOX hoặc COUNTER-INTUITIVE element
      → Framework giải thích TẠI SAO paradox là apparent, not real
      → Ví dụ: "Stay Foolish" — counter-intuitive, framework explains cost mechanism
    Tier 2: Quote phổ biến bị MISAPPLY thường xuyên
      → Framework giúp calibrate đúng
      → Ví dụ: "Follow your passion" — bị apply cứng nhắc
    Tier 3: Quote confirmed bởi NHIỀU cultures/eras
      → Evidence cho universal mechanism
      → Ví dụ: "Know thyself" — từ Socrates tới modern psychology
```

## §3 — TEMPLATE CHUẨN CHO MỖI FILE

```
FILE NAMING:
  Work-[Concept-Keywords].md
  Ví dụ: Work-Goal-And-Why.md, Work-Stay-Hungry-Stay-Foolish.md
  KHÔNG đặt tên theo tác giả — đặt theo CONCEPT.
  Attribution ghi trong file.

YAML HEADER:
  title, version, created, status, scope, purpose, position, dependencies
  dependencies organized by framework area (Gap, Bias, Novelty, Anchor, etc.)

SECTIONS CHUẨN:

  §0 — QUOTE + SOURCE + CONTEXT
    Quote gốc chính xác.
    Source: verified hay floating attribution?
    Context: tác giả nói trong situation nào?
    So sánh với file trước (nếu có pattern).

  §1-§N — GIẢI MÃ TỪNG PHẦN CỦA QUOTE
    Mỗi keyword/phrase = 1 section.
    Mapping vào framework mechanism CỤ THỂ.
    Không chỉ "concept A = concept B" — phải chỉ ra MECHANISM.
    Include: VD cụ thể + research citations.

  §SYNERGY — KẾT HỢP (nếu quote có nhiều phần)
    Tại sao CÁC PHẦN cùng lúc > tổng riêng lẻ.
    Framework insight KHÔNG có trong quote gốc.

  §FAILURE — FAILURE MODES
    Khi nào quote SAI hoặc NGUY HIỂM.
    Mỗi failure = cụ thể (tên, mechanism, VD).
    QUAN TRỌNG NHẤT cho giá trị thực tế.

  §CALIBRATION — CÁCH APPLY KHÔNG CỨNG NHẮC
    Numbered principles (4-6 principles).
    Per-person variation.
    Domain feedback = arbiter.

  §HONEST — HONEST ASSESSMENT
    🟢/🟡/🔴 confidence cho MỖI claim.
    Framework vs re-description evaluation.
    Research citations.

  §CROSS-REF — CROSS-REFERENCES
    Organized by area. Version numbers.

QUALITY STANDARDS:
  - Mỗi file: ~400-500 lines
  - Cross-ref: 10+ framework files
  - Research: 8+ citations
  - Failure modes: ≥ 3
  - Calibration principles: 4-6
  - Honest assessment: PHẢI có 🟡/🔴 (không chỉ 🟢)
```

## §5 — QUY TRÌNH TRIỂN KHAI

```
⭐ WORKFLOW CHO MỖI QUOTE:

  PHASE 1 — VERIFY + READ (~30 min):
    ① Verify quote: exact wording, source, context, date.
    ② Identify framework mechanisms: 3-5 primary files cần đọc.
    ③ Đọc kỹ sections liên quan (không đọc toàn bộ file).
    ④ Check: framework GIẢI THÍCH hay chỉ RE-DESCRIBE?
       Nếu chỉ re-describe → bỏ, chọn quote khác.

  PHASE 2 — PLAN (~15 min):
    ⑤ Draft mapping: quote element → framework mechanism.
    ⑥ Identify failure modes (≥ 3).
    ⑦ Draft calibration principles.
    ⑧ Present plan cho review.

  PHASE 3 — WRITE (~45 min):
    ⑨ Write file theo template §3.
    ⑩ Cross-check: mỗi claim có citation?
    ⑪ Honest assessment: 🟢/🟡/🔴 realistic?

  PHASE 4 — REVIEW:
    ⑫ Re-read: có chỗ nào chỉ re-describe?
    ⑬ Check failure modes: realistic? Có miss obvious one?
    ⑭ Final: file sẵn sàng.

  BATCHING:
    Có thể làm 2-3 quotes/session nếu framework mechanisms overlap.
    Ví dụ: ① + ② (cả hai liên quan Compiled/Fresh) → batch.
    Ví dụ: ⑤ + ⑬ (cả hai liên quan Compiled Quality) → batch.
```
