### §1.1 — Layer 1: Proxy Token for Prediction

```
⭐ MONEY = MATERIALIZED CHUNK PREDICTION:

  Money itself has NO body-base value:
    → Paper: can't eat it, can't stay warm with it
    → Numbers in a bank account: no mass, smell, or taste
    → = Zero body-base input

  Value lies in PREDICTION:
    → "Have $100 → buy a week's food → full → opioid"
    → "Have $1,000 → pay rent → safe → cortisol drops"
    → The brain does NOT reward for the money (paper/numbers)
    → The brain rewards for PREDICTION: "money → body-base GETS FED"

  = Money = distant prediction ENCODED into a physical object/number
  = Like a door: the door is not the room,
    but without the door → can't enter the room

  🟢 Knutson et al. (2001): nucleus accumbens fires when
     ANTICIPATING money, not when receiving it.
     = Prediction-delta from anticipation, not from possession.
  🟢 Kahneman & Deaton (2010): happiness increases with income
     up to ~$75K, then decelerates.
     = Proxy effective for survival, diminishing returns after.
  🟡 "Money = proxy token for prediction" = framework reframe
```

### §1.2 — Layer 2: Obligation Technology

```
⭐ MONEY = TECHNOLOGY FOR OPTIMIZING OBLIGATION (Obligation.md §5.2):

  Money is NOT ONLY a tool for exchanging goods.
  Money is ALSO a technology for optimizing obligation tracking:

  FUNCTION 1 — STANDARDIZE cross-domain cost:
    Without money: "fixing a machine vs typing a document = fair?" → PFC can't compare
    With money: "fix machine = $50, type document = $50" → SAME unit → fair exchange
    → Standardize eliminates asymmetry perception

  FUNCTION 2 — CONVERT implicit → explicit:
    Without money: "I helped you → you owe me... something" = TYPE 3 relational debt
      → Cost_clarity LOW + Endpoint LOW → cortisol sustained
    With money: "I helped you → pay $50 → DONE" = TYPE 1 financial
      → Cost_clarity HIGH + Endpoint HIGH → cortisol stops
    → Convert TYPE 3 → TYPE 1 = REDUCES obligation burden

  FUNCTION 3 — SCALE community exchange:
    Barter: O(n²) matching ("I have eggs, need apples,
      but the person with apples needs fish, not eggs")
    Money: O(1) per transaction ("sell eggs → $5 → buy apples")
    → Scaling solution for exchange at societal scale

  FUNCTION 4 — OPTIMIZE PFC prediction:
    Without money: "if I accept help → I have to give... what???" → overthinking
    With money: "if I accept help → pay $50" → PFC predicts ACCURATELY
    → Reduces cognitive load + reduces cortisol

  🟢 Money as medium of exchange = economics 101
  🟢 Crowding out effect (Frey & Jegen 2001): monetary reward → intrinsic ↓
  🟡 "Money = obligation technology (4 functions)" = framework synthesis
```

## §2 — WHY MONEY EXISTS: EVOLUTIONARY + SOCIAL FOUNDATION

### §2.1 — Evolutionary Roots: From Fighting → Exchange → Money

```
⭐ THE EVOLUTIONARY CHAIN:

  STAGE 0 — FIGHT EVERY TIME (extremely high cost):
    → 20 monkeys, every meal → fight
    → Cost: injuries, death, energy drain, time
    → = Cost PER INTERACTION = too expensive
    (Status.md §0)

  STAGE 1 — STATUS (Fight once, use many times):
    → Evolution selects: fight → remember result → reuse
    → Alpha eats first, sleeps in best spot → NO NEED to fight again
    → = Pre-calibrated resource allocation = STATUS
    → Cost: moderate up-front. Benefit: zero fighting cost thereafter
    → = Status = FIRST TECHNOLOGY to reduce cooperation cost
    (Status.md §0.1)

  STAGE 2 — EXCHANGE (Positive-sum):
    → Status hierarchy = zero-sum (1 alpha, rest subordinate)
    → Evolution selects further: cooperation = BOTH WIN
    → "I have eggs, you have fish → trade → BOTH better off"
    → = Positive-sum > zero-sum → cooperation wins
    → Exchange becomes the MOST COMMON MODE in modern humans
    (Status.md §5)

  STAGE 3 — OBLIGATION TRACKING:
    → Exchange requires TRACKING to be sustainable
    → "I gave you eggs, you HAVEN'T paid me back → DEBT"
    → Obligation = body-level bookkeeping system
    → Detect cheaters → punish → cooperation sustained
    → 🟢 Reciprocal altruism (Trivers 1971)
    (Obligation.md §5.1)

  STAGE 4 — MONEY:
    → Obligation tracking via the brain = LIMITED
      (can only track a few dozen people, high cost)
    → Money = EXTERNAL bookkeeping technology
    → Scales from a few dozen → billions of people
    → = The next technology in the chain of reducing cooperation cost
    (Obligation.md §5.2)


  ⭐ THE LOGICAL CHAIN:
    Fight every time → Status → Exchange → Obligation tracking → Money
    = Each step = TECHNOLOGY that reduces cooperation cost by one more order
    = Money is the CURRENT step in the chain, NOT the final step
    (AI may be the next step — §9)

  🟢 Status: cross-species evidence 350M+ years (Sapolsky, Huber & Kravitz)
  🟢 Exchange: comparative advantage (Ricardo 1817)
  🟢 Reciprocal altruism: Trivers 1971
  🟡 "Money = step in cooperation cost reduction chain" = framework synthesis
```

```
  WHY THIS FUNCTION IS THE STRONGEST:
    → Gap type = Chunk-Miss (expected survival → got less)

    → Both → cortisol FIRES → overrides all other priorities
    → Money = DIRECT TOOL to fill this gap
    → = Reason Schema [money = the best] has HIGH trust (§4.3)

  🟡 Function ① = most effective = framework confirms mainstream
```

### §3.3 — Function ③: Expand the Resource Access Map

```
⭐ HAVING MONEY = STATUS.MD RESOURCE ACCESS MAP EXPANDS:
```
