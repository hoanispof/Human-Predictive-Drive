## §1 — FOUNDATION: WORK WITH THRESHOLD

### §1.1 — Threshold = Hardware = Unchangeable

```
🟢 ESTABLISHED (ADHD-Observation.md §2):

  FOUNDATION PRINCIPLE:
    DRD4 threshold = HARDWARE parameter (genetic)
    → KHÔNG thay đổi bằng training, therapy, hoặc willpower
    → CÓ THỂ COMPENSATE bằng medication (shift threshold temporarily)
    → CÓ THỂ WORK WITH bằng environment + strategy

  Common mistakes:
    ❌ "Hãy tập trung hơn" → yêu cầu threshold GIẢM → impossible
    ❌ "Hãy cố gắng hơn" → yêu cầu PFC SUPPRESS threshold → costly, unsustainable
    ❌ "Meditation sẽ fix" → attention training ≠ changing receptor sensitivity
       (meditation CÓ benefits nhưng KHÔNG THAY ĐỔI DRD4 threshold)

  Correct approach:
    ✅ Hiểu threshold → design environment ABOVE threshold
    ✅ Chọn domain → where natural gaps EXCEED threshold
    ✅ Medication → temporarily shift threshold (chemical compensation)
    ✅ Strategies → work AROUND threshold (external scaffolding)
```

### §1.2 — Domain Selection = HIGHEST LEVERAGE

```
🟡 FRAMEWORK SYNTHESIS + 🟢 RESEARCH SUPPORT:

  🟢 Person-environment fit 2024 (Hotte-Meunier, systematic review):
    → Environment fit = CRITICAL variable for ADHD outcomes
    → Unsuitable environment EXACERBATES symptoms
  🟢 Self-employment 2016 (PMC5005387):
    → ADHD individuals → self-employment preference (customizable conditions)
  🟢 Tran 2026 (17,000+ meta-analysis):
    → ADHD traits + entrepreneurship = positive association

  WHY DOMAIN SELECTION = HIGHEST LEVERAGE:
    → Hardware: can't change
    → Medication: temporary, side effects
    → Strategies: helpful but effortful (PFC cost)
    → Domain selection: once found → SUSTAINABLE without ongoing effort
    → = RIGHT domain → ADHD threshold naturally exceeded → advantage
    → = WRONG domain → ADHD threshold never exceeded → disability
    → CÙNG PERSON, khác domain, khác outcome HOÀN TOÀN

  Gap-Direction Alignment (Gap-Direction.md):
    → Domain with DEEP chunk library → many gaps → clear direction

  HOW TO IDENTIFY RIGHT DOMAIN:
```

### §1.3 — A→B→C: From External to Internal Novelty Source

```
🟡 FRAMEWORK SYNTHESIS:

  Phase A — External-dependent (shallow chunks):
    → New domain → everything novel → VTA fires → engagement HIGH
    → As chunks compile → novelty DECREASES → engagement DROPS
    → = "Honeymoon phase" → then "boring" → then switch domain
    → MOST ADHD people stuck HERE → "can't stick with anything"

  Phase B — Transition (building depth):
    → PUSH through boring phase (external scaffolding needed)
    → Chunks accumulate → gaps emerge → some internal novelty starts
    → = "Sometimes interesting, sometimes boring" → unstable

  Phase C — Internal-driven (deep chunks):
    → Deep chunk library → MANY gaps → RICH recombination
    → Internal novelty > external novelty → self-sustaining
    → = "Expert" territory → ADHD = genuine advantage
    → Cross-domain connections (topic-jumping) = ASSET here
    → = "I never get bored in my field — always something new"

  CRITICAL TRANSITION A→B:
    → A→B = hardest (external novelty fading, internal not yet started)
    → = "The Valley" (Self-Created-Threat.md v1.1 §5.5)
    → MOST ADHD people QUIT HERE
    → Solution: scaffolding SPECIFICALLY for this transition
    → Once past B→C: self-sustaining, scaffolding can withdraw

  ⚠️ A→B→C model = framework construct. "Internal novelty" consistent
  with creativity research (expertise depth × novel connections) but
  specific ADHD transition duration = not measured.
```

## §2 — ENVIRONMENT DESIGN (3+1 CONDITIONS)

### §2.1 — Condition 1: Safety = Explicit Communication

```
🟡 FRAMEWORK SYNTHESIS + 🟢 RESEARCH:

  ADHD-Trade-Off.md §2.2 established: social gaze cueing SELECTIVELY impaired.
  → ADHD gets CONTENT but misses CONTEXT (micro-cues filtered).

  IN UNSAFE ENVIRONMENT:
    → Micro-cues contain IMPORTANT threat signals → ADHD misses
    → Gets blindsided → compile [unpredictable world]
    → → Anxiety accumulation → PFC weakened → worse performance

  IN SAFE ENVIRONMENT:
    → Communication TRỰC TIẾP = explicit channel dominant
    → ADHD gets FULL message through explicit channel
    → No surprise threats → no anxiety accumulation
    → PFC free → better performance → positive spiral

  "SAFE" FOR ADHD = "EXPLICIT":
    → "Tôi thấy vấn đề X, cần sửa Y" = SAFE (explicit, clear)
    → *sigh* + annoyed look = UNSAFE (micro-cue only, no explicit)

  FOR PARENTS:
    → "Con tự biết" = DANGEROUS assumption cho ADHD child
    → NÓI RÕ: "Mẹ muốn con dọn phòng trước 5 giờ" (explicit, timed)
    → KHÔNG: *nhìn phòng bừa bộn và thở dài* (micro-cue only)
    → 🟢 Meta-analysis (59 studies): parental warmth = INVERSE association
      with ADHD negative outcomes

  FOR WORKPLACE:
    → Written feedback > facial expression
    → "Good job on X, improve Y" = WORKS
    → Disappointed look without words = DOES NOT WORK
    → = MATCHING communication to hardware, not "dumbing down"
```

### §2.2 — Condition 2: Arc Decomposition with Tangible Output

```
🟡 FRAMEWORK SYNTHESIS:

  🟢 Concrete vs abstract goals: CONSENSUS in ADHD clinical literature
    → Concrete, specific goals > abstract, vague goals for ADHD

  ADHD big-arc problem:
    → Progress = invisible (no tangible delta until END)
    → Invisible progress → BELOW threshold → "boring" → abandon

  SOLUTION — mini-arcs (Gap-Body-Need.md v1.0 §6):
    → Break big-arc → mini-arcs (3-7 chunks each)
    → Each mini-arc has TANGIBLE output (visible, countable, shareable)

  DESIGN PRINCIPLES cho mini-arcs (ADHD-specific):

    ① SIZE: smaller than neurotypical mini-arcs
      → ADHD dopamine signal SHORT → reward window NARROW
      → Adjust granularity to severity:
        Mild: 2-3 days → tangible output
        Moderate: 1 day → tangible output
        Severe: hours → tangible output

    ② TANGIBILITY: must EXCEED threshold
      → "Made progress" ≠ tangible (abstract, invisible)
      → "Wrote 500 words" = tangible (countable, visible)
      → "Built working feature" = tangible (demonstrable)
      → Rule: if you can't SHOW someone → not tangible enough

    ③ TRANSITIONS: clear boundaries
      → "This mini-arc = done when X exists" (crisp definition)
      → Ambiguous endings → PFC cost → procrastinate
      → ADHD: need EXTERNAL definition (checklist, partner confirms)

    ④ VARIETY: alternate types within big-arc
      → Same type × repetition → habituate → below threshold
      → Mix: research → write → design → test → research...
      → = Keep novelty WITHIN the big-arc

  SCAFFOLDING OPTIONS:
    → Visual progress: Kanban boards, progress bars
    → Accountability: "show me today's output at 3pm"
    → Gamification: streak counters (with caution — see §3.2)
    → AI pair-work: immediate feedback loop
    → Physical artifact: print, check off, display
```

### §2.3 — Condition 3: Environment Fit (Novelty + Autonomy + Immediate Feedback)

```
🟡 FRAMEWORK SYNTHESIS + 🟢 RESEARCH:

  3 ELEMENTS that NATURALLY exceed threshold:

  ① NOVELTY in task:
    → Tasks with inherent novelty → VTA fires → above threshold
    → Software development: each bug = novel puzzle
    → Emergency medicine: each patient = novel challenge
    → Entrepreneurship: each day = novel problem
    → CONTRAST: accounting routine, data entry, scheduled meetings
    → = WHY certain careers ATTRACT ADHD (self-selection for novelty)

  ② AUTONOMY:
    → Choose WHAT to work on → choose ABOVE-threshold tasks
    → No autonomy → FORCED on BELOW-threshold tasks
    → PFC: suppress [want to do X] + hold [must do Y] = DOUBLE COST
    → Autonomy eliminates suppress cost → PFC full budget for work

  ③ IMMEDIATE FEEDBACK:
    → ADHD dopamine signal SHORT → delayed feedback BELOW threshold
    → Programming: compile → error/success → IMMEDIATELY visible
    → Gaming: action → result → IMMEDIATELY visible
    → CONTRAST: write report → feedback in 2 weeks → FAR below threshold
    → = WHY ADHD people love gaming/coding: immediate feedback loop

  WHEN ALL 3 PRESENT:
    → Tasks naturally exceed threshold → less regulation needed
    → Less PFC cost → = "I don't have ADHD when I'm coding"
    → = SAME hardware, threshold EXCEEDED by environment

  🟢 EVIDENCE:
    🟢 Person-environment fit 2024: CRITICAL for ADHD outcomes
    🟢 Tran 2026: ADHD traits over-represented in entrepreneurship
    🟢 Self-employment 2016: better fit, customizable conditions
```

## §3 — REWARD ENGINEERING: ANTICIPATION DEFICIT

### §3.1 — Anticipation vs Consummation: The Core Problem

```
🟢 RESEARCH (ADHD-Trade-Off.md §5.2):

  🟢 Plichta & Scheres 2014: VS hyporesponsive during ANTICIPATION (d=0.48-0.58)
  🟢 PLOS ONE 2014: consummation NORMAL or INCREASED

  = ADHD can't anticipate reward → can't initiate
  = BUT once reward arrives → strong response → engaged

  "MOTIVATION ≠ ACTIVATION":
    → ADHD person WANTS to start (motivated)
    → But CAN'T activate (PFC hypoactivation + dopamine deficit)
    → Urgency compensates: deadline → arousal → initiation
    → = "Activation energy" = PFC initiation threshold

  IMPLICATION FOR OPTIMIZATION:
    → Don't say "just find motivation" → motivation EXISTS
    → Problem: converting motivation → ACTIVATION
    → Solution: reduce activation energy (lower initiation barrier)
    → = Environment design (§2) + reward engineering (§3.2-§3.3)
```

### §3.2 — From Feast-or-Famine to Steady Mini-Rewards

```
🟡 FRAMEWORK SYNTHESIS:

  CURRENT ADHD PATTERN:
    → BIG reward when hyperfocus + success = FEAST (dopamine flood)
    → NOTHING when below-threshold task = FAMINE (dopamine absent)
    → = Binary: everything or nothing

  OPTIMIZATION — convert to steady micro-rewards:

  Step 1 — DETECT pattern:
    → Track 1 week: when engaged? when dead?
    → Map: task × engagement level
    → = Diagnostic baseline

  Step 2 — DESIGN mini-arcs for famine tasks (§2.2):
    → Famine task = below threshold → needs engineering
    → Options:
      a) Attach to meta-task ("clean desk in 5 min while music plays")
      b) Pair with immediate reward ("after email, 5 min X")
      c) Create artificial novelty (same task, different approach)
      d) Social accountability ("show you at 3pm")

  Step 3 — IMPLEMENT adapted work blocks:
    → Standard Pomodoro: 25 min / 5 min break
    → ADHD adaptation: WIDER VARIATION than standard
      🟡 No ADHD-specific Pomodoro research found
      → Framework suggests: adjust by body feedback, not fixed formula
    → KEY: break must be GENUINE reward (ACTIVE, not passive rest)

  Step 4 — MONITOR via body feedback:
    → Mini-arcs too long? → signs: dread, avoidance (shorten)
    → Mini-arcs too short? → signs: momentum broken (lengthen)
    → Body = final arbiter (Ask-AI.md v3.1 §6.1 dual check)
    → = DYNAMIC adjustment — not fixed formula

  ⚠️ This is SCAFFOLDING for necessary below-threshold tasks.
  In above-threshold domain → scaffolding UNNECESSARY (natural flow).
```

### §3.3 — Pair Evaluative + Direct-State Rewards

```
🟡 FRAMEWORK SYNTHESIS (applying Reward-Calibration.md v1.1):

  2 reward types:
    Direct-State: body directly feels (warmth, food, movement)
    Evaluative: PFC-mediated assessment (achievement, progress)

  ADHD IMBALANCE:
    → Evaluative reward WEAKER (needs sustained PFC → under-fueled)
    → Direct-State = NORMAL (body hardware intact)
    → = "Can't feel satisfaction from achievement" but CAN feel physical pleasure

  STRATEGY — pair evaluative WITH direct-state:
    → "Completed report" (evaluative) + "favorite snack" (direct-state)
    → "Finished workout" (evaluative) + "hot shower" (direct-state)
    → = Bridge: evaluative → direct-state → body REGISTERS reward
    → Over time: evaluative alone starts COMPILING with body reward
    → = Building genuine reward pathway for achievement

  ⚠️ TRAP — Over-reward (Reward-Calibration.md v1.1 §3):
    → Too much external reward → "overjustification effect"
    → "Good job" (acknowledge) ≠ "$100 for cleaning room" (overjustify)
    → Balance: reward should ACKNOWLEDGE, not OVERSHADOW
```

## §4 — EXTERNAL PFC SCAFFOLDING

### §4.1 — Concept: Externalize What PFC Can't Sustain

```
🟡 FRAMEWORK SYNTHESIS + 🟢 RESEARCH:

  🟢 Cognitive offloading (Nature 2026): well-established phenomenon
  🟢 ADHD scaffolding meta-analysis 2020: most effective for symptoms

  CORE INSIGHT:
    ADHD = PFC under-fueled → can't sustain certain functions internally
    → Externalize those functions to environment/tools/people
    → = "External PFC" — build external equivalent, don't fix internal

  5 PFC FUNCTIONS TO EXTERNALIZE:

  ① WORKING MEMORY (Hold Duration):
    Internal: PFC holds ~4±1 items, ADHD hold duration SHORT
    External:
      → Voice memos (LOWEST friction — capture before fade)
      → Digital: Notion, Obsidian, task managers
      → Physical: sticky notes, whiteboards
      → AI: "remember this for me"
    KEY: capture system must be FRICTIONLESS
      High friction → won't use → items lost

  ② INHIBITION (Suppress):
    Internal: PFC suppress impulses, ADHD suppress WEAKER
    External:
      → Website blockers (Cold Turkey, Freedom)
      → Phone in another room (PHYSICAL barrier, not willpower)
      → Partner: "tell me if I'm drifting"
    KEY: PHYSICAL BARRIER > willpower
      "Don't check phone" (willpower) = PFC cost → fails
      Phone in another room (physical) = no PFC cost → works

  ③ TIME PERCEPTION:
    Internal: PFC maintains time model, ADHD = time-blind
    External:
      → Visual timers (Time Timer: RED shrinks visually)
      → Phone alarms (multiple, per-task)
      → Calendar blocking (time → visible)
    KEY: make time VISIBLE
      "You have 2 hours" = abstract → body ignores
      Timer counting DOWN visually = concrete → body registers

  ④ PLANNING (Sequential):
    Internal: PFC sequences steps, ADHD = jump to interesting
    External:
      → Written step-by-step (1. 2. 3. — visible order)
      → Checklist (physical: cross off → tangible reward)
      → AI: "break this into steps for me"
    KEY: WRITTEN, not mental
      Mental plan = PFC hold → fade → lost
      Written plan = persistent → can return after interruption

  ⑤ EMOTIONAL REGULATION:
    Internal: PFC modulates response, ADHD = binary response
    External:
      → Pre-agreed protocols ("when frustrated, walk 5 min")
      → Partner: "you seem escalating, let's pause"
      → Breath counting (physical anchor)
    KEY: pre-compiled protocols, not real-time decisions
      Real-time regulation = PFC under load → fails
      Pre-compiled protocol = automatic → works
```

### §5.2 — Sleep: Circadian + Quality

```
🟢 RESEARCH:

  🟢 Sleep/circadian 2025 (Frontiers, systematic review):
    → 73-80% ADHD individuals have circadian alterations
    → DSPD (Delayed Sleep Phase Disorder): 26-78% prevalence in ADHD

  ADHD × SLEEP = COMPOUND PROBLEM:
    → Delayed sleep phase: body clock SHIFTED later (common in ADHD)
    → DMN interference: racing thoughts at bedtime (can't "turn off")
    → → Sleep quality LOWER → PFC repair INSUFFICIENT → worse next day

  OPTIMIZATION:
    ① CONSISTENCY > TIMING:
      → Even if late bedtime → CONSISTENT late bedtime > variable
      → Body clock needs PREDICTABILITY for repair cycle
    ② PRE-SLEEP ROUTINE:
      → Compile body expectation: specific ritual → body expects sleep
      → Screen curfew (blue light + stimulation → threshold)
    ③ MEDICATION TIMING:
      → Stimulant too late → sleep disrupted
      → Stimulant too early → afternoon crash → evening overcompensate
      → = Medical team decision, not self-optimization
    ④ MELATONIN:
      → May help with phase delay (consult medical professional)
      → Framework: doesn't fix mechanism but compensates timing
```

## §7 — BIG-ARC + DRIVE MANAGEMENT

### §7.1 — Valley Prediction + Preparation

```
🟡 FRAMEWORK SYNTHESIS (applying Self-Created-Threat.md v1.1 §5.5):

  Self-Created-Threat.md §5.5: Full Drive Cycle:
    ARC START: mostly PULL (novelty + curiosity → drive high)
    ARC MIDDLE ("The Valley"): PULL fades, PUSH needed
    ARC END: PULL returns (completion near → dopamine rises)

  ADHD-SPECIFIC VALLEY:
    → Valley DEEPER (novelty depleted → BELOW threshold → ZERO drive)
    → Valley WIDER (longer to reach "completion near" from middle)
    → Valley HARDER TO EXIT (PFC under-fueled → can't self-push)

  5 PREPARATION STRATEGIES:

  ① PREDICT WHEN:
    → Most projects: ~20-40% completion = valley entry
    → KNOW IN ADVANCE: "valley = ARCHITECTURE, not personal failure"
    → = Cognitive prep → reduces cortisol spike when valley hits

  ② PRE-POSITION SCAFFOLDING:
    → Before valley: set up accountability (partner expects progress)
    → Before valley: break valley-section into EXTRA-small mini-arcs (§2.2)
    → Before valley: prepare REWARDS
    → = Build scaffolding WHILE PFC has capacity (not during valley)

  ③ REFRESH PULL:
    → Re-visualize Imagine-Final (Imagine-Final.md v3.1 — reconnect with WHY)
    → Share progress with caring person (social reward)
    → Find NOVEL APPROACH to same task (refresh novelty within arc)
    → AVOID: "just push through" (= PFC suppress → will fail)

  ④ PLANNED ARC SWITCHING (not impulsive):
    → Valley too deep → PLANNED switch to different arc temporarily
    → "I'll do Project B for 2 hours, then return to Project A"
    → Return to A: novelty partially refreshed (absence → delta)
    → KEY DISTINCTION:
      Impulsive switching: no plan to return → arc abandoned
      Planned switching: return time SET → arc preserved

  ⑤ ARC GRAVEYARD PREVENTION:
    → Each unfinished arc = [can't finish] Background-Pattern reinforcement (Trade-Off §4.1 ③)
    → Prevention:
      Before starting: "Am I committing through valley? Who's my scaffolding?"
      IF abandoning: CONSCIOUS decision → "I chose to stop because X"
      → "I chose to stop" ≠ "I couldn't finish" → different self-model chunk
```

### §7.2 — 90:10 Pull:Push Applied

```
🟡 FRAMEWORK-DERIVED (from ADHD-Trade-Off.md §7.2):

  PULL strategies (90% of drive):
    → Curiosity: "what happens if..." (generative gap creation)
    → Novelty: vary approach within same goal
    → Connection: share progress with caring person
    → Mini-rewards: tangible output per mini-arc (§2.2)
    → Imagine-Final refresh: weekly vision reminder (externalize — §4.1④)

  PUSH strategies (10% — use SPARINGLY):
    → Time deadline: SHORT-TERM only ("finish mini-arc by 3pm")
    → Social accountability: "I told X I'd have this done"
    → = Brief, bounded, then immediately redirect to pull

  DANGER SIGNS (too much push):
    → Anxiety increasing (cortisol SPIKE pattern — Trade-Off §4.2)
    → "Dreading" the work (threat-compile beginning)
    → Sleep disrupted (cortisol → REM interference)
    → → REDUCE push, INCREASE pull, TAKE BREAK (§5)

  IMAGINE-FINAL PER ADHD:
    → ADHD imagination = STRONG (cross-domain, vivid)
    → BUT: PFC can't SUSTAIN Imagine-Final (hold duration short)
    → SOLUTION: make Imagine-Final TANGIBLE (externalize):
      → Vision board (physical/digital — VISUAL refresh)
      → Written "future letter" to self (read weekly)
      → Prototype/mock-up of end result
    → = Body re-fires each time see tangible reminder
    → = Imagination strength + external persistence = sustained drive
```

### §7.3 — Project Abandonment: Inattention > Hyperactivity

```
🟢 RESEARCH (NEW finding):

  🟢 PMC7515948:
    → INATTENTION (not hyperactivity) predicts switch costs
    → Inattention → goal NEGLECT → costly switching
    → Hyperactivity/impulsivity → may actually HELP some tasks

  FRAMEWORK IMPLICATION:
    → Project abandonment ≠ "too hyperactive to finish"
    → = Goal representation FADES from working memory (inattention)
    → = Valley + hold duration short → goal literally disappears from PFC
    → SOLUTION: EXTERNAL goal persistence (§4.1④ — written plan)
    → = Physical/digital reminder of goal PREVENTS goal neglect

  PER PRESENTATION TYPE:
    → Predominantly Inattentive (ADHD-PI): HIGHER abandonment risk
      → Goal neglect = primary mechanism → external persistence CRITICAL
    → Predominantly H/I: LOWER abandonment risk (impulsivity may drive action)
      → BUT: impulsive SWITCHING risk → planned switching protocol (§7.1④)
    → Combined: both risks compound → full scaffolding needed
```
