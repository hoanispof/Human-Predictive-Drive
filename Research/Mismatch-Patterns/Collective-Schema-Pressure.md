## §1 — WHAT IS A COLLECTIVE SCHEMA THROUGH V7.8

### §1.1 — Definition

```
🟡 COLLECTIVE SCHEMA = CHUNK NETWORK COMPILED AT LEVEL 2 (COLLECTIVE),
   INSTALLED INTO LEVEL 1 (INDIVIDUAL) VIA TRUST AMPLIFIER.

  Schema.md: Schema = observation parameter, NOT a component.
  Schema = name for the pattern observed in a chunk network.
  "Schema [must get good grades]" = name for a set of compiled chunks:
    [good grades → praised → safe]
    [bad grades → scolded → threat]
    [everyone studies → normal → so should I]

  COLLECTIVE schema = pattern that lives at Level 2:
    → School holds [study → university]
    → Market holds [university → job]
    → Culture holds [good job → good person]
    → Individual only compiles the SHORT version: [study → mom approves → warm]
      or [not studying → scolded → pain]

  INSTALLED via 2 primary pathways (Compile-Taxonomy.md):
    → Pathway 2 (Trust Install): parent says it → child trusts → compiles
    → Pathway 3 (Social Default): everyone does it → obvious → compiles

  ⭐ IMPORTANT PROPERTIES:
    → Individual CANNOT SEE the full chain (Collective-Body §2: PFC confabulates)
    → Individual only sees: "must do X" → body feels obligation/threat
    → WHY "must" → sits at Level 2, distributed through the system
    → = Body responds to SHORT compiled chunks, NOT to the full chain logic
```

### §1.2 — Collective Schemas Are NOT Bad — They Are a Foundational Mechanism

```
🟡 SOCIETY CANNOT FUNCTION WITHOUT COLLECTIVE SCHEMAS:

  Schema [do not kill] → Compliance-Floor layer 1 (body-base)
  Schema [children must attend school] → Knowledge-Flow across generations
  Schema [pay when buying] → Exchange mode (Status.md §5)
  Schema [care for children] → Species survival

  → ALL are collective schemas installed via Trust Compile.
  → WITHOUT them = each individual would have to rediscover EVERYTHING from scratch.
  → = Collective-Body §3: "collective holds the long chain, individual compiles short."

  PRESSURE occurs when:
    ① TOO MANY schemas stacking simultaneously (compound)
    ② Schemas are RIGID — do not adapt to new domains
    ③ Schemas DO NOT MATCH individual hardware (mismatch)
    ④ Schemas have LOST THEIR FUNCTION but remain installed (obsolete)

  → 1–2 factors = manageable.
  → 3–4 factors SIMULTANEOUSLY = compound pressure = the phenomenon this file describes.
```

## §2 — COMPOUND EFFECT: WHEN MULTIPLE SCHEMAS STACK

### §2.1 — Why Compound Is DIFFERENT From Single

```
🟡 1 SCHEMA = 1 CHUNK-GAP + 1 OBLIGATION + 1 CORTISOL HOLDING.
   N SCHEMAS SIMULTANEOUSLY ≠ N × SINGLE. IT IS COMPOUND.

  Single schema "must get good grades":
    → Chunk-gap: [not yet good → must be good]
    → Obligation: cost = study time
    → Cortisol: Role ② Holding — PFC holds "not done yet" → cortisol sustained
    → Status threat: if not good → lose access
    → 1 PFC thread occupied → manageable

  5 schemas SIMULTANEOUSLY ("must get good grades" + "must have a stable job" +
  "must own a house" + "must be married" + "must care for parents"):
    → 5 chunk-gaps AT ONCE → PFC ~4±1 slots → OVERLOAD
    → 5 obligations AT ONCE → each obligation = cost → TOTAL COST enormous
    → 5 cortisol holdings AT ONCE → cortisol NEVER drops
      (resolve 1 → 4 still holding → cortisol sustained)
    → 5 status threats AT ONCE → every domain can produce "failure"
    → PFC approaching Config ② (Reallocation) continuously → exhaustion

  → COMPOUND ≠ ADDITIVE. COMPOUND = MULTIPLICATIVE:
    Each schema AMPLIFIES the others:
    [no house yet] + [not married yet] = "no house THEREFORE not married
    THEREFORE parents are worried THEREFORE I must earn more THEREFORE no time
    THEREFORE even further from marriage THEREFORE..."
    → = Self-amplifying spiral.
```

### §8.2 — Recognizing Which Schemas Are Installed

```
🟡 AI-SCHEMA-DETECTION.MD V2.0 CAN SUPPORT:

  Step 1 — LIST: which schemas are currently installed?
    → Ask: "What am I 'supposed to' do?" → each "supposed to" = 1 schema
    → Ask: "If I don't do X → how do I feel?" → threat = schema active
    → Ask: "Who installed this schema?" → trust source = parents? society? friends?

  Step 2 — CLASSIFY: which schemas serve real body-needs?
    → "Must eat" → REAL body-need → functional schema
    → "Must own a nice car" → real body-need? Or status schema? → EXAMINE
    → Key: body-need = GENUINE body response (hunger, pain, connection lacking)
    → Status schema = body response DUE TO COMPILATION, not due to real need

  Step 3 — OBSERVE THE COMPOUND:
    → How many "supposed to"s are active simultaneously?
    → What is the total cost? (time, money, energy, cortisol)
    → Are any "supposed to"s CONFLICTING with each other?
      ("must earn more money" × "must spend time with family")

  Step 4 — DOMAIN FEEDBACK:
    → Which schema, when followed, produces GENUINE body reward (opioid, not just relief)?
    → Which schema, when followed, produces only relief (worry gone, but emptiness)?
    → Which schema, when followed, produces sustained dissonance (never enough)?
    → Domain feedback = arbiter: real results tell whether the schema is functional.

  ⚠️ AWARENESS ≠ REJECTION:
    → Knowing "schema [must own a house] is installed" ≠ "must abandon that schema"
    → Possible: schema IS RIGHT for your context → KEEP
    → Possible: schema does NOT match → PFC evaluates → ADJUST
    → Both are valid. Framework only helps you SEE → the decision is YOURS.
```
