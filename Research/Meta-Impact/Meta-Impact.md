  NHƯNG: framework từ người khác sẽ CÓ LENS KHÁC.
    → Academic lens: mạnh rigorous, yếu practical.
    → Hai framework CÓ THỂ BỔ SUNG thay vì thay thế.
    → First-mover advantage = SET FRAMING cho conversation,
       không phải "ai tìm ra trước."

## 9. Nếu Framework SAI — Failure Mode Analysis

```
🔴 FRAMEWORK TỰ TIN PHẢI TỰ HỎI: "SAI Ở ĐÂU?"

Có 3 LOẠI SAI KHÁC NHAU, hệ quả khác nhau:
```

### 9.1 Sai Cơ Chế Gốc (Node sai)

```
NẾU: Một node nền tảng từ conventional science bị overturn.

Ví dụ:
  - "Dopamine không phải prediction error signal"
    → Schultz (1997) bị falsify
    → §3 Root Channel Novelty sụp
    → Kéo theo: threshold, PE sensitivity, drive equation — TẤT CẢ liên quan dopamine.
    → HPD mất ~30-40% predictive power.

  - "Oxytocin không đặc hiệu cho bonding"
    → Root Channel Oxytocin mất precision
    → Nhưng: observable bonding behavior VẪN TỒN TẠI
    → Có thể: chỉ cần đổi tên + map lại cơ chế, framework structure giữ được.

XÁC SUẤT: THẤP — các nodes đều dựa trên decades of research.
  Nhưng KHÔNG phải zero. Neuroscience vẫn đang tiến hóa.
  → Mitigation: theo dõi replication status của từng node.
  → Xem: Research-References/ cho danh sách evidence per node.
```

### 9.2 Sai Kết Nối (Graph sai)

```
NẾU: Nodes đều đúng, nhưng CÁCH KẾT NỐI giữa chúng sai.

Đây là RỦI RO LỚN NHẤT — vì kết nối là contribution MỚI của HPD,
chưa có independent validation.

Ví dụ:
  - "Threshold KHÔNG quyết định Source choice"
    → Threshold vẫn tồn tại (neurochemistry đúng)
    → Nhưng: mối quan hệ threshold → external/internal KHÔNG như framework mô tả
    → Square Model mất predictive power ở trục Source
    → Hệ quả: cần tìm lại CÁI GÌ quyết định Source → framework restructure.

  - "3 channels KHÔNG independent như mô tả"
    → Novelty, Opioid, Oxytocin interact nhiều hơn framework assume
    → Predictions dựa trên "channel này dominates" sẽ kém chính xác
    → Nhưng: cluster behavior VẪN observable → có thể cần > 3 channels
       hoặc model interaction phức tạp hơn.

  - "PE Sensitivity 4 sub-mechanisms không co-vary"
    → §④ generalization scope KHÔNG correlate với §① amplitude
    → PE Sensitivity KHÔNG phải 1 construct — mà là 2-4 constructs riêng
    → Framework cần tách ra → phức tạp hơn nhưng chính xác hơn.

XÁC SUẤT: TRUNG BÌNH ĐẾN CAO.
  Đây là nơi framework CHẮC CHẮN sẽ cần sửa nhiều nhất.
  Câu hỏi không phải "có sai không" mà "sai BAO NHIÊU."
  → Mitigation: mỗi kết nối cần cross-domain prediction test.
    Nếu predict đúng cross-domain → kết nối có cơ sở.
    Nếu predict sai → kết nối cần xem lại.
```

### 9.3 Sai Ước Lượng Tham Số (Calibration sai)

```
NẾU: Cơ chế đúng, kết nối đúng, nhưng ước lượng mức độ sai.

Ví dụ:
  - "Cortisol inverted-U đúng hình dạng, nhưng peak ở chỗ khác"
    → Framework predict "moderate cortisol = optimal PFC"
    → Thực tế: "moderate" khác nhau PER PERSON hơn framework assume
    → Hệ quả: predictions đúng hướng, sai mức → cần individual calibration.

  - "Threshold calibration CHẬM hơn nhiều so với mô tả"
    → D2 receptor up-regulation cần 6 tháng, không phải vài tuần
    → Practical implications thay đổi (lời khuyên "dopamine fasting" cần timeline khác).

XÁC SUẤT: CAO — gần như chắc chắn sai ở mức chi tiết.
  Đây là normal cho mọi model — Newton đúng hướng, sai chi tiết trước Einstein.
  → Mitigation: luôn nói "hướng + pattern" chứ không nói "con số chính xác."
  → Framework nên tránh đưa ra con số cụ thể khi chưa có empirical data.
```

### 9.4 Tổng Hợp Failure Modes

```
LOẠI SAI:          XÁC SUẤT:     HỆ QUẢ:                    KHẮC PHỤC:
Node sai           Thấp          Sụp cục bộ, nặng            Đợi science resolve
Kết nối sai        TB-Cao        Restructure, framework sống  Test cross-domain
Calibration sai    Cao           Sai chi tiết, đúng hướng     Individual data

→ Framework CHẮC CHẮN sẽ sai ở một số chỗ.
  Câu hỏi hữu ích: "Sai ở loại nào? Kéo theo bao nhiêu?"
  Chứ không phải: "Framework đúng hay sai?" (binary = oversimplify).

→ THÁI ĐỘ ĐÚNG:
  Framework = bản đồ tốt nhất HIỆN TẠI để navigate.
  Bản đồ sẽ được update khi có data mới.
  Bản đồ sai > không có bản đồ — NẾU người dùng BIẾT bản đồ có thể sai.
  Bản đồ sai < không có bản đồ — NẾU người dùng TIN bản đồ là sự thật tuyệt đối.
  → Quay lại §6: rủi ro = oversimplification, không phải framework sai.
```
